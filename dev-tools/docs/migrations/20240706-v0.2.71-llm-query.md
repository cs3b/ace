## Overview of Changes

### Major Changes in v0.2.71

1. **Unified LLM Query Command**: Replaced provider-specific commands with single `llm-query` command using `provider:model` syntax
2. **XDG Cache Compliance**: Migrated cache storage to XDG Base Directory specification
3. **Enhanced Model Information**: Added context size and enhanced metadata across all providers
4. **Improved Cost Tracking**: Integrated with LiteLLM pricing database for accurate cost calculations
5. **Base Client Refactoring**: Reduced code duplication through shared client hierarchy

### Impact Assessment

- **High Impact**: Command syntax changes require script updates
- **Medium Impact**: Cache location changes with automatic migration
- **Low Impact**: Enhanced features with backward-compatible data structures

## Unified LLM Query Command

### Command Migration Table

| Old Command | New Unified Command | Notes |
|-------------|-------|--------|
| `llm-google-query` | `llm-query google:gemini-2.5-flash` | Uses latest default model |
| `llm-gemini-query` | `llm-query google:gemini-2.5-flash` | Same as above |
| `llm-anthropic-query` | `llm-query anthropic:claude-sonnet-4-20250514` | Uses latest Claude model |

#### Conditional Provider Selection

**Before v0.2.71:**
```bash
#!/bin/bash
if [[ "$PROVIDER" == "google" ]]; then
    llm-google-query "$PROMPT"
elif [[ "$PROVIDER" == "anthropic" ]]; then
    llm-anthropic-query "$PROMPT"
elif [[ "$PROVIDER" == "openai" ]]; then
    llm-openai-query "$PROMPT"
fi
```

**After v0.2.71:**
```bash
#!/bin/bash
case "$PROVIDER" in
    google) MODEL="google:gemini-2.5-flash" ;;
    anthropic) MODEL="anthropic:claude-sonnet-4-20250514" ;;
    openai) MODEL="openai:gpt-4o" ;;
    *) echo "Unknown provider: $PROVIDER"; exit 1 ;;
esac

llm-query "$MODEL" "$PROMPT"
```

## XDG-Compliant Cache Migration

### Cache Location Changes

**Old Location**: `~/.coding-agent-tools-cache`
**New Location**: `~/.cache/coding-agent-tools` (or `$XDG_CACHE_HOME/coding-agent-tools`)

### Directory Structure Migration

```
OLD: ~/.coding-agent-tools-cache/
├── google_models.yml
├── lmstudio_models.yml
└── other_provider_models.yml

NEW: $XDG_CACHE_HOME/coding-agent-tools/ or ~/.cache/coding-agent-tools/
├── models/
│   ├── google_models.yml
│   ├── lmstudio_models.yml
│   └── other_provider_models.yml
├── pricing/
│   └── litellm_pricing.json
├── usage/
│   └── usage_tracking.json
└── temp/
```

### Automatic Migration Process

The migration happens automatically on first run after upgrading:

1. **Detection**: Check if `~/.coding-agent-tools-cache` exists
2. **Create Target**: Ensure XDG cache directory exists with proper permissions
3. **Copy Files**: Copy all cache files preserving timestamps and data integrity
4. **Verify**: Confirm successful migration and data consistency
5. **Mark Complete**: Create `.migration_complete` marker file
6. **Preserve Original**: Keep old directory for safety (not automatically deleted)

### Environment Variables

The new cache location respects XDG environment variables:

```bash
# If XDG_CACHE_HOME is set
export XDG_CACHE_HOME="/custom/cache/path"
# Cache location: $XDG_CACHE_HOME/coding-agent-tools

# If XDG_CACHE_HOME is unset (default)
# Cache location: $HOME/.cache/coding-agent-tools

# Fallback if neither HOME nor XDG_CACHE_HOME available
# Cache location: .cache/coding-agent-tools (current directory)
```

### Manual Migration (If Needed)

If automatic migration fails:

```bash
# 1. Create new cache directory
mkdir -p ~/.cache/coding-agent-tools/models

# 2. Copy cache files
cp ~/.coding-agent-tools-cache/* ~/.cache/coding-agent-tools/models/

# 3. Create migration marker
echo "migrated_at: $(date -Iseconds)" > ~/.cache/coding-agent-tools/.migration_complete

# 4. Verify migration
ls -la ~/.cache/coding-agent-tools/models/
```

### Verification Commands

```bash
# Check current cache setup
ls -la ~/.cache/coding-agent-tools/

# Check for legacy cache (still exists after migration)
ls -la ~/.coding-agent-tools-cache/

# Verify cache subdirectories
ls -la ~/.cache/coding-agent-tools/models/
ls -la ~/.cache/coding-agent-tools/pricing/

# Trigger cache usage to see migration in action
```

## Base Client Hierarchy Refactoring

### Migration Strategy Overview

The base client hierarchy refactoring reduces code duplication across provider clients while maintaining full backward compatibility. This is an internal architecture improvement that should be transparent to users.

### Migration Order (Internal Implementation)

1. **GoogleClient** (Most Complex) - Completed
2. **AnthropicClient** (Moderate Complexity) - Completed
3. **OpenAI-Compatible Clients** (Similar Patterns) - Completed
   - OpenAI, Mistral, TogetherAI, and LMStudio

### Client-Specific Considerations

**GoogleClient Specifics**:
- Query parameter authentication preserved
- `candidates` response format handling maintained
- Model-specific URL building unchanged

**AnthropicClient Specifics**:
- `content` blocks array response format preserved
- System message handling unchanged
- Pagination logic maintained

**LMStudioClient Specifics**:
- No credentials requirement preserved
- Server availability checking maintained
- Local server timeout defaults unchanged

**OpenAI/Mistral/TogetherAI Specifics**:
- Standard OpenAI format compatibility maintained
- Bearer authentication preserved
- `choices[0].message.content` response pattern unchanged

### Validation Checklist (Completed)

- ✅ All existing tests pass without modification
- ✅ CLI commands work correctly
- ✅ API calls produce same responses
- ✅ Error handling maintains same behavior
- ✅ Performance characteristics unchanged

## Troubleshooting

### Common Migration Issues

#### Command Not Found Errors

**Issue**:
```bash
$ llm-google-query "test"
command not found: llm-google-query
```

**Solution**: Use the unified command:
```bash
$ llm-query google:gemini-2.5-flash "test"
```

#### Wrong Model Specification

**Issue**:
```bash
$ llm-query google:gpt-4o "test"
Error: Unknown model 'gpt-4o' for provider 'google'
```

**Solution**: Use correct provider:model combination:
```bash
$ llm-query openai:gpt-4o "test"
```

#### Environment Variable Issues

**Issue**:
```bash
$ llm-query google:gemini-2.5-flash "test"
Error: GOOGLE_API_KEY not found
```

**Solution**: Check environment variable names:
```bash
# Make sure you're using the correct variable name
export GOOGLE_API_KEY="your-key"  # Not GEMINI_API_KEY
```

### Cache Migration Issues

#### Permission Denied During Migration

**Issue**:
```
Error: Cache migration failed: Permission denied
```

**Solution**: Ensure write permissions to target directory:
```bash
# Check permissions
ls -la ~/.cache/
# Create directory with correct permissions if needed
mkdir -p ~/.cache/coding-agent-tools
chmod 700 ~/.cache/coding-agent-tools
```

#### Repeated Migration Messages

**Issue**:
```
INFO: Migrated cache from ~/.coding-agent-tools-cache to ~/.cache/coding-agent-tools
```

**Solution**: Check for missing migration marker:
```bash
# Verify marker exists
ls -la ~/.cache/coding-agent-tools/.migration_complete
# Create marker manually if missing
touch ~/.cache/coding-agent-tools/.migration_complete
```

#### Custom XDG_CACHE_HOME Not Recognized

**Issue**:
```
Cache still using ~/.cache instead of custom location
```

**Solution**: Verify environment variable is exported:
```bash
# Check current value
echo $XDG_CACHE_HOME
# Set and export if needed
export XDG_CACHE_HOME="/custom/cache/path"
```

### Performance Issues

#### Slow Cache Migration

**Issue**: Migration takes a long time with large cache

**Solution**: Migration happens once; subsequent operations are fast. Monitor with:
```bash
# Check migration progress
du -sh ~/.coding-agent-tools-cache
du -sh ~/.cache/coding-agent-tools
```

#### Increased Response Times

**Issue**: Queries seem slower after migration

**Solution**: Enhanced features add minimal overhead. Benefits include:
- Better caching reduces redundant API calls
- Cost tracking provides valuable insights
- Context size info helps model selection
