# Capture Raw Input at End of Idea File

## Intention

To ensure that the raw, unedited input provided by the user is preserved at the very end of the generated idea file for complete traceability and reference.

## Problem It Solves

**Observed Issues:**
- Without preserving the raw input, there's no direct reference to the original prompt or idea, making it difficult to understand the context of the generated output or to re-evaluate the original intent.
- In AI-assisted development, understanding the origin of an idea or a requirement is crucial for debugging, auditing, and iterative improvement.
- Currently, there's no explicit mechanism to append the raw user input to the structured output.

**Impact:**
- Loss of direct traceability from the structured idea document back to the original user prompt.
- Hinders the ability to understand the nuances and specific phrasing of the initial request.
- Makes it harder for future iterations or debugging to refer back to the exact input that led to the current state.
- Potential for misinterpretation of the original intent if only the processed output is available.

## Key Patterns from Reflections

- **Documentation-Driven Development**: The project emphasizes clear documentation for all aspects, including the process of idea generation and refinement. Preserving raw input aligns with this principle by documenting the origin of ideas.
- **Structured Output**: The system is designed to transform raw ideas into structured, actionable specifications. This new requirement adds a specific structured element (the raw input) to the output.
- **Traceability**: The project's meta-system architecture and emphasis on ADRs suggest a need for clear traceability throughout the development lifecycle, including the origin of requirements and ideas.

## Solution Direction

1. **Append Raw Input**: A dedicated section, perhaps named "Raw Input" or "Original Prompt," will be added to the very end of the generated idea file.
2. **Identify and Extract Raw Input**: The system needs to reliably capture the user's exact input before any processing or structuring begins.
3. **Format and Append**: The captured raw input will be appended to the structured output, potentially within a markdown code block or a clearly delineated section.
