# `docs-dev` - Development Guides & Workflows

The `docs-dev/` directory contains a collection of standardized development guides, workflow instructions, and templates designed to bring consistency and best practices to AI-assisted software development projects.

## What is `docs-dev`?

This directory provides the "how-to" for development. It includes:
-   **Guides (`guides/`)**: Best practices for coding, testing, documentation, version control, release processes, and more.
-   **Workflow Instructions (`workflow-instructions/`)**: Step-by-step procedures for common development tasks, designed to be followed by developers or AI agents (e.g., initializing project structure, creating user documentation, working on a task).
-   **Templates**: Reusable templates for documents, code snippets, or configurations.

The aim is to streamline development, improve collaboration (human-human and human-AI), and ensure a high standard of quality across projects.

## Contents Overview

-   `guides/`: Contains various development standards and best practices.
-   `workflow-instructions/`: Provides actionable, step-by-step instructions for development tasks.
-   Other directories may contain templates or supporting documentation for the development process.

By leveraging the resources in `docs-dev/`, teams can establish a robust and efficient development environment.
