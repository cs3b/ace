# Generate Project Blueprint Workflow Instruction

## Goal
Create or update the `docs-project/blueprint.md` file with a summary of the current project structure, key files, and links to core project documents. This provides essential context for the AI agent.

## Prerequisites
- Project structure should be relatively stable.
- Core documents (`what-do-we-build.md`, `architecture.md`) should exist and be reasonably up-to-date.

## Process Steps (Initial - Manual/Semi-Automated)

1.  **Identify Core Project Documents:** Verify `docs-project/what-do-we-build.md` and `docs-project/architecture.md` are present.
2.  **Capture Directory Structure:**
    *   Run `tree -L 2` (adjust depth `-L` as needed for project complexity) from the project root.
    *   Copy the relevant output into the `blueprint.md` file under the "Key Directories & Files" section within a code block.
3.  **Identify Key Project-Specific Files:** Manually list critical configuration files, entry points, version files, core modules, or test helpers under the "Key Project-Specific Files" section.
4.  **Update Links:** Ensure the Markdown links to `what-do-we-build.md` and `architecture.md` are correct within `blueprint.md`.
5.  **Save:** Save the updated `docs-project/blueprint.md` file.

## Input
- User confirmation that core documents are ready.
- Optional: User-specified depth for the `tree` command.

## Output / Success Criteria
- [x] `docs-project/blueprint.md` file is created or updated.
- [x] The file contains an accurate directory structure generated by `tree`.
- [x] The file lists key project-specific files.
- [x] The file contains correct links to `what-do-we-build.md` and `architecture.md`.

## Usage Example
Invoke this workflow instruction when:
- Significant structural changes have occurred in the project.
- Before starting a major planning phase (`docs-dev/workflow-instructions/lets-spec-from-prd.md` or `docs-dev/workflow-instructions/lets-spec-from-frd.md`).
- If the context loaded by `docs-dev/workflow-instructions/load-env.md` seems inaccurate regarding project structure.

Example Invocation:
> "Run the `generate-blueprint` workflow instruction to update the project blueprint."

## Future Enhancements
- A script could automate steps 2 and potentially 3 by scanning the project for standard files.
- Could be integrated into `load-env` to automatically refresh if changes are detected (potentially complex).
