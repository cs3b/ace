# Implement ace-prompt push for Contextual History Injection

## Problem
AI agents often require continuity across multiple LLM interactions, especially when transitioning from a research or analysis phase (e.g., using `ace-search` or `ace-review`) to a coding phase. Currently, retrieving the context of the most recent LLM conversation session from the cache (`.cache/ace-prompt/sessions/`) and manually injecting it into a new prompt is cumbersome and non-deterministic. This lack of automated context transfer hinders efficient multi-step workflows.

## Solution
Implement a new CLI command, `ace-prompt push`, which automatically identifies the most recent LLM interaction session managed by `ace-prompt` (or another specified gem's cache), extracts the relevant input and output, summarizes this history, and saves the resulting context into a designated file within the active prompt workspace. This file can then be easily included in subsequent prompt templates using standard ACE prompt inclusion mechanisms.

## Implementation Approach

1.  **Context Discovery (Molecule):** A new Molecule in `ace-prompt` will utilize `Ace::Core::Molecules::PromptCacheManager` (from `ace-support-core`) to locate the latest session directory within `.cache/ace-prompt/sessions/`.
2.  **Extraction and Summarization (Organism):** An Organism will read `system.prompt.md`, `user.prompt.md`, and any cached LLM response. If the combined context exceeds a configurable threshold, it will delegate to `ace-llm` to generate a concise summary of the session's outcome and key decisions.
3.  **Context Storage:** The summarized context will be written to a standardized location, such as `$(ace-prompt config --get workspace)/.context/recent-session.md`. This location is easily referenced by subsequent prompt templates.
4.  **CLI Integration:** The `ace-prompt push` command will orchestrate this process, offering options like `--source-gem` (to pull context from `ace-review` or `ace-docs` caches) and `--max-tokens` (for summarization control).

## Considerations
- **Integration with existing ace-* gems:** The `push` command must be able to read caches generated by other LLM-using gems (e.g., `ace-review`, `ace-docs`) by accepting a `--source-gem` parameter, ensuring cross-tool context flow.
- **Configuration cascade implications:** The target location for the context file should be configurable via the `ace-prompt` configuration cascade, defaulting to a predictable path relative to the current prompt workspace.
- **CLI interface design:** The output should be deterministic, confirming the source session ID and the path where the context was saved, enabling autonomous agents to verify the operation.

## Benefits
- **Improved Agent Efficiency:** Allows agents to seamlessly transition between tasks by automatically loading previous conversation history.
- **Standardized Context Flow:** Enforces a consistent pattern for managing conversational history across the ACE ecosystem.
- **Reduced LLM Redundancy:** Prevents agents from repeating context or decisions already made in a prior session, saving tokens and time.
