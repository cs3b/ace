# Improve Synthesis Report Readability in ace-review

## Problem
The synthesis reports generated by the LLM within `ace-review` are currently poorly formatted, leading to unreadable output in the terminal. This hinders rapid human comprehension of code review results and reduces the overall utility of the tool for developers.

## Solution
Implement a dedicated post-processing and presentation layer within `ace-review` to standardize the output format of LLM-generated synthesis reports. The goal is to ensure the reports use clear, structured Markdown (H2/H3 headings, lists) and leverage terminal capabilities (like colorization via `ace-support-core` utilities) for enhanced readability.

## Implementation Approach
1. **Target Gem:** `ace-review`.
2. **Architecture:** Introduce a new **Molecule** or **Organism** (e.g., `Ace::Review::Organisms::SynthesisPresenter`) responsible for taking the raw LLM response and transforming it into a structured, human-readable format.
3. **Standardization:** Enforce consistent section headers (e.g., 'Summary of Findings', 'Key Recommendations', 'Metrics') and use markdown tables or lists for structured data.
4. **CLI Integration:** Ensure the presentation logic is executed within the `ace-review review` command, respecting the standard CLI options inherited from `Ace::Core::CLI::Base` (especially `--quiet` and `--verbose`). If the output is piped, ensure the raw, deterministic LLM output (if structured, e.g., JSON) is still available via a specific output flag, while the enhanced presentation goes to standard error (stderr) or is the default human-facing output.

## Considerations
- **Deterministic Output:** Ensure the underlying data structure (if the LLM returns structured data) remains accessible for autonomous agents, even if the default terminal display is highly formatted.
- **LLM Agnostic:** The formatting layer must operate reliably regardless of the specific LLM provider used via `ace-llm`.
- **Configuration:** Allow configuration of the presentation style (e.g., minimal vs. detailed report) via the `ace-review` configuration namespace.

## Benefits
- Significantly improves developer experience by making review results instantly digestible.
- Reduces cognitive load when reviewing complex code changes.
- Provides a consistent UX across all `ace-review` operations.
