# Minimal Test Workflow

## Intention

To create a minimal, self-contained workflow that can be executed by an AI agent to verify basic system functionality.

## Problem It Solves

**Observed Issues:**
- AI agents struggle to execute basic commands to verify system health or functionality.
- Existing workflows are often too complex or rely on external context not readily available to an agent.
- Lack of a simple, repeatable test case for AI agent execution environments.

**Impact:**
- AI agents cannot reliably confirm their operational readiness.
- Debugging agent execution environments is challenging due to the absence of a baseline test.
- Difficulty in validating the core functionality of the toolkit from an agent's perspective.

## Key Patterns from Reflections

- **Workflow Self-Containment Principle (ADR-001)**: Workflows must be self-contained and executable without external dependencies, except for three standard context documents.
- **XML Template Embedding Architecture (ADR-002)**: Embedded templates should use XML format for structured content.
- **Consistent Path Standards (ADR-004)**: Paths must be relative to the project root.
- **Universal Document Embedding System (ADR-005)**: Supports embedding various document types within a unified `<documents>` container.
- **ATOM Architecture**: Components are structured into Atoms, Molecules, Organisms, and Ecosystems.
- **Multi-Repository Coordination**: The system relies on interconnected repositories (.ace/handbook, .ace/tools, .ace/taskflow, handbook-meta).
- **CLI Tool Patterns**: Over 25 executables with consistent interfaces are available.

## Solution Direction

1.  **Create a Self-Contained Workflow (`minimal-test.wf.md`)**:
    A new workflow file will be created in `.ace/handbook/workflow-instructions/` that adheres to ADR-001. It will contain all necessary instructions and examples directly within the file.
2.  **Embed Basic Command Execution**:
    The workflow will include steps to execute a simple, universally available CLI command (e.g., `echo "hello"` or `git --version`) and capture its output.
3.  **Define Success Criteria**:
    The workflow will specify clear, verifiable success criteria based on the output of the executed command.

## Critical Questions

**Before proceeding, we need to answer:**
1. What is the most reliable and universally available CLI command across all potential agent execution environments that can serve as a basic "hello world" test?
2. What is the expected output format for the chosen command, and how should success be validated against it?
3. Should this workflow embed any specific templates or context beyond the standard three project documents, even if minimal?

**Open Questions:**
- How will this workflow be discovered and invoked by AI agents if it's not part of a larger agent interaction flow?
- Should this workflow be integrated into any automated testing or CI processes for the toolkit itself?
- What is the minimal set of "Project Context Loading" files required for a workflow that only executes a simple CLI command?

## Assumptions to Validate

**We assume that:**
- An environment capable of running basic shell commands (like `echo` or `git`) will be available to the AI agent. - *Needs validation*
- The AI agent has a mechanism to execute arbitrary CLI commands and interpret their output. - *Needs validation*
- The presence of `git` is a reasonable assumption for a development toolkit environment. - *Needs validation*

## Expected Benefits

- Provides a simple, deterministic way for AI agents to verify their execution environment.
- Serves as a foundational test case for new agent setups or troubleshooting.
- Reduces complexity in debugging agent execution issues by providing a baseline.
- Demonstrates the self-contained nature of workflows as per ADR-001.

## Big Unknowns

**Technical Unknowns:**
- The exact set of CLI tools guaranteed to be present in all agent execution environments.
- The output formatting variations of basic commands across different OS/shell environments.

**User/Market Unknowns:**
- How frequently will AI agents need to perform this kind of "readiness" check?
- What specific errors or failures should this workflow aim to detect?

**Implementation Unknowns:**
- The exact structure and content of the `.wf.md` file, including required sections like `## Project Context Loading` and `## Process Steps`.
- How the `git --version` command's output should be validated (e.g., checking for a specific pattern or just presence).
