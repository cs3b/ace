# Standardize ace-taskflow Retrospective Naming using ace-timestamp Base36 IDs

## Problem
Retrospectives created by `ace-taskflow` currently use simple date-based prefixes or non-standard naming conventions (e.g., `.ace-taskflow/v.X.Y.Z/retro/`). This deviates from the project standard established in Task 149, which mandates the use of Base36 Compact IDs generated by the `ace-timestamp` gem for all managed entities (tasks, ideas, etc.) to ensure global uniqueness, sortability, and compactness.

## Solution
Update the file creation logic within `ace-taskflow` to utilize `ace-timestamp` for generating Base36 Compact IDs as prefixes for retrospective files and directories. This ensures consistency across all taskflow artifacts.

## Implementation Approach
1. **Dependency Update:** Add `ace-timestamp` as a runtime dependency in the `ace-taskflow.gemspec`.
2. **Logic Modification:** Identify the relevant Organism or Molecule within `ace-taskflow` responsible for handling the `retro create` command (likely in `lib/ace/taskflow/organisms/retro_manager.rb`).
3. **ID Generation:** Replace existing date formatting logic with a call to `Ace::Timestamp::Atoms::Base36IDGenerator.generate` to obtain the unique ID prefix.
4. **Audit:** Conduct a brief audit of other `ace-taskflow` file creation points (e.g., release creation, task creation) to ensure they are also using or are migrated to `ace-timestamp` for prefixing, completing the standardization effort.
5. **CLI Interface:** Ensure the `ace-taskflow retro create` command output clearly displays the new, standardized path using the Base36 ID.

## Considerations
- **Backward Compatibility:** While old retros will remain date-prefixed, newly created retros must strictly adhere to the Base36 ID format.
- **Configuration:** No new configuration should be required, as `ace-timestamp` handles generation internally.
- **Deterministic Output:** The CLI output must be deterministic and easily parseable by agents.

## Benefits
- **Consistency:** Achieves full standardization of file naming across all `ace-taskflow` artifacts (tasks, ideas, retros).
- **AI-Native:** Provides compact, unique, and sortable identifiers that improve agent efficiency and navigation via `ace-nav`.
