# Add linked issue metadata and auto sync hooks

## Objective

Teach `ace-task` to understand linked GitHub issue metadata and automatically invoke sync behavior whenever a linked task changes.

## Behavioral Specification

### User Experience

- A user can create a task with `--github-issue` and have the link stored in task metadata.
- A user can update, move, archive, or reparent a linked task without manually revisiting the GitHub issue.
- A user can run a dedicated sync command to backfill or repair GitHub issue tracking for linked tasks.

### Expected Behavior

1. `ace-task create` accepts GitHub issue IDs and stores them in machine-readable frontmatter.
2. `ace-task` supports a dedicated sync command for one task or all linked tasks.
3. Relevant task mutations automatically invoke GitHub issue sync for linked tasks only.
4. Sync triggers include create, title and status updates, path-changing moves, archive moves, and reparenting.
5. Tasks without linked GitHub issues remain unaffected.

### Interface Contract

- **CLI contract**
  ```bash
  ace-task create "Example" --github-issue 276
  ace-task create "Example" --github-issue 276 --github-issue 278
  ace-task github-sync t.abc
  ace-task github-sync --all
  ```
- **Frontmatter contract**
  ```yml
  github:
    issues: [276, 278]
  ```
- **Sync trigger contract**
  - create, update, move, reparent, and archive operations sync linked issues automatically
  - no sync occurs when `github.issues` is absent

### Success Criteria

- [ ] Users can add linked GitHub issue IDs through `ace-task` CLI and frontmatter.
- [ ] Linked task mutations trigger sync automatically.
- [ ] Manual repair and backfill are available through `ace-task github-sync`.
- [ ] Frontmatter validation and docs accept the new GitHub linkage metadata cleanly.

### Validation Questions

## Verification Plan

### Unit / Component Validation

- Verify `--github-issue` input is normalized into `github.issues`.
- Verify frontmatter validators accept valid issue linkage and reject malformed values.
- Verify auto-sync is triggered only for linked tasks.

### Integration / E2E Validation

- Create a linked task and verify that sync is invoked automatically.
- Update a linked task title or status and verify sync runs again.
- Move, archive, or reparent a linked task and verify sync runs with the new task path and ref context.
- Run `ace-task github-sync --all` and verify linked tasks are reconciled.

### Failure / Invalid Path Validation

- Invalid issue IDs produce clear task CLI validation errors.
- GitHub sync failure does not corrupt task metadata or prevent the task file update from being written.
