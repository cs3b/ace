# Expand ace-git with reusable GitHub issue sync primitives

## Objective

Expand `ace-git` so ACE has reusable GitHub issue integration primitives that task management can call safely without depending on `ace-review`.

## Behavioral Specification

### User Experience

- ACE packages that need GitHub issue operations can rely on `ace-git` as the shared integration layer.
- Linked task sync can create or update one sticky issue comment, ensure a stable tracking label, and close or reopen issues safely through reusable `ace-git` primitives.
- GitHub CLI failures remain actionable and consistent across ACE packages.

### Expected Behavior

1. `ace-git` owns the reusable `gh` execution wrapper for GitHub issue operations.
2. `ace-git` exposes primitives for:
   - finding and updating ACE sticky issue comments
   - ensuring and removing the `ace:tracked` label
   - closing and reopening issues
3. One GitHub issue gets at most one ACE-managed sticky comment, updated in place using a stable marker.
4. Multiple ACE tasks linked to the same issue are rendered in a single sticky comment.

### Interface Contract

- **ACE integration contract**
  - `ace-task` calls `ace-git` issue primitives rather than shelling out to `gh` directly
- **Sticky comment contract**
  - stable marker or lookup strategy for ACE ownership
  - one-line body format:
    ```md
    Tracked in ace-task: [8r4.t.ilo](https://github.com/cs3b/ace/blob/main/.ace-tasks/8r4.t.ilo-add-soft-github-issue-integration/8r4.t.ilo-add-soft-github-issue-integration-for-ace.s.md)
    ```
  - when multiple ACE tasks link to one issue, render one linked ref per line after the multi-task rule is resolved during implementation planning
- **Issue lifecycle contract**
  - close issue on ACE terminal state
  - reopen issue on ACE return to non-terminal state

### Success Criteria

- [ ] `ace-git` provides reusable GitHub issue operations needed by `ace-task`.
- [ ] Sticky comment ownership is deterministic and avoids duplicate ACE comments.
- [ ] Stable label management is supported through reusable primitives.
- [ ] Close and reopen issue behavior is available through shared ACE GitHub integration.

### Validation Questions

- Should the reusable `gh` executor migrate from `ace-review` into `ace-git`, or be reimplemented there with the same external behavior?
