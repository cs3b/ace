# Guard unsupported claude --max-tokens in ACE commit flow

## Objective

Make commit message generation resilient across Claude CLI versions by avoiding unsupported token-limit flag usage on incompatible versions.

## Behavioral Specification

### User Experience

- A user running commit automation with `ace-git-commit` should not see hard failures when using a Claude CLI that does not support `--max-tokens`.
- The flow should continue using supported options and still produce expected commit-message behavior when compatible.

### Expected Behavior

1. The commit flow detects or otherwise safely handles Claude CLI variants that reject `--max-tokens` and omits that flag on unsupported variants.
2. Supported CLI variants still use token limiting behavior when available.
3. Unsupported `--max-tokens` handling must not abort the provider fallback chain for commit message generation.
4. This behavior resolves issue [#278](https://github.com/cs3b/ace/issues/278) in the context of the fresh-start hardening effort.

### Interface Contract

- **Runtime contract**
  - Provider option selection must adapt based on available Claude CLI capabilities, with `--max-tokens` included only when the installed Claude CLI supports it.
- **Failure contract**
  - Unsupported flags must not abort the entire provider fallback chain.

### Success Criteria

- [ ] A fresh run with Claude CLI lacking `--max-tokens` does not fail on unsupported-flag errors.
- [ ] Supported flows remain functional for versions where the flag is accepted.
- [ ] Parent PR closes [#278](https://github.com/cs3b/ace/issues/278) when this child is complete.
