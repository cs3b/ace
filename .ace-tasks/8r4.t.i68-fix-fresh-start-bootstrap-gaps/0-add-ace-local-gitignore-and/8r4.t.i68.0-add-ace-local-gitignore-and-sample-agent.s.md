# Add .ace-local gitignore and sample agent guidance during init

## Objective

Ensure fresh project initialization includes durable local artifact hygiene and agent guidance defaults so downstream repos start with repository-safe behavior and predictable `ace-*` usage.

## Behavioral Specification

### User Experience

- A user runs project initialization and ends with canonical `.ace-local` handling already in place.
- New contributors get clear local agent instructions that describe how to use `ace-*` commands and where temporary outputs should live.

### Expected Behavior

1. New initialization sessions produce clear local artifact handling guidance.
2. `.ace-local/` is documented and treated as the canonical place for ACE temporary outputs.
3. Generated guidance explicitly discourages brittle post-processing of `ace-*` command output and recommends reading emitted files directly.
4. The issue [#279](https://github.com/cs3b/ace/issues/279) is resolved by this behavior.

### Interface Contract

- **User-facing contract**
  - A completed initialization flow includes onboarding guidance and local-hygiene expectation in new projects.
- **Failure behavior**
  - If initialization artifacts already exist, behavior should remain idempotent and additive (no destructive overwrite of user-owned content).

### Success Criteria

- [ ] A fresh project has guidance that makes `.ace-local/` usage clear.
- [ ] The guidance avoids `ace-*` output manipulation anti-patterns and points to direct file inspection.
- [ ] Initialization behavior is safe on repeated runs.
- [ ] Mapped issue is explicitly closed via PR when work is complete: [#279](https://github.com/cs3b/ace/issues/279).

## PR / Issue Tracking

  - `Fixes task 8r4.t.i68` (or parent-task equivalent)
  - `Closes #279`

## Validation Plan

- Verify first-run and rerun initialization experiences in a fresh project are consistent and non-disruptive.
- Verify generated agent guidance covers `ace-*` output-file-following behavior and `.ace-local` placement.
