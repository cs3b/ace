# Group .ace config files into single ace-config commit scope

## Behavioral Specification

### User Experience

- **Input**: List of changed files from git (includes `.ace/**` files)
- **Process**: Group files by scope, with all `.ace/**` files grouped as "ace-config"
- **Output**: Commit groups where `.ace/**` files are in single "ace-config" group

### Expected Behavior

When running `ace-git-commit` with changes to multiple `.ace/git/commit.yml` files across packages:

**Current (broken):**
```
15+ separate commits:
- chore(ace-bundle): Update commit config
- chore(ace-docs): Update commit config
- chore(ace-lint): Update commit config
...
```

**Expected (fixed):**
```
1 commit:
- chore(ace-config): Update git commit configurations
  Files: ace-bundle/.ace/git/commit.yml, ace-docs/.ace/git/commit.yml, ...
```

### Interface Contract

```ruby
# CommitGrouper already has this interface:
grouper = CommitGrouper.new
groups = grouper.group(files, project_root: project_root)

# Current behavior: each .ace file gets scope from its package
# New behavior: all .ace/** files get scope "ace-config"
```

### Success Criteria

- [ ] All `.ace/**` files across packages group into single "ace-config" scope
- [ ] Non-.ace files continue to use normal per-file resolution
- [ ] Existing ace-git-commit tests pass
- [ ] New tests verify .ace file grouping behavior
- [ ] Works for nested .ace folders (e.g., `pkg/sub/.ace/`)
- [ ] Root `.ace/` files also grouped into "ace-config"

### Validation Questions

- [x] Should root `.ace/` files also be in "ace-config"? → Yes, all .ace files
- [x] Config for scope name? → Hardcode "ace-config" for now

## Objective

Fix the original issue: `.ace/git/commit.yml` files in nested packages should commit together as "ace-config" instead of 15+ separate package commits.

## Scope of Work

## Technical Approach

### Implementation Strategy

Simple path matching in the `group` method — no external dependencies needed.

In `group()`, before calling `derive_scope_name`, check if the committed file itself is inside a `.ace/` directory using `ace_config_file?(file)`. If so, force scope to "ace-config".

The `ace_config_file?` helper checks:
- `file.include?("/.ace/")` — distributed config (e.g., `ace-bundle/.ace/git/commit.yml`)
- `file.start_with?(".ace/")` — root config (e.g., `.ace/git/commit.yml`)

This is ~5 lines of code total. No ProjectConfigScanner dependency needed.

## Test Cases

### Happy Path

1. **Multiple .ace files from different packages group together**
   - Input: `["ace-bundle/.ace/git/commit.yml", "ace-docs/.ace/git/commit.yml", "ace-lint/.ace/git/commit.yml"]`
   - Expected: Single group with scope_name "ace-config", 3 files

2. **Mixed .ace and non-.ace files separated correctly**
   - Input: `["ace-bundle/.ace/git/commit.yml", "ace-bundle/lib/foo.rb"]`
   - Expected: Two groups: "ace-config" (1 file), "ace-bundle" (1 file)

### Edge Cases

3. **Root .ace/ files included**
   - Input: `[".ace/git/commit.yml"]`
   - Expected: Single group "ace-config" with 1 file

4. **Deeply nested .ace/ folders**
   - Input: `["pkg/sub/.ace/config.yml"]`
   - Expected: File in "ace-config" group

5. **No .ace files**
   - Input: `["foo.rb", "bar.md"]`
   - Expected: Normal grouping, no "ace-config" group created

6. **.ace-defaults excluded**
   - Input: `[".ace-defaults/git/commit.yml"]`
   - Expected: NOT in "ace-config" group (uses normal resolution)
