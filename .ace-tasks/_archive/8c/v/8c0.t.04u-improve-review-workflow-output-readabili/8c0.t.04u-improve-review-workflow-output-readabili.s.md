# Improve Review Workflow Output Readability

## Behavioral Specification

### User Experience
- **Input**: User runs `/ace:review` or `/ace:review-pr`
- **Process**: Workflow executes, Claude presents results
- **Output**: Terminal-friendly formatted output that's easy to scan

### Expected Behavior

When Claude follows the review workflows, action items should be displayed in a vertical list format with color emoji priority indicators, rather than markdown tables that don't render properly in terminal.

### Interface Contract

```
=== ACTION ITEMS ===

🔴 #1 [Critical] Missing error handling
   File: lib/ace/review/cli.rb:45
   Source: gemini-2.5-flash
   Fix: Add rescue block for FileNotFound

🟡 #2 [High] Unused variable
   File: lib/ace/review/molecules/executor.rb:89
   Source: Developer Feedback
   Fix: Remove or use the `temp_path` variable
```

### Success Criteria

- [ ] Action items displayed as vertical list with one item per block
- [ ] Priority shown with color emoji (🔴/🟡/🟢/🔵) and text label
- [ ] Each field on its own indented line (File, Source, Fix)
- [ ] Easy to scan - priority indicators visible at left margin
- [ ] No markdown table syntax in terminal output

### Validation Questions

- [x] Which files need modification? → review.wf.md and review-pr.wf.md
- [x] What format should replace tables? → Vertical list with emoji indicators

## Objective

Replace markdown table format with terminal-friendly vertical list format in review workflow instructions, improving readability when Claude presents action items in the terminal.

## Scope of Work

### Files to Modify
- `ace-review/handbook/workflow-instructions/review.wf.md` - Step 6: Present Results
- `ace-review/handbook/workflow-instructions/review-pr.wf.md` - Step 6: Present Results

### Deliverables
- Updated workflow instructions with vertical list format
- Color/priority mapping: 🔴 Critical, 🟡 High, 🟢 Medium, 🔵 Low

## Out of Scope

- LLM synthesis prompts
- CLI presentation layer
- Individual model review prompts
