# Rename ace-test-e2e-runner Package and Binary for Naming Consistency

## Behavioral Specification

### User Experience

- **Input**: Developer runs `ace-test-e2e` or `ace-test-e2e-suite` commands; references `ace-test-runner-e2e` in Gemfile
- **Process**: All package references, binary names, config paths, and navigation protocols use the new consistent naming
- **Output**: Same CLI behavior as before — only names change, functionality is identical

### Expected Behavior

The test runner packages should follow a consistent naming pattern where the base tool comes first and the variant suffix comes last:

| | Package | Single | Suite | Shell |
|---|---|---|---|---|
| **Unit** | `ace-test-runner` | `ace-test` | `ace-test-suite` | — |
| **E2E** | `ace-test-runner-e2e` | `ace-test-e2e` | `ace-test-e2e-suite` | `ace-test-e2e-sh` |

Currently `ace-test-suite-e2e` breaks the pattern — the `-e2e` qualifier should be an infix (after `test`), not a trailing suffix (after `suite`). The package name `ace-test-e2e-runner` similarly breaks grouping with `ace-test-runner`.

After the rename:
- `ace-test-e2e` works exactly as before (no change)
- `ace-test-e2e-sh` works exactly as before (no change)
- `ace-test-e2e-suite` replaces `ace-test-suite-e2e` with identical behavior
- `gem 'ace-test-runner-e2e'` replaces `gem 'ace-test-e2e-runner'` in Gemfile
- All tests pass, all workflows resolve, all skills function

### Interface Contract

```bash
# These stay the same:
ace-test-e2e run TS-LINT-001a/TC-001-simple-file
ace-test-e2e-sh /path/to/sandbox/command.sh

# This is renamed:
ace-test-e2e-suite                    # was: ace-test-suite-e2e
ace-test-e2e-suite --package ace-lint # was: ace-test-suite-e2e --package ace-lint

# Gemfile reference:
gem 'ace-test-runner-e2e', path: 'ace-test-runner-e2e'  # was: ace-test-e2e-runner
```

**Error Handling:**
- Old binary name `ace-test-suite-e2e`: No backward compatibility shim (ADR-024 — no BC before 1.0)
- Old gem name `ace-test-e2e-runner`: No require path alias needed

**Edge Cases:**
- Cache directory `.cache/ace-test-e2e/` stays unchanged (historical data, no user-facing impact)
- Config namespace `.ace/e2e-runner/` stays unchanged (config cascade uses namespace, not gem name)
- Ruby module `Ace::Test::EndToEndRunner` stays unchanged (internal, no external API)

## Objective

Establish consistent naming across the test runner packages. The current `ace-test-e2e-runner` naming breaks the `ace-test-runner` + variant-suffix pattern used elsewhere in the monorepo, and the `ace-test-suite-e2e` binary name puts the `-e2e` qualifier in the wrong position relative to the `ace-test` / `ace-test-suite` base names.

## Scope of Work

- **Package rename**: Directory, gemspec, gem name, homepage URL
- **Binary rename**: `ace-test-suite-e2e` → `ace-test-e2e-suite` (exe/ file, bin/ wrapper, gemspec executables list)
- **Monorepo references**: Gemfile, Gemfile.lock, commit config, nav protocols, skills metadata, docs
- **No code changes**: Ruby module namespace, class names, and internal require paths stay the same
