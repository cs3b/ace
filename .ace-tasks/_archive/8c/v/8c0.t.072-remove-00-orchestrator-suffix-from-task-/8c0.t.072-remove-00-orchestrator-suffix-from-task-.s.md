# Remove .00 orchestrator suffix from task filenames

## Behavioral Specification

### User Experience

A developer or agent creates an orchestrator task. Currently the orchestrator file is named `NNN.00-orchestrator.s.md` — after this change, it becomes `NNN-orchestrator.s.md`. Subtask files (`.01`, `.02`, etc.) are unchanged. Single-task files are unchanged.

References like `v.0.9.0+task.121` continue to work. References using `.00` (e.g., `121.00`) become invalid and produce a clear error message.

### Expected Behavior

The `.00` suffix was introduced to sort orchestrators before subtasks in directory listings, but it creates confusion: `.00` looks like a subtask number, the naming is inconsistent with single tasks, and canonical IDs never included `.00`. Removing it simplifies the naming convention and aligns filenames with the canonical ID format.

**What changes:**
- Orchestrator files: `NNN.00-orchestrator.s.md` → `NNN-orchestrator.s.md`
- Subtask files: unchanged (`NNN.01-name.s.md`, `NNN.02-name.s.md`, etc.)
- Single task files: unchanged (`NNN-name.s.md`)
- Canonical IDs: unchanged (`v.0.9.0+task.NNN`)

**What does NOT change:**
- Subtask numbering (`.01`, `.02`, etc.)
- Canonical ID format
- Directory naming (`NNN-codename/`)
- Any file outside `.ace-taskflow/`

## Objective

Remove the `.00` suffix from orchestrator task filenames to simplify naming conventions, align filenames with canonical IDs, and eliminate the false impression that orchestrators are subtask zero.

### Design Decisions

- **`121.00` references**: Rejected with clear error. Clean break, no silent coercion to `121`. This avoids ambiguity and forces callers to update.
- **`is_orchestrator?` on references**: Removed from TaskReferenceParser. With the new naming, an orchestrator filename (`NNN-orchestrator.s.md`) looks like a single task (`NNN-name.s.md`) — you can only distinguish them by checking for subtask files in the directory. This is a filesystem concern (TaskLoader/TaskManager), not a reference parsing concern.
- **Archived `.00` files**: Migrated for consistency. Old and new code should see the same filenames.
- **Canonical IDs**: Unchanged. `v.0.9.0+task.121` never included `.00`, so no ID migration needed.
