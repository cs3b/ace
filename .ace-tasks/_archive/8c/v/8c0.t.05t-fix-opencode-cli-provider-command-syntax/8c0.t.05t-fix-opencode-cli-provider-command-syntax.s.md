# Fix OpenCode CLI provider command syntax

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-llm opencode:model "prompt text"` to query an LLM through OpenCode CLI
- **Process**: Command executes the OpenCode CLI with correct syntax and returns the LLM response
- **Output**: Text response from the LLM provider with metadata

### Expected Behavior

The ace-llm command with OpenCode provider should successfully execute the OpenCode CLI and return LLM responses. Currently, the provider uses incorrect command syntax (`opencode generate` with flags like `--prompt`, `--format`, `--temperature`, `--max-tokens`) that do not exist in the current OpenCode CLI interface.

The OpenCode CLI has the following actual interface:
- Command: `opencode run [message..]` (not `opencode generate`)
- Positional argument for prompt (not `--prompt` flag)
- Supported flags: `--model` (for model selection)
- Unsupported flags: `--format`, `--temperature`, `--max-tokens`, `--system`

### Interface Contract

```bash
# CLI Interface
ace-llm opencode:provider/model "prompt text"
# Example: ace-llm opencode:google/gemini-2.5-flash "write ruby code"

# Expected Success Response
[Text output from LLM]

# Expected Error Response
Error: OpenCode CLI failed: [error message from CLI]
```

**Error Handling:**
- OpenCode CLI not found: Clear error message with installation instructions
- OpenCode not authenticated: Error message directing user to run `opencode auth`
- Invalid model: Error indicating model not available
- Execution timeout: Error message indicating timeout duration

**Edge Cases:**
- Empty prompt: Should be handled by OpenCode CLI
- Very long prompt: Should pass through to CLI, may fail if exceeds limits
- Special characters in prompt: Should be properly escaped
- System prompt: Should be prepended to main prompt (no --system flag available)
