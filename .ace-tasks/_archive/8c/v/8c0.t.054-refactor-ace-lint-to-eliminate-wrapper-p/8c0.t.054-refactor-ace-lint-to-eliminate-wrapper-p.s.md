# Refactor ace-lint to eliminate wrapper pattern

## Behavioral Specification

### User Experience
- **Input**: Users run `ace-lint file.md` or `ace-lint "**/*.md" --fix`
- **Process**: Command executes correctly without wrapper layer
- **Output**: Linting results displayed

### Expected Behavior

ace-lint uses wrapper pattern passing through to `LintCommand`. Merge wrapper and logic into single command class.

**Commands to Refactor:**
```bash
ace-lint <pattern> [options]    # Lint files
```

### Success Criteria

- [x] **Lint command refactored**: Wrapper merged with LintCommand
- [x] **Help text works**: `ace-lint --help` shows proper documentation
- [x] **Tests pass**: All tests updated
- [x] **No functional changes**: Command works exactly as before

## Objective

Eliminate wrapper pattern in ace-lint by merging wrapper and business logic into single dry-cli command class.

## Scope of Work

- **User Experience Scope**: ace-lint CLI command
- **System Behavior Scope**: Command execution, linting logic

### Deliverables

#### Modify
- `ace-lint/lib/ace/lint/commands/lint.rb` - Merge business logic from LintCommand
- `ace-lint/lib/ace/lint.rb` - Remove require for deleted file
- `ace-lint/lib/ace/lint/cli.rb` - Remove require for deleted file

#### Delete
- `ace-lint/lib/ace/lint/commands/lint_command.rb`

## Technical Approach

### Architecture Pattern
- **Pattern**: Direct dry-cli command implementation (no wrapper indirection)
- **Rationale**: Simplifies code structure, reduces file count, follows ace-timestamp pattern
- **Integration**: CLI registry (`cli.rb`) already registers `Commands::Lint`, no changes needed to registration

### Implementation Strategy

The current implementation has two layers:
1. `lint.rb` - dry-cli command wrapper that calls `LintCommand.execute(files, clean_options)`
2. `lint_command.rb` - Static class method business logic

**Target architecture:**
1. `lint.rb` - dry-cli command with all business logic inline

Key code migrations:
- Move `execute`, `expand_file_paths`, `prepare_options` methods from `LintCommand` into `Lint` class
- Convert class methods to private instance methods in dry-cli `call` context
- Remove `require_relative "lint_command"` statement from lint.rb
- Remove `require_relative 'lint/commands/lint_command'` from ace/lint.rb
- Remove `require_relative "commands/lint_command"` from cli.rb

### Risk Assessment

#### Technical Risks
- **Risk:** Breaking command behavior during refactor
  - **Probability:** Low
  - **Impact:** High
  - **Mitigation:** Existing tests verify functionality; run tests after each change
  - **Rollback:** Git revert if issues found

#### Integration Risks
- **Risk:** Missing require statements after file deletion
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Grep for all references to `lint_command` before deletion
  - **Monitoring:** `ace-test` will fail if requires are broken
