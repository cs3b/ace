# Implement section-based content organization in ace-context

Enhance ace-context to support structured section-based content organization that allows embedding content into named sections (focus, style, diff, etc.) with XML-style section tags while maintaining full backward compatibility with existing frontmatter format.

## Behavioral Specification

### User Experience

**Input**: User wants to organize context content into structured sections like focus files, style guidelines, and diffs
**Process**: ace-context processes section definitions in frontmatter and generates organized output with XML-style section tags
**Output**: Structured document with named sections containing properly embedded content

### Expected Behavior

- Users can define sections in ace-context frontmatter with `context.sections` key
- Each section has a title, content type, priority, and content configuration
- Output is organized with `## SectionTitle` headers and `<sectionname>` XML tags
- Legacy configurations without sections continue to work unchanged
- Content can be organized by semantic meaning (focus, style, diff, context, etc.)

### Interface Contract

**Frontmatter Input Format**:
```yaml
---
context:
  sections:
    focus:
      title: "Files Under Review"
      content_type: "files"
      priority: 1
      files:
        - path: "src/main.js"
        - path: "docs/api.md"
    style:
      title: "Style Guidelines"
      content_type: "files"
      priority: 2
      files:
        - path: ".eslintrc.js"
        - path: "style-guide.md"
    diff:
      title: "Recent Changes"
      content_type: "diffs"
      priority: 3
      ranges:
        - "origin/main...HEAD"
  # Legacy support
  files:
    - README.md
  commands:
    - git status
---
```

**Expected Output Format**:
```markdown
## Files Under Review
<focus>
  <file path="src/main.js" language="javascript">
    // File content here
  </file>
  <file path="docs/api.md" language="markdown">
    # API Documentation
    <!-- Content here -->
  </file>
</focus>

## Style Guidelines
<style>
  <file path=".eslintrc.js">
    // ESLint config
  </file>
  <file path="style-guide.md">
    <!-- Style guidelines -->
  </file>
</style>

## Recent Changes
<diff>
  <output command="git diff origin/main...HEAD">
    <!-- Diff content -->
  </output>
</diff>

## Attachments
<attachments>
  <!-- Legacy content goes here when no section defined -->
  <!-- README.md content -->
  <!-- git status output -->
</attachments>
```
