# Core Feedback System

## Overview

Implement minimal feedback extraction from review reports using LLM synthesis. This is the right-sized scope after learning from overengineered original subtasks (227.01-227.04).

## Behavioral Specification

### User Experience
- **Input**: Review completes with synthesis report
- **Process**: LLM synthesizes findings from review reports into structured feedback items
- **Output**: YAML feedback files in session directory with unique IDs

### Expected Behavior

The feedback system extracts actionable findings from review reports:

1. **FeedbackItem model** - Immutable struct with id, title, files, priority, status
2. **FeedbackSynthesizer molecule** - Single LLM call to extract findings from multiple reports
3. **FeedbackFileWriter/Reader** - Simple YAML file I/O with atomic writes
4. **FeedbackManager organism** - Basic lifecycle (create, list, update status)
5. **Session-scoped storage** - All feedback stored in `{session}/feedback/`

### Interface Contract

```ruby
# FeedbackItem - immutable struct
FeedbackItem = Data.define(:id, :title, :files, :priority, :status, :finding, :created_at)

# FeedbackSynthesizer - single LLM call
class FeedbackSynthesizer
  def synthesize(reports, options: {})
    # => Array<FeedbackItem>
  end
end

# FeedbackManager - basic lifecycle
class FeedbackManager
  def create(feedback_items)
  def list(status: nil, priority: nil)
  def update_status(id, status:, note: nil)
end
```

### What Was NOT Needed (Lessons Learned)

These were originally planned but removed as overengineered:

- **FeedbackDeduplicator atom** (removed v0.36.6) - LLM handles deduplication in synthesis prompt
- **FeedbackExtractor molecule** (removed v0.36.6) - replaced by simpler FeedbackSynthesizer
- **FeedbackContextResolver molecule** (removed v0.36.8) - ~546 lines removed, session scope is sufficient
- **extraction_model config** (removed v0.36.9) - single `synthesis_model` is enough
- **Task-based path resolution** (removed v0.36.8) - session-scoped storage is sufficient
- **Complex file locking with retries** - simple flock is enough

### Success Criteria

## Out of Scope (Documented Learnings)

- Algorithmic deduplication (Levenshtein) - LLM handles this
- Task-based path resolution - session scope is sufficient
- Separate extraction_model config - single synthesis_model
- Complex file locking with retries - simple flock is enough
- FeedbackContextResolver - unnecessary complexity
