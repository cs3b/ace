# CLI Commands for Feedback Management (Simplified)

## Overview

Add CLI commands for feedback workflow. This is the right-sized scope after learning from overengineered original subtask (227.06).

## Behavioral Specification

### User Experience
- **Input**: CLI commands for managing feedback items
- **Process**: System queries/updates feedback via FeedbackManager
- **Output**: Formatted terminal output or status messages

### Expected Behavior

Simple CLI commands for feedback management:

```bash
# List feedback items
ace-review feedback list [--status STATUS] [--priority PRIORITY]

# Show single item details
ace-review feedback show <id>

# Mark as verified
ace-review feedback verify <id>

# Mark as skipped with reason
ace-review feedback skip <id> [--reason "..."]

# Mark as resolved
ace-review feedback resolve <id>
```

### What Was NOT Needed (Lessons Learned)

These were originally planned but removed as overengineered:

- **Partial ID matching** - Full ID is fine, avoids ambiguity edge cases
- **Task context auto-detection from branch** - Explicit `--session` flag is clearer
- **`--task` option on all commands** - Session-scoped only

### Interface Contract

```ruby
# Simple command structure
class FeedbackListCommand
  def execute(status: nil, priority: nil)
end

class FeedbackShowCommand
  def execute(id)
end

class FeedbackVerifyCommand
  def execute(id)
end

class FeedbackSkipCommand
  def execute(id, reason: nil)
end

class FeedbackResolveCommand
  def execute(id)
end
```

### Success Criteria

- [x] All commands work with `--session` flag
- [x] Default to latest session when no flag provided
- [x] Proper exit codes and error messages
- [x] Help text for all commands
