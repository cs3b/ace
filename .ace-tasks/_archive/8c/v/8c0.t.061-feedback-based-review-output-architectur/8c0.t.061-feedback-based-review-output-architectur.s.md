# Feedback-Based Review Output Architecture

## Overview

Replace the monolithic `synthesis-report.md` with a structured feedback file system. Each finding from reviews becomes a tracked feedback item with its own lifecycle, enabling concurrent verification by multiple agents.

## Final Architecture

```
Session Directory:
{session}/
├── feedback/
│   ├── {id}-{slug}.yml          # Active feedback items
│   └── _archived/               # Resolved/skipped items
└── reviews/
    ├── {id}-{model}-review.md   # Individual model reviews
    └── synthesis.md             # Optional synthesis report
```

## Retrospective: Lessons Learned

### The Real Bug

The entire task was prompted by feedback extraction failures. After 8 subtasks and extensive architecture work, **the root cause was a ~30 line JSON parsing bug** (fixed in v0.36.10):

```ruby
# Bug: Parser expected JSON at start of response
# Fix: Handle text before code fence (Claude Opus pattern)
```

### What Was Overengineered

| Component | Lines | Why Unnecessary |
|-----------|-------|-----------------|
| `FeedbackDeduplicator` | ~80 | LLM handles deduplication in synthesis prompt |
| `FeedbackExtractor` | ~150 | Single FeedbackSynthesizer call is enough |
| `FeedbackContextResolver` | ~546 | Task-based paths unnecessary; session scope sufficient |
| `extraction_model` config | ~50 | Single `synthesis_model` is enough |
| Task-based path resolution | ~200 | Session-scoped storage covers all use cases |

**Total removed: ~1000+ lines** (v0.36.6 through v0.36.9)

### What Worked

1. **FeedbackItem model** - Simple immutable struct
2. **FeedbackSynthesizer** - Single LLM call with good prompt
3. **Session-scoped storage** - Simple, sufficient for all use cases
4. **CLI commands** - Basic CRUD operations

### Key Takeaways

1. **Diagnose before designing** - The JSON parsing bug should have been found first
2. **Start minimal** - Add complexity only when proven necessary
3. **Session scope is enough** - Task-based paths were YAGNI
4. **LLM handles deduplication** - Algorithmic deduplication unnecessary
5. **Simple file I/O** - No need for complex locking strategies

The feedback system now works correctly:

```bash
# Run review with feedback extraction
ace-review --preset code-pr --auto-execute
ls .cache/ace-review/sessions/latest/feedback/

# Manage feedback
ace-review feedback list
ace-review feedback show {id}
ace-review feedback verify {id}
ace-review feedback resolve {id}
```
