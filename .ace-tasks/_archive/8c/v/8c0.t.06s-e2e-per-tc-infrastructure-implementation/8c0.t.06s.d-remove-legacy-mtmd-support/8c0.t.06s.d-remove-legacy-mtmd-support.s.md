## Behavioral Specification

**Input:**
- All 43 `.mt.md` files have been migrated to `TS-*/scenario.yml` format (task 262 complete)
- All orchestration and execution uses the new format (subtasks 01-03 complete)

**Process:**
- Remove `ScenarioParser` molecule (replaced by `ScenarioLoader`)
- Remove `.mt.md` file discovery from `TestDiscoverer` (if dual-mode was maintained)
- Update `config.yml` patterns: `MT-` prefix references â†’ `TS-` prefix

**Output:**
- Clean codebase with only the new `TS-*/scenario.yml` format supported

5. **Clean tests**: Remove test fixtures for old format, update assertions

## Implementation Plan

### 1. Technical Approach

**Removal Strategy: Clean Removal (Not Deprecation)**

Per ADR-024 (no backward compatibility before 1.0.0), all legacy `.mt.md` code can be cleanly deleted. There is no need for deprecation warnings, compatibility shims, or phased removal. By the time this task executes:

1. All 261.01-03 subtasks have added the new TS-format infrastructure with dual-mode support
2. All 262.* migration tasks have converted every `.mt.md` file to `TS-*/scenario.yml`
3. No `.mt.md` files exist anywhere in the monorepo

The dual-mode bridges added in 261.01-03 specifically designed for this cleanup:

- **261.01** added `scenario_format` method to `TestScenario` (`:mt` vs `:ts`), TS-format `short_id` pattern, and made `ScenarioLoader` coexist alongside `ScenarioParser`
- **261.02** added `execute_tc` alongside `execute`, `build_tc` alongside `build`, `parse_tc` alongside `parse` -- all new methods, old methods left unchanged
- **261.03** added `find_scenarios` alongside `find_tests`, dual-mode routing in orchestrators (`scenario.scenario_format == :mt` vs `:ts`), and TS-format display alongside MT-format display

This task removes the `:mt` code path from each of these dual-mode components.

**Verification Approach:**

After each file modification, run targeted grep commands to confirm no stale references remain. After all modifications, run a comprehensive suite-wide grep to verify zero remaining `.mt.md`, `MT-`, or `ScenarioParser` references in production code. CHANGELOG.md and README.md legacy references are acceptable (historical documentation).
