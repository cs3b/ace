# E2E Per-TC Infrastructure Implementation

## Overview

Implement the per-test-case independence architecture for ace-test-e2e-runner. This replaces the current monolithic `.mt.md` format (where setup, fixtures, and test cases are embedded in a single markdown file) with a structured directory-based format where each test case is an independent file with deterministic sandbox setup.

### Motivation

The current architecture sends entire `.mt.md` files (including setup scripts as heredocs) to the LLM for execution. This causes:
- **Slow setup** (30-60s per scenario via LLM interpretation)
- **Non-deterministic environments** (LLM may interpret heredocs differently)
- **No IDE support** for test fixture files (they're strings inside strings)
- **No per-TC independence** (all TCs share a single execution context)

The new architecture separates concerns: deterministic Ruby-driven setup, real fixture files on disk, and independent test case execution.
