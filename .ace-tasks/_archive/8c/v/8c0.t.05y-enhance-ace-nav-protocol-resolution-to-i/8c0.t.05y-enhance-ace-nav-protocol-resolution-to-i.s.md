# Enhance ace-nav Protocol Resolution to Infer File Extensions

## Description

The `ace-nav` tool currently requires the full file extension to be explicitly provided in protocol URIs (e.g., `guide://markdown-style.g` instead of `guide://markdown-style`). This strict matching causes resource discovery failures when agents or developers use the intuitive base name of a handbook item.

This task implements DWIM (Do What I Mean) extension inference - if exact path lookup fails, try common extensions in priority order. This aligns with the DX/AX Dual Optimization principle by making the tool more intuitive and predictable.

## Behavioral Specification

### Input
Protocol URI without full extension:
- `guide://markdown-style` (omitting `.g.md` suffix)
- `wfi://setup` (omitting `.wf.md` suffix)

### Process
1. Attempt exact match with protocol-specific extensions (current behavior)
2. If no match found, infer extensions in priority order:
   - Protocol-specific shorthand (e.g., `.g`, `.wf`, `.ag`)
   - Full protocol extension (e.g., `.g.md`, `.wf.md`, `.ag.md`)
   - Generic markdown (`.md`)
   - Bare filename
3. Return first match found

### Output
- Resolved file path (full path to matched resource)
- Or clear "Resource not found" error if truly not found

## Interface Contract

```bash
# Works today (exact match with partial extension)
ace-nav guide://markdown-style.g
# → /path/to/markdown-style.g.md

# Should also work (inference)
ace-nav guide://markdown-style
# → /path/to/markdown-style.g.md (inferred)

# Still fails if truly not found
ace-nav guide://nonexistent
# → Resource not found: guide://nonexistent
```

### Extension Priority Order (per protocol)

For `guide://` protocol:
1. Try exact: `**/markdown-style` (with protocol extensions `.g.md`, `.guide.md`)
2. Infer: `.g.md`, `.guide.md`, `.md`, then bare filename

For `wfi://` protocol:
1. Try exact: `**/setup` (with protocol extensions `.wf.md`, `.wfi.md`, `.workflow.md`)
2. Infer: `.wf.md`, `.wfi.md`, `.workflow.md`, `.md`, then bare filename

### Configuration Schema

```yaml
# .ace-defaults/nav/config.yml
extension_inference:
  enabled: true
  fallback_order:
    - protocol_shorthand  # e.g., .g, .wf
    - protocol_full       # e.g., .g.md, .wf.md
    - generic_markdown    # .md
    - bare               # no extension
```

### Key Implementation Points

1. **Determinism**: Extension checking order must be deterministic and configurable
2. **No Breaking Changes**: Explicit extensions must continue to work exactly as before
3. **Clear Output**: CLI returns full resolved path, even when inference was used
4. **Config Cascade**: Settings customizable via standard `.ace/nav/config.yml` cascade (ADR-022)
