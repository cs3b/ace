## Original ConfigSummary Specification

### User Experience
- **Input**: Users execute any `ace-*` CLI command with or without explicit parameters
- **Process**: Command starts with a concise 1-3 line configuration summary displayed to stderr
- **Output**: Users receive immediate feedback on effective configuration before command results

### Interface Contract

```bash
# CLI Interface - Configuration Summary Output
ace-review --pr 123
# Output to stderr (before main command execution):
# Config: preset=pr model=claude-sonnet-4.5 pr=123 format=markdown

ace-test test/file_test.rb --quiet
# No config summary (suppressed by --quiet flag)
```

### Summary Format
- Single line, key=value pairs, space-separated
- Machine-readable and human-friendly
- Output to stderr to avoid interfering with stdout
- Excludes sensitive data (tokens, credentials)

## Rollback Strategy

If a Thor migration breaks a package after release:

### Immediate Mitigation
1. **Revert commit**: `git revert <commit>` the migration commit
2. **Re-release patch**: `/ace:release <package> patch` with reverted code
3. **Document issue**: Create follow-up task for proper fix

### Prevention
- Each subtask runs full package test suite before release
- Manual verification of all commands before release
- Backward compatibility table in each subtask must be verified

### Recovery Procedure
```bash
# 1. Identify breaking commit
git log --oneline ace-<package>/

# 2. Revert the migration
git revert <migration-commit>

# 3. Run tests to confirm fix
ace-test ace-<package>

# 4. Release patch
/ace:release ace-<package> patch
```
