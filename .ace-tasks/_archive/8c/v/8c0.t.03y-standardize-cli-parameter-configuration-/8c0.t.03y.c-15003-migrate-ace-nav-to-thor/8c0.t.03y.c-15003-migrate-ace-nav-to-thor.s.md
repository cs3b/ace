# 150.03: Migrate ace-nav to Thor

## Objective

Convert ace-nav from OptionParser CLI to Thor following ADR-018 pattern.

## Implementation

### Thor CLI

```ruby
# lib/ace/nav/cli.rb
require "thor"

module Ace
  module Nav
    class CLI < Thor
      desc "resolve URI", "Resolve resource path or content"
      option :path, type: :boolean, desc: "Display resource path"
      option :content, type: :boolean, desc: "Display resource content"
      option :verbose, type: :boolean, desc: "Detailed information"
      option :quiet, type: :boolean, aliases: "-q", desc: "Suppress config summary"
      def resolve(uri)
        # Handle magic patterns (wildcards → list)
        if uri.include?("*") || uri.include?("?") || uri.match?(/\/$/) || uri.match?(/^\w+:\/\/$/)
          invoke :list, [uri]
          return
        end

        require_relative "commands/resolve_command"
        Commands::ResolveCommand.new(uri, options).execute
      end
      default_task :resolve

      desc "list PATTERN", "List matching resources"
      option :tree, type: :boolean, desc: "Tree format display"
      option :verbose, type: :boolean, desc: "Detailed information"
      option :quiet, type: :boolean, aliases: "-q"
      def list(pattern)
        require_relative "commands/list_command"
        Commands::ListCommand.new(pattern, options).execute
      end

      desc "create URI [TARGET]", "Create resource from template"
      option :verbose, type: :boolean
      option :quiet, type: :boolean, aliases: "-q"
      def create(uri, target = nil)
        require_relative "commands/create_command"
        Commands::CreateCommand.new(uri, target, options).execute
      end

      desc "sources", "Show available sources"
      option :verbose, type: :boolean, desc: "JSON output"
      def sources
        require_relative "commands/sources_command"
        Commands::SourcesCommand.new(options).execute
      end
      # Backward compatibility: --sources flag still works
      map "--sources" => :sources

      # Backward compatibility for --create URI flag
      # The current OptionParser accepts: --create [PATH] where PATH is optional
      # Thor approach: treat --create as taking a required argument (the URI)
      # Usage: ace-nav --create wfi://name → invokes create(wfi://name)
      #
      # NOTE: Thor's map directive passes remaining ARGV after the flag as method args
      # So `ace-nav --create wfi://foo` calls create_from_flag with ARGV=["wfi://foo"]
      desc "create_from_flag URI", "Create resource (backward compat)", hide: true
      def create_from_flag(uri = nil)
        if uri && uri != ""
          invoke :create, [uri]
        else
          puts "Usage: ace-nav create URI [TARGET]"
          puts "       ace-nav --create URI"
        end
      end
      map "--create" => :create_from_flag

      map "--version" => :version  # -v reserved for verbose per usage/ux.md
      desc "version", "Show version"
      def version
        puts "ace-nav #{VERSION}"
      end
    end
  end
end
```

## Magic Pattern Handling

Current CLI auto-detects list mode for:
- `protocol://` (empty path)
- Patterns with `*` or `?`
- Patterns ending with `/`

Thor version handles this in resolve command by invoking list.

## Backward Compatibility

| Original | Thor Migration | Status |
|----------|---------------|--------|
| `ace-nav wfi://setup` | Same (default resolve) | Preserved |
| `ace-nav wfi://*` | Same (auto list) | Preserved |
| `ace-nav --sources` | Same (via map) | Preserved |
| `ace-nav --create URI` | Same (via map) | Preserved |
| `ace-nav sources` | New explicit command | Added |
| `ace-nav create URI` | New explicit command | Added |

**Note**: Both old flag syntax and new subcommand syntax work via Thor `map`.
