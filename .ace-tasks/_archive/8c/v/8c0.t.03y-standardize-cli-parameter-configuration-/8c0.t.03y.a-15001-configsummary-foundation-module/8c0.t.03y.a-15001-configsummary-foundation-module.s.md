# 150.01: ConfigSummary Foundation Module

## Objective

Create shared ConfigSummary module in ace-support-core that all CLI tools can use to display effective configuration state to stderr, showing only values that differ from defaults.

## Deliverables

### Create

1. `ace-support-core/lib/ace/core/atoms/config_summary.rb`
   - `ConfigSummary.display(command:, config:, defaults:, options:, quiet:, summary_keys:)`
   - Config-diffing: only show values that differ from defaults
   - Allowlist support via `summary_keys` parameter
   - Sensitive key filtering (token, password, secret, credential, api_key)
   - Key=value format output to **stderr only**
   - Deterministic ordering (sorted keys)
   - Nested key flattening (dot notation)

2. `ace-support-core/test/atoms/config_summary_test.rb`
   - Test display method outputs to stderr
   - Test config-diffing (only non-default values shown)
   - Test allowlist filtering (summary_keys)
   - Test sensitive filtering
   - Test nested key flattening
   - Test quiet suppression

## Implementation

```ruby
# lib/ace/core/atoms/config_summary.rb
module Ace
  module Core
    module Atoms
      class ConfigSummary
        # Match keys ending with sensitive words (e.g., api_key, auth_token)
        # but not containing them as substrings (e.g., max_tokens, keyboard_layout)
        SENSITIVE_REGEX = /(_|^)(token|password|secret|credential|key|api_key)$/i

        # Display configuration summary to stderr
        #
        # @param command [String] Command name for context
        # @param config [Hash] Effective configuration (merged result)
        # @param defaults [Hash] Default configuration to diff against
        # @param options [Hash] CLI options (Thor options hash)
        # @param quiet [Boolean] Suppress output if true
        # @param summary_keys [Array<String>, nil] Allowlist of keys to include (nil = all non-sensitive)
        def self.display(command:, config: {}, defaults: {}, options: {}, quiet: false, summary_keys: nil)
          return if quiet

          summary = new(command, config, defaults, options, summary_keys).build
          $stderr.puts "Config: #{summary}" unless summary.empty?
        end

        def initialize(command, config, defaults, options, summary_keys)
          @command = command
          @config = flatten_hash(config || {})
          @defaults = flatten_hash(defaults || {})
          @options = options || {}
          @summary_keys = summary_keys
        end

        def build
          pairs = []

          # 1. Add CLI options that were explicitly set (non-nil, non-false, non-default)
          @options.each do |key, value|
            next if value.nil? || value == false
            next if sensitive_key?(key.to_s)
            next if @summary_keys && !@summary_keys.include?(key.to_s)
            pairs << format_pair(key, value)
          end

          # 2. Add config values that differ from defaults (if not already in options)
          @config.each do |key, value|
            # Check both symbol and string keys for indifferent access
            next if @options.key?(key.to_sym) || @options.key?(key.to_s) # Already shown from options
            next if @defaults[key] == value   # Same as default, skip
            next if sensitive_key?(key)
            next if @summary_keys && !@summary_keys.include?(key)
            pairs << format_pair(key, value)
          end

          pairs.sort.join(" ")
        end

        private

        # Flatten nested hash to dot notation
        # { llm: { provider: "google" } } → { "llm.provider" => "google" }
        def flatten_hash(hash, prefix = nil)
          hash.each_with_object({}) do |(key, value), result|
            full_key = prefix ? "#{prefix}.#{key}" : key.to_s
            if value.is_a?(Hash)
              result.merge!(flatten_hash(value, full_key))
            else
              result[full_key] = value
            end
          end
        end

        def format_pair(key, value)
          value_str = case value
                      when true then "true"
                      when Array then value.join(",")
                      else value.to_s
                      end
          "#{key}=#{value_str}"
        end

        def sensitive_key?(key)
          key.to_s.match?(SENSITIVE_REGEX)
        end
      end
    end
  end
end
```

## Integration Examples

### Basic Usage in Thor Command

```ruby
# In a Thor CLI command method
def review
  # Display config summary before command execution
  Ace::Core::Atoms::ConfigSummary.display(
    command: "review",
    config: Ace::Review.config,           # Merged effective config
    defaults: Ace::Review.default_config, # Gem defaults
    options: options,                      # Thor options hash
    quiet: options[:quiet],
    summary_keys: %w[preset model format]  # Optional allowlist
  )

  require_relative "commands/review_command"
  Commands::ReviewCommand.new(options).execute
end
```

### Using cli_defaults from Config

```yaml
# .ace-defaults/review/config.yml
cli_defaults:
  quiet: false
  summary_keys:
    - preset
    - model
    - format
    - pr
```

```ruby
# Load summary_keys from config
cli_defaults = Ace::Review.config.dig("cli_defaults") || {}
Ace::Core::Atoms::ConfigSummary.display(
  command: "review",
  config: Ace::Review.config,
  defaults: Ace::Review.default_config,
  options: options,
  quiet: options[:quiet] || cli_defaults["quiet"],
  summary_keys: cli_defaults["summary_keys"]
)
```

### Output Examples

```bash
# Full options shown (no allowlist)
$ ace-review --preset pr --model gflash
Config: model=gflash preset=pr

# With allowlist (only summary_keys shown)
$ ace-review --preset pr --verbose --model gflash
Config: model=gflash preset=pr
# (verbose not in summary_keys, omitted)

# Quiet mode
$ ace-review --preset pr --quiet
# (no config output)

# Nested config diffing
$ ace-llm query --model gflash
Config: llm.model=gflash llm.provider=google
# (only values differing from defaults)
```
