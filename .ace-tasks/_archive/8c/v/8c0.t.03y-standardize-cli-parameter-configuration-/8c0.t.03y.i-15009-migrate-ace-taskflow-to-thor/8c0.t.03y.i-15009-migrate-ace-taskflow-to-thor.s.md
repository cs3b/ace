# 150.09: Migrate ace-taskflow to Thor

## Objective

Convert ace-taskflow from custom case/when CLI to Thor following ADR-018 pattern.

## Current State

- **Framework**: Custom case/when routing with lazy loading
- **Entry Point**: `ace-taskflow/lib/ace/taskflow/cli.rb`
- **Lines of Code**: 152 (CLI) + 10+ command files
- **Commands**: 12+ (idea, ideas, task, tasks, release, releases, retro, retros, status, doctor, migrate, config)
- **Interface doc**: `_current/usage/ace-taskflow.md`

## Key Challenges

- **Many subcommands**: 12+ commands with their own subcommands
- **Singular/plural pattern**: task/tasks, idea/ideas, etc.
- **Lazy loading**: Commands loaded only when needed
- **Cache clearing**: Caches cleared at CLI start
- **Existing command classes**: Already structured per ADR-018

## Recommended Approach

**Hybrid**: Thor CLI wrapper that delegates to existing command classes.

## Deliverables

### Modify

1. `ace-taskflow/lib/ace/taskflow/cli.rb` - Convert to Thor
2. `ace-taskflow/exe/ace-taskflow` - Thor start wrapper
3. `.ace-defaults/taskflow/config.yml` - Add cli_defaults

Existing command classes remain unchanged.

## Implementation

### Thor CLI

```ruby
# lib/ace/taskflow/cli.rb
require "thor"

module Ace
  module Taskflow
    class CLI < Thor
      def self.start(args = ARGV, config = {})
        clear_caches!
        super
      end

      class_option :quiet, type: :boolean, aliases: "-q", desc: "Suppress config summary"

      # Status and Configuration
      desc "status", "Show current taskflow status"
      option :json, type: :boolean
      option :markdown, type: :boolean
      option :recently_done_limit, type: :numeric
      option :up_next_limit, type: :numeric
      option :include_drafts, type: :boolean
      option :no_activity, type: :boolean
      def status
        display_config_summary("status")
        require_relative "commands/status_command"
        Commands::StatusCommand.new.execute(options_to_args)
      end

      desc "config", "Show current configuration"
      def config
        show_config
      end

      desc "doctor", "Run health checks and auto-fix"
      def doctor
        require_relative "commands/doctor_command"
        Commands::DoctorCommand.new.execute([])
      end

      desc "migrate", "Migrate folder structure"
      def migrate
        require_relative "commands/migrate_command"
        Commands::MigrateCommand.new.execute([])
      end

      # Idea Management
      desc "idea SUBCOMMAND", "Operations on single ideas"
      def idea(*args)
        display_config_summary("idea")
        require_relative "commands/idea_command"
        Commands::IdeaCommand.new.execute(args)
      end

      desc "ideas [PRESET]", "Browse and list ideas"
      option :filter, type: :string
      option :limit, type: :numeric
      option :stats, type: :boolean
      option :tree, type: :boolean
      option :path, type: :boolean
      option :list, type: :boolean
      option :days, type: :numeric
      option :sort, type: :string
      option :release, type: :string
      option :backlog, type: :boolean
      def ideas(preset = nil)
        display_config_summary("ideas")
        require_relative "commands/ideas_command"
        Commands::IdeasCommand.new.execute(build_args(preset))
      end

      # Task Management
      desc "task SUBCOMMAND", "Operations on single tasks"
      def task(*args)
        display_config_summary("task")
        require_relative "commands/task_command"
        Commands::TaskCommand.new.execute(args)
      end

      desc "tasks [PRESET]", "Browse and list tasks"
      option :filter, type: :string
      option :limit, type: :numeric
      option :days, type: :numeric
      option :stats, type: :boolean
      option :tree, type: :boolean
      option :path, type: :boolean
      option :list, type: :boolean
      option :subtasks, type: :boolean
      option :no_subtasks, type: :boolean
      option :flat, type: :boolean
      option :release, type: :string
      option :backlog, type: :boolean
      option :sort, type: :string
      def tasks(preset = nil)
        display_config_summary("tasks")
        require_relative "commands/tasks_command"
        Commands::TasksCommand.new.execute(build_args(preset))
      end

      # Release Management
      desc "release SUBCOMMAND", "Operations on releases"
      def release(*args)
        display_config_summary("release")
        require_relative "commands/release_command"
        Commands::ReleaseCommand.new.execute(args)
      end

      desc "releases [PRESET]", "Browse and list releases"
      option :filter, type: :string
      option :stats, type: :boolean
      option :limit, type: :numeric
      option :sort, type: :string
      def releases(preset = nil)
        display_config_summary("releases")
        require_relative "commands/releases_command"
        Commands::ReleasesCommand.new.execute(build_args(preset))
      end

      # Retrospective Management
      desc "retro SUBCOMMAND", "Operations on retrospectives"
      def retro(*args)
        display_config_summary("retro")
        require_relative "commands/retro_command"
        Commands::RetroCommand.new.execute(args)
      end

      desc "retros", "Browse and list retrospectives"
      option :release, type: :string
      option :current, type: :boolean
      option :backlog, type: :boolean
      option :all, type: :boolean
      option :done, type: :boolean
      option :limit, type: :numeric
      def retros
        display_config_summary("retros")
        require_relative "commands/retros_command"
        Commands::RetrosCommand.new.execute(options_to_args)
      end

      map "--version" => :version  # -v reserved for verbose per usage/ux.md
      desc "version", "Show version"
      def version
        puts "ace-taskflow #{VERSION}"
      end

      private

      def display_config_summary(command)
        return if options[:quiet]
        Ace::Core::Atoms::ConfigSummary.display(
          command: command,
          config: Ace::Taskflow.config,
          defaults: Ace::Taskflow.default_config,
          options: options
        )
      end

      def options_to_args
        # Convert Thor options to command-line style args
        # NOTE: Handle special characters properly - values with spaces or
        # special chars are passed as single array elements, not shell-escaped
        args = []
        options.each do |key, value|
          next if value.nil? || value == false
          flag = "--#{key.to_s.tr('_', '-')}"

          case value
          when true
            args << flag
          when Array
            # Handle array values (repeatable options)
            value.each { |v| args << flag << v.to_s }
          else
            # String/numeric values - pass as-is (no shell escaping needed
            # since we're passing array elements, not constructing a shell command)
            args << flag << value.to_s
          end
        end
        args
      end

      def build_args(preset)
        args = preset ? [preset] : []
        args.concat(options_to_args)
        args
      end

      def self.clear_caches!
        require_relative "molecules/task_loader"
        require_relative "molecules/release_resolver"
        Molecules::TaskLoader.clear_cache!
        Molecules::ReleaseResolver.clear_cache!
      end

      def show_config
        require_relative "molecules/config_loader"
        config = Molecules::ConfigLoader.load
        puts "ace-taskflow Configuration:"
        puts "=" * 50
        # ... existing config display
      end
    end
  end
end
```

## Key Design Decisions

1. **Delegate to existing commands**: Thor CLI acts as wrapper, existing command classes handle logic
2. **Options to args conversion**: Thor options converted to args array for existing commands
3. **Cache clearing preserved**: `self.start` clears caches before Thor dispatch
4. **Lazy loading preserved**: `require_relative` in each method

## Special Character Handling

The `options_to_args` helper passes values as array elements (not shell-escaped strings), so special characters are handled correctly:

```ruby
# Thor options with special chars
options = { filter: "status:in progress", title: "Fix bug #123" }

# Converted to args array (not shell string)
args = ["--filter", "status:in progress", "--title", "Fix bug #123"]

# Command receives values correctly - no quoting issues
Commands::TasksCommand.new.execute(args)
```

**Test cases for special characters:**
- [ ] Values with spaces: `--filter "status:in progress"`
- [ ] Values with colons: `--filter status:pending`
- [ ] Values with quotes: `--title "Fix 'bug'"`
- [ ] Values with hash: `--title "Issue #123"`
