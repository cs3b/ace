## Implementation

### Standard Template

```ruby
desc "command [ARGS]", "One-line description"
long_desc <<~DESC
  Detailed paragraph description (2-3 sentences).

  SYNTAX:
    [Command syntax patterns if complex]

  EXAMPLES:

    # Basic usage
    $ command --option value

    # Advanced usage
    $ command --option1 value1 --option2 value2

  CONFIGURATION:

    Global config:  ~/.ace/<gem>/config.yml
    Project config: .ace/<gem>/config.yml

  OUTPUT:

    [Describe output and exit codes]

  TOOL_SPECIFIC:

    [Tool-specific sections]
DESC
```

### Custom `self.help` Pattern

Use for gems with magic routing or complex concepts:

```ruby
def self.help(shell, subcommand = false)
  super
  shell.say ""
  shell.say "FEATURE_NAME:"
  shell.say "  Description of feature:"
  shell.say "    ace-tool pattern  → ace-tool command pattern"
  shell.say "  Recognized: pattern1, pattern2"
  shell.say ""
  shell.say "Examples:"
  shell.say "  ace-tool --flag value     # Description"
end
```

**Add custom `self.help` to:**
- ace-context (Protocol Routing)
- ace-nav (Magic Wildcard Routing)
- ace-search (Search Type Detection)
- ace-review (Preset System)
- ace-taskflow (Task Reference System)

**Skip for:**
- ace-lint, ace-prompt, ace-git-commit (simple tools)
- ace-llm, ace-git-worktree, ace-test-runner, ace-docs (no special routing)

### Example: ace-lint Enhancement

```ruby
desc "lint [FILES...]", "Lint markdown, YAML, and frontmatter files"
long_desc <<~DESC
  Lint markdown, YAML, and frontmatter files for common issues.

  EXAMPLES:

    # Basic linting (auto-detect file type)
    $ ace-lint README.md
    $ ace-lint docs/**/*.md

    # Auto-fix and format
    $ ace-lint --fix README.md
    $ ace-lint --format --line-width 80 docs/*.md

    # Specific file type
    $ ace-lint --type yaml config.yml
    $ ace-lint --type frontmatter _posts/*.md

  SUPPORTED FILE TYPES:

    markdown     .md files with frontmatter validation
    yaml         .yml, .yaml files with syntax checking
    frontmatter  Files with YAML frontmatter (any extension)

  OUTPUT:

    Exit codes: 0 (success), 1 (errors found), 2 (fatal error)
    Errors printed to stderr in format: "file:line: message"
DESC
```
