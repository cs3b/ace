# 150.13: CLI Base Prerequisites

## Objective

Prepare for Base class adoption by updating the Base class version mapping and fixing `exit_on_failure?` in CLIs that deviate from the standard pattern.

## Deliverables

### 1. Update Base Class Version Mapping

**File**: `ace-support-core/lib/ace/core/cli/base.rb`

- Remove `-V` mapping from `version_command` helper
- Keep only `--version` mapping (no short flag for version)
- `-v` is reserved exclusively for `--verbose`

### 2. Fix exit_on_failure? in 3 CLIs

| File | Change |
|------|--------|
| `ace-prompt/lib/ace/prompt/cli.rb` | Change `exit_on_failure?` from `false` to `true` |
| `ace-llm-models-dev/lib/ace/llm/models_dev/cli.rb` | Change `exit_on_failure?` from `false` to `true` |
| `ace-git-secrets/lib/ace/git/secrets/cli.rb` | Change `exit_on_failure?` from `false` to `true`, refactor `last_exit_code` pattern |

### 3. Update Tests

**File**: `ace-prompt/test/commands/cli_test.rb`
- Update test that asserts `exit_on_failure? == false` to expect `true`

### 4. Refactor ace-git-secrets

- Remove `last_exit_code` class accessor pattern
- Update `exe/ace-git-secrets` to use standard exit code handling
