# Refactor ace-context to markdown-only preset configuration

## Behavioral Context

**Issue**: The ace-context gem had a complex configuration system mixing YAML files (.ace/context.yml), XML tags in markdown, and multiple backward compatibility paths that made it confusing for users.

**Key Behavioral Requirements**:
- Simplified configuration using only markdown files with frontmatter
- Clear and intuitive --output flag replacing confusing --cache/--no-cache
- Industry-standard .example directory pattern for samples

## Objective

Simplify ace-context configuration to use only markdown preset files with YAML frontmatter, eliminating complexity and improving user experience.

## Scope of Work

- Removed all YAML configuration file support (.ace/context.yml)
- Implemented markdown-only preset system with frontmatter
- Replaced confusing cache flags with clear --output modes
- Created example configuration structure

## Implementation Summary

### What Was Done

- **Problem Identification**: Configuration system was overly complex with multiple formats and confusing cache flags
- **Investigation**: Analyzed existing usage patterns and identified core requirements
- **Solution**: Simplified to markdown-only presets with clear frontmatter structure
- **Validation**: Tested all output modes and preset loading successfully

### Technical Details

**New Frontmatter Structure**:
```yaml
---
description: Preset description
params:
  output: cache|stdio|filepath  # Default output mode
  embed_itself: true            # Renamed from embed_document_source
  max_size: 10485760
  timeout: 30
context:
  files: [...]
  commands: [...]
  exclude: [...]
---
```

**CLI Changes**:
- Removed: `--cache`, `--no-cache`
- Added: `--output MODE` with three clear modes (stdio, cache, filepath)

**PresetManager Simplification**:
- Loads only from `.ace/context/*.md`
- No backward compatibility paths
- Clean frontmatter parsing

### Testing/Validation

```bash
# List presets
bundle exec ace-context --list

# Test output modes
bundle exec ace-context project --output stdio
bundle exec ace-context project --output cache
bundle exec ace-context project --output ./custom.md

# Verify glob expansion
bundle exec ace-context gems
```

**Results**: All modes work correctly, glob patterns expand properly, commands execute successfully
