# Show parent task context for orphan subtasks in filtered results

## Behavioral Specification

### User Experience
- **Input**: Running `ace-taskflow tasks` with filters that match subtasks but not their parent
- **Process**: Display orphan subtasks grouped under their parent task with a `[context]` indicator
- **Output**: Users always see parent context when viewing subtasks, regardless of filter criteria

### Expected Behavior

When subtasks match filter criteria but their parent orchestrator task does not, the parent should still be displayed for context.

**Before (current):**
```
$ ace-taskflow tasks --status pending
v.0.9.0+task.202.01 ⚪ Rename ace-llm-query...
v.0.9.0+task.202.02 ⚪ Rename ace-config...
# Parent 202 is missing - no context!
```

**After (proposed):**
```
$ ace-taskflow tasks --status pending
v.0.9.0+task.202 ⚫ Rename Support Gems (Orchestrator) [context]
  .ace-taskflow/v.0.9.0/tasks/202-project-refactor/202-rename-support-gems.s.md
  ├─ v.0.9.0+task.202.01 ⚪ Rename ace-llm-query...
  ├─ v.0.9.0+task.202.02 ⚪ Rename ace-config...
  └─ ...
```

### Success Criteria

- [x] Parent task shown with `[context]` indicator when subtasks are orphaned
- [x] Subtasks displayed under parent with tree connectors
- [x] Parent does not count toward result count (it's context, not a match)
- [x] Works with `--subtasks` display mode

## Objective

Improve UX when filtering tasks by always showing parent context for subtasks, even when the parent's status doesn't match the filter.

## Technical Approach

### Current Code (lines 599-603 in tasks_command.rb)

```ruby
# Display subtasks whose parents are not in the result set (orphan subtasks)
orphan_subtasks = tasks.select do |t|
  t[:parent_id] && !orchestrators.any? { |o| o[:id] == t[:parent_id] }
end
orphan_subtasks.each { |task| display_task_line(task) }
```

**Problem:** Orphan subtasks are displayed as regular tasks, losing parent context.

### Proposed Change

```ruby
# Display subtasks whose parents are not in the result set (orphan subtasks)
# Group by parent and display with parent context
orphan_subtasks = tasks.select do |t|
  t[:parent_id] && !orchestrators.any? { |o| o[:id] == t[:parent_id] }
end

orphan_subtasks.group_by { |t| t[:parent_id] }.each do |parent_id, children|
  # Load and display parent for context
  parent_task = @manager.find_task(parent_id)
  if parent_task
    display_parent_context_line(parent_task)
  end
  children.sort_by { |s| s[:id] || "" }.each_with_index do |subtask, idx|
    connector = idx == children.length - 1 ? "└─" : "├─"
    display_subtask_line(subtask, connector)
  end
end
```

### New Display Method

```ruby
def display_parent_context_line(task_data)
  task = Models::Task.new(task_data)
  status_str = status_icon(task.status)
  ref = task.qualified_reference || task.task_number || task.id || "unknown"
  display_title = strip_task_id_from_title(task.title)

  puts "  #{ref.ljust(15)} #{status_str} #{display_title} (Orchestrator) [context]"
  if task.path
    relative_path = format_relative_path(task.path)
    puts "    #{relative_path}"
  end
end
```
