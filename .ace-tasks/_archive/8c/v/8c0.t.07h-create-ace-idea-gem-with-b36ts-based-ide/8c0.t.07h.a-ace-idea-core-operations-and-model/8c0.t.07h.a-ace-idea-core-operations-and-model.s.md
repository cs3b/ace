# ace-idea Core Operations and Model

## Behavioral Specification

### User Experience
- **Input**: Idea content, references, clipboard data, LLM enhancement requests
- **Process**: IdeaManager organism handles all CRUD operations using ace-support-items shared infrastructure
- **Output**: Idea objects created/loaded/updated with full metadata

### Expected Behavior

Build the complete core operations layer for ideas, leveraging ace-support-items from task 290.01:

1. **IdeaScanner**: Wraps DirectoryScanner for `.idea.s.md` files. Returns scan results with raw b36ts IDs.
2. **IdeaResolver**: Wraps ShortcutResolver for raw idea IDs (no `.t.` type marker). Suffix match on last 3 chars. Must explicitly detect and warn on ambiguity collisions.
3. **IdeaLoader**: Loads idea from directory, parses frontmatter + body, lists attachments (images, files).
4. **IdeaCreator**: Full creation with b36ts ID, slug generation, folder+file creation. Supports:
   - `--clipboard`: Capture system clipboard (text, RTF, HTML, images) — adapted from existing `ace-taskflow/lib/ace/taskflow/atoms/clipboard_reader.rb`. *Note: Text is captured as markdown content; rich media/images are saved strictly as file attachments in the idea folder with markdown references injected.*
   - `--llm-enhance`: Send raw idea through LLM to produce structured 3-Question Brief. *Note: The system prompt for this should be hardcoded in `LlmEnhancer` for this iteration.*
   - `--move-to`: Place in specific folder on creation
5. **IdeaMover**: Delegates to FolderMover from ace-support-items.
6. **IdeaManager**: Orchestrates create, show, list, update, move.

### Interface Contract (Ruby API)

```ruby
manager = Ace::Idea::Organisms::IdeaManager.new

# Create
idea = manager.create("Dark mode would be great for night coding",
                       title: "Dark mode support",
                       tags: ["ux", "design"],
                       move_to: "next")
# => Idea(id: "8ppq7w", status: "pending", ...)

# Create from clipboard
idea = manager.create_from_clipboard(llm_enhance: true)
# => Idea with LLM-enhanced content in 3-Question Brief format

# Show
idea = manager.show("q7w")
# => Idea with full metadata + attachments list

# List
ideas = manager.list(status: "pending", in_folder: "maybe", tags: ["ux"])
# => Array of Idea objects

# Update
manager.update("q7w", set: { "status" => "done" }, add: { "tags" => "implemented" })

# Move
manager.move("q7w", to: "archive")
```

### Success Criteria

- [ ] IdeaScanner finds all `.idea.s.md` files recursively, excluding `.s.md` (task) files
- [ ] IdeaResolver resolves raw b36ts shortcuts (last 3 chars) with duplicate warnings
- [ ] IdeaLoader parses frontmatter + body, enumerates attachments in idea folder
- [ ] IdeaCreator generates raw b36ts ID, creates `{id}-{slug}/{id}-{slug}.idea.s.md`
- [ ] Clipboard capture works for plain text, RTF, HTML, and images (saves attachments to idea folder)
- [ ] LLM enhancement produces 3-Question Brief structure (What I Hope to Accomplish, What "Complete" Looks Like, Success Criteria)
- [ ] `--move-to` on create places idea in target folder immediately
- [ ] IdeaManager orchestrates all operations with config-driven root directory
- [ ] Idea model holds: id, status, title, tags, content, path, file_path, special_folder, created_at, attachments, metadata
- [ ] Full test coverage at atoms/molecules/organisms/models layers
