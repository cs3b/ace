# Fix ace-taskflow idea subcommands for dry-cli compatibility

## Problem

The `idea` subcommands fail with dry-cli because they weren't properly migrated to the nested command pattern:

```bash
ace-taskflow idea create -gc "some idea"
# ERROR: "ace-taskflow idea" was called with arguments "create -gc ..."
```

## Root Cause

The `task` command has proper nested subcommand registrations:
```ruby
register "task create", Commands::Task::Create
register "task show", Commands::Task::Show
```

But `idea` only has:
```ruby
register "idea", CLI::Idea.new  # No nested subcommands!
```

When dry-cli receives `ace-taskflow idea create ...`, it routes to `CLI::Idea` and passes "create -gc ..." as extra arguments. But `CLI::Idea` doesn't define any `argument` declarations to capture them, so dry-cli throws an error.
