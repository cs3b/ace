# 122.02 - Task Scanner Enhancement

## Scope

Update `TaskLoader` to discover orchestrator + subtask patterns and build parent-child relationships during task loading.

## Interface

```ruby
# TaskLoader returns tasks with new fields
task = task_loader.find_task_by_reference("121")
task[:parent_id]      # nil for parent/single tasks
task[:subtask_ids]    # ["v.0.9.0+task.121.01", "v.0.9.0+task.121.02", ...]
task[:is_orchestrator] # true if has subtasks

# Subtask loading
subtask = task_loader.find_task_by_reference("121.01")
subtask[:parent_id]   # "v.0.9.0+task.121"
subtask[:subtask_ids] # [] (subtasks don't have children)
```

## Behavior

### File Pattern Recognition

| Pattern | Type | Example |
|---------|------|---------|
| `{N}.00-*.s.md` | Orchestrator | `121-orchestrator.s.md` |
| `{N}.NN-*.s.md` (NN > 00) | Subtask | `121.01-archive-output.s.md` |
| `{N}-*.s.md` | Single task | `119-feature-name.s.md` |

### Task Directory Scanning

When scanning a task directory like `122-subtask-workflow-support/`:

1. Look for orchestrator file: `122.00-*.s.md`
2. Look for subtask files: `122.01-*.s.md`, `122.02-*.s.md`, etc.
3. If no orchestrator (.00) file but has subtasks:
   - Assume a **virtual parent** based on task number
   - All subtasks get `parent_id: v.0.9.0+task.{N}` (e.g., `v.0.9.0+task.122`)
   - `ace-taskflow task 122` returns nil with message "No orchestrator file found"
   - This maintains consistent data model without promoting subtask to parent
4. If single file `122-*.s.md` -> regular task (no hierarchy)

### Relationship Building

During `load_tasks_from_release()`:
1. First pass: Load all task files
2. Second pass: Build relationships
   - Link subtasks to parents via `:parent_id`
   - Collect subtask IDs into parent's `:subtask_ids`

### Edge Cases

- Task directory with only subtasks (no .00 file) -> use virtual parent (subtasks point to `v.0.9.0+task.{N}`)
- Task directory with orchestrator but no subtasks -> orchestrator without children
- Task references parent field in frontmatter but file missing -> graceful handling (log warning, use derived parent)
- Subtask numbering gaps (121.01, 121.03, no 121.02) -> handled correctly
