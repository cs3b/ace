# 122.03 - CLI Integration

## Scope

Add CLI support for subtask creation (`--child-of`) and hierarchical task listing (`--subtasks`/`--no-subtasks`).

## Interface

### Task Creation

```bash
# Create subtask under existing parent
ace-taskflow task create "Archive + Output" --child-of 121
# Creates: 121-ace-prompt/121.01-archive-output.s.md

ace-taskflow task create "Setup & Reset" --child-of 121
# Creates: 121-ace-prompt/121.02-setup-reset.s.md (auto-increments)

# Short flag
ace-taskflow task create "Context Loading" -p 121
```

### Task Listing

```bash
# Default (config-driven, default: show subtasks)
ace-taskflow tasks

# Explicit flags
ace-taskflow tasks --subtasks      # Show hierarchy
ace-taskflow tasks --no-subtasks   # Hide subtasks
ace-taskflow tasks --flat          # No grouping, all tasks flat
```

**Precedence Rules:**
1. CLI flags always override config
2. `--flat` wins if combined with `--subtasks` or `--no-subtasks`
3. When no flags: `params.subtasks` config controls behavior
4. If config missing: default to `enabled` (show subtasks)

### Task Reference

```bash
ace-taskflow task 121        # Returns orchestrator (121.00)
ace-taskflow task 121.01     # Returns specific subtask
ace-taskflow task 121 --path # Returns orchestrator file path
```

## Behavior

### Subtask Creation (`--child-of`)

1. Parse parent reference using hierarchical parser
2. Find parent task directory
3. Scan for existing subtasks to determine next number (max+1, no gap-fill)
4. Generate subtask file: `{parent}.{NN}-{slug}.s.md`
5. Add `parent: v.0.9.0+task.{N}` to frontmatter (fully qualified ID)
6. Add `dependencies:` field:
   - First subtask (XX.01): `dependencies: []` (no dependency on orchestrator)
   - Subsequent subtasks: `dependencies: [v.0.9.0+task.{N}.{NN-1}]`
   - Uses fully qualified IDs for cross-release compatibility
7. Return created file path

### Task Listing with Hierarchy

Default display (with `params.subtasks: enabled`):
```
📋 Pending Tasks (v.0.9.0)

122 - Subtask Workflow Support [orchestrator]
  ├─ 122.01 - Hierarchical Parser [pending]
  ├─ 122.02 - Task Scanner [pending]
  ├─ 122.03 - CLI Integration [pending]
  └─ 122.04 - Orchestration Workflow [pending]

119 - Another Task [pending]
```

With `--no-subtasks`:
```
📋 Pending Tasks (v.0.9.0)

122 - Subtask Workflow Support [orchestrator] (4 subtasks)
119 - Another Task [pending]
```

With `--flat`:
```
📋 Pending Tasks (v.0.9.0)

119 - Another Task [pending]
122 - Subtask Workflow Support [orchestrator]
122.01 - Hierarchical Parser [pending]
122.02 - Task Scanner [pending]
122.03 - CLI Integration [pending]
122.04 - Orchestration Workflow [pending]
```

### Edge Cases

- Parent task doesn't exist -> helpful error with suggestions
- Parent directory doesn't exist -> create it
- 99 subtasks already exist -> error (max reached)
- Creating subtask of subtask (121.01.01) -> not supported, error
- Parent is single task (no orchestrator):
  - **Error**: "Task {N} is not an orchestrator. Create `{N}.00-orchestrator.s.md` first or use `ace-taskflow task promote {N}` to convert."
  - Do NOT auto-convert (complexity, risk of data loss)
  - Future: May add `task promote` command to convert single task to orchestrator
