# Display Formatting & Lifecycle for Hierarchical Tasks

## Scope

Fix display formatting and lifecycle issues for hierarchical tasks (orchestrators + subtasks).

### Phase 1: Display Issues

1. **Wrong subtask IDs**: Subtasks show parent's ID (`v.0.9.0+task.121`) instead of their own (`v.0.9.0+task.121.01`)
2. **Redundant ID in title**: Titles show `121.01 - Basic: Archive...` instead of just `Basic: Archive...`
3. **Missing subtasks in single task view**: `ace-taskflow task 121` doesn't show subtasks for orchestrators

### Phase 2: Lifecycle Issues (Critical Bug)

4. **Subtask "done" moves entire folder**: Running `ace-taskflow task done 122.04` moved the entire `122-subtask-workflow-support/` folder to `done/`, breaking access to other subtasks.

**Root cause**: `TaskManager#mark_task_as_done` calls `TaskDirectoryMover#move_to_done` which moves the parent folder without checking task type.

## Behavior

### Fix 1: Subtask Qualified Reference

**File**: `models/task.rb` - `qualified_reference` method

Current logic uses `task_number` which is just parent number for subtasks.

**Solution**: Prefer `id` from frontmatter for subtasks, construct qualified ref only for non-hierarchical tasks.

### Fix 2: Strip ID Prefix from Titles

**File**: `commands/tasks_command.rb` - display methods

Strip leading task ID pattern from title (e.g., `122.05 - ` prefix).

Pattern: `/^\d+(\.\d+)?\s*[-–]\s*/`

### Fix 3: Show Subtasks in Single Task View

**File**: `commands/task_command.rb` - `show_task` method

When displaying an orchestrator task:
1. Detect if task is orchestrator (`is_orchestrator` flag or has `subtasks` in frontmatter)
2. Load subtasks from task loader
3. Display subtasks with tree connectors after task details

### Fix 4: Subtask Lifecycle (Don't Move Folder)

**File**: `organisms/task_manager.rb` - `mark_task_as_done` method

**Solution**: Check task type before moving folder:

1. **Subtasks**: Only update status to "done", DON'T move folder
2. **Orchestrators**: Auto-move folder when ALL subtasks have terminal status (done/cancelled)
   - Also auto-update orchestrator status to "done" when all subtasks are done
3. **Single tasks**: Current behavior (move folder to done/)

```ruby
def mark_task_as_done(reference)
  task = find_task(reference)
  update_status(task, "done")

  if task[:parent_id]
    # Subtask: check if parent should auto-complete
    check_orchestrator_completion(task[:parent_id])
    { success: true, message: "Subtask marked as done" }
  elsif task[:is_orchestrator]
    if all_subtasks_terminal?(task)
      move_to_done(task[:path])
    else
      { success: true, message: "Orchestrator done (waiting for subtasks)" }
    end
  else
    move_to_done(task[:path])
  end
end
```
