# 122.04 - Orchestration Workflow

## Scope

Create new `work-on-subtasks.wf.md` workflow for executing orchestration tasks with worktree-per-subtask pattern, including subagent delegation, PR creation, and review cycles.

## Interface

### work-on-task Detection

When `ace:work-on-task` is invoked on an orchestrator task:
1. Detect task type (orchestrator vs regular)
2. If orchestrator -> redirect to work-on-subtasks workflow
3. If regular -> proceed with standard work-on-task

### work-on-subtasks Process

```
For orchestrator task 122:
  1. Load orchestrator, get pending subtasks
  2. For each subtask (122.01, 122.02, ...):
     a. Create worktree: .ace-wt/task.122.01/
     b. Launch subagent in worktree
     c. Subagent executes work-on-task
     d. Create PR: 122.01-* -> 122-*
     e. Review cycle (x2)
     f. User validates manually
     g. Run tests
     h. Mark subtask done
  3. Summarize progress
  4. Wait for user feedback
```

## Behavior

### Task Type Detection

Add to work-on-task.wf.md:
```markdown
## Task Type Detection

After loading task, determine type:

1. **Regular Task** (no subtasks in frontmatter, not .00 suffix)
   -> Execute standard work-on-task workflow

2. **Orchestrator Task** (has subtasks: [...] or filename matches *.00-*.s.md)
   -> Redirect to work-on-subtasks workflow
   -> Load: `ace-nav wfi://work-on-subtasks`
```

### Worktree Creation

```bash
ace-git-worktree create --task 122.01
# Creates: .ace-wt/task.122.01/
# Branch: 122.01-hierarchical-parser
# Updates task status: in-progress
```

### Subagent Delegation

Use Task tool:
```markdown
**Task tool prompt:**
Execute work-on-task workflow for subtask: 122.01

Working directory: .ace-wt/task.122.01/

Follow the complete work-on-task workflow:
1. Read and execute: ace-nav wfi://work-on-task
2. Execute all implementation steps from the task file
3. Run tests: ace-test
4. Update task status to done when complete
5. Return summary of changes made

**Subagent type:** general-purpose
```

### PR Creation

```bash
# From worktree directory
gh pr create \
  --base 122-subtask-workflow-support \
  --head 122.01-hierarchical-parser \
  --title "122.01: Hierarchical Task ID Parser" \
  --body "$(cat <<'EOF'
## Summary
- Extended TaskReferenceParser for hierarchical IDs
- Added support for 121.00, 121.01 formats
- Maintained backward compatibility

## Test Plan
- [ ] All existing tests pass
- [ ] New hierarchical parsing tests pass

🤖 Generated with [Claude Code](https://claude.com/claude-code)
EOF
)"
```

### Review Cycle

```
For each completed subtask:
  1. Run ace-review on PR
  2. If approved -> continue
  3. If feedback:
     a. Delegate to subagent: apply review feedback
     b. Re-run ace-review
     c. Repeat up to 2 times
  4. If still not approved after 2 cycles:
     a. Summarize issues
     b. Ask user how to proceed
```

### Manual Validation

```markdown
**Ask User:**
Subtask 122.01 implementation is complete.

Please validate manually:
1. Test the CLI: `ace-taskflow task 121.01`
2. Verify parser handles: `121`, `121.00`, `121.01`
3. Run: `ace-test test/atoms/task_reference_parser_test.rb`

Is the implementation working correctly?
- [ ] Yes, proceed to next subtask
- [ ] No, need fixes (describe issues)
```

### Test Validation

```bash
# In worktree directory
ace-test

# If failures:
#   1. Delegate to subagent: fix failing tests
#   2. Re-run tests
#   3. Repeat until pass or max retries
```

## Edge Cases

### Workflow Edge Cases
- Subtask already in-progress -> skip, report status
- Subtask already done -> skip
- Worktree already exists -> **Reuse** existing worktree
  - Check for existence, report "Resuming work in existing worktree"
  - Do NOT cleanup/recreate (prevents data loss)
  - Separate `ace-git-worktree prune` command handles cleanup
- Subagent fails repeatedly -> ask user
- Tests never pass -> after N retries, ask user
- PR merge conflicts -> ask user to resolve

### ace-git-worktree Edge Cases
- Subtask ID format (122.01) must work with worktree
- Parent branch (122-*) must exist before subtask branches
- Cleanup worktrees after subtask completion?
