# Multi-Assignment Support

## Overview

Enable multiple concurrent assignments with listing, selection, and filtering. Extends existing single-assignment model with explicit assignment selection while maintaining backwards compatibility.

## Objective

Add multi-assignment management capabilities:
- Explicit assignment selection via `.current` symlink
- `ace-assign list` command to show all assignments
- `ace-assign select` command for assignment switching
- Enhanced `status` showing current job + other assignments
- `--assignment` flag on all commands for explicit targeting
- `ACE_ASSIGN_ID` env var for workflow context propagation

## Technical Research Findings

### Existing Infrastructure

The codebase already has partial multi-assignment support:

1. **AssignmentManager.list()** (lines 121-132 of `assignment_manager.rb`):
   - Implemented but unused
   - Returns all assignments sorted by `updated_at` descending

2. **`.latest` symlink** (lines 174-190):
   - Updated on every assignment create/update
   - Provides O(1) active assignment lookup
   - Used by `find_active()` as fast path

3. **QueueState.summary** (lines 98-106):
   - Returns `{total, done, in_progress, pending, failed}` stats
   - Ready for assignment list display

4. **Registered Commands** (`cli.rb` line 42):
   - `create, status, report, fail, add, retry, prepare`
   - Note: No `start` or `resume` - original spec referenced wrong commands

### Assignment State Derivation

Assignments can be in one of these states (to be computed from QueueState):
- **running**: Has `in_progress` phase
- **paused**: Has `pending` but no `in_progress` (interrupted)
- **completed**: All phases `done`
- **failed**: Has `failed` phase(s)

## Technical Approach

### Symlink Strategy: `.latest` + `.current`

```
.cache/ace-assign/
├── .latest → abc123/          # Auto-updated on any activity
├── .current → def456/         # Explicit user selection (optional)
├── abc123/
│   └── assignment.yaml
└── def456/
    └── assignment.yaml
```

**Resolution order for `find_current()`:**
1. `--assignment <id>` flag (highest priority)
2. `ACE_ASSIGN_ID` env var
3. `.current` symlink (explicit selection)
4. `.latest` symlink (auto, most recent)
5. Scan all assignments (fallback)

### AssignmentDiscoverer Molecule

```ruby
module Ace::Assign::Molecules
  class AssignmentDiscoverer
    def initialize(cache_base:, queue_scanner:)
      @cache_base = cache_base
      @queue_scanner = queue_scanner
    end

    # Find all assignments with computed state
    # @return [Array<AssignmentInfo>]
    def find_all(include_completed: false)
      assignments = assignment_manager.list
      assignments.map { |a| enrich_assignment(a) }
                 .select { |ai| include_completed || !ai.completed? }
    end

    # Find assignments by task reference
    def find_by_task(task_ref:, active_only: true)
      find_all(include_completed: !active_only)
        .select { |ai| ai.assignment.task_ref == task_ref }
    end

    private

    def enrich_assignment(assignment)
      state = @queue_scanner.scan(assignment.phases_dir, assignment: assignment)
      AssignmentInfo.new(assignment: assignment, queue_state: state)
    end
  end
end
```

### AssignmentInfo Model

```ruby
module Ace::Assign::Models
  class AssignmentInfo
    attr_reader :assignment, :queue_state

    def initialize(assignment:, queue_state:)
      @assignment = assignment
      @queue_state = queue_state
    end

    # Computed state: :running, :paused, :completed, :failed
    def state
      return :empty if queue_state.empty?
      return :failed if queue_state.failed.any?
      return :completed if queue_state.complete?
      return :running if queue_state.current
      :paused
    end

    def progress
      s = queue_state.summary
      "#{s[:done]}/#{s[:total]}"
    end

    def current_phase
      queue_state.current&.name || "-"
    end

    # Delegate to assignment
    def id = assignment.id
    def name = assignment.name
    def updated_at = assignment.updated_at
  end
end
```

### CLI Interface

```bash
# List assignments
ace-assign list                    # Active assignments
ace-assign list --all              # Include completed
ace-assign list --task 148         # Filter by task
ace-assign list --format json      # JSON output

# Select assignment
ace-assign select abc123           # Set .current
ace-assign select --clear          # Remove .current

# Status with assignment
ace-assign status                  # Current + other assignments
ace-assign status --assignment abc123 # Specific assignment
ace-assign status --all            # Show completed too
ace-assign status --brief          # Summary only

# Other commands with --assignment
ace-assign report result.md --assignment abc123
ace-assign fail --assignment abc123 --message "blocked"
```
