# Add commit: subject type to ace-review

## Behavioral Specification

### User Experience

- **Input**: Developer runs `ace-review --subject "commit:3cd9afbf"` with a commit hash
- **Process**: ace-review recognizes `commit:` as a valid subject type, extracts the commit's changes using `git diff COMMIT~1..COMMIT`, and includes those changes in the review context
- **Output**: Review runs on the specific commit's changes with clear error messages if the commit hash is invalid

### Expected Behavior

When a developer uses `commit:HASH` as a subject:
- ace-review should recognize `commit:` as a first-class subject type (like `diff:`, `pr:`, `files:`, `task:`)
- The commit's changes should be extracted by comparing the commit to its parent (`COMMIT~1..COMMIT`)
- Invalid commit hash formats should raise clear errors before attempting git operations
- Non-existent commits should fail gracefully with git's error messages

### Interface Contract

```bash
# CLI Interface
ace-review --subject "commit:HASH"
ace-review --subject "commit:3cd9afbf"

# Short hashes (6+ chars) are supported
ace-review --subject "commit:abc123"

# Full hashes are supported
ace-review --subject "commit:3cd9afbf1234567890abcd1234567890abcd1234"

# Error: Invalid format
ace-review --subject "commit:xyz"
# Output: ArgumentError: Invalid commit hash format: 'xyz'. Must be 6-40 hexadecimal characters.

# Error: Empty value
ace-review --subject "commit:"
# Output: ArgumentError: Empty value for commit: subject. Usage: commit:HASH

# Error: Non-existent commit
ace-review --subject "commit:aaaaaaaa"
# Output: git error: fatal: bad revision 'aaaaaaaa~1'
```

**Error Handling:**
- Invalid format (non-hex): ArgumentError with clear message about format requirements
- Empty value: ArgumentError with usage example
- Non-existent commit: Git error propagates to user (no pre-validation needed)

**Edge Cases:**
- Ambiguous short hash (< 6 chars): Rejected with format error
- Very short hash (6 chars): Accepted, let git resolve ambiguity
- Merge commits: Standard git behavior (shows diff vs first parent)
- Root commit (no parent): Git will fail on `~1` syntax
