# Unified CLI help system fix and nested command splitting

## Behavioral Context

**Issue**: Task 274 added two-tier help (`-h` concise / `--help` full) but only 3 of 22 gems differentiated them correctly. Tools with 2-level subcommand nesting (ace-taskflow, ace-review, ace-models) required 3 drill-downs to learn a subcommand.

**Key Behavioral Requirements**:
- `-h` shows concise command listing with footer pointing to `--help`
- `--help` and `help` show full ALL-CAPS format with EXAMPLES section
- Subcommand help works at both tiers (`ace-task create -h` and `ace-task create --help`)
- Nested commands (e.g., `ace-taskflow task create`) split into flat CLIs (`ace-task create`)

## Objective

Fix the help system across all 22 CLI gems and split 3 nested command tools into flat CLIs.

## Scope of Work

- Created shared HelpRouter module in ace-support-core
- Added footer to concise help format ("Run 'tool --help' for more info")
- Added HELP_EXAMPLES support to full help format (tldr-style examples)
- Split ace-taskflow into 4 flat CLIs: ace-task, ace-idea, ace-release, ace-retro
- Split ace-review feedback commands into ace-review-feedback
- Split ace-models provider commands into ace-llm-providers
- Migrated 5 trivial gems to DefaultRouting
- Updated 9 custom start() methods to use HelpRouter
