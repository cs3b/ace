# Implement Help Formatters and Tests in ace-support-core

## Behavioral Specification

### User Experience

- **Input**: User runs `bin/ace-bundle -h` or `bin/ace-bundle load --help` (any binary, any subcommand)
- **Process**: Help renders instantly; no per-gem code touches required
- **Output**:
  - `-h` → compact ~15 line output, scannable in one glance
  - `--help` → full structured reference with ALL-CAPS section headers

### Expected Behavior

All 20 dry-cli gems automatically adopt the new format the moment `ace-support-core` is updated.

**Concise (`-h`) — registry level:**
```
ace-bundle - Load project context from presets, files, or protocol URLs

Usage: ace-bundle COMMAND [OPTIONS]

Commands:
  load [INPUT]          Load context (default)
  version               Show version

Run 'ace-bundle COMMAND --help' for details.
```

**Concise (`-h`) — subcommand:**
```
ace-bundle load - Load context from preset, file, or protocol URL

Usage: ace-bundle load [INPUT] [OPTIONS]

Options:
  --output, -o PATH     Write to file instead of stdout
  --quiet, -q           Suppress non-essential output
  --verbose, -v         Show verbose output
  --help, -h            Show this help

Run 'ace-bundle load --help' for full details.
```

**Full (`--help`) — subcommand:**
```
NAME
  ace-bundle load - Load context from preset, file, or protocol URL

USAGE
  ace-bundle load [INPUT] [OPTIONS]

DESCRIPTION
  INPUT can be a preset name, file path, or protocol URL.
  Protocols: wfi://, guide://, tmpl://, prompt://

ARGUMENTS
  INPUT                             Preset name, file path, or protocol URL

OPTIONS
  --output, -o PATH                 Write to file instead of stdout
  --quiet, -q                       Suppress non-essential output
  --verbose, -v                     Show verbose output
  --debug, -d                       Show debug output
  --help, -h                        Show this help

EXAMPLES
  $ ace-bundle load project           Load project preset
  $ ace-bundle load wfi://commit      Load workflow via protocol
```

### Interface Contract

**New files to create** (all in `ace-support-core/lib/ace/core/cli/dry_cli/`):

```
help_formatter.rb    # Dry::CLI::Banner monkey-patch (--help / full format)
help_concise.rb      # Concise -h format (Banner + Usage variants)
usage_formatter.rb   # Dry::CLI::Usage monkey-patch (first-line desc, grouping)
command_groups.rb    # Mixin + COMMAND_GROUPS constant convention
standard_options.rb  # Canonical desc strings: QUIET_DESC, VERBOSE_DESC, DEBUG_DESC
```

**Files to update:**

- `base.rb` — add 3 `require_relative` lines for the new formatters
- `default_routing.rb` — route `-h` to concise format, `--help` to full format

**Key behavioural rules:**

- `Usage.description(command)` returns **first line only** of multiline `desc` (fixes ace-docs, ace-nav, ace-bundle verbose registry)
- `Banner.section_examples` strips command-name prefix from examples that already include it (fixes ace-docs, ace-git-commit, ace-prompt-prep, ace-test duplicate-name bug)
- `-h` shows options without long descriptions (flag + alias + type only)
- `--help` shows full options with descriptions and defaults
- `COMMAND_GROUPS` constant on CLI registry → grouped headings in `--help` output; absent → flat list (safe default)

**Error handling:**
- If `command.description` is nil → NAME section shows just the command name, no ` - summary`
- If `command.examples` is empty → EXAMPLES section omitted entirely
