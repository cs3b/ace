# Add Ruby Linting Support to ace-lint with StandardRB

## Objective

Add Ruby file linting support to ace-lint using StandardRB (zero-config Ruby validator built on RuboCop). Also fix the autodetection behavior to skip unsupported file types instead of reporting errors.

## Behavioral Specification

### User Experience

- **Input**: Ruby files (.rb, .rake, .gemspec, Rakefile) passed to `ace-lint`
- **Process**: Runs StandardRB validator, displays issues with file:line:column format
- **Output**: Lint results with errors/warnings, optional autofix with `--fix`

### Interface Contract

```bash
# Lint Ruby files
ace-lint lib/ace/lint/cli.rb
ace-lint "**/*.rb"

# Autofix Ruby issues
ace-lint lib/ace/lint/cli.rb --fix

# Force Ruby type detection
ace-lint --type ruby some_file

# Expected output format
lib/ace/lint/cli.rb:15:3: Style/FrozenStringLiteralComment: Missing frozen string literal comment
lib/ace/lint/cli.rb:42:80: Layout/LineLength: Line is too long [95/80]
```

**Error Handling:**
- StandardRB not installed: Helpful error message with install instructions
- Unsupported file type: Skip silently (not an error), report as "skipped"

### Success Criteria

- [ ] Ruby files (.rb, .rake, .gemspec, Rakefile) are linted via StandardRB
- [ ] `--fix` flag applies StandardRB autofix (`standardrb --fix`)
- [ ] Unsupported file types are skipped (not reported as errors)
- [ ] Configuration in `.ace-defaults/lint/ruby.yml` follows ADR-022 pattern
- [ ] Clear error message when StandardRB is not installed
