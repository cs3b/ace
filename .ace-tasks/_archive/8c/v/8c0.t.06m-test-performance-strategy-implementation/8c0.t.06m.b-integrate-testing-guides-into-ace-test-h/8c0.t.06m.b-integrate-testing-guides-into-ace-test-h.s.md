# Integrate Testing Guides into ace-test Handbook

## Behavioral Specification

### User Experience
- **Input**: Agent or developer needs testing guidance via `ace-bundle guide://test-*` protocols
- **Process**: Guides are discoverable through ace-nav and loadable via ace-bundle
- **Output**: Complete testing guidance available inline without leaving workflow

### Expected Behavior

When a developer or AI agent needs testing guidance, they can:
1. Discover available guides via `ace-nav guide://*test*`
2. Load any guide via `ace-bundle guide://test-layer-decision`
3. Reference guides from workflows using standard `guide://` protocol

The 6 new guides integrate seamlessly with existing ace-test documentation.

### Interface Contract

```bash
# Discovery
ace-nav guide://*test*
# Returns: test-layer-decision, test-mocking-patterns, test-suite-health,
#          test-responsibility-map, test-review-checklist, testing-strategy

# Loading
ace-bundle guide://test-layer-decision
# Returns: Full guide content for inline use

# Cross-reference in workflows
ace-bundle wfi://plan-tests
# Guide references within workflow resolve correctly
```

### Success Criteria

- [ ] 6 guides moved to `ace-test/handbook/guides/`
- [ ] All guides follow ACE guide conventions (frontmatter, structure)
- [ ] Guides discoverable via `ace-nav guide://*test*`
- [ ] Protocol references work: `guide://test-layer-decision`
- [ ] Cross-references between guides resolve correctly
- [ ] SUMMARY.md navigation index updated

## Objective

Make the synthesized testing knowledge from Task 253 research accessible through standard ACE guide protocols, enabling agents and developers to access testing guidance inline during workflows.

## Technical Approach

### Architecture Analysis

**Current State:**
- ace-test has 9 existing guides in `handbook/guides/` with varied naming conventions
- ace-test has guide-sources config at `.ace-defaults/nav/protocols/guide-sources/ace-test.yml`
- Bug: ace-nav currently doesn't load ace-test as a guide source (only wfi source shows)
- ace-test-e2e-runner guides ARE discoverable via `guide://e2e-testing`

**Integration Pattern:**
- Copy guides from synthesis artifacts to ace-test handbook
- Update frontmatter to match ACE conventions (see existing guides)
- Fix guide source registration so ace-nav discovers them
- Update SUMMARY.md to include new guides

### Frontmatter Convention

Based on existing guides in ace-test:
```yaml
---
name: test-layer-decision
description: Decision framework for choosing the right test layer
doc-type: guide
purpose: Help developers decide where to test each behavior
search_keywords:
  - test layer
  - unit test
update:
  frequency: on-change
  last-updated: '2026-01-31'
---
```
