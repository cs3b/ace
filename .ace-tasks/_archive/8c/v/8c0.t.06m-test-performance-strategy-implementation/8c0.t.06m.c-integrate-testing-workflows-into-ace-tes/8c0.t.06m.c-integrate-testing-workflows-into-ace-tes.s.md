# Integrate Testing Workflows into ace-test Handbook

## Behavioral Specification

### User Experience
- **Input**: Agent invokes testing workflow via `ace-bundle wfi://plan-tests` or skill
- **Process**: Workflow loads with embedded context, guides agent through testing process
- **Output**: Structured test artifacts (responsibility maps, audit reports, test cases)

### Expected Behavior

When an agent needs to plan, verify, or optimize tests:
1. Discover workflows via `ace-nav wfi://*test*`
2. Load workflow via `ace-bundle wfi://plan-tests`
3. Execute workflow steps with proper tool integrations
4. Produce structured output using embedded templates

The 5 workflows provide complete testing lifecycle coverage.

### Interface Contract

```bash
# Discovery
ace-nav wfi://*test*
# Returns: plan-tests, verify-test-suite, optimize-tests,
#          e2e-sandbox-setup, create-test-cases

# Loading
ace-bundle wfi://verify-test-suite
# Returns: Complete workflow with embedded templates and tool references

# Workflow execution produces
# - Test responsibility maps
# - Performance audit reports
# - E2E test scenarios
```

### Success Criteria

- [ ] 5 workflows moved to `ace-test/handbook/workflow-instructions/`
- [ ] All workflows follow ADR-001 self-containment (embedded templates)
- [ ] Workflows discoverable via `ace-nav wfi://*test*`
- [ ] Tool references (ace-test, ace-bundle) are correct
- [ ] Guide references use `guide://` protocol
- [ ] Template references use `tmpl://` protocol

## Objective

Provide complete testing workflow coverage for agents, enabling them to plan tests before coding, verify test suite health, optimize slow tests, and create E2E scenarios.
