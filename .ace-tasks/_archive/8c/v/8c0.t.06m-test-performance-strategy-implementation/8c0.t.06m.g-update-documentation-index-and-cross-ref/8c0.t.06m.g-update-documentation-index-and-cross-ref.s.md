# Update Documentation Index and Cross-References

## Behavioral Specification

### User Experience
- **Input**: Developer reads ace-test README or docs/tools.md
- **Process**: Finds clear navigation to new testing guides, workflows, skills
- **Output**: Can quickly locate relevant testing documentation

### Expected Behavior

When a developer looks for testing documentation:
1. ace-test README links to new guides under "Testing Guidance" section
2. docs/tools.md includes new testing skills
3. SUMMARY.md in ace-test provides navigation index
4. Cross-references between guides/workflows resolve correctly

### Interface Contract

```markdown
# ace-test/README.md addition
## Testing Guidance

| Guide | Purpose |
|-------|---------|
| [Test Layer Decision](handbook/guides/test-layer-decision.g.md) | Unit vs Integration vs E2E |
| [Test Mocking Patterns](handbook/guides/test-mocking-patterns.g.md) | Stubs, zombies, helpers |
| ... |

## Workflows

Use `/ace:plan-tests` to plan tests before coding.
Use `/ace:verify-test-suite` to audit test health.
...
```

### Success Criteria

- [ ] ace-test README updated with new guides section
- [ ] docs/tools.md updated with testing skills
- [ ] SUMMARY.md navigation index in ace-test/handbook/
- [ ] All internal cross-references use correct protocols
- [ ] No broken links in documentation

## Objective

Ensure discoverability of new testing documentation through updated indexes and navigation, making the research synthesis accessible to developers and agents.

## Technical Approach

### Documentation Structure

**ace-test/handbook/guides/SUMMARY.md** already exists with navigation for existing guides. Need to:
- Add new guides to existing structure
- Organize into logical sections (Core, Performance & Health, Test Planning)

**ace-test/README.md** needs:
- New "Testing Guidance" section
- Links to key guides
- Workflow invocation examples

**docs/tools.md** needs:
- Testing skills added to CLI reference table
- Links to workflow documentation

### Cross-Reference Pattern

All internal references should use protocols:
```markdown
# Within guides:
See [Test Layer Decision](guide://test-layer-decision)

# Within workflows:
Uses template: `tmpl://test-responsibility-map`

# Within README (direct paths work too):
See [handbook/guides/test-layer-decision.g.md](handbook/guides/test-layer-decision.g.md)
```
