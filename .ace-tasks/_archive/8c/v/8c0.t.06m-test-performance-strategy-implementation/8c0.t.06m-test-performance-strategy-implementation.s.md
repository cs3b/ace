# Test Performance Strategy Implementation

## Overview

Implement findings from multi-agent research synthesis on test performance strategy. The research (Task 253 research phase) produced 21 artifacts: 6 guides, 5 workflows, 4 templates, and 6 skills that need to be integrated into the ace-* ecosystem.

### Key Research Findings

1. **The 100ms Rule**: Unit/integration tests >100ms are bugs (hidden I/O leaks)
2. **Stub the Boundary**: Always stub the outermost method that triggers I/O
3. **Fast/Slow Loop Architecture**: Clear separation between fast feedback and E2E validation
4. **Test Responsibility Mapping**: Each behavior → exactly one test layer at lowest level
5. **Planner/Writer Separation**: Distinguish WHAT/WHERE to test from HOW to implement
