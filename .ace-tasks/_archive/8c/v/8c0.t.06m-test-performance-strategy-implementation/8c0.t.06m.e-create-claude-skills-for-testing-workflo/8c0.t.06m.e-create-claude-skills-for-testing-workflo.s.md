# Create Claude Skills for Testing Workflows

## Behavioral Specification

### User Experience
- **Input**: User invokes `/ace:plan-tests` or `/ace:verify-test-suite` in Claude Code
- **Process**: Skill loads corresponding workflow and executes with context
- **Output**: Structured test artifacts following workflow steps

### Expected Behavior

When a user invokes a testing skill:
1. Type `/ace:plan-tests` in Claude Code conversation
2. Skill loads `wfi://plan-tests` workflow
3. Agent executes workflow steps with proper context
4. Produces test responsibility map or other artifacts

The 6 skills provide convenient entry points to testing workflows.

### Interface Contract

```bash
# In Claude Code conversation
/ace:plan-tests
# Loads plan-tests.wf.md and begins Test Planner workflow

/ace:verify-test-suite
# Loads verify-test-suite.wf.md with mode selection (quick/standard/deep)

/ace:optimize-tests
# Loads optimize-tests.wf.md and profiles current package

/ace:test-review
# Loads test-review.wf.md for PR review context

/ace:e2e-sandbox-setup
# Loads e2e-sandbox-setup.wf.md for E2E environment

/ace:test-performance-audit
# Loads audit template and workflow for periodic review
```

### Success Criteria

- [ ] 6 skills created in `.claude/skills/ace_*/`
- [ ] Each skill has `SKILL.md` with correct workflow reference
- [ ] Skills appear in `/help` command listing
- [ ] Skills invoke via `ace-bundle wfi://` protocol
- [ ] Variable handling for skill arguments (e.g., package name)

## Objective

Provide convenient slash command entry points for testing workflows, making them easily discoverable and invocable within Claude Code conversations.

## Technical Approach

### Skill Structure Convention

Based on existing `ace_commit/SKILL.md`:

```yaml
---
name: ace:plan-tests
description: Create a test responsibility map and coverage plan
user-invocable: true
allowed-tools:
  - Bash(ace-bundle:*)
  - Bash(ace-test:*)
  - Read
  - Write
  - Edit
argument-hint: [feature | package | task]
last_modified: 2026-01-31
source: ace-test
---

read and run `ace-bundle wfi://plan-tests`
```

### Skill Naming Convention

| Directory | Slash Command | Workflow |
|-----------|---------------|----------|
| `ace_plan-tests` | `/ace:plan-tests` | `wfi://plan-tests` |
| `ace_verify-test-suite` | `/ace:verify-test-suite` | `wfi://verify-test-suite` |
| `ace_optimize-tests` | `/ace:optimize-tests` | `wfi://optimize-tests` |
| `ace_test-review` | `/ace:test-review` | (guide-based, no workflow) |
| `ace_e2e-sandbox-setup` | `/ace:e2e-sandbox-setup` | `wfi://e2e-sandbox-setup` |
| `ace_test-performance-audit` | `/ace:test-performance-audit` | (template-based) |

### Variable/Argument Handling

Skills can accept arguments via `$1`, `$2`, etc.:
- `$feature` - Feature being tested
- `$package` - Package to analyze
- `$mode` - Audit mode (quick/standard/deep)
