### Key Findings

1. **The 100ms Rule**: Any unit/integration test >100ms is a bug (hidden I/O leak)
2. **Stub the Boundary**: Always stub the outermost method that triggers I/O
3. **Fast/Slow Loop**: Clear separation between fast feedback and E2E validation
4. **Test Responsibility Mapping**: Each behavior → one test layer at lowest level
5. **Zombie Mocks**: Mocks that stub methods no longer called (tests pass but slow)
