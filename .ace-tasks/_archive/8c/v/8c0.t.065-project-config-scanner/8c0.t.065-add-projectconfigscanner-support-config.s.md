# Add ProjectConfigScanner to ace-support-config

## Behavioral Specification

### User Experience

- **Input**: Project root directory and config folder name (default: `.ace`)
- **Process**: Scans project tree downward to find all `.ace` folders
- **Output**: Map of folder locations → config files

### Expected Behavior

As a developer using ace-support-config, I want to discover all `.ace` config folders in my project and query which configs exist where, so that tools like ace-git-commit can group `.ace/**` files correctly.

Current limitation: ConfigFinder only walks UP from a single location. Cannot answer:
- "What `.ace` folders exist in this project?"
- "Give me all `git/commit.yml` configs grouped by location"

### Interface Contract

```ruby
require "ace/support/config"

scanner = Ace::Support::Config::Molecules::ProjectConfigScanner.new(
  project_root: Dir.pwd,  # optional, defaults to cwd
  config_dir: ".ace"      # optional, defaults to .ace
)

# Method 1: Scan all .ace folders
scanner.scan
# => {
#   "." => ["git/commit.yml", "lint/kramdown.yml", ...],
#   "ace-bundle" => ["git/commit.yml"],
#   "ace-docs" => ["git/commit.yml"],
#   ...
# }

# Method 2: Find specific config type across project
scanner.find_all(namespace: "git", filename: "commit")
# => {
#   "." => "/project/.ace/git/commit.yml",
#   "ace-bundle" => "/project/ace-bundle/.ace/git/commit.yml",
#   ...
# }
```

**Error Handling:**
- Non-existent project_root: Return empty hash
- Permission denied on subdirectory: Skip that directory, continue scanning

**Edge Cases:**
- Deeply nested .ace folders: Include all (e.g., `pkg/sub/.ace/`)
- Symlinked directories: Follow symlinks
- Empty .ace folders: Include with empty file list

## Technical Approach

### Architecture Pattern

- **Layer**: Molecule (combines file system traversal with config discovery)
- **Pattern**: Downward traversal scanner (complements ConfigFinder's upward traversal)
- **Integration**: Uses ace-support-fs for path expansion, follows existing molecule patterns

### Technology Stack

- **Dependencies**: ace-support-fs (already available in ace-support-config)
- **Ruby stdlib**: Dir.glob, File.directory?, File.join
- **No new gems required**

### Implementation Strategy

The scanner will:
1. Start from project_root (defaults to cwd)
2. Use Dir.glob with `**/.ace` pattern to find all config directories
3. Skip ignored paths during enumeration
4. For each found `.ace` directory, enumerate config files
5. Return relative paths from project root

**Key Design Decisions:**
- Use Dir.glob for simplicity (handles symlinks by default)
- Return relative paths (consumer can expand if needed)
- No caching (stateless, simple)
- Graceful degradation (return empty hash on errors)

### Technical Risks

- **Risk:** Performance on large monorepos with many directories
  - **Probability:** Low (Dir.glob is efficient)
  - **Impact:** Medium (slow scan)
  - **Mitigation:** SKIP_DIRS excludes common large directories
  - **Rollback:** N/A (stateless operation)

### Integration Risks

- **Risk:** Different behavior across platforms (symlinks on Windows)
  - **Probability:** Low (Ruby handles cross-platform)
  - **Impact:** Low (graceful degradation)
  - **Mitigation:** Follow symlinks by default, skip on errors
  - **Monitoring:** Test on CI matrix
