# Fix ace-review multi-model executor hanging on slow CLI providers

## Objective

Prevent ace-review multi-model execution from hanging indefinitely when a CLI provider (e.g., opencode) becomes unresponsive. Currently, if Ruby's `Timeout.timeout` fails to interrupt a blocking I/O call, `Thread.join` waits forever, blocking the entire batch and eventually producing IOError noise when threads are forcibly terminated.

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-review` with multiple models including potentially slow CLI providers
- **Process**: Multi-model executor runs models concurrently, with a maximum wait time per batch
- **Output**: Review completes within bounded time; timed-out models show failure status; successful models produce reports

### Expected Behavior
- Batch execution completes within `llm_timeout + grace_period` seconds maximum
- Models that exceed timeout are marked as failed with clear error message
- IOError exceptions from killed threads do not leak to stderr
- Successful models in the same batch still produce valid reports
- User sees warning when a thread is forcibly killed

### Interface Contract

```bash
# Normal execution (no change to CLI interface)
ace-review --preset pr

# Expected output when a model times out:
Executing reviews (5 models):
  ⏳ claude:opus: querying...
  ⏳ codex:max: querying...
  ⏳ gemini:pro-latest: querying...
  ⏳ openrouter:minimax: querying...
  ⏳ opencode:zai-coding-plan/glm-4.7: querying...
  ✗ claude:opus: failed (Prompt is too long)
  ✓ openrouter:minimax: complete (40.1s)
  ✓ gemini:pro-latest: complete (111.3s)
  ✓ codex:max: complete (132.8s)
  ⚠ opencode:zai-coding-plan/glm-4.7: killed after 330s timeout
```

**Error Handling:**
- Thread timeout exceeded: Kill thread, mark model as failed, continue batch
- IOError from killed thread: Suppress (do not print to stderr)

**Edge Cases:**
- All models timeout: Batch completes with all failures, synthesis skipped
- Thread killed during response parsing: Result marked as failed with partial data

## Technical Approach

### Current Code (line 80)
```ruby
threads.each(&:join)
```

### Fix
```ruby
# Wait with timeout (LLM timeout + 30s grace period)
join_timeout = @llm_timeout + 30
threads.each do |thread|
  unless thread.join(join_timeout)
    # Thread didn't finish in time, kill it
    thread.kill
    display_progress_warning(thread, join_timeout)
  end
end
```

### Thread Creation (line 73)
```ruby
thread = Thread.new do
  Thread.current.report_on_exception = false  # Suppress IOError noise
  execute_single_model(model, system_prompt, user_prompt, session_dir, batch_results)
end
```
