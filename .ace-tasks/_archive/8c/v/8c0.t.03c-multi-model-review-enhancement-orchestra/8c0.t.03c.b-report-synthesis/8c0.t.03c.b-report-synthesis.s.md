## Objective

Enable ace-review to automatically synthesize multiple LLM model reviews into a unified, actionable consolidated report. When 2+ models complete (from task 126.01 multi-model execution), use LLM-powered analysis to identify consensus findings, unique insights, conflicting views, and prioritized action items - helping users get the best combined insights from diverse AI perspectives.

## Scope of Work

Combine multiple model reviews into a single consolidated report with LLM-powered analysis identifying consensus, conflicts, and prioritized actions.

## User Experience


## Technical Approach

### Architecture Pattern

**Molecule-based Synthesis**:
- Create `ReportSynthesizer` molecule following ATOM architecture
- Input: Array of report file paths
- Output: Consolidated synthesis report with structured sections
- Uses Ace::LLM::QueryInterface for LLM-powered analysis

**Auto-Trigger Pattern**:
- ReviewManager detects when 2+ models complete successfully
- Automatically invokes ReportSynthesizer if config enabled
- Saves synthesis-report.md alongside individual model reports
- Preserves all individual reports for reference

**Backward Compatibility**:
- Single model execution (existing behavior) remains unchanged
- Synthesis only triggers for multi-model execution (2+ models)
- Config option `synthesize_multi_model: true` (default) can be disabled
- CLI flag `--no-synthesize` overrides config

### Technology Stack

**Synthesis LLM**: Ace::LLM::QueryInterface
- Reuse existing ace-llm Ruby API integration
- Same pattern as LlmExecutor.execute
- Default model: Same as primary review model (configurable)
- Fallback: google:gemini-2.5-flash (cost-effective)

**Report Parsing**: File.read + Markdown structure
- Simple file reading (no complex parsing needed)
- Pass full report content to LLM as context
- LLM handles extraction of key findings
- No external markdown parsing dependencies

**Prompt Design**: Handbook-based system prompt
- Create `handbook/prompts/synthesis-system.prompt.md`
- Structured instructions for LLM synthesis task
- Define output format (consensus, unique insights, conflicts, actions)
- Reuse existing prompt composition patterns from ace-review

    ```ruby
    def execute_synthesis(system_prompt, user_prompt, model, session_dir)
      output_file = File.join(session_dir, "synthesis-report.md")

      result = @llm_executor.execute(
        system_prompt: system_prompt,
        user_prompt: user_prompt,
        model: model,
        session_dir: session_dir
      )

      if result[:success]
        # Save synthesis report
        File.write(output_file, result[:response])
        { success: true, output_file: output_file, metadata: result[:metadata] }
      else
        { success: false, error: result[:error] }
      end
    end
    ```

    ```ruby
    def should_synthesize?(results, options)
      # Check if synthesis is enabled in config
      config_enabled = Ace::Review.get('synthesis', 'enabled') != false

      # Check CLI flag override
      cli_disabled = options&.no_synthesize == true

      # Check if we have 2+ successful results
    ```

    ```ruby
    def auto_synthesize(model_results, session_dir, review_data)
      # Collect successful report files
      report_paths = model_results.select { |_, r| r[:success] }
                                  .map { |_, r| r[:output_file] }

      # Determine synthesis model
      synthesis_model = review_data[:synthesis_model] ||
                       review_data[:model] ||
                       "google:gemini-2.5-flash"

      # Invoke synthesizer
      synthesizer = Molecules::ReportSynthesizer.new
      synthesizer.synthesize(
        report_paths: report_paths,
        model: synthesis_model,
        session_dir: session_dir
      )
    rescue => e
      # Log error but don't fail the review
      warn "Warning: Synthesis failed: #{e.message}"
      { success: false, error: e.message }
    end
    ```

## Risk Assessment

### Technical Risks

- **Risk**: LLM synthesis quality varies by model and prompt
  - **Probability**: Medium
  - **Impact**: Medium
  - **Mitigation**: Design clear, structured system prompt with examples; test with multiple models
  - **Rollback**: Users can disable auto-synthesis with --no-synthesize flag

- **Risk**: Large reports exceed LLM context limits
  - **Probability**: Low
  - **Impact**: High
  - **Mitigation**: Document report size limits; add warning if reports too large; consider chunking in future
  - **Rollback**: Synthesis fails gracefully without affecting individual reports

- **Risk**: Synthesis prompt design needs iteration
  - **Probability**: High
  - **Impact**: Low
  - **Mitigation**: Use handbook-based prompt for easy updates; test with real reports; gather feedback
  - **Rollback**: Update prompt without code changes

### Integration Risks

- **Risk**: Auto-trigger breaks existing single-model workflow
  - **Probability**: Low
  - **Impact**: High
  - **Mitigation**: Only trigger for multi-model execution (2+ models); extensive testing
  - **Monitoring**: Full test suite must pass

- **Risk**: Synthesis adds unexpected latency to review workflow
  - **Probability**: Medium
  - **Impact**: Medium
  - **Mitigation**: Synthesis runs after individual reports complete; users can disable; use fast model by default
  - **Monitoring**: Display synthesis progress with timing

### Performance Risks

- **Risk**: Synthesis LLM query adds cost and time
  - **Mitigation**: Use cost-effective model by default (gemini-2.5-flash); configurable
  - **Monitoring**: Display cost/timing in output; track in metadata
  - **Thresholds**: Synthesis should complete < 30s for typical reports
