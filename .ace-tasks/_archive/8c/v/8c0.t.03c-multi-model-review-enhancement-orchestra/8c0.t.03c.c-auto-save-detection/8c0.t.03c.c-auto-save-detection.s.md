# Auto-Save Detection

## Scope

Automatically detect task context from git branch and save final report to appropriate task or release directory (opt-in via config).

## User Experience

```bash
# On branch "121-feature", auto-detects task 121
ace-review --preset pr --auto-execute
# → Auto-saves to: .ace-taskflow/v.0.9.0/tasks/121-.../reviews/

# On main branch, saves to release directory
ace-review --preset pr --auto-execute
# → Auto-saves to: .ace-taskflow/v.0.9.0/reviews/

# Multi-model saves synthesis report (not individual reports)
ace-review --preset pr --model "gemini,gpt-4" --auto-execute
# → Auto-saves: .ace-taskflow/v.0.9.0/tasks/121-.../reviews/20251201-synthesis-pr-review.md

# Explicit task override (highest priority)
ace-review --preset pr --task 114 --auto-execute
# → Saves to task 114, not branch-detected task

# Disable auto-save
ace-review --preset pr --auto-execute --no-auto-save
# → Only saves to session directory
```

## Config Schema

```yaml
# .ace/review/config.yml
defaults:
  auto_save: false  # Disabled by default (opt-in)
  auto_save_path_pattern: "{task|release}/reviews/{timestamp}-{model}-{preset}-review.md"
```

## Directory Structure

```
.ace-taskflow/v.0.9.0/
├── tasks/

│           ├── 20251201-180000-gemini-pr-review.md

└── reviews/                            # Release-level reviews (no task)
```

## Filename Pattern

`{timestamp}-{model-or-synthesis}-{preset}-review.md`
- timestamp: `YYYYMMDD-HHMMSS`
- model: slugified model name (e.g., `gemini-2.5-flash` → `gemini`)
- synthesis: literal `synthesis` for multi-model
- preset: the preset name used

## Technical Approach

### Architecture Pattern Research

**ace-prompt TaskPathResolver Pattern Analysis:**

ace-prompt implements a clean separation between:
- **Atoms**: Pure task resolution logic (`TaskPathResolver.resolve`, `TaskPathResolver.extract_from_branch`)
- **Molecules**: Git I/O operations (`GitBranchReader.current_branch`)

The pattern uses:
1. `GitBranchReader.current_branch` - Executes `git rev-parse --abbrev-ref HEAD` to get branch name
2. `TaskPathResolver.extract_from_branch(branch_name)` - Pure function extracting task ID via regex patterns
3. `TaskPathResolver.resolve(task_id)` - Delegates to `Ace::Taskflow::Organisms::TaskManager.show_task(task_id)`
4. Configurable branch patterns via `.ace/prompt/config.yml` under `task.branch_patterns`

**Default Pattern:** `^(\d+(?:\.\d+)?)-` matches:
- `117-feature` → "117"
- `121.01-archive` → "121.01"
- Does NOT match: `main`, `feature-123` (number not at start)

**Key Insight:** ace-prompt keeps task resolution pure and delegates to ace-taskflow, avoiding duplicate logic.

### Implementation Strategy

**Component Architecture:**

1. **Atoms/TaskAutoDetector** (Pure Function)
   - Extract task ID from branch name (reuse ace-prompt pattern)
   - Configurable regex patterns from config
   - Returns task ID string or nil

2. **Molecules/GitBranchReader** (I/O Operation)
   - Copy from ace-prompt verbatim
   - Returns current branch name or nil
   - Handles detached HEAD gracefully

3. **Molecules/TaskReportSaver Enhancement**
   - Already exists - enhance for multi-model support
   - Add synthesis filename generation
   - Support release directory fallback

4. **Organisms/ReviewManager Integration**
   - Add auto-save orchestration logic
   - Priority: explicit --task > auto-detected > release fallback
   - Respect --no-auto-save flag and config

**Decision Matrix:**

| Scenario | --task flag | auto_save config | --no-auto-save flag | Behavior |
|----------|-------------|------------------|---------------------|----------|
| 1 | Present | Any | Any | Save to explicit task (highest priority) |
| 2 | Absent | false | Any | No auto-save (session only) |
| 3 | Absent | true | Present | No auto-save (flag overrides config) |
| 4 | Absent | true | Absent + branch match | Save to detected task |
| 5 | Absent | true | Absent + no match | Save to release directory |

### Configuration Design

```yaml
# .ace/review/config.yml
defaults:
  auto_save: false  # Opt-in (disabled by default)
  auto_save_branch_patterns:
    - '^(\d+(?:\.\d+)?)-'  # Standard: 121-feature, 121.01-subtask
    - '^feature/(\d+)-'     # Feature branch: feature/123-name
  auto_save_release_fallback: true  # Save to release dir when no task
```

### Risk Assessment

#### Technical Risks

**Risk:** Task resolution fails due to ace-taskflow unavailability
- **Probability:** Low (ace-taskflow is a dependency)
- **Impact:** Medium (feature degradation)
- **Mitigation:** Lazy require with graceful fallback to release directory
- **Rollback:** Auto-save fails silently, session directory still works

**Risk:** Branch name patterns don't match user's git workflow
- **Probability:** Medium (varied team conventions)
- **Impact:** Low (auto-detection fails, manual --task still works)
- **Mitigation:** Configurable patterns in .ace/review/config.yml
- **Monitoring:** User feedback on pattern effectiveness

**Risk:** Multi-model synthesis report conflicts with existing save logic
- **Probability:** Low (synthesis is new feature from 126.02)
- **Impact:** Low (only affects multi-model users)
- **Mitigation:** Clear synthesis vs single-model naming in TaskReportSaver
- **Testing:** Integration tests for both single and multi-model scenarios

#### Integration Risks

**Risk:** Collision with existing --task flag behavior
- **Probability:** Low (explicit flag has highest priority)
- **Impact:** Medium (user confusion)
- **Mitigation:** Clear priority order in documentation
- **Monitoring:** Integration tests verify explicit --task overrides auto-detection

**Risk:** Config changes break existing users
- **Probability:** Very Low (default: false means opt-in)
- **Impact:** Low (no behavior change for existing configs)
- **Mitigation:** Default disabled + clear upgrade path in CHANGELOG
