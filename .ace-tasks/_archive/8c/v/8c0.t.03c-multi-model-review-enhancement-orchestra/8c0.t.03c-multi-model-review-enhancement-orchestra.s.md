# Multi-Model Review Enhancement (Orchestrator)

## Overview

Enhance `ace-review` to support running reviews against multiple LLM models concurrently, synthesizing results, and auto-saving reports to task/release directories.

## Subtasks

| Subtask | Title | Est. | Status | Technical Approach |
|---------|-------|------|--------|-------------------|
| 126.01 | Multi-Model Execution | 8h | pending | Thread-based concurrent LLM queries with progress display |
| 126.02 | Report Synthesis | 6h | pending | LLM-powered consolidation with ReportSynthesizer molecule |
| 126.03 | Auto-Save Detection | 4h | pending | Reuse ace-prompt TaskPathResolver pattern for branch→task |

## Implementation Summary

### 126.01 Multi-Model Execution
- CLI: `--model "gemini,gpt-4,claude"` or multiple `--model` flags
- Preset: `models:` array in YAML configuration
- Concurrency: Ruby Thread-based with configurable limit (default 3)
- Progress: Real-time stderr with emoji indicators (⏳ ✓ ✗)
- Fault isolation: Per-thread exception handling

### 126.02 Report Synthesis
- Command: `ace-review synthesize --session <dir>`
- Auto-trigger: When 2+ models complete (configurable)
- Output: Structured markdown with consensus/unique/conflicts/actions
- LLM: Uses Ace::LLM::QueryInterface for synthesis

### 126.03 Auto-Save Detection
- Detection: Extract task ID from git branch name
- Priority: explicit --task > auto-detected > release fallback
- Config: `auto_save: false` (opt-in by default)
- Pattern: Reuse ace-prompt's TaskPathResolver architecture

## Acceptance Criteria

- [ ] Reviews can be executed against multiple models concurrently
- [ ] Multiple model reports are synthesized into a consolidated report
- [ ] Final reports auto-save to detected task/release directory (opt-in)
- [ ] Each subtask is independently complete with tests and documentation
- [ ] Backward compatibility maintained for single-model workflows
