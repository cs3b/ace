## Objective

Align the package name with the established convention: standalone CLI tools use the `ace-*` pattern, not `ace-support-*`. The `ace-support-*` prefix is reserved for shared infrastructure libraries (core, config, fs). `ace-support-tmux` is a standalone CLI tool with its own binary (`ace-tmux`), so it belongs in the `ace-*` namespace. Per ADR-024, no backward compatibility shims are needed pre-1.0. Follows the precedent of task 264 (ace-coworker → ace-assign).

## Behavioral Specification

### User Experience

- **Input**: No user input required — internal rename with identical CLI interface
- **Process**: Package directory, gemspec, and Ruby module rename. `Ace::Support::Tmux` → `Ace::Tmux`. Binary stays `ace-tmux`. Config stays `.ace/tmux/`.
- **Output**: Identical functionality under correct naming convention

### Interface Contract

```bash
# CLI (unchanged — binary was already ace-tmux)
ace-tmux start [preset]          # Create tmux session from preset
ace-tmux window [preset]         # Add window to running session
ace-tmux list                    # List available presets
ace-tmux --version               # Show version
```
