# Rename ace-config to ace-support-config

## Objective

Rename the `ace-config` gem to `ace-support-config` to follow the established `ace-support-*` naming pattern for infrastructure gems. This is the foundation rename - 17 other gems depend on this.

## Scope of Work

### Directory Rename
- `ace-config/` → `ace-support-config/`

### Namespace Change
- `Ace::Config` → `Ace::Support::Config`
- Version bump: 0.5.1 → 1.0.0 (major due to breaking namespace change)

### lib/ Structure Change
```
ace-config/lib/ace/config.rb       → ace-support-config/lib/ace/support/config.rb
ace-config/lib/ace/config/         → ace-support-config/lib/ace/support/config/
  version.rb                         version.rb
  errors.rb                          errors.rb
  atoms/                             atoms/
  models/                            models/
  molecules/                         molecules/
  organisms/                         organisms/
```

## Risk Assessment

### Technical Risks

- **Risk**: Missing require statement updates
  - **Probability**: Medium
  - **Impact**: High (runtime errors)
  - **Mitigation**: Use ace-search to find all references before committing
  - **Rollback**: Git revert entire commit

- **Risk**: Circular dependency issues during transition
  - **Probability**: Low
  - **Impact**: Medium
  - **Mitigation**: Make all changes in single atomic commit

### Integration Risks

- **Risk**: Breaking ace-support-core functionality
  - **Probability**: Medium
  - **Impact**: High (ace-support-core is widely used)
  - **Mitigation**: Test ace-support-core thoroughly first
  - **Monitoring**: Watch for require errors in CI
