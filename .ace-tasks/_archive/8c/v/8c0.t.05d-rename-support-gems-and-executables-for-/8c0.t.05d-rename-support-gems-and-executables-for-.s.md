# Rename Support Gems and Executables for Naming Consistency

## Behavioral Specification

### User Experience
- **Input**: Decision to standardize gem naming following `ace-support-*` pattern
- **Process**: Sequential execution of subtasks with dependency awareness
- **Output**: Renamed gems with consistent naming, updated dependencies, passing tests

### Expected Behavior

The ace-\* gem ecosystem should follow consistent naming conventions:
- Infrastructure/support gems use `ace-support-*` naming pattern
- Executables can remain with original names for backwards compatibility
- Namespaces follow pattern: `Ace::Support::X` for support gems

After completion:
- `ace-config` becomes `ace-support-config` (namespace: `Ace::Support::Config`)
- `ace-timestamp` becomes `ace-support-timestamp` (namespace: `Ace::Support::Timestamp`, exe: `ace-timestamp`)
- `ace-nav` becomes `ace-support-nav` (namespace: `Ace::Support::Nav`, exe: `ace-nav`)
- `ace-llm-query` executable becomes `ace-llm`

### Interface Contract

```bash
# After completion, these work:
ace-llm --help              # (renamed from ace-llm-query)
ace-timestamp encode        # (executable unchanged, gem renamed)
ace-nav wfi://workflow      # (executable unchanged, gem renamed)

# Ruby requires updated:
require "ace/support/config"
require "ace/support/timestamp"
require "ace/support/nav"
```

### Success Criteria

- [ ] All 4 subtasks completed successfully
- [ ] `bundle install` resolves without errors
- [ ] `ace-test-suite` passes completely
- [ ] All executables work with expected names
- [ ] Documentation updated throughout

## Objective

Establish consistent naming conventions across the ACE gem ecosystem. Support/infrastructure gems should follow the `ace-support-*` pattern established by ace-support-core, ace-support-fs, ace-support-markdown, and ace-support-test-helpers.

## Scope of Work

### Subtasks (Execution Order)

1. **202.01**: Rename ace-llm-query executable to ace-llm (standalone, low risk)
2. **202.02**: Rename ace-config to ace-support-config (foundation, high risk - 17 dependent gems)
3. **202.03**: Rename ace-timestamp to ace-support-timestamp (depends on 202.02)
4. **202.04**: Rename ace-nav to ace-support-nav (depends on 202.02, parallel with 202.03)

### Dependency Graph

```
202.01 (standalone)
         |
    202.02 (foundation)
       /       \
  202.03     202.04  (can run parallel)
```

## Commit Strategy

Each subtask uses **commit per phase** (not single atomic commit):
- Commit after directory structure changes
- Commit after gemspec and Gemfile updates
- Commit after source file updates (requires, namespaces)
- Commit after test updates
- Commit after documentation updates

This improves reviewability while maintaining a clean git history.

## Out of Scope

- Renaming functional gems (ace-context, ace-docs, ace-review, etc.)
- Renaming ace-support-core (already correctly named, though namespace is Ace::Core not Ace::Support::Core)
- Backwards compatibility shims or deprecation warnings
