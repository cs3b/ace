# Fix ace-git-worktree upstream branch configuration

## Description

**BUG FIX**: The `ace-git-worktree` gem fails to consistently set the upstream tracking branch for newly created local branches within a git worktree. Despite existing configuration intended to manage this, the upstream relationship is often not established correctly. This forces both AI agents and human developers to manually configure the upstream branch (e.g., using `git branch --set-upstream-to=origin/branch`) before performing `git pull` or `git push` operations, breaking the intended seamless and autonomous workflow.

## Problem Statement

Current behavior:
- New local branches created in worktrees lack upstream tracking configuration
- Existing configuration to handle upstream setup doesn't work reliably
- Manual intervention required: `git branch --set-upstream-to=origin/branch`
- Breaks autonomous agent workflows and developer productivity

Expected behavior:
- Upstream tracking branch automatically configured when local branch is created
- Git pull/push operations work immediately without manual configuration
- Seamless worktree creation and branch setup

## Acceptance Criteria

- [ ] Upstream tracking branch is automatically set when creating a new local branch in a worktree
- [ ] Configuration correctly applies when checking out existing remote branches into new worktree
- [ ] No manual `git branch --set-upstream-to` intervention required after worktree creation
- [ ] Git pull and push operations work immediately in newly created worktrees
- [ ] Solution respects configuration cascade from `.ace/` directory
- [ ] Comprehensive error handling for scenarios where upstream cannot be set (e.g., remote branch doesn't exist)
- [ ] Unit and integration tests validate upstream configuration across various worktree operations
- [ ] No regressions in existing ace-git-worktree workflows
- [ ] Clear, actionable feedback when upstream configuration fails

## Implementation Notes

### Affected Components
- Identify the Organism or Molecule in `ace-git-worktree` responsible for branch creation/checkout operations
- This is where `git branch --set-upstream-to` integration is needed

### Configuration Integration
- Use `Ace::Core.config.get('ace', 'git-worktree', 'upstream_defaults')` or similar
- Retrieve project-specific or global settings for remote and branch naming conventions
- Ensure configuration cascade is properly respected

### Git Command Integration
- After local branch creation and checkout in worktree:
  - Execute: `git branch --set-upstream-to=<remote>/<branch_name>`
  - Ensure command is deterministic and produces parseable output
- Integrate robustly into existing ace-git-worktree workflow

### Error Handling
- Handle cases where remote branch doesn't exist
- Handle network issues preventing upstream verification
- Provide clear, actionable feedback for all error scenarios

### Testing Requirements
- New unit tests using ace-test-support
- Integration tests covering:
  - Creating new branches from scratch in worktree
  - Checking out existing remote branches into new worktree
  - Configuration cascade application
  - Error scenarios (missing remote branch, network issues)
- Validate upstream is set correctly across all operations

### Considerations
- Backward compatibility with existing workflows
- Configuration cascade correctness (`.ace/` hierarchy)
- No new CLI options expected (bug fix)
- Ensure AI-native principles: deterministic, parseable git output
- Agent autonomy: enable seamless git operations without manual intervention
