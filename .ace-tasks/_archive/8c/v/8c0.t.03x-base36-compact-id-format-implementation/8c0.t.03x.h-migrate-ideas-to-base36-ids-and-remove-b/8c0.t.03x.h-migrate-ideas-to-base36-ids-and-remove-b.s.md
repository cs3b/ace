# Migrate Ideas to Base36 IDs and Remove Backward Compatibility

## Behavioral Specification

### User Experience
- **Input**: Run migration script from task directory
- **Process**: Script scans all ideas/ directories, converts timestamp folders to Base36 IDs
- **Output**: Summary of migrations performed, updated directory structure

### Expected Behavior

#### Migration Discovery
- Scan all `.ace-taskflow/` locations for timestamp-format directories
- Pattern: `YYYYMMDD-HHMMSS-*` (e.g., `20251217-004837-taskflow-add`)
- Recursively find in all locations:
  - `.ace-taskflow/v.0.9.0/ideas/`
  - `.ace-taskflow/_backlog/ideas/`
  - `.ace-taskflow/backlog/ideas/`
  - `.ace-taskflow/_archive/v.*/ideas/`

#### Migration Execution
- Parse timestamp from dirname (e.g., `20251217-004837`)
- Convert to Time object
- Encode to Base36 using `Ace::Timestamp.encode()`
- Construct new dirname: `XXXXXX-title` (e.g., `i50jj3-taskflow-add`)
- Use `git mv` to preserve history
- Track results: migrated count, skipped count, errors

#### Code Cleanup (After Migration)
- Remove all timestamp format detection from codebase
- Simplify to Base36-only format
- Update tests to use Base36 fixtures
- Remove deprecated configuration options

### Interface Contract

```bash
# Migration script usage
ruby .ace-taskflow/v.0.9.0/tasks/149-ci-compact-sortable/149.codemods/migrate-ideas-to-base36.rb

# Dry-run mode (preview changes)
ruby .ace-taskflow/v.0.9.0/tasks/149-ci-compact-sortable/149.codemods/migrate-ideas-to-base36.rb --dry-run
```

**Expected Output:**
```
Found 9 timestamp-format directories to migrate
Migrating: 20251217-004837-taskflow-add → i50jj3-taskflow-add
Migrating: 20251220-212936-taskflow-add → k7j2m1-taskflow-add
...
Summary: 9 migrated, 0 skipped, 0 errors
```

**Error Handling:**
- Invalid timestamp format: Skip directory, log error
- Git mv failure: Stop migration, preserve state for manual intervention
- Target directory exists: Skip, log as already migrated

### Risk Assessment

**Technical Risks**:

- **Risk**: Git mv fails mid-migration leaving partial state
  - **Probability**: Low
  - **Impact**: Medium
  - **Mitigation**: Use dry-run first, work in feature branch
  - **Rollback**: `git reset --hard HEAD`

- **Risk**: Tests fail after fixture updates
  - **Probability**: Medium
  - **Impact**: Low
  - **Mitigation**: Update fixtures in same commit as code changes
  - **Rollback**: Revert commit, update fixtures separately
