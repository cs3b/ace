# Migrate ace-docs to Base36 Compact IDs

## Objective

Update ace-docs to use the new ace-support-timestamp package for generating Base36 compact IDs (6 characters) for analysis session directories and report filenames. Ensure dual-format detection for existing timestamp-based sessions.

## Behavioral Specification

### User Experience

**Before**: Session directories named `.cache/ace-docs/sessions/analyze-consistency-20251023-112940/`
**After**: Session directories named `.cache/ace-docs/sessions/analyze-consistency-000000/` (with year_zero=2000)

### Expected Behavior

**New Sessions**:
- Use Base36 compact ID format (no backward compatibility)
- Use ace-timestamp CompactIdEncoder
- Generate 6-character lowercase IDs
- Already grouped in sessions/ directory (no folder change needed)

**Note**: No backward compatibility for old timestamp format. Task 149.08 will handle migration of existing files.

### Interface Contract

```ruby
# AnalyzeCommand uses new encoder
session_dir = "analyze-consistency-#{Ace::Timestamp::Atoms::CompactIdEncoder.encode(Time.now)}"
# => "analyze-consistency-000000"

# AnalysisReport uses new encoder
filename = "analysis-#{timestamp}.md"
# => "analysis-000000.md"

# ChangeDetector uses new encoder
diff_dir = "diff-#{timestamp}"
# => "diff-000000"
```

### Success Criteria

- [ ] New analysis sessions use 6-character Base36 IDs
- [ ] Config defaults updated
- [ ] Tests pass
