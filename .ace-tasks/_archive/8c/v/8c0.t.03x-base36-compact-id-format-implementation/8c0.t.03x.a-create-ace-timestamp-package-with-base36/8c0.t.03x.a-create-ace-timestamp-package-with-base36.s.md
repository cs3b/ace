# Create ace-timestamp Package with Base36 Compact IDs

## Objective

Create a dedicated `ace-timestamp` Ruby gem that provides configurable Base36 compact ID generation (6 characters) as a replacement for 14-character timestamps.

**Format Design**: Hierarchical field encoding using 6 Base36 digits:
- **Positions 1-2**: Year offset (0-1295) divided by 12 = 108-year coverage from year_zero
- **Position 3**: Day of month (0-35, mapped to actual calendar days 1-31)
- **Position 4**: 40-minute hour block (24 hours × 60 min ÷ 40 = exactly 36 values)
- **Positions 5-6**: Precision within 40-minute window (40×60 = 2400s ÷ 1296 combinations = ~1.85s precision)

Total capacity: 36^6 ≈ 2.17B unique IDs over 108 years.

The package must support configurable year_zero, provide CLI tools for encode/decode/migrate operations, and ensure case-insensitive filesystem compatibility (Base36 lowercase only).

## Behavioral Specification

### User Experience

**Input**: Time objects (Time.now), configuration options (year_zero, alphabet, precision)

**Process**:
- User configures year_zero via `.ace/timestamp/config.yml` (default: 2000, global)
- Per-package override via `.ace/{package}/config.yml` if needed
- User calls `Ace::Timestamp::Atoms::CompactIdEncoder.encode(time)` or uses CLI
- System normalizes to UTC, calculates year offset + elapsed time, encodes as Base36
- User receives 6-character sortable ID

**Output**: 6-character lowercase Base36 ID (e.g., `000000`, `0buzn`, `8hei9`)

### Expected Behavior

**Core Encoding Behavior**:
- Accept Time object and optional config hash (year_zero override)
- Normalize input time to UTC
- Encode hierarchical fields:
  - Year offset = (current_year - year_zero), encode in 2 Base36 digits (÷12 for months)
  - Day of month, encode in 1 Base36 digit (map calendar days 1-31 to 0-30)
  - 40-minute hour block, encode in 1 Base36 digit (36 values cover 24 hours)
  - Seconds within 40-min window, encode in 2 Base36 digits (~1.85s precision)
- Return lowercase string for case-insensitive safety

**Core Decoding Behavior**:
- Accept 6-character Base36 string and optional config hash
- Validate format (length, characters)
- Decode and return Time object in UTC
- Raise descriptive errors for invalid input

**Configuration Behavior**:
- Load year_zero from ace-config cascade (default: 2000)
- Priority: package config > global config > .ace-defaults
- Support 108-year coverage from configured base year
- Allow runtime config override (parameter) for flexibility

**CLI Behavior**:
- `ace-timestamp encode "2025-01-01 00:00:00"` → outputs compact ID
- `ace-timestamp decode "000000"` → outputs timestamp
- `ace-timestamp migrate [package]` → migrates existing files

### Interface Contract

**Ruby API**:

```ruby
# Encode time to compact ID
Ace::Timestamp::Atoms::CompactIdEncoder.encode(time)
# => "000000"

# With config override
Ace::Timestamp::Atoms::CompactIdEncoder.encode(time, year_zero: 2025)
# => "1o0000"

# Decode compact ID to time
Ace::Timestamp::Atoms::CompactIdEncoder.decode("000000")
# => 2000-01-01 00:00:00 UTC

# Validate format
Ace::Timestamp::Atoms::CompactIdEncoder.valid?("000000")
# => true

Ace::Timestamp::Atoms::CompactIdEncoder.valid?("invalid")
# => false
```

**CLI Interface**:

```bash
# Encode timestamp
ace-timestamp encode "2025-01-01 00:00:00"
# Output: 000000

# Decode compact ID
ace-timestamp decode "000000"
# Output: 2000-01-01 00:00:00 UTC

# Migrate package files
ace-timestamp migrate ace-taskflow --dry-run
# Output: Would migrate 25 idea directories

ace-timestamp migrate ace-taskflow
# Output: Migrated 25 idea directories (in-place rename)

ace-timestamp migrate ace-prompt --in-place --pattern "sessions/*"
# Output: Migrated 150 session files

# Show configuration
ace-timestamp config
# Output: year_zero: 2000 (global)
#         alphabet: base36
#         encoding: hierarchical (year/day/hour-block/second-precision)
#         precision: ~1.85s within 40-minute windows
```

**Error Handling**:

```ruby
module Ace
  module Timestamp
    class Error < StandardError; end
    class InvalidIdError < Error; end
    class RangeError < Error; end
    class ConfigError < Error; end
  end
end
```

- `Ace::Timestamp::InvalidIdError`: Non-Base36 characters or wrong length
- `Ace::Timestamp::RangeError`: Year outside configured coverage (e.g., before 2000 with default config)
- `Ace::Timestamp::ConfigError`: Invalid configuration values

**Edge Cases**:

- Year boundary transitions (1999→2000, 2107→2108)
- Leap year dates (Feb 29 handling)
- End of month/day rollovers
- Minimum and maximum representable timestamps

### Success Criteria

- [ ] **6-Character IDs**: All generated IDs are exactly 6 lowercase Base36 characters
- [ ] **Sortability**: Lexicographic string sort matches chronological order
- [ ] **Roundtrip**: `encode(decode(id)) == id` for all valid IDs
- [ ] **Configurable Year Zero**: Default 2000, overridable via config and parameter
- [ ] **CLI Tools**: All encode/decode/migrate commands work correctly
- [ ] **108-Year Coverage**: Covers year_zero to year_zero + 108 years with hierarchical encoding (year/day/hour/second precision)
- [ ] **Case-Insensitive Safe**: Uses only 0-9a-z characters (Base36 lowercase)
- [ ] **UTC Normalization**: All timestamps normalized to UTC internally

### Architectural Decisions

**Decision 1: Year Zero Configuration**
- ✅ **RESOLVED**: Global default (2000) with per-package override via ace-config
- Rationale: Most packages use standard 2000-2107 range; long-lived packages can override
- Implementation: `.ace/timestamp/config.yml` for global, `.ace/{package}/config.yml` for override
- Loading: Use `Ace::Config.create.resolve_namespace("timestamp")` cascade

**Decision 2: Migration Strategy**
- ✅ **RESOLVED**: In-place rename (atomic file operations)
- Rationale: Preserves file metadata, simpler rollback, no disk space duplication
- Implementation: `File.rename(old_path, new_path)` with transaction safety
- CLI: `--dry-run` flag for preview, `--in-place` flag for explicit confirmation

**Decision 3: Backward Compatibility**
- ✅ **RESOLVED**: Dual-format detection (compact ID + timestamp)
- Rationale: Gradual migration, no breaking changes, existing files remain accessible
- Implementation: Format detection heuristics (length: 6 vs 14, pattern matching)
- Detection order: Try compact ID first, fallback to timestamp format

**Decision 4: Error Handling Strategy**
- ✅ **RESOLVED**: Explicit exceptions, no silent fallbacks
- Rationale: Fail-fast with clear errors, predictable behavior
- Exceptions:
  - `InvalidIdError`: Malformed ID (wrong length, invalid characters)
  - `RangeError`: Timestamp outside supported year range
  - `ConfigError`: Invalid configuration values
- No graceful fallback - errors indicate actual problems requiring attention

## Design Notes

**CLI Tool in ace-timestamp Gem**
- `ace-timestamp` includes `exe/ace-timestamp` CLI tool
- This is a user-facing utility gem (not `ace-support-*`) so CLI is appropriate
- Primary use: Migration operations (`ace-timestamp migrate`), encode/decode debugging
- User-facing packages (ace-taskflow, ace-prompt, etc.) use the Ruby API, not the CLI
- Similar to: `ace-git-worktree` which provides worktree management utilities

**Dual-Format Detection**
- Detection order: Try compact ID (6 chars, Base36) first, then timestamp (14 chars with dash)
- Pattern matching:
  - Compact ID: `/^[0-9a-z]{6}$/` - exactly 6 lowercase alphanumeric (Base36)
  - Timestamp: `/^\d{8}-\d{6}$/` - YYYYMMDD-HHMMSS pattern with dash separator
- Return: `:compact`, `:timestamp`, or `:unknown`
- Used by: File loaders, migration tools, compatibility layers

**Detection Implementation Example**:
```ruby
module Ace
  module Timestamp
    module Formats
      COMPACT_REGEX = /^[0-9a-z]{6}$/
      TIMESTAMP_REGEX = /^\d{8}-\d{6}$/

      def self.detect(id)
        case id
        when COMPACT_REGEX then :compact
        when TIMESTAMP_REGEX then :timestamp
        else :unknown
        end
      end
    end
  end
end
```
