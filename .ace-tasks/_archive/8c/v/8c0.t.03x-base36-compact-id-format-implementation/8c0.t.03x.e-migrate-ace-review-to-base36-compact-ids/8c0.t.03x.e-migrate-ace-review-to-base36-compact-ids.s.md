# Migrate ace-review to Base36 Compact IDs

## Objective

Update ace-review to use the new ace-support-timestamp package for generating Base36 compact IDs (6 characters) for review session directories and report filenames. Ensure dual-format detection for existing timestamp-based sessions.

## Behavioral Specification

### User Experience

**Before**: Session directories named `.cache/ace-review/sessions/review-20260104-001424/`
**After**: Session directories named `.cache/ace-review/sessions/review-000000/` (with year_zero=2000)

### Expected Behavior

**New Sessions**:
- Use Base36 compact ID format (no backward compatibility)
- Use ace-timestamp CompactIdEncoder
- Generate 6-character lowercase IDs
- Already grouped in sessions/ directory (no folder change needed)

**Note**: No backward compatibility for old timestamp format. Task 149.08 will handle migration of existing files.

### Interface Contract

```ruby
# ReviewManager uses new encoder
session_dir = "review-#{Ace::Timestamp::Atoms::CompactIdEncoder.encode(Time.now)}"
# => "review-000000"

# TaskReportSaver uses new encoder
filename = "#{timestamp}-synthesis.md"
# => "000000-synthesis.md"
```

### Success Criteria

- [ ] New review sessions use 6-character Base36 IDs
- [ ] Config defaults updated
- [ ] Tests pass

## Scope of Work

### Deliverables

#### Modify

- `lib/ace/review/organisms/review_manager.rb` (lines 804, 830)
  - Add ace-support-timestamp dependency
  - Use CompactIdEncoder.encode() for session directories
