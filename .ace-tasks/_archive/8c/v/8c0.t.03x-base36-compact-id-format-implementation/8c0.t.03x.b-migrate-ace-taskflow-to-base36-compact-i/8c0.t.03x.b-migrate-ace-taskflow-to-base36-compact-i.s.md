# Migrate ace-taskflow to Base36 Compact IDs

## Objective

Update ace-taskflow to use the new ace-support-timestamp package for generating Base36 compact IDs (6 characters) instead of 14-character timestamps for idea and task directories. Ensure dual-format detection so existing timestamp-based directories continue to work.

## Behavioral Specification

### User Experience

**Before**: Ideas created as `20251117-231038-my-idea/`
**After**: Ideas created as `000000-my-idea/` (with year_zero=2000)

**Existing Behavior**: Existing timestamp directories continue to work

### Expected Behavior

**New Ideas/Tasks**:
- Default to Base36 compact ID format
- Use ace-support-timestamp CompactIdEncoder
- Generate 6-character lowercase IDs

**Existing Ideas/Tasks**:
- Dual-format detection recognizes both timestamp and compact ID formats
- No migration required for existing directories
- Read/load operations work transparently

**Configuration**:
- Add `file_naming.id_format: base36` to config defaults
- Support `timestamp` as fallback option
- Allow per-configuration override

### Interface Contract

```ruby
# FileNamer uses new encoder
namer = Ace::Taskflow::Molecules::FileNamer.new(config)
dirname = namer.generate(metadata)
# => "000000-my-idea" (base36 format)

# Dual-format detection in loaders
# Both formats recognized:
# - "20251117-231038-my-idea" (old)
# - "000000-my-idea" (new)
```

**Config Updates**:

```yaml
# .ace-defaults/taskflow/config.yml
file_naming:
  id_format: base36  # New default
  title_max_length: 50
```

### Success Criteria

- [ ] New ideas use 6-character Base36 IDs
- [ ] Existing timestamp directories still load correctly
- [ ] Config defaults updated
- [ ] Tests pass for both formats
- [ ] Documentation updated

## Scope of Work

### Deliverables

  - Add `file_naming.id_format: base36` default

- `lib/ace/taskflow.gemspec`
  - Add dependency: `ace-support-timestamp ~> 0.1`

#### Create

- Tests for compact ID generation
