## Objective

Add deprecation notice to ace-git-diff, keeping it as a thin wrapper for backward compatibility. This allows existing code to continue working while encouraging migration to ace-git.

## Technical Approach

### Architecture Pattern

- **Pattern**: Deprecation Shim
- **Rationale**: Maintains backward compatibility while directing users to new package
- **Integration**: ace-git-diff becomes a dependency wrapper around ace-git
- **Impact**: Minimal - existing code continues to work with deprecation warning

### Technology Stack

- **Dependencies**:
  - ace-git (~> 0.3.0) - the replacement package
  - Remove: ace-support-core (~> 0.11) - inherited from ace-git
  - Remove: thor (~> 1.3) - no longer needed (CLI deprecated)
- **Ruby Version**: >= 3.0.0 (unchanged)

### Implementation Strategy

1. **Update gemspec**: Replace dependencies with ace-git
2. **Modify main entry point**: Add deprecation warning and forward to ace-git
3. **Keep CLI**: Thin wrapper that calls ace-git diff
4. **Update documentation**: README migration guide, CHANGELOG entry

## Risk Assessment

### Technical Risks

- **Risk:** ace-git API differs from ace-git-diff API
  - **Probability:** Low (task 140.01 explicitly migrates all components)
  - **Impact:** High (breaks backward compatibility)
  - **Mitigation:** Verify API compatibility before release
  - **Rollback:** Revert to 0.1.3 if issues found

- **Risk:** ace-git not published when deprecation is released
  - **Probability:** Low (dependencies enforce order)
  - **Impact:** High (gem won't install)
  - **Mitigation:** Task 140.01 and 140.05 are prerequisites
  - **Rollback:** Hold release until ace-git is available

### Integration Risks

- **Risk:** Existing code breaks due to subtle API differences
  - **Probability:** Medium
  - **Impact:** Medium
  - **Mitigation:** Test common usage patterns
  - **Monitoring:** Monitor issues after release
