## Objective

Create unified ace-git package by merging ace-git-diff functionality with new context/PR/branch features. The ace-git package already exists as a workflow-only gem (v0.2.2) with rebase, PR creation, and squash workflows. This task transforms it into a full CLI package while preserving existing workflow functionality.

## Technical Approach

### Architecture Pattern

- **Pattern**: Atomic Design (Atoms/Molecules/Organisms)
- **Rationale**: Consistent with ace-git-diff and other ACE packages
- **Integration**: Extends existing ace-git package structure
- **Impact**: Adds CLI layer on top of existing workflow-only gem

### Technology Stack

- **CLI Framework**: Thor (~> 1.3) - consistent with ace-git-diff
- **Dependencies**:
  - ace-support-core (~> 0.11) - already a dependency
  - thor (~> 1.3) - for CLI
- **Ruby Version**: >= 3.2.0 (aligned with existing ace-git requirement)

### Implementation Strategy

1. **Migrate**: Copy ace-git-diff components into ace-git, renaming module from `Ace::GitDiff` to `Ace::Git`
2. **Consolidate**: Import atoms/molecules from ace-review and ace-context with namespace changes
3. **Extend**: Add new context/PR/branch commands and supporting components
4. **Preserve**: Keep existing ace-git workflows in handbook/ directory

## Risk Assessment

### Technical Risks

- **Risk:** Module namespace conflicts during migration
  - **Probability:** Medium
  - **Impact:** High
  - **Mitigation:** Careful find/replace with verification
  - **Rollback:** Revert to separate packages

- **Risk:** gh CLI dependency failures in tests
  - **Probability:** Medium
  - **Impact:** Medium
  - **Mitigation:** Mock gh CLI in tests
  - **Rollback:** Skip PR-related tests in CI if needed

### Integration Risks

- **Risk:** Breaking existing ace-git workflow users
  - **Probability:** Low
  - **Impact:** High
  - **Mitigation:** Preserve handbook/ directory unchanged
  - **Monitoring:** Check workflow accessibility after changes

### Performance Risks

- **Risk:** Context command too slow with PR fetch
  - **Probability:** Medium
  - **Impact:** Medium
  - **Mitigation:** Timeout on gh CLI calls, async fetch option
  - **Thresholds:** <100ms local, <500ms with PR

## Behavioral Specification

### CLI Interface

```bash
ace-git context              # Repository context (branch, PR, task pattern)
ace-git diff [range]         # Git diff (migrate from ace-git-diff)
ace-git pr [number]          # PR information
ace-git branch               # Branch info only
```

### Output Format (context subcommand)

```markdown
# Repository Context

**Branch:** 140-add-git-context
**Remote:** origin/140-add-git-context (up to date)
**Task Pattern:** 140

## Pull Request

| Field | Value |
|-------|-------|
| Number | #75 |
| Title | Add Git Context |
| Status | open |
| Target | main |
| Author | username |
| URL | https://github.com/owner/repo/pull/75 |
```
