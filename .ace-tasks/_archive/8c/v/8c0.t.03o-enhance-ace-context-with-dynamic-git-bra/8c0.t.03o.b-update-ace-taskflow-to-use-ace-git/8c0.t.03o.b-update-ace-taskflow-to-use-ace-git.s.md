## Objective

Integrate ace-taskflow with ace-git by:
1. Adding an `ace-taskflow context` subcommand that combines git context with task-aware information
2. Replacing the internal `git_executor.rb` molecule with dependency on `ace-git` gem

This task creates a task-aware context layer on top of ace-git's repository context, providing developers and agents with unified task + git state information.

## Technical Approach

### Architecture Pattern

- **Pattern**: Thin wrapper over ace-git with task enrichment
- **Rationale**: Avoid duplicating git functionality; ace-git handles git operations, ace-taskflow adds task awareness
- **Integration**: Use ace-git's `Ace::Git::Organisms::RepoContextLoader` for repository context
- **Impact**: Reduces code duplication, centralizes git operations

### Technology Stack

- **Dependencies**:
  - ace-git (~> 0.3.0) - Repository context, diff, and commit operations
  - ace-support-core (~> 0.10) - Already present
- **No new external dependencies** - ace-git provides all git functionality

### Implementation Strategy

1. **Add Dependency**: Add ace-git to gemspec
2. **Create Context Command**: New command using ace-git + task resolution
3. **Create GitCommitter Molecule**: Thin wrapper for ace-git commit operations
4. **Update IdeaWriter**: Replace GitExecutor usage with GitCommitter
5. **Remove GitExecutor**: Delete obsolete molecule and tests
6. **Update Tests**: Adapt tests for new dependency

## Risk Assessment

### Technical Risks

- **Risk:** ace-git API changes before 140.01 completion
  - **Probability:** Low (140.01 spec is finalized)
  - **Impact:** Medium
  - **Mitigation:** Use interfaces from 140.01 specification
  - **Rollback:** Keep thin wrapper layer for easy adaptation

- **Risk:** GitCommitter Result interface incompatibility
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Create compatible Result struct in GitCommitter
  - **Rollback:** Shim layer to translate between interfaces

### Integration Risks

- **Risk:** IdeaWriter commit behavior changes
  - **Probability:** Low
  - **Impact:** High
  - **Mitigation:** Maintain exact Result struct interface
  - **Monitoring:** Run existing IdeaWriter tests before and after

### Performance Risks

- **Risk:** Context command too slow with full task resolution
  - **Probability:** Low
  - **Impact:** Low
  - **Mitigation:** Task resolution is local filesystem only
  - **Thresholds:** <200ms without PR, <700ms with PR
