# Standardize Symbol Keys in Section Processing

## Objective

Refactor `merge_section_data` and related methods to standardize on symbol keys earlier in the processing pipeline, reducing verbosity and potential bugs from symbol/string key mismatches.

**Source:** google-pro-preview review (item #13)

## Problem

Current code has verbose dual-key checking throughout:

```ruby
# Example from section_processor.rb
processed_files = data[:_processed_files] || data['_processed_files'] || []
processed_commands = data[:_processed_commands] || data['_processed_commands'] || []
processed_diffs = data[:_processed_diffs] || data['_processed_diffs'] || []
```

This pattern:
- Repeated in many methods
- Error-prone (easy to miss one key type)
- Verbose and hard to read
- Indicates unclear data contract

---

## Implementation Plan

### Technical Approach: Layered Normalization with Safe Boundaries

The approach normalizes keys at strategic entry points while maintaining backward compatibility at public API boundaries. This ensures:
1. Internal methods work with consistent symbol keys
2. External callers can still pass string or symbol keys
3. Minimal risk of breaking existing functionality

### Key Design Decisions

1. **Normalization at merge boundaries**: Normalize keys when data enters `merge_section_data` and similar methods, not at outer API boundaries
2. **Deep key symbolization**: Use recursive transform for nested hashes (section data often has nested structures)
3. **Preserve ContentChecker dual-key support**: ContentChecker is a shared atom used by multiple callers; it will retain dual-key support for now, but internal callers will benefit from normalized input
4. **Incremental simplification**: After normalization is proven, simplify accessors in follow-up PR

### Phase 1: Add Symbolization Helper (Non-Breaking)

**File:** `ace-context/lib/ace/context/molecules/section_processor.rb`
**Location:** Private methods section (after line 284)

```ruby
# Recursively converts all string keys to symbols
# @param obj [Hash, Array, Object] object to symbolize
# @return [Hash, Array, Object] object with symbolized keys
def symbolize_keys_deep(obj)
  case obj
  when Hash
    obj.each_with_object({}) do |(key, value), result|
      sym_key = key.respond_to?(:to_sym) ? key.to_sym : key
      result[sym_key] = symbolize_keys_deep(value)
    end
  when Array
    obj.map { |item| symbolize_keys_deep(item) }
  else
    obj
  end
end
```

**Test:** Add to `section_processor_test.rb`:
```ruby
def test_symbolize_keys_deep_converts_string_keys
  input = { 'title' => 'Test', 'nested' => { 'key' => 'value' } }
  result = @processor.send(:symbolize_keys_deep, input)

  assert_equal({ title: 'Test', nested: { key: 'value' } }, result)
end

def test_symbolize_keys_deep_preserves_symbol_keys
  input = { title: 'Test', nested: { key: 'value' } }
  result = @processor.send(:symbolize_keys_deep, input)

  assert_equal({ title: 'Test', nested: { key: 'value' } }, result)
end

def test_symbolize_keys_deep_handles_arrays
  input = { 'items' => [{ 'name' => 'a' }, { 'name' => 'b' }] }
  result = @processor.send(:symbolize_keys_deep, input)

  assert_equal({ items: [{ name: 'a' }, { name: 'b' }] }, result)
end
```

### Phase 3: Update ContextLoader.has_processed_section_content?

**File:** `ace-context/lib/ace/context/organisms/context_loader.rb`
**Method:** `has_processed_section_content?` (line 823)

**Before:**
```ruby
def has_processed_section_content?(context)
  return false unless context.has_sections?

  context.sections.any? do |_name, data|
    processed_files = data[:_processed_files] || data['_processed_files'] || []
    processed_commands = data[:_processed_commands] || data['_processed_commands'] || []
    processed_diffs = data[:_processed_diffs] || data['_processed_diffs'] || []
    processed_files.any? || processed_commands.any? || processed_diffs.any?
  end
end
```

**After:**
```ruby
def has_processed_section_content?(context)
  return false unless context.has_sections?

  context.sections.any? do |_name, data|
    # Section data is normalized to symbol keys by SectionProcessor
    processed_files = data[:_processed_files] || []
    processed_commands = data[:_processed_commands] || []
    processed_diffs = data[:_processed_diffs] || []
    processed_files.any? || processed_commands.any? || processed_diffs.any?
  end
end
```

**Note:** This simplification is safe because SectionProcessor now normalizes keys before storage.

| Test coverage gaps | Low | Medium | Existing tests cover merge behavior |
| Performance overhead | Very Low | Very Low | Transform is O(n) on hash size, hashes are small |
