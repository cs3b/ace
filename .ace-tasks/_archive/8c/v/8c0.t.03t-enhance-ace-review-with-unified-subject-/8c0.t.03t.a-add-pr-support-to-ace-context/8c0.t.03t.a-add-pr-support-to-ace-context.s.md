# Add pr: support to ace-context

## Objective

Add `pr:` as a new source type in ace-context, enabling PR diff content to be loaded alongside files, diffs, and commands.

## Behavioral Specification

### User Experience

```yaml
# ace-context configuration with pr: support
files: ["lib/**/*.rb"]
diffs: ["origin/main..HEAD"]
pr: 123                                    # NEW - PR number
pr: "owner/repo#456"                       # Cross-repo PR
pr: "https://github.com/owner/repo/pull/789"  # Full URL
```

### Interface Contract

```bash
# Load PR diff via ace-context
ace-context -f config.yml  # where config.yml has pr: 123

# Inline via ace-review (after 145.02)
ace-review --subject pr:123
```

### Expected Output Format

PR diff content is wrapped in `<diff>` tags with `source` attribute indicating the PR:

```xml
<diff source="pr:123">
diff --git a/lib/foo.rb b/lib/foo.rb
index abc123..def456 100644
--- a/lib/foo.rb
+++ b/lib/foo.rb
@@ -1,3 +1,4 @@
 class Foo
+  # new method
 end
</diff>
```

For multiple PRs, each gets its own `<diff>` block:

```xml
<diff source="pr:123">
  ... diff content ...
</diff>
<diff source="pr:456">
  ... diff content ...
</diff>
```

### Multiple PR Support

```yaml
# Single PR
pr: 123

# Multiple PRs (array syntax)
pr:
  - 123
  - "owner/repo#456"
  - "https://github.com/org/project/pull/789"
```

### PR Identifier Parsing (Atom - Pure Function)

```ruby
# lib/ace/context/atoms/pr_identifier_parser.rb
module Ace::Context::Atoms
  module PrIdentifierParser
    ParseResult = Data.define(:number, :repo)

    def self.call(input)
      case input.to_s
      when /^(\d+)$/                           # Simple number
        ParseResult.new(number: $1, repo: nil)
      when /^([^\/]+\/[^#]+)#(\d+)$/           # owner/repo#123
        ParseResult.new(number: $2, repo: $1)
      when /github\.com\/([^\/]+\/[^\/]+)\/pull\/(\d+)/  # Full URL
        ParseResult.new(number: $2, repo: $1)
      else
        raise ArgumentError, "Invalid PR identifier: #{input}"
      end
    end
  end
end
```

### GH PR Executor (Molecule - Side Effects)

```ruby
# lib/ace/context/molecules/gh_pr_executor.rb
module Ace::Context::Molecules
  class GhPrExecutor
    def initialize(identifier)
      @parsed = Atoms::PrIdentifierParser.call(identifier)
    end

    def fetch_diff
      stdout, stderr, status = execute_gh_command(build_args)
      raise GhCommandError, format_error(stderr) unless status.success?
      stdout
    end

    protected

    # Override in tests for mocking
    def execute_gh_command(args)
      Open3.capture3(*args)
    end

    private

    def build_args
      args = ["gh", "pr", "diff", @parsed.number]
      args += ["-R", @parsed.repo] if @parsed.repo
      args
    end

    def format_error(stderr)
      case stderr
      when /not found/i
        "PR ##{@parsed.number} not found in repository"
      when /auth/i
        "Not authenticated with GitHub. Run: gh auth login"
      else
        "gh pr diff failed: #{stderr}"
      end
    end
  end

  class GhCommandError < StandardError; end
end
```

## Technical Approach

### Architecture Pattern

**Pattern Selected:** ATOM layering with ace-review molecule reuse

**Rationale:**
- **Atom (PrIdentifierParser):** Pure parsing logic for PR identifiers, no side effects
- **Molecule (GhPrExecutor):** Wraps ace-review's existing GhPrFetcher for safe execution
- **Integration (ContextLoader):** Add PR processing after diffs section (line ~876)

**Integration with Existing Architecture:**
- Follows same pattern as `diffs:` processing in `process_template_config`
- Reuses ace-review's proven PR fetching implementation
- Maintains existing error handling patterns (errors added to context metadata)

**Impact on System Design:**
- No breaking changes - purely additive feature
- ace-context gains dependency on ace-review (already transitively present)
- PR diffs treated as additional diff content (same `<diff>` wrapper as git diffs)

### Technology Stack

**Libraries/Frameworks:**
- ace-review (existing): PrIdentifierParser, GhPrFetcher for PR operations
- Open3 (stdlib): Safe command execution (via ace-review)
- gh CLI (external): GitHub PR diff fetching

**Version Compatibility:**
- Ruby 3.1+ (existing requirement)
- ace-review ~> 0.2.0 (existing dependency)
- gh CLI >= 2.0 (external tool, user-installed)

**Performance Implications:**
- Network-bound: PR fetching requires GitHub API access
- Cached by gh CLI: Repeated requests are fast
- Parallel-safe: Multiple PRs can be fetched concurrently (future optimization)

**Security Considerations:**
- gh CLI handles authentication securely (OAuth token)
- No credential storage in ace-context
- User must authenticate separately (`gh auth login`)

### Integration Code Snippet

Add after line 875 in `context_loader.rb`:

```ruby
# Process PRs
if config['pr'] && !config['pr'].to_s.empty?
  data[:diffs] ||= []
  pr_refs = config['pr'].is_a?(Array) ? config['pr'] : [config['pr']]

  pr_refs.each do |pr_ref|
    begin
      parsed = Atoms::PrIdentifierParser.call(pr_ref)
      executor = Molecules::GhPrExecutor.new(parsed)
      result = executor.fetch_diff

      if result[:success]
        # Determine source label
        source = if parsed.repo
          "pr:#{parsed.repo}##{parsed.number}"
        else
          "pr:#{parsed.number}"
        end

        data[:diffs] << {
          range: source,
          output: result[:diff],
          success: true,
          source: :pr
        }
      else
        data[:diffs] << {
          range: "pr:#{pr_ref}",
          output: "",
          success: false,
          error: result[:error],
          source: :pr
        }
        data[:errors] << "PR fetch failed for '#{pr_ref}': #{result[:error]}"
      end
    rescue StandardError => e
      data[:errors] << "PR processing failed for '#{pr_ref}': #{e.message}"
    end
  end
end
```

## Risk Assessment

### Technical Risks

**Risk:** ace-review dependency not available in ace-context
- **Probability:** Low (ace-review is already a transitive dependency)
- **Impact:** Medium (feature won't work without it)
- **Mitigation:** Verify ace-review is in Gemfile.lock, add explicit dependency if needed
- **Rollback:** Graceful degradation - skip PR processing if ace-review unavailable

**Risk:** gh CLI not installed on user's system
- **Probability:** Medium (users may not have gh installed)
- **Impact:** Low (clear error message, easy to install)
- **Mitigation:** Provide clear error: "GitHub CLI (gh) not installed. Install with: brew install gh"
- **Rollback:** N/A - user error, not code issue

**Risk:** GitHub API rate limiting
- **Probability:** Low (gh CLI handles rate limits gracefully)
- **Impact:** Low (temporary failure, retry works)
- **Mitigation:** Rely on gh CLI's built-in rate limit handling
- **Rollback:** Users can wait or use git diffs instead

### Integration Risks

**Risk:** Conflict with existing diffs: key in config
- **Probability:** Low (pr: is a new key, no conflict)
- **Impact:** Low (both can coexist)
- **Mitigation:** pr: and diffs: are separate config keys, both stored in data[:diffs]
- **Monitoring:** Test combined usage (files + diffs + pr)

**Risk:** Output format incompatibility with existing tooling
- **Probability:** Low (uses same <diff> wrapper as git diffs)
- **Impact:** Low (downstream tools already handle <diff> tags)
- **Mitigation:** Use consistent source attribute pattern
- **Monitoring:** Verify output format in integration tests

### Performance Risks

**Risk:** Network latency for PR fetching
- **Mitigation:** gh CLI caches responses, multiple PRs fetch serially
- **Monitoring:** Log fetch times in debug mode
- **Thresholds:** Warn if single PR fetch >5s, fail if >30s (gh CLI timeout)

**Risk:** Multiple PR fetches blocking context load
- **Mitigation:** Accept serial fetching for v1, optimize to parallel later if needed
- **Monitoring:** Track total context load time
- **Thresholds:** Acceptable if <10s for 3-5 PRs
