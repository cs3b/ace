## Objective

Trace how `ace-taskflow idea create --llm` works and determine if the idea-manager templates in dev-handbook are still needed or should be integrated into the existing capture-idea.wf.md workflow.

## Background

From task 180 discussion: The idea-manager templates in dev-handbook may be obsolete or may contain valuable content that should be integrated with the existing ace-taskflow idea creation workflow.

## Technical Approach

### Architecture Analysis

The investigation revealed:

1. **Current Implementation Location**: `ace-taskflow/lib/ace/taskflow/`
   - `commands/idea_command.rb` - Main CLI command handler
   - `molecules/idea_arg_parser.rb` - Parses `--llm-enhance` and other flags
   - `organisms/idea_writer.rb` - Writes ideas with optional LLM enhancement
   - `molecules/idea_enhancer.rb` - LLM enhancement logic

2. **Template Usage**:
   - **Active template**: `ace-taskflow/templates/idea_enhancement.system.md`
     - Used by `IdeaEnhancer` for LLM system prompt
     - Embeds project context via `ace-context project-base`
     - Produces JSON output with: filename, title, enhanced_description

3. **dev-handbook Templates** (under investigation):
   - `dev-handbook/templates/idea-manager/idea.template.md` - Structured idea template
   - `dev-handbook/templates/idea-manager/system.prompt.md` - LLM system prompt

### Key Finding: Parallel Implementations

There are two separate idea enhancement approaches:

| Aspect | ace-taskflow (active) | dev-handbook (legacy?) |
|--------|----------------------|----------------------|
| System prompt | `idea_enhancement.system.md` | `system.prompt.md` |
| Template | Dynamic (JSON-based) | `idea.template.md` |
| Output format | Frontmatter + markdown | Structured sections |
| Context source | `ace-context project-base` | Inline ACE references |

### Implementation Strategy

This is an **investigation task** (documentation/workflow type) - skip test planning steps.

**Investigation Methodology:**
1. Compare template content side-by-side
2. Check for unique content in dev-handbook templates
3. Determine if dev-handbook adds value beyond ace-taskflow
4. Document recommendation with rationale
