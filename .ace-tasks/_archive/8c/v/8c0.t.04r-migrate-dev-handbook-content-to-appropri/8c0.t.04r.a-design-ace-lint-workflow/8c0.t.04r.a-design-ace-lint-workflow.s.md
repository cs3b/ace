## Objective

Create a comprehensive lint workflow for ace-lint package that enables AI-assisted linting with:
1. A `run-lint.wf.md` workflow with autofix option and report generation
2. A `process-lint-report.wf.md` workflow for creating tasks from lint issues
3. Integration with .claude/commands for `/ace:lint` and `/ace:lint-process`

## Technical Approach

### Architecture Pattern

- **Workflow-first design**: Workflows coordinate CLI tools, don't replace them
- **JSON report format**: Structured output for programmatic processing
- **ace-taskflow integration**: Task creation via existing task infrastructure
- **Protocol registration**: Register `wfi://run-lint` and `wfi://process-lint-report`

### Technology Stack

- **Lint engine**: Existing ace-lint CLI (kramdown, Psych)
- **Report format**: JSON with structured lint result schema
- **Task creation**: ace-taskflow draft-task workflow
- **Context embedding**: Use workflow context embedding pattern

### Implementation Strategy

1. Create handbook structure in ace-lint package
2. Implement run-lint workflow with report generation
3. Implement process-lint-report workflow with task creation
4. Add .claude/commands integration
5. Update ace-nav sources for protocol discovery

## Risk Assessment

### Technical Risks

- **Risk:** JSON report format may not capture all lint details
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Design schema based on existing ValidationError model
  - **Rollback:** Can extend schema without breaking existing reports

- **Risk:** Fix proposals may be inaccurate for complex issues
  - **Probability:** Medium
  - **Impact:** Low
  - **Mitigation:** Mark proposals as suggestions, not automated fixes
  - **Rollback:** Users review proposals before applying

### Integration Risks

- **Risk:** Protocol registration may conflict with existing sources
  - **Probability:** Low
  - **Impact:** Low
  - **Mitigation:** Follow established nav/sources.yml pattern
  - **Monitoring:** Test `ace-nav wfi://` after implementation

## Out of Scope

- Actual linting logic changes (handled by ace-lint atoms/molecules)
- Migrating fix-linting-issue-from.wf.md content (archived with dev-handbook)
