## Objective

Convert all existing agents from `.ag.md` format to the skill command format (`.claude/commands/`). Skills are the preferred pattern: lightweight command files that delegate to workflow instructions via `ace-context wfi://`. This aligns with Claude Code's skill system.

## Background

### Agent vs Skill Pattern Analysis

**Agents (.ag.md):**
- Full markdown documents with extensive inline instructions
- Define expected_params, response formats, workflows inline
- Located in `handbook/agents/` within packages
- Symlinked to `.claude/agents/` for @agent invocation
- Pattern: `type: agent` in frontmatter

**Skills (.claude/commands/):**
- Lightweight command definitions (~5-10 lines)
- Frontmatter: description, allowed-tools, argument-hint, source
- Body delegates to workflow: `read and run ace-context wfi://...`
- Invoked via `/ace:command` in Claude Code
- Pattern: Thin wrapper + workflow instruction

## Technical Approach

### Architecture Pattern

The skill pattern follows the existing `.claude/commands/` structure:

```markdown
---
description: Short description
allowed-tools: Bash, Read
argument-hint: "[args]"
source: ace-package-name
---

read and run `ace-context wfi://workflow-name`
```

For agents without workflow instructions, extract agent logic to new workflow instruction files.

### Conversion Strategy

**Type A - Has Workflow:** Create skill command that delegates to existing workflow
**Type B - No Workflow:** Extract agent logic to new workflow, then create skill command
**Type C - CLI Delegation:** Create skill that delegates to CLI tool directly

### Implementation Strategy

1. **Phase 1:** Convert agents with existing workflows (Type A) - 3 agents
2. **Phase 2:** Create workflow instructions for complex agents (Type B) - 6 agents
3. **Phase 3:** Create simple CLI delegation skills (Type C) - 2 agents
4. **Phase 4:** Clean up old agent files and symlinks

  - **Impact:** High
