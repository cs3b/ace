# Create E2E Tests for ace-git

## Context

ace-git is the git operations toolkit providing real git state inspection, branch operations, and subprocess execution. It wraps common git operations with consistent output formatting. No E2E coverage currently exists.

This is a P2 (medium priority) task — broadens coverage to git infrastructure.

## Behavioral Specification

Create 1 scenario with ~4 test cases using the standalone goal-mode TC format.

### Scenario: TS-GIT-001-git-operations (smoke)

Core git inspection and status operations.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | Git status | `ace-git status` | Output matches actual working tree state |
| TC-002 | Git diff | `ace-git diff` | Diff output for known staged changes |
| TC-003 | Branch info | `ace-git branch` | Current branch display correct |
| TC-004 | Log summary | `ace-git log` | Recent commit output format correct |

- Tags: `[smoke, "use-case:git"]`
- E2E justification: Real git state inspection, branch operations, subprocess execution

## Interface Contract

### Files to Produce

```
ace-git/test/e2e/
└── TS-GIT-001-git-operations/
    ├── scenario.yml           # mode: goal, tags: [smoke, "use-case:git"]
    ├── TC-001.runner.md       # Git status
    ├── TC-001.verify.md       # Verify working tree state
    ├── TC-002.runner.md       # Git diff
    ├── TC-002.verify.md       # Verify diff output
    ├── TC-003.runner.md       # Branch info
    ├── TC-003.verify.md       # Verify branch display
    ├── TC-004.runner.md       # Log summary
    └── TC-004.verify.md       # Verify commit output
```

### CLI Execution

```bash
ace-test-e2e run ace-git --provider claude:haiku
```
