# Create E2E Tests for ace-tmux

## Context

ace-tmux provides tmux session management including session listing, creation, and lifecycle operations. E2E tests require a real tmux environment which adds environment constraints.

This is a P3 (low priority / deferred) task — requires tmux environment, niche use case.

## Behavioral Specification

Create 1 scenario with ~2 test cases using the standalone goal-mode TC format.

### Scenario: TS-TMUX-001-tmux-management (deep)

Tmux session management operations.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | List sessions | `ace-tmux list` | Session listing output |
| TC-002 | Create session | `ace-tmux create --name test` | Session created and visible |

- Tags: `[deep, "use-case:tmux"]`
- E2E justification: Real tmux subprocess, session state management

## Interface Contract

### Files to Produce

```
ace-tmux/test/e2e/
└── TS-TMUX-001-tmux-management/
    ├── scenario.yml           # mode: goal, tags: [deep, "use-case:tmux"]
    ├── TC-001.runner.md       # List sessions
    ├── TC-001.verify.md       # Verify session listing
    ├── TC-002.runner.md       # Create session
    └── TC-002.verify.md       # Verify session created
```

### CLI Execution

```bash
ace-test-e2e run ace-tmux --provider claude:haiku
```
