# Create E2E Tests for ace-docs

## Context

ace-docs provides document management with real filesystem doc discovery and YAML validation. It discovers documentation files, validates YAML frontmatter, and reports coverage status. No E2E coverage currently exists.

This is a P2 (medium priority) task — broadens coverage to documentation infrastructure.

## Behavioral Specification

Create 1 scenario with ~3 test cases using the standalone goal-mode TC format.

### Scenario: TS-DOCS-001-docs-operations (smoke)

Core document management operations.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | Discover docs | `ace-docs discover` | Document listing output |
| TC-002 | Validate docs | `ace-docs validate` | YAML frontmatter validation results |
| TC-003 | Status check | `ace-docs status` | Coverage summary output |

- Tags: `[smoke, "use-case:docs"]`
- E2E justification: Real filesystem doc discovery, YAML frontmatter parsing, cross-file validation

## Interface Contract

### Files to Produce

```
ace-docs/test-e2e/scenarios/
└── TS-DOCS-001-docs-operations/
    ├── scenario.yml           # mode: goal, tags: [smoke, "use-case:docs"]
    ├── TC-001.runner.md       # Discover docs
    ├── TC-001.verify.md       # Verify document listing
    ├── TC-002.runner.md       # Validate docs
    ├── TC-002.verify.md       # Verify validation results
    ├── TC-003.runner.md       # Status check
    └── TC-003.verify.md       # Verify coverage summary
```

### CLI Execution

```bash
ace-test-e2e run ace-docs --provider claude:haiku
```
