# Create E2E Tests for ace-test-runner

## Context

ace-test-runner tests the test infrastructure itself. It performs real test discovery, subprocess execution, and exit code propagation across the monorepo. E2E tests for the test runner validate that the tool correctly discovers test files, delegates to the right executors, and propagates results.

This is a P1 (high priority) task — the test runner is foundational infrastructure and its correctness is critical.

## Behavioral Specification

Create 2 scenarios with ~5 test cases using the standalone goal-mode TC format.

### Scenario: TS-TEST-001-test-execution (smoke)

Core test execution workflows.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | Run package tests | `ace-test {package}` | Exit code 0, test count in output |
| TC-002 | Run specific file | `ace-test {file}` | Only that file's tests run |
| TC-003 | Run test group | `ace-test atoms` | Only atom-layer tests execute |

- Tags: `[smoke, "use-case:testing"]`
- E2E justification: Real subprocess execution, test discovery across filesystem, exit code propagation

### Scenario: TS-TEST-002-suite-execution (deep)

Cross-package suite execution and failure reporting.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | Run full suite | `ace-test-suite` | Cross-package execution, aggregate report |
| TC-002 | Verify failure propagation | Intentionally fail one package | Non-zero exit code reflects failures |

- Tags: `[deep, "use-case:testing"]`
- E2E justification: Cross-package coordination, aggregated subprocess management

## Interface Contract

### Files to Produce

```
ace-test-runner/test-e2e/scenarios/
├── TS-TEST-001-test-execution/
│   ├── scenario.yml           # mode: goal, tags: [smoke, "use-case:testing"]
│   ├── TC-001.runner.md       # Run package tests
│   ├── TC-001.verify.md       # Verify exit code + output
│   ├── TC-002.runner.md       # Run specific file
│   ├── TC-002.verify.md       # Verify scoped execution
│   ├── TC-003.runner.md       # Run test group
│   └── TC-003.verify.md       # Verify group filtering
└── TS-TEST-002-suite-execution/
    ├── scenario.yml           # mode: goal, tags: [deep, "use-case:testing"]
    ├── TC-001.runner.md       # Run full suite
    ├── TC-001.verify.md       # Verify cross-package execution
    ├── TC-002.runner.md       # Trigger failure
    └── TC-002.verify.md       # Verify non-zero exit
```

### CLI Execution

```bash
# Run all ace-test-runner E2E tests
ace-test-e2e run ace-test-runner --provider claude:haiku

# Run only smoke tests
ace-test-e2e run ace-test-runner --tags smoke --provider claude:haiku
```
