# Create E2E Tests for ace-search

## Context

ace-search is a unified search tool with real ripgrep integration and filesystem traversal. It auto-detects search mode (content vs file search) and supports multiple output formats. No E2E coverage currently exists.

This is a P2 (medium priority) task — broadens coverage to search infrastructure.

## Behavioral Specification

Create 1 scenario with ~3 test cases using the standalone goal-mode TC format.

### Scenario: TS-SEARCH-001-search-workflow (smoke)

Core search operations across different modes.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | Content search | `ace-search "pattern"` | Ripgrep output with matched lines |
| TC-002 | File search | `ace-search --files "*.rb"` | File list output |
| TC-003 | Count mode | `ace-search --mode count "pattern"` | Count format output |

- Tags: `[smoke, "use-case:search"]`
- E2E justification: Real ripgrep subprocess, filesystem traversal, auto-detection of search mode

## Interface Contract

### Files to Produce

```
ace-search/test/e2e/
└── TS-SEARCH-001-search-workflow/
    ├── scenario.yml           # mode: goal, tags: [smoke, "use-case:search"]
    ├── TC-001.runner.md       # Content search
    ├── TC-001.verify.md       # Verify matched lines
    ├── TC-002.runner.md       # File search
    ├── TC-002.verify.md       # Verify file list
    ├── TC-003.runner.md       # Count mode search
    └── TC-003.verify.md       # Verify count format
```

### CLI Execution

```bash
ace-test-e2e run ace-search --provider claude:haiku
```
