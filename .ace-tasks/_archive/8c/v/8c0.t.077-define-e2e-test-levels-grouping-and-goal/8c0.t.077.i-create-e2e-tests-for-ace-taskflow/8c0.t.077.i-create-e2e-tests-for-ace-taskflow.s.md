# Create E2E Tests for ace-taskflow

## Context

ace-taskflow is the most complex CLI tool in the project (30+ commands across 5 binaries: `ace-task`, `ace-idea`, `ace-backlog`, `ace-retro`, `ace-assign-prep`). It manages multi-command workflows with filesystem state (`.ace-taskflow/` dirs), inter-command dependencies, and YAML frontmatter parsing. No E2E coverage currently exists.

This is a P1 (high priority) task — ace-taskflow is the backbone of the task management system and unblocks confidence in core tools.

## Behavioral Specification

Create 3 scenarios with ~11 test cases using the standalone goal-mode TC format (`TC-*.runner.md` + `TC-*.verify.md` pairs).

### Scenario: TS-TASK-001-task-lifecycle (smoke)

Multi-step task lifecycle from creation through completion.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | Create task | `ace-task create --title "..." --status draft` | `.ace-taskflow/tasks/` file created with correct frontmatter |
| TC-002 | Start task | `ace-task start {id}` | Status changes to `in-progress` |
| TC-003 | Complete task | `ace-task done {id}` | Status changes to `done` |
| TC-004 | List tasks | `ace-task list` | Output includes created task with correct status |

- Tags: `[smoke, "use-case:taskflow"]`
- E2E justification: Multi-step filesystem workflow with inter-command state dependencies

### Scenario: TS-TASK-002-task-queries (happy-path)

Task query and filtering operations.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | Show task details | `ace-task show {id}` | Full frontmatter and body output |
| TC-002 | Filter by status | `ace-task list --status draft` | Only matching tasks shown |
| TC-003 | Status summary | `ace-task status` | Count output correct |

- Tags: `[happy-path, "use-case:taskflow"]`
- E2E justification: Real filesystem traversal, YAML parsing, and CLI output formatting

### Scenario: TS-IDEA-001-idea-lifecycle (happy-path)

Idea capture and management lifecycle.

| TC | Goal | CLI Command | Verification |
|----|------|-------------|--------------|
| TC-001 | Capture idea | `ace-idea capture` | `.ace-taskflow/ideas/` file created |
| TC-002 | List ideas | `ace-idea list` | Captured idea appears in output |
| TC-003 | Prioritize idea | `ace-idea prioritize` | Priority field updated |

- Tags: `[happy-path, "use-case:taskflow"]`
- E2E justification: Real filesystem state management, separate binary from ace-task
