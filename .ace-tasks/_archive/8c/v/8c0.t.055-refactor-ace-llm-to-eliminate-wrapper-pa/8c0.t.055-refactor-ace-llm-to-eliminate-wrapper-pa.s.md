- **Input**: Users run `ace-llm query "prompt"` or `ace-llm list-providers`

**Commands to Refactor:**

## Objective

Eliminate wrapper pattern in ace-llm by merging wrapper and business logic into single dry-cli command classes.

## Technical Approach

### Architecture Analysis

The wrapper pattern elimination has already been completed:

1. **Query Command** (`lib/ace/llm/commands/query.rb`):
   - Already a complete dry-cli command class (311 lines)
   - Includes all business logic: help display, config summary, query execution, message building, client creation, response formatting
   - Uses `Ace::Core::CLI::DryCli::Base` mixin

2. **ListProviders Command** (`lib/ace/llm/commands/list_providers.rb`):
   - Already a complete dry-cli command class (67 lines)
   - Includes all business logic: provider listing with status display

3. **Legacy Command Classes** (dead code):
   - `QueryCommand` (241 lines) - Never required, not used
   - `ListProvidersCommand` (46 lines) - Never required, not used

4. **CLI Registry** (`lib/ace/llm/cli.rb`):
   - Already registers the dry-cli commands directly
   - No references to legacy `*Command` classes

### Implementation Strategy

This is a cleanup task - simply delete the unused legacy files.
