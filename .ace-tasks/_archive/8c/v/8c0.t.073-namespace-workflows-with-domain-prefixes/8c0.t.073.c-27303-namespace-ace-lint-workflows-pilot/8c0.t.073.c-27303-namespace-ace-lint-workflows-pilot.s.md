# 273.03: Namespace ace-lint Workflows (pilot)

## Behavioral Specification

### Expected Behavior

Pilot migration — move 2 ace-lint workflows into `lint/` subdirectory. This validates the full rename workflow before applying to larger packages.

### Workflow Mapping

| Current | New File | wfi:// URI | Slash Command |
|---------|----------|------------|---------------|
| run-lint.wf.md | lint/run.wf.md | wfi://lint/run | ace:lint-run |
| process-lint-report.wf.md | lint/process-report.wf.md | wfi://lint/process-report | ace:lint-process-report |

### Per-Workflow Checklist

For each workflow:
- [ ] Move file to `ace-lint/handbook/workflow-instructions/lint/` subdirectory
- [ ] Rename file (drop redundant domain words)
- [ ] Update frontmatter `name:` field
- [ ] Rename `.claude/skills/` directory (e.g., `ace_run-lint` → `ace_lint-run`)
- [ ] Update SKILL.md: `name:`, wfi:// reference, `assign.source` if present
- [ ] Search for and update cross-references to old wfi:// URI

### Success Criteria

- [ ] `ace-nav wfi://lint/run` resolves correctly
- [ ] `ace-nav wfi://lint/*` lists both lint workflows
- [ ] `ace-bundle wfi://lint/run` loads content
- [ ] `/ace:lint-run` slash command works
- [ ] No stale references to old names remain
