# 273.14: Update Documentation for Namespaced Workflows

## Behavioral Specification

### Expected Behavior

After all workflow migrations are complete, update project documentation to reflect the new namespaced naming convention.

### Documents to Update

1. **CLAUDE.md** — Update command examples and quick reference table
2. **docs/tools.md** — Update all workflow/skill references
3. **docs/architecture.md** — Mention namespace convention in Workflow section
4. **docs/blueprint.md** — Update directory structure description
5. **ace-integration-claude** — Update skill registry and integration docs
6. **Package READMEs** — Update any workflow references in individual package READMEs

### Key Changes

- All `wfi://old-name` references → `wfi://namespace/new-name`
- All `/ace:old-name` slash command references → `/ace:namespace-action`
- Directory structure examples updated to show subdirectories
- Add namespace browsing documentation (`wfi://task/*`)

### Success Criteria

- [ ] No stale wfi:// references anywhere in documentation
- [ ] No stale slash command references
- [ ] Namespace browsing documented
- [ ] grep for old workflow names across entire codebase returns zero results
- [ ] Global sweep of `wfi://` URIs inside `.wf.md` files confirms all cross-references updated (run: `grep -r 'wfi://' --include='*.wf.md'` and verify no old flat names remain)
