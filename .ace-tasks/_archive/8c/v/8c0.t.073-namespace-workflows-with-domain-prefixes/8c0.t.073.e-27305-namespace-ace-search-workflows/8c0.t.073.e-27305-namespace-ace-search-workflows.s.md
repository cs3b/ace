# 273.05: Namespace ace-search Workflows

## Behavioral Specification

### Expected Behavior

This migration fixes the **shadowing bug**: ace-search's `research.wf.md` and `search.wf.md` are currently invisible via wfi:// because ace-handbook's `research.wf.md` takes priority. After namespacing, both resolve uniquely.

### Workflow Mapping (3 workflows → `search/`)

| Current | New File | wfi:// URI | Slash Command |
|---------|----------|------------|---------------|
| search.wf.md | search/run.wf.md | wfi://search/run | ace:search-run |
| research.wf.md | search/research.wf.md | wfi://search/research | ace:search-research |
| feature-research.wf.md | search/feature-research.wf.md | wfi://search/feature-research | ace:search-feature-research |

### Per-Workflow Checklist

For each workflow:
- [ ] Move file to `ace-search/handbook/workflow-instructions/search/`
- [ ] Update frontmatter `name:` field
- [ ] Rename `.claude/skills/` directory
- [ ] Update SKILL.md: `name:`, wfi:// reference
- [ ] Update cross-references

### Success Criteria

- [ ] `ace-nav wfi://search/*` lists all 3 search workflows
- [ ] `wfi://search/research` and `wfi://handbook/research` resolve to different files (shadowing fixed)
- [ ] All slash commands work
