# 273.02: Verify ProtocolScanner Subdirectory Resolution

## Behavioral Specification

### User Experience
- **Input**: `wfi://namespace/action` URI (e.g., `wfi://lint/run`)
- **Process**: ProtocolScanner resolves URI to a subdirectory file path
- **Output**: Correct file path or clear error

### Expected Behavior

Before any workflow migration, verify that `ace-nav` and `ace-bundle` correctly resolve `wfi://` URIs with `/` separators to subdirectory paths. Create a test fixture with a workflow in a subdirectory and confirm resolution works end-to-end.

Test scenarios:
1. Single resource: `wfi://lint/run` → `handbook/workflow-instructions/lint/run.wf.md`
2. Glob listing: `wfi://lint/*` → lists all files in `lint/` subdirectory
3. Content loading: `ace-bundle wfi://lint/run` → loads workflow content
4. Error: `wfi://nonexistent/foo` → appropriate error message

### Success Criteria

- [ ] `wfi://prefix/name` URI resolution works with subdirectories
- [ ] Glob patterns work within namespaces (`wfi://prefix/*`)
- [ ] `ace-bundle` loads content correctly from subdirectory paths
- [ ] Test coverage added for subdirectory URI resolution
- [ ] If ProtocolScanner needs changes, they are implemented and backward-compatible
- [ ] Tests added to `ace-support-nav/test/` for subdirectory URI resolution

## Scope

**Task scope**: Verify ProtocolScanner supports subdirectory resolution. If changes are needed, implement the fix and add tests — do not file a separate blocker.

- **Package**: ace-support-nav
- **Key file**: `ace-support-nav/lib/ace/support/nav/molecules/protocol_scanner.rb`
- **Config**: `.ace/nav/protocols/wfi.yml`
