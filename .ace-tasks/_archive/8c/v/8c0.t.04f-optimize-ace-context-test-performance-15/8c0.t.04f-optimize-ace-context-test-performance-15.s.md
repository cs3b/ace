# Optimize ace-context test performance (15s to under 5s)

## Behavioral Specification

### User Experience
- **Input**: Developer runs `ace-test ace-context` to execute test suite
- **Process**: Tests execute with properly mocked commands and optimized file I/O
- **Output**: Full test results in <5 seconds (down from 8.78s baseline)

### Expected Behavior
Developers experience faster test execution. CLI tests consistently use mocks instead of spawning subprocesses. Git repository tests use MockGitRepo pattern.

### Interface Contract

```bash
# No changes to public interfaces
# Test structure improvements:
# - CLI integration tests mock subprocess calls
# - Git tests use MockGitRepo from ace-support-test-helpers
```

### Success Criteria

- [x] Test suite runs in <5 seconds (baseline: 8.78s) → **3.62s achieved (58% improvement)**
- [x] All 177 tests pass
- [x] CLI integration tests use mock patterns (no Open3.capture3 subprocess)
- [x] Git repo tests use DiffOrchestrator stubbing pattern

## Objective

Reduce ace-context test execution time by ~45% (from 8.78s to <5s) by replacing subprocess calls with mocks in CLI integration tests and using MockGitRepo for git tests.

## Technical Analysis

### Current Performance Baseline (profiled 2026-01-03)

| Group | Time | Tests | Avg |
|-------|------|-------|-----|
| atoms | 55ms | 25 | 2ms |
| molecules | 100ms | 75 | 1ms |
| organisms | 980ms | 63 | 16ms |
| integration | 7.64s | 14 | 546ms |
| **Total** | **8.78s** | **177** | |

### Root Cause Analysis

**Primary bottleneck: Integration tests (87% of total time)**

1. **CLI subprocess spawning (7+ seconds)**:
   - `cli_embed_source_test.rb`: Uses `Open3.capture3(@ace_context_bin, *args)` - 6 tests
   - `cli_preset_composition_test.rb`: Uses `Open3.capture3(@ace_context_bin, *args)` - 6 tests
   - `cli_api_parity_test.rb`: Uses backticks for CLI execution - 1 test (2.01s alone!)
   - Each subprocess spawn: ~400-800ms overhead

2. **Real git repository creation (4 tests, ~600ms)**:
   - `context_loader_test.rb:722-803`: Uses `system("git init")` + commits
   - Tests: `test_load_inline_yaml_with_flat_diffs_config`, `test_load_inline_yaml_with_nested_context_diffs_config`, `test_load_inline_yaml_flat_and_nested_produce_same_output`

### Existing Infrastructure (already in place)

- `CommandMockHelper` - Intercepts `Ace::Core::Atoms::CommandExecutor.execute`
- `PrMockFixtures` - Mock diff data for PR tests
- `MockGitRepo` from `ace-support-test-helpers` - Fast mock git repos
- `AceTestCase` base class with `with_temp_dir` helper

### Execution Steps

#### Phase 1: Replace Open3.capture3 in CLI Tests (estimated savings: ~4-5s)

- [x] **cli_embed_source_test.rb** - Convert subprocess tests to API tests
  - Replaced `run_ace_context` method with direct `Ace::Context.load_file` calls
  - Pattern: Test CLI flags via API options instead of spawning subprocess
  - **Result:** 6 tests now run in <10ms (was ~2.5s)

- [x] **cli_preset_composition_test.rb** - Convert subprocess tests to API tests
  - Replaced `run_ace_context` with `Ace::Context.load_multiple_presets` and `inspect_config`
  - Test preset composition via API, not CLI subprocess
  - **Result:** 5 tests now run in <15ms (was ~2.4s)

- [x] **cli_api_parity_test.rb** - Kept as E2E validation
  - This test explicitly validates CLI and API produce identical output
  - Accepts ~2s overhead for this single E2E validation test

#### Phase 2: Replace Git Init with DiffOrchestrator Stubbing (estimated savings: ~600ms)

  - `test_load_inline_yaml_flat_and_nested_produce_same_output` simplified (no git needed)
  - **Result:** 3 tests now run in <10ms (was ~600ms)

## Risk Assessment

### Technical Risks

- **Risk:** Mocked tests may miss real subprocess behavior
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Keep ONE real E2E test per integration test file
  - **Rollback:** Revert to subprocess tests if issues found in production

- **Risk:** MockGitRepo may not provide enough fidelity for diff tests
  - **Probability:** Low
  - **Impact:** Low
  - **Mitigation:** Stub at DiffOrchestrator level instead of git level
  - **Rollback:** Keep real git tests if mocking proves insufficient

### Performance Risks

- **Risk:** Target of <5s may not be achievable
  - **Mitigation:** Each phase provides incremental improvement
  - **Monitoring:** Profile after each phase
