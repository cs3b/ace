## Behavioral Specification

### User Experience (CORRECTED - See Course Correction Section Below)

**Original (Incorrect):** ~~Hierarchical prompt library with discovery, protocols, frontmatter~~
**Corrected:** Simple queue-based workflow with single default file

- **Input**: Developers write prompts in a single default file: `.cache/ace-prompt/prompts/the-prompt.md` (or task-specific location)
- **Process**: ace-prompt reads the file, archives it with timestamp, optionally enhances via LLM, and outputs to stdout
- **Output**: Prompt content ready to execute or pipe to other tools

### Expected Behavior (CORRECTED)

Developers need a simple prompt queue workflow that solves the problem of Claude Code's limited in-editor prompt writing capabilities. The tool should:

1. **Single File Workflow**: Read from one default file (`the-prompt.md`), not a named prompt library
2. **Automatic Archiving**: Copy prompt to `archive/TIMESTAMP.md` before reading, keeping history
3. **Previous Link**: Update `_previous.md` symlink to latest archived prompt for easy access
4. **Task Context Support**: Allow task-specific prompts via `--task N` flag
5. **Optional Enhancement**: Optionally enhance prompts via LLM for clarity (disabled by default)
6. **Simple Integration**: Work with `/prompt` slash command in Claude Code

The focus is on **workflow simplicity**: making it easy to write prompts in your editor, automatically archive them, and use them in AI-assisted development - **NOT** building a prompt template library.

### Interface Contract (CORRECTED)

```bash
# CLI Interface - Simple Queue Workflow

# Default command (no arguments) - reads, archives, outputs
ace-prompt
# Behavior:
#   1. Find default prompt: .cache/ace-prompt/prompts/the-prompt.md
#   2. Archive to: .cache/ace-prompt/prompts/archive/YYYYMMDD-HHMMSS.md
#   3. Update symlink: _previous.md → archive/YYYYMMDD-HHMMSS.md
#   4. Read content
#   5. Optionally enhance (if config.enhancement.enabled = true)
#   6. Output to stdout
# Exit code: 0 on success, 1 on error

# Task-specific prompt location
ace-prompt --task 117
# Finds task directory: .ace-taskflow/v.0.9.0/tasks/117-*/
# Uses prompt: {task-path}/prompts/the-prompt.md
# Archives in: {task-path}/prompts/archive/
# Exit code: 0 on success, 1 on error

# Skip enhancement (raw output)
ace-prompt --raw
# Same archive behavior, but skip LLM enhancement step
# Exit code: 0 on success, 1 on error

# Combined flags
ace-prompt --task 117 --raw
# Task-specific prompt, no enhancement

# Integration via slash command
/prompt
# Uses: ace-prompt (default behavior)
```

**Prompt File Format** (Simple - No Frontmatter):
```markdown

1. Security issues
2. Test coverage
3. Code quality

Focus on the authentication changes.
```

**Configuration Format (.ace/prompt/config.yml)** (Simplified):
```yaml
prompt:
  # Default prompt location
  default_dir: .cache/ace-prompt/prompts
  default_file: the-prompt.md
  archive_subdir: archive

  # Optional enhancement (disabled by default)
  enhancement:
    enabled: false  # Opt-in
    model: google:gemini-2.0-flash-lite
    temperature: 0.3
    cache_enabled: true
    cache_path: .cache/ace-prompt/enhanced
    system_prompt_path: handbook/prompts/enhance-system.md
```

**Error Handling:**
- **Prompt file not found**: "Prompt file not found: {path}" (exit 1)
- **Task not found**: "Task 117 not found in .ace-taskflow/" (exit 1)
- **Enhancement failure**: "Enhancement failed: {error}. Outputting raw prompt." (warning, exit 0)
- **Archive write failure**: "Failed to archive prompt: {error}" (warning, continue)

**Edge Cases:**
- **Empty prompt file**: Output empty string (exit 0)
- **No archive directory**: Create directory automatically
- **Enhancement disabled**: Skip enhancement, output raw content
- **Cache miss**: Enhance and cache result
- **Multiple task matches**: Use most recent version directory

### What Was Actually Built vs. What Was Needed

**What Was Built:**
A comprehensive **prompt library management system** with:
- Named prompt discovery and hierarchical search
- Protocol-based discovery (`prompt://`)
- YAML frontmatter parsing for metadata
- Context preset merging with ace-context integration
- Multiple CLI commands (read, enhance, list, info, config)
- Named prompts with discovery across search paths
- Complex configuration system

**What Was Actually Needed:**
A simple **prompt queue workflow tool**:
- Read from ONE default file: `the-prompt.md`
- Archive with timestamp before reading
- Execute/output the content
- Support project/task contexts
- Optional LLM enhancement
- Simple, focused interface

### Root Cause Analysis

The task evolved from a simple workflow tool into a sophisticated library manager by interpreting "prompt management" through the lens of common prompt engineering patterns (templates, discovery, libraries) rather than the actual use case: **a queue-based prompt workflow for AI-assisted development**.

The actual requirement was solving: "Claude Code's editor isn't great, let me write prompts in a file, archive them automatically, and execute them."

### Implementation Approach

**Preset Merging Strategy:**
Following ace-context patterns, implement preset array merging:
1. Load config default_presets: [base, project]
2. Parse prompt frontmatter context: [task, diff]
3. Merge arrays: [base, project, task, diff]
4. Execute: `ace-context base project task diff`
5. Prepend output to prompt content

**Enhancement Strategy:**
Use ace-llm QueryInterface with Haiku:
1. Load enhancement system prompt (configurable via prompt://enhance-system)
2. Build enhancement query: system prompt + original prompt
3. Execute via `Ace::LLM::QueryInterface.query(prompt, model: 'haiku', temperature: 0.3)`
4. Cache result keyed by content hash (MD5)
5. Return enhanced prompt

**Protocol Registration:**
Register with ace-nav in .ace.example/nav/protocols/wfi-sources/ace-prompt.yml:
```yaml
sources:
  - type: directory
    path: .ace/prompts
    protocol: prompt
  - type: directory
    path: .cache/prompts/queue
    protocol: prompt
```

## Risk Analysis

### Technical Risks

**Risk: ace-context preset not found**
- Mitigation: Validate presets exist before calling ace-context
- Fallback: Skip missing presets with warning, continue with available

**Risk: LLM enhancement timeout/failure**
- Mitigation: Set reasonable timeout (30s), implement retry logic
- Fallback: Return original prompt with warning if enhancement fails

**Risk: Cache corruption**
- Mitigation: Validate cached content before returning
- Fallback: Re-enhance if cache invalid, log warning

**Risk: Protocol registration conflicts**
- Mitigation: Check if prompt:// already registered
- Fallback: Document conflict resolution in README

### Rollback Strategy

**Phase 1-2 failures:** Delete gem directory, no impact on other gems
**Phase 3 failures:** ace-context integration isolated, doesn't affect ace-context gem
**Phase 4 failures:** Enhancement optional, can disable in config
**Phase 5-6 failures:** Commands independent, can disable specific commands

### Performance Impact

**Positive impacts:**
- Cached enhancements reduce repeated LLM calls
- Preset merging eliminates manual context copying
- Fast Haiku model provides <2s enhancements

**Monitoring:**
- Log enhancement latency for performance tracking
- Monitor cache hit rate
- Track ace-context execution time

**Thresholds:**
- Enhancement: <2 seconds for typical prompts
- Cache retrieval: <100ms
- Protocol resolution: <10ms
