# Refactor env loading to centralize in ace-core without ENV pollution

## Behavioral Context

**Issue**: Environment variable loading logic was duplicated across ace-llm and dev-tools, with ace-llm having its own cascade discovery implementation that duplicated ace-core functionality.

**Key Behavioral Requirements**:
- Single source of truth for env cascade loading
- No pollution of global ENV unless explicitly requested
- Clean API for other gems to access env vars from cascade
- Performance optimization through caching

## Objective

Centralized all environment variable cascade loading in ace-core, providing a clean API that other ace gems can use without duplicating logic or polluting ENV.

## Scope of Work

- Enhanced ace-core with cascade loading capabilities
- Added public API for clean env access
- Simplified ace-llm to delegate to ace-core
- Removed duplicate cascade logic from ace-llm

### Deliverables

#### Create
- `ace-core/lib/ace/core/molecules/env_loader.rb::load_cascade` - Load .env files without setting ENV
- `ace-core/lib/ace/core.rb::get_env` - Public API for env access
- `ace-core/lib/ace/core.rb::clear_env_cache` - Cache management
- `ace-core/lib/ace/core.rb::cascade_vars` - Private caching method

#### Modify
- `ace-core/lib/ace/core.rb` - Added public API methods
- `ace-core/lib/ace/core/molecules/env_loader.rb` - Added cascade loading
- `ace-llm/lib/ace/llm/atoms/env_reader.rb` - Simplified to use ace-core

#### Delete
- Removed `load_env_cascade_without_setting` from ace-llm (replaced by ace-core)
- Deprecated duplicate cascade logic in ace-llm

## Implementation Summary

### What Was Done

- **Problem Identification**: Recognized that env loading should be handled at ace-core level, not duplicated
- **Investigation**: Found ace-core already had EnvLoader and ConfigDiscovery for cascade support
- **Solution**: Enhanced ace-core with clean API and updated ace-llm to delegate
- **Validation**: Tested that ENV remains clean and ace-git-commit works

### Technical Details

The refactoring involved:

1. **ace-core enhancements**:
   - Added `EnvLoader.load_cascade` to load from cascade without setting ENV
   - Added `Ace::Core.get_env` public API that checks ENV first, then cascade
   - Implemented caching for performance

2. **ace-llm simplification**:
   - Updated `EnvReader.get_api_key` to use `Ace::Core.get_env`
   - Deprecated duplicate cascade methods
   - Removed redundant ConfigDiscovery usage

### Testing/Validation

# Test ace-git-commit still works

**Results**:
- ENV remains clean (nil before and after)
- Keys are loaded from cascade on-demand
