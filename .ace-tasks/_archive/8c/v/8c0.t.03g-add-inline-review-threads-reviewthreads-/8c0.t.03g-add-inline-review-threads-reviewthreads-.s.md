# Add inline review threads (reviewThreads) to PR comment fetching

## Behavioral Specification

### User Experience
- **Input**: PR review via `ace-review --pr <N>` (same as parent task)
- **Process**: ace-review fetches `reviewThreads` in addition to comments/reviews, includes inline code comments with file:line context in dev-feedback report
- **Output**: `review-dev-feedback.md` includes "Inline Code Comments" section with thread IDs, file paths, line numbers, and resolution status

### Expected Behavior

When a developer reviews a PR that has inline code comments (review threads), they see:

1. **Inline Code Comments Section** in `review-dev-feedback.md`:
   - File path and line number for each comment thread
   - Thread ID (PRRT_xxx) for resolution tracking
   - Resolution status (Resolved/Unresolved)
   - All comments in the thread with authors

2. **Integration with Resolution Workflow**:
   - Thread IDs enable the existing `/ace:review-pr` workflow's resolution steps
   - `gh api graphql` mutation can use the thread ID to resolve

### Interface Contract

```bash
# GitHub API: reviewThreads fetched via GraphQL (not available in gh CLI --json)
gh api graphql -f query='
  query($owner: String!, $repo: String!, $number: Int!) {
    repository(owner: $owner, name: $repo) {
      pullRequest(number: $number) {
        reviewThreads(first: 100) {
          nodes { id, isResolved, path, line, comments(first: 50) { nodes { id, body, author { login }, createdAt }}}
        }
      }
    }
  }'

# reviewThreads field structure:
{
  "path": "lib/foo.rb",
  "line": 42,
  "isResolved": false,
  "id": "PRRT_abc123",
  "comments": [
    {"author": {"login": "alice"}, "body": "This needs null checking"},
    {"author": {"login": "bob"}, "body": "Agreed, will fix"}
  ]
}
```

**Expected Output Format:**
```markdown
## Inline Code Comments

### lib/foo.rb:42 (thread: PRRT_abc123) - Unresolved
> @alice: This needs null checking
> @bob: Agreed, will fix

### lib/bar.rb:15 (thread: PRRT_def456) - Resolved
> @alice: Typo in variable name
> @bob: Fixed in abc1234
```

**Error Handling:**
- [No review threads]: Graceful skip, no "Inline Code Comments" section
- [API field missing]: Continue without threads (backward compatible)
