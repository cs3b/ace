# Migrate ace-taskflow to use ace-support-markdown

## Behavioral Specification

### User Experience
- **Input**: Developers use ace-taskflow commands (task creation, status updates, idea management)
- **Process**: ace-taskflow uses ace-support-markdown for all markdown file operations
- **Output**: Tasks and ideas are created/modified safely without corruption risk

### Expected Behavior

ace-taskflow will use the unified ace-support-markdown API instead of direct file operations and scattered markdown handling code. This eliminates the corruption risk demonstrated in tasks 076 and 078.

**Current Problem**: ace-taskflow uses direct `File.write` calls and custom markdown handling in DoctorFixer, TaskManager, and IdeaWriter, which led to file corruption when regex replacements failed.

**Solution**: Replace all direct file operations with ace-support-markdown's DocumentEditor and SafeFileWriter.

### Interface Contract

**Before (current):**
```ruby
# DoctorFixer
def rebuild_content_with_frontmatter(frontmatter, body)
  # Custom YAML serialization + string concatenation
  File.write(file_path, new_content)
end

# TaskManager
File.write(task_path, task_content)

# IdeaWriter
File.write(idea_path, idea_content)
```

**After (migration):**
```ruby
# DoctorFixer
editor = Ace::Support::Markdown::DocumentEditor.new(file_path)
editor.update_frontmatter(frontmatter)
editor.save!(backup: true, validate: true)

# TaskManager
Ace::Support::Markdown::SafeFileWriter.write(
  task_path,
  task_content,
  backup: true,
  validate: true
)

# IdeaWriter
Ace::Support::Markdown::SafeFileWriter.write(
  idea_path,
  idea_content,
  backup: true,
  validate: true
)
```
