# Laser-Focused Review Presets (code-correctness, code-quality, code-polish)

## Overview

Create 3 laser-focused review presets that each target a specific concern layer, replacing the current broad single-pass approach. Wire them into ace-assign's review cycle system so each pass produces precise, actionable feedback.

## Problem Statement

Current review approach:
- The `code` preset loads ALL focus modules (security, performance, tests, docs, architecture, ruby) in a single pass
- ace-assign review cycles differentiate focus only via natural-language instructions while using the same preset
- Every review cycle gets the same broad prompt — the LLM tries to catch everything
- Result: diluted feedback, findings that don't match the cycle's intent

Laser-focused approach:
1. **code-correctness**: Does it work? Does it do what the task requires?
2. **code-quality**: Is it well-structured? Performant? Follows standards?
3. **code-polish**: Can we simplify? Non-blocking refinements only

## Design

### Implementation approach: YAML + Markdown only

No Ruby code changes needed. The existing preset composition system (`prompt://` protocol, section-based composition) already supports everything we need:
- Each preset selects which focus prompt modules to include
- Focus prompts contain explicit IGNORE sections to prevent scope creep
- ace-assign step instructions reference the preset name

### Preset naming: `code-*` prefix

Follow existing naming pattern (`code.yml`, `code-pr.yml`, `code-wip.yml`):
- `code-correctness` — not just `correctness` (avoids ambiguity, groups with code presets)
- `code-quality` — distinct from standalone `performance` or `security` presets
- `code-polish` — clearly a code review phase, not a docs review

### Also: `code-deep` backward compatibility

Existing ace-assign presets reference `code-deep` which doesn't exist. Create it as a composition of `code` with detailed format.
