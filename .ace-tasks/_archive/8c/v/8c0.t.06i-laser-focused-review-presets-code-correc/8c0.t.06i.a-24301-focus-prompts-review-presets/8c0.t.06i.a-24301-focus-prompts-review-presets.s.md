## Objective

Create 3 laser-focused review presets for ace-review with corresponding focus prompts. Each preset composes only the relevant focus modules and includes explicit IGNORE sections to prevent scope creep. Also create `code-deep` preset for backward compatibility.

## Behavioral Specification

### User Experience

```bash
# Phase 1: Does it work?
ace-review --preset code-correctness --pr 123

# Phase 2: Is it well-structured?
ace-review --preset code-quality --pr 123

# Phase 3: Can we simplify?
ace-review --preset code-polish --pr 123

# Backward compat (referenced by existing ace-assign presets)
ace-review --preset code-deep --pr 123
```

Each preset produces focused findings — correctness won't mention style, quality won't suggest rewrites, polish frames everything as non-blocking suggestions.

## Technical Approach

### Architecture Pattern

This task follows the existing ace-review preset composition pattern. Presets are standalone YAML files that reference prompt modules via `prompt://` protocol URIs. The PromptResolver resolves these URIs to markdown files under `ace-review/handbook/prompts/`. No new architecture is introduced.

**Pattern confirmed from existing presets:**
- `security.yml` — standalone preset, references `prompt://focus/quality/security` directly
- `code.yml` — base preset, bundles multiple focus areas with `prompt://base/system` as base
- `code-pr.yml` / `code-wip.yml` — composition presets using `presets: [code]` to inherit

**Key structural observation:** The `security.yml` preset puts `base` at the `instructions` level (not nested under `bundle`), while `code.yml` puts `base` inside `bundle`. The task spec's example YAML puts `base` under `instructions` at top level. We will follow the `security.yml` pattern for the 3 standalone presets since it is the closest analog (single-focus standalone presets).

### Technology Stack

- No new dependencies
- No Ruby code changes
- YAML preset files (parsed by ace-review's existing preset loader)
- Markdown prompt files (resolved by ace-review's PromptResolver via `prompt://` protocol)

### Implementation Strategy

1. Create focus prompt files first (they are referenced by presets)
2. Create standalone phase presets (code-correctness, code-quality, code-polish)
3. Create composition preset (code-deep)
4. Validate all presets load correctly via `--dry-run`
