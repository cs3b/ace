## Technical Approach

### Architecture Pattern
- **Pattern**: Direct dry-cli command class with inline business logic
- **Rationale**: Eliminates unnecessary indirection while maintaining clean separation. Each command file becomes self-contained with both CLI definition (options, arguments, help text) and execution logic
- **Integration**: No change to CLI registry (`cli.rb`) - continues to reference `Commands::Review`, etc.

### Code Structure Analysis

**Current Pattern (Wrapper):**
```ruby
# review.rb - thin wrapper
class Review < Dry::CLI::Command
  def call(**options)
    ReviewCommand.new([], clean_options).execute
  end
end

# review_command.rb - business logic
class ReviewCommand
  def initialize(args, options); end
  def execute; end
end
```

**Target Pattern (Merged):**
```ruby
# review.rb - combined CLI + logic
class Review < Dry::CLI::Command
  def call(**options)
    # business logic directly here
    execute_review(options)
  end

  private
  def execute_review(options); end
  def build_review_options(options); end
  # ... other private methods
end
```
