# 233.03: Chunked Strategy

## Goal

Split large diffs into chunks at file boundaries, review in parallel, synthesize.

## Deliverables

1. `ace-review/lib/ace/review/atoms/diff_boundary_finder.rb` - Parse diff into file blocks
2. `ace-review/lib/ace/review/molecules/strategies/chunked_strategy.rb` - Chunk strategy
3. Unit tests

## Interface

```ruby
# Parse diff into file blocks
blocks = Ace::Review::Atoms::DiffBoundaryFinder.parse(diff_text)
# → [
#     { path: "lib/foo.rb", content: "diff --git...", lines: 45 },
#     { path: "test/foo_test.rb", content: "diff --git...", lines: 30 }
#   ]

# Chunked strategy
strategy = Ace::Review::Molecules::SubjectStrategy.for(:chunked, {
  max_tokens_per_chunk: 100_000,
  include_change_summary: true
})

strategy.prepare(subject, context)
# → [
#     { content: "Summary: Changes in lib/foo.rb, test/foo_test.rb\n\ndiff --git...",
#       metadata: { strategy: :chunked, chunk_index: 0, total_chunks: 2, files: ["lib/foo.rb"] } },
#     { content: "Summary: Changes in lib/foo.rb, test/foo_test.rb\n\ndiff --git...",
#       metadata: { strategy: :chunked, chunk_index: 1, total_chunks: 2, files: ["test/foo_test.rb"] } }
#   ]
```

## Chunking Rules

1. Never split within a file (atomic file boundaries)
2. Each chunk includes summary of changed files (see Summary Size Limits below)
3. Chunks are sized to fit within `max_tokens_per_chunk`
4. If single file exceeds limit, handle according to Overflow Behavior below

### Summary Size Limits

For PRs with many files, the summary must be capped to avoid consuming too many tokens:

- **≤20 files**: List all files with change type (added/modified/deleted)
- **21-100 files**: Group by directory, show counts (e.g., "lib/atoms/: 5 files modified")
- **>100 files**: Show only directory-level summary with totals

### Overflow Behavior

When a single file exceeds the context limit:

1. **Line-based truncation**: Include first N lines that fit, append `[TRUNCATED: {remaining} lines omitted]`
2. **Metadata preserved**: Always include file path, total lines, change type
3. **Warning in output**: Mark chunk metadata with `{ overflow: true, truncated_lines: N }`

```ruby
# Example overflow handling
if file_tokens > max_tokens_per_chunk
  available = max_tokens_per_chunk - HEADER_RESERVE
  truncated = truncate_to_tokens(file_content, available)
  {
    content: truncated + "\n[TRUNCATED: #{remaining_lines} lines omitted]",
    metadata: { overflow: true, truncated_lines: remaining_lines }
  }
end
```
