# 233.02: Subject Strategy Interface + Full Strategy

## Goal

Define the strategy pattern interface and implement the `full` (no-split) strategy.

## Deliverables

1. `ace-review/lib/ace/review/molecules/subject_strategy.rb` - Interface + factory
2. `ace-review/lib/ace/review/molecules/strategies/full_strategy.rb` - Pass-through
3. Update `ReviewManager` to use strategy interface
4. Unit tests

## Interface

```ruby
# Factory method
strategy = Ace::Review::Molecules::SubjectStrategy.for(:full, config)

# Check if strategy can handle the subject
strategy.can_handle?(subject_text, model_context_limit)
# → true/false

# Prepare subject for review (returns array of review units)
strategy.prepare(subject_text, context)
# → [{ content: "full subject...", metadata: {} }]
```

## Context Parameter Type

The `context` parameter is a Hash with the following structure:

```ruby
# @param context [Hash] Review context
# @option context [String] :system_prompt Base system prompt for the reviewer
# @option context [String] :user_prompt User instructions or focus areas
# @option context [String] :model Model identifier (e.g., "google:gemini-2.5-pro")
# @option context [Integer] :model_context_limit Token limit for the model
# @option context [Hash] :preset Full preset configuration
# @option context [Array<String>] :file_list List of files being reviewed (for summaries)

context = {
  system_prompt: "You are a code reviewer...",
  user_prompt: "Focus on security issues",
  model: "google:gemini-2.5-pro",
  model_context_limit: 1_000_000,
  preset: { name: "security", ... },
  file_list: ["lib/foo.rb", "test/foo_test.rb"]
}
```

## Full Strategy Behavior

The `full` strategy is the simplest - it passes the complete subject through unchanged:

```ruby
class FullStrategy
  def can_handle?(subject, model_context)
    TokenEstimator.estimate(subject) < model_context * 0.85
  end

  def prepare(subject, context)
    [{
      content: subject,
      metadata: { strategy: :full, chunk_index: 0, total_chunks: 1 }
    }]
  end
end
```
