# CLI Args Passthrough for LLM Providers

## Objective

Add `--cli-args` option to pass arbitrary flags to CLI-based LLM providers (Claude Code, Codex, etc.), enabling commands like:

```bash
ace-llm claude:sonnet "prompt" --cli-args dangerously-skip-permissions
ace-e2e-test ace-coworker MT-001 --cli-args dangerously-skip-permissions
```

## Behavioral Specification

### User Experience

- **Input**: `--cli-args` flag with space-separated args (auto-prefixed with `--`)
- **Process**: Args passed through to CLI provider subprocess
- **Output**: LLM response with provider flags applied

### Interface Contract

```bash
# Single arg
ace-llm claude:sonnet "hello" --cli-args dangerously-skip-permissions
# Translates to: claude --dangerously-skip-permissions ...

# Multiple args
ace-llm claude:sonnet "hello" --cli-args "dangerously-skip-permissions verbose"
# Translates to: claude --dangerously-skip-permissions --verbose ...
```

### Success Criteria

- [ ] `cli_args` normalized in ace-llm-providers-cli (auto-prefix `--`)
- [ ] All CLI provider clients accept and pass through cli_args
- [ ] `--cli-args` option added to `ace-llm` CLI command
- [ ] Threading through `QueryInterface` and `FallbackOrchestrator`

### Execution Steps

- [x] Add ArgsNormalizer atom for cli_args
  ```ruby
  # lib/ace/llm/providers/cli/atoms/args_normalizer.rb
  module Ace::LLM::Providers::CLI::Atoms
    class ArgsNormalizer
      def normalize_cli_args(cli_args)
        return [] if cli_args.nil? || cli_args.empty?

        args = cli_args.is_a?(String) ? cli_args.split : cli_args
        args.map { |arg| arg.start_with?('--') ? arg : "--#{arg}" }
      end
    end
  end
  ```

- [x] Update ClaudeClient to accept cli_args
  ```ruby
  def query(prompt, system: nil, cli_args: [], **options)
    normalized_args = ArgsNormalizer.new.normalize_cli_args(cli_args)
    cmd = build_command(prompt, system: system, extra_args: normalized_args)
    execute(cmd)
  end
  ```

## Pass-Through Principle

Do NOT validate or filter CLI args. Pass whatever the user provides directly to the provider. If the provider doesn't support a flag, it will fail and surface the error - this is expected behavior.
