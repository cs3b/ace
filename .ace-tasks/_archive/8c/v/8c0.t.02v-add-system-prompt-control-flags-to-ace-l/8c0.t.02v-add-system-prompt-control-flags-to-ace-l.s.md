# Add system prompt control flags to ace-llm-query with Claude provider support

## Problem Statement

When using ace-llm for tools like ace-git-commit that require precise, deterministic prompts, we currently cannot:
1. Fully control the system prompt without Claude Code's defaults interfering
2. Choose between replacing vs appending to default system prompts
3. Actually use system prompts at all with Claude (due to a bug where ClaudeCodeClient uses non-existent `--system` flag)

This prevents us from leveraging Claude's subscription for fast, focused responses with models like Haiku while maintaining exact prompt control.

## User Experience

### What Users Will Be Able To Do

**Replace system prompt entirely** for precise control:
```bash
ace-llm-query claude:haiku --system my-exact-prompt.md --prompt "Generate a commit message"
# Uses ONLY the system prompt from my-exact-prompt.md
```

**Append to default system prompt** when defaults are helpful:
```bash
ace-llm-query claude:haiku --system-append extra-context.md --prompt "Review this code"
# Keeps Claude's helpful defaults AND adds extra-context.md
```

**Use both for maximum flexibility**:
```bash
ace-llm-query claude:haiku --system base.md --system-append context.md --prompt "Task"
# Replaces default with base.md, then appends context.md
```

**ace-git-commit gets fast, deterministic responses**:
```bash
ace-git-commit --provider claude:haiku
# Now works correctly with precise system prompts for <2 second commit generation
```

### Provider Behavior

**Providers that support both flags** (Claude):
- `--system` maps to `--system-prompt` (replace)
- `--system-append` maps to `--append-system-prompt` (append)

**Providers that support only one system prompt** (Anthropic API, OpenAI, Google):
- Both flags concatenate into a single system prompt
- User gets consistent interface regardless of provider limitations
