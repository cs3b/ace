# Grouped Diff Stats Format for ace-git diff

## Behavioral Specification

### User Experience
- **Input**: `ace-git diff main..HEAD --format grouped-stats` — same CLI interface, new format option
- **Process**: Parses `git diff --numstat` output, groups files by package and layer, formats as aligned monospace text
- **Output**: Stats-first, package-grouped file change summary with vertically aligned numbers, suitable for terminal display and PR description embedding

### Expected Behavior

#### Grouped Stats Output

**Current behavior**: `ace-git diff` supports `--format diff` (full diff) and `--format summary` (file list with change counts). Neither groups files by package or aligns stats columns.

**Target behavior**: `ace-git diff --format grouped-stats` produces output like:

```
+209, -386   15 files   total

ace-taskflow/  (+161, -30)

 +94,  -26    2 files   lib/
 +93,  -25     organisms/task_manager.rb
  +1,   -1     version.rb

 +65,   -4    2 files   test/
 +56,   -2     organisms/task_manager_reorganization_test.rb
  +9,   -2     organisms/task_manager_test.rb

  +2,   -2    3 files   handbook/
  +1           workflow-instructions/task/draft-batch.wf.md
  +1,   -1     workflow-instructions/task/draft.wf.md
        -1     workflow-instructions/task/reorganize.wf.md
```

**Formatting rules**:
- Stats in columns 0-9: `%4s, %4s` (4-char right-aligned + val, comma-space, 4-char right-aligned - val)
- When only + or only -: pad the absent side with spaces (total still 10 chars)
- File paths start at fixed column 15 (5 spaces after 10-char stat block)
- Group headers show subtotals + file count between stats and path
- Markdown mode: wrap groups in `<details>` when file count exceeds `collapse_above` threshold

#### Grouping Tiers

1. **Package** — top-level directory (`ace-taskflow/`, `ace-review/`, etc.)
2. **Layer** — subdirectory within package (`lib/`, `test/`, `handbook/`)
3. **Dotfiles** — directories starting with `.` (`.ace-taskflow/`, `.ace/`)
4. **Root** — files not under any package directory ("Project root")

Each group carries aggregate stats (sum of +/- across its files) and file count.

### Interface Contract

**CLI addition**: New value `grouped-stats` for the existing `--format` option in `ace-git diff`.

```bash
# Terminal usage
ace-git diff main..HEAD --format grouped-stats

# With range
ace-git diff HEAD~5 --format grouped-stats

# Markdown mode for PR embedding
ace-git diff main..HEAD --format grouped-stats --output /tmp/stats.md
```

**Error Handling:**
- Empty diff (no changes): Output nothing (same as other formats)
- Binary files in diff: Show file path with `(binary)` instead of +/- stats
- Renamed files: Show as single entry with `→` arrow between old and new path

**Edge Cases:**
- Single-file diff: Still shows total line + package group (no special-casing)
- Files only at root level (no package dirs): All grouped under "Project root"
- Very large diffs (100+ files): Markdown mode collapses groups above threshold

### Success Criteria

- [ ] `ace-git diff main..HEAD --format grouped-stats` produces correctly grouped, vertically-aligned output
- [ ] Numbers right-align in their columns across all files
- [ ] Packages auto-detected from top-level directories without configuration
- [ ] Layer grouping (lib/test/handbook) works within each package
- [ ] Dot-directories grouped separately

## Objective

Provide a reusable, machine-generated file change summary format that replaces manual PR description file listings. The grouped-stats format organizes changes by package and layer with aligned columns, making diffs scannable at a glance. This is the foundation for evidence-based PR descriptions (281.07).

## Scope of Work

- **User Experience Scope**: New `--format grouped-stats` output from existing `ace-git diff` command
- **System Behavior Scope**: Numstat parsing, file grouping, aligned formatting
- **Interface Scope**: CLI format option addition, optional config in `.ace/git/config.yml`

### What to Build

#### 1. New atom: `DiffNumstatParser`

Parse `git diff --numstat` output into per-file stat structs.

Input: `"93\t25\tace-taskflow/lib/ace/taskflow/organisms/task_manager.rb\n1\t1\tace-taskflow/lib/ace/taskflow/version.rb"`

Output: `[{path: "ace-taskflow/lib/.../task_manager.rb", additions: 93, deletions: 25}, ...]`

#### 2. New atom: `FileGrouper`

Group file stat entries by configurable rules (package → layer → dotfile → root). Each group carries aggregate stats and file count.

#### 3. New atom: `GroupedStatsFormatter`

Format grouped file stats as aligned monospace text following the formatting rules above.

#### 4. Configuration model

Via `.ace/git/config.yml` (extend existing diff config):

```yaml
git:
  diff:
    grouped_stats:
      layers: [lib, test, handbook]
      collapse_above: 5
      show_full_tree: collapsible
      dotfile_groups: [.ace-taskflow, .ace]
```

Defaults should work well with zero configuration.

#### 5. Wire into existing CLI

- Add `grouped-stats` to the `--format` option in `diff.rb` CLI command
- Add `format_grouped_stats(result)` method alongside existing `format_summary`
- Generate numstat data in `DiffOrchestrator` when format is `grouped-stats`
