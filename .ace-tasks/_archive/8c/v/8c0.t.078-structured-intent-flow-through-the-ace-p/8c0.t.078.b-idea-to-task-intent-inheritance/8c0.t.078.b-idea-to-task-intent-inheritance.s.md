# Idea-to-Task Intent Inheritance

## Behavioral Specification

### User Experience
- **Input**: User invokes `/ace-task-draft` with an enhanced idea file as source material (no change to invocation)
- **Process**: The drafting agent reads the idea's 3-question sections and maps them into the corresponding task template sections, rather than restating intent from scratch
- **Output**: Draft task file where Objective, Expected Behavior, and Success Criteria trace back to the source idea's intent

### Expected Behavior

When the task draft workflow (step 1: "Gather Behavioral Requirements") processes an enhanced idea that contains the 3-question structure, the drafting agent should:

1. **Map idea intent into task sections**:
   - Idea `## What I Hope to Accomplish` â†’ Task `## Objective` (the *why*)
   - Idea `## What "Complete" Looks Like` â†’ Task `### Expected Behavior` (the observable end state)
   - Idea `## Success Criteria` â†’ Task `### Success Criteria` (verifiable checks, expanded as needed)

2. **Preserve intent, add specificity**: The agent should carry the idea's intent forward and add behavioral detail (interface contracts, edge cases, user experience), not discard the idea's framing and start over.

3. **Handle pre-3Q ideas gracefully**: Older ideas that use the previous format (`Problem / Solution / Implementation Approach / Considerations / Benefits`) should still work â€” the drafting agent treats them as unstructured input and gathers requirements normally (current behavior, no regression).

4. **Act on gap markers**: If the idea contains italicized gap markers (e.g., `_What specific outcome would tell you this idea succeeded?_`), the drafting agent should surface these as questions to the user during step 4 (verification), not silently fill them in.

### Interface Contract

The workflow instruction file (`draft.wf.md`) is the interface. No CLI or API changes.

**Error Handling:**
- Idea has some but not all 3 sections: Map what exists, treat missing sections as gaps to clarify with user
- Idea has no recognizable structure: Fall through to current behavior (gather requirements from user)

**Edge Cases:**
- Multiple source ideas combined into one task: Agent synthesizes intent from all sources, noting which idea contributed which intent
- Idea was not LLM-enhanced (raw text only): Treat as unstructured input, current behavior applies

## Objective

Close the intent gap between idea capture and task drafting. Currently, even well-structured ideas lose their intent when a task is drafted â€” the drafting agent starts from scratch instead of inheriting the idea's stated purpose, end state, and success criteria. This subtask makes the handoff explicit in the workflow instruction.
