# Auto-update Provider Model Lists from models.dev

## Behavioral Specification

### User Experience

- **Input**: Provider config files (`.ace/llm/providers/*.yml`), models.dev API cache
- **Process**: Compare current model lists with models.dev, show diff, optionally update
- **Output**: Diff report with added/removed/deprecated models, alias suggestions

### Expected Behavior

Users can:
1. Preview what models would be added/removed with dry-run (default)
2. Apply updates to provider config files with `--apply`
3. Commit changes automatically with `--commit`
4. Sync specific provider only with `--provider X`
5. See suggested aliases for new models (informational)

### Interface Contract

```bash
# Preview changes (dry-run is default)
ace-llm-models sync-providers

# Apply changes to provider files
ace-llm-models sync-providers --apply

# Apply and commit changes
ace-llm-models sync-providers --apply --commit

# Sync specific provider only
ace-llm-models sync-providers --provider anthropic

# Target specific config directory
ace-llm-models sync-providers --config-dir .ace/llm/providers
```

**Output format:**
```
Syncing provider configs with models.dev...

anthropic:
  + claude-sonnet-4-20250514      (new)
  + claude-opus-4-20250514        (new)
  - claude-3-sonnet-20240229      (deprecated)

  Suggested aliases:
    s4: claude-sonnet-4-20250514
    o4: claude-opus-4-20250514

google:
  + gemini-2.5-pro               (new)
  = gemini-2.0-flash             (unchanged)

Summary: 5 added, 1 removed, 12 unchanged across 3 providers

Run with --apply to update config files.
(Alias suggestions are informational only - add manually if desired)
```

**Error Handling:**
- Provider not in models.dev: Skip with warning
- No changes detected: Report "already up to date"
- Write permission denied: Error with suggestion to use project config
- models.dev cache stale: Auto-sync first (or prompt user)

## Objective

Keep provider model lists in sync with models.dev, reducing manual maintenance and ensuring users have access to latest models.

## Scope of Work

### Deliverables

1. **sync-providers command**: New CLI command in ace-llm-models-dev
2. **Diff generation**: Compare current vs models.dev model lists
3. **YAML preservation**: Update files without destroying structure/comments
4. **Alias suggestions**: Pattern-based suggestions for new models
5. **Commit integration**: Optional auto-commit via ace-git-commit

### Out of Scope

- Auto-applying alias suggestions
- Creating new provider configs (only updates existing)
- Syncing other provider fields (api_key, capabilities, etc.)
