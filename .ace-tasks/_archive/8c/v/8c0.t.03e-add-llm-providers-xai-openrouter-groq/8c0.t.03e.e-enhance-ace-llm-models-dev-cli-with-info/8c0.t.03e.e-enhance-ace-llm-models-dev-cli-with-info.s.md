# Enhance ace-llm-models-dev CLI with info command, search filters, and JSON output

## Behavioral Specification

### User Experience

- **Input**: Model IDs, search queries, filter expressions, output format flags
- **Process**: Query cached models.dev data with filtering and formatting options
- **Output**: Model information in human-readable or JSON format

### Expected Behavior

Users can:
1. Get complete model information with `ace-llm-models info MODEL_ID`
2. Filter search results with `--filter key:value` (repeatable)
3. Get JSON output from `info`, `cost`, and `search` commands via `--json`
4. Search without query to list all models (useful with filters)

### Interface Contract

```bash
# Info command - show complete model details
ace-llm-models info MODEL_ID [--json]

# Cost command - existing + JSON output
ace-llm-models cost MODEL_ID [-i tokens] [-o tokens] [--json]

# Search command - enhanced with filters and optional query
ace-llm-models search [QUERY] [--filter KEY:VALUE]... [--provider X] [--limit N] [--json]

# Filter types supported:
#   provider:openai       - Filter by provider ID
#   reasoning:true/false  - Has reasoning capability
#   tool_call:true/false  - Has tool calling
#   attachment:true/false - Has attachment support
#   open_weights:true/false - Is open weights
#   modality:image        - Supports specific input modality
#   min_context:100000    - Minimum context window
#   max_input_cost:5      - Maximum input cost per million tokens
```

**Human-readable info output:**

**Error Handling:**
- Invalid model ID: "Error: Model 'x' not found in provider 'y'"
- Invalid filter syntax: "Error: Invalid filter format 'x'. Use key:value"
- Unknown filter key: Ignored (allows forward compatibility)

## Objective

Extend ace-llm-models-dev CLI to provide comprehensive model information and flexible querying capabilities, enabling users and AI agents to discover and filter models programmatically.

2. **--json flag**: JSON output for info, cost, and search commands

### Out of Scope

- Provider-level information command
- Caching of filter results
- Complex query syntax beyond key:value
