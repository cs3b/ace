# Fix OpenRouter model sync false positives with canonicalization

## Behavioral Context

**Issue**: The `ace-llm-models providers sync` command was incorrectly reporting OpenRouter models with routing suffixes (`:nitro`, `:floor`, `:online`, etc.) as "removed" because models.dev only contains canonical model names without these suffixes.

**Key Behavioral Requirements**:
- Models like `openai/gpt-4:nitro` should match against `openai/gpt-4` in models.dev
- Only truly missing models should be reported as removed
- Non-OpenRouter providers should be unaffected by canonicalization

## Objective

Add model name canonicalization to strip OpenRouter-specific routing suffixes before comparing against models.dev reference list.

## Scope of Work

- Created ModelNameCanonicalizer Atom for stripping OpenRouter suffixes
- Updated ProviderSyncDiff to use canonical names for comparison
- Added comprehensive tests following ADR-017 flat structure

### Deliverables

#### Create
- `ace-llm-models-dev/lib/ace/llm/models_dev/atoms/model_name_canonicalizer.rb`
- `ace-llm-models-dev/test/atoms/model_name_canonicalizer_test.rb`

#### Modify
- `ace-llm-models-dev/lib/ace/llm/models_dev.rb` - Added require for new atom
- `ace-llm-models-dev/lib/ace/llm/models_dev/molecules/provider_sync_diff.rb` - Use canonical names
- `ace-llm-models-dev/test/molecules/provider_sync_diff_test.rb` - Added canonicalization tests

## Implementation Summary

### What Was Done

- **Problem Identification**: User reported false positives for `:nitro` suffixed models
- **Investigation**: Researched OpenRouter documentation for complete suffix list
- **Solution**: Created canonicalization atom that strips known suffixes for OpenRouter provider
- **Validation**: All 213 tests pass, verified with actual sync command

### Technical Details

OpenRouter suffixes handled:
- Dynamic: `:nitro`, `:floor`, `:online`
- Static: `:free`, `:extended`, `:exacto`, `:thinking`

### Testing/Validation

```bash
ace-test  # 213 tests, 0 failures
ace-llm-models providers sync --since 2025-12-01  # No false positives for :nitro models
```

**Results**: Models like `openai/gpt-oss-120b:nitro` no longer falsely reported as removed
