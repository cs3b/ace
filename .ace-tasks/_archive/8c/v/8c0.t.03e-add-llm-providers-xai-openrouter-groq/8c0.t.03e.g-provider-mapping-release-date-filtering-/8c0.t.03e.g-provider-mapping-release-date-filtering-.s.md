# Provider Mapping & Release Date Filtering for sync-providers

## Behavioral Specification

### User Experience

- **Input**: Provider configs with optional `models_dev_id` and `last_synced` fields
- **Process**: Filter models by release date, map provider names to models.dev IDs
- **Output**: Only new models since last sync, helpful hints for unmapped providers

### Expected Behavior

Users can:
1. Map custom provider names (e.g., `claude` → `anthropic`) via `models_dev_id` field
2. See only models released after `last_synced` date (default behavior)
3. Use `--all` to see all models regardless of date
4. Use `--since DATE` for custom date filtering
5. Get helpful hints when provider names don't match models.dev

### Interface Contract

```bash
# Default: only show models released after last_synced
ace-llm-models sync-providers

# Show ALL models regardless of date (full sync)
ace-llm-models sync-providers --all

# Show models released after specific date
ace-llm-models sync-providers --since 2025-11-01

# Other existing options still work
ace-llm-models sync-providers --provider anthropic --apply --commit
```

**Provider YAML schema update:**
```yaml
name: claude
models_dev_id: anthropic  # Optional - maps to models.dev provider ID
last_synced: 2025-12-01   # Auto-updated by sync-providers --apply
models:
  - claude-sonnet-4-20250514
```

**Output format:**
```
Syncing provider configs with models.dev...
(Showing models released after 2025-12-01)

anthropic:
  + claude-sonnet-4.5-20251205    (new, released: 2025-12-05)

  Last synced: 2025-12-01

claude:
  ⚠ Provider 'claude' not found in models.dev
    Hint: Add 'models_dev_id: anthropic' to provider config

Summary: 1 new model across 1 provider (1 provider skipped)

Run with --apply to update config files.
Run with --all to see all models (not just new releases).
```
