# Add LLM Providers: x.ai, OpenRouter, Groq

## Behavioral Specification

### User Experience
- **Input**: User specifies provider:model (e.g., `xai:grok-3`, `openrouter:openai/gpt-4o`, `groq:llama-3.3-70b-versatile`) or aliases (e.g., `grok`, `or-gpt4`, `groq-fast`)
- **Process**: ace-llm resolves provider, authenticates via environment variable, sends request to provider API
- **Output**: LLM response text with metadata (tokens, model used, finish reason)

### Expected Behavior
Users can query three new LLM providers through ace-llm-query CLI and QueryInterface:
- **x.ai (Grok)**: Access to Grok models via OpenAI-compatible API
- **OpenRouter**: Access to 400+ models from multiple providers via unified API
- **Groq**: Ultra-fast inference for Llama and other models

All providers use OpenAI-compatible chat completion format, making integration consistent with existing providers.

### Interface Contract

```bash
# x.ai (Grok) - uses XAI_API_KEY
ace-llm-query xai:grok-3 "prompt text"
ace-llm-query grok "prompt text"  # alias

# OpenRouter - uses OPENROUTER_API_KEY
ace-llm-query openrouter:openai/gpt-4o "prompt text"
ace-llm-query or-sonnet "prompt text"  # alias

# Groq - uses GROQ_API_KEY
ace-llm-query groq:llama-3.3-70b-versatile "prompt text"
ace-llm-query groq-fast "prompt text"  # alias
```

**Error Handling:**
- Missing API key: Clear error message with env var name
- Invalid model: List available models for provider
- Rate limit: Appropriate retry with fallback support

### Success Criteria

- [ ] All 3 providers work with ace-llm-query CLI
- [ ] All 3 providers work with QueryInterface Ruby API
- [ ] Aliases resolve correctly for each provider
- [ ] Fallback orchestrator works with new providers
- [ ] Tests pass for all new provider clients
- [ ] ace-llm-models-dev gem validates model names locally

## Objective

Expand ace-llm provider ecosystem with three popular OpenAI-compatible providers, giving users access to Grok (x.ai), multi-provider routing (OpenRouter), and ultra-fast inference (Groq).
