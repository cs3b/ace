# CLI DX Restructure with Git-Style Subcommands

## Behavioral Specification

### User Experience

- **Input**: CLI commands with new git-style subcommand structure
- **Process**: Organized command groups (cache, providers, models) with intuitive naming
- **Output**: Clear, consistent responses with JSON support across all commands

### Expected Behavior

Users interact with the CLI through organized command groups:

1. **Cache commands** - Manage local models.dev cache
2. **Provider commands** - Work with provider configurations
3. **Model commands** - Search, query, and price models

Top-level shortcuts preserve convenience for common operations.

### Interface Contract

```bash
# Cache management
ace-llm-models cache sync       # Fetch from models.dev API
ace-llm-models cache status     # Show cache info (fresh, age, counts)
ace-llm-models cache diff       # Show changes since last sync
ace-llm-models cache clear      # Clear local cache

# Provider management
ace-llm-models providers list   # List all providers with model counts
ace-llm-models providers show PROVIDER  # Show provider details
ace-llm-models providers sync   # Sync provider YAML configs with models.dev

# Model operations
ace-llm-models models search [QUERY]    # Search with filters
ace-llm-models models info MODEL_ID     # Show model info (brief by default)
ace-llm-models models info MODEL_ID --full  # Show complete details
ace-llm-models models cost MODEL_ID     # Show pricing

# Top-level shortcuts (convenience)
ace-llm-models search [QUERY]   # → models search
ace-llm-models info MODEL_ID    # → models info
ace-llm-models sync             # → cache sync

# Global options
--json                          # JSON output (all commands)
--help                          # Help for any command
```

**Error Handling:**
- Unknown subcommand: Show help for command group
- Missing required argument: Clear error with usage example
- Cache not available: Suggest `cache sync` first

**Edge Cases:**
- `ace-llm-models cache` without subcommand: Show cache group help
- `ace-llm-models` alone: Show main help with all command groups

## Technical Approach

### Thor Subcommand Pattern

```ruby
# lib/ace/llm/models_dev/cli.rb
module Ace::LLM::ModelsDev
  class CLI < Thor
    desc "cache SUBCOMMAND", "Manage local cache"
    subcommand "cache", CacheCLI

    desc "providers SUBCOMMAND", "Manage providers"
    subcommand "providers", ProvidersCLI

    desc "models SUBCOMMAND", "Work with models"
    subcommand "models", ModelsCLI

    # Top-level shortcuts
    desc "search [QUERY]", "Search models (shortcut for: models search)"
    def search(query = nil)
      ModelsCLI.new.search(query)
    end
  end
end
```

### File Structure

```
lib/ace/llm/models_dev/
  cli.rb                    # Main CLI (register subcommands + shortcuts)
  cli/
    cache_cli.rb            # cache subcommand group
    providers_cli.rb        # providers subcommand group
    models_cli.rb           # models subcommand group
```
