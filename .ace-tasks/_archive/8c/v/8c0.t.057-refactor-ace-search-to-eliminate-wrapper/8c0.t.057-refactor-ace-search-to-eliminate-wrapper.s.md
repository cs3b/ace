### User Experience

## Objective

Eliminate wrapper pattern in ace-search by merging wrapper and business logic into single dry-cli command class. The current implementation has two files:
1. `search.rb` - dry-cli wrapper that defines options and delegates to SearchCommand
2. `search_command.rb` - Business logic class that handles execution

This refactoring merges both into a single cohesive command class following the dry-cli pattern.

## Technical Approach

### Architecture Pattern
- **Pattern**: Single-class dry-cli command (no delegation)
- **Rationale**: Eliminates unnecessary indirection while maintaining testability
- **Integration**: Follows established ace-lint pattern (`Lint` class + `LintCommand` for complex logic)
- **Impact**: Simplifies command structure, reduces file count

### Technology Stack
- **Framework**: dry-cli (already in use)
- **Dependencies**: No new dependencies required
- **Version compatibility**: Ruby 3.0+, dry-cli 1.0+

### Implementation Strategy

The refactoring moves all business logic from `SearchCommand` into the dry-cli `Search` class `call` method:

1. Keep option/argument definitions in `search.rb`
2. Move private methods from `search_command.rb` into `search.rb`
3. Inline the `execute` logic into `call` method
4. Update require statements in `cli.rb`
5. Delete `search_command.rb`
6. Update tests
