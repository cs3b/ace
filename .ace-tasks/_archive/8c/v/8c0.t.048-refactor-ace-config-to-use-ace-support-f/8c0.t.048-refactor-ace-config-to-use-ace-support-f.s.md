# Refactor ace-config to use ace-support-fs directly

## Behavioral Context

**Issue**: After extracting ace-support-fs gem (task 159), ace-config still had wrapper classes (PathExpander, ProjectRootFinder) that duplicated code instead of delegating directly to ace-support-fs.

**Key Behavioral Requirements**:
- ace-config should use ace-support-fs classes directly, not through wrappers
- No external packages depend on ace-config's PathExpander/ProjectRootFinder
- All tests must pass after refactoring

## Objective

Remove unnecessary wrapper classes from ace-config and use ace-support-fs directly, reducing code duplication.

## Scope of Work

- Deleted wrapper classes that duplicated ace-support-fs functionality
- Updated all internal references to use ace-support-fs classes directly
- Removed redundant tests (covered by ace-support-fs tests)
- Added PathError alias for backward compatibility

### Deliverables

#### Delete
- `ace-config/lib/ace/config/atoms/path_expander.rb` - wrapper class
- `ace-config/lib/ace/config/molecules/project_root_finder.rb` - wrapper class
- `ace-config/test/atoms/path_expander_test.rb` - redundant tests
- `ace-config/test/molecules/project_root_finder_test.rb` - redundant tests

#### Modify
- `ace-config/lib/ace/config.rb` - use ace-support-fs directly
- `ace-config/lib/ace/config/errors.rb` - add PathError alias
- `ace-config/lib/ace/config/molecules/directory_traverser.rb` - use ace-support-fs
- `ace-config/lib/ace/config/molecules/config_finder.rb` - use ace-support-fs
- `ace-config/test/test_helper.rb` - update cache clearing
- `ace-config/test/integration/config_cascade_test.rb` - update references
- `ace-config/test/integration/performance_test.rb` - update references

## Implementation Summary

### What Was Done

- **Investigation**: Explored ace-config to find that PathExpander and ProjectRootFinder were only used internally, with no external dependencies
- **Solution**: Deleted wrapper classes and updated all internal code to use `Ace::Support::Fs::Atoms::PathExpander` and `Ace::Support::Fs::Molecules::ProjectRootFinder` directly
- **Validation**: All 130 ace-config tests pass, full monorepo suite (4522 tests) passes

### Technical Details

**Before**: ace-config had ~311 lines of wrapper code duplicating ace-support-fs
**After**: ace-config uses ace-support-fs directly with only a PathError alias

Key changes:
- `Atoms::PathExpander.expand()` → `Ace::Support::Fs::Atoms::PathExpander.expand()`
- `ProjectRootFinder.find()` → `Ace::Support::Fs::Molecules::ProjectRootFinder.find()`
- `PathError = Ace::Support::Fs::PathError` (alias for backward compat)
