# Refactor ace-taskflow to use ace-git-commit

## Objective

Refactor ace-taskflow to use ace-git-commit for all commit operations instead of creating a new ace-support-git gem.

**Rationale:** ace-git-commit already has a complete programmatic API that supports both LLM-generated messages (default) and explicit messages (bypasses LLM when provided). No new gem needed.

## Scope

Update ace-taskflow to delegate commit operations to ace-git-commit:
- Add convenience API to ace-git-commit (optional, for simpler usage)
- Refactor ace-taskflow's GitCommitter to use ace-git-commit
- Remove ace-taskflow's direct git command execution

## Technical Notes

**ace-git-commit API (already exists):**
```ruby
# Full API via orchestrator
orchestrator = Ace::GitCommit::Organisms::CommitOrchestrator.new
options = Ace::GitCommit::Models::CommitOptions.new(
  message: "chore(task-123): mark as done",
  files: ["path/to/file.md"],
  quiet: true
)
orchestrator.execute(options)

# Convenience API (to be added)
Ace::GitCommit.commit(
  files: ["path/to/task.md"],
  message: "chore(task-123): mark as done"
)
```

**Key insight:** `CommitOptions#use_llm?` returns `false` when message is provided - LLM is completely bypassed.

**Dependencies:**
- ace-taskflow gains dependency on ace-git-commit (which pulls in ace-llm)
- This is acceptable since LLM is only used when message is NOT provided
