# Extract ace-idea from ace-taskflow

## Objective

Extract ideation functionality from ace-taskflow into a standalone `ace-idea` gem with soft dependency on ace-taskflow for release detection. This reduces ace-taskflow complexity and elevates ideation to a first-class concern.

**Key Goals:**
1. Ideas become independent of task workflow
2. Ideation/prioritization gets dedicated focus
3. ace-taskflow becomes lighter and more focused on tasks
4. Configurable storage: `.ace-taskflow/{release}/ideas/` OR standalone `.ace-ideas/`

## Scope of Work

### Phase 0: Shared Utility Extractions (Prerequisites)
Extract utilities currently in ace-taskflow that ace-idea will need:
- **ace-support-git**: Git commit operations (GitCommitter)
- **ace-support-yaml**: YAML/frontmatter parsing (new gem)
- **ace-support-slug**: Slug sanitization and generation

### Phase 1: Create ace-idea Gem
- Scaffold ace-idea with ATOM structure
- Move ~2,500 LOC from ace-taskflow (models, molecules, organisms, commands)
- Implement ReleaseProvider with soft ace-taskflow dependency

### Phase 2: Update ace-taskflow
- Remove idea code from ace-taskflow
- Add backward compatibility wrapper for `ace-idea`

### Phase 3: Update Skills & Workflows
- Update `ace_capture-idea`, `ace_prioritize-ideas` skills
- Move idea workflows to ace-idea/handbook/

## Technical Notes

### ReleaseProvider Logic
```ruby
def use_taskflow_storage?
  defined?(Ace::Taskflow) &&
  Dir.exist?(".ace-taskflow") &&
  taskflow_release.present?
end
```

**Behavior:**
- If ace-taskflow loaded + has release info → `.ace-taskflow/{release}/ideas/`
- Otherwise → `.ace-ideas/{release}/` (standalone)
