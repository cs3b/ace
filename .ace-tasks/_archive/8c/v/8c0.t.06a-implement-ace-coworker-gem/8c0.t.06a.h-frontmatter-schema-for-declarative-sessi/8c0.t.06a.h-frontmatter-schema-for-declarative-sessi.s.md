## Technical Approach

### Core `coworker:` Schema

The frontmatter schema follows the `bundle:` pattern - supports both wrapped and flat formats.

### Complete Schema Reference

```yaml
coworker:
  # ═══════════════════════════════════════════════════════════════════
  # SESSION - Session tracking and hierarchy
  # ═══════════════════════════════════════════════════════════════════
  session:
    required: true              # Require tracked session (default: false)
    inherit: true               # Inherit config from parent session (default: true)
    parent: "${parent_id}"      # Explicit parent session ID (optional)
    id_prefix: "task-228"       # Custom prefix for session ID (optional)

  # ═══════════════════════════════════════════════════════════════════
  # CONTEXT - What context to load before execution
  # ═══════════════════════════════════════════════════════════════════
  context:
    bundle: "task-context"      # Preset name from ace-bundle
    files:                      # Additional files to include
      - path/to/spec.md
      - path/to/related.md
    commands:                   # Commands to run for dynamic context
      - ace-taskflow task $task
      - git log --oneline -5
    protocols:                  # Protocol URLs to fetch
      - wfi://work-on-task
      - guide://testing
    variables:                  # Variables available in context
      task: "$task_id"
      session: "$session_id"
      working_dir: "$PWD"

  # ═══════════════════════════════════════════════════════════════════
  # INSTRUCTIONS - What to tell the agent
  # ═══════════════════════════════════════════════════════════════════
  instructions:
    template: "wfi://work-on-task"   # Workflow instruction template
    inline: |                        # OR inline instructions
      Execute the task implementation.
      Follow the spec at $spec_path.
    variables:                       # Variables for interpolation
      task: "$task_id"
      spec_path: ".ace-taskflow/.../task.s.md"
    append: |                        # Additional instructions appended
      When complete, report with: ace-coworker report <file>

  # ═══════════════════════════════════════════════════════════════════
  # REPORT - How agent should report back
  # ═══════════════════════════════════════════════════════════════════
  report:
    format: markdown            # Report format (default: markdown)
    required: true              # Report required to advance (default: true)
    required_sections:          # Sections that must be present
      - summary
      - changes
      - test_results
    template: "tmpl://step-report"  # Report template (optional)
    bubble_up: true             # Aggregate to parent session (default: false)
    store_path: "reports/"      # Where to store reports (default: reports/)

  # ═══════════════════════════════════════════════════════════════════
  # VERIFICATION - How to verify step completion
  # ═══════════════════════════════════════════════════════════════════
  verification:
    commands:                   # Commands to run (exit 0 = pass)
      - ace-test
      - ace-lint --check
      - ace-review --preset quick
    retries: 3                  # Max retry attempts (default: 3)
    on_repeated_failure: 3      # Stop after N identical errors (default: 3)
    timeout: 300                # Command timeout in seconds (default: 300)
    continue_on_warning: true   # Continue if exit code = 0 but warnings (default: false)

  # ═══════════════════════════════════════════════════════════════════
  # DELEGATION - Subagent spawning configuration
  # ═══════════════════════════════════════════════════════════════════
  delegation:
    enabled: true               # Allow subagent delegation (default: false)
    inherit_session: true       # Children link to parent session (default: true)
    inherit_context: true       # Children inherit parent context (default: true)
    worktree: true              # Use git worktree for isolation (default: false)
    max_concurrent: 3           # Max concurrent child sessions (default: 1)
    timeout: 3600               # Delegation timeout in seconds (default: 3600)

  # ═══════════════════════════════════════════════════════════════════
  # GATES - Human approval gates
  # ═══════════════════════════════════════════════════════════════════
  gates:
    before_start: false         # Require approval before workflow starts
    before_step: false          # Require approval before each step
    before_delegation: true     # Require approval before spawning subagent
    on_failure: true            # Require approval to retry after failure
    on_complete: false          # Require approval before marking complete

  # ═══════════════════════════════════════════════════════════════════
  # WORKFLOW - Workflow definition (inline or reference)
  # ═══════════════════════════════════════════════════════════════════
  workflow:
    name: "task-completion"     # Workflow name
    ref: "wfi://task-completion"  # OR external workflow reference
    steps: [...]                # OR embedded step definitions (see below)
```

### Step-Level Schema

Each step in `workflow.steps` can have its own configuration:

```yaml
coworker:
  workflow:
    steps:
      # ─────────────────────────────────────────────────────────────
      # Step 1: Implementation with full configuration
      # ─────────────────────────────────────────────────────────────
      - name: implement
        description: "Implement the task according to spec"

        # Step-specific context (merged with workflow-level)
        context:
          bundle: implement-context
          files:
            - $spec_path
          commands:
            - ace-bundle wfi://work-on-task

        # Step instructions
        instructions:
          template: wfi://work-on-task
          variables:
            task: $task_id

        # Step report requirements
        report:
          required: true
          required_sections: [summary, changes, files_modified]

        # Step verification
        verification:
          commands: [ace-test]
          retries: 5
          on_repeated_failure: 3

        # Step-specific delegation
        delegation:
          enabled: false        # No subagents for this step

      # ─────────────────────────────────────────────────────────────
      # Step 2: Review with human gate
      # ─────────────────────────────────────────────────────────────
      - name: review
        description: "Review changes before release"

        gate: human             # Shorthand for gates.before_step: true

        context:
          commands:
            - git diff --stat
            - ace-review --preset pr

        instructions:
          inline: |
            Review the implementation:
            1. Check test coverage
            2. Verify acceptance criteria
            3. Approve or request changes

        report:
          required: true
          required_sections: [decision, feedback]

      # ─────────────────────────────────────────────────────────────
      # Step 3: Delegated subtasks
      # ─────────────────────────────────────────────────────────────
      - name: subtasks
        description: "Delegate to subtask workers"

        delegation:
          enabled: true
          inherit_session: true
          max_concurrent: 3

        instructions:
          inline: |
            For each subtask, spawn a subagent:

            Task tool prompt:
            """
            ace-coworker start --task $subtask_id --parent $session_id
            Follow instructions from ace-coworker status
            """

        report:
          bubble_up: true       # Collect all subtask reports

      # ─────────────────────────────────────────────────────────────
      # Step 4: Release
      # ─────────────────────────────────────────────────────────────
      - name: release
        description: "Prepare and publish release"

        context:
          bundle: release-context

        instructions:
          template: wfi://draft-release

        verification:
          commands:
            - ace-test-suite
            - ace-lint

        gates:
          on_complete: true     # Final approval before done
```

### Flat Format (Alternative)

For simpler configs, flat keys are supported:

```yaml
coworker:
  # Flat equivalents of nested keys
  session_required: true
  session_inherit: true

  context_bundle: "task-context"
  context_files: [spec.md]

  instructions_template: "wfi://work-on-task"

  report_required: true
  report_format: markdown

  verification_commands: [ace-test, ace-lint]
  verification_retries: 3

  delegation_enabled: true
  delegation_max_concurrent: 3

  gates_on_failure: true
```

### Variable Interpolation

Variables available in all string fields:

| Variable | Description | Example |
|----------|-------------|---------|
| `$task_id` | Current task ID | `228` |
| `$session_id` | Current session ID | `8or5kx` |
| `$parent_id` | Parent session ID | `8or5kx` (for child `8or5kx.1`) |
| `$step_name` | Current step name | `implement` |
| `$step_index` | Current step index (0-based) | `0` |
| `$attempt` | Current attempt number (1-based) | `1` |
| `$working_dir` | Working directory | `/path/to/project` |
| `$spec_path` | Task spec file path | `.ace-taskflow/.../228.s.md` |
| `$cache_dir` | Session cache directory | `.cache/ace-coworker/8or5kx` |

### Session Hierarchy Model

Child session IDs extend parent IDs with dot notation:

```
8or5kx           # Parent (orchestrator task 234)
8or5kx.1         # Child (subtask 234.01)
8or5kx.2         # Child (subtask 234.02)
8or5kx.1.1       # Grandchild (nested delegation)
```

### Directory Structure

```
.cache/ace-coworker/
├── 8or5kx/              # Parent session
│   ├── job.json         # { children: ["8or5kx.1", "8or5kx.2"] }
│   └── log.jsonl        # Logs child_started, child_completed events
├── 8or5kx.1/            # Child session
│   └── job.json         # { parent: "8or5kx" }
└── 8or5kx.2/
    └── job.json         # { parent: "8or5kx" }
```

### CoworkerConfigParser Atom

```ruby
module Ace
  module Coworker
    module Atoms
      class CoworkerConfigParser
        # Parse coworker configuration from frontmatter hash
        # @param frontmatter [Hash] Parsed frontmatter
        # @return [Hash] Result with :config, :valid, :errors
        def self.parse(frontmatter)
          return empty_result if frontmatter.nil? || frontmatter.empty?

          coworker = frontmatter["coworker"] || frontmatter[:coworker]
          return empty_result unless coworker

          config = unwrap_config(coworker)
          errors = validate(config)

          {
            config: config,
            valid: errors.empty?,
            errors: errors
          }
        end

        def self.unwrap_config(coworker)
          # Support nested and flat formats for all keys
          {
            # Session
            session_required: extract_bool(coworker, %w[session required], false),
            session_inherit: extract_bool(coworker, %w[session inherit], true),
            session_parent: extract_string(coworker, %w[session parent]),

            # Context
            context_bundle: extract_string(coworker, %w[context bundle]),
            context_files: extract_array(coworker, %w[context files]),
            context_commands: extract_array(coworker, %w[context commands]),
            context_protocols: extract_array(coworker, %w[context protocols]),
            context_variables: extract_hash(coworker, %w[context variables]),

            # Instructions
            instructions_template: extract_string(coworker, %w[instructions template]),
            instructions_inline: extract_string(coworker, %w[instructions inline]),
            instructions_variables: extract_hash(coworker, %w[instructions variables]),
            instructions_append: extract_string(coworker, %w[instructions append]),

            # Report
            report_format: extract_string(coworker, %w[report format], "markdown"),
            report_required: extract_bool(coworker, %w[report required], true),
            report_required_sections: extract_array(coworker, %w[report required_sections]),
            report_template: extract_string(coworker, %w[report template]),
            report_bubble_up: extract_bool(coworker, %w[report bubble_up], false),

            # Verification
            verification_commands: extract_array(coworker, %w[verification commands]),
            verification_retries: extract_int(coworker, %w[verification retries], 3),
            verification_on_repeated_failure: extract_int(coworker, %w[verification on_repeated_failure], 3),
            verification_timeout: extract_int(coworker, %w[verification timeout], 300),

            # Delegation
            delegation_enabled: extract_bool(coworker, %w[delegation enabled], false),
            delegation_inherit_session: extract_bool(coworker, %w[delegation inherit_session], true),
            delegation_inherit_context: extract_bool(coworker, %w[delegation inherit_context], true),
            delegation_worktree: extract_bool(coworker, %w[delegation worktree], false),
            delegation_max_concurrent: extract_int(coworker, %w[delegation max_concurrent], 1),

            # Gates
            gates_before_start: extract_bool(coworker, %w[gates before_start], false),
            gates_before_step: extract_bool(coworker, %w[gates before_step], false),
            gates_before_delegation: extract_bool(coworker, %w[gates before_delegation], false),
            gates_on_failure: extract_bool(coworker, %w[gates on_failure], false),
            gates_on_complete: extract_bool(coworker, %w[gates on_complete], false),

            # Workflow
            workflow_name: extract_string(coworker, %w[workflow name]),
            workflow_ref: extract_string(coworker, %w[workflow ref]),
            workflow_steps: extract_array(coworker, %w[workflow steps])
          }
        end
      end
    end
  end
end
```

## Risk Assessment

### Technical Risks

- **Risk:** Frontmatter parsing may conflict with existing bundle: handling
  - **Probability:** Low
  - **Impact:** Low
  - **Mitigation:** coworker: is separate from bundle:; both can coexist

- **Risk:** Session ID dot notation may cause filesystem issues
  - **Probability:** Very Low
  - **Impact:** Medium
  - **Mitigation:** Dots are valid in directory names on all platforms

- **Risk:** Hierarchy depth could grow unbounded
  - **Probability:** Low
  - **Impact:** Low
  - **Mitigation:** Practical limit of 3-4 levels; log depth in hierarchy metadata

### Integration Risks

- **Risk:** Changes break existing 234.01-234.02 functionality
  - **Probability:** Low
  - **Impact:** High
  - **Mitigation:** Frontmatter config is optional; existing behavior preserved

- **Risk:** Parent session updates create race conditions
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Use atomic file writes; child_sessions array is append-only
