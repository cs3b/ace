## Objective

- Session-specific `--session` flag for status/resume commands
- `--new-session` flag to force new session creation

## Technical Approach

### Architecture Pattern

Following ATOM architecture with clear separation:
- **Atoms**: Pure functions for session ID generation and list formatting (no I/O)
- **Molecules**: Filesystem operations for session discovery and management
- **CLI**: Commands for list, start (with session flags), status, resume

### Session ID Generation

Uses ace-support-timestamp for compact, sortable, unique IDs:

```ruby
module Ace
  module Coworker
    module Atoms
      class SessionIdGenerator
        # Generate compact session ID using ace-timestamp
        # @param time [Time] Optional time in UTC (default: current UTC time)
        # @return [String] 6-character Base36 session ID (e.g., "8or5kx")
        def self.generate(time: nil)
          time ||= Time.now.utc
          Ace::Support::Timestamp.encode(time)
        end

        # Validate session ID format
        # @param id [String] Session ID to validate
        # @return [Boolean] true if valid 6-char Base36 ID
        def self.valid?(id)
          return false unless id.is_a?(String)
          Ace::Support::Timestamp.valid?(id)
        end
      end
    end
  end
end
```

### Session Discovery Implementation

```ruby
module Ace
  module Coworker
    module Molecules
      class SessionDiscoverer
        # Find all sessions in cache directory
        # @param cache_dir [String] Path to cache directory
        # @return [Array<Hash>] Array of session info hashes
        def self.find_all(cache_dir:)
          pattern = File.join(cache_dir, "*/job.json")
          Dir.glob(pattern).map do |job_path|
            session_id = File.basename(File.dirname(job_path))
            job = JSON.parse(File.read(job_path), symbolize_names: true)
            {
              session_id: session_id,
              task_id: job[:task_id],
              workflow: job[:workflow],
              status: job[:status],
              current_step: job[:current_step],
              total_steps: job[:steps]&.size || 0,
              updated_at: job[:updated_at],
              created_at: job[:created_at]
            }
          rescue JSON::ParserError, Errno::ENOENT
            nil
          end.compact
        end

        # Find sessions for a specific task
        # @param cache_dir [String] Path to cache directory
        # @param task_id [String] Task ID to filter by
        # @param active_only [Boolean] Only include running/paused sessions
        # @return [Array<Hash>] Matching sessions sorted by updated_at desc
        def self.find_by_task(cache_dir:, task_id:, active_only: true)
          sessions = find_all(cache_dir: cache_dir)
          sessions = sessions.select { |s| s[:task_id] == task_id.to_s }

          if active_only
            sessions = sessions.select { |s| %w[running paused].include?(s[:status]) }
          end

          # Sort by updated_at descending (most recent first)
          sessions.sort_by { |s| s[:updated_at] || "" }.reverse
        end

        # Find a specific session by ID
        # @param cache_dir [String] Path to cache directory
        # @param session_id [String] Session ID to find
        # @return [Hash, nil] Session info or nil if not found
        def self.find_by_id(cache_dir:, session_id:)
          job_path = File.join(cache_dir, session_id, "job.json")
          return nil unless File.exist?(job_path)

          job = JSON.parse(File.read(job_path), symbolize_names: true)
          {
            session_id: session_id,
            task_id: job[:task_id],
            workflow: job[:workflow],
            status: job[:status],
            current_step: job[:current_step],
            total_steps: job[:steps]&.size || 0,
            updated_at: job[:updated_at],
            created_at: job[:created_at]
          }
        rescue JSON::ParserError, Errno::ENOENT
          nil
        end
      end
    end
  end
end
```

### Session List Formatter

```ruby
module Ace
  module Coworker
    module Atoms
      class SessionListFormatter
        # Format sessions as table output
        # @param sessions [Array<Hash>] Session info hashes
        # @return [String] Formatted table output
        def self.format_table(sessions)
          return "No sessions found." if sessions.empty?

          headers = %w[SESSION TASK WORKFLOW STATUS STEP PROGRESS]
          rows = sessions.map do |s|
            step_name = s[:current_step_name] || "-"
            progress = "#{s[:current_step] + 1}/#{s[:total_steps]}"
            [s[:session_id], s[:task_id], s[:workflow], s[:status], step_name, progress]
          end

          # Calculate column widths
          widths = headers.map.with_index do |h, i|
            [h.length, rows.map { |r| r[i].to_s.length }.max || 0].max
          end

          # Format header
          header_line = headers.map.with_index { |h, i| h.ljust(widths[i]) }.join("  ")
          separator = widths.map { |w| "-" * w }.join("  ")

          # Format rows
          row_lines = rows.map do |row|
            row.map.with_index { |cell, i| cell.to_s.ljust(widths[i]) }.join("  ")
          end

          [header_line, separator, *row_lines].join("\n")
        end

        # Format sessions as JSON output
        # @param sessions [Array<Hash>] Session info hashes
        # @return [String] JSON output
        def self.format_json(sessions)
          JSON.pretty_generate(sessions)
        end
      end
    end
  end
end
```

### Multi-Session Resume Rules

When `start --task X` is called:

1. **Explicit Session**: If `--session <id>` provided, resume that session directly
2. **Force New**: If `--new-session` flag, create new session regardless of existing
3. **Auto-Resume**: Otherwise:
   - Scan for sessions where `task_id == X`
   - Filter to only `running` or `paused` status (not `completed`, `rejected`, `failed`)
   - Select the most recently updated session (by `updated_at`)
   - If found: resume that session
   - If not found: create new session

### CLI Interface Updates

#### Start Command (Updated)

```ruby
module Ace
  module Coworker
    module CLI
      module Commands
        class Start < Dry::CLI::Command
          include Ace::Core::CLI::DryCli::Base

          desc "Start or resume workflow session"

          option :task, type: :string, aliases: ["-t"], desc: "Task ID to work on"
          option :workflow, type: :string, aliases: ["-w"], desc: "Workflow name (default: task-completion)"
          option :session, type: :string, aliases: ["-s"], desc: "Resume specific session by ID"
          option :new_session, type: :boolean, desc: "Force create new session even if one exists"

          def call(**options)
            # Validation
            if options[:session] && options[:task]
              raise Ace::Core::CLI::Error.new("Cannot specify both --session and --task")
            end

            if options[:new_session] && options[:session]
              raise Ace::Core::CLI::Error.new("Cannot specify both --new-session and --session")
            end

            unless options[:session] || options[:task]
              raise Ace::Core::CLI::Error.new("Must specify --task or --session")
            end

            # Session resolution logic
            if options[:session]
              resume_session(options[:session])
            elsif options[:new_session]
              create_new_session(options[:task], options[:workflow])
            else
              auto_resume_or_create(options[:task], options[:workflow])
            end
          end

          private

          def auto_resume_or_create(task_id, workflow)
            # Find existing active sessions for this task
            sessions = SessionDiscoverer.find_by_task(
              cache_dir: cache_dir,
              task_id: task_id,
              active_only: true
            )

            if sessions.any?
              # Resume most recently updated session
              session = sessions.first
              puts "Resuming existing session #{session[:session_id]} for task #{task_id}"
              resume_session(session[:session_id])
            else
              create_new_session(task_id, workflow)
            end
          end
        end
      end
    end
  end
end
```

#### List Command (New)

```ruby
module Ace
  module Coworker
    module CLI
      module Commands
        class List < Dry::CLI::Command
          include Ace::Core::CLI::DryCli::Base

          desc "List all workflow sessions"

          option :format, type: :string, default: "table", desc: "Output format (table, json)"
          option :task, type: :string, aliases: ["-t"], desc: "Filter by task ID"
          option :status, type: :string, desc: "Filter by status (running, paused, completed, etc.)"
          option :all, type: :boolean, aliases: ["-a"], desc: "Include completed/rejected sessions"

          def call(**options)
            sessions = Molecules::SessionDiscoverer.find_all(cache_dir: cache_dir)

            # Apply filters
            sessions = apply_filters(sessions, options)

            # Format output
            output = case options[:format]
            when "json"
              Atoms::SessionListFormatter.format_json(sessions)
            else
              Atoms::SessionListFormatter.format_table(sessions)
            end

            puts output
          end

          private

          def apply_filters(sessions, options)
            sessions = sessions.select { |s| s[:task_id] == options[:task] } if options[:task]
            sessions = sessions.select { |s| s[:status] == options[:status] } if options[:status]

            unless options[:all]
              # Default: show only active sessions
              sessions = sessions.select { |s| %w[running paused].include?(s[:status]) }
            end

            sessions
          end

          def cache_dir
            Ace::Coworker.config[:cache_dir] || ".cache/ace-coworker"
          end
        end
      end
    end
  end
end
```

## Risk Assessment

### Technical Risks

- **Risk:** Session ID collision (two sessions created at same second)
  - **Probability:** Very Low
  - **Impact:** Medium
  - **Mitigation:** ace-timestamp uses 2-second resolution; typical workflow takes longer to start

- **Risk:** Race condition when multiple agents start sessions simultaneously
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Each agent creates unique session ID; filesystem operations are atomic at directory level

- **Risk:** Performance with many sessions (filesystem scanning)
  - **Probability:** Low
  - **Impact:** Low
  - **Mitigation:** Glob is efficient; cache directory is local; session cleanup is manual

### Integration Risks

- **Risk:** Changes break 234.01-234.06 functionality
  - **Probability:** Medium
  - **Impact:** High
  - **Mitigation:** Session ID generator change is additive; run full test suite after integration

- **Risk:** Start command logic conflicts with existing behavior
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Existing single-session behavior preserved when no flags provided; new flags are additive

## Out of Scope

- Automatic session cleanup/pruning
- Session TTL (time-to-live) expiration
- Cross-directory session discovery
- Session locking for concurrent access
- Session migration between directories
- Worktree integration (ace-overseer responsibility)
- Remote session synchronization
