## Technical Approach

### Architecture Pattern

Following ATOM architecture with clear separation:
- **Atoms**: Pure functions for variable interpolation (no I/O)
- **Molecules**: Already exist from 234.01-234.04
- **Integration**: Variable interpolation called before instruction execution

### Workflow Parameter Interpolation

Workflow files use shell-style variable interpolation:
- `$task` → replaced with `--task` argument value
- `$session` → replaced with current session ID
- Variables are expanded before instruction execution

Example: `ace-bundle wfi://work-on-task $task` with `--task 228` becomes `ace-bundle wfi://work-on-task 228`

**Not supported in MVP:**
- Environment variable fallback (`${VAR:-default}`)
- Nested expansion
- Command substitution

### Full 8-Step Workflow Definition

```yaml
name: task-completion
description: Complete task lifecycle from implementation to PR update
version: 1

steps:
  - name: implement
    instructions: |
      Execute task implementation workflow:
      ace-bundle wfi://work-on-task $task

      When complete, report results with: ace-coworker report <file>
    verifications:
      - command: ace-test
        description: All tests pass
    retries: 5

  - name: commit
    instructions: |
      Commit implementation changes:
      ace-bundle wfi://commit

      When complete, report results with: ace-coworker report <file>

  - name: test
    instructions: |
      Run full test suite to verify implementation:
      ace-test

      Report test results: ace-coworker report <file>
    retries: 5

  - name: review-gate
    gate: human
    prompt: "Review implementation before creating PR. Approve with: ace-coworker resume --approve"

  - name: create-pr
    instructions: |
      Create pull request for the changes:
      ace-bundle wfi://create-pr

      When complete, report PR URL with: ace-coworker report <file>

  - name: self-review
    instructions: |
      Run self-review on the PR:
      ace-bundle wfi://review-pr

      Report review findings: ace-coworker report <file>

  - name: apply-feedback
    instructions: |
      Apply feedback from self-review:
      Review the synthesis report and implement any required fixes.

      When complete, report changes with: ace-coworker report <file>
    verifications:
      - command: ace-test
        description: All tests pass

  - name: update-pr
    instructions: |
      Push updates and refresh PR:
      git push
      ace-bundle wfi://update-pr-description

      Report final PR state: ace-coworker report <file>
```

### Variable Interpolator Implementation

```ruby
module Ace
  module Coworker
    module Atoms
      class VariableInterpolator
        VARIABLE_PATTERN = /\$([a-z_][a-z0-9_]*)/i

        # Pure function: (text:, variables:) -> interpolated_text
        # variables is a hash like { task: "228", session: "abc123" }
        def self.interpolate(text:, variables:)
          text.gsub(VARIABLE_PATTERN) do |match|
            var_name = Regexp.last_match(1).downcase.to_sym
            variables.fetch(var_name, match)  # Keep original if not found
          end
        end

        # Pure function: (text:) -> array of variable names
        def self.extract_variables(text:)
          text.scan(VARIABLE_PATTERN).flatten.map { |v| v.downcase.to_sym }.uniq
        end
      end
    end
  end
end
```
