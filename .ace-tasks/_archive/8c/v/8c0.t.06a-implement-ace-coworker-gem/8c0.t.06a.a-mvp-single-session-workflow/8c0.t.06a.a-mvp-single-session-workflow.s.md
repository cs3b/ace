# MVP - Single session workflow

> **ARCHIVED**: This task has been superseded by task 237 (ace-coworker MVP with Work Queue Model).
> The simplified implementation is tracked in 237-orchestrator.s.md.

## Objective

Create the smallest working ace-coworker that can execute a two-step workflow with state persistence.

## Technical Approach

### Architecture (ATOM Pattern)

```
ace-coworker/
├── lib/ace/coworker/
│   ├── atoms/
│   │   ├── job_parser.rb          # Pure: Parse job.json to hash
│   │   ├── session_id_generator.rb # Pure: Generate unique session IDs
│   │   └── step_resolver.rb        # Pure: Determine next step from job state
│   ├── molecules/
│   │   ├── job_manager.rb         # Stateful: Read/write job.json
│   │   └── session_manager.rb     # Stateful: Find/create session directories
│   ├── organisms/
│   │   └── workflow_executor.rb   # Orchestrate: Coordinate workflow execution
│   ├── models/
│   │   ├── job.rb                 # Data: Job state structure
│   │   └── step.rb                # Data: Step structure
│   └── cli/
│       └── commands/
│           ├── start.rb           # CLI: Start workflow command
│           ├── status.rb          # CLI: Status check command
│           └── report.rb          # CLI: Submit report command
```

### Dependencies

- `dry-cli ~> 1.0` - CLI framework (ADR-023)
- `ace-support-core ~> 0.18` - Base infrastructure

## Behavioral Specification

### User Experience

**Input:** User invokes `/ace:coworker-work-on 228`

**Process:**
1. Agent calls `ace-coworker start --task 228` -> gets step 1 instructions
2. Agent follows instructions (wfi://work-on-task)
3. Agent calls `ace-coworker report <file>` -> stores report, advances to step 2
4. Agent follows step 2 instructions (wfi://draft-release)
5. Agent calls `ace-coworker report <file>` -> workflow complete

**Output:** Task completed through two-step workflow with state tracked

### Interface Contract

```bash
# Start workflow - creates session, returns first step instructions
ace-coworker start --task 228
# Output: Session created, step instructions printed

# Check current step and instructions
ace-coworker status
# Output: Current step name, instructions, progress

# Store agent report and advance to next step
ace-coworker report <file>
# Output: Report stored, advanced to next step (or workflow complete)
```
