## Objective

Track what goes to agent (delegation) and what comes back (report) with structured logging. This provides:
- Audit trail of all workflow events
- Reproducibility by capturing exact instructions sent
- Debugging capability via event stream
- Retry support with attempt numbering

## Scope of Work

Add logging and reporting capabilities to ace-coworker:
- JSONL event stream for all workflow events

## Technical Approach

### Architecture Pattern

Following ATOM architecture with clear separation:
- **Atoms**: Pure functions for event serialization (JSON string generation) and file naming (no I/O)
- **Molecules**: File I/O operations for event logging (JSONL append), delegation/report storage
- **Integration**: Logging calls added to existing workflow executor

### Technology Stack

- **ace-support-timestamp**: Provides `Ace::Support::Timestamp.encode(Time.now)` for 6-char compact IDs
- **JSON**: Standard library for JSONL serialization
- **FileUtils**: Standard library for directory creation

### Implementation Strategy

1. Create atoms first (pure functions, easy to test)
2. Build molecules on atoms (file I/O with clear interfaces)
3. Integrate into existing commands (minimal changes)
4. Add comprehensive tests at each layer

## File Structure

**Session Files (after this task):**
```
.cache/ace-coworker/{session-id}/
├── job.json                              # Session state (from 234.01)
├── log.jsonl                             # NEW: event stream
└── reports/                              # NEW: delegation and report files
    ├── 001-01-implement-delegation.md    # Step 1, attempt 1: instructions
    ├── 001-01-implement-report.md        # Step 1, attempt 1: response
    ├── 001-02-implement-delegation.md    # Retry: step 1, attempt 2
    ├── 001-02-implement-report.md
    ├── 002-01-release-delegation.md      # Step 2, attempt 1
    └── 002-01-release-report.md
```

**File naming convention:** `{step_number}-{attempt_number}-{step_name}-{type}.md`
- Step number: 3-digit zero-padded (001, 002, ...)
- Attempt number: 2-digit zero-padded (01, 02, ...)
- Type: `delegation` or `report`

## Log Event Schema

```jsonl
{"ts":"8or5kx","event":"workflow_started","workflow":"work-on-task","task_id":"228","session_id":"8or5kx"}
{"ts":"8or5ky","event":"step_started","step":"implement","step_index":0,"attempt":1,"session_id":"8or5kx"}
{"ts":"8or5kz","event":"delegation_sent","step":"implement","attempt":1,"file":"reports/001-01-implement-delegation.md"}
{"ts":"8or5m0","event":"report_received","step":"implement","attempt":1,"file":"reports/001-01-implement-report.md"}
{"ts":"8or5m1","event":"step_completed","step":"implement","attempt":1}
```

**Schema fields:**
- `ts`: 6-character ace-timestamp (UTC, via ace-support-timestamp)
- `event`: Event type identifier
- `session_id`: Session ID for correlation
- `step`: Step name (for step-related events)
- `step_index`: Zero-based step index (optional)
- `attempt`: Attempt number (1-based)
- `file`: Relative path to file (for file events)
