# Replace ace-nav wfi:// with ace-context wfi:// in Claude Code commands and workflows

## Objective

Standardize on `ace-context` for context loading operations across Claude Code commands and workflows, providing a more intuitive agent experience with direct content display (leveraging task 152's auto-format feature).

## Behavioral Specification

### User Experience
- **Input**: Claude Code agents invoke workflows via `ace-context wfi://workflow-name`
- **Process**: ace-context resolves protocol, loads content, and returns directly (for small files) or via path (for large files)
- **Output**: Workflow content displayed directly to agent context for immediate use

### Expected Behavior

When agents and workflows need to load context:
- `ace-context wfi://workflow-name` → Returns workflow content directly (< 500 lines) or path (>= 500 lines)
- Explicit `--output` flags still work for specific needs
- Cross-references between workflows use consistent ace-context pattern

## Technical Approach

### Phase 1: ace-context Enhancement

Enhance `load_template` method to fall back to raw content when no context configuration is found:

```ruby
# In load_template, after checking for context config
# If no context config found, return file content directly
context = Models::ContextData.new
context.content = File.read(path)
context.metadata = frontmatter.transform_keys(&:to_sym)
context
```

## Risk Assessment

### Technical Risks
- **Risk:** ace-context changes break existing preset loading
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Only add fallback, don't change existing paths
  - **Rollback:** Revert load_template changes

### Integration Risks
- **Risk:** Some workflows may need file paths, not content
  - **Probability:** Low
  - **Impact:** Low
  - **Mitigation:** Agents can use `ace-nav` when path is specifically needed
