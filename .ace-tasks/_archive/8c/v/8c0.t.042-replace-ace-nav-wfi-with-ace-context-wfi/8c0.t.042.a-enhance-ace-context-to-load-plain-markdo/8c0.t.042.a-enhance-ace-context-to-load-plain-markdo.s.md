# Enhance ace-context to load plain markdown files

## Objective

Make `ace-context wfi://workflow-name` work by adding fallback handling for files with metadata-only frontmatter (no context configuration keys).

## Behavioral Specification

### User Experience
- **Input**: `ace-context wfi://commit` or `ace-context path/to/file.wf.md`
- **Process**: Protocol resolves to file path, file content loaded with frontmatter parsed as metadata
- **Output**: File content returned (inline for < 500 lines, path for >= 500 lines per auto-format feature)

### Current Behavior (Bug)
```bash
$ ace-context wfi://commit --output stdio
Error: No valid configuration found
```

### Expected Behavior
```bash
$ ace-context wfi://commit --output stdio
---
description: Commit Changes
allowed-tools: Bash(git:*), Read, Glob, Grep
---

# Commit Workflow
...
```

### Success Criteria

- [ ] `ace-context wfi://commit` returns workflow content
- [ ] `ace-context path/to/file.wf.md` returns file content with metadata preserved
- [ ] Frontmatter metadata available in result metadata
- [ ] Existing template loading (files with `context:` key) unchanged
- [ ] All ace-context tests pass

## Root Cause Analysis

Current code path in `context_loader.rb`:
1. Protocol resolves via ace-nav → returns file path
2. File has YAML frontmatter → detected as "template" by `TemplateParser.template?`
3. `load_template` called → expects context config keys (`files`, `commands`, `include`)
4. Workflow files have only metadata keys (`description`, `allowed-tools`) → **falls through to body parsing**
5. Body parsing fails with "No valid configuration found"

## Scope of Work

### Modify

**ace-context/lib/ace/context/organisms/context_loader.rb** (lines 428-492):
- Add fallback case in `load_template` method
- After checking for `context` key or template config keys
- If frontmatter exists but has no context config → return raw content with metadata

### Create

**ace-context/test/integration/plain_markdown_load_test.rb**:
- Test loading plain markdown file by path
- Test loading via wfi:// protocol
- Test frontmatter metadata preservation

## Technical Approach

```ruby
# In load_template, after lines 430-431 (config key check)
# Add else clause:

if frontmatter.key?('context') || has_template_config_keys?(frontmatter)
  # Existing template loading logic...
else
  # NEW: Plain markdown with metadata-only frontmatter
  context = Models::ContextData.new
  context.content = template_content  # Raw markdown body
  context.metadata = frontmatter.transform_keys(&:to_sym)
  context.source = path
  return context
end
```
