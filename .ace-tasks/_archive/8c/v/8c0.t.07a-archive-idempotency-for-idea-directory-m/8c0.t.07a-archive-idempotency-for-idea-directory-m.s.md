# Archive Idempotency for Idea Directory Mover

## Behavioral Specification

### User Experience
- **Input**: `ace-idea done <idea-ref>` where the idea is already in `_archive/`
- **Process**: The system recognizes the idea is already archived, updates metadata (completion timestamp), and returns success
- **Output**: `"Idea already in _archive/"` — no file movement, no double-nesting

### Expected Behavior

**Current behavior (broken)**:
```
$ ace-idea done 8pnwx0-intent-engineering
Idea '8pnwx0-intent-engineering' marked as done and moved to _archive/

# But the idea was ALREADY in _archive/, so now it's at:
# ideas/_archive/_archive/8pnwx0-intent-engineering/
```

The mover calculates the archive path relative to the idea's current parent directory. When the idea is already inside `_archive/`, the target becomes `_archive/_archive/` — an infinitely nestable bug.

**Target behavior**:
```
$ ace-idea done 8pnwx0-intent-engineering
Idea already in _archive/
Completed at: 2026-02-25 15:10:40
```

The system:
1. Detects that the idea path already contains `/_archive/`
2. Updates frontmatter metadata (`status: done`, `completed_at`) in place
3. Returns success with the existing path — no file movement

**Same behavior for `move_to_maybe`**:
```
# Idea already in _maybe/
$ ace-idea park 8pnwx0-some-idea
Idea already in _maybe/
```

**Target-already-exists case**: When the target path exists in the archive (e.g., a previous partial move left artifacts), return success instead of failure — matching `TaskDirectoryMover` behavior.

### Interface Contract

```bash
# Idea already archived — idempotent success
$ ace-idea done my-archived-idea
Idea already in _archive/
Completed at: 2026-02-25 15:10:40

# Idea not yet archived — normal move (unchanged behavior)
$ ace-idea done my-active-idea
Idea 'my-active-idea' marked as done and moved to _archive/
Completed at: 2026-02-25 15:10:40

# Idea already in _maybe/ — idempotent success
$ ace-idea park my-parked-idea
Idea already in _maybe/
```

**Error Handling:**
- Idea not found: `"Idea not found: <path>"` (unchanged)
- Idea path is nil: `"Idea path not provided"` (unchanged)
- Frontmatter update fails on already-archived idea: Log warning, still return success (the path is correct even if metadata update fails)

**Edge Cases:**
- Idea passed as file path (not folder): Normalize to folder, then check if folder is in `_archive/`
- Idea in `_archive/` but missing frontmatter: Add frontmatter with `status: done` and `completed_at`
- Nested `_archive/_archive/` already exists from previous bug: This fix prevents new occurrences; existing double-nesting needs manual cleanup (out of scope)

### Success Criteria

- [ ] `ace-idea done` on an already-archived idea returns `success: true` without moving files
- [ ] No `_archive/_archive/` directory is created
- [ ] Frontmatter is updated with `status: done` and `completed_at` even on idempotent calls
- [ ] `move_to_maybe` on an already-parked idea returns `success: true` without moving
- [ ] No `_maybe/_maybe/` directory is created
- [ ] Target-already-exists check returns success (not failure) — matches `TaskDirectoryMover`
- [ ] Normal move behavior (idea not in archive) is unchanged
- [ ] `TaskDirectoryMover` pattern is followed (consistency across movers)

### Validation Questions

- [x] Should idempotent calls update metadata or purely no-op? → **Update metadata** (timestamp refreshed, status confirmed as `done`)
- [x] Should `move_to_maybe` get the same fix? → **Yes** — same vulnerability with `_maybe/_maybe/`
- [x] Should target-exists also become idempotent? → **Yes** — return success, matching TaskDirectoryMover

## Objective

Make archive a terminal state for ideas. Once an idea is in `_archive/`, subsequent `ace-idea done` calls should be safe no-ops (with metadata updates), not destructive moves into nested directories. The `TaskDirectoryMover` already has this protection — the `IdeaDirectoryMover` should match. This bug was hit during the task-281 drafting session and required manual cleanup.

## Scope of Work

- **User Experience Scope**: `ace-idea done` and `ace-idea park` are safe to call multiple times
- **System Behavior Scope**: `IdeaDirectoryMover.move_to_archive` and `move_to_maybe` gain idempotence guards
- **Interface Scope**: Return value changes from `success: false` to `success: true` for already-in-target cases

### Deliverables

#### Code Changes
- `IdeaDirectoryMover.move_to_archive` — add pre-check for already-archived
- `IdeaDirectoryMover.move_to_maybe` — add pre-check for already-parked
- Both methods: change target-exists from error to success
- Add `update_completion_metadata_in_place` helper for idempotent metadata updates

#### Tests
- `test_move_to_archive_idempotent_when_already_archived`
- `test_move_to_maybe_idempotent_when_already_in_maybe`
- `test_move_to_archive_updates_metadata_when_already_archived`

## Out of Scope

- Cleaning up existing `_archive/_archive/` directories (manual, one-time fix)
- Changes to `TaskDirectoryMover` (already correct)
- Changes to the `ace-idea done` CLI command itself (the fix is in the mover layer)
- Changes to `restore_from_archive` / `restore_from_maybe` (already have guards)
