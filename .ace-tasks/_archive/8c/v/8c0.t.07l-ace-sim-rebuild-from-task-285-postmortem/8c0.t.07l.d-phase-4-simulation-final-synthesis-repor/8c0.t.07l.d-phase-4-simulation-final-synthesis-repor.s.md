# Phase 4: simulation final synthesis report (unplanned)

## Behavioral Context

**Issue**: After Phase 3, `ace-sim` produced per-step chain artifacts and `synthesis.yml`, but there was no final actionable report generated from all step outputs.

**Key Behavioral Requirements**:
- `ace-sim run` must optionally execute a final synthesis stage that consumes chain outputs.
- Runtime must support explicit controls for final synthesis workflow and provider.
- Final artifact must be a markdown report (`suggestions.report.md`) saved in a deterministic location.
- Existing chain execution must remain unchanged when synthesis is not requested.

## Objective

Add an optional final synthesis stage to `ace-sim` that builds a suggestions report from simulation outputs and integrates it into run status and metadata.

## Scope of Work

- Added CLI options for final synthesis control:
  - `--synthesis-workflow`
  - `--synthesis-provider`
- Implemented a dedicated final stage executor that:
  - builds prompt context from chain step outputs,
  - invokes `ace-bundle` and `ace-llm`,
  - writes final artifact at `final/suggestions.report.md`.
- Wired final-stage execution into `SimulationRunner` after chain completion.
- Extended `session.yml` / `synthesis.yml` generation with final-stage metadata.
- Updated tests and E2E scenario to validate final-stage behavior and artifacts.
