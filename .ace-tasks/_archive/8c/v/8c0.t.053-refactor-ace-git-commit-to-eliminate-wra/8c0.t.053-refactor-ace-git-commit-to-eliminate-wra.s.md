# Refactor ace-git-commit to eliminate wrapper pattern

## Behavioral Specification

### User Experience
- **Input**: Users run `ace-git-commit` or `ace-git-commit -i "fix bug"`
- **Process**: Command executes correctly without wrapper layer
- **Output**: Commit created with LLM-generated message

### Expected Behavior

ace-git-commit uses wrapper pattern passing through to `CommitCommand`. Merge wrapper and logic into single command class.

**Commands to Refactor:**
```bash
ace-git-commit [options]    # Generate commit with LLM
```

### Success Criteria

- [ ] **Commit command refactored**: Wrapper merged with CommitCommand
- [ ] **Help text works**: `ace-git-commit --help` shows proper documentation
- [ ] **Tests pass**: All tests updated
- [ ] **No functional changes**: Command works exactly as before

## Objective

Eliminate wrapper pattern in ace-git-commit by merging wrapper and business logic into single dry-cli command class.

## Scope of Work

- **User Experience Scope**: ace-git-commit CLI command
- **System Behavior Scope**: Command execution, LLM integration

### Deliverables

#### Modify
- `ace-git-commit/lib/ace/git_commit/commands/commit.rb` - Merge business logic from CommitCommand
- `ace-git-commit/lib/ace/git_commit.rb` - Remove require for deleted file

#### Delete
- `ace-git-commit/lib/ace/git_commit/commands/commit_command.rb`

## Technical Approach

### Architecture Pattern
- **Pattern**: Direct dry-cli command implementation (no wrapper indirection)
- **Rationale**: Simplifies code structure, reduces file count, follows ace-timestamp pattern
- **Integration**: CLI registry (`cli.rb`) already registers `Commands::Commit`, no changes needed there

### Implementation Strategy

The current implementation has two layers:
1. `commit.rb` - dry-cli command wrapper that calls `CommitCommand.new(files, options).execute`
2. `commit_command.rb` - Instance-based business logic

**Target architecture:**
1. `commit.rb` - dry-cli command with all business logic inline

Key code migrations:
- Move `execute`, `display_config_summary`, `commit_options`, `load_effective_config`, `default_config` methods from `CommitCommand` into `Commit` class
- Convert instance methods to work within dry-cli `call` method context
- Remove `require_relative "commit_command"` statement

### Risk Assessment

#### Technical Risks
- **Risk:** Breaking command behavior during refactor
  - **Probability:** Low
  - **Impact:** High
  - **Mitigation:** Existing tests verify functionality; run tests after each change
  - **Rollback:** Git revert if issues found

#### Integration Risks
- **Risk:** Missing require statements after file deletion
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Grep for all references to `commit_command` before deletion
  - **Monitoring:** `ace-test` will fail if requires are broken
