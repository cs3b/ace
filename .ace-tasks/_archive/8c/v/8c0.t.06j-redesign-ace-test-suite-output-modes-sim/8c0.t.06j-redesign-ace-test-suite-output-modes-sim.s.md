# Redesign ace-test-suite output modes: simple default + --progress flag

## Behavioral Specification

### User Experience

- **Input**: User runs `ace-test-suite` with optional `--progress` flag
- **Process**:
  - Default (simple mode): User sees minimal, line-by-line output suitable for piping and agent consumption
  - Progress mode: User sees animated live progress bars with ANSI updates
- **Output**: Test results summary with pass/fail status and exit code

### Expected Behavior

**Default Mode (Simple):**
When running `ace-test-suite` without flags, the output should be:
- One line when tests start: "Running tests for N packages..."
- One line per package when it completes (printed in completion order)
- Summary section at the end (same format as current)
- No ANSI cursor control or screen clearing
- Clean output when piped (`ace-test-suite 2>&1 | cat` works cleanly)

**Progress Mode:**
When running `ace-test-suite --progress`:
- Live animated display with progress bars (current behavior)
- Skip redundant final results table (already shown inline during updates)
- Show only summary at the end

### Interface Contract

```bash
# Default: Simple line-by-line output (agent/pipe-friendly)
ace-test-suite
# Output:
# Running tests for 27 packages...
# [ace-support-core]      ✅ 100 tests, 200 assertions, 0 failures (2.34s)
# [ace-support-config]    ✅ 50 tests, 120 assertions, 0 failures (1.23s)
# [ace-bundle]            ❌ 285 tests, 775 assertions, 2 failures (9.83s)
# ...
# ═════════════════════════════════════════════════════════════════
# ✅ ALL TESTS PASSED!
#
# Packages:  27/27 passed, 0 failed
# Tests:     1500/1500 passed, 0 failed
# Duration:  45.23s (wall time)
# ═════════════════════════════════════════════════════════════════

# Progress mode: Live animated display
ace-test-suite --progress
# Shows live progress bars with ANSI updates (current behavior)
# Skips redundant final results table
# Shows summary at end

# Existing flags still work
ace-test-suite --parallel 16 --group foundation
ace-test-suite --progress --parallel 16
```

**Error Handling:**
- Failed package: Print line with ❌ icon and failure count
- Interrupted (Ctrl+C): Print partial results and exit 130
- No packages found: Print error message and exit 1

**Edge Cases:**
- Single package: Still shows "Running tests for 1 packages..."
- All packages fail: Summary shows failure count, exit 1
- Mixed results: Show all results, appropriate summary, exit 1

### Success Criteria

- [x] **Default output is simple**: Running `ace-test-suite` produces line-by-line output without ANSI cursor control
- [x] **Pipe-friendly**: `ace-test-suite 2>&1 | cat` produces clean, readable output
- [x] **Progress mode available**: `--progress` flag enables current animated display
- [x] **No redundant output**: Progress mode doesn't redraw final results table (already shown inline)
- [x] **Same exit codes**: Exit 0 for success, 1 for failures, 130 for interrupt
- [x] **Same summary format**: Summary section content unchanged in both modes

## Objective

The current animated ANSI output causes problems when piped or captured by agents. This task adds a simple line-by-line output mode as the default, making `ace-test-suite` agent-friendly while preserving the current animated display via `--progress` flag.

## Scope of Work

- **User Experience Scope**: Two output modes - simple (default) and progress (flag-enabled)
- **System Behavior Scope**: Display manager selection, output formatting, flag parsing
- **Interface Scope**: New `--progress` CLI flag, unchanged exit codes and summary format

### Deliverables

#### Create

- `ace-test-runner/lib/ace/test_runner/suite/simple_display_manager.rb`
  - New class for line-by-line output without ANSI cursor control
  - Methods: `initialize_display`, `update_package`, `refresh`, `show_final_results`, `show_summary`

#### Modify

- `ace-test-runner/exe/ace-test-suite`
  - Add `--progress` flag to OptionParser
  - Wire flag to config: `config["test_suite"]["progress"] = true`

- `ace-test-runner/lib/ace/test_runner/suite/orchestrator.rb`
  - Select display manager based on `config.dig("test_suite", "progress")`
  - Default to `SimpleDisplayManager`, use `DisplayManager` when `--progress`

- `ace-test-runner/lib/ace/test_runner/suite/display_manager.rb`
  - Modify `show_final_results` to skip redundant table (inline updates already show results)
  - Keep summary output unchanged

- `ace-test-runner/lib/ace/test_runner/suite.rb`
  - Add `require_relative "suite/simple_display_manager"`

## Out of Scope

- **Future Enhancements**: JSON output mode, verbose mode, other output formats
- **Auto-detection**: Automatic mode selection based on TTY (explicit flag only)
