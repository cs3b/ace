# Molecules — worktree provisioning, tmux, assignment, status collection

## Objective

Implement the molecule layer that composes sibling gem SDKs. Each molecule handles one I/O concern: creating worktrees, opening tmux windows, launching assignments, collecting status, and checking prune safety.

## Behavioral Specification

### Design Principle

All molecules call sibling gems via their **public Ruby SDK** (organisms/molecules). No internal file reading. Use `Dir.chdir(worktree_path)` + `reset_config!` to scope SDK calls to the correct worktree context.

### Molecules

**`WorktreeProvisioner`** — creates or finds a worktree for a task:
```ruby
provisioner.provision(task_ref)
# → { worktree_path: "/path/.ace-wt/task.230", branch: "230-...", created: true }
# If worktree already exists, returns it with created: false
```
Uses: `Ace::Git::Worktree::Organisms::WorktreeManager` (.switch, .create_task)

**`TmuxWindowOpener`** — opens a tmux window for a worktree:
```ruby
opener.open(worktree_path:, window_name: "t230", session_name: "ace", preset: "cc")
# → { window_name: "t230" }
# Creates session first if it doesn't exist (detached)
```
Uses: `Ace::Tmux::Organisms::{SessionManager, WindowManager}`

**`AssignmentLauncher`** — prepares and creates an assignment in a worktree:
```ruby
launcher.launch(worktree_path:, preset_name: "work-on-task", task_ref: "230")
# → { assignment_id: "8or5kx", first_phase: "010-onboard" }
# Dir.chdir(worktree_path) → reset_config! → load preset → expand → write job.yaml → executor.start
```
Uses: `Ace::Assign::Atoms::PresetExpander`, `Ace::Assign::Organisms::AssignmentExecutor`

**`WorktreeContextCollector`** — gathers status from one worktree:
```ruby
collector.collect(worktree_path)
# → Models::WorkContext with assignment_status, git_status filled in
# Dir.chdir(worktree_path) → reset_config! → executor.status + RepoStatusLoader.load
# Rescues NoActiveAssignmentError → assignment_status = nil
```
Uses: `Ace::Assign::Organisms::AssignmentExecutor`, `Ace::Git::Organisms::RepoStatusLoader`

**`PruneSafetyChecker`** — evaluates if a worktree is safe to remove:
```ruby
checker.check(worktree_path:, task_ref:)
# → Models::PruneCandidate with assignment_complete, task_done, git_clean, reasons
# Safe only when: assignment completed + task done + git clean
```

### Dir.chdir + reset_config! Pattern

Critical pattern used by AssignmentLauncher and WorktreeContextCollector:
```ruby
Dir.chdir(worktree_path) do
  Ace::Assign.reset_config!  # Clear memoized config → cache_dir resolves to worktree
  # ... SDK calls here ...
ensure
  Ace::Assign.reset_config!  # Restore for caller's context
end
```

### Success Criteria

- [ ] WorktreeProvisioner: creates new worktree OR returns existing one
- [ ] TmuxWindowOpener: creates session if needed, then adds window with correct name and root
- [ ] AssignmentLauncher: creates assignment in worktree's `.cache/ace-assign/` (not main tree's)
- [ ] WorktreeContextCollector: returns WorkContext with assignment + git status from correct worktree
- [ ] PruneSafetyChecker: correctly identifies safe-to-prune candidates (all 3 conditions)
- [ ] All molecules tested with mocked sibling SDK calls
- [ ] `ace-test ace-overseer` passes
