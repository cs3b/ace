# ace-overseer v0.1.0 — Project Control Plane

## Overview

ace-overseer is a project control plane that composes existing tools (ace-assign, ace-tmux, ace-git-worktree, ace-taskflow, ace-git) via their **public Ruby SDK interfaces** to automate multi-worktree task management.

Today, starting work on a task requires 4-5 manual steps: create worktree, open tmux window, prepare assignment, create assignment, start drive. There's also no single place to see what's happening across all active tasks. ace-overseer solves both.

**Boundary:** ace-assign owns single-worktree execution (phases, quality progression). ace-overseer owns multi-worktree setup, monitoring, and lifecycle.

**Architectural principle:** Compose via public SDK calls — never read sibling gem internals. If a tool doesn't expose what overseer needs, add it to that tool first.

### Commands (v0.1.0)

```bash
ace-overseer work-on --task 230 [--preset implement-with-pr]
# Creates worktree → opens tmux window → prepares + creates assignment → ready to drive

ace-overseer status
# Unified dashboard: all active worktrees with assignment progress, git state, release info

ace-overseer prune [--dry-run] [--yes]
# Safe cleanup of completed worktrees (assignment done + task done + git clean)
```

### Key Design Decisions

- **SDK calls via Dir.chdir** — not subprocess spawning. `Ace::Assign.cache_dir` uses `ProjectRootFinder.find_or_current` (Dir.pwd), so `Dir.chdir(worktree_path)` + `reset_config!` naturally scopes to each worktree
- **One tmux session per project** ("ace"), each task gets a window (e.g., "t230")
- **Preset resolution cascade:** task spec `assign:` frontmatter → `--preset` flag → config default
- **Safe prune:** only when assignment completed + task done + git clean + branch merged

## Out of Scope (v0.1.0)

- HIL (human-in-the-loop) gates — deferred to backlog
- Quality progression (code-valid/fit/shine) — ace-assign's responsibility
- Auto-drive (sending keys to tmux) — manual handoff for now
- Cross-repository orchestration
- Daemon/polling — status is always a snapshot
