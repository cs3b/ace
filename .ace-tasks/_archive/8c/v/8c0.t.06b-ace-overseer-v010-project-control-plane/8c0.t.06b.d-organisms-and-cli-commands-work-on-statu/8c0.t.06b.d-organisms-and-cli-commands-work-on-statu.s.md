# Organisms and CLI commands (work-on, status, prune)

## Objective

Implement the organism layer that orchestrates molecules into complete workflows, and the CLI commands that expose them. After this subtask, all three commands work end-to-end.

## Behavioral Specification

### Command 1: `ace-overseer work-on --task 230`

```bash
ace-overseer work-on --task 230
# Creating worktree for task 230...
# Worktree: /project/.ace-wt/task.230 (branch: 230-streamline-task-lifecycle)
# Opening tmux window t230...
# Preparing assignment (preset: work-on-task)...
# Assignment created: 8or5kx
#
# Ready! Switch to tmux window 't230' and run /ace:assign-drive

ace-overseer work-on --task 230 --preset fix-bug
# Same flow, using fix-bug preset instead of default
```

**Organism: `WorkOnOrchestrator`**
1. Load task metadata from ace-taskflow (check for `assign:` frontmatter)
2. Resolve preset: task frontmatter → `--preset` flag → config default
3. Provision worktree via WorktreeProvisioner (create or reuse)
4. Open tmux window via TmuxWindowOpener
5. Launch assignment via AssignmentLauncher
6. Print summary with next steps

**Edge cases:**
- Worktree already exists → reuse it, skip creation
- Assignment already exists in worktree → warn and skip (don't create duplicate)
- Tmux session doesn't exist → create it first (detached)

### Command 2: `ace-overseer status`

```bash
ace-overseer status
# ACE PROJECT STATUS
#
# Release: v.0.9.0 — 221/236 tasks done (94%)
#
# ACTIVE WORKTREES
# ┌──────┬────────────────────────────────┬───────────┬─────────────────┬──────────┐
# │ Task │ Branch                         │ Assign    │ Current Phase   │ Git      │
# ├──────┼────────────────────────────────┼───────────┼─────────────────┼──────────┤
# │ 230  │ 230-streamline-task-lifecycle  │ running   │ 020 work-on-task│ clean    │
# │ 243  │ 243-laser-review-presets       │ completed │ —               │ 1 dirty  │
# └──────┴────────────────────────────────┴───────────┴─────────────────┴──────────┘
#
# No active worktrees without tasks.

ace-overseer status --format json
# { "release": {...}, "worktrees": [...] }
```

**Organism: `StatusCollector`**
1. List all task-associated worktrees via `ace-git-worktree list` SDK
2. For each worktree, collect context via WorktreeContextCollector
3. Load release info via `ace-taskflow` SDK
4. Format via StatusFormatter atom (table or json)

### Command 3: `ace-overseer prune`

```bash
ace-overseer prune --dry-run
# Candidates for cleanup:
#   task.243 — assignment completed, task done, git clean ✓
#
# 1 worktree can be safely pruned.
# Run without --dry-run to remove.

ace-overseer prune
# The following worktrees will be removed:
#   task.243 (/project/.ace-wt/task.243)
# Continue? [y/N] y
# Removed worktree task.243
# Closed tmux window t243
# 1 worktree pruned.

ace-overseer prune --yes
# Skips confirmation prompt
```

**Organism: `PruneOrchestrator`**
1. Find candidates via PruneSafetyChecker for each task worktree
2. Filter to safe-only (assignment completed + task done + git clean)
3. If `--dry-run`: show candidates and exit
4. If not `--yes`: prompt for confirmation
5. For each candidate: remove worktree (WorktreeManager) + close tmux window
6. Print summary

**Safety:** Never prune worktrees with uncommitted changes, active assignments, or undone tasks. Show reasons why unsafe candidates were skipped.

### Success Criteria

- [ ] `ace-overseer work-on --task <ref>` creates worktree + tmux window + assignment in one command
- [ ] `ace-overseer work-on` reuses existing worktree if already created
- [ ] `ace-overseer work-on` resolves preset from task frontmatter when present
- [ ] `ace-overseer status` shows all active worktrees with assignment + git state
- [ ] `ace-overseer status --format json` outputs valid JSON
- [ ] `ace-overseer prune --dry-run` lists candidates without removing
- [ ] `ace-overseer prune` prompts for confirmation before removing
- [ ] `ace-overseer prune --yes` skips confirmation
- [ ] Prune refuses to remove worktrees with dirty git or active assignments
- [ ] `ace-test ace-overseer` passes — organisms and commands tested
