# Fix ace-taskflow doctor to respect configured archive folder

## Objective

Fix the `ace-taskflow doctor` command to use the configured archive directory name from ace-config instead of hardcoded `/done/` pattern. Currently it reports false positives for tasks in `_archive/` because it looks for `/done/` in file paths.

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-taskflow doctor`
- **Process**: Doctor scans task files and validates their status vs location
- **Output**: No false warnings about done tasks "not being in done/ directory" when they are correctly in `_archive/`

### Expected Behavior

When `ace-taskflow doctor` checks task locations:
1. Load the `directories.completed` config value from ace-config (default: `_archive`)
2. Use this configured value instead of hardcoded `/done/`
3. Correctly identify tasks as archived when they are in the configured directory

### Interface Contract

```bash
# Before fix (current broken behavior):
ace-taskflow doctor
# ⚠️ Task v.0.9.0+task.001 has status 'done' but is not in done/ directory
#    (.ace-taskflow/v.0.9.0/tasks/_archive/001-feat-core/task.001.s.md)

# After fix (correct behavior):
ace-taskflow doctor
# ✓ No warnings for archived tasks (they are in _archive/ as configured)
```

**Configuration Source (ADR-022):**
- Default: `.ace-defaults/taskflow/config.yml` → `directories.completed: "_archive"`
- Override: `.ace/taskflow/config.yml` → `directories.completed: "done"` (if user prefers)

## Scope of Work

### Deliverables

#### Modify

- `ace-taskflow/lib/ace/taskflow/molecules/doctor.rb` (or equivalent)
  - Changes: Replace hardcoded `/done/` check with config lookup
  - Use: `Ace::Taskflow.config.dig("directories", "completed")` or equivalent

### Technical Approach

1. Find where the "not in done/ directory" check is implemented
2. Replace hardcoded string with config value from ace-config
3. Ensure fallback to default `_archive` if config missing
