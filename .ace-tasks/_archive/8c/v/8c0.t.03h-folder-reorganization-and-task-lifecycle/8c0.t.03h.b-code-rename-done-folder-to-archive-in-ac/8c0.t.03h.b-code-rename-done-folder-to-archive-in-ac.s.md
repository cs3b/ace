# Code: Rename done folder to _archive in ace-taskflow

## Description

Rename the `done` folder in ace-taskflow to `_archive` to:
1. Avoid confusion with the task status "done"
2. Follow the `_` prefix convention for system/special folders
3. Sort system folders to the top in directory listings

Using `_archive` clarifies that this folder is for completed/archived tasks that are no longer active in the workflow, while the underscore prefix makes it visually distinct from content folders.

## Acceptance Criteria

- [ ] Update `configuration.rb`: Change `done_dir` default from `done` to `_archive`
- [ ] Update all code references from `done` folder to `_archive` folder
- [ ] Update all documentation references from "done folder" to "_archive folder"
- [ ] Update CLI commands and help text that reference the done folder
- [ ] Add backward compatibility: detect old `done/` folders when `_archive/` doesn't exist
- [ ] Log warning when old folder name detected (once per CLI invocation)
- [ ] All tests pass with the new folder name
- [ ] Update any examples or usage documentation

**Note**: Physical folder renames are handled in task 131.05 (data migration).

## Backward Compatibility Rules

### Precedence Order
When both `done/` and `_archive/` folders exist:
1. **`_archive/` takes priority** - use the new folder, ignore the old one
2. **No automatic merge** - contents remain separate until user runs migration

### Warning Behavior
- **Trigger**: When old `done/` folder is detected and `_archive/` doesn't exist
- **Format**: `[DEPRECATED] Using 'done/' folder. Run 'ace-taskflow migrate' to upgrade.`
- **Frequency**: Once per CLI invocation (not per file operation)
- **Implementation**: Use module-level flag to track if warning has been shown

### Test Cases for Mixed States
- [ ] Only `_archive/` exists → use normally, no warning
- [ ] Only `done/` exists → use with deprecation warning
- [ ] Both exist → use `_archive/`, log info about old folder being ignored
- [ ] Neither exists → create `_archive/` on first write

### Deprecation Timeline
- **v.0.9.0**: Old folder names deprecated with warnings
- **v.1.0.0**: Old folder name support removed (breaking change)
