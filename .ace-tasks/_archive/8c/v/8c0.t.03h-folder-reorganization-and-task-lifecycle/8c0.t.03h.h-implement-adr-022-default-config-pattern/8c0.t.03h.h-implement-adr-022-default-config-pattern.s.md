# Implement ADR-022 default config pattern in ace-taskflow

## Objective

Refactor ace-taskflow's configuration system to follow ADR-022 (Configuration Default and Override Pattern). Move from hardcoded `DEFAULT_CONFIG` to loading defaults from `.ace.example/taskflow/config.yml` at runtime, making the example config the single source of truth for defaults.

## Scope of Work

- Enhance `.ace.example/taskflow/config.yml` with all configuration options
- Update `ConfigLoader` to load gem defaults from `.ace.example/`
- Add tests for default config loading and merging behavior
- Remove hardcoded `DEFAULT_CONFIG` constant (keep only minimal fallback)
- Maintain backward compatibility with existing configs

## Technical Approach

### Architecture Pattern

The implementation follows the three-tier config pattern from ADR-022:
1. **Gem defaults**: `.ace.example/taskflow/config.yml` (loaded at runtime)
2. **User overrides**: `.ace/taskflow/config.yml` (cascade from home → project)
3. **CLI options**: Command-line flags (not changed in this task)

### Implementation Strategy

**Key insight**: The `load_file` method already extracts the `taskflow:` section from YAML. We can reuse this for loading gem defaults.

```ruby
def self.load
  config = load_gem_defaults  # NEW: Load from .ace.example/
  config_paths = find_config_paths

  config_paths.each do |path|
    if File.exist?(path)
      file_config = load_file(path)
      config = deep_merge(config, file_config)
    end
  end

  config
end

def self.load_gem_defaults
  gem_root = File.expand_path("../../../..", __dir__)
  default_file = File.join(gem_root, ".ace.example", "taskflow", "config.yml")

  if File.exist?(default_file)
    load_file(default_file)
  else
    MINIMAL_FALLBACK  # Only essential values if file missing
  end
end
```

### Caching

Defaults should be cached since `.ace.example/` doesn't change at runtime:

```ruby
def self.load_gem_defaults
  @gem_defaults ||= begin
    # ... load from file
  end
end

def self.reset_gem_defaults!
  @gem_defaults = nil
end
```

## Risk Assessment

### Technical Risks

- **Risk:** Gem root path detection fails in different environments
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Use `__dir__` relative path navigation
  - **Rollback:** Keep MINIMAL_FALLBACK as safety net

- **Risk:** Circular dependency during gem loading
  - **Probability:** Low
  - **Impact:** High
  - **Mitigation:** Load defaults lazily, cache result
  - **Rollback:** Revert to DEFAULT_CONFIG pattern

### Integration Risks

- **Risk:** Tests depend on specific DEFAULT_CONFIG values
  - **Probability:** Medium
  - **Impact:** Low
  - **Mitigation:** Review test assertions, update as needed
  - **Monitoring:** Run full test suite after each change
