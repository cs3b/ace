## Technical Approach

### Fixture Extraction Strategy

1. Parse all `cat > {path} << 'EOF' ... EOF` blocks from Test Data sections → `fixtures/{path}`
2. Preserve directory structure (e.g., `batch/`, `app/models/`, `invalid/`)
3. Files created dynamically in TC steps (not Test Data) remain as TC step commands

### Setup Step Mapping

| Pattern | scenario.yml step |
|---------|-------------------|
| Git repo needed | `git-init` |
| Heredoc test files | `copy-fixtures` |
| Subdirectory git repos | `run: git -C {subdir} init --quiet .` |
| Config files from TC-000 | `write-file:` |
| PROJECT_ROOT_PATH | `env: { PROJECT_ROOT_PATH: "." }` |

### TC Extraction Rules

- Strip `ace-test-e2e-sh "$TEST_DIR"` wrappers from commands
- Generate slug from title: lowercase, spaces→hyphens, remove special chars
- Omit `**Actual:**`, `**Status:**` checkboxes, and `---` separators
- Incorporate `**Note:**` addenda into Steps or Expected as appropriate

### Special Handling

- **MT-LINT-004 TC-000:** Config file creation → merge into setup as `write-file:` step
- **MT-LINT-005 & MT-LINT-008:** Subdirectories need git-init → `run:` commands in setup
- **MT-LINT-007:** Config files created in TC steps → keep as TC commands (not setup)
- **MT-LINT-008 TC-004:** Document known issue (exit code bug in ace-lint doctor)
