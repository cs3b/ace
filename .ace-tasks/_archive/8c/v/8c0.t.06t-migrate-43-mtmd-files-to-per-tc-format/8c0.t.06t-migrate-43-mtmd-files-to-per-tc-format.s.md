# Migrate 43 .mt.md Files to Per-TC Format

## Overview

Convert all existing `.mt.md` E2E test scenario files across 10 packages to the new directory-based per-TC format (`TS-*/scenario.yml` + `fixtures/` + `TC-*.tc.md`), **filtered through the E2E Value Gate**. Each subtask runs the 3-stage pipeline (review → plan → execute) before migrating, combining format conversion and test restructuring into one pass.

**Empirical basis:** The ace-lint migration (262.01) proved that 1:1 format conversion is wasteful — a faithful migration of 31 TCs was immediately followed by restructuring down to 9 TCs (~71% reduction, ~60% duplicated unit test coverage). The 3-stage pipeline (v0.14.0) now exists to avoid this wasted intermediate step.

### Migration Pattern

Each subtask follows a two-phase approach:

**Phase 0: Coverage Review & Change Plan**
1. Run `/ace:review-e2e-tests {PACKAGE}` — produces coverage matrix
2. Run `/ace:plan-e2e-changes {PACKAGE}` — classifies each TC as REMOVE / KEEP / MODIFY / CONSOLIDATE / ADD
3. Approve plan before proceeding

**Phase 1: Informed Migration** (only for Value Gate-passing TCs)
1. Create directory: `test/e2e/TS-{AREA}-{NNN}-{slug}/`
2. Extract metadata → `scenario.yml` (test-id, title, area, package, priority, requires)
3. Convert setup heredocs → `setup:` steps in `scenario.yml` + real files in `fixtures/`
4. Extract each approved TC → individual `TC-{NNN}-{slug}.tc.md` file
5. Validate: `ace-test-e2e setup <package> <scenario-id>` succeeds
6. Remove original `.mt.md` file

### Special Case: ace-coworker

The ace-coworker package has **sequential test dependencies** that must be handled differently:

- MT-COWORKER-002: 8 sequential TCs → merge into single fork context TC
- MT-COWORKER-003a/b/c/d: mixed independent + sequential
