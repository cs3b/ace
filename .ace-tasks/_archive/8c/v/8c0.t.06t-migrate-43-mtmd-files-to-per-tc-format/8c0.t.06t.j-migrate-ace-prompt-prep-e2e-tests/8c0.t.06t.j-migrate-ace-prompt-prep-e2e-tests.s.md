# Migrate ace-prompt-prep E2E Tests

## Scope

Convert `.mt.md` files in `ace-prompt-prep/test/e2e/` to directory-based per-TC format, **filtered through the E2E Value Gate**. The migration inventory below shows the source material; actual TC count will be determined by Phase 0's coverage review. Smallest migration task.

**Source material:** 1 `.mt.md` file, 5 TCs (pre-review)
**Expected output:** Fewer TCs after Value Gate filtering and consolidation

## Phase 0: Coverage Review & Change Plan

Before migrating, run the 3-stage pipeline to determine which TCs have genuine E2E value:

1. **Review:** `/ace:review-e2e-tests ace-prompt-prep` — produces coverage matrix (functionality × unit tests × E2E)
2. **Plan:** `/ace:plan-e2e-changes ace-prompt-prep` — classifies each existing MT TC as REMOVE / KEEP / MODIFY / CONSOLIDATE / ADD
3. **Approve plan** before proceeding to migration

The ace-lint migration showed ~60% of TCs duplicate unit test coverage. Expect significant TC reduction.

### Migration scope is determined by the plan:
- **KEEP TCs** → migrate to TS format as-is
- **MODIFY TCs** → migrate to TS format with changes applied
- **CONSOLIDATE TCs** → merge during migration (single TS TC from multiple MT TCs)
- **REMOVE TCs** → skip (don't create TS files, delete MT files)
- **ADD TCs** → create new TS TCs for uncovered features (if any)

## Implementation Plan

### Phase 0: Review & Plan
- [ ] Run `/ace:review-e2e-tests ace-prompt-prep` — capture coverage matrix
- [ ] Run `/ace:plan-e2e-changes ace-prompt-prep` — get TC classifications
- [ ] Present plan for approval (expected: N REMOVE, N KEEP, N MODIFY, N CONSOLIDATE, N ADD)

### Phase 1: Informed Migration
- [ ] For each KEEP/MODIFY TC: extract to TS format following migration-pattern.md
- [ ] For each CONSOLIDATE group: merge into single TC during extraction
- [ ] For each ADD: create new TS TC based on plan
- [ ] Validate: `ace-test-e2e setup ace-prompt-prep TS-PREP-{NNN}` for each scenario
- [ ] Remove all original `.mt.md` files
- [ ] Final verification: scenario count, TC count, no stale references

## Acceptance Criteria

- [ ] Phase 0 coverage review completed and change plan approved
- [ ] All KEEP/MODIFY TCs migrated to TS format with setup validation passing
- [ ] CONSOLIDATE groups merged into single TCs during migration
- [ ] REMOVE TCs skipped — no TS equivalent created
- [ ] All original `.mt.md` files removed after successful migration
- [ ] Fixture files are real files on disk (not heredoc creation commands)
- [ ] Final scenario count: 2-5 TCs per scenario (cost-aware guideline)
- [ ] `ace-test-e2e-sh` wrapper calls removed from TC steps

## Out of Scope

- ❌ Changes to ace-test-e2e-runner code
