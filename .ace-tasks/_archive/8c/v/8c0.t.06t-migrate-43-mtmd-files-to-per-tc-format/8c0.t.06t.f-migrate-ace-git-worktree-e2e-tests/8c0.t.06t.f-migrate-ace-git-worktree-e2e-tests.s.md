# Migrate ace-git-worktree E2E Tests

## Scope

Convert `.mt.md` files in `ace-git-worktree/test-e2e/scenarios/` to directory-based per-TC format, **filtered through the E2E Value Gate**. The migration inventory below shows the source material; actual TC count will be determined by Phase 0's coverage review.

**Source material:** 2 `.mt.md` files, 21 TCs (pre-review)
**Expected output:** Fewer scenarios/TCs after Value Gate filtering and consolidation

## Phase 0: Coverage Review & Change Plan

Before migrating, run the 3-stage pipeline to determine which TCs have genuine E2E value:

1. **Review:** `/ace:review-e2e-tests ace-git-worktree` — produces coverage matrix (functionality × unit tests × E2E)
2. **Plan:** `/ace:plan-e2e-changes ace-git-worktree` — classifies each existing MT TC as REMOVE / KEEP / MODIFY / CONSOLIDATE / ADD
3. **Approve plan** before proceeding to migration

The ace-lint migration showed ~60% of TCs duplicate unit test coverage. Expect significant TC reduction.

### Migration scope is determined by the plan:
- **KEEP TCs** → migrate to TS format as-is
- **MODIFY TCs** → migrate to TS format with changes applied
- **CONSOLIDATE TCs** → merge during migration (single TS TC from multiple MT TCs)
- **REMOVE TCs** → skip (don't create TS files, delete MT files)
- **ADD TCs** → create new TS TCs for uncovered features (if any)

## Technical Approach

### TS-WORKTREE-001 (git-only, no fixtures)

```yaml
setup:
  - git-init
  - run: |
      git checkout -b feature/test-worktree --quiet
      echo "Feature content" > feature.txt
      git add feature.txt && git commit -m "Add feature" --quiet
      git checkout main --quiet
      git checkout -b bugfix/test-fix --quiet
      echo "Bugfix content" > bugfix.txt
      git add bugfix.txt && git commit -m "Add bugfix" --quiet
      git checkout main --quiet
  - env:
      PROJECT_ROOT_PATH: .
```

No `fixtures/` directory needed — all test data is git structure.

### TS-WORKTREE-002 (taskflow fixtures)

```yaml
setup:
  - git-init
  - copy-fixtures
  - run: |
      mkdir -p .ace-wt
      git add .ace-taskflow/ .ace/ && git commit -m "Add taskflow" --quiet
  - env:
      PROJECT_ROOT_PATH: .
```

Fixtures include nested `.ace-taskflow/v.test/tasks/` structure with task spec files.
