# Optimize ace-git test performance (11s to under 5s)

**Note**: Original target was 11s → <5s. Previous optimizations reduced this to 6.55s baseline. This PR achieves 6.55s → 4.99s.

## Behavioral Specification

### User Experience
- **Input**: Developer runs `ace-test ace-git` to execute test suite
- **Process**: Tests execute with consolidated Open3 mocking and optimized timeout tests
- **Output**: Full test results in <5 seconds (achieved 4.99s, down from 6.55s baseline)

### Expected Behavior
Developers experience faster test execution. Open3 mock setup is consolidated. Timeout tests use mock delays instead of real sleep.

### Success Criteria

- [x] Test suite runs in <5 seconds (achieved 4.99s, down from 6.54s)
- [x] All 446 tests pass
- [x] Open3 mocking consolidated
- [x] No real sleep(10) in tests

## Results

**Performance Achievement: 4.99s (24% reduction from 6.54s baseline)**

- Organisms layer: 4.82s → 3.19s (34% reduction)
- All 446 tests pass with 1053 assertions
- No reduction in test coverage
- Tests remain readable and maintainable

## Implementation

Created consolidated stub helpers to reduce deep nesting overhead:

1. **`repo_status_loader_test.rb`**: Added `with_mock_repo_load` helper
   - Replaces 6-7 levels of nested stubs with single helper call
   - Supports overrides for repository_type, usable?, repository_state, branch_info, task_pattern, prs, git_status, commits
   - All 27 tests refactored to use new helper

2. **`diff_orchestrator_test.rb`**: Added `with_mock_diff_orchestrator` helper
   - Replaces repeated stub nesting for ConfigLoader, DiffGenerator, DiffFilter
   - Uses single temp directory created in setup, cleaned in teardown
   - Eliminated per-test `Dir.mktmpdir` calls

## Objective

Reduce ace-git test execution time by 55%+ (from 11s to <5s) by consolidating Open3 mock setup and optimizing timeout tests.

### Optimization Strategy

#### Option 1: Optimize `repo_status_loader_test.rb` (Recommended)
The 27 tests in `repo_status_loader_test.rb` have deep nesting (6-7 levels of stubs). Each test creates a new stub stack:

```ruby
# Current pattern - slow due to deep stub nesting
Ace::Git::Atoms::RepositoryChecker.stub :repository_type, :normal do
  Ace::Git::Atoms::RepositoryChecker.stub :usable?, true do
    Ace::Git::Atoms::RepositoryStateDetector.stub :detect, :clean do
      # ... 4 more levels
    end
  end
end
```

**Optimization**: Create a test helper method that pre-stubs common dependencies:

```ruby
def with_mock_repo_load(context: @mock_context, **overrides)
  stubs = {
    repository_type: :normal,
    usable?: true,
    repository_state: :clean,
    branch_info: @mock_branch_info,
    task_pattern: nil,
    prs: @empty_prs
  }.merge(overrides)

  Ace::Git::Atoms::RepositoryChecker.stub :repository_type, stubs[:repository_type] do
    Ace::Git::Atoms::RepositoryChecker.stub :usable?, stubs[:usable?] do
      # ... reduced nesting
    end
  end
end
```

#### Option 2: Replace `Dir.mktmpdir` with in-memory mocks
The `diff_orchestrator_test.rb` creates actual temporary directories. Replace with a mock file system.

#### Option 3: Split test files
If individual files are too large, split `repo_status_loader_test.rb` into:
- `repo_status_loader_basic_test.rb` (core load behavior)
- `repo_status_loader_pr_test.rb` (PR-related tests)
- `repo_status_loader_activity_test.rb` (activity tests)
