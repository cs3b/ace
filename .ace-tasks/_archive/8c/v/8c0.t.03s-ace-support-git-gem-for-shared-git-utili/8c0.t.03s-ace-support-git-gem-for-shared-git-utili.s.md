# ace-support-git gem for shared git utilities

## Behavioral Specification

### User Experience
- **Input**: Developers using ace-* gems need to access git repository information (branch names, worktree status, repository state)
- **Process**: Call standardized git utility functions from a shared library instead of duplicating code
- **Output**: Consistent, reliable git information across all ace-* gems with reduced code duplication

### Expected Behavior

Developers using ace-prompt, ace-review, and other ace-* gems should be able to:
- Query git branch information using a shared, tested implementation
- Detect worktree status consistently across all gems
- Execute git commands with standardized helpers
- Access repository state information through a unified interface

This consolidates currently duplicated git utilities (GitBranchReader, worktree detection, etc.) into a single, well-tested gem that all ace-* tools can depend on.

### Interface Contract

```ruby
# Gem API Interface
require 'ace/support/git'

# Branch name reading
branch_name = Ace::Support::Git.current_branch
# => "main" or "feature/my-branch"

# Worktree detection
is_worktree = Ace::Support::Git.worktree?
# => true or false

# Repository state detection
state = Ace::Support::Git.repository_state
# => :clean, :dirty, :rebasing, etc.

# Git command execution helpers
result = Ace::Support::Git.execute('status', '--short')
# => Command output with proper error handling
```

**Error Handling:**
- **Not in git repository**: Raise clear exception with actionable message
- **Git command failure**: Capture and report stderr with exit status
- **Permission issues**: Provide clear error about repository access

**Edge Cases:**
- **Detached HEAD state**: Return appropriate identifier
- **Nested worktrees**: Detect and handle correctly
- **Bare repositories**: Handle gracefully or indicate unsupported

## Objective

Consolidate duplicated git utility code across ace-* gems into a single, well-tested shared library (ace-support-git). This reduces maintenance burden, ensures consistent behavior, and provides a reliable foundation for git operations across the entire ace toolkit.
