# Add parallel E2E test execution with subagents

## Context

With the enhanced report structure (task 244) enabling suite-level report aggregation, the opportunity arose to create a parallel test execution capability using Claude Code's Task subagent feature. This allows running multiple E2E tests concurrently with aggregated results.

## Objective

Create an orchestrator skill that enables running multiple E2E tests in parallel using Task subagents, with aggregated suite-level reporting.

## Description

Implement a new `/ace:run-e2e-tests` orchestrator skill (plural form) that spawns parallel Task subagents for concurrent E2E test execution. Each subagent runs a single test using the existing `/ace:run-e2e-test` worker skill and returns structured results for suite aggregation.

### Key Features

1. **Parallel execution**: Multiple tests run concurrently via Task tool
2. **Subagent delegation**: Each test runs in isolated subagent context
3. **Structured return contract**: Worker skill returns machine-parseable results
4. **Suite aggregation**: Orchestrator collects and summarizes all results

## Implementation Notes

### Orchestrator Pattern

The orchestrator uses Claude Code's Task tool to spawn subagents:

```markdown
## Execution Pattern

For each test in the suite:
1. Spawn Task subagent with `subagent_type: general-purpose`
2. Instruct subagent to run `/ace:run-e2e-test {test-path}`
3. Subagent returns structured result block
4. Orchestrator aggregates results into suite report
```

### Subagent Return Contract

Worker skill returns structured XML block for parsing:

```xml
<e2e-test-result>
  <test-id>{test-id}</test-id>
  <status>pass|fail</status>
  <summary-path>{path}</summary-path>
  <experience-path>{path}</experience-path>
</e2e-test-result>
```

### Usage

```
/ace:run-e2e-tests ace-git-commit/test-e2e/scenarios/*.e2e.md
```

Runs all matching tests in parallel and produces aggregated suite report.
