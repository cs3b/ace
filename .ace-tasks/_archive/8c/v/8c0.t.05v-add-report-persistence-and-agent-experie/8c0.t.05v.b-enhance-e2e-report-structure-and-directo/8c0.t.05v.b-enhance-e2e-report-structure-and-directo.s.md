# Enhance E2E report structure and directory convention

## Context

During implementation of task 221 (E2E runner report persistence), it became clear that the existing directory structure needed improvement to support better report organization and multi-test suite runs. This work emerged organically from the parent task requirements.

## Objective

Improve E2E test directory naming conventions and report file locations to provide:
- Clear namespace identification (`ace-test-e2e` prefix)
- Unique sandbox identification via test IDs
- Persistent reports outside sandboxes for suite aggregation
- Consistent conventions across all E2E tests

## Description

Restructure E2E test cache directories and report output locations to support better organization, discoverability, and suite-level report aggregation.

### Changes Made

1. **Cache directory renamed**: `test-e2e` → `ace-test-e2e` for clear namespace
2. **Sandbox naming enhanced**: Added test ID to folder names (`{timestamp}-{package}-{test-id}/`)
3. **Reports moved outside sandbox**: Reports now saved as sibling files to sandbox directory
   - `.summary.r.md` - Human-readable summary report
   - `.experience.r.md` - Agent experience report
   - `.metadata.yml` - Structured metadata for tooling
4. **Suite final report**: Added generation for multi-test runs

## Implementation Notes

The new directory structure follows this pattern:

```
~/.cache/ace-test-e2e/
├── {timestamp}-{package}-{test-id}/     # Sandbox directory
│   ├── repo/                            # Test repository
│   └── ...                              # Other sandbox contents
├── {timestamp}-{package}-{test-id}.summary.r.md
├── {timestamp}-{package}-{test-id}.experience.r.md
└── {timestamp}-{package}-{test-id}.metadata.yml
```

This structure enables:
- Easy cleanup of sandboxes while preserving reports
- Suite-level report aggregation by scanning sibling report files
- Clear association between reports and their source sandboxes
