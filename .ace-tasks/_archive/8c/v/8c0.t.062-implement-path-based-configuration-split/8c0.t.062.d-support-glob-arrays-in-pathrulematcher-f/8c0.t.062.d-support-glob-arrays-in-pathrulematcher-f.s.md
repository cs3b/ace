# Support glob arrays in PathRuleMatcher for simplified ace-config grouping

## Behavioral Specification

### User Experience

- **Input**: Path rules config with `glob:` as either a string OR an array of strings
- **Process**: PathRuleMatcher checks file against all globs in the array (OR logic)
- **Output**: Match if file matches ANY of the provided globs

### Expected Behavior

As a developer configuring path-based commit splitting, I want to specify multiple glob patterns for a single scope, so that I can group related files from different locations (like all `.ace/**` folders across packages) into one commit group.

**Current limitation**: `glob:` only accepts a single string pattern.

**New capability**: `glob:` accepts string OR array of strings.

### Interface Contract

```yaml
# .ace/git/commit.yml - BEFORE (single glob)
paths:
  ace-config:
    glob: ".ace/**"

# .ace/git/commit.yml - AFTER (glob array)
paths:
  ace-config:
    glob:
      - ".ace/**"           # Root .ace folder
      - "*/.ace/**"         # Direct child packages
      - "**/.ace/**"        # Any nested .ace folder
    type_hint: chore
    description: "All ACE configuration files"
```

**Ruby API (unchanged):**
```ruby
matcher = PathRuleMatcher.new(path_rules, project_root: project_root)
result = matcher.match("ace-bundle/.ace/git/commit.yml")
# => MatchResult(name: "ace-config", config: {...})
```

**Error Handling:**
- Empty array: No match (same as empty string)
- Mixed types in array: Convert all to strings
- Nil in array: Skip nil entries

### Success Criteria

- [ ] `glob:` accepts a single string (backward compatible)
- [ ] `glob:` accepts an array of strings (new capability)
- [ ] File matches if ANY glob in array matches (OR logic)
- [ ] First matching rule still wins (existing behavior)
- [ ] All existing PathRuleMatcher tests pass
- [ ] New tests cover array glob scenarios

### Validation Questions

- [x] Should array globs use OR or AND logic? → OR (match any)
- [x] Empty array behavior? → No match
- [x] Order of globs in array matter? → No, any match wins

## Objective

Enable simplified `.ace/**` file grouping by allowing multiple glob patterns per path rule, eliminating the need for ProjectConfigScanner and CommitGrouper modifications (tasks 228.02 and 228.03).

**This is the pragmatic solution**: Configuration-only change instead of new code.

## Technical Approach

### Architecture Pattern

- **Layer**: Atom modification (pure function, no side effects)
- **Pattern**: Extend existing glob matching with array iteration
- **Integration**: Fully backward compatible

### Implementation Strategy

The change is minimal:
1. In `match` method, check if `glob` is an Array
2. If array: iterate and return on first match
3. If string: existing behavior (unchanged)

**Key insight**: This solves the original problem (`.ace/**` files grouping) with ~10 lines of code instead of ~200 lines across two new components.
