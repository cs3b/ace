# ace-retro Doctor Command

## Behavioral Specification

### User Experience
- **Input**: `ace-retro doctor` (optionally with `--auto-fix`, `--check`, `--verbose`)
- **Process**: RetroDoctor organism runs health checks across all retros in `.ace-retros/`, reports issues, optionally auto-fixes safe problems
- **Output**: Health report with score, issue list, fix summary

### Expected Behavior

Following ace-idea's IdeaDoctor pattern (`ace-idea/lib/ace/idea/organisms/idea_doctor.rb`), build a comprehensive health check system for retros:

1. **Structure checks**: Folder naming conventions, file naming (`*.retro.md`), orphan files (files in retro folder that aren't the main `.retro.md`), empty directories, backup files
2. **Frontmatter checks**: Required fields present (`id`, `title`, `type`, `status`, `created_at`), valid field values, valid b36ts ID format
3. **Scope/status consistency**: Archived retros should have `status: done`, active retros shouldn't be in `_archive/`
4. **B36TS ID validation**: Folder name matches frontmatter `id:`, ID format matches `\A[0-9][0-9a-z]{5}` pattern
5. **Duplicate detection**: No two retros share the same b36ts ID
6. **Auto-fix**: Safe fixes for common issues â€” add missing `status:` field, add missing `created_at:` from git log, fix archive path structure

### Interface Contract

```bash
# Full health check
ace-retro doctor

# Auto-fix safe issues
ace-retro doctor --auto-fix

# Run specific check only
ace-retro doctor --check structure
ace-retro doctor --check frontmatter
ace-retro doctor --check scope

# Verbose output
ace-retro doctor --verbose
```

### Ruby API

```ruby
doctor = Ace::Retro::Organisms::RetroDoctor.new(root_path, options)
result = doctor.run_diagnosis
# => { valid: true/false, health_score: 0-100, issues: [...], stats: {...}, duration: ... }

# Auto-fix check
doctor.auto_fixable?(issue) # => true/false
```

### Success Criteria

- [ ] `ace-retro doctor` scans all retros and reports issues with severity levels (error, warning, info)
- [ ] Health score computed (100 minus weighted deductions for errors/warnings)
- [ ] Structure check catches: orphan files, empty dirs, bad folder naming, backup files
- [ ] Frontmatter check catches: missing required fields, invalid field values, malformed YAML
- [ ] Scope check catches: status/folder inconsistency (done retro not in archive, active retro in archive)
- [ ] B36TS validation catches: invalid ID format, ID mismatch between folder and frontmatter, duplicates
- [ ] `--auto-fix` repairs safe issues (missing fields, simple path fixes) without data loss
- [ ] `--check` runs only the specified check type
- [ ] `--verbose` shows detailed per-file output
- [ ] Exit code 0 when healthy, non-zero when errors found
- [ ] Full test coverage at atoms/molecules/organisms layers
