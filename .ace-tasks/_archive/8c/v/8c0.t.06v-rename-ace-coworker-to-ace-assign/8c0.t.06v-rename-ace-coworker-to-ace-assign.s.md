## Objective

Align the package name with its function. "ace-coworker" describes *who* does the work; "ace-assign" describes *what* the tool does — you **assign** work to an agent. The internal concepts also rename for collision avoidance — "session" and "job" are overloaded in tech (CI/CD, HTTP, cron), "step" conflicts with pipeline/workflow step. The new terms **assignment** and **phase** are distinctive and unambiguous. Additionally, combine three separate skills (prepare → create → drive) with a new `/ace:assign-start` entry point.

## Behavioral Specification

### User Experience

- **Input**: No user input required — internal rename with identical CLI interface
- **Process**: All CLI commands change from `ace-coworker <cmd>` to `ace-assign <cmd>`. Ruby module `Ace::Coworker` → `Ace::Assign`. Internal "session" → "assignment", "step" → "phase". Skills `/ace:coworker-*` → `/ace:assign-*`. New combined `/ace:assign-start` skill.
- **Output**: Identical functionality under clearer naming

### Interface Contract

```bash
# CLI (renamed from ace-coworker)
ace-assign create job.yaml        # Create assignment from YAML spec
ace-assign status [--flat]        # Show current queue state
ace-assign report <file>          # Complete current phase with report
ace-assign fail --message "msg"   # Mark current phase as failed
ace-assign add <name> [opts]      # Add phase dynamically
ace-assign retry <phase-ref>      # Retry failed phase
```

```bash
# Skills (renamed from /ace:coworker-*)
/ace:assign-start [preset] [args]  # NEW: Combined prepare + create
/ace:assign-prepare [preset]       # Prepare job.yaml from preset
/ace:assign-create job.yaml        # Create assignment from YAML
/ace:assign-drive                  # Drive execution through active assignment
```
