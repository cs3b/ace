# Migrate to global test_mode with per-test opt-out

## Objective

Complete the ace-config test_mode rollout by flipping the default: enable test_mode globally and use `with_real_config` blocks for tests that need filesystem access. This completes the work started in 157.12.

## Technical Approach

### Architecture Analysis

Test infrastructure hierarchy:
- **BaseTestCase** (ace-support-test-helpers): Inherited by packages that opted-in to shared test support
- **Per-package TestCase**: Some packages (ace-config, ace-review, etc.) define their own Minitest::Test subclasses

Packages extending BaseTestCase/AceTestCase (will get global test_mode automatically):
- ace-llm, ace-llm-models-dev, ace-search, ace-taskflow, ace-git, ace-git-commit

Packages with own TestCase (won't inherit from BaseTestCase):
- ace-config (needs real config for all tests - OK)
- ace-review, ace-nav, ace-git-worktree, ace-git-secrets, ace-support-fs, ace-handbook, ace-integration-claude, ace-test-runner

### Strategy

1. Enable test_mode in BaseTestCase.setup - affects packages using shared test infrastructure
2. Remove per-package `Ace::Config.test_mode = true` lines and "Note:" comments
3. Add `with_real_config` blocks to tests that verify config loading behavior
4. Tests in packages not using BaseTestCase are unaffected (they don't inherit the global setting)

## Scope

### In Scope
- Enable test_mode globally in BaseTestCase setup
- Remove redundant per-package test_mode lines (5 packages)
- Remove "Note:" comments from packages that couldn't use test_mode (5 packages)
- Wrap config-testing tests with `with_real_config` blocks
- Ensure all 4705 tests pass

### Out of Scope
- Performance optimizations beyond config loading
- Changes to ace-config test_mode implementation itself
- Packages not using BaseTestCase (they continue working as-is)

## Implementation Notes

The `with_real_config` helper (ace-support-test-helpers/lib/ace/test_support/config_helpers.rb):
```ruby
def with_real_config
  original_test_mode = Ace::Config.test_mode
  Ace::Config.test_mode = false
  yield
ensure
  Ace::Config.test_mode = original_test_mode
end
```

Pattern for config-testing tests:
```ruby
def test_config_cascade_hierarchy
  with_real_config do
    # Test that needs real filesystem config access
    with_temp_config(".ace" => { "config.yml" => "key: value" }) do
      config = Ace::Config.create.resolve
      assert_equal "value", config.get("key")
    end
  end
end
```
