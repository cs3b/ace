# Design Generic API for ace-config

## Objective

Design the public API for ace-config that makes folder names configurable, enabling use outside the ace ecosystem.

## Target API

```ruby
# Main entry point - Factory method (not constructor, to avoid Ruby idiom confusion)
module Ace
  module Config
    def self.create(**options)
      Organisms::ConfigResolver.new(**options)
    end
  end
end
```

**Implementation Note**: `Ace::Config.create` is a convenience factory method that returns an instance of `Ace::Config::Organisms::ConfigResolver`. This keeps the public API clean (`Ace::Config.create`) while the internal implementation uses the Organism layer. Users interact with the resolver through methods like `.resolve`, `.resolve_for`, and `.get`.

```ruby
# Factory options
config = Ace::Config.create(
  config_dir: ".ace",              # User config folder (default: ".ace")
  defaults_dir: ".ace-defaults",   # Gem defaults folder (default: ".ace-defaults")
  gem_path: GEM_ROOT,              # Optional: gem root for defaults
  project_markers: [".git", "Gemfile", "package.json"],  # Optional: root detection
  merge_strategy: :replace         # Optional: array merge strategy
)

# Resolution methods
config.resolve                     # Full cascade resolution → Config model
config.resolve_for(["file.yml"])   # Specific file patterns
config.get("key", "nested")        # Direct value access

# Finder API (lower level)
finder = Ace::Config::Finder.new(config_dir: ".my-app")
finder.find_file("settings.yml")
finder.find_all_files("settings.yml")

# Path resolution
expander = Ace::Config::PathExpander.for_file(source_file)
expander.resolve("./relative")
```

## Cascade Priority

```
1. $CWD/{config_dir}/              # Highest (priority 0)
2. ... intermediate dirs ...        # Between CWD and PROJECT_ROOT (priority 10, 20, ...)
3. $PROJECT_ROOT/{config_dir}/     # Project-level
4. $HOME/{config_dir}/             # User preferences (priority 1000)
5. $GEM_PATH/{defaults_dir}/       # Gem defaults (LOWEST - priority 2000)
```
