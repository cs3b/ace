## Target API

```ruby
# Generic configuration cascade
config = Ace::Config.create(
  defaults_dir: ".my-defaults",  # Gem defaults folder name
  config_dir: ".my-config",      # User config folder name
  gem_path: GEM_ROOT,            # Optional: for gem defaults
  project_markers: [".git", "Gemfile"]  # Optional: customize root detection
)

config.resolve                     # Returns Config model
config.resolve_for(["settings.yml"])  # Specific file patterns
config.get("key", "nested")        # Direct value access
```

## Cascade Order (configurable)

```
1. $CWD/{config_dir}/              # Highest priority
2. ... intermediate dirs ...        # All dirs between CWD and PROJECT_ROOT
3. $PROJECT_ROOT/{config_dir}/      # Project-level
4. $HOME/{config_dir}/              # User preferences
5. $GEM_PATH/{defaults_dir}/        # Gem defaults (LOWEST - fallback only)
```

## Components to Extract

### Move to ace-config

**Atoms:**
- `DeepMerger` - Pure deep merge functions
- `YamlParser` - YAML parsing utilities
- `PathExpander` - Path resolution with protocols

**Molecules:**
- `ConfigFinder` - Find config files in cascade
- `DirectoryTraverser` - Walk directory tree
- `ProjectRootFinder` - Detect project root via markers
- `YamlLoader` - Load YAML files

**Organisms:**
- `ConfigResolver` - Full cascade resolution
- `VirtualConfigResolver` - Virtual filesystem view

**Models:**
- `CascadePath` - Cascade path with priority
- `Config` - Configuration data structure

**Errors:**
- `ConfigNotFoundError`, `YamlParseError`, `PathError`, `MergeStrategyError`

### Keep in ace-support-core (ace-specific)

- `ConfigTemplates` - Scans for ace-* gems
- `ConfigInitializer` - Initializes from gem examples
- `ConfigDiff` - Compares local vs gem configs
- `EnvironmentManager` - .env file handling
- `EnvLoader`, `EnvParser` - Environment utilities
