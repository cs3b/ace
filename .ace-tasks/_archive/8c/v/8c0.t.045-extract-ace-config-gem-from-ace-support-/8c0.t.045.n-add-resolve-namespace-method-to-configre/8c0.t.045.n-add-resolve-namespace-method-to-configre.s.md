# Add resolve_namespace() method to ConfigResolver

## Objective

Add a new `resolve_namespace()` method to ConfigResolver that accepts path segments and automatically handles .yml/.yaml extension patterns. This simplifies gem config loading by centralizing extension handling and reducing boilerplate across ace-* gems.

## Behavioral Specification

### User Experience

- **Input**: Path segments like `("docs", "config")` or `("lint", "kramdown")`
- **Process**: Method builds file patterns by combining segments with .yml/.yaml extensions
- **Output**: Returns `Models::Config` with merged configuration from cascade

### Interface Contract

```ruby
# Current pattern (explicit, repetitive)
resolver.resolve_for(["docs/config.yml", "docs/config.yaml"])

# Proposed pattern (path segments)
resolver.resolve_namespace("docs", "config")  # Auto-adds .yml/.yaml

# Also support single segment with default filename
resolver.resolve_namespace("lint")  # Looks for lint/config.yml, lint/config.yaml

# Custom filename override
resolver.resolve_namespace("lint", filename: "kramdown")  # lint/kramdown.yml, lint/kramdown.yaml
```

### Expected Behavior

1. Accepts 1-N path segments as arguments
2. Uses "config" as default filename if not specified
3. Automatically generates patterns: `["path/to/config.yml", "path/to/config.yaml"]`
4. Delegates to existing `resolve_for()` for cascade resolution
5. Returns `Models::Config` object (same as resolve_for)

## Technical Approach

### Architecture Pattern

- **Delegation pattern**: `resolve_namespace` is a convenience wrapper around `resolve_for`
- **Single responsibility**: Method only handles segment-to-pattern transformation
- **No caching**: Each call generates fresh results (consistent with `resolve_for` behavior)

### Implementation Strategy

The method builds file patterns from segments and delegates to `resolve_for`:

```ruby
def resolve_namespace(*segments, filename: "config")
  # Build base path from segments
  base_path = segments.join("/")

  # Generate both .yml and .yaml patterns
  patterns = [
    "#{base_path}/#{filename}.yml",
    "#{base_path}/#{filename}.yaml"
  ]

  # Delegate to existing resolve_for
  resolve_for(patterns)
end
```

### Why This Approach

1. **Minimal code change**: Builds on existing, tested `resolve_for` method
2. **Consistent behavior**: Returns same `Models::Config` as `resolve_for`
3. **Flexible API**: Supports single segment, multiple segments, and custom filenames
4. **DRY improvement**: Eliminates repeated `.yml/.yaml` pattern across 10+ gems

### Happy Path

| Scenario | Input | Expected Output |
|----------|-------|-----------------|
| Single segment | `resolve_namespace("docs")` | Config from `docs/config.{yml,yaml}` |
| Multiple segments | `resolve_namespace("git", "worktree")` | Config from `git/worktree/config.{yml,yaml}` |
| Custom filename | `resolve_namespace("lint", filename: "kramdown")` | Config from `lint/kramdown.{yml,yaml}` |

### Edge Cases

| Scenario | Input | Expected Behavior |
|----------|-------|-------------------|
| No matching files | `resolve_namespace("nonexistent")` | Returns empty `Models::Config` |
| Empty segments | `resolve_namespace()` | Returns config from `config.{yml,yaml}` in root |
