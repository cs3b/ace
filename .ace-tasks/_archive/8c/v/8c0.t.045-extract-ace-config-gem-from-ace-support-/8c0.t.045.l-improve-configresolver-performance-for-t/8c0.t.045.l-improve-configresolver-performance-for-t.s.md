# Improve ConfigResolver performance for test environments

## Objective

ace-git unit tests are slow (5.7s for 29 organism tests) because ConfigResolver performs filesystem searches on every call. Add test mode to ace-config that skips filesystem searches and returns empty/mock config immediately.

**Dependency**: This task runs AFTER Task 157 (ace-config extraction) completes.

## Behavioral Specification

### User Experience
- **Input**: Test runner executes ace-git test suite
- **Process**: Tests complete quickly without filesystem searches for config files
- **Output**: Tests pass in < 1s (currently 5.7s)

### Expected Behavior

ConfigResolver should NOT perform filesystem searches in test environments. In test mode:
1. Skip filesystem searches entirely
2. Return empty config immediately
3. Allow tests to provide mock config if needed

### Interface Contract

```ruby
# Test Mode Detection (Priority Order):
# 1. Constructor parameter (explicit)
Ace::Config.create(test_mode: true).resolve  # Returns empty immediately

# 2. Class-level flag
Ace::Config.test_mode = true
Ace::Config.create.resolve  # Returns empty immediately

# 3. Auto-detection (ENV or Minitest defined)
# ACE_CONFIG_TEST_MODE=1 or defined?(Minitest)

# Mock config injection
Ace::Config.create(test_mode: true, mock_config: { "key" => "value" }).resolve

# Opt-out for integration tests
with_real_config do
  # Uses real filesystem-based config loading
end
```

### Success Criteria
- [x] **Performance**: Config loading 30x faster in test mode (0.01ms vs 0.3ms per call)
- [x] **Test Independence**: Tests using test_mode don't depend on filesystem state
- [x] **Backward Compatibility**: Non-test code unaffected (test_mode defaults to false)
- [x] **Opt-out Available**: `with_real_config` helper for integration tests

**Note**: The original target of "ace-git tests < 1s" was based on an incorrect hypothesis. Investigation revealed config loading was already fast (0.3ms/call). The test slowness is due to deeply nested stubs and unmocked git operations - a separate issue.

## Investigation: ace-git Test Slowness

**Finding**: The original hypothesis that config loading was causing the 5.7s test time was partially incorrect.

**Measurements**:
- Config loading WITHOUT test_mode: 0.3ms/call
- Config loading WITH test_mode: 0.01ms/call (30x faster)
- Total config overhead for 29 tests: ~9ms (negligible)

**Actual Cause of Slowness**:
The ace-git organism tests are slow due to:
1. Deeply nested stub blocks (6-8 levels of `.stub` calls per test)
2. Missing stubs for `GitStatusFetcher` and `RecentCommitsFetcher` in some tests
3. These unmocked calls make real git subprocess calls

**Resolution**: Config test mode is working correctly. The remaining test slowness is a separate issue that requires fixing the test structure (out of scope for this task).
