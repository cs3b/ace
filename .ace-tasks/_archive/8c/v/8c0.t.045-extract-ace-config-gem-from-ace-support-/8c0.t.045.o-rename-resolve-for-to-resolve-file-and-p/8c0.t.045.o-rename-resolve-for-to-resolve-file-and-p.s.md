# Rename resolve_for() to resolve_file() and promote config.with() usage

## Objective

Two related improvements to the ace-config API:
1. Rename `resolve_for()` to `resolve_file()` for clarity (takes explicit file patterns)
2. Update gems to use `config.with()` instead of calling `DeepMerger.merge()` directly

The Config model already has `with()` method that handles deep merging internally - gems just aren't using it.

## Behavioral Specification

### Problem

Currently gems follow this verbose pattern:
```ruby
config = resolver.resolve_for(["docs/config.yml", "docs/config.yaml"])
data = config.data  # Extract raw hash
merged = Ace::Config::Atoms::DeepMerger.merge(data, cli_options)  # Manual merge
```

This:
- Loses the Config object's benefits
- Requires gems to know about DeepMerger internals
- Doesn't leverage the existing `with()` method

### Proposed Solution

```ruby
# Rename for clarity
config = resolver.resolve_file(["docs/config.yml", "docs/config.yaml"])

# Use Config's built-in merge capability
merged_config = config.with(cli_options)  # Returns new Config with merged data
```

### Interface Contract

**ConfigResolver changes:**
```ruby
# Add alias for backward compatibility
alias_method :resolve_for, :resolve_file

# Or deprecation warning
def resolve_for(patterns)
  warn "DEPRECATED: Use resolve_file instead of resolve_for"
  resolve_file(patterns)
end
```

**Gem usage pattern change:**
```ruby
# Before (ace-test-runner, ace-git, ace-nav, etc.)
config = Ace::Config::Atoms::DeepMerger.merge(config.data, options)

# After
config = config.with(options)  # Returns Config object, not Hash
```
