# Extract Models & Errors to ace-config

## Objective

Move data models and config-related errors from ace-support-core to ace-config.

## Models to Extract

### 1. CascadePath

**Source:** `ace-support-core/lib/ace/core/models/cascade_path.rb`
**Target:** `ace-config/lib/ace/config/models/cascade_path.rb`

Changes:
- Namespace: `Ace::Core::Models::CascadePath` → `Ace::Config::Models::CascadePath`
- No logic changes (pure data model)

### 2. Config

**Source:** `ace-support-core/lib/ace/core/models/config.rb`
**Target:** `ace-config/lib/ace/config/models/config.rb`

Changes:
- Namespace update
- Update DeepMerger require to new path

## Errors to Extract

**Source:** `ace-support-core/lib/ace/core/errors.rb`

Extract config-related errors:
```ruby
# ace-config/lib/ace/config/errors.rb
module Ace
  module Config
    class Error < StandardError; end
    class ConfigNotFoundError < Error; end
    class ConfigInvalidError < Error; end
    class YamlParseError < Error; end
    class PathError < Error; end
    class MergeStrategyError < Error; end
  end
end
```

Keep in ace-support-core:
- `EnvParseError` (if exists)
- Any ace-specific errors

## NOT Extracted (ace-specific)

- `ConfigTemplates` - Scans for ace-* gems specifically
