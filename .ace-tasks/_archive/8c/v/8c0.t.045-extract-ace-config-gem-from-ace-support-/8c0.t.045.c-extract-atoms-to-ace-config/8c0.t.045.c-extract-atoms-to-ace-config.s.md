### 3. PathExpander

**Source:** `ace-support-core/lib/ace/core/atoms/path_expander.rb`
**Target:** `ace-config/lib/ace/config/atoms/path_expander.rb`

Changes:
- Namespace: `Ace::Core::Atoms::PathExpander` → `Ace::Config::Atoms::PathExpander`
- Update error class references to Ace::Config::* namespace

**Architectural Fix Required**: PathExpander currently violates Atom purity by calling `Molecules::ProjectRootFinder.new` internally (lines 40, 55). This must be refactored:

```ruby
# BEFORE (violation): Atom calls Molecule
def self.for_file(source_file)
  project_root = Molecules::ProjectRootFinder.new.find || Dir.pwd  # ❌
  new(source_dir: source_dir, project_root: project_root)
end

# AFTER (pure): Accept project_root as parameter, let caller resolve
def self.for_file(source_file, project_root: nil)
  project_root ||= Dir.pwd  # Caller provides or uses default
  new(source_dir: source_dir, project_root: project_root)
end
```

**Migration approach**:
1. Add `project_root:` parameter with nil default
2. Keep internal ProjectRootFinder call as fallback (deprecation warning)
3. Update callers to pass project_root explicitly
4. Remove fallback in future version
