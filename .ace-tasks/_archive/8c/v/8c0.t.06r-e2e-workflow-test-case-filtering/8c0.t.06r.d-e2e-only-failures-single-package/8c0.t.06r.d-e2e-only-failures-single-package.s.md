# E2E Only-Failures Single Package

## Behavioral Specification

### User Experience
- **Input**: `--only-failures` flag on ace-test-e2e command
- **Process**: CLI scans cache for failed test cases, passes them as --test-cases
- **Output**: Only failed test cases are re-executed

### Expected Behavior
The ace-test-e2e CLI should accept an `--only-failures` flag that:
1. Scans `.cache/ace-test-e2e/` for metadata.yml files
2. Extracts `failed_test_cases` arrays from recent test runs
3. Passes failed test case IDs as `--test-cases` to re-execute

This enables quick re-running of only the test cases that failed in the previous execution.

### Interface Contract

**CLI Invocation:**
```bash
# Re-run only failed test cases
ace-test-e2e ace-lint --only-failures

# Dry run to see what would run
ace-test-e2e ace-lint --only-failures --dry-run

# Combined with specific test
ace-test-e2e ace-lint MT-LINT-003 --only-failures
```

**Error Handling:**
- No failures found: Message "No failed test cases found"
- Mutually exclusive with --test-cases: Error message

**Edge Cases:**
- No previous test runs: Error message
- Cache directory missing: Create directory or error
- Multiple test files: Aggregate failures across all

## Risk Assessment

### Technical Risks
- **Risk:** Cache directory may not exist or be empty
  - **Probability:** Medium (new installations, cache cleanup)
  - **Impact:** Low (graceful error handling)
  - **Mitigation:** Check directory existence, provide helpful error message
  - **Rollback:** Not needed (graceful degradation)

### Integration Risks
- **Risk:** YAML parsing may fail for malformed metadata files
  - **Probability:** Low (metadata.yml is generated by system)
  - **Mitigation:** Use YAML.safe_load, skip malformed files with warning
  - **Monitoring:** Log skipped files for debugging
