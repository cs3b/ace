# E2E Workflow Test Case Filtering

## Behavioral Specification

### User Experience
- **Input**: Test case IDs as comma-separated argument (e.g., `TC-001,002` or `tc-001,tc-003`)
- **Process**: Workflow normalizes test case IDs to `TC-NNN` format, validates they exist in the test file, and executes ONLY those test cases
- **Output**: Test reports containing only executed test cases with clear indication of filtering applied

### Expected Behavior
The run-e2e-test workflow should accept an optional `TEST_CASES` argument that allows users to execute specific test cases within a test scenario, rather than running all test cases. When provided:
1. Parse comma-separated test case IDs
2. Normalize each ID to `TC-NNN` format (zero-padded, uppercase)
3. Validate that all requested test cases exist in the test file
4. Execute ONLY the specified test cases
5. Generate reports that include only executed test cases

### Interface Contract

**Workflow Invocation (Manual):**
```bash
# Run specific test case
/ace:run-e2e-test ace-lint MT-LINT-003 TC-002

# Run multiple test cases with various formats
/ace:run-e2e-test ace-lint MT-LINT-003 TC-001,tc-003,002
```

**TEST_CASES Argument Formats:**
- `TC-001` → `TC-001` (already normalized)
- `tc-001` → `TC-001` (uppercase)
- `001` → `TC-001` (add prefix)
- `1` → `TC-001` (zero-pad and add prefix)
- `TC-1` → `TC-001` (zero-pad)

**Error Handling:**
- Non-existent test case: Error message with available test cases listed
- Invalid format: Clear error message indicating expected format

**Edge Cases:**
- Empty TEST_CASES: Execute all test cases (default behavior)
- Single test case: Execute only that test case
- All test cases specified: Execute all (same as default)
- Duplicate IDs (after normalization): Execute each test case only once
