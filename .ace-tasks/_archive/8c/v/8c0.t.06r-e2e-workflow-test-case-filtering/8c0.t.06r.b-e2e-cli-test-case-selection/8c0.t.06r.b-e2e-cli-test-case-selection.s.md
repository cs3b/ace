# E2E CLI Test Case Selection

## Behavioral Specification

### User Experience
- **Input**: `--test-cases` flag with comma-separated test case IDs (e.g., `--test-cases tc-001,002`)
- **Process**: CLI normalizes test case IDs, passes them to workflow which filters execution
- **Output**: Test reports containing only executed test cases

### Expected Behavior
The ace-test-e2e CLI command should accept a `--test-cases` flag that allows users to run specific test cases. When provided:
1. Parse comma-separated test case IDs
2. Normalize each ID to `TC-NNN` format
3. Pass normalized IDs to the workflow (which handles filtering)
4. Generate reports showing only executed test cases

Additionally, a `--dry-run` flag should preview which test cases would run without executing.

### Interface Contract

**CLI Invocation:**
```bash
# Run specific test cases
ace-test-e2e ace-lint MT-LINT-003 --test-cases tc-001,002

# Dry run to preview
ace-test-e2e ace-lint MT-LINT-003 --test-cases tc-002 --dry-run

# With provider selection
ace-test-e2e ace-lint MT-LINT-003 --test-cases TC-001 --provider claude:sonnet
```

**Test Case ID Formats:**
- Same normalization as workflow: `tc-001`, `001`, `TC-1` → `TC-001`

**Error Handling:**
- Invalid test case ID: Clear error message
- Dry run output: List test cases that would execute

**Edge Cases:**
- Empty --test-cases: Error or run all tests
- Dry run with test cases: List without executing
- Combined with --only-failures: Error (mutually exclusive)

## Risk Assessment

### Technical Risks
- **Risk:** Ruby normalization may differ from bash normalization in workflow
  - **Probability:** Medium
  - **Impact:** High (inconsistent behavior between CLI and manual invocation)
  - **Mitigation:** Use identical normalization logic, test both paths
  - **Rollback:** Revert Ruby parser changes

### Integration Risks
- **Risk:** Workflow may not receive test cases correctly via skill invocation
  - **Probability:** Low (skill invocation follows standard pattern)
  - **Mitigation:** Test skill command format manually
  - **Monitoring:** Verify workflow receives TEST_CASES argument
