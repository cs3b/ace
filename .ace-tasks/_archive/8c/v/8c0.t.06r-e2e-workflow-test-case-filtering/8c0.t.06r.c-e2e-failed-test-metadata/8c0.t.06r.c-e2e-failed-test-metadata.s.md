# E2E Failed Test Metadata

## Behavioral Specification

### User Experience
- **Input**: None (automatic enhancement during test execution)
- **Process**: ReportWriter extracts failed test case IDs from test results
- **Output**: metadata.yml includes `failed_test_cases` array

### Expected Behavior
When E2E tests execute, the metadata.yml file should include a `failed_test_cases` array listing all test case IDs that failed. This enables:
1. Re-running only failed tests with --only-failures flag
2. Tracking which specific test cases failed within a test scenario
3. Better debugging and failure analysis

### Interface Contract

**New metadata.yml structure:**
```yaml
run-id: "8p5jo2-lint-mt001"
test-id: "MT-LINT-001"
package: "ace-lint"
status: "fail"
results:
  passed: 1
  failed: 1
  total: 2
failed_test_cases:
  - "TC-002"
```

**When all tests pass:**
```yaml
status: "pass"
failed_test_cases: []
# or omit the key entirely
```

**Error Handling:**
- Missing test_cases in result: Empty array
- Partial completion: List failed test cases that were executed

### Success Criteria
- [x] **Metadata includes failed_test_cases array**: Populated from test results
- [x] **Empty when no failures**: Array is empty when all tests pass
- [x] **Format is consistent**: Always TC-NNN format strings
- [x] **Backward compatible**: Existing parsers ignore new field

### Validation Questions
- [x] **Omit vs empty array**: Decision: use empty array `[]` (more explicit, easier for parsers)
- [x] **Partial test execution**: Only failed cases from executed set are reported (natural behavior)

## Objective

Track which specific test cases failed in E2E test execution. This metadata enables --only-failures functionality and provides better failure analysis.
