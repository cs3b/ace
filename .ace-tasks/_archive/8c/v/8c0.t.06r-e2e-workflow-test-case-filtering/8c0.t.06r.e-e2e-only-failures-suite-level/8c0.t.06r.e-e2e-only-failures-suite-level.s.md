# E2E Only-Failures Suite Level

## Behavioral Specification

### User Experience
- **Input**: `--only-failures` flag on ace-test-suite-e2e command
- **Process**: Suite orchestrator collects failures by package, runs each with filtered test cases
- **Output**: Only failed test cases re-run across all packages

### Expected Behavior
The ace-test-suite-e2e CLI should accept an `--only-failures` flag that:
1. Scans cache for failed test cases across all packages
2. Groups failures by package and test ID
3. Executes tests with only failed test cases (using FailureFinder)
4. Aggregates results across all packages

This enables re-running only failed test cases across the entire test suite.

### Interface Contract

**CLI Invocation:**
```bash
# Re-run only failed tests across all packages
ace-test-suite-e2e --only-failures

# Re-run only failed tests for specific package
ace-test-suite-e2e ace-lint --only-failures

# Combined with --affected
ace-test-suite-e2e --affected --only-failures
```

**Error Handling:**
- No failures found: Message "No failed test cases found in cache"
- Mutually exclusive with --test-cases: Error message

**Edge Cases:**
- Empty cache: Helpful error message
- Package with no failures: Skip with log message
- Combined with --affected: Apply both filters
