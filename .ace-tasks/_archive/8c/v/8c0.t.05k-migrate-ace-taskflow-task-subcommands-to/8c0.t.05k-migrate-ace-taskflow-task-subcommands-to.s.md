# Migrate ace-taskflow task subcommands to proper dry-cli nested commands

## Behavioral Specification

### User Experience
- **Input**: Users run commands like `ace-taskflow task move 187 --child-of 150`
- **Process**: The command should work correctly and show proper help with `--help`
- **Output**: Tasks are moved/shown/started correctly with clear feedback

### Expected Behavior

Users interact with ace-taskflow task subcommands through CLI. Currently, 10 task subcommands are broken due to legacy wrapper pattern that uses `options[:args]` which dry-cli doesn't populate.

**Current Broken Behavior:**
```bash
$ ace-taskflow task move --help
ERROR: "ace-taskflow task" was called with arguments "move --help"

$ ace-taskflow task move 187 --child-of 150
ERROR: "ace-taskflow task" was called with arguments "move --child-of 150"
```

**Expected Fixed Behavior:**
```bash
$ ace-taskflow task move --help
Usage:
  ace-taskflow task move TASK_REF [options]

Options:
  --child-of=PARENT     Make subtask of specified parent
  --release=RELEASE    Move to specified release
  --dry-run, -n        Preview without moving
  --help, -h           Show this help message

Move a task to a different location

$ ace-taskflow task move 187 --child-of 150
Task 187 moved under task 150

$ ace-taskflow task show 187
Title: Fix task create command
Status: In Progress
...
```

### Interface Contract

**CLI Commands to Fix:**
```bash
# All should work with proper --help and argument parsing
ace-taskflow task show <task_ref>              # Show task details
ace-taskflow task start <task_ref>             # Mark task as in-progress
ace-taskflow task done <task_ref>              # Mark task as complete
ace-taskflow task undone <task_ref>            # Mark task as incomplete
ace-taskflow task defer <task_ref>             # Defer task
ace-taskflow task undefer <task_ref>           # Undefer task
ace-taskflow task move <task_ref> [options]    # Move task location
ace-taskflow task update <task_ref> [options]  # Update task metadata
ace-taskflow task add-dependency <task_ref> <dependency_ref>  # Add dependency
ace-taskflow task remove-dependency <task_ref> <dependency_ref>  # Remove dependency
```

**Error Handling:**
- Invalid task reference: Clear error message with reference format examples
- Missing required arguments: Show usage hint
- Invalid options: Show available options
- Move errors (file conflicts): Explain what went wrong

**Edge Cases:**
- Moving task to itself (should error)
- Moving task with children (should preserve hierarchy)
- Updating non-existent fields (should error or warn)
- Adding duplicate dependencies (should warn or skip)

## Objective

Fix broken task subcommands by migrating from legacy wrapper pattern to proper dry-cli nested subcommands. The current implementation uses `options[:args]` which dry-cli doesn't populate, causing all subcommands except `create` to fail.

## Technical Approach

### Architecture Pattern
- **Pattern**: Nested dry-cli subcommand registration (per ADR-023)
- **Integration**: Each subcommand is a standalone class under `Commands::Task::` namespace
- **Impact**: Replaces legacy wrapper pattern with direct `register "task <subcmd>", Commands::Task::<Subcmd>`

### Technology Stack
- **Framework**: dry-cli with `Ace::Core::CLI::DryCli::Base` module
- **Business Logic**: Direct calls to `Organisms::TaskManager` methods (already exist)
- **No new dependencies required** - all infrastructure exists

### Implementation Strategy
1. Create 10 new command classes following `Commands::Task::Create` pattern
2. Register each as nested subcommand in `cli.rb`
3. Delete legacy `CLI::Task` wrapper and `TaskCommand` dispatcher
4. Retain `TaskCommand` helper methods (display_task_*) if needed, or migrate to new location

## File Modifications

### Create
- `lib/ace/taskflow/commands/task/show.rb` - Show task details
  - Purpose: Display task by reference
  - Key components: `argument :task_ref`, call `TaskManager#show_task`
- `lib/ace/taskflow/commands/task/start.rb` - Mark task as in-progress
  - Purpose: Start working on task
  - Key components: `argument :task_ref`, call `TaskManager#start_task`
- `lib/ace/taskflow/commands/task/done.rb` - Mark task as complete
  - Purpose: Complete task and move to archive
  - Key components: `argument :task_ref`, call `TaskManager#complete_task`
- `lib/ace/taskflow/commands/task/undone.rb` - Reopen completed task
  - Purpose: Restore task from done status
  - Key components: `argument :task_ref`, call `TaskManager#reopen_task`
- `lib/ace/taskflow/commands/task/defer.rb` - Defer task
  - Purpose: Move task to deferred directory
  - Key components: `argument :task_ref`, call `TaskManager#defer_task`
- `lib/ace/taskflow/commands/task/undefer.rb` - Restore deferred task
  - Purpose: Restore task from deferred
  - Key components: `argument :task_ref`, call `TaskManager#undefer_task`
- `lib/ace/taskflow/commands/task/move.rb` - Move task location
  - Purpose: Move/reorganize tasks (release move, promote, demote, convert)
  - Key components: `argument :task_ref`, options for `--child-of`, `--release`, `--dry-run`
  - Calls: `TaskManager#move_task`, `#promote_to_standalone`, `#demote_to_subtask`, `#convert_to_orchestrator`
- `lib/ace/taskflow/commands/task/update.rb` - Update task fields
  - Purpose: Update arbitrary task metadata fields
  - Key components: `argument :task_ref`, `option :field` (repeatable), call `TaskManager#update_task_fields`
- `lib/ace/taskflow/commands/task/add_dependency.rb` - Add task dependency
  - Purpose: Add dependency relationship
  - Key components: `argument :task_ref`, `argument :dependency_ref`, call `TaskManager#add_dependency`
- `lib/ace/taskflow/commands/task/remove_dependency.rb` - Remove task dependency
  - Purpose: Remove dependency relationship
  - Key components: `argument :task_ref`, `argument :dependency_ref`, call `TaskManager#remove_dependency`

### Modify
- `lib/ace/taskflow/cli.rb`
  - Changes: Add 10 new `register "task <subcmd>"` lines, require new command files
  - Integration: Keep `register "task", CLI::Task.new` for base `ace-taskflow task` (show next task)
- `lib/ace/taskflow/cli/task.rb`
  - Changes: Simplify to only handle "show next task" case (no subcommand routing)
  - Remove legacy `options[:args]` pattern

### Delete
- `lib/ace/taskflow/commands/task_command.rb` (legacy dispatcher)
  - Reason: All logic migrated to TaskManager and new command classes
  - Note: Some helper methods (display_task_*, status_icon) may need to be preserved in a shared module
