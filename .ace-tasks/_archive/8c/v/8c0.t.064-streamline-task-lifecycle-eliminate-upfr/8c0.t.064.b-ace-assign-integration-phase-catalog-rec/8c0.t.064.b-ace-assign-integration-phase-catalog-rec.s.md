# ace-assign Integration (phase catalog, recipes, composition rules)

## Behavioral Specification

### User Experience

- **Input**: User runs `/ace:assign-start` with implement-with-pr recipe for a task
- **Process**: ace-assign composes assignment with JIT plan-task phase automatically placed before work-on-task
- **Output**: Assignment with phases: onboard → plan-task (JIT) → work-on-task → ... where plan-task output is ephemeral (phase report only)

### Expected Behavior

When ace-assign composes an assignment for task implementation:
1. The plan-task phase appears between onboard and work-on-task
2. plan-task is optional (can be skipped for trivial tasks)
3. plan-task output goes to phase report (ephemeral), not task spec
4. Composition rules enforce correct ordering (onboard → plan → work)

### Interface Contract

```bash
# compose assignment — plan-task phase auto-included
ace-assign compose --recipe implement-with-pr --taskref 230.01
# Result: job.yaml with phases: onboard, plan-task, work-on-task, ...

# drive assignment — plan-task runs as JIT phase
ace-assign status
# Phase 010: onboard [done]
# Phase 020: plan-task [in_progress]  ← JIT planning
# Phase 030: work-on-task [pending]
```

### Success Criteria

- [ ] `plan-task.phase.yml` updated: produces `[implementation-plan]` only (no `task-spec`), consumes `[project-context, task-spec]`
- [ ] `implement-with-pr.recipe.yml` includes optional `plan-task` between onboard and work-on-task
- [ ] `implement-simple.recipe.yml` includes optional `plan-task` between onboard and work-on-task
- [ ] `composition-rules.yml` has `plan-before-implementation` and `onboard-before-plan` ordering rules
- [ ] `composition-rules.yml` has conditional suggestion for plan-task when work-on-task is included

## Objective

Integrate the JIT plan-task phase into ace-assign's phase catalog, recipes, and composition rules so that implementation assignments automatically include planning as an ephemeral step.
