# Task Status Migration and Verification

## Behavioral Specification

### User Experience

- **Input**: Run verification commands against all active tasks
- **Process**: Ensure all non-completed, non-in-progress tasks follow new lifecycle model
- **Output**: All tasks correctly in draft or pending status with behavioral specs only (no stale implementation plans in task files)

### Expected Behavior

1. All tasks with `status: pending` that have detailed implementation plans embedded should be evaluated:
   - If plan is stale/outdated → revert to `status: draft`
   - If task is actively being worked (243.xx) → leave as-is
2. Verify no workflow cross-references are broken after Stage 1 changes
3. Confirm new lifecycle is consistent across all workflow files

### Interface Contract

```bash
# Check for pending tasks with embedded plans
ace-taskflow tasks --status pending

# Verify no stale cross-references
grep -r "plan-task" ace-taskflow/handbook/workflow-instructions/ | grep -v "JIT\|ephemeral\|ace-assign"

# Current known state (no reversions expected):
# 230, 235: already draft
# 243.01, 243.02: pending, excluded (active orchestrator)
```

### Success Criteria

- [ ] No stray pending tasks with stale implementation plans (excluding 243.xx)
- [ ] All workflow .wf.md files have consistent cross-references (no stale plan-task lifecycle references)
- [ ] `ace-taskflow tasks --status draft` shows expected tasks
- [ ] `ace-taskflow tasks --status pending` shows only 243.xx subtasks

## Objective

Final verification pass ensuring the new streamlined lifecycle is consistently applied across all active tasks and workflow files.

## Scope of Work

### Deliverables

#### Verify
- Task status distribution matches new lifecycle model

#### Modify (if needed)
