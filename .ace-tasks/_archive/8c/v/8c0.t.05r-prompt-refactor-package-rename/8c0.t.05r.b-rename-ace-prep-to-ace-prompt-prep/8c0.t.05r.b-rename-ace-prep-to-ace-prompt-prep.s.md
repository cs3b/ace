# Rename ace-prep to ace-prompt-prep

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-prompt-prep` command (instead of `ace-prep`)
- **Process**: All functionality remains identical - prompt workspace management with archiving, context loading, and LLM enhancement
- **Output**: Same outputs as before, but with updated naming throughout (cache dirs, config paths, etc.)

### Expected Behavior

The current name `ace-prep` is semantically ambiguous - it could mean "ACE preparation" rather than "prompt preparation". The tool prepares prompts for use, and this connection should be explicit in the name.

After this rename:
- Users run `ace-prompt-prep` to prepare prompts
- Configuration lives in `.ace/prompt-prep/`
- Cache files go to `.cache/ace-prompt-prep/`
- The `prompt://` protocol remains unchanged (it refers to the domain, not the gem)

This follows the compound naming pattern established by `ace-git-commit`, `ace-git-secrets`, and `ace-git-worktree` where:
- First part (`prompt`) = domain
- Second part (`prep`) = specific function

### Interface Contract

```bash
# CLI Interface (renamed)
ace-prompt-prep                        # Process and archive prompt
ace-prompt-prep --enhance              # Enhance prompt via LLM
ace-prompt-prep --task 121             # Use task-specific prompts
ace-prompt-prep setup                  # Initialize workspace
ace-prompt-prep setup --template bug   # Initialize with bug template
```

**Configuration Paths:**
- Old: `.ace/prep/config.yml` → New: `.ace/prompt-prep/config.yml`
- Old: `.ace-defaults/prep/` → New: `.ace-defaults/prompt-prep/`
- Old: `.cache/ace-prep/` → New: `.cache/ace-prompt-prep/`

**Protocol (unchanged):**
- `prompt://` protocol continues to work (domain reference, not gem name)
- `tmpl://` protocol continues to work

## Objective

Rename the `ace-prep` gem to `ace-prompt-prep` to make the semantic meaning clear - this tool prepares prompts, not "ACE preparation". This follows the compound naming pattern used by other ACE gems.
