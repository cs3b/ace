# Rename ace-prompt to ace-prep

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-prep` command (instead of `ace-prompt`)
- **Process**: All functionality remains identical - setup, process, enhance operations work the same way
- **Output**: Same results, better naming that reflects the tool's purpose as prompt preparation before the main session

### Expected Behavior

The package rename from `ace-prompt` to `ace-prep` should be transparent to users except for the command name change. "Prep" better captures the tool's purpose:
- **setup** = preparing the workspace
- **process** = preparing the prompt for use (archiving, outputting)
- **enhance** = preparing/improving the prompt via LLM

Users should be able to:
- Run `ace-prep` instead of `ace-prompt`
- Use `ace-prep setup`, `ace-prep process`, `ace-prep process --enhance`
- Configure via `.ace/prep/config.yml` (instead of `.ace/prompt/config.yml`)
- Find cached files in `.cache/ace-prep/` (instead of `.cache/ace-prompt/`)

### Interface Contract

```bash
# CLI Interface (renamed from ace-prompt)
ace-prep --version          # Show version (0.15.0)
ace-prep --help             # Show help with updated names
ace-prep setup              # Create prompt workspace
ace-prep process            # Process and archive prompt
ace-prep process --enhance  # Enhance via LLM

# Config cascade (renamed)
.ace/prep/config.yml        # Project config
~/.ace/prep/config.yml      # User config (if exists)

# Cache directory (renamed)
.cache/ace-prep/prompts/    # Prompt workspace
```

**Error Handling:**
- Old command `ace-prompt` will not exist (no backward compatibility per ADR-024)
- Old config `.ace/prompt/` will not be read (no migration)

**Edge Cases:**
- Protocol URI `prompt://` stays unchanged (it's for prompts, not gem name)
- Internal class names with "Prompt" (e.g., `PromptReader`) stay as-is (they work WITH prompts)
