# Migrate ace-git-worktree Configuration

## Objective

Remove the dual configuration system (hardcoded `Configuration` module constants + `ConfigLoader`) and consolidate to a single ace-core cascade pattern.

## Current State (Anti-Pattern)

**File: `ace-git-worktree/lib/ace/git/worktree/configuration.rb:21-36`**
```ruby
# Hardcoded constants duplicating cascade config
DEFAULT_ROOT_PATH = ".ace-wt"
DEFAULT_DIRECTORY_FORMAT = "task.{id}"
DEFAULT_BRANCH_FORMAT = "{id}-{slug}"
DEFAULT_GIT_TIMEOUT = 30
DEFAULT_MISE_TIMEOUT = 5
DEFAULT_TASK_TIMEOUT = 10
MAX_SLUG_LENGTH = 50
MIN_SLUG_LENGTH = 3
```

**Problem**: Configuration is defined in two places:
1. Hardcoded constants in `Configuration` module
2. Cascade config via `ConfigLoader` molecule

## Target State (Standard Pattern)

All configuration should come from `.ace.example/git/worktree.yml` through the ace-core cascade, with no hardcoded constants.

```ruby
def self.config
  @config ||= Ace::Core.config.get('ace', 'git', 'worktree') || {}
end

# Access via config method, not constants
def root_path
  config['root_path'] || '.ace-wt'
end
```

   - Update to use ace-core cascade

2. **`ace-git-worktree/lib/ace/git/worktree/molecules/config_loader.rb`**
   - Ensure proper cascade usage
   - Remove fallback to hardcoded constants

3. **`ace-git-worktree/.ace.example/git/worktree.yml`**
   - Ensure all removed constants are present as config keys
   - Add timeout values
   - Add slug length limits
