Update ADR-022 (Configuration Default and Override Pattern) with findings from Task 143 research, including the `.ace-defaults/` rename decision and config path standardization.

## Scope of Work

### 1. Update ADR-022

**File: `docs/decisions/ADR-022-configuration-default-and-override-pattern.md`**

Add sections for:

#### A. Rename: `.ace.example/` → `.ace-defaults/`

**Rationale:**
- "example" implies "copy to use" but users DON'T copy
- These files are **runtime defaults** loaded by gems
- `.ace-defaults/` clearly indicates purpose

**Migration:**
```bash
# Per gem:
git mv .ace.example .ace-defaults
# Update gemspec, loading code, docs
```

#### B. Config Path Standard

**File structure:**
```
.ace/{tool}/config.yml       # Main config
.ace/{tool}/{feature}.yml    # Feature-specific

Examples:
.ace/git/config.yml          # Main git config
.ace/git/worktree.yml        # Worktree-specific
.ace/git/commit.yml          # Commit-specific
.ace/taskflow/config.yml     # Taskflow config
```

**Loading pattern:**
```ruby
Ace::Core.get("tool", file: "config")    # Main config
Ace::Core.get("tool", file: "feature")   # Feature-specific
```

## ADR-022 Update Template

```markdown
## Status Update (December 2025)

This ADR was updated following Task 143 research to:
1. Rename `.ace.example/` → `.ace-defaults/` for clarity
2. Standardize config path patterns
3. Document compliance status across packages

### Rename Decision: .ace.example → .ace-defaults

**Context:** The name "example" implies files to copy, but these are runtime
defaults loaded by gems. Users don't copy them - they create `.ace/` overrides.

**Decision:** Rename to `.ace-defaults/` which clearly indicates purpose.

**Migration:**
1. Rename directory in each gem
2. Update gemspec to include `.ace-defaults/`
3. Update loading code path references
4. Update documentation references

### Config Path Standard

All ace-* gems use consistent path patterns:

**Directory structure:**
.ace/{tool}/config.yml       # Main configuration
.ace/{tool}/{feature}.yml    # Feature-specific config

**Loading API:**
Ace::Core.get("tool", file: "config")    # → .ace/tool/config.yml
Ace::Core.get("tool", file: "feature")   # → .ace/tool/feature.yml

**DO NOT use:**
- get("ace", "tool") - deprecated pattern
- Hardcoded namespace prefixes

### Package Compliance Status

**Compliant:**
- ace-git, ace-git-secrets

**Needs Migration:**
- ace-taskflow, ace-git-worktree, ace-git-commit, ace-docs, ace-lint
- ace-nav, ace-prompt, ace-review, ace-search, ace-test-runner

**Deferred:**
- ace-llm, ace-llm-providers-cli (ENV-based, separate task)
```
