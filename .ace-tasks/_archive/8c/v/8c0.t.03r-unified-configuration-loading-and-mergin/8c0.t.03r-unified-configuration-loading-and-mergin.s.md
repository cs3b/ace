### Anti-Patterns Identified

1. **Hardcoded paths**: `.ace-taskflow/done/` in ace-search, ace-docs
2. **Hardcoded timeouts**: `DEFAULT_TIMEOUT` constants in multiple packages
3. **Inline defaults**: Full config hashes/YAML embedded in Ruby code
4. **Custom YAML loading**: `YAML.load_file` instead of Ace::Core
5. **Deprecated API**: Using `get("ace", "tool")` instead of `get("tool", file: "config")`

### Naming Decision

**Rename `.ace.example/` → `.ace-defaults/`**
- "example" implies files to copy, but users don't copy
- These are runtime defaults loaded by gems
- `.ace-defaults/` clearly indicates purpose

## Target Pattern

### Cascade Order (highest to lowest priority)

```
1. $CWD/.ace/                    # Current working directory
2. ... intermediate dirs ...     # All .ace/ dirs between CWD and PROJECT_ROOT
3. $PROJECT_ROOT/.ace/           # Project root (auto-discovered via markers)
4. $HOME/.ace/                   # User home preferences
5. $GEM_PATH/.ace-defaults/      # Gem defaults (LOWEST - fallback only)
```

**Note:** DirectoryTraverser already handles #1-#4. Adding #5 (gem defaults) is deferred to Task 157.

### Task 143 Approach (Use Existing API)

Task 143 moves hardcoded defaults to `.ace.example/` files and uses the **existing** Ace::Core API:

```ruby
# Task 143: Move defaults to .ace.example/, use existing cascade
def self.config
  @config ||= Ace::Core.config.get("taskflow", file: "config")
end
```

Gems manually load their `.ace.example/` defaults and merge with user config.

### Target API (Implemented in Task 157)

After Task 157 extracts ace-config with `gem_path` support:

```ruby
# Task 157: Full cascade including gem defaults
def self.config
  @config ||= begin
    gem_root = Gem.loaded_specs["ace-taskflow"]&.gem_dir || File.expand_path("../..", __dir__)
    Ace::Config.create(gem_path: gem_root).resolve_for(["taskflow/config.yml"]).data
  end
end
```

**Important**: The `gem_path` parameter does not exist until Task 157 implements it.

## Objective

Provide unified, consistent configuration experience across all ace-* packages:
1. All defaults in `.ace.example/` files (renamed to `.ace-defaults/` in Task 157.08)
2. Gem defaults integrated into cascade via `gem_path` parameter
3. Deep merge with priority: CWD → intermediate → PROJECT → HOME → GEM_DEFAULTS
4. Hardcoded values become configurable
