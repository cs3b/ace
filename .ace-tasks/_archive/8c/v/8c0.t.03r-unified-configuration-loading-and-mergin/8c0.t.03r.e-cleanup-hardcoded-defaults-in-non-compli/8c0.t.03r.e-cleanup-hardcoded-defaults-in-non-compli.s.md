# Cleanup Hardcoded Defaults in Non-Compliant Packages

## Objective

Migrate hardcoded default_config methods and DEFAULT_CONFIG constants to `.ace.example/` files across all non-compliant packages.

## Research Findings (Dec 2025)

### Packages With Hardcoded Defaults

| Package | Issue | Location |
|---------|-------|----------|
| ace-git-commit | `default_config` method | `organisms/commit_orchestrator.rb:82-93` |
| ace-docs | `default_config` method | `docs.rb:28-50` |
| ace-lint | `default_kramdown_config` | `lint.rb:66-75` |
| ace-prompt | `default_config` method | `prompt.rb:64-87` |
| ace-review | `default_config` + presets | `review.rb:72-123` |
| ace-search | `default_config` method | `search.rb:18-33` |

### Additional Anti-Patterns

**Hardcoded Paths:**
- `ace-search/lib/ace/search.rb:23` - `.ace-taskflow/done/**/*`
- `ace-docs/lib/ace/docs/organisms/document_registry.rb:304` - regex pattern

**Hardcoded Timeouts:**
- `ace-context/lib/ace/context/molecules/gh_pr_executor.rb:14` - `DEFAULT_TIMEOUT = 30`
- `ace-docs/lib/ace/docs.rb:35` - `llm_timeout => 600`

## Target State

All packages should:
1. Load defaults from `.ace.example/<package>/config.yml`
2. Load gem defaults manually + merge with `Ace::Core.config.get()` cascade (same pattern as 143.01)
3. Deep merge user config over gem defaults
4. Have NO hardcoded DEFAULT_CONFIG constants or `default_config` methods

## Acceptance Criteria

- [ ] No `DEFAULT_CONFIG` constants in Ruby code
- [ ] No `default_config` methods returning hardcoded hashes
- [ ] No `default_kramdown_config` or similar methods
- [ ] All defaults defined in `.ace.example/` files
- [ ] All packages load gem defaults at runtime
- [ ] All packages merge user config over gem defaults
- [ ] All package tests pass
- [ ] No hardcoded `.ace-taskflow/done/` paths in Ruby code
- [ ] No hardcoded timeout constants in Ruby code

## Out of Scope

- ❌ Changing default values (only making them configurable)
- ❌ Adding new features
- ❌ Modifying package behavior
- ❌ Renaming `.ace.example/` to `.ace-defaults/` (separate subtask)
