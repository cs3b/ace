# Migrate ace-test-runner Configuration

## Objective

Replace the `DEFAULT_CONFIG` hash in `config_loader.rb` with consistent ace-core cascade pattern usage.

## Current State (Anti-Pattern)

**File: `ace-test-runner/lib/ace/test_runner/molecules/config_loader.rb:27-53`**
```ruby
DEFAULT_CONFIG = {
  version: 1,
  patterns: { atoms: "test/...", molecules: "test/..." },
  groups: { unit: %w[atoms molecules ...], all: %w[...] },
  defaults: { reporter: "progress", ... },
  failure_limits: { max_display: 7 }
}.freeze
```

**Problem**: Default configuration is embedded in Ruby code. While the package does use `Ace::Core::ConfigDiscovery`, it falls back to this hardcoded hash.

## Target State (Standard Pattern)

```ruby
def self.config
  @config ||= Ace::Core.config.get('ace', 'test-runner') || load_gem_defaults_from_example
end
```

All defaults should be in `.ace.example/test-runner/config.yml` with no fallback to hardcoded Ruby hashes.
