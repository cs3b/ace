# Migrate ace-taskflow Configuration

## Objective

Remove inline YAML defaults from `configuration.rb` and ensure all configuration is loaded from `.ace.example/taskflow/config.yml` using existing Ace::Core API. (Full cascade with `gem_path` comes in Task 157.09.)

## Current State (Anti-Pattern)

**File: `ace-taskflow/lib/ace/taskflow/configuration.rb:198-250`**
```ruby
def default_config_yaml
  <<~YAML
    taskflow:
      root: ".ace-taskflow"
      task_dir: "t"
      directories:
        completed: "_archive"
        backlog: "_backlog"
      # ... many more defaults embedded in Ruby code
  YAML
end
```

**Problem**: Full configuration is embedded as a YAML string in Ruby code instead of being loaded from `.ace.example/`.

## Target State (Task 143 Pattern)

```ruby
def self.config
  @config ||= begin
    # Load gem defaults from .ace.example/
    gem_root = Gem.loaded_specs["ace-taskflow"]&.gem_dir ||
               File.expand_path("../..", __dir__)
    gem_defaults_path = File.join(gem_root, ".ace.example", "taskflow", "config.yml")
    gem_defaults = YAML.safe_load_file(gem_defaults_path, permitted_classes: [], aliases: true) if File.exist?(gem_defaults_path)

    # Load user config via existing cascade
    user_config = Ace::Core.config.get("taskflow", file: "config") || {}

    # Merge: user config over gem defaults
    Ace::Core::Atoms::DeepMerger.merge(gem_defaults || {}, user_config)
  end
end
```

**⚠️ Implementation Note**: This pattern uses `YAML.safe_load_file` directly, which requires manual error handling for missing files and parse errors. Consider wrapping in a helper method that:
- Handles `Errno::ENOENT` gracefully when file doesn't exist
- Wraps `Psych::SyntaxError` in a descriptive error

Task 157 will provide `Ace::Config::Atoms::YamlParser` with built-in error handling. Until then, each gem implements this pattern locally.

**Note**: Task 157.09 will simplify this to `Ace::Config.create(gem_path:)` after extraction.
