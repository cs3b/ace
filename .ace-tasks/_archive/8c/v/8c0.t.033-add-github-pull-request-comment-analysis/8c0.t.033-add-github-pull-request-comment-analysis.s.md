# Add GitHub Pull Request comment analysis mode to ace-review

## Behavioral Specification

### User Experience
- **Input**: PR review via `ace-review --pr <N>` (comments fetched by default), optional `--no-pr-comments` to skip
- **Process**: ace-review fetches PR diff AND PR comments, generates LLM reviews plus a "developer feedback" report, synthesizes all sources together
- **Output**: Session directory with `review-*.md` files (LLM reviews) plus `review-dev-feedback.md` (PR comments), synthesis includes all perspectives

### Expected Behavior

When a developer or AI agent uses `ace-review --pr 123`, they experience:

1. **PR Content Extraction**: Fetches PR diff (existing) AND PR comments/reviews (new)
2. **Developer Feedback Report**: Transforms PR comments into `review-dev-feedback.md` with:
   - Unresolved questions and feedback
   - Approved/resolved discussions
   - Inline code comments with file:line references
   - Comment/thread IDs for resolution tracking
3. **Unified Synthesis**: ReportSynthesizer treats `review-dev-feedback.md` as another review source
   - Mixed with LLM model reviews in synthesis
   - "Developer Feedback" appears alongside model names
4. **Comment Resolution Workflow**: Via `/ace:review-pr` workflow, after fixing comment-sourced items:
   - Reply to PR comment with commit reference
   - Resolve the conversation thread

### Interface Contract

```bash
# PR review with comments (default behavior)
ace-review --pr 123 --auto-execute
# Output includes: review-gemini.md, review-gpt4.md, review-dev-feedback.md, synthesis-report.md

# PR review WITHOUT comments
ace-review --pr 123 --no-pr-comments --auto-execute
# Output includes: review-gemini.md, review-gpt4.md, synthesis-report.md (no dev-feedback)

# Configuration in .ace/review/config.yml
defaults:
  pr_comments: true  # Default: fetch comments for --pr

# Expected synthesis includes "Developer Feedback" as a source:
"""
## Overview
- **Models**: gemini-2.5-flash, gpt-4, Developer Feedback
- **Synthesis Model**: gemini-2.5-flash
"""
```

**Error Handling:**
- [No comments found]: Graceful skip, no `review-dev-feedback.md` created
- [gh CLI errors]: Inherit Task 116 error handling (auth, network, rate limits)

**Edge Cases:**
- [Empty comments]: Skip report generation
- [Only bot comments]: Filter bots, may result in empty report
- [Closed/merged PRs]: Comments still fetched and analyzed

## Objective

Enable ace-review to automatically include PR comments as a "developer feedback" review source when reviewing PRs, so that human reviewer feedback is synthesized alongside LLM analysis for comprehensive code review insights.

## Technical Approach

### Architecture Pattern

**Integration Strategy**: Extend existing `--pr` flow rather than create parallel pipeline

The implementation adds to ace-review's existing PR review workflow:
- **Atoms**: `pr_comment_formatter.rb` - Pure transformation of comments to markdown
- **Molecules**: `gh_pr_comment_fetcher.rb` - Fetch comments via gh CLI, `gh_comment_resolver.rb` - Reply/resolve
- **Organisms**: Modify `review_manager.rb` to integrate comment fetching
- **CLI**: Add `--[no-]pr-comments` flag

**Key Design Decisions**:
1. **Integration over Separation**: Comments fetched as part of `--pr` flow, not separate command
2. **Report-Based Integration**: Comments become `review-dev-feedback.md`, discovered by existing synthesizer
3. **Simple Format**: No LLM analysis of comments - direct transformation to report format
4. **Resolution via Workflow**: Comment resolution handled in `/ace:review-pr` workflow, not tool

### Report Format

`review-dev-feedback.md`:
```markdown
---
source: pr-comments
pr_number: 123
fetched_at: 2025-12-08T12:00:00Z
---

# Developer Feedback from PR #123

## Summary
- Total comments: 8
- Unresolved threads: 5
- Reviewers: @alice, @bob

## Unresolved Feedback

### @alice (thread: PRRT_abc123)
> `src/auth.rb:42` - Add null checks here

### @bob (comment: IC_def456)
> Should we add integration tests?

## Resolved Feedback

- @alice approved changes
- Thread "naming convention" resolved

## Inline Code Comments

| File:Line | Author | Status | Comment | ID |
|-----------|--------|--------|---------|-----|
| src/auth.rb:42 | @alice | Open | Add null checks | PRRT_abc123 |
```
