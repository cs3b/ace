# Create ace-bundle Package (Copy & Rename)

## Behavioral Specification

### User Experience

- **Input**: Developer requests to create ace-bundle package alongside ace-context
- **Process**: Copy ace-context directory and rename all internal references
- **Output**: New ace-bundle package with fully functional code, tests passing

### Expected Behavior

1. **Package Creation**: ace-bundle directory exists alongside ace-context
   - `ace-bundle/ace-bundle.gemspec` with updated name and version
   - `exe/ace-bundle` executable
   - `lib/ace/bundle/` with all modules renamed
   - All 25 lib files use `Ace::Bundle` module namespace
   - All 25 test files updated with new requires and namespaces

2. **Module Updates**: All references updated
   - `Ace::Context` → `Ace::Bundle`
   - `require 'ace/context'` → `require 'ace/bundle'`

3. **Verification**: Package is functional
   - Tests pass: `cd ace-bundle && ace-test`
   - CLI works: `ace-bundle/exe/ace-bundle --help`
   - No ace-context references remain in lib

### Success Criteria

- [ ] **Package Structure**: ace-bundle directory exists with correct structure
- [ ] **Module Namespace**: All code uses `Ace::Bundle` module
- [ ] **Tests Pass**: `ace-test ace-bundle` succeeds
- [ ] **No Old References**: `grep -r "Ace::Context" ace-bundle/lib` returns nothing

### Validation Questions

- [x] **Coexistence**: Should both ace-context and ace-bundle exist temporarily?
  - **Answer**: Yes. ace-context remains untouched until subtask 206.08. ace-bundle is created as a copy, then tweaked to work independently. Dependencies migrate progressively, and ace-context is deleted only in 206.08.
- [x] **Test Scope**: Are all test files included in the namespace updates?
  - **Answer**: Yes, all test files in ace-bundle/test/ must be updated.
- [x] **Version Bump**: Should version be bumped to 0.29.0 in this subtask?
  - **Answer**: Yes, ace-bundle starts at 0.29.0.

## Objective

Create new ace-bundle package by copying ace-context and renaming all internal references, enabling both packages to coexist during migration.

## Scope of Work

**Package Creation:**
- Copy ace-context → ace-bundle directory
- Rename gemspec, executable, lib directory
- Update all module namespaces
- Update all test files
- Bump version to 0.29.0

**Out of Scope:**
- Updating dependent packages (subtasks 206.05-206.06)
- Updating documentation/skills (subtask 206.07)
- Removing ace-context (subtask 206.08)
