# Update ace-bundle Configuration Paths

## Behavioral Specification

### User Experience

- **Input**: Developer has ace-bundle package from subtask 206.01
- **Process**: Update all configuration paths to use bundle namespace
- **Output**: ace-bundle uses `.ace/bundle/` and `.cache/ace-bundle/` paths

### Expected Behavior

1. **Config Directory**: `.ace-defaults/context/` → `.ace-defaults/bundle/`
   - Config root key: `context:` → `bundle:`
   - Cache path: `.cache/ace-context` → `.cache/ace-bundle`

2. **Config Namespace**: Update code to use new namespace
   - `resolve_namespace("context")` → `resolve_namespace("bundle")` in `lib/ace/bundle.rb`

3. **Nav Protocol**: `ace-context.yml` → `ace-bundle.yml`
   - Name updated to ace-bundle
   - Description mentions bundle

4. **Project Presets**: Copy project presets for coexistence
   - Copy `.ace/context/presets/` → `.ace/bundle/presets/`
   - Both directories exist during migration

5. **Verification**: Configuration loads correctly
   - `ace-bundle --list` shows presets
   - Cache writes to `.cache/ace-bundle/`

### Success Criteria

- [ ] **Config Updated**: `.ace-defaults/bundle/config.yml` uses bundle namespace
- [ ] **Namespace Updated**: `resolve_namespace("bundle")` in code
- [ ] **Nav Protocol**: Protocol file renamed and updated
- [ ] **Presets Copied**: `.ace/bundle/presets/` exists with project presets
- [ ] **Functional**: CLI commands work with new paths
- [ ] **No Old Paths**: No references to ace-context paths in ace-bundle

### Validation Questions

- [x] **Cache Migration**: Should existing ace-context cache be migrated or left as-is?
  - **Answer**: Left as-is. Cache is ephemeral data; ace-bundle writes to `.cache/ace-bundle/` fresh.
- [x] **Fallback**: Should there be fallback to ace-context paths for compatibility?
  - **Answer**: No. Clean break per orchestrator decision (line 84). ace-bundle works only as ace-bundle; ace-context remains untouched until 206.08.

## Objective

Update all ace-bundle configuration paths to use bundle namespace instead of context.

## Scope of Work

- Update `.ace-defaults/bundle/config.yml`
- Update `resolve_namespace("context")` → `resolve_namespace("bundle")` in `lib/ace/bundle.rb`
- Rename and update nav protocol file
- Copy `.ace/context/presets/` → `.ace/bundle/presets/` (coexistence)
- Update any hardcoded paths in code

## References

- Depends on: v.0.9.0+task.206.01 (ace-bundle package must exist)
