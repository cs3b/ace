# Automatic Review Verification & Auto-Judgment for ace-coworker

## Overview

Enable ace-coworker to automatically process review feedback without manual user intervention. Instead of asking questions during reviews, use Chain of Verification (CoV) pattern to validate findings and apply verified feedback automatically.

**Key insight**: Severity labels are unreliable (Critical often wrong, Low often useful). Instead, judge by **verification status** using independent reasoning.

## Problem Statement

Current ace-coworker review flow:
1. Run review â†’ get findings
2. Ask user questions to verify each finding
3. Apply confirmed findings

Desired automated flow:
1. Run review â†’ extract feedback items (Task 227)
2. Auto-verify findings using CoV pattern
3. Apply only `verified` items automatically
4. Flag `uncertain` items for human review

## Design: Chain of Verification (CoV)

Based on research ([CoV in LLM Evaluations](https://www.rohan-paul.com/p/chain-of-verification-in-llm-evaluations), [Verified Code Reasoning](https://arxiv.org/html/2509.26546v1)):

```
Finding â†’ Verification Questions â†’ Independent Answers â†’ Confidence Score
```

**Key principle**: Verify without original bias - don't show the original finding's reasoning during verification.

### Three Auto-Apply Levels

| Level | What gets auto-applied | Default |
|-------|------------------------|---------|
| `none` | Nothing - pure inspection mode | |
| `verified` | Only verified items | **Yes** |
| `all` | All draft + pending items | |

### Verification Questions (per finding)

1. "Does the code at {location} actually have this issue?"
2. "Would the suggested fix introduce new bugs?"
3. "Is this change aligned with {task goal}?"

### Confidence Scoring

- `verified` - All questions answered positively
- `uncertain` - Mixed answers, needs human review
- `invalid` - Verification found the finding was wrong
