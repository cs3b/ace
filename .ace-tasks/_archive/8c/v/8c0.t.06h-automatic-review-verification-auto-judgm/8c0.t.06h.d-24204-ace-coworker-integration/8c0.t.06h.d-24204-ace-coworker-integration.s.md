# 242.04: ace-coworker Integration

## Objective

Integrate automatic verification and feedback application into ace-coworker workflow, enabling fully automated review cycles.

## Behavioral Specification

### User Experience

- **Input**: Review step in coworker job with `auto_apply` config
- **Process**: Extract findings → verify → apply verified items → continue
- **Output**: Applied fixes + feedback report

### Interface Contract

**Job Configuration:**

```yaml
# job.yaml
session:
  name: feature-implementation
  auto_apply: verified  # none | verified | all (default: verified)

steps:
  - name: implement-feature
    skill: ace:work-on-task
    instructions:
      - Implement the feature per task spec

  - name: review-cycle
    skill: ace:review-pr
    auto_apply: verified  # Can override session default
    instructions:
      - Review the implementation
      - Extract feedback items
      - Verify and apply automatically

  - name: final-review
    skill: ace:review-pr
    auto_apply: none  # Force manual review for final pass
```

**Workflow Behavior:**

```
Review step completes:
  ↓
Extract feedback items (Task 227)
  ↓
[If auto_apply != none]
  ↓
Run verification on draft items (Task 242.01-03)
  ↓
[For each verified item]
  → Insert "apply-feedback" step
  → Mark feedback as done after application
  ↓
[For uncertain items]
  → Log for human review (don't block)
  ↓
Continue to next step
```

### Expected Behavior

1. **Extraction**: After review, automatically extract findings to feedback items
2. **Verification**: Run CoV on all draft items
3. **Application**: For `verified` items, insert fix steps into queue
4. **Reporting**: Log uncertain/invalid items without blocking workflow

## Implementation Notes

### Auto-Apply Levels

| Level | Behavior |
|-------|----------|
| `none` | Extract feedback only, no verification or application |
| `verified` | Verify all, apply only verified items (default) |
| `all` | Apply all draft items without verification (aggressive) |

### Workflow Integration Points

```markdown
### After Review Steps (in drive-coworker-session.wf.md)

When completing a step with `skill: ace:review-pr`:

1. Check session/step `auto_apply` setting
2. If `none`: Advance to next step (extraction only)
3. If `verified` or `all`:
   a. Run `ace-review feedback extract` (from Task 227)
   b. If `verified`: Run `ace-review feedback verify-all --status draft`
   c. For each applicable item:
      - Run `ace-coworker add "apply-feedback-{id}" --instructions "{fix instructions}"`
   d. Advance to next step (inserted fix steps will execute in order)
```

### Feedback Application Step Template

```yaml
name: apply-feedback-8o7abc
skill: ace:apply-fix
instructions:
  - "Apply verified fix: SQL Injection in User Input Handler"
  - "Location: src/handlers/user.rb:42-55"
  - "Fix: Use parameterized queries instead of string concatenation"
  - "After applying, mark feedback 8o7abc as done"
```

### Non-Blocking Uncertain Items

When items are `uncertain`:
- Log: "Feedback 8o7abd needs human review: {reason}"
- Continue workflow without blocking
- User can review uncertain items after workflow completes
- Consider: Add optional `--pause-on-uncertain` flag for strict workflows
