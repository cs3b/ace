# 242.02: Feedback Model Extensions

## Objective

Extend Task 227's FeedbackItem model with verification-related fields to support the CoV verification workflow.

## Behavioral Specification

### User Experience

- **Input**: Existing feedback item + verification results
- **Process**: Store verification data in frontmatter
- **Output**: Feedback item file with verification section

### Interface Contract

**Extended Feedback Item Frontmatter:**

```yaml
---
id: 8o7abc
title: SQL Injection in User Input Handler
files:
  - src/handlers/user.rb:42-55
reviewer: google:gemini-2.5-flash
reviewers: [google:gemini-2.5-flash, anthropic:claude-sonnet]  # Consensus tracking
status: draft
priority: high
tags: [security, bug]
verification:
  status: verified  # verified | uncertain | invalid | pending
  questions:
    - question: "Does the code at src/handlers/user.rb:42-55 actually have SQL injection?"
      answer: "YES - user input is concatenated directly into SQL query"
      verdict: yes
    - question: "Would the suggested fix introduce new bugs?"
      answer: "NO - parameterized queries are a safe replacement"
      verdict: no
    - question: "Is this change aligned with task goal: Improve security?"
      answer: "YES - SQL injection is a critical security vulnerability"
      verdict: yes
  verified_at: 2026-01-29T17:00:00Z
  verified_by: google:gemini-2.5-flash
created: 2026-01-22T19:00:00Z
updated: 2026-01-29T17:00:00Z
---
```

### Expected Behavior

1. **Verification Status Lifecycle**:
   ```
   (no verification) → pending → verified | uncertain | invalid
   ```

2. **Consensus Tracking**: `reviewers` array tracks which reviewers found this issue (for weighting)

3. **Verification Persistence**: Questions, answers, and timestamps stored for full auditability

## Implementation Notes

### Verification Status Values

| Status | Meaning | Next Actions |
|--------|---------|--------------|
| `pending` | Not yet verified | Run verification |
| `verified` | CoV passed, safe to auto-apply | Auto-apply or manual apply |
| `uncertain` | Mixed results, needs human review | Present to user for decision |
| `invalid` | CoV found issue doesn't exist | Mark as skip/invalid |

### Model Structure

```ruby
module Ace::Review::Models
  class FeedbackItem
    VerificationQuestion = Struct.new(:question, :answer, :verdict, keyword_init: true)

    Verification = Struct.new(
      :status,        # :pending | :verified | :uncertain | :invalid
      :questions,     # Array<VerificationQuestion>
      :verified_at,   # Time
      :verified_by,   # String (model that did verification)
      keyword_init: true
    )

    attr_accessor :verification, :reviewers

    def verification_status
      verification&.status || :pending
    end
  end
end
```
