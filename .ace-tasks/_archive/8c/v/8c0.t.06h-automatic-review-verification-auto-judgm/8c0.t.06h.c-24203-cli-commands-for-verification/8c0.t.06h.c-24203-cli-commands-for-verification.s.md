# 242.03: CLI Commands for Verification

## Objective

Add CLI commands to ace-review for running and managing feedback verification.

## Behavioral Specification

### User Experience

- **Input**: Feedback item ID or filter criteria
- **Process**: Run verification on specified items
- **Output**: Verification results and updated feedback files

### Interface Contract

```bash
# Verify single feedback item
ace-review feedback verify 8o7abc
# Output:
# Verifying feedback 8o7abc: SQL Injection in User Input Handler
#   Q1: Does code have this issue? → YES
#   Q2: Would fix introduce bugs? → NO
#   Q3: Aligned with task goal? → YES
# Result: verified ✓

# Batch verify all draft items
ace-review feedback verify-all --status draft
# Output:
# Verifying 5 feedback items...
#   8o7abc: verified ✓
#   8o7abd: uncertain ⚠ (needs review)
#   8o7abe: invalid ✗
#   8o7abf: verified ✓
#   8o7abg: uncertain ⚠ (needs review)
# Summary: 2 verified, 2 uncertain, 1 invalid

# List by verification status
ace-review feedback list --verification-status uncertain
# Output:
# Uncertain feedback items (need human review):
#   8o7abd: Performance issue in loop (uncertain - conflicting answers)
#   8o7abg: Missing error handling (uncertain - goal alignment unclear)

# Dry-run verification (show questions without running)
ace-review feedback verify 8o7abc --dry-run
# Output:
# Would verify 8o7abc with questions:
#   1. Does the code at src/handlers/user.rb:42-55 actually have SQL injection?
#   2. Would the suggested fix introduce new bugs?
#   3. Is this change aligned with task goal: Improve security?
```

### Expected Behavior

1. **Single Item Verify**: Run CoV on one feedback item, update file with results
2. **Batch Verify**: Run CoV on all items matching filter, show summary
3. **List by Verification**: Filter feedback list by verification status
4. **Dry-Run**: Preview verification questions without LLM calls

## Implementation Notes

### Command Structure

```ruby
module Ace::Review::CLI::Commands
  class Feedback < Dry::CLI::Command
    # Existing from 227.06: list, show, update

    class Verify < Dry::CLI::Command
      desc "Verify a feedback item using Chain of Verification"
      argument :id, required: true, desc: "Feedback item ID"
      option :dry_run, type: :boolean, default: false, desc: "Show questions without running"
      option :task, desc: "Task context for goal alignment"
    end

    class VerifyAll < Dry::CLI::Command
      desc "Verify all feedback items matching filter"
      option :status, desc: "Filter by status (draft, pending)"
      option :task, desc: "Task context for goal alignment"
    end
  end
end
```
