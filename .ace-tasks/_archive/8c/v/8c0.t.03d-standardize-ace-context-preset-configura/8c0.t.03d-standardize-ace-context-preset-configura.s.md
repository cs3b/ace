# Standardize ace-context Preset Configuration Loading

## Behavioral Specification

### User Experience

- **Input**: Preset or file with `context: presets: [preset-name]` at top level
- **Process**: ace-context loads referenced presets, merges their content (files, commands, params), with current config taking precedence
- **Output**: Combined context with content from all referenced presets

### Expected Behavior

When a preset or file configuration contains `context: presets: [base-preset]` at the top level (not nested within sections), ace-context should:
1. Load each referenced preset using the existing composition mechanism
2. Merge files, commands, and parameters from referenced presets
3. Apply "current config wins" precedence (same as sections)
4. Remove the `presets` key after processing

Currently, this only works when presets are nested within sections (`context: sections: my-section: presets: [...]`).

### Interface Contract

```yaml
# SHOULD WORK (currently broken):
---
context:
  presets:
    - project-base
  files:
    - additional-file.md
---

# ALREADY WORKS:
---
context:
  sections:
    my-section:
      presets:
        - project-base
      files:
        - additional-file.md
---
```

### Success Criteria

- [x] Top-level `context: presets: [...]` loads and merges referenced presets

- [x] Missing presets return error (consistent with existing composition behavior)

- [x] All existing tests pass
