# Address Review Feedback (121.04) - LLM Enhancement Fixes

## Summary

Fix the broken LLM enhancement feature in ace-prompt based on code review findings from PR #54. The feature is non-functional due to prompt:// source not being registered and unnecessary ModelAliasResolver class.

## Issues to Fix

### P0 - Blocking

1. **prompt:// source not registered** - Enhancement can't load system prompt
2. **ModelAliasResolver is redundant** - ace-llm handles aliases internally; remove this class

### P1 - Important

3. **Cache key incomplete** - Only hashes content, ignores model/system-prompt/temperature
4. **Archive path mismatch** - Returns original archive_path instead of enhanced version

### P2 - Quality

5. **Missing integration tests** - No CLI test for `--enhance` flow
6. **Missing documentation** - README/usage.md don't describe enhancement

## Scope

- Register ace-prompt as prompt:// source in ace-nav configuration
- Remove redundant ModelAliasResolver (pass aliases directly to ace-llm)
- Fix prompt:// protocol resolution in PromptEnhancer
- Update cache key to include model/system-prompt/temperature
- Fix archive path to return enhanced version
- Add integration test for --enhance flow
- Update documentation

## Implementation Plan

### Step 1: Register prompt:// source for ace-prompt

Create `ace-prompt/.ace.example/nav/protocols/prompt-sources/ace-prompt.yml`:
```yaml
---
name: ace-prompt
type: gem
description: Prompt enhancement system prompts
priority: 10

config:
  relative_path: handbook/prompts
  pattern: "*.md"
  enabled: true

categories:
  - enhancement
```

Also create project-level copy at `.ace/nav/protocols/prompt-sources/ace-prompt.yml`

### Step 2: Remove ModelAliasResolver

- Delete `ace-prompt/lib/ace/prompt/atoms/model_alias_resolver.rb`
- Delete `ace-prompt/test/atoms/model_alias_resolver_test.rb`
- Update `ace-prompt/lib/ace/prompt.rb` - Remove require
- Update `prompt_enhancer.rb` - Pass model directly to `Ace::LLM::QueryInterface.query()`

### Step 3: Fix prompt:// resolution

In `prompt_enhancer.rb`, change:
```ruby
result = engine.resolve(uri_or_path)
if result && result[:success] && result[:path]
```
To:
```ruby
result = engine.resolve(uri_or_path, content: true)
return result if result.is_a?(String) && !result.empty?
```

### Step 4: Fix cache key

In `enhancement_tracker.rb`, update to include all parameters:
```ruby
def self.cache_key(content, model, system_prompt_uri, temperature)
  key_material = "#{content}|#{model}|#{system_prompt_uri}|#{temperature}"
  Digest::SHA256.hexdigest(key_material)
end
```

### Step 5: Return enhanced archive path

In `prompt_processor.rb`, after writing enhanced archive:
```ruby
Molecules::PromptArchiver.update_symlink(enhanced_archive_path)
archive_path = enhanced_archive_path
```

### Step 6: Add integration test

Add test to `cli_integration_test.rb` that:
1. Creates a prompt file
2. Runs `ace-prompt -e` with mocked LLM
3. Asserts archive points to enhanced content

### Step 7: Update documentation

- `ace-prompt/README.md` - Add enhancement section
- `ace-prompt/docs/usage.md` - Document `--enhance`, `--model`, config
