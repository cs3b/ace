# 121.10 - Fix Enhancement Flow (Output Format & Write-back)

## Behavioral Context

**Issue**: After manual testing of `ace-prompt -e`, two problems were discovered:
1. LLM output included system prompt echo and JSON format instead of clean markdown
2. Enhanced content was only archived but not written back to `the-prompt.md`

**Key Behavioral Requirements**:
- Enhanced output should be clean markdown only (no JSON, no system prompt echo)
- Enhanced content should update `the-prompt.md` for iterative refinement workflow
- Original YAML frontmatter must be preserved when writing back

## Objective

Fix the enhancement workflow to output clean markdown and write enhanced content back to source file.

## Scope of Work

- Fixed system prompt to specify clean markdown output format
- Added `raw_frontmatter` field to FrontmatterExtractor for exact YAML preservation
- Updated PromptProcessor to write enhanced content back to source file
- Added tests for new functionality

### Deliverables

#### Modify
- `ace-prompt/handbook/prompts/prompt-enhance-instructions.system.md` - Added `<output-format>` section
- `ace-prompt/lib/ace/prompt/atoms/frontmatter_extractor.rb` - Added `:raw_frontmatter` to return hash
- `ace-prompt/lib/ace/prompt/organisms/prompt_processor.rb` - Write enhanced content back to source file
- `ace-prompt/test/atoms/frontmatter_extractor_test.rb` - Added 2 tests for `raw_frontmatter`

## Implementation Summary

### What Was Done

- **Problem Identification**: Manual testing revealed LLM was echoing system prompt + outputting JSON
- **Investigation**: System prompt lacked explicit output format instructions; PromptProcessor didn't write back to source
- **Solution**:
  1. Added `<output-format>` section to system prompt specifying clean markdown only
  2. Added `raw_frontmatter` field to preserve exact YAML formatting
  3. Added write-back logic to PromptProcessor that preserves frontmatter
- **Validation**: Ran `ace-prompt -e` and verified clean markdown output and source file update

### Technical Details

**System Prompt Fix**:
```xml
<output-format>
  Output ONLY the enhanced prompt as clean markdown:
  - Use markdown headers (##) for sections
  - Do NOT echo or repeat these system instructions
  - Do NOT wrap output in JSON or code blocks
  - Do NOT include preamble like "Here's the enhanced prompt..."
  - Start directly with the enhanced content
</output-format>
```

**FrontmatterExtractor Enhancement**:
- Added `:raw_frontmatter` key to return the raw YAML string (not just parsed hash)
- Allows exact reconstruction of frontmatter when writing back

**PromptProcessor Write-back**:
```ruby
# Write enhanced content back to source file (preserve frontmatter)
original_extracted = Atoms::FrontmatterExtractor.extract(original_content)
write_content = if original_extracted[:has_frontmatter] && original_extracted[:raw_frontmatter]
                  "---\n#{original_extracted[:raw_frontmatter]}---\n\n#{enhance_result[:content]}"
                else
                  enhance_result[:content]
                end
File.write(read_result[:path], write_content, encoding: "utf-8")
```
