# Address Review Feedback (121.02)

## Behavioral Specification

### User Experience

- **Input**: User runs `ace-prompt setup` or `ace-prompt setup --template bug` from a project directory
- **Process**: System initializes prompt workspace in project's `.cache/ace-prompt/prompts/`, archives existing prompt if present, copies template
- **Output**: Prompt file created at `{project_root}/.cache/ace-prompt/prompts/the-prompt.md` with confirmation message

### Expected Behavior

1. **Project Root Usage**: `ace-prompt setup` creates files in the current project's `.cache/ace-prompt/prompts/` directory, NOT in `~/.cache/ace-prompt/`

2. **Consolidated Setup Command**: Single `setup` command that:
   - Archives existing `the-prompt.md` if present (unless `--no-archive`)
   - Copies template to `the-prompt.md`
   - Always outputs to `the-prompt.md` regardless of template used

3. **Template Short Form**: Users can specify templates using:
   - Short form: `--template bug` → resolves to `tmpl://ace-prompt/the-prompt-bug`
   - Full URI: `--template tmpl://ace-prompt/the-prompt-bug`

4. **ace-nav Ruby API**: Template resolution uses ace-nav's Ruby API directly (not shelling out to CLI)

### Interface Contract

```bash
# Setup with default template (always archives existing)
ace-prompt setup
# Output:
# Prompt initialized:
#   Path: /path/to/project/.cache/ace-prompt/prompts/the-prompt.md
#   Archive: /path/to/project/.cache/ace-prompt/prompts/archive/the-prompt-20251128-191500.md

# Setup with short template name
ace-prompt setup --template bug
# Uses tmpl://ace-prompt/the-prompt-bug

# Setup with full template URI
ace-prompt setup --template tmpl://ace-prompt/the-prompt-bug

# Setup without archiving existing
ace-prompt setup --no-archive

# Force overwrite (alias for --no-archive, backward compat)
ace-prompt setup --force
```

**Error Handling:**
- Template not found: `Error: Template not found: tmpl://ace-prompt/the-prompt-bug`
- No project root: Falls back to current directory

**Edge Cases:**
- No existing prompt: Creates new prompt without archive step
- Invalid short form: Tries to resolve as full URI, then fails gracefully
