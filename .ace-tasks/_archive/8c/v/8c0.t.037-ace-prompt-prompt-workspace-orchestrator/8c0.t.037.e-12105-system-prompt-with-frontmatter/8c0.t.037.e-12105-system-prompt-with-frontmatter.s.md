# 121.05 - System Prompt with Frontmatter

## Scope

System prompt can have its own frontmatter for context loading via ace-context.

## Interface

No new CLI flags - extends existing `--enhance` behavior.

## System Prompt Format

```markdown
---
context:
  files:
    - docs/architecture.md
    - docs/ace-gems.g.md
  presets:
    - project
---

You are a prompt enhancement assistant. Given the project context above,
improve the user's prompt for clarity and completeness.

## Enhancement Guidelines

1. Make requirements explicit
2. Add relevant context references
3. Specify expected output format
...
```

## Behavior

When `--enhance` is used:

1. Load system prompt via config `system-prompt` (default: `prompt://prompt-enhance-instructions`)
2. Check if system prompt has `context` frontmatter
3. If yes: process via ace-context (same delegation pattern)
4. Final system prompt = context-enriched system prompt
5. Proceed with normal enhancement flow

### Edge Cases for Testing

#### EnhancementSessionManager
- System prompt has no context frontmatter (use as-is)
- System prompt has context but `--no-context` flag (skip context loading)
- System prompt file not found (helpful error)
- System prompt context loading fails (use raw system prompt, log warning)
- Both system and user prompts have context (process both independently)
- Circular context references (max depth enforcement)

#### System Prompt Loading
- Custom `--system-prompt` protocol resolution
- Protocol returns nil/empty (use default)
- Multiple system prompt sources (priority: CLI > config > default)
- System prompt with embedded variables (future consideration)
- System prompt file is empty
- System prompt with only frontmatter, no instructions

### Technical Details

**EnhancementSessionManager Flow:**
1. Load system prompt content via `PromptEnhancer.load_system_prompt()`
2. Check for `context:` key in frontmatter
3. If present, resolve protocol URI to file path (e.g., `prompt://` → `/path/to/file.md`)
4. Process via `Ace::Context.load_file()` with format: markdown
5. Return context-enriched system prompt or fallback to body if loading fails

**System Prompt Frontmatter:**
```yaml
context:
  presets:
    - project
```

**Graceful Fallback:**
- If ace-context not available → use raw system prompt body
- If context loading fails → use raw body, log warning
- If protocol resolution fails → use raw body
