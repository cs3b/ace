# 121.06 - Task-ID Folder Support

## Scope

Use task directory for prompts via CLI flag or branch name detection.

## Interface

```bash
ace-prompt --task 117          # Use task 117's prompt folder
ace-prompt -t 117              # Short flag
```

## Configuration

```yaml
# .ace/prompt/config.yml
prompt:
  params:
    task-detection: false  # --task / auto-detect from branch (117-feature-name → task 117)
```

## Behavior

### With `--task N` flag:
1. Look for task N in current release via ace-taskflow
2. Use `{task-dir}/prompts/` as prompt directory
3. All operations (read, archive, setup, reset) use task path
4. CLI flag always overrides config/detection

### With branch detection (config enabled):
1. Parse current branch name for task number
2. Pattern: `117-feature-name` or `121.01-archive-output` → extract first number
3. If valid task found in release, use its directory
4. If not found, fall back to default location

## File Structure with Task

```
.ace-taskflow/v.0.9.0/tasks/117-feature/
└── prompts/
    ├── the-prompt.md
    ├── _previous.md
    └── archive/
        └── 20251127-143000.md
```

### Edge Cases for Testing

#### TaskPathResolver
- Task ID not found in release (helpful error)
- Multiple matching task directories (117-feature-a, 117-feature-b - use first match)
- Task directory has no prompts/ subdirectory (create it)
- Nested task structure (subtasks like 121.01)
- Invalid task ID format (non-numeric - helpful error)
- Task ID with subtask format (121.01 - resolve parent task)

#### Branch Detection
- Branch name without task ID (`main`, `feature-x` - no detection)
- Branch with multiple numbers (`fix-123-update-456` - use first)
- Branch with subtask format (`121.01-archive-output`) - first try to resolve `121.01` directory, fall back to parent `121`
- Detached HEAD state (no detection, use default)
- Git not available (graceful fallback to default)

#### CLI
- `--task` overrides branch detection
- `--task` with non-existent task (helpful error with available tasks)
- Task detection disabled in config (ignore branch name)
- Task prompts/ directory missing (create or error based on command)
- `--task` combined with all other flags (`-t 117 -ce`)
