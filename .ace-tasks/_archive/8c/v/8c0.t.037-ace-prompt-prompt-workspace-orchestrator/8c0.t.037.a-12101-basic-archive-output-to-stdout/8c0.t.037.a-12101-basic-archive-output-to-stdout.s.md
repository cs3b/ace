# 121.01 - Basic: Archive + Output to stdout

## Scope

Minimal viable ace-prompt - read prompt file, archive it with timestamp, output to stdout.

## Interface

```bash
ace-prompt                    # Read, archive, output to stdout
ace-prompt --output path      # Output to file instead of stdout
ace-prompt --output -         # Explicit stdout (same as default)
```

## Behavior

1. Read `.cache/ace-prompt/prompts/the-prompt.md`
2. Archive to `archive/YYYYMMDD-HHMMSS.md`
3. Update `_previous.md` symlink to point to archived file
4. Output content to stdout (or file with --output)

## File Structure

```
.cache/ace-prompt/prompts/
├── the-prompt.md         # Active prompt (user edits this)
├── _previous.md          # Symlink to last archived
└── archive/
    └── 20251127-143000.md
```

### Edge Cases for Testing

#### PromptReader
- Empty prompt file (0 bytes)
- File with only whitespace
- Non-existent file path (helpful error)
- Unicode content (日本語, Привет, café)
- Very large file (100KB+)
- File with Windows line endings (\r\n)
- File with UTF-8 BOM marker
- Symlinked prompt file

#### PromptArchiver
- Archive directory doesn't exist (should create)
- Source file missing (return nil gracefully)
- Empty content archiving
- Concurrent archiving (thread safety)
- Very long content (16KB+)
- UTF-8 content preservation
- `_previous.md` symlink already exists (overwrite)
- Invalid symlink target
- Timestamp collision (same second - append suffix like -1, -2 to avoid overwriting)

#### ContentHasher
- Empty string input
- nil input (return default/empty)
- Unicode content hashing
- Very long content (consistent hash)

#### CLI
- `--output` to non-existent directory (error)
- `--output -` (stdout explicit)
- No arguments (default behavior)
- Permission denied on output path
