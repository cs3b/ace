# 121.04 - ace-llm-query Enhancement

## Scope

Optional LLM-powered prompt enhancement with caching.

## Interface

```bash
ace-prompt --enhance           # Enable LLM enhancement
ace-prompt -e                  # Short flag
ace-prompt --no-enhance        # Disable LLM enhancement
ace-prompt -ce                 # Combine context + enhance
```

## Configuration

```yaml
# .ace/prompt/config.yml
prompt:
  params:
    enhance: false              # --enhance / --no-enhance
    model: glite                # --model (alias for google:gemini-2.0-flash-lite)
    temperature: 0.3            # --temperature
    system-prompt: prompt://prompt-enhance-instructions  # --system-prompt
```

## Model Aliases

Built-in convenience aliases:
- `glite` → `google:gemini-2.0-flash-lite` (default, fast)
- `claude` → `anthropic:claude-sonnet-4-20250514`
- `haiku` → `anthropic:claude-3-haiku-20240307`

## Behavior

1. Archive original prompt (always, before enhancement)
2. Hash content for cache lookup
3. If cached: return cached enhancement
4. If not cached:
   - Load system prompt from config `system-prompt` (default: `prompt://prompt-enhance-instructions`)
   - Call `ace-llm-query` with system + user prompt
   - Save result as markdown (human-readable)
   - Archive enhanced result with `_e001` suffix
5. Output enhanced content

## Archive Naming

```
archive/
├── 20251127-143000.md        # Original
├── 20251127-143000_e001.md   # Enhanced (iteration 1)
└── 20251127-143000_e002.md   # Re-enhanced (iteration 2)
```

### Edge Cases for Testing

#### ModelAliasResolver
- Unknown alias (return as-is, let LLM handle)
- nil input (return default model)
- Empty string alias (return default model)
- Full provider:model format (pass through unchanged)
- Case sensitivity (`Glite` vs `glite` - normalize)

#### EnhancementTracker
- No previous enhancements (start at _e001)
- Existing _e001, _e002 (create _e003)
- Gap in sequence (_e001, _e003 exists - create _e004)
- Cache hit (same content hash - return cached)
- Cache directory doesn't exist (create it)
- Corrupted cache file (ignore, regenerate)

#### PromptEnhancer
- Missing ace-llm-query gem (return original, log warning)
- LLM API error (return original, log warning)
- Empty prompt content
- Very long prompt (handle gracefully, may truncate)
- Enhancement returns empty (use original)
- `--no-enhance` overrides config `enhance: true`
- `--system-prompt` custom path not found (helpful error)
- System prompt with invalid frontmatter

#### Archive Naming
- Multiple enhancements same second (_e001, _e002)
- Original not archived yet (archive first)
- Enhancement of already enhanced content (_e001 → _e002) - re-running `--enhance` on same prompt creates _e002, _e003 etc. based on matching original timestamp
