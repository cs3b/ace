# ace-prompt: Address Review Feedback (121.01)

## Overview

Address code review feedback from 3 AI reviewers (GPT-5.1, GPro, GLM-4.6) for task 121.01 (Basic Archive + Output). The Kimi-K2 review produced no output.

**Review Reports**: `review/task.121.01-01/`

## Objective

Improve code quality and ACE pattern compliance based on multi-model review feedback. Primary focus: CLI exit pattern violation that impacts testability.

## Scope

Fix HIGH and MEDIUM priority issues identified in reviews:
- CLI exit pattern violation (ACE testing patterns)
- Missing CLI integration tests
- README ADR-004 path violation
- Unused parameter cleanup
- Code simplification

## Technical Approach

### Architecture Pattern

The fix follows the established ACE CLI pattern used in ace-taskflow:

**Current (violation):**
```ruby
# cli.rb - calls exit directly
exit 1 if error

# exe/ace-prompt - ignores return value
Ace::Prompt::CLI.start(ARGV)
```

**Target (ACE pattern):**
```ruby
# cli.rb - returns status codes
return 1 if error
return 0  # success

# exe/ace-prompt - captures and exits
exit_code = Ace::Prompt::CLI.start(ARGV)
exit(exit_code || 0)
```

### Reference Implementation

`ace-taskflow/exe/ace-taskflow:6-7` demonstrates the correct pattern:
```ruby
exit_code = Ace::Taskflow::CLI.start(ARGV)
exit(exit_code || 0)
```
