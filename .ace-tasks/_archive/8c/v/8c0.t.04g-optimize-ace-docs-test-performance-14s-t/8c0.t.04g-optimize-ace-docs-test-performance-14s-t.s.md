# Optimize ace-docs test performance (14s to under 5s)

## Behavioral Specification

### User Experience
- **Input**: Developer runs `ace-test ace-docs` to execute test suite
- **Process**: Tests execute with properly mocked git operations via DiffOrchestrator
- **Output**: Full test results in ~1.5 seconds (down from 13.97s, 89% reduction)

### Expected Behavior
Developers experience faster test execution. Git diff operations are mocked at the `DiffOrchestrator.generate` level to prevent real git I/O. Tests that previously took 1+ second each now complete in milliseconds.

### Success Criteria

- [x] Test suite runs in <5 seconds (currently 13.97s) → **Now 1.5-1.7s (89% reduction)**
- [x] All 153 tests pass
- [x] Git operations properly mocked in unit tests
- [x] No real git diff operations in ChangeDetector unit tests

## Objective

Reduce ace-docs test execution time by 64%+ (from 14s to <5s) by properly mocking git operations in ChangeDetector tests.

## Scope of Work

### Root Cause Analysis (actual finding)
- **Primary Issue**: 5 tests in `change_detector_test.rb` were stubbing `ChangeDetector.stub :execute_git_command` but the actual code path uses `Ace::Git::Organisms::DiffOrchestrator.generate`
- Tests were running real git operations (~1 second each) because the wrong method was being mocked
- Total overhead: ~2.4s from change_detector_test alone (out of 4.1s total)

### Files Modified
- `ace-docs/test/molecules/change_detector_test.rb` - Fixed mocking pattern in 5 tests

### Optimization Applied
Changed 5 tests to use the correct mock pattern:
```ruby
# Before (wrong - execute_git_command not called in this path)
ChangeDetector.stub :execute_git_command, "" do

# After (correct - DiffOrchestrator.generate is the actual call)
empty_result = Ace::Git::Models::DiffResult.empty
Ace::Git::Organisms::DiffOrchestrator.stub :generate, empty_result do
```

### Tests Fixed
1. `test_diff_with_exclude_renames_option` (line 122)
2. `test_determine_since_uses_document_last_updated` (line 142) - was taking 1.09s
3. `test_determine_since_with_explicit_date` (line 163) - was taking 1.07s
4. `test_determine_since_with_string` (line 181)
5. `test_get_diff_for_document_multi_subject` (line 266) - was taking 0.19s

## Verification

To reproduce the performance benchmarks:

```bash
ace-test ace-docs --profile 10
```

## Lessons Learned

**Profile before assuming**: Initial hypothesis focused on DocumentRegistry filesystem discovery and LLM stubs. Actual root cause was incorrect mock patterns for git operations. Running `ace-test --profile 10` immediately revealed the slow tests.

**Match mocks to actual code paths**: When code evolves (e.g., `execute_git_command` → `DiffOrchestrator.generate`), tests may continue to pass while mocks become ineffective. Always verify mock methods match actual call paths.

**Extract test helpers for repeated patterns**: After fixing, the same stub pattern appeared 6 times. Extracting `with_empty_git_diff` helper improved maintainability.
