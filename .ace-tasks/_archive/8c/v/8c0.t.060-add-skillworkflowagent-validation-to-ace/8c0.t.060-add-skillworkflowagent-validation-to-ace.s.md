# Add skill/workflow/agent validation to ace-lint

## Behavioral Context

**Issue**: The ace-lint tool could lint Ruby code, Markdown files, and YAML configs, but had no validation for skill definitions (SKILL.md), workflow instructions (*.wf.md), or agent configs (*.ag.md). These files have specific structural requirements that should be validated.

**Key Behavioral Requirements**:
- Auto-detect file types based on naming patterns
- Validate required frontmatter fields per file type
- Check allowed-tools declarations against known tool schemas
- Validate comment blocks have proper structure
- Support schema-driven validation with customizable defaults

## Objective

Add comprehensive validation support for skill, workflow, and agent markdown files to ace-lint, including required field checks, allowed-tools validation, and comment structure validation.

## Scope of Work

- Extend TypeDetector to recognize skill/workflow/agent files
- Create schema loader for skill validation rules
- Implement allowed-tools validator against known Claude tools
- Implement comment validator for markdown comment blocks
- Create SkillValidator molecule orchestrating validation
- Integrate into LintOrchestrator for automatic detection
- Add comprehensive test coverage

## Implementation Summary

### What Was Done

- **Type Detection**: Extended TypeDetector to recognize SKILL.md, *.wf.md, and *.ag.md files with appropriate categorization
- **Schema System**: Created SkillSchemaLoader that loads validation rules from YAML config files with sensible defaults
- **Tool Validation**: Built AllowedToolsValidator that checks declared tools against known Claude Code tools (Read, Write, Edit, Bash, Glob, Grep, etc.)
- **Comment Validation**: Implemented CommentValidator for verifying HTML-style markdown comments have proper structure
- **Orchestration**: SkillValidator molecule ties together field validation, tool checks, and comment validation
- **Integration**: LintOrchestrator now auto-routes skill/workflow/agent files to SkillValidator

### Technical Details

File type detection patterns:
- `SKILL.md` → skill type
- `*.wf.md` → workflow type
- `*.ag.md` → agent type

Validation checks performed:
1. Required frontmatter fields (configurable per type)
2. Allowed-tools against known Claude tools list
3. Comment block structure (opening/closing tags)
4. Field format validation
