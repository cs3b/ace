## Objective

Eliminate duplicate wrapper classes in ace-context by merging wrapper and business logic into single dry-cli command classes. This reduces indirection and aligns with established patterns in ace-llm-models-dev.

## Technical Research Findings

### Validation Questions Answered

1. **Organism layer**: Yes, `Organisms::ContextLoader` exists and provides the core context loading API. The `LoadCommand` class uses `Ace::Context.load_auto`, `Ace::Context.inspect_config`, and `Ace::Context.load_multiple_inputs` - these are module-level methods that delegate to `ContextLoader`. The `LoadCommand` primarily handles CLI-specific concerns: option processing, output mode handling, cache file writing, and config summary display.

2. **Test locations**: Tests are in:
   - `test/commands/cli_routing_test.rb` - Tests CLI routing (does NOT test LoadCommand/ListCommand directly)
   - `test/integration/cli_*.rb` - Integration tests that invoke CLI through `Ace::Context::CLI.start()`

3. **Dependencies**: `LoadCommand` and `ListCommand` are ONLY used from their respective wrapper classes (`load.rb` and `list.rb`). No external imports.

### Reference Pattern Analysis

The **ace-llm-models-dev** package demonstrates the target pattern where business logic is directly in dry-cli commands:

```ruby
# ace-llm-models-dev/lib/ace/llm/models_dev/cli/models/search.rb
class Search < Dry::CLI::Command
  include Ace::Core::CLI::DryCli::Base

  def call(query: nil, **options)
    # Direct business logic - no wrapper delegation
    searcher = Molecules::ModelSearcher.new
    filters = parse_filters(options[:filter])
    result = searcher.search(query, provider: options[:provider], ...)
    # Handle output directly
  end
end
```

### Current Pattern Analysis

**Load command** (`load.rb` + `load_command.rb`):
- `load.rb` (113 lines): CLI definition, option handling, help text, option preprocessing
- `load_command.rb` (189 lines): Business logic - process_options, handle_output, write_to_cache, display_config_summary

**List command** (`list.rb` + `list_command.rb`):
- `list.rb` (31 lines): CLI definition, minimal
- `list_command.rb` (31 lines): Single `execute` method - very simple

## Technical Approach

### Architecture Pattern
- **Pattern**: Direct dry-cli integration (matching ace-llm-models-dev)
- **Rationale**: Eliminates unnecessary abstraction layer; single file per command is easier to maintain
- **Integration**: Commands continue to use `Ace::Context.*` API methods which delegate to `ContextLoader`

### Implementation Strategy
1. Merge `load_command.rb` private methods into `load.rb`
2. Merge `list_command.rb` logic into `list.rb`
3. Update require statements in `cli.rb`
4. Delete legacy `*_command.rb` files
5. Run tests to verify no regressions

### Risk Assessment

**Technical Risks:**
- **Risk:** Merge introduces subtle behavioral changes
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Run full test suite before/after, manual CLI testing
  - **Rollback:** Git revert if issues discovered

- **Risk:** Test mocks in `cli_routing_test.rb` may break
  - **Probability:** Medium (tests stub `LoadCommand.new`, `ListCommand.new`)
  - **Impact:** Low (tests need updating)
  - **Mitigation:** Update test mocks to work with merged commands
