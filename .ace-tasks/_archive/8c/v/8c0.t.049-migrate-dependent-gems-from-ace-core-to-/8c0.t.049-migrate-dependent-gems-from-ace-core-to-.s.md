## Technical Approach

### Migration Pattern

Each file requires two changes:

1. **Update require statement:**
```ruby
# Before:
require "ace/core/molecules/project_root_finder"

# After:
require "ace/support/fs"
```

2. **Update class references:**
```ruby
# Before:
Ace::Core::Molecules::ProjectRootFinder.find_or_current

# After:
Ace::Support::Fs::Molecules::ProjectRootFinder.find_or_current
```

### Test Stub Pattern

Tests using stubs need to stub the ace-support-fs version:
```ruby
# Before:
Ace::Core::Molecules::ProjectRootFinder.stub :find_or_current, "/path" do

# After:
Ace::Support::Fs::Molecules::ProjectRootFinder.stub :find_or_current, "/path" do
```

### API Compatibility

Verified: No production code uses `config_dir_name:` parameter (old ace-support-core API).
All DirectoryTraverser callers use `config_dir:` or no named parameter, so no API changes needed.

### Dependency Strategy

All gems already depend on ace-support-core, which transitively depends on ace-support-fs.
No gemspec changes required - ace-support-fs is automatically available.
