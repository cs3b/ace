# Migrate CLI Framework from Thor to dry-cli

## Objective

Replace Thor CLI framework with dry-cli across all 13 ACE CLI gems to eliminate:
1. **Option consumption conflicts** - Thor consumes declared options before calling method
2. **Nested subcommand limitations** - Thor issue #489 open since 2014
3. **Default command workarounds** - Manual handling needed
4. **Help flag boilerplate** - Required `if args.first == "--help"` everywhere

This is a framework swap - user-facing behavior must remain **identical**.

## Technical Approach

### Architecture Pattern

**Selected**: Direct dry-cli usage with thin wrapper in ace-support-core

**Key Files**:
- `ace-support-core/lib/ace/core/cli/dry_cli/base.rb` - Base module
- `ace-*/lib/ace/*/cli.rb` - Per-gem CLI (13 files)

### Per-Gem Migration Pattern

Each gem migration follows this pattern:
1. Convert `cli.rb` from Thor class to dry-cli Registry
2. Update gemspec: add `dry-cli ~> 1.1`
3. Run existing tests (must pass)
4. Manual verification of key commands
5. Update CHANGELOG

## Rollback Strategy

### Per-Gem Rollback (Mid-Migration Failure)

If a gem migration fails and needs to be reverted:

```bash
# 1. Restore original cli.rb from git
git checkout HEAD -- ace-{gem}/lib/ace/{gem}/cli.rb

# 2. Remove dry-cli from gemspec (keep thor)
# Edit ace-{gem}/ace-{gem}.gemspec

# 3. Restore original exe entry point if modified
git checkout HEAD -- ace-{gem}/exe/ace-{gem}

# 4. Verify tests pass
ace-test ace-{gem}
```

### Full Migration Rollback

If entire migration needs to be abandoned:

```bash
# 1. Revert all gem CLIs to Thor
git checkout origin/main -- ace-*/lib/ace/*/cli.rb

# 2. Remove dry-cli additions from ace-support-core
git checkout origin/main -- ace-support-core/lib/ace/core/cli/

# 3. Restore all gemspecs
git checkout origin/main -- ace-*/*.gemspec

# 4. Run full test suite
ace-test-suite
```
