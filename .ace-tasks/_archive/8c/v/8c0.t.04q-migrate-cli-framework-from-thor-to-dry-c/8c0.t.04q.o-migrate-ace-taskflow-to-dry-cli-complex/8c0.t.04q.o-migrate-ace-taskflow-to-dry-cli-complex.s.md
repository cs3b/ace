# Migrate ace-taskflow to dry-cli (Complex)

## Objective

Migrate ace-taskflow CLI from Thor to dry-cli. This is the most complex migration due to:
- Many subcommands (task, tasks, idea, ideas, release, releases, retro, retros, status, etc.)
- Default command routing (unknown → task)
- Class options shared across commands
- Cache clearing on start

## CLI Analysis

**Complexity**: Very Complex
**Commands**: task, tasks, idea, ideas, release, releases, retro, retros, status, doctor, migrate, config
**Aliases**: context → status
**Default routing**: Unknown commands → task (e.g., `ace-taskflow 150` → `ace-taskflow task 150`)
**Class options**: json, markdown, status, stats, tree, format, limit, etc.

## Implementation Notes

### Default Command Routing Pattern

Use the exe wrapper pattern validated in 179.02:

```ruby
#!/usr/bin/env ruby
require_relative "../lib/ace/taskflow"

# Known commands that should NOT be routed to "task"
KNOWN_COMMANDS = %w[
  task tasks idea ideas release releases retro retros
  status context doctor migrate config
  version --help -h --version
].freeze

# Route unknown first args to "task" command
# e.g., "ace-taskflow 150" → "ace-taskflow task 150"
unless KNOWN_COMMANDS.include?(ARGV.first) || ARGV.empty?
  ARGV.unshift("task")
end

result = Dry::CLI.new(Ace::Taskflow::CLI).call
exit(result.is_a?(Integer) ? result : 0)
```

### KNOWN_COMMANDS Pattern (Task 180)

Use the standardized REGISTERED_COMMANDS + BUILTIN_COMMANDS pattern from task 180:
```ruby
require "set"
# ...
module CLI
  extend Dry::CLI::Registry

  # Single source of truth for application commands
  REGISTERED_COMMANDS = %w[
    task tasks idea ideas
    release releases retro retros
    status doctor migrate config
  ].freeze

  # dry-cli built-ins (standard across all CLI gems)
  BUILTIN_COMMANDS = %w[version help --help -h --version].freeze

  # Auto-derived - no manual maintenance needed
  KNOWN_COMMANDS = Set.new(REGISTERED_COMMANDS + BUILTIN_COMMANDS).freeze

  DEFAULT_COMMAND = "task"

  def self.start(args)
    if args.empty? || !KNOWN_COMMANDS.include?(args.first)
      args = [DEFAULT_COMMAND] + args
    end
    Dry::CLI.new(self).call(arguments: args)
  end
end
```

Note: The exe wrapper pattern shown in Implementation Notes can also use this pattern.
