# Standardize KNOWN_COMMANDS pattern across dry-cli gems

## Risk Assessment

- **Test Impact**: All gems with KNOWN_COMMANDS changes will need test verification
- **Backward Compatibility**: CLI behavior is preserved (routing logic unchanged, only refactored)
- **Migration Risk**: Low - pattern is already partially implemented in some gems

## Behavioral Specification

### User Experience
- **Input**: Developers modifying CLI command registrations in dry-cli-based gems
- **Process**: When adding a new command, developers update one source of truth (REGISTERED_COMMANDS)
- **Output**: KNOWN_COMMANDS is automatically derived, maintaining consistency

### Expected Behavior

The current pattern requires manual maintenance of two separate lists:
1. Command registrations (`register "cmd", Commands::Cmd.new`)
2. KNOWN_COMMANDS array/Set for routing logic

This creates a duplication problem where adding a new command requires updating both places, leading to potential inconsistencies.

The standardized pattern centralizes command definitions:
- `REGISTERED_COMMANDS`: Lists all application commands (single source of truth)
- `BUILTIN_COMMANDS`: Lists dry-cli built-ins (help, version, etc.)
- `KNOWN_COMMANDS`: Auto-derived from the above two constants

### Interface Contract

```ruby
require "set"

module CLI
  extend Dry::CLI::Registry

  # Single source of truth for application commands
  REGISTERED_COMMANDS = %w[load list].freeze

  # dry-cli built-ins (standard across all CLI gems)
  BUILTIN_COMMANDS = %w[version help --help -h --version].freeze

  # Auto-derived - no manual maintenance needed
  KNOWN_COMMANDS = Set.new(REGISTERED_COMMANDS + BUILTIN_COMMANDS).freeze

  DEFAULT_COMMAND = "load"

  def self.start(args)
    if args.empty? || !KNOWN_COMMANDS.include?(args.first)
      args = [DEFAULT_COMMAND] + args
    end
    Dry::CLI.new(self).call(arguments: args)
  end
end
```

**Error Handling:**
- Missing command in REGISTERED_COMMANDS: Routed to default command with unknown token as argument
- Command not registered but in KNOWN_COMMANDS: dry-cli raises UnknownCommandError
- **Note**: Unknown tokens are intentionally routed to DEFAULT_COMMAND for tools that treat unknown arguments as file paths/patterns (e.g., `ace-lint file.md` routes to lint command with file.md as argument)

**Edge Cases:**
- Empty args: Routes to DEFAULT_COMMAND
- Flag as first arg: Passed through to dry-cli (--help, --version)
- Unknown token as first arg: Routed to DEFAULT_COMMAND with token as first argument (by design for DWIM tools)
