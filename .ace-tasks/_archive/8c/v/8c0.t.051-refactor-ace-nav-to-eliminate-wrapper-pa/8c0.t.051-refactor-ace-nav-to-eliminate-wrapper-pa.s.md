## Objective

Eliminate wrapper pattern in ace-nav by merging wrapper and business logic into single dry-cli command classes.

## Technical Approach

### Architecture Pattern
- **Pattern**: Merge dry-cli wrapper with business logic class
- **Rationale**: Eliminates indirection, simplifies maintenance, follows ace-timestamp pattern
- **Integration**: No changes to organisms/navigation_engine.rb or other dependencies

### Implementation Strategy
Each command file (resolve.rb, list.rb, etc.) currently:
1. Defines dry-cli options and arguments
2. Creates instance of *Command class
3. Calls execute() on the command

After refactoring, each file will:
1. Define dry-cli options and arguments (unchanged)
2. Contain the execute logic directly in `call()` method
3. Include all private helper methods from *Command class

### Key Considerations
- `resolve.rb` has special `magic_wildcard_pattern?` logic for auto-routing to list
- `resolve.rb` delegates to `Organisms::CommandDelegator` for `cmd://` protocol
- All commands share common config loading pattern via `display_config_summary`, `load_effective_config`, `default_config`
- Tests in `cli_test.rb` test `magic_wildcard_pattern?` via `send(:magic_wildcard_pattern?)` - need to ensure method remains accessible
