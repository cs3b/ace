# Migrate ace-docs to use ace-support-markdown

## Behavioral Specification

### User Experience
- **Input**: Developers use ace-docs commands for documentation generation and analysis
- **Process**: ace-docs uses ace-support-markdown for all markdown frontmatter operations
- **Output**: Documentation files are created/modified safely with validated frontmatter

### Expected Behavior

ace-docs will use the unified ace-support-markdown API instead of its custom FrontmatterManager and FrontmatterParser implementations. This consolidates duplicate code and ensures consistent, safe markdown handling across all ACE gems.

**Current Problem**: ace-docs has duplicate YAML parsing logic in FrontmatterParser and custom frontmatter manipulation in FrontmatterManager. This duplication creates maintenance burden and potential inconsistencies.

**Solution**: Replace FrontmatterManager with ace-support-markdown's DocumentEditor and delete FrontmatterParser in favor of FrontmatterExtractor.

### Interface Contract

**Before (current):**
```ruby
# FrontmatterManager
def update_document(document, updates)
  # Custom frontmatter update logic
  # Direct file write
end

# FrontmatterParser
def parse(content)
  # Custom YAML extraction
end
```

**After (migration):**
```ruby
# Replace FrontmatterManager
editor = Ace::Support::Markdown::DocumentEditor.new(doc_path)
editor.update_frontmatter(updates)
editor.save!(backup: true, validate: true)

# Replace FrontmatterParser (use directly)
doc = Ace::Support::Markdown::MarkdownDocument.parse(content)
frontmatter = doc.frontmatter
```

## Migration Notes

### Backward Compatibility Strategy

Since ace-docs is pre-1.0, we can make breaking changes, but we'll keep FrontmatterManager as a thin wrapper for smoother migration:

```ruby
# FrontmatterManager (backward compatible wrapper)
class FrontmatterManager
  def self.update_document(document, updates)
    editor = Ace::Support::Markdown::DocumentEditor.new(document.path)
    editor.update_frontmatter(updates)
    result = editor.save!(backup: true, validate: true)

    result[:success] ? document : raise(Error, result[:errors].join(", "))
  end
end
```

### FrontmatterParser Replacement

Direct usage patterns:

```ruby
# Before
result = FrontmatterParser.parse(content)
frontmatter = result[:frontmatter]
body = result[:body]

# After
doc = MarkdownDocument.parse(content)
frontmatter = doc.frontmatter
body = doc.raw_body
```
