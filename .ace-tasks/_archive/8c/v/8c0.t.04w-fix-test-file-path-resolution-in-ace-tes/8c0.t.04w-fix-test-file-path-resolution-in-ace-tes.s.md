# Fix Test File Path Resolution in ace-test-suite

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-test-suite` which executes tests across multiple packages
- **Process**: When tests fail, user sees failure summary with package names and file references
- **Output**: User receives accurate, usable file paths to locate failure details

### Expected Behavior

When `ace-test-suite` reports failed packages, the file paths shown in stdout must:
1. Point to files that actually exist
2. Be usable for navigation (can be copied/clicked to open the file)
3. Fall back gracefully when expected files don't exist

**Current Problem**: ace-test-suite shows paths like `ace-context/test-reports/latest/failures.json` but the file may not exist because:
- `failures.json` is only created when there ARE test failures (not when package fails for other reasons)
- The `latest` symlink may not exist if tests crashed before report generation
- Tests may have failed before any reports were created

**Desired Behavior**:
- Only show paths to files that exist
- If `failures.json` doesn't exist, show the best available alternative (summary.json, report.md, etc.)
- If no report files exist, show a helpful fallback message

### Interface Contract

```bash
# Current (problematic) output:
Failed packages:
  - ace-context: 2 failures, 1 error
    -> See ace-context/test-reports/latest/failures.json  # FILE DOESN'T EXIST!

# Expected output (when failures.json exists):
Failed packages:
  - ace-context: 2 failures, 1 error
    -> See ace-context/test-reports/latest/failures.json  # File exists, usable

# Expected output (when failures.json doesn't exist but summary.json does):
Failed packages:
  - ace-context: 0 failures, 1 error
    -> See ace-context/test-reports/latest/summary.json  # Falls back to available file

# Expected output (when no report files exist):
Failed packages:
  - ace-context: 0 failures, 1 error
    Error: No test results found (summary.json missing)
    -> Check ace-context/test-reports/ for details  # Helpful fallback
```

## Objective

Fix the broken user experience where ace-test-suite points users to non-existent files, breaking the debugging workflow for both human developers and AI agents.
