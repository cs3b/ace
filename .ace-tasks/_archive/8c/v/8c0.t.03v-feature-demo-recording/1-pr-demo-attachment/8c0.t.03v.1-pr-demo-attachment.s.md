# PR Demo Attachment

## Behavioral Specification

### User Experience
- **Input**: GIF path + PR number (or auto-detect from branch)
- **Output**: PR comment with embedded demo GIF
- **CLI**: `ace-demo attach <file> --pr <number>` and `ace-demo record <tape> --pr <number>`

### Expected Behavior

Attach a demo GIF to a GitHub PR as an inline-image comment. The GIF is uploaded to a `demo-assets` GitHub release (created on demand) for a stable URL, then a markdown comment with the embedded image is posted to the PR.

The system should:
- Upload GIF to a `demo-assets` GitHub release for stable URLs
- Create the `demo-assets` release on demand if it doesn't exist
- Format a markdown comment with `![Demo](url)` and recording metadata
- Post the comment to the specified PR via `gh pr comment`
- Support `--dry-run` to preview without posting
- Combine with `record` command via `--pr` flag for one-step record-and-attach

### Interface Contract

```bash
# Attach an existing GIF to a PR
ace-demo attach .ace-local/demo/hello.gif --pr 123
# → Uploaded: hello.gif → https://github.com/.../releases/download/demo-assets/hello-<timestamp>.gif
# → Posted demo comment to PR #123

# Record and attach in one step
ace-demo record hello --pr 123
# → Recorded: .ace-local/demo/hello.gif
# → Uploaded: hello.gif → https://github.com/.../releases/download/demo-assets/hello-<timestamp>.gif
# → Posted demo comment to PR #123

# Dry run (preview without posting)
ace-demo attach .ace-local/demo/hello.gif --pr 123 --dry-run
# → [dry-run] Would upload: hello.gif
# → [dry-run] Would post comment to PR #123:
# →   ## Demo: hello
# →   ![Demo](https://github.com/...)
# →   _Recorded at 2026-03-05 12:00:00_

# Missing PR
ace-demo attach .ace-local/demo/hello.gif --pr 999
# → Error: PR #999 not found

# gh not authenticated
ace-demo attach .ace-local/demo/hello.gif --pr 123
# → Error: gh CLI not authenticated. Run: gh auth login
```

### ATOM Component Design

| Layer | Component | Responsibility |
|-------|-----------|---------------|
| Atom | `DemoCommentFormatter` | Formats markdown comment body with `![Demo](url)`, tape name, timestamp metadata (pure function) |
| Molecule | `GhAssetUploader` | Uploads GIF to `demo-assets` GitHub release via `gh release upload`; creates release if needed |
| Molecule | `DemoCommentPoster` | Posts formatted comment to PR via `gh pr comment --body-file` |
| Organism | `DemoAttacher` | Orchestrates: upload GIF → format comment → post to PR |
| CLI | `Attach` | dry-cli command parsing `<file>`, `--pr`, `--dry-run` options |

### Image Hosting Strategy

Upload to a `demo-assets` GitHub release (tag: `demo-assets`):
- **Created on demand**: If the release doesn't exist, create it with `gh release create demo-assets --title "Demo Assets" --notes "Auto-generated release for demo GIF hosting"`
- **Unique filenames**: Append timestamp to avoid collisions (e.g., `hello-1709654400.gif`)
- **Stable URLs**: GitHub release asset URLs are permanent and publicly accessible
- **No branch pollution**: Keeps demo artifacts out of the git tree

### Success Criteria

- [ ] **GIF Upload**: Demo GIF uploaded to `demo-assets` release with stable URL
- [ ] **Release Creation**: `demo-assets` release created on demand when it doesn't exist
- [ ] **PR Comment**: Comment posted with inline `![Demo](url)` image visible in PR
- [ ] **Combined Flow**: `ace-demo record <tape> --pr <number>` records and attaches in one step
- [ ] **Dry Run**: `--dry-run` previews upload and comment without side effects
- [ ] **Error Handling**: Clear messages for missing PR, auth failure, upload failure
