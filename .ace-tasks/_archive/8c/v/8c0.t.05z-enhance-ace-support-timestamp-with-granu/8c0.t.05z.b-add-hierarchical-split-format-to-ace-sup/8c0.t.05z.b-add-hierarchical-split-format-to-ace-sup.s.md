# Add Hierarchical Split Format to ace-support-timestamp

## Description

Add a `--split` option to ace-support-timestamp that enables hierarchical folder structures from timestamps. The split format creates a parent-child hierarchy: month → week-in-month → day → block → precision.

This builds on the existing granular format templates (v0.3.0) by adding a new hierarchical encoding scheme designed for path construction.

**Simplification:** Always use day-of-month (0-30) for the `day` component. No day-of-week - less is more.

## Behavioral Specification

### Input
- `ace-timestamp encode "2025-01-06 12:30:00" --split month,week,day,block`
- Split levels: month, week, day, block (each is child of previous)

### Process
1. Parse split levels and validate hierarchy
2. Encode time using hierarchical format:
   - month: 2 chars (months offset)
   - week: 1 char (week-in-month, 0-4)
   - day: 1 char (day-of-month, 0-30)
   - block: 1 char (40-min block, 0-35)
   - rest: remaining precision
3. Output components and constructed path

### Output
```yaml
month: i5
week: 1
day: 5
block: j
rest: j3
path: i5/1/5/j/j3
full: i515jj3
```

### Interface Contract

```bash
# Encode with split (hierarchical)
ace-timestamp encode "2025-01-06 12:30:00" --split month
# → month: i5, rest: 5jj3, path: i5/5jj3

ace-timestamp encode "2025-01-06 12:30:00" --split month,week
# → month: i5, week: 1, rest: 5jj3, path: i5/1/5jj3

ace-timestamp encode "2025-01-06 12:30:00" --split month,day
# → month: i5, day: 5, rest: jj3, path: i5/5/jj3

ace-timestamp encode "2025-01-06 12:30:00" --split month,week,day
# → month: i5, week: 1, day: 5, rest: jj3, path: i5/1/5/jj3

ace-timestamp encode "2025-01-06 12:30:00" --split month,week,day,block
# → month: i5, week: 1, day: 5, block: j, rest: j3, path: i5/1/5/j/j3

# Path-only output for shell usage
ace-timestamp encode now --split month,week --path-only
# → i5/1/5jj3

# JSON output for programmatic use
ace-timestamp encode now --split month,week --json
# → {"month":"i5","week":"1","rest":"5jj3","path":"i5/1/5jj3","full":"i515jj3"}

# Decode from path (auto-detect separators)
ace-timestamp decode "i5/1/5/j/j3"
# → 2025-01-06 12:XX:XX UTC
```

**Error Handling:**
- `--split month,block` → Error: "block requires day"
- `--split day,month` → Error: "levels must be in order: month → week → day → block"
- `--split month,week,day,hour` → Error: "unknown level: hour (valid: month, week, day, block)"
- `ace-timestamp encode now --split month,week --format ms` → Error: "--split and --format are mutually exclusive"

## Ruby API

### Encoding with Split
```ruby
Ace::Support::Timestamp.encode_split(time, levels: [:month, :week])
# => { month: "i5", week: "1", rest: "5jj3", path: "i5/1/5jj3", full: "i515jj3" }

Ace::Support::Timestamp.encode_split(time, levels: [:month, :week], path_only: true)
# => "i5/1/5jj3"

Ace::Support::Timestamp.encode_split(time, levels: [:month, :day])
# => { month: "i5", day: "5", rest: "jj3", path: "i5/5/jj3", full: "i55jj3" }
```

### Decoding from Path
```ruby
Ace::Support::Timestamp.decode_path("i5/1/5/j/j3")
# => Time (2025-01-06 12:XX:XX UTC)

Ace::Support::Timestamp.decode_path("i5/5/jj3")
# => Time (2025-01-06 XX:XX:XX UTC)
```

## Clarifications

### Split vs Format
`--split` and `--format` are mutually exclusive. Split creates hierarchical output; format creates flat output of specified length.

### Day is Always Day-of-Month
- `day` = day-of-month (0-30 for days 1-31) in all contexts
- No day-of-week option - simpler mental model
- Week is optional: `month,day` and `month,week,day` both valid

### Default Output
Without `--path-only` or `--json`, `--split` outputs YAML-like key-value pairs:
```
month: i5
week: 1
rest: 5jj3
path: i5/1/5jj3
```

### Rest Component
`rest` contains the remaining timestamp precision after the last split level:
- After `month`: 4 chars (day + block + precision)
- After `month,week`: 4 chars (day + block + precision)
- After `month,day`: 3 chars (block + precision)
- After `month,week,day`: 3 chars (block + precision)
- After `month,week,day,block`: 2 chars (precision only)

### Valid Level Combinations
All combinations follow the hierarchy but allow skipping intermediate levels:
- `month`
- `month,week`
- `month,day`
- `month,week,day`
- `month,week,day,block`
- `month,day,block`

**Note:** `week` is optional between `month` and `day`. `block` always requires `day`.
