# Add Gemini CLI Provider to ace-llm-providers-cli

## Behavioral Specification

### User Experience
- **Input**: Users provide prompts via ace-llm with `gemini` provider alias (e.g., `ace-llm "prompt" -m gemini:gemini-2.5-flash`)
- **Process**: GeminiClient executes `gemini` CLI in headless mode, handles JSON output parsing
- **Output**: Structured response with text content and metadata (tokens, model, timing)

### Expected Behavior

Users should be able to query Google Gemini models through the unified ace-llm interface:
- Provider automatically registers when ace-llm-providers-cli gem loads
- All existing ace-llm options work (temperature, max_tokens, system prompt)
- System prompts are embedded in user prompt since Gemini CLI lacks native `--system-prompt` flag
- JSON output is parsed for structured metadata extraction

### Interface Contract

```bash
# Basic usage through ace-llm
ace-llm "Explain Ruby blocks" -m gemini:gemini-2.5-flash
ace-llm "Summarize this code" -m gemini:gemini-2.5-pro

# With system prompt (embedded in user prompt)
ace-llm "Write a haiku" -m gemini:gemini-2.5-flash --system "You are a poet"

# With temperature control
ace-llm "Creative story" -m gemini:gemini-2.5-flash --temperature 0.9

# Direct gemini CLI command structure (internal)
gemini --prompt "System: You are helpful\n\nUser: Hello" --output-format json --model gemini-2.5-flash
```

**Error Handling:**
- CLI not installed: Clear error message with installation instructions (`npm install -g @google/gemini-cli`)
- Not authenticated: Guide to run `gemini` interactively first for OAuth setup
- API errors: Parse and forward error messages from CLI stderr

**Edge Cases:**
- Empty system prompt: Skip embedding, use prompt directly
- Multi-line prompts: Properly escape for CLI argument passing
- Unicode content: Pass through without modification
- Long prompts: No truncation, let CLI handle limits

## Technical Approach

### Architecture Pattern
- Follow existing CLI provider pattern (CodexClient, ClaudeCodeClient)
- Use subprocess execution via Open3.capture3
- JSON output parsing for structured responses
- System prompt embedding (no native flag support in Gemini CLI)

### Key Implementation Details

**Command Construction:**
```ruby
def build_gemini_command(prompt, options)
  cmd = ["gemini", "--prompt", prompt, "--output-format", "json"]
  cmd << "--model" << @model if @model != DEFAULT_MODEL
  cmd
end
```

**System Prompt Embedding (CodexClient pattern):**
```ruby
def format_messages_as_prompt(messages)
  # ... format messages ...
  if system_content && !prompt.include?("System:")
    "System: #{system_content}\n\nUser: #{prompt}"
  else
    prompt
  end
end
```

**JSON Response Parsing:**
```json
{
  "response": "The AI-generated answer",
  "stats": {
    "models": [...],
    "tokens": {...}
  }
}
```

### Risk Assessment

**Technical Risks:**
- **Risk:** Gemini CLI output format may vary between versions
  - **Probability:** Medium
  - **Impact:** Medium
  - **Mitigation:** Defensive JSON parsing with fallback to raw text
  - **Rollback:** Use raw stdout if JSON parsing fails

- **Risk:** Authentication flow differs from other CLI providers
  - **Probability:** Low
  - **Impact:** Medium
  - **Mitigation:** Clear error messages with setup instructions
