# Create Synthesize Research Workflow

## Behavioral Specification

### User Experience
- **Input**: User invokes `/ace:synthesize-research [research_folder]`
- **Process**: Agent follows 5-phase workflow to combine agent outputs
- **Output**: Unified synthesis with report, comparison matrix, merged artifacts

### Expected Behavior

The workflow guides an agent through combining parallel agent research outputs into unified results. It follows 5 phases: inventory, compare, resolve, synthesize, validate.

### Interface Contract

**Inputs**:
- `research_folder`: Path containing agent outputs (required)
- `output_folder`: Where to write synthesis (default: `{research_folder}/synthesis`)

**Outputs**:
- `synthesis/report.md` - Unified research report
- `synthesis/comparison-matrix.md` - Decisions documented
- `synthesis/artifacts/` - Merged guides, workflows, templates

## Implementation Plan

### Execution Steps

- [ ] Step 1: Create workflow structure
  - Goal section
  - Prerequisites
  - Project context loading
  - High-level execution plan

- [ ] Step 2: Phase 1 - Inventory
  - List all agent outputs
  - Create comparison matrix from template
  - Mark artifact presence and quality

- [ ] Step 3: Phase 2 - Compare
  - Side-by-side analysis by artifact type
  - Note conflicts and disagreements
  - Rate depth, accuracy, actionability

- [ ] Step 4: Phase 3 - Resolve
  - Document conflicts with both positions
  - Research/verify factual disagreements
  - Make decisions with rationale

- [ ] Step 5: Phase 4 - Synthesize
  - Select base (most comprehensive)
  - Merge unique contributions
  - Ensure consistent terminology
  - Credit source reports

- [ ] Step 6: Phase 5 - Validate
  - Completeness check
  - Consistency check
  - Gap identification

- [ ] Step 7: Add embedded templates and examples
  - Synthesis report template
  - Conflict resolution template
  - Example outputs
