# Create Parallel Research Workflow

## Behavioral Specification

### User Experience
- **Input**: User invokes `/ace:parallel-research [topic]`
- **Process**: Agent follows workflow to set up and run parallel research
- **Output**: Research folder with agent outputs ready for synthesis

### Expected Behavior

The workflow guides setting up and running parallel agent research. It covers task formulation, agent selection, consistent prompting, dispatching work, monitoring completion, and orchestrating cross-review before handoff to synthesis.

### Interface Contract

**Inputs**:
- `topic`: Research question or topic
- `agents`: Agent list (default: claude, gemini, codex)
- `output_folder`: Where to store results

**Outputs**:
- Research folder with:
  - `{timestamp}-{agent}-report.md` for each agent
  - `{timestamp}-{agent}-supplementary/` for artifacts
  - Ready for `wfi://synthesize-research`

- [ ] Workflow created at `ace-handbook/handbook/workflow-instructions/parallel-research.wf.md`

## Implementation Plan

### Execution Steps

- [ ] Step 1: Create workflow structure
  - Goal section
  - Prerequisites
  - Project context loading
  - High-level execution plan

- [ ] Step 2: Phase 1 - Setup
  - Define research question clearly
  - Select agents (quality > diversity)
  - Create task context and shared materials
  - Prepare consistent prompt template

- [ ] Step 3: Phase 2 - Dispatch
  - Launch parallel research with identical prompts
  - Assign unique timestamps to each agent
  - Create output folder structure

- [ ] Step 4: Phase 3 - Monitor
  - Track agent completion
  - Handle failures/retries
  - Collect outputs

- [ ] Step 5: Phase 4 - Cross-Review
  - Distribute reports to all agents
  - Each agent reviews peers
  - Collect enhanced reports

- [ ] Step 6: Phase 5 - Handoff
  - Verify all outputs collected
  - Prepare for synthesis phase
  - Link to `wfi://synthesize-research`

- [ ] Step 7: Embed templates
  - Research prompt template
  - Cross-review prompt template
  - Output folder structure
