# Standardize gemspec configurations across all ace-* packages

## Behavioral Context

**Issue**: Following tasks 162 (remove individual Gemfiles) and 163 (add MIT licenses), a healthcheck review revealed inconsistencies across gemspec configurations including missing license declarations, outdated Ruby versions, inconsistent URLs, and varying spec.files patterns.

**Key Behavioral Requirements**:
- All gems should declare MIT license in gemspec (not just have LICENSE file)
- All gems should require Ruby >= 3.3.0 (Ruby 3.2 EOL: March 31, 2026)
- All gems should use consistent homepage URL pattern
- All gems should use deterministic Dir.glob for spec.files

## Objective

Standardize all 24 ace-* gemspec configurations for consistency and future-proofing.

## Scope of Work

- Add `spec.license = "MIT"` to 9 gemspecs missing declaration
- Standardize homepage URLs to `https://github.com/cs3b/ace-meta` (12 gemspecs)
- Update `required_ruby_version` to `>= 3.3.0` (all 24 gemspecs)
- Standardize `spec.files` to `Dir.glob` pattern (13 gemspecs)

### Deliverables

#### Modify
- All 24 `ace-*/ace-*.gemspec` files with standardized configurations

## Implementation Summary

### What Was Done

- **Problem Identification**: Healthcheck audit of all gemspecs revealed inconsistencies
- **Investigation**: Found 9 missing licenses, 12 wrong URLs, varying Ruby versions (2.7-3.2), 4 different spec.files patterns
- **Solution**: Batch updates to all 24 gemspecs following established patterns
- **Validation**: All gem builds succeed, 4794 tests pass (1 flaky perf test unrelated)

### Technical Details

**Phase 1 - License declarations (9 gemspecs):**
- ace-context, ace-docs, ace-git-secrets, ace-lint, ace-llm-models-dev
- ace-prompt, ace-search, ace-support-core, ace-support-test-helpers

**Phase 2 - Homepage URLs (12 gemspecs):**
- Updated template URLs (`your-org/ace-meta`) and wrong org patterns
- Standardized to `https://github.com/cs3b/ace-meta`
- Also fixed `allowed_push_host` and `changelog_uri` patterns

**Phase 3 - Ruby version (24 gemspecs):**
- Upgraded from mixed versions (2.7.0, 3.0.0, 3.1.0, 3.2.0) to `>= 3.3.0`
- Rationale: Ruby 3.2 EOL is March 31, 2026 (3 months away)

**Phase 4 - spec.files pattern (13 gemspecs):**
- Replaced `IO.popen git ls-files` and `Dir.chdir + backticks` patterns
- Standardized to deterministic `Dir.glob` pattern:
```ruby
spec.files = Dir.glob(%w[
  lib/**/*
  exe/*
  .ace-defaults/**/*
  *.md
  LICENSE
  Rakefile
]).select { |f| File.file?(f) }
```
