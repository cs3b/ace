# ace-lint: migrate to standard help pattern

## Scope

Migrate ace-lint from custom start() to standard multi-command help pattern.

**Binary:** `ace-lint`
**CLI module:** `ace-lint/lib/ace/lint/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** lint, doctor
**Current routing:** Custom `start()` with path-aware `known_command?`

## Custom Logic to Preserve

Path-aware routing: args containing `/` or starting with `.` are treated as file paths, not commands (e.g., `ace-lint ./version` → `lint ./version`). With explicit subcommands this is no longer needed, but verify no edge cases remain.

## Code Changes

- [ ] Remove custom `start()` method and `known_command?`
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] No args → show help

## HELP_EXAMPLES (curate)

```ruby
HELP_EXAMPLES = [
  ["Lint current directory", "ace-lint lint"],
  ["Lint specific files", "ace-lint lint src/file.rb"],
  ["Run with autofix", "ace-lint lint --fix"],
  ["Run doctor checks", "ace-lint doctor"],
].freeze
```
