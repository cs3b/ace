# ace-test-e2e: migrate to standard help pattern

## Scope

Migrate ace-test-e2e from custom start() to standard multi-command help pattern.

**Binary:** `ace-test-e2e`
**CLI module:** `ace-test-runner-e2e/lib/ace/test/end_to_end_runner/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** run, suite, setup
**Current routing:** Custom `start()` with extended `known_command?` guards

## Custom Logic to Preserve

Extended `known_command?`: args with `/`, starting with `.`, or starting with `-` are treated as non-commands (routed to default `run`). With explicit subcommands this is mostly unnecessary, but verify edge cases.

## Code Changes

- [ ] Remove custom `start()` method and `known_command?`
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] No args → show help

HELP_EXAMPLES = [
