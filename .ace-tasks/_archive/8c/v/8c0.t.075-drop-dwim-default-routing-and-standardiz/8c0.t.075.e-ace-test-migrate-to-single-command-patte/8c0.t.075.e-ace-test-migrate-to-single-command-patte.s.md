# ace-test: migrate to single-command pattern

## Scope

Migrate ace-test from registry+DefaultRouting to single-command pattern.

**Binary:** `ace-test`
**CLI module:** `ace-test-runner/lib/ace/test_runner/cli.rb`
**Pattern:** single-command
**Current commands:** test (only real command)
**Current routing:** DefaultRouting (clean delete, no custom logic)

## Code Changes

- [ ] Remove `extend Ace::Core::CLI::DryCli::DefaultRouting`
- [ ] Remove `extend Dry::CLI::Registry`
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND`, `BUILTIN_COMMANDS` constants
- [ ] Switch to `Dry::CLI.new(Commands::Test).call` in exe/ace-test
- [ ] Handle `--version` as a command option
- [ ] Remove `start()` method (inherited from DefaultRouting)

## Update References

- [ ] Update any workflow files, guides, docs referencing ace-test CLI patterns
