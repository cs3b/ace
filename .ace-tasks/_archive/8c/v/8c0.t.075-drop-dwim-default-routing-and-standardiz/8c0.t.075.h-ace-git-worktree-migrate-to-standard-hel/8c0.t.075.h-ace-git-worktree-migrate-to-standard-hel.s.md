# ace-git-worktree: migrate to standard help pattern

## Scope

Migrate ace-git-worktree from DefaultRouting to standard multi-command help pattern.

**Binary:** `ace-git-worktree`
**CLI module:** `ace-git-worktree/lib/ace/git/worktree/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** create, list (alias ls), switch (alias cd), remove (alias rm), prune, config
**Current routing:** DefaultRouting — clean delete, no custom logic. Has `COMMAND_ALIASES = %w[ls cd rm]`.

## Code Changes

- [ ] Remove `extend Ace::Core::CLI::DryCli::DefaultRouting`
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND`, `BUILTIN_COMMANDS`, `COMMAND_ALIASES` constants
- [ ] Remove `start()` method
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] Update exe/ wrapper to use `Dry::CLI.new(CLI).call` directly
- [ ] No args → show help
