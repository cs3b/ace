# ace-taskflow: migrate to standard help pattern

## Scope

Migrate ace-taskflow from DefaultRouting+custom override to standard multi-command help pattern.

**Binary:** `ace-taskflow`
**CLI module:** `ace-taskflow/lib/ace/taskflow/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** status (alias context), doctor, config
**Current routing:** DefaultRouting + custom `start()` override for cache clearing

## Custom Logic to Preserve

Cache clearing before dispatch: `Ace::Taskflow::Molecules::TaskLoader.clear_cache!` and `Ace::Taskflow::Molecules::ReleaseResolver.clear_cache!`. Move to exe/ wrapper.

## Code Changes

- [ ] Remove `extend Ace::Core::CLI::DryCli::DefaultRouting`
- [ ] Remove custom `start()` override
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] Move cache clearing to exe/ace-taskflow
- [ ] No args → show help

## HELP_EXAMPLES (curate)

```ruby
HELP_EXAMPLES = [
  ["Show taskflow status", "ace-taskflow status"],
  ["Run health checks", "ace-taskflow doctor"],
  ["Show configuration", "ace-taskflow config"],
  ["Show config as YAML", "ace-taskflow config --format yaml"],
].freeze
```
