# ace-overseer: migrate to standard help pattern

## Scope

Migrate ace-overseer from custom start() to standard multi-command help pattern.

**Binary:** `ace-overseer`
**CLI module:** `ace-overseer/lib/ace/overseer/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** work-on, status, prune
**Current routing:** Custom `start()` — no default command; empty args shows usage

## Custom Logic to Preserve

Empty args already shows usage (via `Dry::CLI::Usage.call`). This aligns with the new pattern (no args → show help), so no special preservation needed — the HelpCommand replaces this behavior.

## Code Changes

- [ ] Remove custom `start()` method
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] No args → show help (replaces custom usage display)
