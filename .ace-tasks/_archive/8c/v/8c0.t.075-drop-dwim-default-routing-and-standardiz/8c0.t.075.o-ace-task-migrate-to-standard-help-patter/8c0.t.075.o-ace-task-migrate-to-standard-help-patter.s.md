# ace-task: migrate to standard help pattern

## Scope

Migrate ace-task (TaskCLI) from DefaultRouting+custom override to standard multi-command help pattern.

**Binary:** `ace-task`
**CLI module:** `ace-taskflow/lib/ace/taskflow/cli/task_cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** list, show, create, start, done, undone, defer, undefer, move, update, add-dependency, remove-dependency
**Current routing:** DefaultRouting + custom `start()` override for cache clearing

## Custom Logic to Preserve

Cache clearing: `TaskLoader.clear_cache!` and `ReleaseResolver.clear_cache!`. Move to exe/ wrapper.

## Code Changes

- [ ] Remove `extend Ace::Core::CLI::DryCli::DefaultRouting`
- [ ] Remove custom `start()` override
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] Move cache clearing to exe/ace-task
- [ ] No args → show help

## HELP_EXAMPLES (curate)

HELP_EXAMPLES = [
