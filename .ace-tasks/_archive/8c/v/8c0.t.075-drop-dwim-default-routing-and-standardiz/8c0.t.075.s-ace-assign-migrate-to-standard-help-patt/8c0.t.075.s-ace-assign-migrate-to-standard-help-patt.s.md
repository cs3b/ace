# ace-assign: migrate to standard help pattern

## Scope

Migrate ace-assign from custom start() to standard multi-command help pattern.

**Binary:** `ace-assign`
**CLI module:** `ace-assign/lib/ace/assign/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** create, status, report, fail, add, retry, list, select, fork-run
**Current routing:** Custom `start()` with exit code capture and deprecated command remapping

## Custom Logic to Preserve

1. **Exit code capture**: `wrap_command()` intercepts integer return values from commands and stores in `@captured_exit_code`. `start()` resets and returns this value. Must be preserved in exe/ wrapper.
2. **Deprecated command remapping**: `start` → `create` with stderr warning. Can likely be dropped or moved to exe/.

## Code Changes

- [ ] Remove custom `start()` DWIM routing logic
- [ ] Keep exit code capture mechanism — move to exe/ace-assign
- [ ] Remove or move deprecated `start` → `create` remapping
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] No args → show help
