# ace-bundle: migrate to standard help pattern

## Scope

Migrate ace-bundle from DefaultRouting to standard multi-command help pattern with HelpCommand.

**Binary:** `ace-bundle`
**CLI module:** `ace-bundle/lib/ace/bundle/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** load, list
**Current routing:** DefaultRouting — default command is `load` (DWIM: `ace-bundle project` → `ace-bundle load project`)

## Code Changes

- [ ] Remove `extend Ace::Core::CLI::DryCli::DefaultRouting`
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND`, `BUILTIN_COMMANDS` constants
- [ ] Remove `start()` method
- [ ] Add `HELP_EXAMPLES` constant with curated examples
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] Update exe/ace-bundle to use `Dry::CLI.new(CLI).call` directly
- [ ] No args → show help (not implicit `load`)

## HELP_EXAMPLES (curate)

```ruby
HELP_EXAMPLES = [
  ["Load project context", "ace-bundle load project"],
  ["Load workflow protocol", "ace-bundle load wfi://bundle"],
  ["List available presets", "ace-bundle list"],
  ["Load from file path", "ace-bundle load path/to/file.md"],
].freeze
```

## Update References

- [ ] Update workflow files, guides, docs — `ace-bundle project` → `ace-bundle load project` everywhere
- [ ] Update CLAUDE.md examples if they reference bare `ace-bundle project`

## Acceptance Criteria

- [ ] `ace-bundle --help` shows formatted help with commands + examples, exits 0
- [ ] `ace-bundle` (no args) shows help
- [ ] `ace-bundle load project` works
- [ ] `ace-bundle project` no longer implicitly routes to `load` (shows help or error)
- [ ] `ace-test` passes in ace-bundle
