# ace-nav: migrate to standard help pattern

## Scope

Migrate ace-nav from custom start() to standard multi-command help pattern.

**Binary:** `ace-nav`
**CLI module:** `ace-support-nav/lib/ace/support/nav/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** resolve, list, create, sources
**Current routing:** Custom `start()` with backward-compat flag routing

## Custom Logic to Preserve

Backward compatibility flags in `start()`:
- `--sources` → routes to `sources` command
- `--create URI [target]` → reconstructs as `create URI [target]` with remaining flags

Decision needed: drop these legacy flags, handle in exe/ wrapper, or register as hidden commands.

## Code Changes

- [ ] Remove custom `start()` method
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] Handle `--sources` and `--create` backward compat (move to exe/ wrapper or drop)
- [ ] No args → show help

## HELP_EXAMPLES (curate)

```ruby
HELP_EXAMPLES = [

  ["List available resources", "ace-nav list"],

].freeze
```
