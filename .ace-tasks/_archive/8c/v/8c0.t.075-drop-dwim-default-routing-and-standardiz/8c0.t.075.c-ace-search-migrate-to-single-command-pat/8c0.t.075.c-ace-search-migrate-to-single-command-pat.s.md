# ace-search: migrate to single-command pattern

## Scope

Migrate ace-search from registry pattern to single-command pattern using `Dry::CLI.new(command).call`.

**Binary:** `ace-search`
**CLI module:** `ace-search/lib/ace/search/cli.rb`
**Pattern:** single-command
**Current commands:** search (only real command)
**Current routing:** Custom `start()` with path-aware `known_command?` — skips args starting with `/` or `.`

## Code Changes

- [x] Remove registry (`extend Dry::CLI::Registry`)
- [x] Remove custom `start()` method and `known_command?`
- [x] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [x] Switch to `Dry::CLI.new(Commands::Search).call` in exe/ace-search
- [x] Handle `--version` as a command option (not registered subcommand)
- [x] Path-like first args (e.g., `ace-search ./version`) no longer need special handling — single-command pattern passes all args to the command directly

## Update References

- [x] Update any workflow files (.wf.md), guides (.g.md), docs/usage.md, README.md referencing ace-search CLI patterns

## Acceptance Criteria

- [x] `ace-search --help` shows dry-cli built-in help, exits 0
- [x] `ace-search "pattern" /path` works (no subcommand needed)
- [x] `ace-search --version` prints version
- [x] `ace-test` passes in ace-search
