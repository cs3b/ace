# ace-tmux: migrate to standard help pattern

## Scope

Migrate ace-tmux from custom start() to standard multi-command help pattern.

**Binary:** `ace-tmux`
**CLI module:** `ace-tmux/lib/ace/tmux/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** start, window, list
**Current routing:** Custom `start()` with context-aware routing based on `ENV["TMUX"]`

## Custom Logic to Preserve

Context-aware default routing:
- Empty args inside tmux → routes to `window`
- Empty args outside tmux → routes to `start`
- Unknown first arg inside tmux → prepends `window`
- Unknown first arg outside tmux → prepends `start`

This DWIM behavior is being dropped. All subcommands become explicit. Move context-dependent logic to exe/ wrapper only if absolutely needed, otherwise drop.

## Code Changes

- [ ] Remove custom `start()` method
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] No args → show help (not context-dependent routing)
- [ ] Drop DWIM context routing — users must type `ace-tmux start` or `ace-tmux window` explicitly

## HELP_EXAMPLES (curate)

```ruby
HELP_EXAMPLES = [
  ["Start tmux session", "ace-tmux start"],
  ["Open new window", "ace-tmux window"],
  ["Open named window", "ace-tmux window my-task"],
  ["List sessions", "ace-tmux list"],
].freeze
```
