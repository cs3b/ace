# ace-review: migrate to standard help pattern

## Scope

Migrate ace-review from custom start() to standard multi-command help pattern.

**Binary:** `ace-review`
**CLI module:** `ace-review/lib/ace/review/cli.rb`
**Pattern:** multi-command (registry)
**Current commands:** review, list-presets, list-prompts
**Current routing:** Custom `start()` with array option preprocessing

## Custom Logic to Preserve

Array option preprocessing: dry-cli only keeps last `--subject`/`--model` occurrence. Custom `start()` accumulates all occurrences:
- `--subject` values joined with ASCII Unit Separator `\x1F` (preserves internal commas like `files:a.rb,b.rb`)
- `--model` values joined with `,`
- Methods: `preprocess_array_options`, `extract_flag_value`, `skip_to_next_arg`

This preprocessing must be preserved — move to exe/ wrapper or a before_callback.

## Code Changes

- [ ] Remove custom `start()` DWIM routing logic
- [ ] Keep array option preprocessing — move to exe/ace-review or dedicated preprocessor
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Add `HELP_EXAMPLES` constant
- [ ] Register HelpCommand for `--help` and `-h`
- [ ] No args → show help
