# Add Provider-Grouped Revocation Report

## Objective

Add a compact `-providers.md` report alongside the existing JSON report that groups unique tokens by provider for easier revocation workflow. Users need to quickly see which tokens to revoke at each provider without wading through duplicate entries.

## Behavioral Specification

### User Experience

- **Input**: User runs `ace-git-secrets scan` as usual
- **Process**: Scan completes and generates reports automatically
- **Output**: Two report files created:
  - `<timestamp>-report.json` (existing behavior)
  - `<timestamp>-providers.md` (new - grouped by provider for revocation)

### Expected Behavior

When a scan finds tokens, an additional markdown report is auto-generated alongside the JSON report. This providers report:
- Groups unique tokens by provider (GitHub, AWS, Anthropic, etc.)
- Shows each unique token only once (deduplicated by value)
- Lists ALL locations (commit:file:line) where each token appears
- Orders providers with revocable ones first, manual-revocation last

### Interface Contract

**Output Format** (`-providers.md`):

```markdown
# Tokens to Revoke

**Scan**: 2025-12-21 00:31:13 | **Unique tokens**: 12 | **Providers**: 3

## GitHub (8 tokens)

### 1. `ghp_1234****wxyz` (github_pat_classic)

**Locations:**
- `abc1234` config.env:12
- `def5678` config.env:12
- `ghi9012` .env:3

### 2. `gho_5678****ABCD` (github_oauth)

**Locations:**
- `jkl3456` secrets.yml:45

## AWS (3 tokens)

### 1. `AKIA****7890` (aws_access_key)

**Locations:**
- `mno7890` deploy.sh:8
- `pqr1234` scripts/setup.sh:22

## Manual Revocation Required (1 token)

### 1. `AIza****xyz1` (google_api_key)

**Locations:**
- `stu5678` app.js:156
```

**Error Handling:**
- No tokens found: Providers report not generated (only clean repos)
- File write error: Error logged, main report still saved
