## Objective

Enforce markdown typography standards from the [Markdown Style Guide](guide://markdown-style) through automated linting. This enables consistent, terminal-friendly documentation by detecting problematic characters like em-dashes and smart quotes before they enter the codebase.

## Scope of Work

### In Scope

- Detect em-dashes (U+2014) and warn to use spaced hyphen
- Detect smart/curly quotes (U+201C, U+201D, U+2018, U+2019) and warn to use straight quotes
- Skip typography checks inside fenced code blocks
- Configurable severity (warn/error/off) via `.ace/lint/markdown.yml`
- Integration with existing MarkdownLinter molecule
- Tests for all typography validation scenarios

### Out of Scope

- Autofix functionality (future enhancement)
- Other markdown style rules beyond typography (line spacing, bullet chars)
- Changes to ace-docs
- Detection in non-markdown files

## Technical Approach

### Architecture Pattern

Typography validation integrates into the existing `MarkdownLinter` molecule:
- Pattern: Add typography checks as extension of `check_markdown_style` method
- Rationale: Typography is a style concern; existing method already handles markdown style checks
- Impact: Minimal - extends existing functionality without architectural changes

### Technology Stack

- **No new dependencies** - Uses Ruby string matching and regex
- **Unicode handling** - Ruby native string methods with UTF-8 encoding
- **Configuration** - Follows ADR-022 cascade pattern (existing infrastructure)

### Implementation Strategy

1. Add typography checks to `MarkdownLinter.check_markdown_style` method
2. Create new config file `markdown.yml` for markdown-specific settings
3. Skip checks within fenced code blocks (track state while iterating lines)
4. Respect configured severity levels from config cascade

## Test Case Planning

### Happy Path Scenarios

| Test Case | Input | Expected |
|-----------|-------|----------|
| Clean markdown | Text with straight quotes and hyphens | No warnings |
| Em-dash detected | "The toolkit—provides" | Warning at correct line |
| Smart quote detected | '"Smart quotes"' | Warning at correct line |
| Mixed issues | Both em-dash and smart quotes | Multiple warnings |

### Edge Case Scenarios

| Test Case | Input | Expected |
|-----------|-------|----------|
| Em-dash in code block | `\`\`\`\n—\n\`\`\`` | No warning (skipped) |
| Smart quote in inline code | `` `"quoted"` `` | No warning (skipped) |
| Multiple em-dashes same line | "A—B—C" | Single or multiple warnings per char |
| Empty file | "" | No warnings |
| File with only code blocks | All fenced code | No warnings |

### Configuration Scenarios

| Test Case | Config | Expected |
|-----------|--------|----------|
| em_dash: error | Error severity | Result.errors contains em-dash issue |
| em_dash: warn | Warning severity | Result.warnings contains em-dash issue |
| em_dash: off | Disabled | No em-dash warnings |
| smart_quotes: off | Disabled | No smart quote warnings |
| Both off | All disabled | No typography warnings |
