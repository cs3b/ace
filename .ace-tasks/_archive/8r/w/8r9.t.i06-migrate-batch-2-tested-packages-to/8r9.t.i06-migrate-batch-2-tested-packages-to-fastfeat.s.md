# Migrate batch 2 tested packages to the fast/feat/e2e testing model

## Objective

Apply the validated `ace-b36ts` pilot taxonomy to the remaining tested packages that were not covered by batch 1, keeping one explicit migration subtask per package.

## Behavioral Specification

### User Experience

- Every package assigned to batch 2 gets an explicit migration task.
- Batch 2 covers the lower-level support and handbook-integration packages omitted from batch 1 because of the one-character child-slot limit in `8r9.t.i05`.
- Package tasks state the current package shape explicitly: fast-only or fast+feat.
- No current batch 2 package owns scenario runtime work, so this batch should not invent `ace-test-e2e <package>` work or `test/e2e/` directories.

### Expected Behavior

1. Run subtask `8r9.t.i06.0` first to refresh all batch 2 child specs from the pilot outputs and current repo state.
2. Execute one batch 2 package migration subtask at a time.
3. Use explicit package tasks for all remaining tested packages.
4. Keep `ace-test <package>` defaulting to `fast` for every batch 2 package.
5. Use `ace-test <package> feat` only for the packages that currently own deterministic `test/integration/` coverage: `ace-support-config`, `ace-support-core`, and `ace-support-markdown`.
6. Keep `ace-test-e2e <package>` out of the batch 2 contract unless a real package-owned scenario surface is discovered during implementation; current repo state says none apply.
7. Hand completion to sibling cleanup task `8r9.t.i07` once all batch 2 package migrations are done.

### Interface Contract

- `ace-test <package>`
- `ace-test <package> feat` for `ace-support-config`, `ace-support-core`, and `ace-support-markdown`
- `ace-test <package> all`

### Validation Questions

- None. Current repo inspection already resolves the package split: fast-only packages are the five handbook integrations plus `ace-hitl`, `ace-support-cli`, `ace-support-fs`, `ace-support-items`, `ace-support-mac-clipboard`, `ace-support-test-helpers`, and `ace-test`; fast+feat packages are `ace-support-config`, `ace-support-core`, and `ace-support-markdown`.

### Success Criteria

- [ ] Every package assigned to batch 2 has one explicit migration subtask.
- [ ] Batch 2 package tasks state whether they are fast-only or fast+feat using the current repo state.
- [ ] The refresh subtask runs before batch 2 migration implementation begins.
- [ ] No batch 2 child task teaches `ace-test <package> e2e`.
- [ ] No batch 2 child task invents `test/e2e/` or `ace-test-e2e <package>` ownership without new package-owned scenario work.
- [ ] Combined with `8r9.t.i05`, all non-pilot tested packages are covered exactly once.
