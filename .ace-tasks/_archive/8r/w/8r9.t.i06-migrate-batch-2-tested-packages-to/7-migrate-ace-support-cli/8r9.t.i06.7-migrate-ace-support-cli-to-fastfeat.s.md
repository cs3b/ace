# Migrate ace-support-cli to the fast/feat/e2e testing model

## Objective

Move ace-support-cli into the new `fast`/`feat`/`e2e` model with a package-specific migration shape that matches its current test surface.

## User Experience

- `ace-test ace-support-cli` becomes the only public deterministic loop for this package.
- The package stays fast-only in this migration because the current repo state contains only deterministic ATOM-layer tests and no `test/integration/` or package-owned scenarios.
- Package docs teach a single fast loop instead of implying an extra feature layer.

## Package Migration Shape

- Fast-only package with current ATOM-style deterministic tests.
- Move ATOM folders into `test/fast/`.
- Do not create `test/feat/` or `test/e2e/` in this migration.

## Expected Behavior

1. Move current atoms/models/molecules/organisms tests into `test/fast/`.
2. Preserve package-level deterministic coverage in the fast loop by default.
3. Update package docs and references to teach the fast-only contract.
4. Do not invent `test/feat/` or `test/e2e/`.

## Interface Contract

- `ace-test ace-support-cli`
- `ace-test ace-support-cli all`

## Validation Questions

- None. Current repo inspection found ATOM-layer tests only, with no `test/integration/`, `test/e2e/`, or scenario surface that requires a separate `feat` or `e2e` command.

## Vertical Slice Decomposition

- This task is one standalone package migration slice for `ace-support-cli`. It ships independently once the package exposes the fast-only contract.

## Verification

- `ace-test ace-support-cli`
- `ace-test ace-support-cli all`
- Verify the migration does not add `test/feat/` or `test/e2e/` for this package.
