# Migrate ace-support-config to the fast/feat/e2e testing model

## Objective

Move ace-support-config into the new `fast`/`feat`/`e2e` model with a package-specific migration shape that matches its current test surface.

## User Experience

- `ace-test ace-support-config` becomes the default fast loop for deterministic package tests.
- `ace-test ace-support-config feat` exposes the deterministic feature coverage that currently lives under `test/integration/`.
- The package remains deterministic-only for this migration and does not gain a scenario runtime layer.

## Package Migration Shape

- Fast + feat package.
- Move unit-style deterministic tests into `test/fast/`.
- Move current `test/integration/` suites into `test/feat/`.
- Keep `test/e2e/` absent in this migration.

## Expected Behavior

1. Move fast-loop deterministic coverage into `test/fast/`.
2. Move current integration-like deterministic coverage into `test/feat/`.
3. Keep the public package contract on `fast`, `feat`, and `all`.
4. Update package docs and test references to match the new layout.

## Interface Contract

- `ace-test ace-support-config`
- `ace-test ace-support-config feat`
- `ace-test ace-support-config all`

## Validation Questions

- None. Current repo inspection confirmed existing `test/integration/` coverage and no package-owned `test/e2e/` or scenario surface.

## Vertical Slice Decomposition

- This task is one standalone package migration slice for `ace-support-config`. It ships independently once the package exposes the required `fast` / `feat` contract.

## Verification

- `ace-test ace-support-config`
- `ace-test ace-support-config feat`
- `ace-test ace-support-config all`
- Verify the migration does not add `test/e2e/` or a package-owned `ace-test-e2e` contract.

## Success Criteria

- [x] Deterministic fast-loop coverage lives under `test/fast/`.
- [x] Existing `test/integration/` suites move to `test/feat/`.
- [x] Package docs and test references stop teaching `integration` or `int` as the public layer name.
