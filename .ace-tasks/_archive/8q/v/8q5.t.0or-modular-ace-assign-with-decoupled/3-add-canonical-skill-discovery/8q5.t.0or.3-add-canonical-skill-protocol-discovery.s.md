# Add canonical skill protocol discovery

## Behavioral Specification

### User Experience
- **Input**: Developers and agents ask nav to list or resolve canonical package skills with `skill://` the same way they already use `wfi://`, `guide://`, or `agent://`.
- **Process**: Nav scans registered canonical package skill sources, applies existing precedence rules, and resolves only canonical package-owned skills rather than projected provider files.
- **Output**: Users can discover every canonical skill across packages without knowing which provider tree currently exposes it.

### Expected Behavior

This slice adds canonical discovery on top of the typed skill platform:
1. `skill://` becomes a first-class nav protocol.
2. Registered gems expose `handbook/skills` sources to nav.
3. `skill://*` lists all canonical package skills regardless of `skill.kind`.
4. `skill://<name>` resolves the canonical skill path for one skill name.
5. Duplicate names follow deterministic precedence aligned with nav's registered-source order.
6. Discovery operates on canonical package skills only, not generated provider projections.

### Interface Contract

```bash
# List all canonical skills
ace-nav skill://*

# Resolve one canonical skill
ace-nav skill://as-b36ts
ace-nav skill://as-task-plan
ace-nav skill://as-assign-start
```

```yaml
# Example canonical skill source registration
name: ace-task
type: gem
description: Canonical skills from ace-task
priority: 10
config:
  relative_path: handbook/skills
  pattern: "*/SKILL.md"
```

**Error Handling:**
- Unknown `skill://` targets return the normal nav not-found behavior with searched-source context.
- Duplicate skill names resolve deterministically and report which canonical source won.
- Gems without `handbook/skills` contribute no entries rather than failing the whole protocol.

**Edge Cases:**
- `skill://*` lists capability, workflow, and orchestration skills together; type filtering is the responsibility of downstream consumers.
- Project-level overrides may shadow gem-level canonical skills through the existing precedence model.
- Partial migrations are allowed; `skill://` lists whatever canonical skills exist now.
