# Plan-first task execution foundation for plan/work workflows and plan CLI

## Behavioral Specification

### User Experience
- **Input**: Maintainer invokes task planning/execution workflows for a selected task.
- **Process**: Agent always derives or reuses a detailed plan before implementation work begins.
- **Output**: A predictable plan-driven workflow and a CLI entry point that resolves fresh plans for work sessions.

### Expected Behavior
The task system should support a deterministic loop: draft -> review -> pending -> plan -> work. Work execution should consume an explicit plan artifact, not informal reasoning from task text alone. The combined result should reduce context drift, improve observability, and keep slices within single-context execution limits.

### Interface Contract
```bash
# Validate orchestrator/subtask structure
mise exec -- ace-task show 8q2.t.hy4 --tree

# Validate review-gate progression
mise exec -- ace-task list --status draft
mise exec -- ace-task list --status pending
```

**Error Handling:**
- Missing subtask spec sections must block promotion to pending.
- Any contradictory workflow directive must be resolved before work starts.

**Edge Cases:**
- Subtasks can finish in any order unless dependencies require sequencing.
- Parent task remains non-executable until all required subtasks are pending.

### Success Criteria
- [ ] Three subtasks exist and each maps to one vertical slice outcome.
- [ ] Each subtask includes explicit verification plan and bundle context.
- [ ] Parent task provides clear integration objective and sequencing.
- [ ] Task set can be promoted to pending without unresolved behavioral questions.

### Validation Questions
- [ ] Should work enforce hard failure when a fresh plan cannot be generated?
- [ ] Should plan freshness include only task + bundle files, or also git state?

### Vertical Slice Decomposition (Task/Subtask Model)
- **Slice Type**: Orchestrator
- **Slice Outcome**: Three independent but compatible changes produce plan-first execution end-to-end.
- **Advisory Size**: medium
- **Context Dependencies**: task plan/work wf files, ace-task CLI and config paths.
