# Add ace-task plan command with freshness reuse and regeneration

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-task plan <ref>`.
- **Process**: Command resolves task, checks latest cached plan freshness, reuses if fresh, otherwise generates a new plan artifact.
- **Output**: Prints selected plan path (or content with `--content`) and exits successfully.

### Expected Behavior
The command must provide a single entry point to get a usable plan for work. It should support strict freshness checks against task/context mtimes and forced regeneration via `--refresh`. Generated plans are stored at `.cache/task/{task-id}/{b36ts}-plan.md`, with `latest-plan.md` pointer updates.

### Interface Contract
```bash
# default resolve/reuse/generate
mise exec -- ace-task plan <ref>

# force regenerate
mise exec -- ace-task plan <ref> --refresh

# print full plan content
mise exec -- ace-task plan <ref> --content

# optional model override
mise exec -- ace-task plan <ref> --model gemini:flash-latest
```

**Error Handling:**
- Unknown task ref -> non-zero with "task not found" guidance.
- Plan generation backend unavailable -> non-zero with actionable remediation.

**Edge Cases:**
- Fresh cached plan exists -> no regeneration.
- Missing latest pointer but timestamped plans exist -> choose newest valid plan.
- Missing tracked context file in plan metadata -> mark stale and regenerate.
