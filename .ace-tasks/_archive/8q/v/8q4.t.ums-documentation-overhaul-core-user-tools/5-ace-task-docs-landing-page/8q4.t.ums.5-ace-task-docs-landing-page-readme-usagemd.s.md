# ace-task docs: landing page README, usage.md, handbook.md

## Objective

Improve ace-task documentation quality beyond the initial batch-generated output, establishing the template pattern for all other ace-* packages.

## Issue

The initial documentation overhaul (8q4.t.ums.2) produced a basic landing page but lacked:
- Value-first sell tagline and user-perspective features
- Interoperability mentions (ace-idea, ace-overseer, ace-git-worktree, ace-assign)
- Agent skills documentation in README
- Full CLI command reference (usage.md)
- Handbook catalog (handbook.md)
- Expanded demo tape covering list/create-with-options/show/archive scenarios

## Solution

Rewrote README as a sell-first landing page and created two new reference docs.

### Files Modified

- `ace-task/README.md` — rewritten as value-first landing page with sell tagline, features, Works With (4 integrations), Agent Skills (lifecycle summary), docs links. No commands in README.
- `ace-task/docs/getting-started.md` — expanded to 6-step tutorial with common commands table and next-steps links
- `ace-task/docs/usage.md` — **created**: full CLI reference for all 7 commands (create, show, list, update, status, plan, doctor) with options tables and examples
- `ace-task/docs/handbook.md` — **created**: catalog of 21 skills, 14 workflow instructions, 3 guides, 10 templates
- `ace-task/docs/demo/ace-task-getting-started.tape` — expanded demo: list, create-with-options, show+tree, archive
- `.ace/demo/tapes/ace-task-getting-started.tape` — synced

### Key Decisions

- README contains NO commands — just links to getting-started.md
- Agent Skills section shows only task-owned skills; cross-domain skills (bugs, ideas, testing, docs, retros) noted separately
- Common Commands table appears in both getting-started.md and usage.md for discoverability
- Handbook.md groups skills by domain with one-sentence descriptions
