# Route public entrypoints through create and drive

## Behavioral Specification

### User Experience
- **Input**: Users create assignments through `/as-assign-create` and execute them through `/as-assign-drive`; if they explicitly ask to start immediately, they use `/as-assign-create ... --run`.
- **Process**: Public docs, workflows, and skill wrappers present create/drive as the single supported creation path, while legacy public `prepare` and `start` entrypoints disappear from the normal user experience.
- **Output**: Users no longer need to choose between prepare, start, and create; they learn one public creation path and one execution path.

### Expected Behavior

This slice makes the public surface coherent once the tracer proves the hidden-spec model:

1. `/as-assign-create` becomes the user-facing workflow for presets, freeform intent, and `--run`.
2. `/as-assign-drive` remains the execution workflow.
3. `/as-assign-prepare` and `/as-assign-start` are removed from the public skill surface and no longer appear in docs, examples, or test fixtures as recommended entrypoints.
4. Advanced direct CLI usage remains valid through `ace-assign create FILE`, but it is documented as a lower-level path rather than the primary UX.
5. `--run` is defined as a workflow-level handoff into drive, not as new natural-language behavior inside the raw CLI.

### Interface Contract

```bash
/as-assign-create "work on task 123 and create a PR"
/as-assign-create "work on task 123 and create a PR" --run
/as-assign-drive
```

**Error Handling:**
- If a user tries the removed public flow described in stale repo docs, the docs/tests must be updated so the repo no longer teaches that path.
- If `--run` is requested and the created assignment has no workable phase, creation still succeeds and reports why drive cannot continue.

**Edge Cases:**
- Direct `ace-assign create FILE` remains supported for hand-authored specs and debugging.
- The public surface does not promise backward-compatibility aliases for removed pre-1.0 skills.

### Success Criteria

- [ ] **Single public creation story**: Repo-facing docs and skills teach create + drive, not prepare/start.
- [ ] **Immediate execution path exists**: `/as-assign-create ... --run` hands off to drive as the last step of create.
- [ ] **Legacy teaching surface removed**: Fixtures, workflow docs, and skill wrappers no longer normalize prepare/start as public entrypoints.
- [ ] **CLI escape hatch preserved**: Low-level file-driven `ace-assign create FILE` remains available and documented as advanced usage.

### Validation Questions

- [x] **Should `--run` live in the workflow or CLI?** -> Workflow/skill layer only.
