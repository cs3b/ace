# Unify ace-assign finish reporting with --message flag

## Behavioral Specification

### User Experience
- **Input**: Report content provided via `--message/-m` flag (inline string or file path) or piped stdin
- **Process**: The command auto-detects whether `--message` value is an existing file path (reads content) or an inline string (uses directly). Falls back to stdin if no flag given.
- **Output**: Phase completed with report content saved; confirmation message printed

### Expected Behavior

Replace the `--report` option on `ace-assign finish` with `--message/-m`, matching the naming pattern already used by `ace-assign fail --message`. The new flag uses smart detection to determine whether the provided value is a file path or an inline string:

- If the value is an existing file path, read and use the file's content
- Otherwise, treat the value as an inline string

This eliminates the need for agents and users to create temporary files for simple reports. The resolution order is: `--message` (file or string) > stdin > error.

No backward compatibility shim is needed for `--report` (ADR-024, pre-1.0).

### Interface Contract

```bash
# --help output:
#   --message=VALUE, -m    Report content: string, file path, or pipe stdin

# Inline string (primary use case for agents)
ace-assign finish --message "## Report\nImplementation complete."
ace-assign finish -m "Done: all tests pass"

# File path (auto-detected when path exists on disk)
ace-assign finish --message report.md
ace-assign finish -m /tmp/detailed-report.md

# Stdin piping (unchanged, lowest priority)
echo "Phase complete" | ace-assign finish

# Error: no input provided
ace-assign finish
# => Error: "Missing report input: provide --message <string|file> or pipe stdin."
```

**Error Handling:**
- No `--message` and no stdin: `"Missing report input: provide --message <string|file> or pipe stdin."`
- `--message` with empty string: Same error as no input
- `--message` file path that doesn't exist: Treated as inline string (not an error)

**Edge Cases:**
- Value that looks like a path but doesn't exist on disk: used as inline string
- Value that is an existing file path: file content is read
- Both `--message` and stdin provided: `--message` takes precedence (matches current `--report` > stdin behavior)

## Objective

Simplify the `ace-assign finish` reporting workflow by replacing the file-only `--report` flag with a smart `--message/-m` flag that accepts inline strings or file paths. This eliminates the overhead of creating temporary report files, making the reporting process more ergonomic for agents and interactive users alike. Aligns `finish` with the `fail --message` pattern for CLI consistency.
