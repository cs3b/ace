# Rename skill prefix from ace- to as- for namespace separation

## Behavioral Context

**Issue**: Non-Claude agents (Codex, Gemini) see skill names like `ace-git-commit` and execute the CLI binary instead of loading the skill instructions. The `ace-` prefix collides with CLI tool executables.

**Key Behavioral Requirements**:
- Skill directories and `name:` fields use `as-*` prefix ("ace skill")
- CLI tool binaries remain `ace-*` (unchanged)
- All slash-command invocations (`/ace-*`) become `/as-*`
- All `skill:` YAML references become `as-*`
- Package names, `source:` fields, `allowed-tools:` entries stay unchanged

## Objective

Rename all 89 skill directories and references from `ace-*` to `as-*` to create clear namespace separation between skills (AI workflows) and CLI tools (deterministic binaries).

- Renamed 89 skill directories in `.agent/skills/`

## Implementation Summary

### What Was Done

- **Critical regex distinction**: Only skill references were changed. CLI tool references (`ace-bundle`, `ace-test`, etc.) were preserved by targeting specific patterns:
  - Leading `/` (slash command): `/ace-task-work` → `/as-task-work`
  - YAML `skill:` key: `skill: ace-task-work` → `skill: as-task-work`
  - SKILL.md `name:` field: `name: ace-task-work` → `name: as-task-work`
  - Skill directory names: `.agent/skills/ace-task-work/` → `.agent/skills/as-task-work/`
