# ace-compressor: compact mode

## Behavioral Specification

### User Experience
- **Input**: One or more Markdown/plain-text sources passed to `ace-compressor compress ... --mode compact`.
- **Process**: The compressor classifies content, compresses narrative and examples aggressively, preserves reasoning-critical rules and facts, and refuses unsafe compaction when required.
- **Output**: Smaller `ContextPack/1` output than exact mode for narrative-heavy sources, with explicit refusal or fallback behavior for safety-sensitive content.

### Expected Behavior
`compact` is the user-facing name for the lossy mode. It prioritizes prompt efficiency while preserving imperative rules, decisions, and critical facts. Narrative prose, repeated examples, and bulky illustrations may collapse into references or summaries. If a source cannot be compacted safely under these rules, the command must fail clearly and recommend rerunning with `--mode exact`.

### Interface Contract
```bash
ace-compressor compress docs/vision.md --mode compact
ace-compressor compress README.md docs/ --mode compact
ace-compressor compress docs/vision.md docs/decisions.md --mode compact --verbose
```

**Operating Modes / Defaults:**
- Standard ACE CLI flags (`--quiet`, `--verbose`, `--debug`, `--help`) are inherited behavior.
- `--verbose` should expose per-source classification and policy decisions.
- `--quiet` should suppress extra explanation but still surface refusal/fallback outcomes.
- `--dry-run` and `--force` are outside this orchestrator's defined compact-mode behavior.

**Error Handling:**
- Safety-sensitive source cannot be compacted without dropping required rules: exit non-zero and recommend `--mode exact`.
- Unsupported binary or non-text input: exit non-zero with the same scope guidance as exact mode.
- Mixed multi-source runs with at least one refusal: emit per-source compact outputs for safe sources, emit `REFUSAL`/`GUIDANCE` metadata for refusing sources, then exit non-zero for the invocation.

**Edge Cases:**
- Mixed corpus with narrative and rule-heavy files may compact some sections but must preserve rule sections exactly.
- Repeated examples collapse to references instead of repeated text.
- Mixed-outcome invocation rule: one refusing source must not suppress compact output already produced for safe sources.

### Success Criteria
- [ ] `compact` produces smaller output than `exact` for narrative-heavy documents.
- [ ] Imperative rules, decisions, and critical facts remain preserved.
- [ ] Fidelity preflight checks gate compact success for imperative rules, numeric facts, and table structure.
- [ ] Unsafe compacting results in explicit refusal, not silent degradation.
- [ ] Per-source policy decisions are explicit for mixed inputs.
- [ ] Hybrid mode remains out of scope for this task.
