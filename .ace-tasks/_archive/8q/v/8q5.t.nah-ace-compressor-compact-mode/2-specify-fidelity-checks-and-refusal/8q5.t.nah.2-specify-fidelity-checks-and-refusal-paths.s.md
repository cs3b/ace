# Specify fidelity checks and refusal paths

## Behavioral Specification

### User Experience
- **Input**: Compact-mode runs on safe and unsafe documentation.
- **Process**: The command validates whether compact output still preserves critical meaning.
- **Output**: Successful compact output when safe, or a refusal message with a recommendation to use `--mode exact`.

### Expected Behavior
Compact mode must have explicit stop conditions. It is not acceptable to produce a smaller pack that silently loses imperative rules, numeric facts, or required table structure. Refusal is a first-class behavior, not an edge case.

### Interface Contract
```bash
ace-compressor compress docs/decisions.md --mode compact
ace-compressor compress docs/decisions.md --mode compact --verbose
ace-compressor compress docs/vision.md docs/decisions.md --mode compact --quiet
```

**Operating Modes / Defaults:**
- Standard ACE CLI flags (`--quiet`, `--verbose`, `--debug`, `--help`) are inherited behavior.
- `--verbose` exposes the fidelity checks that passed/failed per source and why a refusal was triggered.
- `--quiet` suppresses explanatory detail but still emits refusal outcomes and `--mode exact` guidance.
- `--dry-run` and `--force` are not defined by this slice and must not bypass refusal safety.

**Error Handling:**
- Safety validation fails: exit non-zero and print why compact mode was refused.
- Required imperatives dropped during a compact pass: fail validation and refuse output.
- Required numeric facts dropped or altered: fail validation and refuse output.
- Required table structure (schema/key rows) dropped: fail validation and refuse output.
- Refusal output includes actionable guidance to retry with `--mode exact`.

**Edge Cases:**
- A mostly narrative document with one critical policy section succeeds only if that section remains exact.
- A mixed document may compact narrative sections while preserving policy sections exactly.
- Compact output may meet size targets only when fidelity checks pass; size pressure cannot override safety.
- Compact-mode size targets never override safety validation.

### Success Criteria
- [ ] Compact mode validates imperative-rule, numeric-fact, and table-structure preservation before success.
- [ ] Refusal guidance is actionable and points to `--mode exact`.
- [ ] Size targets do not override safety.
- [ ] Mixed narrative/policy sources succeed only when critical policy sections remain exact.
- [ ] `--quiet` output still surfaces refusal outcomes.
- [ ] Failure modes are explicit and testable.

### Validation Questions
- [x] Safety validation is required before compact output is accepted.
- [x] Refusal is the default response to unsafe compacting.
- [x] Mixed narrative/policy docs may still succeed if critical sections stay exact.
- [x] Compression ratio is secondary to fidelity.
