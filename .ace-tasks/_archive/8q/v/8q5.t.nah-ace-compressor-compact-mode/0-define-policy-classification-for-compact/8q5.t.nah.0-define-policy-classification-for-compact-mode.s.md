# Define policy classification for compact mode

## Behavioral Specification

### User Experience
- **Input**: A document corpus with mixed file types such as vision docs, guides, decisions, and workflows.
- **Process**: Compact mode classifies input before deciding what can be reduced aggressively.
- **Output**: A predictable policy choice per source or section that explains why content was compacted, preserved, or refused.

### Expected Behavior
Compact mode must classify before it compacts. Narrative-heavy documents are good compact candidates. Rule-heavy policy and decision documents must either preserve rule-bearing sections exactly or refuse compacting when safety would be compromised.

### Policy Classification Contract
- **narrative-heavy**: aggressive compacting is allowed for prose and repeated examples while preserving key facts.
- **mixed**: compact narrative sections but preserve rule-bearing sections exactly.
- **rule-heavy**: refuse compact mode unless rule-bearing content can be preserved exactly.
- **unknown**: use conservative compact behavior, report fallback class explicitly, and avoid aggressive reductions.

**Classification precedence:**
- Detect rule-bearing sections first (imperative language and decision-oriented policy content).
- Classify as `mixed` when both narrative and rule-bearing sections are present.
- Classify as `unknown` when signals are insufficient or ambiguous.

### Interface Contract
```bash
ace-compressor compress docs/vision.md docs/decisions.md --mode compact
ace-compressor compress docs/vision.md docs/decisions.md --mode compact --verbose
```

**Operating Modes / Defaults:**
- Standard ACE CLI flags (`--quiet`, `--verbose`, `--debug`, `--help`) are inherited behavior.
- `--verbose` exposes per-source classification and selected policy path.
- `--quiet` suppresses explanatory details but still emits refusal/fallback outcomes.
- `--dry-run` and `--force` are not defined by this slice; if introduced later they must not bypass safety refusal rules.

**Error Handling:**
- Unknown file class: use conservative compact behavior and report fallback class as `unknown`.
- Source classified as unsafe for compacting: exit non-zero with guidance to use `--mode exact`.
- Mixed source where rule-bearing sections cannot be preserved exactly: exit non-zero with refusal guidance.
- Mixed multi-source invocation with at least one refusal: keep successful per-source output visible, then exit non-zero for the overall command.

**Edge Cases:**
- Mixed files in one invocation may take different policy paths per source.
- A narrative doc containing a short imperative rule is classified as `mixed` and preserves that rule exactly.
- One unknown source in a mixed invocation does not force refusal for other safe sources.

### Success Criteria
- [ ] File/section classes are behaviorally distinct.
- [ ] Narrative-heavy inputs are compacted more aggressively.
- [ ] Rule-bearing content stays exact or triggers refusal.
- [ ] Unknown class behavior is conservative and explicit.
- [ ] Policy decision is surfaced per source/section with class and chosen action.
- [ ] Verbose/quiet policy reporting behavior is explicit and testable.

### Validation Questions
- [x] Classification happens before aggressive compacting.
- [x] Unknown classes do not silently opt into the most destructive policy.
- [x] Rule-bearing sections keep exact semantics even in compact mode.
- [x] Refusal is preferred over unsafe compacting.
- [x] Mixed inputs are allowed to produce different policy decisions per source.
- [x] Output mode (`verbose`/`quiet`) does not hide refusal outcomes.
