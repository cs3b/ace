# Compact narrative docs end to end

## Behavioral Specification

### User Experience
- **Input**: Narrative-heavy docs such as vision/readme/guides, plus ambiguous text docs that may fall into `unknown`.
- **Process**: The CLI classifies the source, emits runtime policy metadata, applies `aggressive_compact` for `narrative-heavy`, and falls back to `conservative_compact` for `unknown`.
- **Output**: A working narrative compact path with smaller output than exact mode and a safe non-blocking fallback for ambiguous docs.

### Expected Behavior
This slice must run end to end on narrative-heavy docs. It is not enough to label a source as `narrative-heavy`; the output must actually be smaller and visibly compacted, with `POLICY|...|action=aggressive_compact` or equivalent runtime metadata. Ambiguous docs in this slice must not jump to aggressive reduction: they use `class=unknown` with `action=conservative_compact` and remain successful unless a later fidelity check fails.

### Interface Contract
```bash
ace-compressor compress docs/vision.md --mode compact
ace-compressor compress README.md --mode compact --verbose
ace-compressor compress docs/custom-notes.md --mode compact --verbose
```

**Error Handling:**
- Unsupported binary input: exit non-zero.
- Unknown class in this slice: fall back conservatively, not aggressively.

**Edge Cases:**
- Repeated examples may collapse to refs.
- Short imperative lines inside otherwise narrative docs remain explicit.
- `class=unknown` is reported explicitly and may emit a terse note explaining the fallback.

### Success Criteria
- [x] Narrative docs compact materially more than exact mode.
