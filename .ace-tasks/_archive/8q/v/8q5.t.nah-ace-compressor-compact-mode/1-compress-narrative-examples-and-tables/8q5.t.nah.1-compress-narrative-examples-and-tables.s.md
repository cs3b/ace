# Handle mixed-source refusals end to end

## Behavioral Specification

### User Experience
- **Input**: One invocation containing both safe narrative docs and unsafe rule-heavy docs, or one mixed doc that blends narrative and policy sections.
- **Process**: The CLI compacts safe sources, preserves critical rule-bearing sections exactly when possible, refuses unsafe sources with explicit metadata, and exits non-zero only when a source reaches refusal.
- **Output**: A mixed-outcome compact run with compact data, exact-preserved sections where needed, and refusal records when fidelity fails.

### Expected Behavior
This slice delivers the hardest compact-mode product behavior: one command can produce usable output for some sources and a refusal for others, and one mixed source can still succeed if its rule-bearing sections remain exact. Safe-source output must not disappear just because one source refuses.

### Interface Contract
```bash
ace-compressor compress docs/vision.md docs/decisions.md --mode compact --verbose
ace-compressor compress docs/blueprint.md --mode compact --verbose
```

**Error Handling:**
- Any source failing fidelity preflight emits `REFUSAL|...` and `GUIDANCE|retry_with=--mode exact`.
- Overall process exits non-zero when any source refuses.

**Edge Cases:**
- Rule-heavy sections inside mixed docs may keep exact sections rather than full-source refusal if fidelity stays intact.
- Safe-source compact output is emitted before the process exits.
- `class=mixed` may emit `action=compact_with_exact_rule_sections` and `FIDELITY|...|status=pass`.
- `class=rule-heavy` may only proceed via refusal in this slice.

### Success Criteria
- [ ] Mixed-source runs can contain both compact output and refusal metadata.
- [ ] Unsafe sources never compact silently.
- [ ] Overall exit code is non-zero when any source refuses.
- [ ] Safe-source output remains available to the caller.
- [ ] Mixed single-source docs can still succeed when critical policy sections are preserved exactly.

### Validation Questions

- [x] Refusal guidance points to `--mode exact`.
- [x] Per-source divergence is first-class behavior.
- [x] Mixed-source output is part of the CLI contract, not just docs.
