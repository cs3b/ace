# Compress narrative examples and tables

## Behavioral Specification

### User Experience
- **Input**: Narrative-heavy docs, repeated examples, and tables that may be summarized safely.
- **Process**: Compact mode reduces prose and examples aggressively while keeping reasoning-critical table shape or summaries explicit.
- **Output**: Smaller records that still communicate concepts, constraints, and key numeric structure.

### Expected Behavior
Narrative paragraphs may collapse into dense summaries or fact records. Repeated examples may become symbolic references. Tables may be compacted to schema-plus-key-rows or explicit summaries, but they may not disappear silently.

### Interface Contract
```bash
ace-compressor compress README.md docs/vision.md --mode compact
ace-compressor compress README.md docs/ace-gems.g.md --mode compact --verbose
```

**Operating Modes / Defaults:**
- Standard ACE CLI flags (`--quiet`, `--verbose`, `--debug`, `--help`) are inherited behavior.
- `--verbose` exposes per-source narrative/example/table handling decisions and any declared loss records.
- `--quiet` suppresses explanatory detail but still surfaces refusal and loss-signaling outcomes.
- `--dry-run` and `--force` are not defined by this slice and must not bypass table-safety boundaries.

**Error Handling:**
- Table too important to summarize safely: preserve it structurally or refuse compact mode.
- Example required for output mimicry: keep it explicitly instead of collapsing it.
- Table summary attempt drops required schema or key numeric values: exit non-zero and recommend `--mode exact`.

**Edge Cases:**
- Behavioral guard: repeated-example collapse keeps one explicit canonical example with provenance instead of silently dropping all instances.
- Repeated examples across files collapse to one reference plus provenance.
- Large tables retain schema and key values or are explicitly summarized with loss declared.
- Mixed sources may compact narrative sections aggressively while preserving rule-bearing example blocks exactly.
- Multiple tables in one source may use different strategies (`preserve`, `schema-plus-key-rows`, or explicit-loss summary) per table.

### Success Criteria
- [ ] Narrative prose shrinks materially versus exact mode.
- [ ] Examples collapse to references when safe.
- [ ] Tables remain represented explicitly, even when summarized.
- [ ] Loss declarations are explicit wherever structure is reduced.
- [ ] Table strategy choice is explicit per table (`preserve`, `schema-plus-key-rows`, `summarize-with-loss`).
- [ ] Verbose and quiet behavior for narrative/example/table decisions is explicit and testable.
