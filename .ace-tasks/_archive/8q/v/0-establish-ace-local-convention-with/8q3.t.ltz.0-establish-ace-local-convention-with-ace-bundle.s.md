# Establish `.ace-local` convention with ace-bundle proof

## Problem

Need to establish the `.ace-local/` directory convention starting with the foundational pieces: gitignore, core path builder, and ace-bundle as the proof-of-concept consumer.

## Changes Required

### 1. `.gitignore`

Replace `.cache` entry with `.ace-local`.

### 2. `docs/blueprint.md`

Update ignored-paths documentation to reference `.ace-local` instead of `.cache`.

### 3. `ace-support-core/lib/ace/core/molecules/prompt_cache_manager.rb`

Central session cache path builder. Change default from `.cache/{gem_name}` to `.ace-local/{short_name}` where short_name is derived via `gem_name.sub(/^ace-/, "")`.

### 4. `ace-bundle/.ace-defaults/bundle/config.yml`

Update default `cache_dir` value from `.cache/ace-bundle` to `.ace-local/bundle`.

### 5. `ace-bundle/lib/ace/bundle.rb`

Update fallback path from `.cache/ace-bundle` to `.ace-local/bundle`.

### 6. ace-bundle tests

Update any test expectations referencing `.cache/ace-bundle` to `.ace-local/bundle`.

## Verification

- `cd ace-bundle && ace-test` — all tests pass
- `ace-bundle project` output references `.ace-local/bundle/` (not `.cache/ace-bundle/`)
- `.gitignore` covers `.ace-local`
