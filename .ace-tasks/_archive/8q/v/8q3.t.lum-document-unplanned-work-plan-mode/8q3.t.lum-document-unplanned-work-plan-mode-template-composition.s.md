# Document unplanned work: plan-mode template composition for ace-task plan

## Behavioral Context

**Issue**: Planning outputs drifted into execution guidance (permission requests and command handoff) instead of returning a pure plan artifact.

**Key Behavioral Requirements**:
- Keep task planning in strict plan mode.
- Build system prompt composition through ace-bundle config, not ad-hoc string wrapping.
- Repeat plan-mode constraints before and after planning workflow context.

## Objective

Implemented reusable plan-mode prompt composition so `ace-task plan` generates planning-focused system prompts via `ace-bundle` sections and a dedicated template resource.

## Scope of Work

- Added a reusable plan-mode template in `ace-llm` for `tmpl://agent/plan-mode`.
- Registered `ace-llm` under nav tmpl sources for protocol resolution.
- Switched `TaskPlanPromptBuilder` to section-based system config composition:
  - `base: tmpl://agent/plan-mode`
  - `workflow` section with `wfi://task/plan`
  - `project_context` section with `presets: [project]`
  - `repeat_instruction` section with `tmpl://agent/plan-mode`
- Updated prompt-builder tests to validate new composition and ordering.
