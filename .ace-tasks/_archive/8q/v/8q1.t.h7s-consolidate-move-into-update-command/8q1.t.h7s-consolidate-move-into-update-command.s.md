# Consolidate move into update command

## Behavioral Context

**Issue**: The new gems (ace-task, ace-idea, ace-retro) had a `move` command that only relocated items between folders, and an `update` command that only modified frontmatter fields. Routine tasks like "mark done and archive" required two separate commands.

**Key Behavioral Requirements**:
- `update --move-to` replaces standalone `move` across all 3 gems
- `update --move-as-child-of` enables reparenting (ace-task only)
- Auto-archive hook: when all subtasks reach terminal status, parent auto-moves to archive
- "next", "root", "/" all mean "move back to root"

## Objective

Merge `move` functionality into `update` via `--move-to` and `--move-as-child-of`, remove standalone `move` command, add auto-archive hook.

## Scope of Work

- Add `SpecialFolderDetector.move_to_root?` with aliases: next, root, /
- Add `FolderCompletionDetector` atom for terminal-status checking
- Extend `update()` in TaskManager, IdeaManager, RetroManager with `move_to:` param
- Add `--move-to` / `-m` CLI option to all 3 gems' update commands
- Add `TaskReparenter` molecule and `--move-as-child-of` to ace-task
- Remove standalone `move` command and `move()` method from all 3 gems

## Implementation Summary

### Phase 1: SpecialFolderDetector
Added `MOVE_TO_ROOT_ALIASES` constant and `move_to_root?` class method. Case-insensitive matching.

### Phase 2: FolderCompletionDetector
New atom that globs spec files, parses frontmatter, checks if all have terminal status (done/skipped/blocked). Supports `recursive: true` for subtask subdirectories.

### Phase 3: Manager update() extensions
All 3 managers gained `move_to:` param. TaskManager also got auto-archive hook and `move_as_child_of:` param. Field updates happen before move, so the moved file has updated frontmatter.

### Phase 4: CLI --move-to flag
Added to all 3 gems. Validation updated: `--move-to` alone is sufficient. Git commit uses `root_dir` when move is involved (path changed).

### Phase 5: TaskReparenter (ace-task only)
Three operations: "none" (promote subtask to standalone), "self" (convert to orchestrator), "<ref>" (demote to subtask). Handles ID reassignment, frontmatter updates, folder moves.

### Phase 6: Remove standalone move
Deleted command files, removed CLI registrations, removed `move()` from managers, updated all tests.

8,681 tests pass across 33 packages (ace-test-suite). Net change: -541/+387 lines across 23 files + 4 new files.
