# ReviewFinding Model and Runner Interface Contract

## Behavioral Specification

### User Experience
- **Input**: Spike concept inventory defining which models and interfaces survived validation
- **Process**: Formalize the ReviewFinding model and runner interface as production-quality ATOM components
- **Output**: Stable ReviewFinding model in `models/` and runner base class, with full test coverage

### Expected Behavior

Based on the spike's concept inventory, this subtask formalizes:
1. `ReviewFinding` model — immutable value object representing a single finding from any source
2. Runner interface — base class or module defining the `#execute(subject)` contract
3. Relationship between ReviewFinding and existing FeedbackItem (ReviewFinding feeds into FeedbackItem during synthesis, or FeedbackItem gains a `source` field)

The ReviewFinding model captures:
- `source`: identifies the runner and tool (e.g., "llm:google:gemini-2.5-flash", "lint:standardb")
- `file_path`: file where finding was identified
- `line_number`: specific line (nil for general findings)
- `severity`: "error", "warning", "info"
- `message`: human-readable description
- `category`: optional classification ("style", "correctness", "security", "performance")

### Interface Contract

```ruby
# ReviewFinding - immutable value object
finding = Ace::Review::Models::ReviewFinding.new(
  source: "lint:standardb",
  file_path: "lib/foo.rb",
  line_number: 42,
  severity: "warning",
  message: "Unused variable `x`",
  category: "style"
)

finding.source       # => "lint:standardb"
finding.deterministic? # => true (non-LLM source)
finding.to_h         # => { "source" => "lint:standardb", ... }

# Runner interface
class LintRunner < Ace::Review::Runners::Base
  def execute(subject)  # => Array<ReviewFinding>
  end
end
```

**Error Handling:**
- Invalid severity value: raises ArgumentError with valid options
- Missing required fields (source, message): raises ArgumentError

**Edge Cases:**
- Finding without file_path (general observation): allowed, file_path is nil
- Finding without line_number (file-level issue): allowed, line_number is nil

### Success Criteria

- [ ] **ReviewFinding model**: Immutable value object with source, file_path, line_number, severity, message, category
- [ ] **Deterministic detection**: `#deterministic?` method returns true for non-LLM sources
- [ ] **Runner base class**: Defines `#execute(subject)` contract with NotImplementedError default
- [ ] **Serialization**: `#to_h` and `#to_yaml` produce consistent output
- [ ] **Full test coverage**: All model validations and edge cases tested
