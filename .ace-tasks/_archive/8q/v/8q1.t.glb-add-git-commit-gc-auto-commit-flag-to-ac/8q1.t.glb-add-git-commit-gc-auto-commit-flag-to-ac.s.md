# Add --git-commit / --gc auto-commit flag to ace-task, ace-idea, ace-retro

## Description

Mutating CLI commands (create, update, move) in ace-task, ace-idea, and ace-retro required a separate manual `ace-git-commit` invocation to commit changes. This task adds a `--git-commit` / `--gc` flag that auto-commits after a successful operation by shelling out to `ace-git-commit <path> -i "<intention>"`.

## Changes

### New: GitCommitter molecule (ace-support-items)

- `lib/ace/support/items/molecules/git_committer.rb` — shared molecule using `system()` to invoke `ace-git-commit` with paths and intention
- `test/molecules/git_committer_test.rb` — 5 tests covering command construction, failure returns, special characters
- `lib/ace/support/items.rb` — added require_relative

### Modified: 9 command files across 3 packages

Each gets `option :git_commit, type: :boolean, aliases: %w[--gc]` and post-operation commit logic:

**ace-task:** create.rb, update.rb, move.rb
**ace-idea:** create.rb, update.rb, move.rb
**ace-retro:** create.rb, update.rb, move.rb

- `create` and `update` commit the item's `.path` (folder)
- `move` commits `manager.root_dir` (captures both old path deletion + new path creation)
- Dry-run + --gc correctly skips the commit
