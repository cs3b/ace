# ace-compressor: exact mode runnable CLI

## Behavioral Specification

### User Experience
- **Input**: Markdown/text files or directories passed to `ace-compressor compress ... --mode exact`.
- **Process**: The CLI loads sources, parses Markdown structure, preserves rules/numerics/tables, and emits compact structured output instead of raw Markdown.
- **Output**: A runnable `ace-compressor` gem that emits deterministic `ContextPack/2` output for exact mode, with subtask `8q5.t.na8.3` explicitly allowed to supersede that wire format with `ContextPack/3` semantic compression.

### Expected Behavior
This phase must produce the first real, testable `ace-compressor` package. After this parent is done, a developer can install or run the gem locally and use `--mode exact` on single files, directories, and multi-file inputs. Each subtask adds a new runnable capability, not just internal plumbing.
Subtasks `8q5.t.na8.0` through `8q5.t.na8.2` establish the runnable gem through `ContextPack/2`; subtask `8q5.t.na8.3` may upgrade the exact-mode wire format to `ContextPack/3` as a public replacement, not an internal-only refactor.

### Interface Contract
```bash
ace-compressor compress docs/vision.md --mode exact
ace-compressor compress docs/ --mode exact
ace-compressor compress docs/vision.md docs/architecture.md --mode exact
```

**Operating Modes / Defaults:**
- Standard ACE CLI flags (`--quiet`, `--verbose`, `--debug`, `--help`) follow normal dry-cli behavior.
- Callers must choose `--mode exact` explicitly in this phase.
- Output is written to stdio in this phase; file/cache output targets are out of scope.

**Error Handling:**
- Unsupported binary input: exit non-zero with a clear text-only scope error.
- Empty file or empty directory of supported sources: exit non-zero with actionable guidance.
- Image-only chart references: keep command successful and emit unresolved chart records.

**Edge Cases:**
- Repeated explicit file arguments are collapsed to one unique source in output.
- Directory traversal ignores unsupported files and surfaces them only in verbose diagnostics.
- Duplicate paragraphs may deduplicate in representation, but provenance stays auditable.
