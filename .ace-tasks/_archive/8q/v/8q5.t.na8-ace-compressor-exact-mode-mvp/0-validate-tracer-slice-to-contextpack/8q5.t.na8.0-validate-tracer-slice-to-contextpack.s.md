# Bootstrap runnable exact single-file path

## Behavioral Specification

### User Experience
- **Input**: A single Markdown file passed to `ace-compressor compress <file> --mode exact`.
- **Process**: The new gem boots, parses one file, extracts basic structured units, and emits a minimal exact pack.
- **Output**: One runnable end-to-end exact compression path for a single file.

### Expected Behavior
This subtask must create the smallest real slice that proves the package exists and works. It includes gem scaffold, CLI entrypoint, one-file ingest/parse/emit flow, and at least one test proving the command works.

### Interface Contract
```bash
ace-compressor compress docs/vision.md --mode exact
```

**Error Handling:**
- Missing file path: exit non-zero with usage guidance.
- Empty file: exit non-zero and explain that exact mode requires content.

**Edge Cases:**
- Empty frontmatter does not block output.
- A file with only headings still emits a valid exact pack.

### Success Criteria
- [ ] `ace-compressor` command exists and runs.
- [ ] One Markdown file is converted to `ContextPack/1`.
- [ ] The output contains structured records, not raw Markdown.
- [ ] At least one automated test covers the single-file path.
