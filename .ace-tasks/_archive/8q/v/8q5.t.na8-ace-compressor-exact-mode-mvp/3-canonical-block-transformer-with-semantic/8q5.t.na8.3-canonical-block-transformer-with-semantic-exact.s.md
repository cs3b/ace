# Canonical block transformer with semantic exact encoding

## Behavioral Specification

### User Experience
- **Input**: Same files/directories as current exact mode (`ace-compressor compress ... --mode exact`).
- **Process**: Replace verbatim F| record encoding with typed semantic records. Add a normalization layer between `MarkdownParser` and `ContextPack` that transforms each markdown block into compact canonical form.
- **Output**: ContextPack/3 format that preserves all *information content* but is materially smaller than the original in both bytes and tokens. Output should look like a data structure, not a slightly uglier markdown file.

### Expected Behavior -- The 8 transformations

1. **Prose to semantic records**: Paragraphs become typed records (`FACT|`, `RULE|`, `SUMMARY|`, `CONSTRAINT|`) based on content signals, not raw `F|` copies. Imperative modalities (must, never, required) become `RULE|`. Factual statements become `FACT|`. Overview text becomes `SUMMARY|`.

2. **Strip markdown formatting**: Remove `**bold**`, `` `backticks` ``, `[link](url)`, blockquote markers, decorative separators, emoji in headings before encoding. `**Example: \`ace-git-commit\`**` becomes `EXAMPLE|tool=ace-git-commit`.

3. **Lists to arrays**: Bullet lists become typed array records. `- Context bloat\n- No isolation boundary\n- Prompt fragility` becomes `PROBLEMS|[context_bloat,isolation_boundary,prompt_fragility]`. Optionally add detail records per item when explanations are present.

4. **Code fences to typed blocks**: Shell commands become `CMD|`. File trees become `TREE|`. File listings become `FILES|[...]`. Generic code becomes `CODE|lang|...`. No more `B|1|fenced-code|` wrapper.

5. **Drop layout artifacts**: Separator lines (`---`), decorative emoji in headings, numbering prefixes (`### 1.`) -- all dropped. They carry no semantic content.

6. **Implicit scoping**: `FILE|path` sets file context. `SEC|key` sets section context. No source IDs repeated on every record. Replace `S|1|path` + `M|1|1|Title` + `F|1|text` with `FILE|path` + `SEC|title` + `FACT|text`.

7. **Heading normalization**: `### 1. CLI-First, Agent-Agnostic` becomes `SEC|cli_first_agent_agnostic`. Lowercase, underscored, no emoji, no numbering prefix.

8. **ContextPack/3 header**: `H|ContextPack/3|exact` replaces `H|ContextPack/2|exact`.

## Objective
Evolve exact mode from ContextPack/2 format-relabeling to ContextPack/3 semantic compression. The current output keeps raw prose verbatim, adds format overhead (repeated `F|1|` prefixes), preserves markdown syntax, and keeps layout artifacts -- producing output that is actually larger than the original. This subtask inserts a canonical block transformer between parsing and encoding that normalizes each block into compact semantic form, achieving real compression while preserving all information content.
