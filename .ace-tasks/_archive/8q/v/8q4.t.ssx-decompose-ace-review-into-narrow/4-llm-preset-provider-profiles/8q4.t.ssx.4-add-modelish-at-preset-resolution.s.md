# Add preset-qualified model targets in ace-llm

## Behavioral Specification

### User Experience
- **Input**: A maintainer references a model target as `alias`, `provider`, `provider:model`, or any of those with `@preset`, such as `gflash@review-fast` or `claude:sonnet@review-deep`.
- **Process**: `ace-llm` resolves the provider/model target, loads a named execution preset, and merges preset behavior with explicit runtime overrides. `ace-review` passes the same target through unchanged when used in reviewer/provider definitions.
- **Output**: The same model can be reused with named execution profiles across review and non-review workflows without adding ad hoc runtime fields to review configs.

### Expected Behavior

`ace-llm` must own preset-qualified model targeting as a generic runtime feature. The `@preset` suffix must not mean "review preset" or "prompt bundle." It must mean a reusable execution profile that can shape how a provider/model is invoked across any `ace-llm` consumer.

The accepted target forms must remain backward compatible with existing alias and `provider:model` inputs while adding optional `@preset` resolution. Explicit runtime options must still win over preset defaults so callers can override behavior when needed.

`ace-review` must be able to consume preset-qualified targets in provider or reviewer definitions without introducing string-fragile logic. Context-limit lookup, file naming, slugs, and metadata must continue to behave predictably when the model target includes a preset suffix.

### Interface Contract

```bash
# Generic ace-llm target forms
ace-llm query gflash "Summarize this diff"
ace-llm query gflash@review-fast "Summarize this diff"
ace-llm query claude:sonnet@review-deep "Summarize this diff"
ace-llm query claude:sonnet --preset review-deep "Summarize this diff"

# Generic preset config surface
.ace/llm/presets/<name>.yml

# ace-review provider or reviewer references
model: "gflash@review-fast"
model: "claude:sonnet@review-deep"
```

**Error Handling:**
- Missing preset definition: fail with an actionable message naming the missing preset.
- Invalid target form such as `gflash@`: fail before provider execution.
- Duplicate preset selection via both suffix and flag: fail with a clear conflict message.
- Preset-qualified targets must not corrupt context-limit lookup, slug generation, or report naming in `ace-review`.

**Edge Cases:**
- Provider-only inputs such as `google@review-fast` must still resolve to that provider's default model.
- Legacy plain targets without `@preset` must behave exactly as they do today.
- Preset names must stay generic and reusable outside `ace-review`.

### Success Criteria

- [x] `ace-llm` accepts `alias`, `provider`, and `provider:model` targets with an optional `@preset` suffix.
- [x] `ace-llm` exposes a generic preset config surface that is reusable outside `ace-review`.
- [x] Explicit runtime options override preset defaults deterministically.
- [x] `ace-review` can consume preset-qualified targets without breaking feedback generation, context-limit lookup, or output metadata.

### Validation Questions

- [x] **Suffix vs Flag**: The first cut ships `model@preset` and `--preset`; using both together is an explicit error.
- [x] **Preset Scope**: v1 preset files may define runtime invocation settings such as timeout, token/output limits, temperature, CLI args, environment, and system-append text; review semantics stay out of scope.
- [x] **Override Model**: v1 supports provider-specific overrides inside a generic preset because runtime knobs differ across providers.
- [x] **Display Labels**: Metadata retains the full `model@preset` token, while filenames and context-limit lookup use canonical provider/model data plus an optional sanitized preset suffix only where provenance is needed.

## Objective

Introduce a generic `ace-llm` execution-preset layer so reviewer/provider definitions can select reusable provider-runtime behavior with `model@preset` syntax instead of embedding ad hoc runtime knobs inside review configs.

## Scope of Work

- **User Experience Scope**: Maintainers can reference preset-qualified model targets consistently across `ace-llm` and `ace-review`.
- **System Behavior Scope**: Target parsing, preset resolution, override precedence, and downstream review-runtime compatibility.
- **Interface Scope**: `ace-llm` target syntax, `ace-llm` preset config surface, and review provider/reviewer target references.

### Resolved Defaults

- Accepted target forms in v1:
  - `alias`
  - `provider`
  - `provider:model`
  - `alias@preset`
  - `provider@preset`
  - `provider:model@preset`
- CLI support in v1:
  - suffix form: `model@preset`
  - explicit flag form: `--preset review-deep`
  - conflict rule: supplying both is a hard error
- Allowed preset-controlled fields in v1:
  - `timeout`
  - `max_tokens` or equivalent output limit
  - `temperature`
  - `cli_args`
  - `system_append`
  - `subprocess_env`
- Explicitly out of preset scope:
  - reviewer focus
  - review policy
  - correctness/security/simplicity semantics
- Override precedence:
  - explicit runtime args
  - preset values
  - provider defaults
  - global config defaults
- Provenance and naming defaults:
  - preserve full `model@preset` in metadata
  - use canonical provider/model data for context-limit lookup
  - keep filenames and slugs stable by avoiding raw unsanitized `@preset` tokens

### Deliverables

#### Behavioral Specifications
- Generic `ace-llm` target syntax contract with optional preset qualification
- Preset configuration contract for runtime invocation behavior
- Review-runtime compatibility contract for preset-qualified targets

#### Validation Artifacts
- Usage scenarios covering normal, override, and invalid forms
- Verification scenarios covering parsing, resolution, and review compatibility

## Out of Scope

- ❌ Defining review semantics such as correctness, security, or simplicity focus
- ❌ Designing skeptic/referee adjudication
- ❌ Migrating all existing review presets in the same slice
- ❌ Introducing review-only provider config knobs that duplicate generic preset behavior
