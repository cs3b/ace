# Enhance ArgsNormalizer to Shellwords-split array elements

## Behavioral Specification

### User Experience
- **Input**: Array of CLI arg strings, where each element may contain multiple tokens (e.g., `"--sandbox danger-full-access"`)
- **Process**: Each array element is Shellwords-split before normalization
- **Output**: Flat array of individual CLI tokens with auto-prefixed bare words

### Expected Behavior

When `normalize_cli_args` receives an Array, each element is tokenized via `Shellwords.split` before the normalization loop. This allows natural YAML like:
```yaml
codex:
  - "--sandbox danger-full-access"
  - "--ask-for-approval never"
```
to produce `["--sandbox", "danger-full-access", "--ask-for-approval", "never"]`.

Currently, `Array(cli_args)` keeps multi-word elements intact as single strings, causing them to be treated as single tokens in the command array.

### Interface Contract

```ruby
normalizer = ArgsNormalizer.new

# Current behavior (unchanged for strings):
normalizer.normalize_cli_args("--sandbox danger-full-access")
# => ["--sandbox", "danger-full-access"]

# New behavior (array elements now tokenized):
normalizer.normalize_cli_args(["--sandbox danger-full-access", "verbose"])
# => ["--sandbox", "danger-full-access", "--verbose"]

# Single-word array elements (unchanged):
normalizer.normalize_cli_args(["foo", "--bar"])
# => ["--foo", "--bar"]
```

**Error Handling:**
- Empty strings in array: filtered out (existing behavior via `reject(&:empty?)`)
- Malformed shell escaping: `Shellwords.split` raises `ArgumentError` (caught by existing rescue)

**Edge Cases:**
- Array with nil elements: `Array(cli_args).flat_map { |a| Shellwords.split(a.to_s) }` handles nil → `""`
- Single-element array with spaces: `["--model sonnet"]` → `["--model", "sonnet"]`
