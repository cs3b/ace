# Rename `.cache/ace-*` to `.ace-local/{short-name}/`

## Problem

Project-local working data (sessions, reports, simulation runs, prompt workspaces) lives under `.cache/ace-{gem-name}/`. This naming:

- Does not align with `.ace/{short-name}/` config convention
- Poorly communicates intent (`.cache` suggests throwaway data; some content is persistent workspace state)
- Uses full gem names (`ace-review`) instead of short names (`review`)

## Behavioral Scope

All 12 gems using project-relative `.cache/ace-*` paths migrate to `.ace-local/{short-name}/`:

| Before | After |
|--------|-------|
| `.cache/ace-bundle/` | `.ace-local/bundle/` |
| `.cache/ace-review/` | `.ace-local/review/` |
| `.cache/ace-sim/` | `.ace-local/sim/` |
| `.cache/ace-task/` | `.ace-local/task/` |
| `.cache/ace-assign/` | `.ace-local/assign/` |
| `.cache/ace-docs/` | `.ace-local/docs/` |
| `.cache/ace-prompt-prep/` | `.ace-local/prompt-prep/` |
| `.cache/ace-lint/` | `.ace-local/lint/` |
| `.cache/ace-git-secrets/` | `.ace-local/git-secrets/` |
| `.cache/ace-test-runner-e2e/` | `.ace-local/test-runner-e2e/` |
| `.cache/ace-support-nav/` | `.ace-local/support-nav/` |
| `.cache/ace-release/` | `.ace-local/release/` |

Short-name convention: `gem_name.sub(/^ace-/, "")` — same as `.ace/` config dirs.

No backward compatibility shim (per ADR-024). No API changes — only default path values change.

## Out of Scope

- `~/.cache/ace-llm` and `~/.cache/ace-models` (XDG home-based, not project-local)
- Config key name standardization
- Archived task/retro/idea files

## Verification

- `ace-test-suite` passes
- `grep -r '\.cache/ace-' --include='*.rb' --include='*.yml' --include='*.md'` returns zero hits in active code/docs (excluding archives, changelogs, ADRs)
- `ace-bundle project` references `.ace-local/bundle/`
