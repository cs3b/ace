# Add `.ace-local/tmp/` workspace with B36TS time-partitioned keys

## Problem

Agents use system `/tmp/` (`Dir.mktmpdir`, `Tempfile`) for workspace operations. Tmp files vanish on reboot and scatter across system dirs with no traceability. Debugging agent runs requires inspecting intermediate artifacts, which is impossible when they disappear.

## Behavioral Scope

A new `TmpWorkspace` atom in ace-support-core creates project-local tmp directories under `.ace-local/tmp/{ym}/{w}/{b36ts}-{label}/`. Only **workspace session** tmp operations migrate — atomic writes and security-sensitive ops stay as-is.

### Migration categories

| Category | Examples | Migrate? |
|----------|----------|----------|
| **Workspace sessions** | `feedback_synthesizer.rb` → `Dir.mktmpdir("feedback-synthesis")` | Yes |
| **Atomic writes** | `safe_file_writer.rb`, `feedback_file_writer.rb`, `assignment_launcher.rb` — temp in same dir for atomic rename | **No** — must stay co-located for cross-fs rename safety |
| **Security-sensitive** | `git_rewriter.rb` → `/dev/shm` for tokens | **No** — needs memory-backed tmpfs |

### What stays unchanged

- `safe_file_writer.rb` — atomic writes in target dir
- `document_editor.rb` — atomic writes
- `feedback_file_writer.rb` — atomic writes with lock coordination
- `assignment_launcher.rb` — atomic writes
- `gh_comment_poster.rb` — temp file for gh CLI, cleaned up in block
- `git_rewriter.rb` — security-sensitive, needs `/dev/shm`
- `diff.rb` — path validation only, not creating files
- All test helpers — test isolation is fine in system tmp

## Changes Required

### 1. Create `ace-support-core/lib/ace/core/atoms/tmp_workspace.rb`

New atom `Ace::Core::Atoms::TmpWorkspace` with a single class method:

```ruby
def self.create(label, project_root: nil, time: Time.now)
```

- Resolves project root via `ProjectRootFinder.find_or_current` if not given
- Computes date partition via `DatePartitionPath.compute(time)` → `"8p/4"`
- Encodes timestamp via `Ace::B36ts.encode(time)` → `"i50jj3"`
- Creates and returns `.ace-local/tmp/{partition}/{b36ts}-{label}/`

Path example: `.ace-local/tmp/8p/4/i50jj3-feedback-synthesis/`

No block form — directories persist for inspection. Cleanup is by age (manual or future GC task).

### 2. Create `ace-support-core/test/atoms/tmp_workspace_test.rb`

Test that:
- Creates directory under `.ace-local/tmp/` with correct partition structure
- Includes b36ts prefix and label in directory name
- Returns the created directory path
- Accepts custom `project_root:` and `time:` parameters

### 3. Wire require in `ace-support-core/lib/ace/core.rb`

Add require/autoload for `tmp_workspace` following existing atom loading pattern.

### 4. Migrate `ace-review/lib/ace/review/molecules/feedback_synthesizer.rb`

Replace `Dir.mktmpdir("feedback-synthesis")` (line ~437) with:

```ruby
Ace::Core::Atoms::TmpWorkspace.create("feedback-synthesis")
```

### 5. Migrate `ace-git-secrets/lib/ace/git/secrets/atoms/gitleaks_runner.rb`

Replace `Tempfile.create(["gitleaks-report", ".json"])` (line ~123) with TmpWorkspace — create workspace dir, write report file inside it. Reports become inspectable for debugging.
