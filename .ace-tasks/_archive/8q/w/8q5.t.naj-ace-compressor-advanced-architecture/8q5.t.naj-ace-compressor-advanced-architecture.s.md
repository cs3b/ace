# ace-compressor: stable updates and patches

## Behavioral Specification

### User Experience
- **Input**: Exact-mode compression runs on the same file(s) over time, including changed content.
- **Process**: The CLI emits stable semantic IDs alongside exact-mode `ContextPack/3` output, compares changed content to prior exact output, and can emit minimal patch/update artifacts.
- **Output**: A runnable update-friendly exact path that avoids full resend when only a subset changed.

### Expected Behavior
This phase is no longer spec-only. After it is done, users can observe stable semantic IDs in exact output, generate a minimal `PatchPack/1` for single-file changes, and rely on validator/provenance/dedup behavior for update flows. This work extends the current `ContextPack/3` exact path; it does not revert the compressor to older `ContextPack/1` contracts.

### Interface Contract
```bash
ace-compressor compress docs/vision.md --mode exact
ace-compressor patch prev.contextpack new.md --mode exact
ace-compressor compress docs/ --mode exact --verbose
```

**Error Handling:**
- Missing or incompatible prior baseline for patching: exit non-zero with base-compatibility guidance.
- Non-stable ID churn caused by formatting-only changes is a test failure.

**Edge Cases:**
- Whitespace/frontmatter-only edits do not rotate IDs.
- Repeated facts across files deduplicate without losing provenance.

### Success Criteria
- [ ] Exact output exposes stable semantic IDs without breaking current `ContextPack/3` consumers.
- [ ] Single-file delta patches are smaller than full resends and runnable.
- [ ] Dedup/provenance remain explicit in update flows.
- [ ] Validator checks are testable and visible.
