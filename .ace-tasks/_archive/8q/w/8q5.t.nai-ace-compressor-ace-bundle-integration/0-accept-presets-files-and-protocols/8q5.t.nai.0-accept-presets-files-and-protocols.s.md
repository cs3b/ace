# Resolve presets and config inputs

## Behavioral Specification

### User Experience
- **Input**: Preset names, config files, and explicit file paths passed to `ace-compressor compress`.
- **Process**: The CLI resolves inputs using ACE-style auto-detection and sends the resolved sources into compression.
- **Output**: A runnable merged compression flow for presets and config/file inputs.

### Expected Behavior
This slice delivers the first ACE-native integration increment. A user can point `ace-compressor` at `project`, a config file, or an explicit Markdown path and get runnable compression without touching `ace-bundle` manually.

### Interface Contract
```bash
ace-compressor compress project --mode exact
ace-compressor compress ./custom-context.md --mode exact
ace-compressor compress ./custom-context.yml --mode exact
```

**Error Handling:**
- Unknown preset: exit non-zero and name it.
- Missing file/config: exit non-zero and name it.

**Edge Cases:**
- Existing local file path wins over preset-name interpretation.
- Mixed preset + file input resolves in one merged run.

### Success Criteria
- [ ] Preset names resolve directly in `ace-compressor`.
- [ ] File/config inputs resolve directly in `ace-compressor` without altering the existing `ContextPack/3` output shape.
- [ ] Output is runnable in merged mode.
- [ ] Automated tests cover input auto-detection.

### Validation Questions
