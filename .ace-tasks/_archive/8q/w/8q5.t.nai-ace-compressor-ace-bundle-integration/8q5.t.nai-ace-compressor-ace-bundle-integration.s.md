# ace-compressor: ACE-native source inputs

## Behavioral Specification

### User Experience
- **Input**: Preset names, config files, plain files, and protocol URLs passed directly to `ace-compressor`.
- **Process**: The CLI resolves ACE-native inputs and compresses them in `merged` or `per-source` scope.
- **Output**: A runnable integration path that feels native in the ACE ecosystem while preserving the current `ContextPack/3` output contracts.

### Expected Behavior
This phase integrates the already-runnable compressor with ACE input conventions. The user should not need `ace-bundle | ...`. `merged` remains default. `per-source` is explicit. Errors must identify which source failed. This task changes source resolution and routing only; it does not redesign the current `exact|compact|agent` output formats.

### Interface Contract
```bash
ace-compressor compress project --mode exact
ace-compressor compress custom-context.yml --mode exact
ace-compressor compress wfi://task/draft --mode compact --source-scope per-source
```

**Operating Modes / Defaults:**
- `--source-scope merged` is the default.
- `--source-scope per-source` is explicit.
- Input auto-detection follows ACE conventions: existing file path wins over preset interpretation.

**Error Handling:**
- Unknown preset, missing file, or unresolved protocol: exit non-zero and name the failing source.
- Mixed valid/invalid sources: fail overall while preserving any already-emitted safe-source output where the scope allows it.

**Edge Cases:**
- Merged mode still retains per-source provenance.
- Per-source mode preserves resolved-source ordering.

### Success Criteria
- [ ] `ace-compressor` accepts ACE-native source forms directly without changing current `ContextPack/3` output shape.
