# ace-demo README and Getting Started

## Objective

Rewrite ace-demo's README from a 30-line bare command listing into a proper landing page. The current README is the opposite problem from most packages -- it is too terse, showing only raw CLI examples with no explanation of what the tool does, why it matters, or how to get started. Meanwhile docs/usage.md (520 lines) is an exhaustive reference with 14 scenarios, full option tables, and troubleshooting. Create a tutorial-style getting-started.md. Create docs/usage.md (CLI reference) and docs/handbook.md (skill catalog). Update gemspec metadata.

## Behavioral Specification

### User Experience
- **Input**: Current ace-demo/README.md (30 lines), docs/usage.md (520 lines)
- **Process**: Rewrite README as landing page, create getting-started tutorial, write VHS tape, create usage.md + handbook.md, update gemspec, audit handbook
- **Output**: Rewritten README.md (~60 lines), new docs/getting-started.md, new VHS tape + GIF committed to docs/demo/, new docs/usage.md, docs/handbook.md, updated gemspec

### Expected Behavior

New README should convey: "Record terminal demos and attach them to PRs -- visual proof of features with one command."

Key messaging:
- Record deterministic terminal demos from VHS tape scripts
- Attach GIFs directly to GitHub pull requests for visual proof
- Create tapes from shell commands or use built-in presets
- Works for both human developers and AI agents in CI

Current README problems to fix:
- No tagline or explanation of what the tool does
- No "Why" section explaining the value proposition
- Raw CLI examples without context or narrative
- No installation instructions
- No link to deeper documentation
- Missing features overview

### Interface Contract

README.md (~60 lines):
- Tagline: "Record terminal demos and attach them to pull requests"
- Demo GIF (meta: a demo of ace-demo itself)
- Why section (value-first, no commands)
- Works With (integration packages)
- Agent Skills (package-owned only)
- Features: VHS tape recording, PR attachment, inline recording, tape presets
- Documentation links to getting-started.md and --help
- Part of ACE footer

docs/getting-started.md:
- Prerequisites (Ruby, VHS, optional: gh CLI)
- First recording: `ace-demo record hello`
- Listing presets: `ace-demo list`
- Recording and attaching to a PR: `ace-demo record hello --pr 42`
- Creating a custom tape: `ace-demo create my-demo -- "git status"`
- Inline recording: `ace-demo record my-demo -- "echo hello"`
- Common Commands table
- What to try next (custom tape presets, format options, dry-run)

VHS tape: Show `ace-demo list` then `ace-demo record hello` producing a GIF
