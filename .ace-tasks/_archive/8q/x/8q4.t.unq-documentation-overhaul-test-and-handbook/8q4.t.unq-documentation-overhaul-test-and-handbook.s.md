# Documentation Overhaul - Test and Handbook

## Objective

Rewrite documentation for ace-test, ace-test-runner, ace-test-runner-e2e, and ace-handbook to create a user-first landing experience. Transform reference-heavy READMEs into landing pages with hero GIFs, taglines, and quick start guides. Create tutorial-style getting-started.md files. Create docs/usage.md (CLI reference), docs/handbook.md (skill/workflow catalog), and update gemspec metadata. Audit handbook for misplaced skills and dead content.

See retros 8qkop7, 8qku8y, 8qkvc4 for learnings applied here.

Research basis: Diataxis framework, clig.dev, top OSS README patterns (gum, lazygit, httpie, ripgrep).

## Behavioral Specification

### User Experience

- **Input**: Existing package READMEs, --help output, current docs/usage.md files
- **Process**: Rewrite each package README as landing page, create getting-started tutorial, record demo GIF, create usage.md + handbook.md, update gemspec, audit handbook, record demo GIF to docs/demo/
- **Output**: Per package: rewritten README.md, new docs/getting-started.md, docs/usage.md, docs/handbook.md, updated gemspec, docs/demo/*.gif committed to git

### Expected Behavior

When a first-time visitor opens any package README on GitHub, they should:
1. See a clear tagline explaining the tool's value proposition
2. See a hero GIF demo (30s or less) showing the tool's core value
3. Find a "Why" section with value-first framing (no commands)
4. Find feature bullets framed as benefits
5. Find "Works With" showing integration packages
6. Find "Agent Skills" listing package-owned skills only
7. Find documentation links to getting-started, usage, handbook
8. NOT find --help output, commands, full config schemas, or API docs in the README

### Interface Contract

Each package follows the landing page template established by the documentation overhaul initiative:
- README.md: hero GIF, tagline, Why section (value-first, no commands), Works With, Agent Skills, features, documentation links
- docs/getting-started.md: tutorial-style walkthrough with Common Commands table
- docs/demo/*.gif: hero GIF recording committed to docs/demo/ (30s or less, showing core value)
- docs/usage.md: full CLI reference (all flags, subcommands)
- docs/handbook.md: skill/workflow catalog (skills, workflows, guides, prompts)
