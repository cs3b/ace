# ace-search README and Getting Started

## Objective

Rewrite ace-search's README from 271-line reference manual into a landing page. The current README contains full configuration YAML schemas, preset file examples, a lengthy troubleshooting section with 5 subsections, debug output examples, architecture details, dependency lists, development commands, migration notes, and a Claude Code agent integration section -- all of which belong in getting-started.md, --help, or separate docs. Create a tutorial-style getting-started.md.

## Behavioral Specification

### User Experience
- **Input**: Current ace-search/README.md (271 lines)
- **Process**: Rewrite README as landing page, create getting-started tutorial, write VHS tape, create usage.md + handbook.md, update gemspec, audit handbook
- **Output**: Rewritten README.md (~60 lines), new docs/getting-started.md, new VHS tape + GIF committed to docs/demo/, new docs/usage.md, docs/handbook.md, updated gemspec

### Expected Behavior

New README should convey: "Unified codebase search -- one command that auto-detects whether you want files or content."

Key messaging:
- DWIM mode automatically detects file vs content search, so you just search
- Project-wide by default: always searches from project root regardless of current directory
- Dual backend (ripgrep + fd) provides best-in-class speed for both content and file search
- Preset system lets you save and reuse common search patterns

Current README problems to fix:
- Configuration section shows full YAML schema that belongs in --help or getting-started
- Preset file example is tutorial content, not landing page material
- Troubleshooting section (5 subsections, 50+ lines) dominates the README
- Debug output section with multi-line terminal examples is reference material
- Architecture section with ATOM pattern details is developer-facing
- Migration section from dev-tools is historical, not relevant to new users
- Claude Code agent integration section (40+ lines) is a separate concern
