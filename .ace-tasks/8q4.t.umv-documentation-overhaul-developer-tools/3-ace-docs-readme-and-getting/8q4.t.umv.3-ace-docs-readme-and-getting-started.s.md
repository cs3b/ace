### Expected Behavior

New README should convey: "Keep documentation current -- track freshness, detect drift, and get LLM-powered update suggestions."

Key messaging:
- Frontmatter-based tracking means your docs know when they were last updated and what code they cover
- Change analysis uses git history to show exactly what changed since a doc was last touched
- LLM-powered analysis generates actionable update suggestions, not just diffs
- Cross-document consistency checking catches terminology conflicts and version drift

Current README problems to fix:
- Quick Start section is 6 numbered subsections with full command examples (guide-level content)
- Consistency analysis example output block (50 lines) is demo material, not landing page
- ISO 8601 UTC timestamp migration guide (90 lines) is a migration guide embedded in the README
- Configuration YAML schema with document type definitions is reference
- Requirements and development sections are boilerplate
- Common use cases section duplicates getting-started tutorial content
