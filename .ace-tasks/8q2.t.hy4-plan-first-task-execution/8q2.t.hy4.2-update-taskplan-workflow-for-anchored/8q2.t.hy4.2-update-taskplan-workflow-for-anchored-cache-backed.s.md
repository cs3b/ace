# Update task/plan workflow for anchored cache-backed plans

## Behavioral Specification

### User Experience
- **Input**: Agent invokes task planning for a selected task ref.
- **Process**: Workflow loads task + bundle context, creates actionable anchored todo plan, and stores plan artifact in cache.
- **Output**: Fresh plan file at `.cache/task/{task-id}/{b36ts}-plan.md` plus stdout pointer to the generated/reused plan.

### Expected Behavior
The planning workflow must stop being ephemeral-only and define persistent plan artifacts that work sessions can consume directly. Plan output should be structured as an execution checklist with stable step IDs, path:line anchors into the task spec, and per-step verification commands.

### Interface Contract
```bash
# planning workflow source
mise exec -- ace-bundle wfi://task/plan

# expected artifact pattern
.cache/task/<task-id>/<b36ts>-plan.md
.cache/task/<task-id>/latest-plan.md
.cache/task/<task-id>/latest-plan.meta.yml
```

**Error Handling:**
- If task ref cannot be resolved, planning returns explicit error and no cache write.
- If required context file is missing, plan marks stale inputs and reports gap.

**Edge Cases:**
- Existing fresh plan may be reused.
- Any context mtime newer than plan marks plan stale.

### Success Criteria
- [ ] `plan.wf.md` requires persistent cache-backed plan output.
- [ ] Plan template includes step IDs, anchors (`path:line`), dependencies, and verification commands.
- [ ] Workflow defines freshness metadata capture (task file + loaded context files).
- [ ] Workflow metadata (`name`, `description`, `allowed-tools`) and markdown lint issues are fixed.

### Validation Questions
- [ ] Should stale context force regeneration automatically or prompt first?

- **Slice Outcome**: Planning phase yields deterministic, inspectable artifacts that can be reused by work.
- **Advisory Size**: small
- **Context Dependencies**: task planning workflow file, draft/review standards, task config defaults.
