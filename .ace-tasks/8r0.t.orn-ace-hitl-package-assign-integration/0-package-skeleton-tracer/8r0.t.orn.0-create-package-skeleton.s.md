# Create ace-hitl Package Skeleton

## Objective

Create the initial `ace-hitl` package structure in the monorepo so the later HITL store and `ace-assign` integration work lands in a real package rather than being spread across unrelated gems. This is the tracer slice for the new direction.

## Behavioral Specification

### User Experience

- **Input**: A developer adds the new HITL package to the repo.
- **Process**: The package is scaffolded using existing ACE gem conventions, with the executable entry points, gem registration, library root, and baseline docs in place.
- **Output**: The repo has a valid `ace-hitl` package that can be required and invoked, even before the full HITL store behavior is implemented.

### Expected Behavior

1. Add a new top-level package directory `ace-hitl/`.
2. Register `gem 'ace-hitl', path: 'ace-hitl'` in the root `Gemfile`.
3. Add `ace-hitl/exe/ace-hitl` and root wrapper `bin/ace-hitl`.
4. Add package baseline files:
   - `ace-hitl/ace-hitl.gemspec`
   - `ace-hitl/lib/ace/hitl.rb`
   - `ace-hitl/lib/ace/hitl/version.rb`
   - `ace-hitl/lib/ace/hitl/cli.rb`
   - `ace-hitl/README.md`
   - `ace-hitl/CHANGELOG.md`
   - `ace-hitl/Rakefile`
5. The initial CLI may be minimal, but it must establish the v1 command registry shape for `list`, `show`, `create`, and `update`.
6. Package docs must state that `ace-hitl` owns HITL semantics while `ace-support-items` remains generic support infrastructure.

### Interface Contract

Minimum executable contract:

```bash
bin/ace-hitl help
ace-hitl help
```

Minimum library contract:

```ruby
require "ace/hitl"
Ace::Hitl::VERSION
Ace::Hitl::HitlCLI.start(ARGV)
```

### Success Criteria

- [ ] `ace-hitl/` exists with the standard ACE gem skeleton
- [ ] Root and package executables are wired correctly
- [ ] `Gemfile` includes the new local workspace gem
- [ ] The package establishes CLI registration for the intended v1 commands
- [ ] Package docs clearly define the ownership boundary

## Vertical Slice Decomposition

Single tracer slice:
- **Outcome**: New `ace-hitl` package exists and is wired into the monorepo
- **Advisory size**: small
- **Context**: follow `ace-retro` package patterns for CLI registry and wrapper wiring

## Verification Plan

### Validation
- Run the executable wrapper and confirm the CLI starts
- Confirm bundler can resolve the new package from the root `Gemfile`
- Confirm docs and package root say `ace-hitl` owns HITL semantics

### Failure/Invalid Path Validation
- Verify no HITL-specific classes are added to `ace-support-items` as the primary deliverable for this slice
- Verify the package is not created without a root wrapper or Gemfile registration

### Verification Commands
- `ruby bin/ace-hitl --help`
- `cd ace-hitl && ace-test`
