# Orchestrator Multi-Runner Coordination

## Behavioral Specification

### User Experience
- **Input**: ReviewManager receives review options that may include multiple runner types
- **Process**: Orchestrator identifies active runners, executes each, collects all ReviewFinding instances, passes to synthesis
- **Output**: Unified review report with findings from all runners, each labeled by source

### Expected Behavior

ReviewManager gains runner coordination capabilities:
1. Reads `runners:` config from preset to determine which runners are active
2. Instantiates configured runners (LLM runners from existing `reviewers:` config, LintRunner when enabled)
3. Executes all runners, collecting `Array<ReviewFinding>` from each
4. Passes combined findings to synthesis step
5. Synthesis produces report sections that distinguish deterministic vs subjective findings

The existing LLM execution path (MultiModelExecutor) is wrapped as an "LLM runner" that produces ReviewFinding instances, maintaining backward compatibility.

Default behavior (no `runners:` config): only LLM runner is active, identical to today.

### Interface Contract

```ruby
# ReviewManager orchestration flow
manager = Ace::Review::Organisms::ReviewManager.new
result = manager.execute_review(options)
# result[:findings] includes ReviewFinding from all runners
# result[:report] synthesis distinguishes sources

# Runner registry (internal)
manager.runners
# => [#<LlmRunner ...>, #<LintRunner ...>]
```

**Error Handling:**
- A runner raises an unexpected error: Caught, logged as warning, other runners continue
- All runners fail: Review reports error, no findings produced
- Runner timeout: Configurable per-runner timeout, defaults to 60s for lint, existing timeout for LLM

**Edge Cases:**
- Only LLM runner configured (default): Behavior identical to current implementation
- Only lint runner configured: Produces lint-only report (no LLM analysis)
- Runner produces empty findings: Included in report as "no issues found by [source]"

### Success Criteria

- [ ] **Multi-runner execution**: ReviewManager executes all configured runners
- [ ] **Findings aggregation**: All ReviewFinding instances collected into single array
- [ ] **Source-labeled synthesis**: Report clearly shows which runner produced each finding
- [ ] **Fault isolation**: One runner failing doesn't block others
- [ ] **Backward compatible**: No `runners:` config means LLM-only, identical to today
