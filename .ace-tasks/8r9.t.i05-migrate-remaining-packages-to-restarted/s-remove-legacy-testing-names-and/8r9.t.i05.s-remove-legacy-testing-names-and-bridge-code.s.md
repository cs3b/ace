# Remove legacy testing names and bridge code after migration

## Objective

Remove temporary compatibility layers after all package migrations to `fast` / `feat` / `e2e` are complete.

## Expected Behavior

1. Delete legacy public target handling for `unit`, `integration`, and `int` from shared runner surfaces.
2. Remove `test/integration` fallback logic from `ace-test-runner-e2e` once packages are migrated to `test/feat`.
3. Remove package-local bridge configs and leftover compatibility wording.
4. Normalize shared docs, guides, and task references so they teach only `fast`, `feat`, and `e2e`.
5. Keep `ace-test-e2e` as the only scenario entrypoint and keep `ace-test <pkg> all` as `fast` then `feat`.

## Success Criteria

- [ ] Shared runner code no longer exposes `unit`, `integration`, or `int` as public targets.
- [ ] `ace-test-runner-e2e` no longer falls back to `test/integration`.
- [ ] No migrated package still depends on compatibility names or bridge config.
- [ ] Public docs teach only `fast`, `feat`, and `e2e`.
