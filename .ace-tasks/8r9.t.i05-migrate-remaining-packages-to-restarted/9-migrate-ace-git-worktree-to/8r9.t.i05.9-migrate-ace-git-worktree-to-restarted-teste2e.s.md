# Migrate ace-git-worktree to restarted test/e2e and scenario model

## Objective

Migrate ace-git-worktree to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-WORKTREE-001-basic-lifecycle: TC-001-help-survey, TC-002-list-and-create, TC-003-switch-and-formats, TC-004-dry-run-ops, TC-005-remove-worktree, TC-006-prune-orphaned
- TS-WORKTREE-002-task-aware: TC-001-help-survey, TC-002-create-task-worktree, TC-003-list-and-filter, TC-004-switch-by-task, TC-005-multi-task, TC-006-json-metadata, TC-007-remove-and-cleanup
2. Use these destination file(s):
- ace-git-worktree/test/e2e/ts_worktree_001_test.rb
- ace-git-worktree/test/e2e/ts_worktree_002_test.rb
3. Scenario decision:
- Keep one real task-worktree management scenario.
- Scenario should ask the agent to create, inspect, and clean up a worktree-backed task flow from docs and help, with verification from final response plus git and worktree state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.

## Verification Commands

- ace-test ace-git-worktree e2e
- ace-test-e2e ace-git-worktree
