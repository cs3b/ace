# Migrate ace-git-commit to the fast/feat/e2e testing model

## Objective

Move ace-git-commit into the new `fast`/`feat`/`e2e` model and create explicit deterministic `feat` coverage where the package currently relies on old flat tests or scenario-owned smoke behavior.

## User Experience

- `ace-test ace-git-commit` becomes the default fast feedback loop for deterministic package tests.
- `ace-test ace-git-commit feat` exposes deterministic feature coverage that is no longer hidden inside scenario-only flows.
- `ace-test-e2e ace-git-commit` remains the only operator command for real workflow scenarios.

## Expected Behavior

1. Move existing unit-style package tests into `test/fast/`, preserving ATOM subfolders where they already exist.
2. Create `test/feat/` for deterministic feature coverage that should not remain hidden inside `test/e2e/` scenarios.
3. Keep only real workflow scenarios under `test/e2e/`.
4. Update package-local test config, docs, demo assets, and references so the public contract is only `fast`, `feat`, and `e2e`.
5. Do not reintroduce deterministic `*_test.rb` files under `test/e2e/`.

## Package Migration Shape

- Existing ATOM or flat package tests move into `test/fast/`.
- Deterministic smoke coverage that was previously implied by scenario runtime should become explicit `test/feat/` coverage.
- `test/e2e/` remains only for agent-runtime scenarios that still justify the full workflow cost.

## Interface Contract

- `ace-test ace-git-commit`
- `ace-test ace-git-commit feat`
- `ace-test ace-git-commit all`
- `ace-test-e2e ace-git-commit`

## Validation Questions

- None. The package currently lacks `test/integration/`, so this task explicitly creates deterministic `feat` coverage from flat tests or scenario-owned smoke checks.

## Vertical Slice Decomposition

- This task is one standalone package migration slice for `ace-git-commit`. It ships independently once the package exposes the required `fast` / `feat` / `e2e` contract.

## Verification

- `ace-test ace-git-commit`
- `ace-test ace-git-commit feat`
- `ace-test ace-git-commit all`
- `ace-test-e2e ace-git-commit`

## Success Criteria

- [ ] ace-git-commit exposes the default deterministic loop through `test/fast/`.
- [ ] ace-git-commit has an explicit `test/feat/` layer for deterministic feature coverage.
- [ ] `ace-test ace-git-commit feat` validates the new feature layer.
- [ ] `ace-test-e2e ace-git-commit` runs only real workflow scenarios.
- [ ] Package-local docs and config stop teaching `unit`, `integration`, `int`, or deterministic `test/e2e` test files.
