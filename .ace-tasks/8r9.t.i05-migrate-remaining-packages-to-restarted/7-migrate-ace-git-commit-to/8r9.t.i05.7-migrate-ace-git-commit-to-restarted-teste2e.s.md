# Migrate ace-git-commit to restarted test/e2e and scenario model

## Objective

Migrate ace-git-commit to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-COMMIT-001-commit-workflow: TC-001-help-survey, TC-002-basic-commit, TC-003-dry-run-and-paths, TC-004-delete-and-rename, TC-005-auto-split, TC-006-no-split
2. Use these destination file(s):
- ace-git-commit/test/e2e/ts_commit_001_test.rb
3. Scenario decision:
- Keep one real scoped-commit scenario.
- Scenario should ask the agent to complete a real commit job from docs and help, with verification from final response plus git history and state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.

- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
