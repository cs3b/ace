# Migrate ace-retro to restarted test/e2e and scenario model

## Objective

Migrate ace-retro to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-RETRO-001-retro-cli-smoke: TC-001-help-and-usage, TC-002-create-list-show-lifecycle, TC-003-folder-and-filter-views, TC-004-doctor-health-to-failure
2. Use these destination file(s):
- ace-retro/test/e2e/ts_retro_001_test.rb
3. Scenario decision:
- Keep one real retro-creation or synthesis scenario.
- Scenario should ask the agent to create or synthesize a retro from docs and help and verify final response plus retro state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.
