# Migrate ace-assign to restarted test/e2e and scenario model

## Objective

Migrate ace-assign to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-ASSIGN-001-core-workflow: TC-001-help-survey, TC-002-workflow-lifecycle, TC-003-fork-context, TC-004-prepare-workflow, TC-005-no-skip-policy
- TS-ASSIGN-002-hierarchy: TC-001-injection-renumbering, TC-002-auto-completion, TC-003-hierarchy-errors, TC-004-display-audit, TC-005-fork-subtree
2. Use these destination file(s):
- ace-assign/test/e2e/ts_assign_001_test.rb
- ace-assign/test/e2e/ts_assign_002_test.rb
3. Scenario decision:
- Keep one real assignment workflow scenario.
- Give the agent docs and --help, then one end-to-end assignment-management job.
- Verify from final response plus resulting assignment, git, and filesystem state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.

## Verification Commands

- ace-test ace-assign e2e
- ace-test-e2e ace-assign
