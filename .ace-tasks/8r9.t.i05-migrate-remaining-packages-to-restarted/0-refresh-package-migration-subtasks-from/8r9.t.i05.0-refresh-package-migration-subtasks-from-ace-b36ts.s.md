# Refresh package migration subtasks from ace-b36ts pilot outputs

## Objective

Apply the final ace-b36ts pilot truth to every child migration subtask before package work starts so all package tasks use the same helper API, target names, scenario contract, and documentation model.

## Expected Behavior

1. Re-read the completed pilot outputs.
2. Update every child task so its commands, helper references, scenario contract, and deterministic destination files match the pilot.
3. Lock the following as the only valid restart contract for all children:
   - deterministic tests in test/e2e
   - ace-test <pkg> e2e
   - ace-test-suite --target e2e
   - scenario-only ace-test-e2e
   - final-response-only scenario-owned text output
4. Record any package-specific deviations discovered during the pilot and push them into the child specs before implementation begins.

## Success Criteria

- [ ] Every package child task matches the pilot-owned interface contract.
- [ ] No child task refers to test-e2e/integration.
- [ ] No child task refers to deterministic phase execution inside ace-test-e2e.
- [ ] Any helper API or scenario-contract changes from the pilot are reflected in every child.
