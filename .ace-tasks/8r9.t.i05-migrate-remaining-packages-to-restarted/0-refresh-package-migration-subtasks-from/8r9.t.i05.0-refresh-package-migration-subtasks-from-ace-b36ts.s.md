# Refresh package migration subtasks from ace-b36ts pilot outputs

## Objective

Apply the final `ace-b36ts` pilot truth to every child migration subtask before package work starts so all package tasks use the same public contract, directory layout, and verification commands.

## Expected Behavior

1. Re-read the completed pilot outputs.
2. Update every child task so its commands, helper references, scenario contract, and deterministic destination files match the pilot.
3. Lock the following as the only valid migration contract for all children:
   - deterministic tests in `test/fast/`
   - deterministic feature tests in `test/feat/`
   - scenario workflows in `test/e2e/`
   - `ace-test <pkg>` defaulting to `fast`
   - `ace-test <pkg> feat`
   - `ace-test-e2e <pkg>`
4. Record any package-specific deviations discovered during the pilot and push them into the child specs before implementation begins.

## Success Criteria

- [ ] Every package child task matches the pilot-owned `fast` / `feat` / `e2e` interface contract.
- [ ] No child task refers to `test-e2e/integration` or `test-e2e/scenarios`.
- [ ] No child task refers to deterministic phase execution inside `ace-test-e2e`.
- [ ] No child task teaches `ace-test <pkg> e2e`.
- [ ] Any helper API or scenario-contract changes from the pilot are reflected in every child.
