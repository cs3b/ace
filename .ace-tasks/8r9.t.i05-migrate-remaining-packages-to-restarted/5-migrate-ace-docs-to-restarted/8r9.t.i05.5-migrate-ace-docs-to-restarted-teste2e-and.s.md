# Migrate ace-docs to the fast/feat/e2e testing model

## Objective

Move ace-docs from the legacy `unit`/`integration` naming into the new `fast`/`feat`/`e2e` model without losing deterministic coverage or scenario validation.

## Expected Behavior

1. Move existing unit-style package tests into `test/fast/`, preserving ATOM subfolders where they already exist.
2. Move existing deterministic `test/integration/` coverage into `test/feat/` and rename files to reflect feature contracts rather than legacy layer names.
3. Keep or simplify `test/e2e/` scenarios so they validate only real workflow behavior that still needs agent runtime.
4. Update package-local test config, docs, demo assets, and references so the public contract is only `fast`, `feat`, and `e2e`.
5. Remove any deterministic `test/e2e/*_test.rb` destination assumptions from the package.

## Package Migration Shape

- Existing deterministic coverage already exists and should migrate from `test/integration/` to `test/feat/`.
- Existing ATOM or flat package tests should move under `test/fast/`.
- Scenario assets stay under `test/e2e/` only when they still validate real workflows.

## Verification

- `ace-test ace-docs`
- `ace-test ace-docs feat`
- `ace-test ace-docs all`
- `ace-test-e2e ace-docs`

## Success Criteria

- [ ] ace-docs exposes deterministic tests only through `test/fast/` and `test/feat/`.
- [ ] `ace-test ace-docs` defaults to the `fast` loop.
- [ ] `ace-test ace-docs feat` runs the former integration coverage from `test/feat/`.
- [ ] Any remaining scenario coverage stays under `test/e2e/` and is executed only with `ace-test-e2e`.
- [ ] Package-local docs and config stop teaching `unit`, `integration`, `int`, or deterministic `test/e2e` test files.
