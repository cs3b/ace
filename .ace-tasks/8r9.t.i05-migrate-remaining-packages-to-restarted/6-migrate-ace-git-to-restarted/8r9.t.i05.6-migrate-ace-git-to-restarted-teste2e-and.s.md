# Migrate ace-git to restarted test/e2e and scenario model

## Objective

Migrate ace-git to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-GIT-001-git-operations: TC-001-git-status, TC-002-git-diff, TC-003-branch-info, TC-004-pr-summary, TC-005-diff-output-path-security, TC-006-status-json-no-pr
2. Use these destination file(s):
- ace-git/test/e2e/ts_git_001_test.rb
3. Scenario decision:
- Keep one real repo-inspection scenario.
- Scenario should ask the agent to inspect and summarize a repo task using package docs and help, with verification from final response plus repo state and output.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.
