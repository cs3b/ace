# Migrate ace-search to the fast/feat/e2e testing model

## Objective

Move ace-search from the legacy `unit`/`integration` naming into the new `fast`/`feat`/`e2e` model without losing deterministic coverage or scenario validation.

## User Experience

- `ace-test ace-search` becomes the default fast feedback loop for deterministic package tests.
- `ace-test ace-search feat` runs the deterministic feature layer that replaces legacy integration naming.
- `ace-test-e2e ace-search` remains the only operator command for real workflow scenarios.

## Expected Behavior

1. Move existing unit-style package tests into `test/fast/`, preserving ATOM subfolders where they already exist.
2. Move existing deterministic `test/integration/` coverage into `test/feat/` and rename files to reflect feature contracts rather than legacy layer names.
3. Keep or simplify `test/e2e/` scenarios so they validate only real workflow behavior that still needs agent runtime.
4. Update package-local test config, docs, demo assets, and references so the public contract is only `fast`, `feat`, and `e2e`.
5. Remove any deterministic `test/e2e/*_test.rb` destination assumptions from the package.

## Package Migration Shape

- Existing deterministic coverage already exists and should migrate from `test/integration/` to `test/feat/`.
- Existing ATOM or flat package tests should move under `test/fast/`.
- Scenario assets stay under `test/e2e/` only when they still validate real workflows.

## Interface Contract

- `ace-test ace-search`
- `ace-test ace-search feat`
- `ace-test ace-search all`
- `ace-test-e2e ace-search`

## Validation Questions

- None. The package already has deterministic `test/integration/` coverage, so this task only remaps that layer into `test/feat/` and aligns the public commands.

## Vertical Slice Decomposition

- This task is one standalone package migration slice for `ace-search`. It ships independently once the package exposes the required `fast` / `feat` / `e2e` contract.

## Verification

- `ace-test ace-search`
- `ace-test ace-search feat`
- `ace-test ace-search all`
- `ace-test-e2e ace-search`

## Success Criteria

- [ ] ace-search exposes deterministic tests only through `test/fast/` and `test/feat/`.
- [ ] `ace-test ace-search` defaults to the `fast` loop.
- [ ] `ace-test ace-search feat` runs the former integration coverage from `test/feat/`.
- [ ] Any remaining scenario coverage stays under `test/e2e/` and is executed only with `ace-test-e2e`.
- [ ] Package-local docs and config stop teaching `unit`, `integration`, `int`, or deterministic `test/e2e` test files.
