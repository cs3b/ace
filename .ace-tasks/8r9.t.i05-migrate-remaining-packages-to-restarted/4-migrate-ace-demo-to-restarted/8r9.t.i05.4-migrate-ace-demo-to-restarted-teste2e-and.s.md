# Migrate ace-demo to restarted test/e2e and scenario model

## Objective

Migrate ace-demo to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-DEMO-001-cli-smoke: TC-001-help-and-command-surface, TC-002-create-and-show-tape-lifecycle, TC-003-record-inline-dry-run-preview, TC-004-attach-pr-validation-error
2. Use these destination file(s):
- ace-demo/test/e2e/ts_demo_001_test.rb
3. Scenario decision:
- Keep one real demo-recording scenario.
- Scenario should ask the agent to record or prepare a meaningful terminal demonstration from available docs and help and validate the final artifact plus final response.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.
