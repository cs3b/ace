# Migrate ace-task to restarted test/e2e and scenario model

## Objective

Migrate ace-task to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-TASK-001-core-cli-smoke: TC-001-help-survey, TC-002-create-show-list, TC-003-update-and-archive, TC-004-doctor-health
2. Use these destination file(s):
- ace-task/test/e2e/ts_task_001_test.rb
3. Scenario decision:
- Keep one real task-lifecycle scenario.
- Scenario should ask the agent to complete a real task-management job from docs and help and verify final response plus task state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.
