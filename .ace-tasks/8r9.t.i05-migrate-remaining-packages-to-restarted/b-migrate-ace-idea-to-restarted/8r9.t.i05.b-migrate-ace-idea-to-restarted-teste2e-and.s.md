# Migrate ace-idea to restarted test/e2e and scenario model

## Objective

Migrate ace-idea to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-IDEA-001-idea-lifecycle: TC-001-create-idea, TC-002-list-ideas, TC-003-move-idea, TC-004-archive-idea
2. Use these destination file(s):
- ace-idea/test/e2e/ts_idea_001_test.rb
3. Scenario decision:
- Keep one real idea-capture and refinement scenario.
- Scenario should ask the agent to turn raw input into an idea artifact using docs and help and verify final response plus idea state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.

## Verification Commands

- ace-test ace-idea e2e
- ace-test-e2e ace-idea
