# Migrate ace-tmux to restarted test/e2e and scenario model

## Objective

Migrate ace-tmux to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-TMUX-001-tmux-management: TC-001-list-presets, TC-002-start-session, TC-003-add-window
2. Use these destination file(s):
- ace-tmux/test/e2e/ts_tmux_001_test.rb
3. Scenario decision:
- Keep one real tmux session and window scenario.
- Scenario should ask the agent to complete a real tmux management task from docs and help and verify final response plus tmux and state outputs.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.

## Verification Commands

- ace-test ace-tmux e2e
- ace-test-e2e ace-tmux
