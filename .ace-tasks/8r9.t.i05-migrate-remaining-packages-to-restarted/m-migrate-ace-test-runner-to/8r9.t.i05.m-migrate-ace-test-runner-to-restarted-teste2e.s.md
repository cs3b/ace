# Migrate ace-test-runner to restarted test/e2e and scenario model

## Objective

Migrate ace-test-runner to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-TEST-001-test-execution: TC-001-run-package-tests, TC-002-run-specific-file, TC-003-run-test-group
- TS-TEST-002-suite-execution: TC-001-run-full-suite, TC-002-verify-failure-propagation
2. Use these destination file(s):
- ace-test-runner/test/e2e/ts_test_001_test.rb
- ace-test-runner/test/e2e/ts_test_002_test.rb
3. Scenario decision:
- Keep one real focused test-running scenario.
- Scenario should ask the agent to diagnose or run a realistic test job from docs and help and verify final response plus reports and output state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.

## Verification Commands

- ace-test ace-test-runner e2e
- ace-test-e2e ace-test-runner
