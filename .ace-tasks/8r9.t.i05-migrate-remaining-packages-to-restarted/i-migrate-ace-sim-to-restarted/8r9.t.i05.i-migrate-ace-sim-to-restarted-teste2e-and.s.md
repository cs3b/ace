# Migrate ace-sim to restarted test/e2e and scenario model

## Objective

Migrate ace-sim to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-SIM-001-next-phase-smoke: TC-001-help-survey, TC-002-preset-contract, TC-003-run-chain-artifacts, TC-004-full-chain-synthesis, TC-005-validate-task-preset, TC-006-synthesis-provider-guard
2. Use these destination file(s):
- ace-sim/test/e2e/ts_sim_001_test.rb
3. Scenario decision:
- Keep one real simulation scenario.
- Scenario should ask the agent to complete a realistic simulation workflow from docs and help and verify final response plus produced state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.

- ace-test ace-sim e2e
- ace-test-e2e ace-sim
