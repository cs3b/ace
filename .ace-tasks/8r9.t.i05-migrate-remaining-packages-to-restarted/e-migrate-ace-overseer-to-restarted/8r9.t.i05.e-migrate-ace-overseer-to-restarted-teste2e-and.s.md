# Migrate ace-overseer to restarted test/e2e and scenario model

## Objective

Migrate ace-overseer to the restarted testing structure by moving deterministic sandboxed coverage into ordinary Minitest under test/e2e and applying the pilot scenario decision for this package.

## Expected Behavior

1. Migrate the following deterministic source coverage into test/e2e:
- TS-OVERSEER-001-overseer-workflow: TC-001-help-survey, TC-002-work-on, TC-003-idempotent-rerun, TC-004-preset-override, TC-005-prune-workflow
2. Use these destination file(s):
- ace-overseer/test/e2e/ts_overseer_001_test.rb
3. Scenario decision:
- Keep one real work-on and prune scenario.
- Scenario should ask the agent to use overseer for a real task-management job from docs and help and verify final response plus worktree and task state.
4. Remove package-local dependence on test-e2e/integration and on deterministic execution through ace-test-e2e.

## Success Criteria

- [ ] Every listed deterministic TC is re-expressed in test/e2e as explicit test methods.
- [ ] The destination file layout matches this task specification.
- [ ] Package scenarios, when kept, follow the final-response-only contract from the pilot.
- [ ] Package docs and local testing references no longer point to the old deterministic layout.
