# Migrate remaining packages to the fast/feat/e2e testing model

## Objective

Apply the validated `ace-b36ts` pilot taxonomy to all remaining packages through one package-specific migration subtask per package, plus one final cleanup subtask that removes temporary compatibility bridges after the migration lands.

## Behavioral Specification

### User Experience

- Every remaining package gets an explicit migration task.
- Each package task states how deterministic tests move into `test/fast/` and `test/feat/`, and how `test/e2e/` is simplified.
- No package migration starts from stale `unit`/`integration`/`e2e` assumptions.

### Expected Behavior

1. Run subtask `8r9.t.i05.0` first to refresh all child specs from the `ace-b36ts` pilot outputs.
2. Execute one package migration subtask at a time.
3. Make `ace-test <package>` default to `fast`.
4. Use `ace-test <package> feat` for deterministic feature coverage.
5. Keep `ace-test-e2e <package>` as the only scenario E2E entrypoint.
6. Run final cleanup subtask `8r9.t.i05.s` after package migrations to remove compatibility names and bridge logic.

### Interface Contract

- `ace-test <package>`
- `ace-test <package> feat`
- `ace-test <package> all`
- `ace-test-e2e <package>`

### Success Criteria

- [ ] Every remaining package has one explicit migration subtask.
- [ ] Every package subtask lists deterministic migration destinations under `test/fast/` and `test/feat/`.
- [ ] The refresh subtask runs before package migration implementation begins.
- [ ] No child task teaches `ace-test <package> e2e`.
- [ ] Final cleanup task removes legacy naming and temporary bridge code after migrations land.

## Vertical Slice Decomposition

- `8r9.t.i05.0` refresh all package subtasks from pilot outputs
- `8r9.t.i05.1` through `8r9.t.i05.r` migrate or simplify package-local coverage under `fast` / `feat` / `e2e`
- `8r9.t.i05.s` remove legacy `unit` / `integration` naming and bridge code after migration
