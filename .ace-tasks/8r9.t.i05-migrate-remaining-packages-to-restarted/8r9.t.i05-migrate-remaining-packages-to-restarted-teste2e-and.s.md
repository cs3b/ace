# Migrate batch 1 packages to the fast/feat/e2e testing model

## Objective

Apply the validated `ace-b36ts` pilot taxonomy to the first migration batch of packages through one package-specific migration subtask per package.

### User Experience

- Batch 1 operators can pick up any child task and see the exact `fast` / `feat` / `e2e` contract it must implement.
- Reviewers can confirm that the batch no longer teaches legacy `unit` / `integration` / `e2e` command usage before implementation starts.
- The batch stays blocked until every child spec is complete enough to execute without inventing missing migration behavior.

## Behavioral Specification

### User Experience

- Every package assigned to batch 1 gets an explicit migration task.
- Each package task states how deterministic tests move into `test/fast/` and `test/feat/`, and how `test/e2e/` is simplified.
- No batch 1 migration starts from stale `unit`/`integration`/`e2e` assumptions.

### Expected Behavior

1. Run subtask `8r9.t.i05.0` first to refresh all batch 1 child specs from the `ace-b36ts` pilot outputs.
2. Execute one batch 1 package migration subtask at a time.
3. Make `ace-test <package>` default to `fast`.
4. Use `ace-test <package> feat` for deterministic feature coverage.
5. Keep `ace-test-e2e <package>` as the only scenario E2E entrypoint.
6. Hand remaining tested packages to sibling batch task `8r9.t.i06`.
7. Run sibling cleanup task `8r9.t.i07` only after both migration batches are complete.

### Interface Contract

- `ace-test <package>`
- `ace-test <package> feat`
- `ace-test <package> all`
- `ace-test-e2e <package>`

### Success Criteria

- [ ] Every package assigned to batch 1 has one explicit migration subtask.
- [ ] Every batch 1 package subtask lists deterministic migration destinations under `test/fast/` and `test/feat/`.
- [ ] The refresh subtask runs before batch 1 migration implementation begins.
- [ ] No batch 1 child task teaches `ace-test <package> e2e`.
- [ ] Batch 1 cleanly hands off the remaining tested packages to `8r9.t.i06`.

## Vertical Slice Decomposition

- `8r9.t.i05.0` refresh all batch 1 package subtasks from pilot outputs
- `8r9.t.i05.1` through `8r9.t.i05.r` migrate or simplify batch 1 package-local coverage under `fast` / `feat` / `e2e`
- sibling `8r9.t.i06` owns batch 2 packages that do not fit under the child-slot limit of `8r9.t.i05`
- sibling `8r9.t.i07` removes legacy `unit` / `integration` naming and bridge code after both batches land

## Validation Questions

- None. The batch contract, package split, and command surface are already established by `8r9.t.hzr`, `8r9.t.i05.0`, and the current package layouts in this repo.

## Verification Plan

2. Confirm that every child task follows the pilot-owned `fast` / `feat` / `e2e` contract and that no child teaches `ace-test <pkg> e2e` or deterministic `test/e2e/*_test.rb` migration.
3. Re-run `ace-task show 8r9.t.i05 --content` after all children are `pending` and promote the parent only when no draft child remains in the subtree.
