# Migrate remaining packages to restarted test/e2e and scenario model

## Objective

Apply the validated ace-b36ts pilot architecture to all remaining packages through one package-specific subtask per package, with one refresh subtask that updates every child from the pilot outputs before migration starts.

## Behavioral Specification

### User Experience

- After the pilot lands, every remaining package gets its own explicit migration task.
- Each package task states exactly which deleted deterministic TCs move to which test/e2e file(s), and whether the package keeps or drops an agent scenario.
- No package migration starts from stale assumptions; all child tasks are refreshed once from pilot truth.

### Expected Behavior

1. Run subtask 8r9.t.i05.0 first to refresh all child specs from pilot outcomes.
2. Execute one package subtask at a time, preserving package-local migration scope.
3. Migrate deterministic coverage to test/e2e through ace-test.
4. Keep or simplify agent scenarios under test-e2e/scenarios based on package classification.
5. Use the pilot’s shared sandbox helper, scenario prompt model, and verification contract everywhere.

### Interface Contract

- ace-test <package> e2e
- ace-test-suite --target e2e
- ace-test-e2e <package>

### Success Criteria

- [ ] Every remaining package has one explicit migration subtask.
- [ ] Every package subtask lists exact deterministic migration sources and destinations.
- [ ] The refresh subtask runs before package migration implementation begins.
- [ ] No child task reintroduces deterministic execution into ace-test-e2e.

## Vertical Slice Decomposition

- 8r9.t.i05.0 refresh all package subtasks from pilot outputs
- 8r9.t.i05.1 through 8r9.t.i05.o migrate package-local deterministic and scenario coverage
- 8r9.t.i05.p through 8r9.t.i05.r simplify existing scenario-only packages under the restarted model
