# Root README Landing Page Rewrite

## Objective

Rewrite the root README.md as a landing page that converts first-time visitors into users. The current README reads as a reference manual with a tools table and configuration cascade. The new version should lead with why ACE exists, show a hero GIF, and make the value immediately obvious.

This subtask establishes the landing page template that all other package READMEs will follow.

## Behavioral Specification

### User Experience
- **Input**: Current root README.md, docs/vision.md for messaging, existing package list
- **Process**: Rewrite README as landing page following top OSS patterns (gum, lazygit, httpie)
- **Output**: New README.md that serves as project landing page on GitHub

### Expected Behavior

When a first-time visitor opens the root README on GitHub, they should experience:

1. **Hero section**: Tagline + hero GIF showing ACE in action (e.g., ace-review or ace-git-commit demo)
2. **Problem statement**: 2-3 sentences on what problem ACE solves (draw from docs/vision.md)
3. **Tools table**: Package name + one-line benefit + mini-GIF/screenshot for top 4-6 tools
4. **Quick Start**: Install and run first command in <2 minutes
5. **Principles**: Core principles as one-liners (not paragraphs)
6. **Documentation links**: Links to vision, architecture, contributing

### Interface Contract

README.md structure:

```markdown
# ACE - Agentic Coding Environment

> Tagline: one sentence value proposition

![Demo](docs/demo.gif)

## The Problem

2-3 sentences from vision.md, reframed for first-time visitors.

## Tools

| Tool | What it does |
|------|-------------|
| ace-review | Multi-model code review |
| ace-git-commit | LLM-powered commits |
| ace-bundle | Project context loading |
| ace-task | Task management with AI plans |
| ... | ... |

## Quick Start

gem install ace-review ace-git-commit
ace-git-commit -i "fix auth bug"

## Principles

1. **Same Tools** - Humans and agents use identical CLI commands
2. **Transparent** - Review sessions, dry-run, readable YAML configs
3. **Modular** - Every capability is an installable gem
4. **Provider Freedom** - Switch LLM providers without changing workflow

## Documentation

- [Vision](docs/vision.md)
- [Architecture](docs/architecture.md)
- [Contributing](docs/contributing/)

## License

MIT
```
