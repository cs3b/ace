# ace-test README and Getting Started

## Objective

Rewrite ace-test's README from 233-line reference manual into a landing page. The current README is packed with protocol listings (ace-nav guide://, wfi://, agent://), full file structure trees, performance target tables, IO isolation rules, guide description catalogs, workflow tables, and an architecture section -- all reference material that belongs in getting-started.md or the guides themselves. Despite an important callout that this is a "pure documentation package (no CLI tool)", it buries this under layers of protocol examples. Create a tutorial-style getting-started.md. Clarify the relationship: ace-test is the testing knowledge base, ace-test-runner is the execution engine.

- **Input**: Current ace-test/README.md (233 lines)
- **Process**: Rewrite README as landing page, create getting-started tutorial, write VHS tape, create usage.md + handbook.md, update gemspec, audit handbook
- **Output**: Rewritten README.md (~60 lines), new docs/getting-started.md, new VHS tape + GIF committed to docs/demo/, new docs/usage.md, docs/handbook.md, updated gemspec

New README should convey: "Testing knowledge base for ACE -- guides, patterns, and workflows for fast, reliable tests."

- Covers IO isolation, mocking patterns, performance targets, and TDD cycles

Current README problems to fix:
