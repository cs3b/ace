# Documentation Overhaul - Test and Handbook

## Objective

Rewrite documentation for ace-test, ace-test-runner, ace-test-runner-e2e, and ace-handbook to create a user-first landing experience. Transform reference-heavy READMEs into landing pages with hero GIFs, taglines, and quick start guides. Create tutorial-style getting-started.md files. Delete usage.md files (CLI reference stays in --help only).

Research basis: Diataxis framework, clig.dev, top OSS README patterns (gum, lazygit, httpie, ripgrep).

## Behavioral Specification

### User Experience

- **Input**: Existing package READMEs, --help output, current docs/usage.md files
- **Process**: Rewrite each package README as landing page, create getting-started tutorial, record demo GIF, delete usage.md
- **Output**: Per package: rewritten README.md, new docs/getting-started.md, new docs/demo.gif, deleted docs/usage.md

### Expected Behavior

When a first-time visitor opens any package README on GitHub, they should:
1. See a clear tagline explaining the tool's value proposition
2. See a hero GIF demo (30s or less) showing the tool's core value
3. Find a Quick Start section to get running in under 2 minutes
4. Find feature bullets framed as benefits
5. Find links to getting-started.md tutorial and --help for CLI reference
6. NOT find --help output, full config schemas, or API docs in the README

### Interface Contract

Each package follows the landing page template established by the documentation overhaul initiative:
- README.md: hero GIF, tagline, quick start, benefit-oriented features, links
- docs/getting-started.md: tutorial-style walkthrough for first-time users
- docs/demo.gif: hero GIF recording (30s or less, showing core value)
- docs/usage.md: deleted (CLI reference lives in --help only)

- [ ] Each package README opens with a one-line tagline
