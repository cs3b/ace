# Migrate canonical skills to .agent/skills/ (flat structure)

## Context

Skills currently live in `.claude/skills/` as the canonical source, with provider dirs (`.codex/`, `.gemini/`, `.pi/`) symlinking to it. This couples the canonical location to a single provider.

The goal is a provider-neutral canonical location: **`.agent/skills/`** (singular, following the emerging AGENTS.md community standard and consistent with all existing provider dir naming conventions which use singular: `.claude/`, `.cursor/`, `.codex/`).

The previous task formulation proposed a nested `ace/*/` structure. This was **abandoned**: nested paths make skill names inconsistent across providers (Claude Code would call it `/ace:git:commit` while other providers interpret the path differently or not at all). Flat names (`ace-git-commit`) are universally unambiguous.

## Behavioral Specification

### User Experience

- **Input**: Repository with `.claude/skills/` as canonical skills location
- **Process**: Move skills to `.agent/skills/`, replace all provider symlinks to point there
- **Output**: `.agent/skills/` is canonical; all provider dirs resolve to the same skills without behavior change

### Expected Behavior

After migration:

1. **`.agent/skills/`** contains all 94+ skills (flat `ace-*` names, unchanged)
2. **`.claude/skills`** is a symlink â†’ `../.agent/skills`
3. **`.codex/skills`**, **`.gemini/skills`**, **`.pi/skills`** symlinks updated â†’ `../.agent/skills`
4. All skills remain invocable in Claude Code at the same `/ace-*` names
5. The nested `ace-e2e-run` symlink (â†’ `../../ace-test-runner-e2e/handbook/skills/ace-e2e-run`) resolves correctly through the chain

### Interface Contract

```bash
# Before
ls -la .claude/skills/ace-git-commit/   # real dir

# After

ls -la .claude/skills/ace-git-commit/   # resolves via symlink chain
ls -la .codex/skills/ace-git-commit/    # resolves via symlink chain
```

### Success Criteria

- [ ] `.agent/skills/` exists and contains all skills
- [ ] `.claude/skills` is a symlink to `../.agent/skills`
