# Fork stall diagnostics: surface last message and enable resume

## Overview

When a forked agent session stalls (e.g., asks a question in non-interactive `codex exec` mode), the orchestrator has zero visibility into what happened. The only error is "did not complete within spawned session" with no agent context. Recovery requires manually digging through `~/.codex/sessions/` JSONL files.

This task adds:
1. **Diagnostics**: Capture and surface the agent's last message when a fork stalls
2. **Resume**: New `fork-resume` command to send a follow-up message to a stalled session
3. **Docs**: Update the drive workflow with fork stall recovery patterns

### Root Cause (from incident on task 278.03)
A forked Codex agent ran `bundle install` during release, which caused unrelated files in `git status`. The agent correctly asked for user direction — but in non-interactive mode nobody could answer. Timeout with no diagnostic output.

### Packages Affected
- `ace-llm-providers-cli` (CodexClient: `--output-last-message`, resume support)
- `ace-llm` (QueryInterface: thread new options)
- `ace-assign` (ForkSessionLauncher, ForkRun, Phase model, Status, new ForkResume command)

## Subtasks

- **01**: Surface agent last message on fork stall (`--output-last-message` + `stall_reason`)
- **02**: Fork resume command (`ace-assign fork-resume`)
- **03**: Update drive workflow with fork stall recovery patterns
