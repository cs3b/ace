# Specify GitHub repo name, description, and topics

## Behavioral Specification

### User Experience
- **Input**: Approved naming policy and selected slug
- **Process**: Maintainer gets an ordered GitHub settings checklist that can be executed without ambiguity
- **Output**: Repository metadata reflects ACE brand and ADE category while preserving link continuity

### Expected Behavior
This subtask defines the exact GitHub-side changes required:
- Repository name/slug update decision path
- Repository description/headline/tagline copy
- Topics and social-preview alignment
- Post-rename redirect and link-resolution verification checks

### Interface Contract
```text
Required GitHub output checklist:
- Repo slug target:
- Repo description:
- Profile/README headline line:
- Topics list:
- Redirect verification checks:
```

## Acceptance Criteria

- [ ] Checklist includes exact field-level updates and expected final values
- [ ] Redirect checks cover old URL resolution and clone URL continuity
- [ ] Failure/rollback conditions are specified

### Defaults
- **Description max length**: 350 characters (GitHub limit)
- **Topics limit**: Max 20 topics, prioritize discoverability terms
- **Redirect grace period**: GitHub provides automatic redirects indefinitely for renamed repos; verification checks should confirm this within 24h of rename

## Verification Plan

### Checklist Completeness
- Every GitHub metadata field (name, description, topics, social preview) has an explicit target value
- Redirect verification checks cover: old HTTPS URL, old clone URL (SSH + HTTPS), old API endpoint
- Rollback conditions are defined (what triggers reverting the rename)

### Copy Quality
- Description fits within 350-character GitHub limit
- Headline and tagline are distinct (not just copies of each other)
- Topics list includes both brand terms and category/discovery terms

### Negative Check
- Checklist does not include actual `gh api` or `gh repo rename` execution commands (spec only)

## Deliverables

- [ ] GitHub metadata update checklist
- [ ] Approved description/headline/tagline copy variants
- [ ] Redirect verification script/checklist (behavioral requirements only)
