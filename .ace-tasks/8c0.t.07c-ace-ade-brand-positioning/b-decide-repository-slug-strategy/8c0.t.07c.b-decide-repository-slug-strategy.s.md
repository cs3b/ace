# Decide repository slug strategy

## Behavioral Specification

### User Experience
- **Input**: Surface inventory and product-positioning direction (ACE brand + ADE category)
- **Process**: Author sees a side-by-side decision matrix with weighted criteria and explicit tradeoffs
- **Output**: A final decision statement for canonical repository slug and naming language

### Expected Behavior
The subtask defines objective criteria (brand continuity, discoverability, uniqueness, ecosystem fit, migration risk)
and records a recommendation with rationale. The expected default is:
- Brand: `ACE`
- Expansion: `Agentic Coding Environment`
- Category language: "Agentic Development Environment (ADE)"
- Repository slug preference order: `ace` (if available) -> keep `ace-meta` -> `agentic-coding-environment` only if brand slug is unavailable/unacceptable

### Interface Contract
```text
Decision output format:
1) Canonical brand name
2) Canonical expanded phrase
3) Canonical category phrase
4) Repository slug decision and fallback order
5) One-paragraph public rationale
```

## Acceptance Criteria

- [x] Decision matrix includes explicit pass/fail thresholds for each criterion
- [x] Final recommendation includes both primary and fallback slug options
- [x] Public-facing wording is consistent with prior decision: ACE brand, ADE category

## Deliverables

- [x] Naming decision matrix
- [x] Canonical naming policy statement
- [x] Final repo slug recommendation with fallback logic

### Defaults
- **Scoring method**: Simple weighted matrix (1-5 scale per criterion, weights summing to 100%)
- **Default weights**: Brand continuity 30%, discoverability 25%, uniqueness 20%, ecosystem fit 15%, migration risk 10%
- **Tie-breaking rule**: If scores are within 10%, prefer the option with lower migration risk

## Verification Plan

### Decision Completeness
- Decision matrix contains all 5 criteria with explicit scores for each slug candidate
- Final recommendation states primary slug and at least one fallback
- Public rationale paragraph exists and references the matrix scores

### Consistency Check
- Canonical brand name, expanded phrase, and category phrase are each stated exactly once with no contradictions
- Slug preference order is unambiguous (no "either/or" without resolution)

### Negative Check
- Decision does not include implementation steps (no `gh repo rename` commands)
