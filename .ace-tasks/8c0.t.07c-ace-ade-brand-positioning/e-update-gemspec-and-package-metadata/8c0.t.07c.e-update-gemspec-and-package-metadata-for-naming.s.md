# Update gemspec and package metadata for naming

## Behavioral Specification

### User Experience
- **Input**: Canonical naming policy and docs terminology matrix
- **Process**: Maintainer receives a package-level metadata checklist across all relevant gems
- **Output**: Gemspec summaries/descriptions and package readme metadata remain consistent with ACE/ADE positioning

### Expected Behavior
This subtask defines:
- Which `*.gemspec` fields need updates (summary, description, homepage where relevant)
- Which package `README.md` descriptors must align to the naming policy
- Compatibility checks to ensure no accidental changes to gem names, executable names, or require paths

### Interface Contract
```text
Per-package metadata checklist:
- Package:
- Gemspec fields to update:
- README sections to update:
- Must-not-change identifiers (gem name/bin/module):
```

## Acceptance Criteria

- [ ] Metadata update plan distinguishes copy changes from identity changes
- [ ] Plan explicitly preserves backward-compatible gem/binary identifiers
- [ ] Verification checklist includes scan for stale `ace-meta` references after updates

### Defaults
- **Gemspec fields in scope**: `summary`, `description`, `homepage` — gem `name`, `executables`, and `require_paths` are explicitly do-not-change
- **Package README scope**: Only the first paragraph (project description line) of each per-package README
- **Coverage**: All 42 gemspec files and their corresponding README.md files
- **Module/class names**: `Ace::*` namespace is unchanged (not a rename target)

## Verification Plan

### Coverage Check
- Alignment matrix lists all 42 gemspecs with explicit update/no-change status for each field
- Package README matrix covers all per-package READMEs identified in subtask a
- Must-not-change identifiers are listed per package (gem name, binary name, module namespace)

### Consistency Check
- All updated `summary` and `description` fields use the canonical terminology from subtask b
- No gemspec contains both old and new terminology after updates

### Safety Check
- Verify that `spec.name`, `spec.executables`, and `require` paths are explicitly marked do-not-change
- Verification checklist includes `bundle exec rake test` equivalent (`ace-test`) for at least a sample of updated gems
- Post-update scan command (`rg "ace-meta" --glob "*.gemspec"`) is defined to catch stale references

## Deliverables

- [ ] Gemspec metadata alignment matrix
- [ ] Package README alignment matrix
- [ ] Post-update verification checklist for naming consistency and compatibility
