# Inventory naming surfaces and blast radius

## Behavioral Specification

### User Experience
- **Input**: Current repository naming terms (`ACE`, `Agentic Coding Environment`, `ADE`) and candidate slugs (`ace`, `ace-meta`, `agentic-coding-environment`)
- **Process**: Author receives a concrete inventory of where names appear and what each change would affect
- **Output**: A complete map of update surfaces and risk notes for each surface

### Expected Behavior
The subtask produces a traceable inventory covering repository metadata, root docs, per-gem READMEs, gemspec summaries/descriptions,
workflow/skill references, badges, and links. Each surface is labeled as required update, optional update, or do-not-change.

### Interface Contract
```bash
# Discovery commands expected during implementation phase
rg -n "Agentic Coding Environment|Agentic Development Environment|ACE|ace-meta"
rg --files -g "*.gemspec"
```

### Success Criteria
- [x] Inventory includes GitHub metadata surfaces and local repository files
- [x] Every update candidate is tagged with ownership and risk level
- [x] No implementation changes are proposed in this phase

### Defaults
- **Output format**: Markdown table sections appended to this task file under a `## Results` heading
- **Naming surface definition**: Any file or metadata field containing the strings `ACE`, `Agentic Coding Environment`, `ace-meta`, or the repo URL
- **Risk levels**: critical (breaks builds/installs), high (breaks links/discovery), medium (inconsistent branding), low (cosmetic/internal)
- **Exclusions**: Files under `_legacy/`, `.ace-taskflow/done/`, and test fixtures are excluded from the inventory

## Scope

- Root-level docs and metadata files
- Per-package `README.md` and `*.gemspec` files
- Taskflow/workflow references that include repository naming

## Deliverables

- [x] Naming surface inventory table
- [x] Blast-radius summary with priority labels (critical/high/medium/low)
- [x] Recommended sequencing constraints for later subtasks

## Verification Plan

### Completeness Check
- Run `rg --files -g "*.gemspec" | wc -l` and confirm inventory covers all gemspec files (expected: 42)
- Run `rg --files -g "README.md" | wc -l` and confirm inventory covers all README files in active packages
- Run `rg -l "Agentic Coding Environment" --glob '!_legacy/**' --glob '!.ace-taskflow/done/**'` and confirm all matches appear in inventory

### Structure Check
- Every row in the inventory table has: surface name, file path, current value, update category (required/optional/do-not-change), risk level
- Blast-radius summary groups surfaces by risk level with counts
- Sequencing constraints reference subtask IDs (b through e)

### Negative Check
- Inventory does not include any `sed`/`rg --replace` commands or concrete replacement text (that belongs to later subtasks)

### 3. Recommended Sequencing Constraints

1. **Subtask b** (slug decision) must complete first — all URL updates depend on knowing the final slug (`ace` confirmed available)
2. **Subtask c** (GitHub metadata) should execute the rename before subtasks d and e, so that redirect verification can run against the live state
3. **Subtask d** (root docs) and **subtask e** (gemspec/package metadata) can proceed in parallel after c, but should be implemented in a single PR to avoid partial branding inconsistency
4. **Stale URLs found**: `ace-ecosystem/ace-meta` in ace-git-worktree and `ace-meta/ace-core` in ace-support-core should be flagged as pre-existing bugs to fix during subtask e regardless of rename decision
5. **Do-not-change surfaces**: CHANGELOGs, retrospectives, migration docs, and test fixture comments are historical records — do not update
