# Migrate ace-support-cli to the fast/feat/e2e testing model

## Objective

Move ace-support-cli into the new `fast`/`feat`/`e2e` model with a package-specific migration shape that matches its current test surface.

## Package Migration Shape

- Primarily fast package with current ATOM-style deterministic tests.
- Move ATOM folders into `test/fast/`.
- Create `test/feat/` only if package-level contract tests are discovered during refresh.

## Expected Behavior

1. Move current atoms/models/molecules/organisms tests into `test/fast/`.
2. Preserve package-level deterministic coverage in the fast loop by default.
3. Create `test/feat/` only when the refresh task identifies a deterministic feature/contract layer that should not remain in fast.
4. Do not invent `test/e2e/`.

## Verification

- `ace-test ace-support-cli`
- `ace-test ace-support-cli all`

## Success Criteria

- [ ] Existing ATOM-style tests run from `test/fast/`.
- [ ] `test/feat/` is only introduced if justified by package-owned feature coverage.
- [ ] No scenario layer is introduced without a real package-owned need.
