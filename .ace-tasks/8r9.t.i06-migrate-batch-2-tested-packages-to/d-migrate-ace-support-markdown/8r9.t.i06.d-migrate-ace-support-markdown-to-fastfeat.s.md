# Migrate ace-support-markdown to the fast/feat/e2e testing model

## Objective

Move ace-support-markdown into the new `fast`/`feat`/`e2e` model with a package-specific migration shape that matches its current test surface.

## Package Migration Shape

- Fast + feat package.
- Move smoke and atom tests into `test/fast/`.
- Move current `test/integration/` suites into `test/feat/`.
- Keep `test/e2e/` absent unless the package later owns real scenarios.

## Expected Behavior

1. Move fast-loop deterministic coverage into `test/fast/`.
2. Move current integration-like deterministic coverage into `test/feat/`.
3. Keep the public package contract on `fast`, `feat`, and `all`.
4. Update docs and package-local config to match the new layout.

## Verification

- `ace-test ace-support-markdown`
- `ace-test ace-support-markdown feat`
- `ace-test ace-support-markdown all`

## Success Criteria

- [ ] Smoke and atom tests run from `test/fast/`.
- [ ] Existing `test/integration/` suites move to `test/feat/`.
- [ ] Package docs and config stop teaching `integration` or `int` as the public layer name.
