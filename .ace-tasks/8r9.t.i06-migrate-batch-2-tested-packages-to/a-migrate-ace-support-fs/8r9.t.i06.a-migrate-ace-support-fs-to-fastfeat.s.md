# Migrate ace-support-fs to the fast/feat/e2e testing model

## Objective

Move ace-support-fs into the new `fast`/`feat`/`e2e` model with a package-specific migration shape that matches its current test surface.

## User Experience

- `ace-test ace-support-fs` becomes the only public deterministic loop for this package.
- The package remains fast-only in this migration because the current repo state contains only atom and molecule tests.
- Package docs teach the fast-only contract instead of implying extra layers.

## Package Migration Shape

- Fast-only package with existing deterministic tests.
- Move deterministic tests into `test/fast/`, preserving ATOM-style subfolders where they already exist.
- Do not create `test/feat/` or `test/e2e/` in this migration.

## Expected Behavior

1. Move current deterministic tests into `test/fast/`.
2. Keep the package default loop on `fast`.
3. Update package docs and references to teach the fast-only contract.
4. Do not invent `test/feat/` or `test/e2e/`.

## Interface Contract

- `ace-test ace-support-fs`
- `ace-test ace-support-fs all`

## Validation Questions

- None. Current repo inspection found only atom and molecule tests, with no `test/integration/`, `test/e2e/`, or scenario surface.

## Vertical Slice Decomposition

- This task is one standalone package migration slice for `ace-support-fs`. It ships independently once the package exposes the fast-only contract.

## Verification

- `ace-test ace-support-fs`
- `ace-test ace-support-fs all`
- Verify the migration does not add `test/feat/` or `test/e2e/` for this package.

## Success Criteria

- [ ] Existing deterministic tests run from `test/fast/`.

- [ ] No scenario layer is introduced without a real package-owned need.
