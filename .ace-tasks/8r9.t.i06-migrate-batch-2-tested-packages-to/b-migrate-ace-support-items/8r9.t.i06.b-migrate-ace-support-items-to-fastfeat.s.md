# Migrate ace-support-items to the fast/feat/e2e testing model

## Objective

Move ace-support-items into the new `fast`/`feat`/`e2e` model with a package-specific migration shape that matches its current test surface.

## Package Migration Shape

- Primarily fast package with existing deterministic tests.
- Move deterministic tests into `test/fast/`, preserving ATOM-style subfolders where they already exist.
- Create `test/feat/` only if the refresh task identifies a deterministic feature/contract layer that should not remain in the fast loop.

## Expected Behavior

1. Move current deterministic tests into `test/fast/`.
2. Keep the package default loop on `fast`.
3. Only introduce `test/feat/` if a deterministic feature layer is actually present and should be isolated from the fast loop.
4. Do not invent `test/e2e/`.

## Verification

- `ace-test ace-support-items`
- `ace-test ace-support-items all`

## Success Criteria

- [ ] Existing deterministic tests run from `test/fast/`.
- [ ] `test/feat/` is only introduced when justified by package-owned behavior.
- [ ] No scenario layer is introduced without a real package-owned need.
