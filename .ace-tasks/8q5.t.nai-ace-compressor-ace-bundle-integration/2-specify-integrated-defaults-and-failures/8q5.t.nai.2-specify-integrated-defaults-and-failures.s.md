# Specify integrated defaults and failures

## Behavioral Specification

### User Experience
- **Input**: Successful and failing bundle-style compressor invocations.
- **Process**: The command applies defaults consistently and reports integrated-resolution failures clearly.
- **Output**: Predictable default behavior for mode and source-scope, plus actionable error messages for bad inputs.

### Expected Behavior
Integrated compressor usage must be understandable without reading implementation code. Missing or invalid sources must identify the exact failing input. Defaults must stay consistent: `merged` source scope by default, mode explicitly chosen by the caller, and no hidden requirement to run another ACE tool first.

### Interface Contract
```bash
ace-compressor compress project --mode exact
ace-compressor compress unknown-preset --mode exact
ace-compressor compress wfi://missing/resource --mode compact --source-scope per-source
```

**Error Handling:**
- Unknown preset or missing file: exit non-zero and identify the source.
- Unresolved protocol: exit non-zero and identify the resource.

**Edge Cases:**
- Mixed valid and invalid sources fail loudly instead of producing partial success silently.
- Successful merged output with unresolved inner chart/image references still succeeds and marks unresolved records.

### Success Criteria
- [ ] Defaults are explicit and documented.
- [ ] Bad source errors are actionable.
- [ ] Mixed valid/invalid source sets fail loudly.
- [ ] Successful runs can still surface unresolved inner artifacts without becoming hard failures.

### Validation Questions
- [x] Integrated defaults should be explicit, not inferred from hidden state.
- [x] Mixed valid/invalid source sets fail non-zero.
- [x] Inner unresolved artifacts in otherwise valid content stay visible but do not become resolution failures.
- [x] Users should not need to know `ace-bundle` internals to interpret failures.
