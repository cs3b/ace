# Harden integrated errors and defaults

## Behavioral Specification

### User Experience
- **Input**: Successful and failing ACE-native compressor invocations.
- **Process**: The CLI applies default `merged` behavior, surfaces actionable source-specific failures, and preserves traceability in integrated output.
- **Output**: A hardened ACE-native integration path ready for normal use.

### Expected Behavior
This slice finishes the integration phase. Defaults must be explicit, invalid sources must be reported precisely, and successful integrated runs must keep provenance clear even when inner unresolved artifacts are present.

### Interface Contract
```bash
ace-compressor compress project --mode exact
ace-compressor compress unknown-preset --mode exact
ace-compressor compress wfi://missing/resource --mode compact --source-scope per-source
```

**Error Handling:**
- Unknown preset, missing file, and unresolved protocol each identify the failing source.
- Mixed valid/invalid source lists fail overall with actionable messaging.

**Edge Cases:**
- `merged` remains default when `--source-scope` is absent.
- Inner unresolved chart/image references inside valid sources do not become resolution errors.

### Success Criteria
- [ ] Default integrated behavior is explicit and testable.
- [ ] Resolution failures are actionable and source-specific.
- [ ] Mixed valid/invalid source runs fail clearly.
- [ ] Provenance and unresolved inner artifacts remain visible on successful runs.

### Validation Questions
- [x] `merged` remains the default scope here.
- [x] Inner unresolved artifacts are not resolution failures.
- [x] Mixed source lists fail overall if any source cannot be resolved.
- [x] This slice finishes the integrated user-facing contract.
