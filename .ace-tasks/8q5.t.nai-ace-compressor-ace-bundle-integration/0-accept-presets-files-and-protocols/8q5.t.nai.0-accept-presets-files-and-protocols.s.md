# Accept presets files and protocols

## Behavioral Specification

### User Experience
- **Input**: Preset names, explicit files, and protocol URLs passed to `ace-compressor compress`.
- **Process**: The command resolves inputs using ACE conventions users already know.
- **Output**: A normalized set of resolved sources ready for compression.

### Expected Behavior
`ace-compressor` must accept the same high-level source types that `ace-bundle` already accepts: presets, file paths/config files, and protocol URLs. Users should not need to run `ace-bundle` first or pipe its output manually.

### Interface Contract
```bash
ace-compressor compress project --mode exact
ace-compressor compress ./custom-context.md --mode exact
ace-compressor compress wfi://task/draft --mode compact
```

**Error Handling:**
- Unknown preset: exit non-zero and name the preset.
- Unresolved protocol: exit non-zero and name the protocol URL.

**Edge Cases:**
- Mixed preset + file inputs resolve in one invocation.
- File inputs that exist locally take precedence over preset-name interpretation.

### Success Criteria
- [ ] Preset names resolve.
- [ ] Files and config files resolve.
- [ ] Protocol URLs resolve.
- [ ] No shell piping is required.

### Validation Questions
- [x] Input auto-detection should match ACE conventions.
- [x] Local files win over preset interpretation when the path exists.
- [x] Protocol URLs are first-class compressor inputs.
- [x] Resolution errors name the failing source.
