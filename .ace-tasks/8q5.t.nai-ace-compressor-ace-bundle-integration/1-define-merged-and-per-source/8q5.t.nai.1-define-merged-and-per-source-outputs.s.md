# Define merged and per-source outputs

## Behavioral Specification

### User Experience
- **Input**: Resolved ACE-style sources passed to `ace-compressor`.
- **Process**: The command compresses either all resolved sources into one pack or emits one pack per source.
- **Output**: `merged` output by default, or `per-source` output when requested explicitly.

### Expected Behavior
`merged` is the default source scope. It creates one compressed artifact from all resolved sources while preserving source-level provenance. `per-source` is explicit and yields separate compressed outputs for each resolved source so agents can load or patch them independently.

### Interface Contract
```bash
ace-compressor compress project --mode exact --source-scope merged
ace-compressor compress project --mode compact --source-scope per-source
```

**Error Handling:**
- Unknown `source-scope` value: exit non-zero with allowed values.
- `per-source` output requested with zero resolved sources: exit non-zero.

**Edge Cases:**
- `merged` output still retains source provenance for each fact.
- `per-source` output preserves source ordering from resolution.

### Success Criteria
- [ ] `merged` is the default behavior.
- [ ] `per-source` is explicit.
- [ ] Source provenance survives both modes.
- [ ] Output behavior is predictable for agents and developers.
