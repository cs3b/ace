# Support protocol and per-source flows

## Behavioral Specification

### User Experience
- **Input**: Protocol URLs and other resolved ACE-native inputs used with `--source-scope per-source`.
- **Process**: The CLI resolves protocol-backed sources and emits one compressed output per resolved source.
- **Output**: A runnable per-source integration path.

### Expected Behavior
This slice extends ACE-native integration from merged mode to per-source mode. Users can run protocol-backed inputs such as `wfi://task/draft` and obtain separate outputs per resolved source in stable order.

### Interface Contract
```bash
ace-compressor compress wfi://task/draft --mode compact --source-scope per-source
ace-compressor compress project --mode exact --source-scope per-source
```

**Error Handling:**
- Unresolved protocol: exit non-zero and name the URL.
- Invalid `--source-scope` value: exit non-zero with allowed values.

**Edge Cases:**
- Per-source output preserves source ordering.
- Provenance remains tied to each emitted source pack.

### Success Criteria
- [ ] Protocol-backed inputs resolve directly in `ace-compressor`.
- [ ] `--source-scope per-source` emits one output per resolved source.
