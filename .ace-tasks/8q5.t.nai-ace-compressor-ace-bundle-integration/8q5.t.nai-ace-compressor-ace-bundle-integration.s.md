# ace-compressor: ace-bundle integration

## Behavioral Specification

### User Experience
- **Input**: `ace-bundle`-style sources such as preset names, file paths, config files, and protocol URLs passed directly to `ace-compressor`.
- **Process**: `ace-compressor` resolves the same source shapes ACE users already know, then compresses them either into one merged output or one output per resolved source.
- **Output**: `ContextPack/1` output for either `merged` or `per-source` scope, with preserved provenance per source.

### Expected Behavior
This task integrates the compressor with ACE context loading conventions without requiring shell piping between ace tools. `ace-compressor` must accept preset-like names, explicit files, and protocol URLs directly. The default source scope is `merged`; `per-source` is an explicit mode that emits one compressed artifact per resolved source.

### Interface Contract
```bash
ace-compressor compress project --mode exact
ace-compressor compress wfi://task/draft --mode compact --source-scope per-source
ace-compressor compress -f custom.yml -p project --mode exact --source-scope merged
```

**Error Handling:**
- Unknown preset or unresolved protocol: exit non-zero with the failing source called out explicitly.
- Mixed valid and invalid bundle-style sources: fail non-zero and report which source could not be resolved.

**Edge Cases:**
- `per-source` mode preserves provenance per source artifact.
- `merged` mode keeps source-level provenance so facts still trace back to individual inputs.

### Success Criteria
- [ ] `ace-compressor` accepts bundle-style inputs directly.
- [ ] `merged` is the default source scope.
- [ ] `per-source` is explicit and produces one output per resolved source.
- [ ] No shell piping between `ace-bundle` and `ace-compressor` is required.
