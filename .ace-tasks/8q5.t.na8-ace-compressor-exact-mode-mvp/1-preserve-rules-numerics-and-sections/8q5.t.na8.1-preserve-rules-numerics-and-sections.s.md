# Add exact directory and multi-file runs

## Behavioral Specification

### User Experience
- **Input**: Multiple file paths or one directory passed to exact mode.
- **Process**: The CLI resolves supported sources, orders them deterministically, and emits one merged exact pack with per-source provenance.
- **Output**: One runnable exact flow for multi-file and directory inputs.

### Expected Behavior
This slice extends the package from one-file operation to the real repository use case: compressing sets of Markdown/text sources together. It must preserve provenance per source and avoid duplicated output when the same explicit file path appears twice.

### Interface Contract
```bash
ace-compressor compress docs/vision.md docs/architecture.md --mode exact
ace-compressor compress docs/ --mode exact --verbose
```

**Error Handling:**
- Directory with no supported sources: exit non-zero and explain what qualifies as supported input.
- Explicit binary file path in a multi-source invocation: exit non-zero.

**Edge Cases:**
- Repeated explicit paths are collapsed once.
- Unsupported files discovered during directory traversal are ignored unless `--verbose` is used.

### Success Criteria
- [ ] Multi-file exact compression works end to end.
- [ ] Directory exact compression works end to end.
- [ ] Output ordering is deterministic.
- [ ] Per-source provenance is preserved.

### Validation Questions
- [x] This slice must cover both directory and explicit multi-file inputs.
- [x] One merged exact pack is the output shape in phase 1.
- [x] Explicit bad paths fail loudly.
- [x] Traversal-discovered unsupported files do not kill the whole run by default.
