# Preserve rules numerics and sections

## Behavioral Specification

### User Experience
- **Input**: Markdown/plain-text sources with rules, lists, sections, and numeric facts.
- **Process**: Exact mode compacts presentation but preserves imperative language and numeric details exactly.
- **Output**: `ContextPack/1` records that keep rule modality, values, and section coverage intact.

### Expected Behavior
This slice defines the preservation contract. Any imperative rule or numeric fact that appears in the source must survive exact compression unchanged in meaning. Section coverage must remain auditable so a user can tell that a section was preserved, compacted, or marked unresolved.

### Interface Contract
```bash
ace-compressor compress docs/decisions.md --mode exact
```

**Error Handling:**
- Ambiguous numeric extraction: report the raw source fragment in provenance instead of rewriting the value.
- Unknown markdown block type: keep a compact fallback record rather than dropping the block.

**Edge Cases:**
- Repeated rule text across files keeps one canonical record plus provenance list.
- Tables remain represented, even if compacted structurally.

### Success Criteria
- [ ] Rule modality is preserved exactly.
- [ ] Numeric values are preserved exactly.
- [ ] Every major section is represented or marked unresolved.
- [ ] No table or rule disappears silently.

### Validation Questions
- [x] Exact mode never rewrites imperative rule strength.
- [x] Exact mode keeps numeric fidelity over readability.
- [x] Unknown blocks degrade to explicit fallback records instead of omission.
- [x] Coverage is section-based, not paragraph-count based.
