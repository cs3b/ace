# ace-compressor: exact mode MVP

## Behavioral Specification

### User Experience
- **Input**: One or more Markdown or plain-text files, directories, or explicit file paths passed to `ace-compressor compress ... --mode exact`.
- **Process**: The command parses source structure, compacts formatting noise, preserves reasoning-critical semantics, and reports unresolved non-text artifacts explicitly.
- **Output**: Deterministic `ContextPack/1` output that is smaller than the source while preserving rules, numeric values, headings, and provenance anchors.

### Expected Behavior
`exact` is the user-facing name for the lossless MVP. The command must preserve meaning fully while reducing token-heavy presentation details such as redundant frontmatter fields, repeated whitespace, markdown decoration, and duplicated explanatory prose. The MVP only accepts Markdown and plain text. Image-only charts are not interpreted; they are emitted as unresolved items with source provenance.

### Interface Contract
```bash
ace-compressor compress README.md docs/architecture.md --mode exact
ace-compressor compress docs/vision.md --mode exact --output stdio
```

**Operating Modes / Defaults:**
- Standard ACE CLI flags (`--quiet`, `--verbose`, `--debug`, `--help`) follow the repo's normal dry-cli conventions and are in-scope as inherited behavior, not bespoke MVP features.
- This task defines `--output stdio` behavior only. File-path and cache output targets are out of scope for the MVP.
- This task does not define an implicit compression-mode default. Callers must choose `--mode exact` explicitly.

**Error Handling:**
- Unsupported binary or non-text input: exit non-zero with a message that MVP input is limited to Markdown/plain text.
- Unresolved chart/image reference: continue successfully, emit unresolved chart record, do not guess numeric content.

**Edge Cases:**
- Duplicate paragraphs across sections: preserve one canonical payload plus provenance.
- Missing or noisy frontmatter: compact what is usable and continue without inventing fields.
- Empty file: exit non-zero with a clear message instead of emitting an empty pack.
- Directory input with no supported Markdown/plain-text files: exit non-zero and say that no supported sources were found.
- Repeated explicit file arguments for the same source: compress the source once and preserve one provenance entry per unique source path.
- Directory traversal includes only Markdown/plain-text sources; unsupported files discovered during traversal are skipped and reported only through verbose output.

- [ ] `ace-compressor compress <input...> --mode exact` emits deterministic `ContextPack/1` output for Markdown/plain-text inputs.
- [ ] Imperative rules, decisions, and numeric values are preserved exactly.
- [ ] Formatting-only noise is compacted without semantic loss.
- [ ] Unresolved charts/images are surfaced explicitly rather than guessed.
- [ ] File inputs, directory inputs, empty inputs, and repeated explicit inputs have explicit and testable behavior.
- [ ] Standard ACE CLI operating modes are inherited cleanly, while bespoke MVP output behavior stays decision-complete.

## Objective

Ship the first usable `ace-compressor` behavior around exact preservation so ACE can compact prompt context without losing imperative rules, numeric values, or document structure.
