# Harden exact preservation and input failures

## Behavioral Specification

### User Experience
- **Input**: Rule-heavy, numeric, and table-bearing docs plus invalid inputs.
- **Process**: Exact mode preserves mandatory semantics and emits explicit failures or unresolved markers where required.
- **Output**: A hardened exact path suitable for policies, ADRs, and workflow docs.

### Expected Behavior
This slice turns exact mode from “works” into “safe enough to trust”. It must preserve imperative modalities, numeric values, and table structure while clearly handling unknown blocks, image-only charts, empty inputs, and binary files.

### Interface Contract
```bash
ace-compressor compress docs/decisions.md --mode exact
ace-compressor compress docs/ace-gems.g.md --mode exact --verbose
```

**Error Handling:**
- Binary file path: exit non-zero with Markdown/text-only guidance.
- Empty file or empty directory: exit non-zero.
- Image-only chart reference inside supported doc: emit unresolved record and succeed.

**Edge Cases:**
- `must`, `must not`, `never`, `required`, and `only` remain exact.
- Large tables remain represented even if compacted structurally.

### Success Criteria
- [ ] Rule modality is preserved exactly.
- [ ] Numeric facts are preserved exactly.
- [ ] Tables are never dropped silently.
- [ ] Exact-mode failure behavior is explicit and tested.

### Validation Questions
- [x] Policies and ADRs are primary exact-mode targets.
- [x] Unresolved chart/image references are non-blocking inside supported docs.
- [x] Unknown markdown blocks degrade to explicit fallback records.
- [x] Exact mode favors fidelity over prettiness.
