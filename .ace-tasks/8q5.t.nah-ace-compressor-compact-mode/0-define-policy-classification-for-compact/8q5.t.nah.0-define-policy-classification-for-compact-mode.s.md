# Compact narrative docs end to end

## Behavioral Specification

### User Experience
- **Input**: Narrative-heavy docs such as vision/readme/guides passed to compact mode.
- **Process**: The CLI classifies the source as safe for aggressive compacting and emits dense structured output plus policy metadata.
- **Output**: A working narrative compact path with smaller output than exact mode.

### Expected Behavior
This slice must run end to end on narrative-heavy docs. It is not enough to label a source as `narrative-heavy`; the output must actually be smaller and visibly compacted, with `POLICY|...|action=aggressive_compact` or equivalent runtime metadata.

### Interface Contract
```bash
ace-compressor compress docs/vision.md --mode compact
ace-compressor compress README.md --mode compact --verbose
```

**Error Handling:**
- Unsupported binary input: exit non-zero.
- Unknown class in this slice: fall back conservatively, not aggressively.

**Edge Cases:**
- Repeated examples may collapse to refs.
- Short imperative lines inside otherwise narrative docs remain explicit.

### Success Criteria
- [ ] Narrative docs compact materially more than exact mode.
- [ ] Output includes policy metadata for compact decisions.
- [ ] Facts/summaries remain usable for downstream agents.
- [ ] At least one automated test covers narrative compacting.

### Validation Questions
- [x] This slice includes actual compact output, not just classification.
- [x] `POLICY|...` output is runtime-visible.
- [x] Unknown sources fall back conservatively.
- [x] Imperative lines inside narrative docs still survive.
