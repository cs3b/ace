# Compress tables and examples with loss metadata

## Behavioral Specification

### User Experience
- **Input**: Table-heavy and example-heavy docs passed to compact mode.
- **Process**: The CLI shrinks rows/examples/code references where safe and emits explicit metadata describing what was lost or retained.
- **Output**: Compact output with visible `LOSS|...` and `EXAMPLE_REF|...` style records.

### Expected Behavior
This slice turns compact mode into something trustworthy for structured docs. Tables may be summarized to schema-plus-key-rows, examples may collapse to refs, and code blocks may compact to essential lines or refs, but every reduction must be explicitly marked.

### Interface Contract
```bash
ace-compressor compress docs/ace-gems.g.md --mode compact --verbose
ace-compressor compress README.md docs/vision.md --mode compact
```

**Error Handling:**
- Table too sensitive to summarize safely: preserve more structure or refuse.
- Missing loss metadata for a summarized table/example is a test failure.

**Edge Cases:**
- Repeated examples across sources collapse to one ref with provenance.
- Header rows are not counted as retained data rows in loss metadata.

### Success Criteria
- [ ] Tables compact with explicit `LOSS|...` metadata when rows are elided.
- [ ] Repeated examples compact to refs.
- [ ] Sensitive structured content stays explicit or refuses.
- [ ] Automated tests cover table/example loss signaling.
