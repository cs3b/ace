# Handle mixed-source refusals end to end

## Behavioral Specification

### User Experience
- **Input**: One invocation containing both safe narrative docs and unsafe rule-heavy docs.
- **Process**: The CLI compacts safe sources, refuses unsafe ones with explicit metadata, and exits non-zero for the overall run.
- **Output**: A mixed-outcome compact run with both compact data and refusal records.

### Expected Behavior
This slice delivers the hardest compact-mode product behavior: one command can produce usable output for some sources and a refusal for others. Safe-source output must not disappear just because one source refuses.

### Interface Contract
```bash
ace-compressor compress docs/vision.md docs/decisions.md --mode compact --verbose
```

**Error Handling:**
- Any source failing fidelity preflight emits `REFUSAL|...` and `GUIDANCE|retry_with=--mode exact`.
- Overall process exits non-zero when any source refuses.

**Edge Cases:**
- Rule-heavy sections inside mixed docs may keep exact sections rather than full-source refusal if fidelity stays intact.
- Safe-source compact output is emitted before the process exits.

### Success Criteria
- [ ] Mixed-source runs can contain both compact output and refusal metadata.
- [ ] Unsafe sources never compact silently.
- [ ] Overall exit code is non-zero when any source refuses.
- [ ] Safe-source output remains available to the caller.

### Validation Questions
- [x] This slice must preserve safe-source output on partial failure.
- [x] Refusal guidance points to `--mode exact`.
- [x] Per-source divergence is first-class behavior.
- [x] Mixed-source output is part of the CLI contract, not just docs.
