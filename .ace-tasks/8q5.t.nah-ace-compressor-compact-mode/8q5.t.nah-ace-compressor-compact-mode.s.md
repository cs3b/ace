# ace-compressor: compact mode runnable CLI

## Behavioral Specification

### User Experience
- **Input**: Markdown/text files or directories passed to `ace-compressor compress ... --mode compact`.
- **Process**: The CLI classifies each source, applies the matching compact action, preserves rule-bearing sections or refuses unsafe compacting, and emits explicit policy/fidelity/loss metadata.
- **Output**: A runnable compact mode that is materially smaller than exact mode and still safe to use.

### Expected Behavior
This phase builds on the runnable package from `na8`. Compact mode must be executable, testable, and capable of mixed outcomes per source. The old `nah` semantics stay intact as runtime contracts: `POLICY|...`, `REFUSAL|...`, `GUIDANCE|...`, `LOSS|...`, and `FIDELITY|...` are real CLI outputs, not doc-only annotations.

### Compact Runtime Contract

**Classification classes:**
- `narrative-heavy`
- `mixed`
- `rule-heavy`
- `unknown`

**Classification precedence:**
- Detect rule-bearing sections first.
- Use `mixed` when narrative and rule-bearing signals both exist.
- Use `unknown` when the classifier cannot establish a narrower safe class.

**Action tokens:**
- `aggressive_compact`
- `compact_with_table_guards`
- `compact_with_exact_rule_sections`
- `conservative_compact`
- `refuse`

**Fidelity gates:**
- `imperative_rules`
- `numeric_facts`
- `table_structure`

### Interface Contract
```bash
ace-compressor compress docs/vision.md --mode compact
ace-compressor compress docs/vision.md docs/decisions.md --mode compact --verbose
ace-compressor compress docs/custom-notes.md --mode compact --verbose
ace-compressor compress docs/blueprint.md --mode compact --verbose
```

**Operating Modes / Defaults:**
- Standard ACE CLI flags apply.
- `--verbose` exposes per-source classification and policy decisions.
- `--quiet` may suppress extra explanation but never hides refusal outcomes.
- `--dry-run` and `--force` are out of scope here.

**Error Handling:**
- Unsafe source: emit refusal metadata and exit non-zero.
- Mixed invocation with at least one refusing source: preserve safe-source output, emit refusal for unsafe sources, exit non-zero.
- Unsupported binary input: same explicit non-text error as exact mode.

**Edge Cases:**
- `class=unknown` uses `action=conservative_compact`; it does not trigger failure by itself.
- `class=mixed` may succeed with `action=compact_with_exact_rule_sections` when rule-bearing sections remain exact.
- Rule-heavy sections inside mixed docs may stay exact while narrative sections compact.
- Repeated examples collapse to refs with provenance.
- Tables may shrink, but strategy and loss must be explicit.

### Success Criteria
- [ ] Narrative-heavy docs compact materially more than exact mode.
- [ ] Compact classification uses `narrative-heavy`, `mixed`, `rule-heavy`, and `unknown`.
- [ ] `POLICY|...|action=...` exposes one of the approved action tokens at runtime.
- [ ] Fidelity gates over `imperative_rules`, `numeric_facts`, and `table_structure` determine success or refusal.
- [ ] Unsafe compacting refuses explicitly instead of degrading silently.
- [ ] Mixed runs preserve safe-source output and fail overall only when at least one source reaches `action=refuse`.
- [ ] `--verbose` and `--quiet` behavior is explicit and testable.
