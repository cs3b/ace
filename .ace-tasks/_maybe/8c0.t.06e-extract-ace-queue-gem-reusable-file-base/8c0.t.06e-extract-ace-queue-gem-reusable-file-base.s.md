# Extract ace-queue gem - Reusable File-Based Queue Infrastructure

## Behavioral Specification

### User Experience

- **Input**: Directory path and schema configuration for queue behavior
- **Process**:
  - Consumer gems initialize queue with path and schema
  - Queue manages file creation, numbering, and frontmatter
  - Operations (list, add, update, reschedule) work consistently across consumers
- **Output**: Ordered queue items with consistent behavior across ace-coworker, ace-taskflow, ace-review

### Expected Behavior

The ace-queue gem provides shared infrastructure for file-based queue management:

1. **Hierarchical Numbering**: Automatic numbering with configurable gaps (010, 020, 030) and subtask support (.01, .02) up to 3 levels deep.

2. **Frontmatter Management**: Safe frontmatter reading/writing with schema validation per consumer. Timestamps, status tracking, and custom fields.

3. **Lexicographic Ordering**: Queue order determined by filename sorting - no index file needed.

4. **Queue Operations**: List, add, update, reschedule, advance - consistent API for all consumers.

5. **Schema Extension**: Consumers define their frontmatter requirements while inheriting core queue behavior.

### Interface Contract

```ruby
# Initialize queue with path and schema
queue = Ace::Queue.new(path: "jobs/", schema: :coworker)

# List items (lexicographic order)
queue.list                      # → Array of QueueItem
queue.list(status: :pending)    # → Filtered by status
queue.current                   # → Item with status: in_progress

# Add item (auto-numbering)
queue.add(name: "fix-bug", instructions: "...")
# Creates 041-fix-bug.md if last was 040

# Add subtask (hierarchical)
queue.add(name: "setup", parent: "030")
# Creates 030.01-setup.md

# Update frontmatter
queue.update("040", status: :failed, error: "Test failed")

# Reschedule (move to new position)
queue.reschedule("040", after: "050")
# Renames to next available after 050

# Advance queue
queue.advance(report: "Content appended to current item")
```

```bash
# CLI (optional, for debugging/admin)
ace-queue list jobs/
ace-queue show jobs/040
ace-queue reschedule jobs/040 --after 050
```

**Numbering Schema:**
```
010-init.md           # Main item (10-step gaps)
010.01-setup.md       # Subtask (.01 suffix)
010.01.01-create.md   # Sub-subtask (3 levels max)
020-build.md          # Next main
041-hotfix.md         # Injected after 040
```

**Frontmatter Schema (base):**
```yaml
---
name: item-name           # Required
status: pending           # pending | in_progress | done | failed
started_at: null          # ISO8601 timestamp
completed_at: null        # ISO8601 timestamp
error: null               # Error message (if failed)
added_by: null            # null | dynamic | retry_of:040
parent: null              # Parent reference (for subtasks)
---
```

**Error Handling:**
- Invalid parent reference: "Error: Parent 099 not found"
- Max depth exceeded: "Error: Maximum 3 levels allowed (010.01.01)"
- Reschedule conflict: Auto-increment to next available position

**Edge Cases:**
- Empty queue: Returns empty list, no current item
- All items done: Returns null for current, allows adding new items
- Concurrent modifications: Last write wins (file-based)

### Success Criteria

- [ ] `Ace::Queue.new(path:, schema:)` initializes queue with configurable schema
- [ ] `queue.list` returns items sorted lexicographically by filename

## Objective

Extract the file-based queue infrastructure designed for ace-coworker into a reusable gem. This enables consistent queue behavior across multiple ACE tools (jobs, tasks, reviews) without code duplication.
