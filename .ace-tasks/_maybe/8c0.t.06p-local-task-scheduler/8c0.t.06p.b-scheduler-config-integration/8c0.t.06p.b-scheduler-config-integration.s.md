# Scheduler Configuration and Integration

## Objective

Configure default scheduled tasks, integrate with E2E test runner and release workflows. This replaces the CI-based performance gate (253.06) with local automation.

## Behavioral Specification

### User Experience

**During Release Draft:**
- E2E tests run automatically via `ace-scheduler emit release-started`
- Results verified before release tasks created

**During Release Publish:**
- Final verification runs
- E2E results checked

### Interface Contract

```bash
# When running /ace:draft-release
# Step added: ace-scheduler emit release-started

# Manual maintenance
ace-scheduler run daily-tests
ace-scheduler emit pr-merged
```

### Success Criteria

- [ ] Default project scheduler config created
- [ ] Release workflows emit scheduler events
- [ ] E2E tests run before release finalization
- [ ] Documentation updated

## Scope of Work

### Deliverables

#### Create

**Project Configuration:**
```yaml
# .ace/scheduler/config.yml
version: 1

tasks:
  daily-tests:
    description: "Run test suite with profiling"
    cron: "0 9 * * *"
    command: ace-test-suite --profile 10
    enabled: true
    timeout: 3600

  weekly-e2e:
    description: "Full E2E test run"
    cron: "0 6 * * 1"
    command: ace-e2e-test-suite --all
    enabled: true
    timeout: 7200

  weekly-perf-audit:
    description: "Performance audit"
    cron: "0 7 * * 1"
    command: ace-test-suite --profile 20
    enabled: true
    timeout: 1800

events:
  pr-merged:
    description: "Post-merge verification"
    triggers:
      - command: ace-test-suite
        timeout: 1800

  release-started:
    description: "Pre-release gates"
    triggers:
      - command: ace-test-suite
        timeout: 1800
      - command: ace-e2e-test-suite --all
        timeout: 7200
```
