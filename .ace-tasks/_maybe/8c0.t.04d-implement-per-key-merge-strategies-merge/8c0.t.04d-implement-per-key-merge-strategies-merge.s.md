# Implement per-key merge strategies (_merge directive)

## Scope

Add support for per-key merge strategies in YAML configs using a `_merge` metadata key. Currently, merge strategy is global to the entire ConfigResolver. This enables fine-grained control over how arrays are merged in the configuration cascade.

**Current behavior** (global):
```ruby
ConfigResolver.new(merge_strategy: :replace)  # Applies to ALL arrays
```

**Proposed behavior** (per-key):
```yaml
dependencies:
  _merge: concat
  - lodash
  - moment

plugins:
  _merge: union
  - plugin-a
  - plugin-b
```

## Implementation Notes

### File to Modify

`ace-config/lib/ace/config/atoms/deep_merger.rb`

### Changes (~30-40 lines)

```ruby
def merge(base, other, options = {})
  other.each do |key, other_value|
    local_strategy = options[:array_strategy] || :replace

    if other_value.is_a?(Hash) && other_value.key?('_merge')
      local_strategy = other_value['_merge'].to_sym
      other_value = other_value.except('_merge')
    end

    # ... existing merge logic with local_strategy ...
  end
end
```

### Edge Cases

1. **Conflicting directives**: Overlay `_merge` wins
2. **_merge on non-arrays**: Ignore or warn
3. **Nested arrays**: Strategy applies to top-level only
4. **Empty with _merge**: Treat as explicit empty
