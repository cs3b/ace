# Container-Based E2E Test Isolation for macOS

## Behavioral Specification

### User Experience
- **Input**: Developer runs E2E tests via `ace-test-e2e` or agent runs via `/ace:run-e2e-test`
- **Process**: Tests execute inside a lightweight Linux container (Lima/OrbStack) with isolated git repository
- **Output**: Test results with full filesystem isolation, no host pollution

### Expected Behavior

When E2E tests require true filesystem isolation:
1. Container image with Ruby + project gems pre-installed is available
2. Test artifacts are mounted to `.cache/test-e2e/` on host for persistence
3. Git operations inside container don't affect host repositories
4. Tests run identically on CI (Linux) and local development (macOS)

### Interface Contract

```bash
# Option 1: Direct container execution
ace-test-e2e --container MT-WORKTREE-002

# Option 2: Container wrapper script
ace-container-test ace-git-worktree MT-WORKTREE-002

# Mounts
.cache/test-e2e/  -> /workspace/artifacts  (read-write)
project/          -> /workspace/src        (read-only, optional)
```

**Error Handling:**
- Container not running: Auto-start with helpful message
- Image missing: Pull or build with progress indicator
- Mount permissions: Clear error with resolution steps

## Objective

E2E tests that manipulate git repositories need true filesystem isolation. The current approach creates temporary directories on the host, which:
- Can leak test artifacts if cleanup fails
- Cannot be safely run in parallel
- Behaves differently on macOS vs Linux (CI)

Container-based isolation provides:
- Hermetic test environments
- Parallel execution safety
- Cross-platform consistency
- Simple cleanup (just remove container)

## Technical Notes

### Container Technology Options

**Lima (recommended)**
- Native macOS virtualization (Virtualization.framework)
- Light resource usage
- Supports containerd/nerdctl
- Free and open source

**OrbStack (alternative)**
- Drop-in Docker replacement for macOS
- Excellent performance
- Commercial product (free tier available)
- Simpler setup than Lima

### Volume Mounts

```yaml
# Lima config excerpt
mounts:
  - location: "~/.cache/test-e2e"
    writable: true
  - location: "~/Projects/ace-task"
    writable: false
```
