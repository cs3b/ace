# ace-mq: Markdown querying tool for precise content extraction

## Behavioral Specification

### User Experience
- **Input**: Users provide a Markdown file path and a query expression specifying what content to extract (e.g., specific headers, sections, code blocks, frontmatter fields)
- **Process**: The tool parses the Markdown structure, evaluates the query against the document hierarchy, and extracts matching content
- **Output**: Users receive structured data (JSON, YAML, or raw Markdown) containing only the queried content, ready for consumption by other tools or AI agents

### Expected Behavior

ace-mq provides jq/yq-like querying capabilities specifically designed for Markdown documents. Users can precisely extract content based on structural elements without processing entire documents or relying on fuzzy string matching.

**Core Capabilities:**
- Query by header levels (H1, H2, H3, etc.)
- Extract specific sections by title
- Filter content by block types (code blocks, lists, quotes)
- Access frontmatter fields with path expressions
- Combine queries for complex extractions
- Output in multiple formats for tool integration

**User Stories:**
- As an AI agent, I need to extract only the "Problem" section from a guide to reduce token usage
- As a developer, I want to query all H2 headers in documentation to build a table of contents
- As a workflow automation, I need to extract code blocks from a specific section for validation
- As a documentation tool, I want to retrieve frontmatter metadata for content indexing

### Interface Contract

```bash
# Basic section extraction
ace-mq query <file> --section "Problem"
# Output: Raw Markdown content under "Problem" header

# Query by header level
ace-mq query <file> --headers 2
# Output: JSON array of all H2 headers and their content

# Path-based queries (JSONPath-like for Markdown)
ace-mq query <file> --path '$.sections[?(@.level == 2)]' --output json
# Output: JSON array of sections at header level 2

# Extract code blocks from specific section
ace-mq query <file> --section "Implementation" --blocks code
# Output: All code blocks within "Implementation" section

# Frontmatter queries
ace-mq query <file> --frontmatter "title,status,estimate"
# Output: JSON object with specified frontmatter fields

# Combined queries
ace-mq query <file> --section "Behavioral Specification" --subsections --output yaml
# Output: YAML with section and all its subsections

# Multiple format outputs
ace-mq query <file> --headers 1 --output json    # JSON format
ace-mq query <file> --headers 1 --output yaml    # YAML format
ace-mq query <file> --headers 1 --output md      # Raw Markdown
ace-mq query <file> --headers 1 --output text    # Plain text (stripped)
```

**Error Handling:**
- Invalid query syntax: "Error: Invalid query expression at position X: expected..."
- Section not found: "Error: Section 'NonExistent' not found in document"
- Invalid file path: "Error: Cannot read file: <path> (file not found)"
- Malformed Markdown: "Warning: Skipping malformed header at line X"
- Empty results: Returns empty array/object with exit code 0 (not an error)

**Edge Cases:**
- Duplicate section names: Returns all matching sections in document order
- Nested sections with same title: Query path distinguishes by hierarchy
- Mixed header levels: Respects document structure, not sequential numbering
- Empty sections: Includes header but indicates no content
- No frontmatter: Frontmatter queries return empty object
- Large documents: Streams output for memory efficiency

### Success Criteria

- [ ] **Precise Content Extraction**: Users can query specific sections, headers, or blocks without receiving irrelevant content
- [ ] **Multiple Output Formats**: Supports JSON, YAML, raw Markdown, and plain text output formats
- [ ] **AI Agent Efficiency**: AI agents can reduce token usage by extracting only required sections (measurable: 50-90% reduction in irrelevant context)
- [ ] **Query Language Clarity**: Query syntax is intuitive and well-documented with clear examples
- [ ] **Deterministic Results**: Same query against same document always produces identical output
- [ ] **Error Messages**: Invalid queries or missing content produce clear, actionable error messages
- [ ] **Performance**: Queries against large documents (10MB+) complete in under 2 seconds
- [ ] **Integration Ready**: Output format allows seamless piping to jq, yq, and other CLI tools

### Validation Questions

- [ ] **Query Language Design**: Should we use JSONPath-like syntax, XPath-like syntax, or a custom Markdown-specific query language?
- [ ] **Section Matching**: How should the tool handle partial section name matches or case-sensitivity?
- [ ] **Hierarchy Representation**: How should nested sections be represented in JSON/YAML output?
- [ ] **Code Block Metadata**: Should code blocks include language tags and other metadata in structured output?
- [ ] **Streaming vs. Loading**: For very large files, should we support streaming queries or require full document loading?
- [ ] **Default Behavior**: When no output format is specified, what should the default be?
- [ ] **Integration with ace-docs**: Should ace-mq leverage ace-docs for frontmatter parsing or implement its own?

## Objective

Enable precise, programmatic extraction of content from Markdown documents, significantly improving AI agent efficiency and enabling advanced documentation automation within the ACE ecosystem.

**User Value:**
- Developers gain precise documentation querying capabilities
- AI agents reduce token usage by 50-90% through targeted content extraction
- Automation workflows can reliably extract specific content without parsing entire documents
- Documentation tools can build advanced features on top of structured Markdown querying
