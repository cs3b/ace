# Migrate skills to nested ace/* directory structure

## Description

Migrate `.claude/skills/` from flat `ace_*` directories to nested `ace/*/` subfolder structure once Claude Code supports nested skill directories.

**Current state** (Claude Code v2.1.7 does NOT support nested):
```
.claude/skills/
├── ace_lint/SKILL.md         # name: ace:lint
├── ace_commit/SKILL.md       # name: ace:commit
├── meta-manage-agents/SKILL.md  # name: meta-manage-agents
└── ...
```

**Target state** (when supported):
```
.claude/skills/
├── ace/
│   ├── lint/SKILL.md         # name: lint
│   ├── commit/SKILL.md       # name: commit
│   ├── meta/
│   │   └── manage-agents/SKILL.md  # name: meta/manage-agents
│   └── ...
├── ace-release/              # NOT migrated (hyphenated)
└── create-api-docs/          # NOT migrated (misc)
```

## Blocker

Claude Code does not currently support nested skill directories. Attempted migration on 2026-01-14 resulted in "Unknown skill" errors.

Monitor Claude Code release notes for nested skill directory support.

## Migration Scope

| Source | Count | Before | After |
|--------|-------|--------|-------|
| `ace_*` | 48 | `ace_lint/` → name: `ace:lint` | `ace/lint/` → name: `lint` |
| `meta-*` | 8 | `meta-manage-agents/` → name: `meta-manage-agents` | `ace/meta/manage-agents/` → name: `meta/manage-agents` |

**NOT migrating**: 5 `ace-*` hyphenated skills and 5 miscellaneous skills (stay as-is)

## Implementation Notes

**Migration script**: `codemods/migrate-skills.rb` (in this task directory)

```bash
# Dry run (preview changes)
ruby .ace-taskflow/_backlog/tasks/214-skills-refactor/codemods/migrate-skills.rb --dry-run

# Execute migration
ruby .ace-taskflow/_backlog/tasks/214-skills-refactor/codemods/migrate-skills.rb

# Verify
ls -la .claude/skills/ace/
ls -la .claude/skills/ace/meta/
```

The script:
- Derives new path from SKILL.md `name:` field (not directory name)
- Handles `ace_support_timestamp` → `ace/timestamp/` correctly
- Updates SKILL.md name field (removes `ace:` prefix)
- Reports errors and counts
