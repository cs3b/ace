# Create ace-support-yaml gem

## Objective

Create a new `ace-support-yaml` gem to consolidate YAML and frontmatter parsing across all ACE gems. This provides a single source of truth for YAML utilities.

## Scope

Consolidate YAML parsing from multiple locations:
- `ace-taskflow/atoms/yaml_parser.rb` → deprecate
- `ace-taskflow/atoms/safe_yaml_parser.rb` → move to ace-support-yaml
- `ace-support-markdown/atoms/frontmatter_extractor.rb` → move to ace-support-yaml

## Deliverables

### Create

- `ace-support-yaml/lib/ace/support/yaml/atoms/parser.rb`
- `ace-support-yaml/lib/ace/support/yaml/atoms/frontmatter_extractor.rb`

## Technical Notes

**FrontmatterExtractor API:**
```ruby
result = Ace::Support::Yaml::Atoms::FrontmatterExtractor.extract(content)
# => { frontmatter: {...}, body: "...", valid: true, errors: [] }
```

**SafeParser API:**
```ruby
Ace::Support::Yaml::Molecules::SafeParser.parse_with_recovery(content)
# Returns parsed YAML or best-effort recovery
```

**Important:** SafeParser should NOT include domain-specific validation (checking for `id`, `status` fields). The current ace-taskflow SafeYamlParser has taskflow-specific checks at lines 138-139 that must be removed when moving to the generic support gem. Domain validation belongs in ace-taskflow or ace-idea, not in a shared YAML utility.
