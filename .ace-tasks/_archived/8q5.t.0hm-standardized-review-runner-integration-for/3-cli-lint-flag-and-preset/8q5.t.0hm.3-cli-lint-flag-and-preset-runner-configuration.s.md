# CLI Lint Flag and Preset Runner Configuration

## Behavioral Specification

### User Experience
- **Input**: User passes `--lint` flag on CLI, or configures `runners:` in preset YAML
- **Process**: CLI flag and preset config are merged through configuration cascade into ReviewOptions, which tells ReviewManager which runners to activate
- **Output**: Review executes with the requested runner combination

### Expected Behavior

Two ways to enable runners:

1. **CLI flag** `--lint`: Ad-hoc enablement of the lint runner for any preset
2. **Preset config** `runners:` key: Persistent per-preset runner configuration

The `--lint` flag adds a LintRunner to whatever runners are already configured. If a preset already has lint in its `runners:` config, `--lint` is a no-op (idempotent).

ReviewOptions gains a `runners` attribute that holds the resolved runner configuration. The configuration cascade applies: CLI flags override preset config.

### Interface Contract

```bash
# CLI usage
ace-review --preset code-deep --lint --pr 123
ace-review --lint                    # uses default preset + lint
ace-review --preset code-deep        # no lint (unless preset configures it)

# Preset configuration
# .ace/review/presets/comprehensive.yml
runners:
  - type: llm
  - type: lint
    config:
      format: json

# Default config (no runners key = LLM only)
# .ace-defaults/review/config.yml
# (no runners key needed - LLM is implicit default)
```

**Error Handling:**
- `--lint` but ace-lint not installed: Warning at CLI level, review proceeds without lint
- Unknown runner type in preset config: Warning logged, unknown runner skipped

**Edge Cases:**
- `--lint` with a preset that already configures lint runner: Idempotent, no duplicate execution
- `--no-lint` to explicitly disable lint in a preset that enables it: Not needed for v1, but flag structure supports it
- Empty `runners:` array in preset: Treated as "no runners", falls back to default LLM
