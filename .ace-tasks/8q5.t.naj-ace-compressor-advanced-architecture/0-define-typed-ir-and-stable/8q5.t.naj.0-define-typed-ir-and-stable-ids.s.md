# Add stable IDs to exact output

## Behavioral Specification

### User Experience
- **Input**: The same doc compressed multiple times in exact mode.
- **Process**: The CLI assigns stable semantic IDs to files, sections, rules, facts, tables, and examples while preserving the current exact-mode `ContextPack/3` layout.
- **Output**: Exact output with IDs stable enough for future patching.

### Expected Behavior
This slice makes stable IDs observable from the live CLI. IDs must not be based on line numbers; whitespace-only or frontmatter-only edits keep the same identities.

### Interface Contract
```bash
ace-compressor compress docs/vision.md --mode exact --verbose
```

**Error Handling:**
- ID churn after formatting-only change is a failing invariant.
- Unsupported content lacking stable identity still emits deterministic fallback IDs.

**Edge Cases:**
- Heading punctuation changes do not rotate section IDs.
- Frontmatter edits do not rotate semantic unit IDs.

### Success Criteria
- [ ] Exact output exposes stable IDs for core unit types without regressing current exact consumers that rely on `FILE|`, `SEC|`, and typed record ordering.
- [ ] IDs survive formatting-only edits.
- [ ] At least one automated repeat-run test proves stability.
- [ ] IDs are visible enough to support the next slice.

### Validation Questions
- [x] Stable IDs are runtime-visible in this slice.
- [x] Positional IDs are not acceptable.
- [x] Fallback IDs may exist for unsupported structures.
- [x] This slice prepares patch generation directly.
