# ace-compressor: advanced architecture

## Behavioral Specification

### User Experience
- **Input**: Confirmed learnings from exact and compact mode plus the advanced architecture concept doc.
- **Process**: The task defines the long-term internal contracts required for patchable, provenance-aware, typed compression.
- **Output**: Decision-complete architecture for typed IR, stable IDs, `PatchPack/1`, provenance, deduplication, and validation strategy.

### Expected Behavior
This task is specification-only. It does not deliver new runtime behavior directly. Instead, it defines the contracts that future implementation tasks must satisfy so the compressor can support stable identifiers, compact provenance, incremental updates, and deterministic validation without re-deciding core architecture later.

### Interface Contract
```text
ContextPack/1
FILE|id=docs/vision.md|type=vision
RULE|id=rule:cli_first|modality=must|action=be developer friendly
P|id=rule:cli_first|src=docs/vision.md#core-principles

PatchPack/1
PATCH|base=sha256:...
R|id=rule:cli_first|value=...
D|id=example:legacy_cli
```

**Error Handling:**
- Stable-ID design depends on line numbers: reject the design.
- Patch design without provenance or base compatibility: reject the design.

**Edge Cases:**
- Whitespace-only or frontmatter-only edits must not rotate stable IDs.
- Multiple near-duplicate facts across files keep one canonical unit plus provenance list.

### Success Criteria
- [ ] Typed intermediate representation is decision-complete.
- [ ] Stable-ID rules survive formatting-only edits.
- [ ] `PatchPack/1` is ID-addressed rather than line-addressed.
- [ ] Provenance, deduplication, and validation strategy are locked for future implementation.

### Validation Questions
- [x] This task is spec-only and does not require code implementation.
- [x] Stable IDs must survive whitespace and frontmatter-only changes.
- [x] Patches target IDs, not line numbers.
- [x] Provenance remains mandatory even when compact.
