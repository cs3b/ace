# Emit single-file delta patches

## Behavioral Specification

### User Experience
- **Input**: A previous exact baseline and a changed single source file.
- **Process**: The CLI compares stable IDs and emits a minimal `PatchPack/1` with add/replace/delete operations.
- **Output**: A runnable patch-generation path for one-file updates.

### Expected Behavior
This slice must produce a smaller update artifact than a full resend for normal edits. It starts narrow on purpose: one changed file against one prior baseline.

### Interface Contract
```bash
ace-compressor patch prev.contextpack docs/vision.md --mode exact
```

**Error Handling:**
- Missing baseline: exit non-zero.
- Incompatible baseline digest: exit non-zero with compatibility guidance.

**Edge Cases:**
- No content change may emit an empty/no-op patch.
- One changed fact may emit a single `R|...` patch operation.

### Success Criteria
- [ ] `PatchPack/1` is emitted for one-file updates.
- [ ] Patch operations reference stable IDs.
- [ ] No-op changes do not emit noisy full patches.
- [ ] Automated tests compare patch size against full resend for simple edits.

### Validation Questions
- [x] This slice starts with one-file updates only.
- [x] Base-compatibility checks are mandatory.
- [x] ID-addressed patches are the product, not unified diff.
- [x] No-op patch behavior is in scope.
