# ace-sim README and Getting Started

## Objective

Rewrite ace-sim's README from a 28-line bare stub into a proper landing page. The current README shows only two CLI examples and a reference to docs/usage.md with no explanation of what simulations are, why they matter, or how the tool works. Meanwhile docs/usage.md (79 lines) contains preset-driven usage, override flags, synthesis details, artifact structure, and behavior notes. Create a tutorial-style getting-started.md. Delete docs/usage.md.

- **Input**: Current ace-sim/README.md (28 lines), docs/usage.md (79 lines)
- **Process**: Rewrite README as landing page, create getting-started tutorial, write VHS tape, delete usage.md

### Expected Behavior

New README should convey: "Run multi-provider LLM simulations to validate ideas and tasks -- get diverse AI perspectives before committing to a plan."

Key messaging:
- Run chained LLM simulations across multiple providers for diverse perspectives
- Built-in presets for idea validation and task review
- File-chained steps (draft, plan, work) build on each other
- Final synthesis aggregates all chains into actionable suggestions

Current README problems to fix:
- No explanation of what a "simulation" means in this context
- No value proposition or "Why" section
- Two raw CLI examples without narrative
- "See docs/usage.md for full usage" is not a landing page
- No installation instructions
- No features overview
