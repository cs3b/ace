# Pilot restarted E2E architecture on ace-b36ts and sync testing docs

## Objective

Prove the restarted testing architecture on ace-b36ts, using one package to validate all of the following at once:
- deterministic sandboxed Minitest in test/e2e
- scenario-only ace-test-e2e
- shared sandbox helper reuse
- ace-test and ace-test-suite target-based execution
- updated docs, workflows, and skills for the new structure

## Behavioral Specification

### User Experience

- A developer can run deterministic sandboxed ace-b36ts coverage with ace-test ace-b36ts e2e.
- A developer can run the package’s one agent scenario with ace-test-e2e ace-b36ts.
- Testing docs and workflows describe one clear split:
  - test/e2e for deterministic Minitest
  - test-e2e/scenarios for agent scenarios
- ace-test-suite --target e2e becomes the monorepo entrypoint for deterministic sandboxed E2E-style Minitest.

### Expected Behavior

1. Add e2e as a first-class ace-test and ace-test-suite target backed by test/e2e/**/*_test.rb.
2. Add all-with-e2e as a package-level ace-test-runner config-only group alias while keeping default all excluding e2e. Do not add a suite-level alias; ace-test-suite only adds --target e2e pass-through.
3. Extract one shared sandbox package-copy helper into ace-support-test-helpers.
   - helper creates sandboxes under .ace-local/test-e2e/
   - helper copies the target package into the sandbox and returns both sandbox root and sandbox package root
   - helper owns PROJECT_ROOT_PATH=sandbox root and ACE_E2E_SOURCE_ROOT=repo root
   - helper is reusable from ordinary test/e2e Minitest and from ace-test-e2e scenario setup
4. Migrate all deleted deterministic ace-b36ts TCs into ace-b36ts/test/e2e/ts_b36ts_001_test.rb:
   - TC-001-help-survey
   - TC-002-encode-today
   - TC-003-decode-token
   - TC-004-error-behavior
   - TC-005-output-routing
   - TC-006-structured-output
   - TC-007-roundtrip-pipeline
   - TC-008-batch-sort
5. Make ace-test-e2e scenario-only and remove deterministic phase execution from its runtime contract.
6. Add one real ace-b36ts agent scenario:
   - agent receives docs and --help
   - agent is given five dated notes
   - agent must rename notes with correct ace-b36ts prefixes, reorganize them into year/month/week, and return a final reflection
   - only the final response is saved as scenario-owned text output
   - verification is based on final response plus resulting filesystem state
7. Update global testing docs, workflows, and skills to the new split.
   - ace-test-runner README and docs/usage for e2e target and all-with-e2e config alias
   - ace-test-runner-e2e README, docs/usage, handbook/guides/e2e-testing.g.md, and workflow instructions under handbook/workflow-instructions/e2e/
   - ace-test-runner-e2e skills that instruct scenario creation, execution, or repair
   - ace-assign README/docs references to routine test execution
   - ace-handbook testing/release references that currently point readers only at ace-test-e2e flows

### Interface Contract

- ace-test ace-b36ts e2e
- ace-test-suite --target e2e
- ace-test-e2e ace-b36ts

Success rules:
- ace-test resolves the e2e target to test/e2e/**/*_test.rb and default all excludes e2e
- ace-test-suite adds --target and passes e2e through to package-level ace-test without inventing a separate suite alias
- ace-test-e2e discovers and runs only test-e2e/scenarios
- the shared helper is the only place that defines sandbox package-copy setup and environment ownership
- the ace-b36ts scenario uses docs/help plus a job description, not a command recipe

### Success Criteria

- [ ] ace-test ace-b36ts e2e passes.
- [ ] ace-test-e2e ace-b36ts passes.
- [ ] ace-test-suite --target e2e works for the new deterministic target.
- [ ] ace-b36ts deterministic coverage fully replaces the deleted 8 TC runner cases.
- [ ] ace-b36ts agent scenario follows the new final-response-only model.
- [ ] Global testing docs, workflows, and skills consistently describe the restarted structure and commands.
- [ ] The shared sandbox helper is owned outside ace-test-runner-e2e.
- [ ] The pilot updates every currently affected handbook/doc surface that still describes deterministic phase execution or the old scenario contract.
