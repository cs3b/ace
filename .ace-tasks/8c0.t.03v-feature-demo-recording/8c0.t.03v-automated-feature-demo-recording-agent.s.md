# Automated CLI Demo Recording

## Behavioral Specification

### User Experience
- **Input**: VHS `.tape` script (path or preset name), optional PR number
- **Process**: Resolve tape file via config cascade, run VHS to produce recording, optionally attach to PR
- **Output**: GIF/MP4/WebM artifact at `.ace-local/demo/<name>.gif`, optionally posted as PR comment

### Expected Behavior

A CLI tool (`ace-demo`) that records terminal command demos using VHS tape scripts and attaches the resulting GIF artifacts to GitHub pull requests. This enables AI agents and developers to provide visual proof of feature functionality directly in PRs.

The system should:
- Run VHS `.tape` scripts to produce deterministic CLI recordings
- Resolve tape files via config cascade (`.ace-defaults/demo/tapes/` → `~/.ace/demo/tapes/` → `.ace/demo/tapes/` → direct path)
- Output recordings to `.ace-local/demo/` by default
- Upload GIF artifacts to GitHub releases for stable URLs
- Post PR comments with inline demo GIFs
- Ship default demo tapes and make them discoverable

### Interface Contract

```bash
# Record a demo from a tape file
ace-demo record <tape-or-preset> [--output path] [--format gif|mp4|webm]

# Attach an existing demo to a PR
ace-demo attach <file> --pr <number>

# Record and attach in one step
ace-demo record <tape> --pr <number>

# Discover available tapes
ace-demo list
ace-demo show <name>

# Examples:
ace-demo record hello                           # Record preset tape → .ace-local/demo/hello.gif
ace-demo record ./custom.tape --format mp4      # Record custom tape as MP4
ace-demo record ace-test --pr 123               # Record and attach to PR
ace-demo attach .ace-local/demo/hello.gif --pr 45  # Attach existing GIF to PR
ace-demo list                                   # Show available tapes with descriptions
ace-demo show hello                             # Display tape file contents

# Expected outputs:
# Success: "Recorded: .ace-local/demo/hello.gif"
#          "Attached demo to PR #123"
# Error:   "VHS not found. Install: https://github.com/charmbracelet/vhs"
#          "Tape not found: missing.tape"
```

**Error Handling:**
- VHS binary not found: Report with install instructions
- Tape file not found: Search cascade paths, report searched locations
- VHS execution fails: Capture stderr, report with tape path context
- PR upload fails: Show `gh` CLI errors, suggest manual upload
- GIF too large: Warn with file size, suggest `--format webm`

**Edge Cases:**
- Tape name without extension: Try `<name>.tape` in cascade paths
- Output directory missing: Create `.ace-local/demo/` automatically
- PR does not exist: Fail with clear message (do not create draft PR)
- `gh` CLI not authenticated: Report auth status, suggest `gh auth login`

### Success Criteria

- [ ] **VHS Recording**: `.tape` files produce GIF artifacts via VHS
- [ ] **Config Cascade**: Tapes discovered from `.ace-defaults/`, user, and project paths
- [ ] **PR Attachment**: Demo GIFs uploaded and embedded in PR comments
- [ ] **Tape Discovery**: `list` and `show` commands find and display available tapes
- [ ] **Error Handling**: Clear messages for missing VHS, missing tapes, failed recordings
- [ ] **Deterministic Output**: Same tape produces consistent recordings across runs

### Validation Questions (Resolved)

- [x] **Recording Tool**: VHS — deterministic `.tape` scripts, CI-friendly, headless, produces GIF/MP4/WebM
- [x] **Demo Script Format**: Native VHS `.tape` files — leverage VHS ecosystem directly
- [x] **Artifact Storage**: GitHub release assets (`demo-assets` release, created on demand) — stable URLs via `gh`
- [x] **Gem Structure**: Standalone `ace-demo` gem — different concern and dependencies from ace-tmux

## Objective

Enable CLI demo recordings via VHS tape scripts with PR attachment. Bridge the gap between passing tests and showing tangible feature behavior, improving PR review quality with minimal infrastructure overhead.

## Scope of Work

- **Gem**: New `ace-demo` gem following standard ace-* structure
- **Recording**: VHS tape execution producing GIF/MP4/WebM output
- **Discovery**: Config cascade for tape files, `list`/`show` commands
- **PR Integration**: Upload to GitHub release assets, post PR comments with inline images
- **CLI**: `record`, `attach`, `list`, `show` commands via dry-cli

### Subtask Decomposition

1. **VHS Recording Spike** — End-to-end: resolve tape → run VHS → produce GIF. Gem scaffold.
2. **PR Demo Attachment** — Upload GIF to GitHub release, post PR comment with inline image.
3. **Demo Tape Library & Discovery** — Ship default tapes, `list`/`show` commands, metadata parsing.

Subtasks 2 and 3 can be built in parallel after Subtask 1.

### Deliverables

#### Implementation (ace-demo gem)
- VHS command builder atom and executor molecule
- Tape resolver with config cascade
- Demo recorder orchestrator
- GIF upload and PR comment posting
- Tape scanner and metadata parser
- CLI commands: record, attach, list, show

#### Default Content
- `hello.tape` — minimal echo demo
- `ace-test.tape` — shows ace-test run

## Out of Scope

- VM/container provisioning or environment management
- Feature deployment automation
- GUI or browser recording (terminal only)
- Video editing or post-processing
- Live streaming or interactive demos
- Custom recording DSL (use native VHS `.tape` format)
