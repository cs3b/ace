# Demo Tape Library and Discovery

## Behavioral Specification

### User Experience
- **Input**: `ace-demo list` or `ace-demo show <name>`
- **Output**: List of available demo tapes with descriptions, or detail view of a specific tape

### Expected Behavior

Ship default demo tapes and make them discoverable. Tapes are scanned from config cascade paths, metadata is extracted from VHS comments, and users can list available tapes or view their contents.

The system should:
- Scan all config cascade paths for `.tape` files
- Extract metadata from VHS comments (e.g., `# Description: ...`, `# Tags: ...`)
- Display a summary list with tape name, source path, and description
- Show full tape contents with metadata for a specific tape
- Ship default tapes in `.ace-defaults/demo/tapes/`

### Interface Contract

```bash
# List all available tapes
ace-demo list
# → Available demo tapes:
# →   hello          Built-in echo demo                    (.ace-defaults/demo/tapes/)
# →   ace-test       Shows ace-test run                    (.ace-defaults/demo/tapes/)
# →   my-feature     Custom feature demo                   (.ace/demo/tapes/)

# Show a specific tape
ace-demo show hello
# → Tape: hello
# → Source: .ace-defaults/demo/tapes/hello.tape
# → Description: Built-in echo demo
# →
# → --- Contents ---
# → Output .ace-local/demo/hello.gif
# → Set FontSize 14
# → Set Width 800
# → Set Height 400
# → Type "echo 'Hello from ace-demo!'"
# → Enter
# → Sleep 2s

# Tape not found
ace-demo show nonexistent
# → Error: Tape not found: nonexistent
#          Available tapes: hello, ace-test
```

### ATOM Component Design

| Layer | Component | Responsibility |
|-------|-----------|---------------|
| Atom | `TapeMetadataParser` | Extracts VHS comments as metadata hash — parses `# Description:`, `# Tags:`, etc. (pure function) |
| Molecule | `TapeScanner` | Scans config cascade paths for `.tape` files, returns list with resolved paths and parsed metadata |
| CLI | `List` | dry-cli command displaying tape summary table |
| CLI | `Show` | dry-cli command displaying tape detail with contents |

### Config Cascade

Search order for `.tape` files:
1. `.ace-defaults/demo/tapes/` — built-in defaults shipped with the gem
2. `~/.ace/demo/tapes/` — user-level tapes
3. `.ace/demo/tapes/` — project-level tapes
4. Direct file path — explicit path to a `.tape` file

Later entries override earlier ones with the same name (project > user > defaults).

### Default Tapes


### Metadata Comment Convention

VHS comments at the top of `.tape` files serve as metadata:
```
# Description: One-line description of what the tape demonstrates
# Tags: comma, separated, tags
# Author: name
```

Only `# Description:` is used in `list` output. Other fields are optional and shown in `show` detail view.

### Success Criteria

- [ ] **Tape Scanning**: Discovers tapes from all config cascade levels
- [ ] **Metadata Parsing**: Extracts `# Description:` and other comment-based metadata
- [ ] **List Output**: Shows tape name, description, and source path for all discovered tapes
- [ ] **Show Output**: Displays tape metadata and full file contents
- [ ] **Override Semantics**: Project tapes override user tapes override defaults with same name
- [ ] **Default Tapes**: `hello.tape` and `ace-test.tape` ship in `.ace-defaults/demo/tapes/`

### Verification Plan

- **Unit tests**: `TapeMetadataParser` — pure parsing of comment lines, no stubs needed
- **Stubbed molecule tests**: `TapeScanner` — stub filesystem paths, verify cascade discovery order and override behavior
- **CLI tests**: Verify `list` and `show` output formatting
