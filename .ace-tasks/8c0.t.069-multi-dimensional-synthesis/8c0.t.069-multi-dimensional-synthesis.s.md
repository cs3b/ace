# Weighted Multi-Dimensional Synthesis

## Goal

Activate the dormant reviewer metadata (weight, critical, focus) so that multi-reviewer presets produce weighted consensus, surface critical findings prominently, and group output by focus area.

## Current State

The Reviewer model already stores `weight` (0.0–1.0), `critical` (boolean), and `focus` (string) attributes, and `Reviewer.from_preset_config()` can parse them from preset YAML. However:

- **PresetManager** loads preset config but never instantiates Reviewer objects
- **ReviewManager** creates Reviewers via `from_preset_config` but does not pass them through the execution chain
- **MultiModelExecutor** receives a flat `models:` array of strings — no Reviewer objects, no metadata
- **FeedbackSynthesizer** uses a hardcoded `CONSENSUS_THRESHOLD = 3` (simple count) — weights are ignored
- **FeedbackItem** has no `confidence`, `focus`, or `critical_source` fields — no way to carry metadata downstream

The result: multi-reviewer presets execute in parallel and synthesize findings, but all reviewers are treated identically regardless of their configured weight, criticality, or focus area.

## Concept Inventory

| Concept | Description | Introduced In |
|---------|-------------|---------------|
| Reviewer pipeline | Thread Reviewer objects through PresetManager → ReviewOptions → ReviewManager → MultiModelExecutor → FeedbackSynthesizer | 069.a |
| Weighted consensus | Replace count-based threshold with weight-aware confidence scoring | 069.b |
| Confidence score | Float 0.0–1.0 on FeedbackItem computed from reviewer count × weights × agreement ratio | 069.b |
| Critical surfacing | Fast-path for findings from `critical: true` reviewers to appear first in output | 069.c |
| Focus-area grouping | Group synthesized feedback by reviewer `focus` when metadata is available | 069.c |

## Subtasks

- **069.a**: Connect reviewer pipeline to execution chain
  - Wire Reviewer objects through the full pipeline so metadata is available at each stage
  - Foundation for 069.b and 069.c — no behavioral change to output yet
  - Advisory size: medium

- **069.b**: Weight-aware consensus and confidence scoring
  - Add `confidence` field to FeedbackItem
  - Replace hardcoded consensus threshold with configurable weighted threshold
  - Depends on 069.a
  - Advisory size: medium

- **069.c**: Critical finding surfacing and focus-area grouping
  - Tag findings from `critical: true` reviewers, surface them first
  - Group feedback listing by `focus` area when metadata available
  - Depends on 069.a, parallelizable with 069.b
  - Advisory size: small

## Acceptance Criteria

- [ ] Reviewer objects with weight/critical/focus flow through the full execution pipeline
- [ ] FeedbackItem carries confidence score derived from reviewer weights
- [ ] Consensus detection uses weighted threshold instead of hardcoded count
- [ ] Critical reviewers' findings are surfaced prominently in output
- [ ] Focus areas group feedback when metadata is available
- [ ] Single-reviewer presets still work unchanged (backward compatible)
- [ ] All existing tests pass without modification

## Out of Scope

- Map-reduce condensation (separate task when synthesis token limits are hit in production)
- Per-reviewer subject filtering (already works via SubjectFilter)
- Optimal weight algorithms or second-order agreement patterns
- Reviewer reliability tracking over time
