# VHS Recording Spike

## Behavioral Specification

### User Experience
- **Input**: `.tape` file path or preset name
- **Output**: GIF file at `.ace-local/demo/<name>.gif`
- **CLI**: `ace-demo record <tape-or-preset> [--output path] [--format gif|mp4|webm]`

### Expected Behavior

End-to-end validation: resolve a VHS tape file via config cascade, execute VHS to produce a recording, and return the output path. This subtask scaffolds the `ace-demo` gem and proves the core recording pipeline works.

The system should:
- Resolve tape files by name through config cascade paths (`.ace-defaults/demo/tapes/`, `~/.ace/demo/tapes/`, `.ace/demo/tapes/`, direct path)
- Build correct VHS command arrays from tape path and options
- Execute VHS via `Open3.capture3` with proper error capture
- Detect and report missing VHS binary with install instructions
- Output recordings to `.ace-local/demo/` by default
- Support `--format` option for gif, mp4, or webm output

- [ ] **VHS Detection**: Reports missing VHS binary with install link when not found

- [ ] **Error Messages**: Clear, actionable messages for all failure modes
