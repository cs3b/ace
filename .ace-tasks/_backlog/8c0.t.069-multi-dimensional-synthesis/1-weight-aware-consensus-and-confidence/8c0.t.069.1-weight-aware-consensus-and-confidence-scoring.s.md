# Weight-Aware Consensus and Confidence Scoring

## Goal

Replace the hardcoded consensus threshold (count >= 3) with a weight-aware confidence score, so that findings from higher-weight reviewers carry proportionally more influence in consensus detection.

## Current State

- `FeedbackSynthesizer` uses `CONSENSUS_THRESHOLD = 3` — consensus is true when 3+ reviewers report the same finding
- All reviewers contribute equally regardless of their configured `weight` (0.0–1.0)
- `FeedbackItem` has a boolean `consensus` field but no numeric confidence score
- Reviewer weights exist in the model but are never consulted during synthesis

## Behavioral Spec

### Input → Process → Output

**Input**: Synthesized findings with reviewer associations and weight metadata (available after 069.0).

**Process**: For each FeedbackItem, compute a confidence score from the weighted sum of agreeing reviewers relative to total available weight. Use this score (instead of raw count) to determine consensus.

**Output**: Each FeedbackItem carries a `confidence` float (0.0–1.0). Consensus is true when confidence meets the configurable threshold.

### Interface Contract

**FeedbackItem** gains a `confidence` field:

- Type: Float, range 0.0–1.0
- Default: nil (unscored, for backward compatibility with items created without reviewer metadata)
- Serialized in `to_h` output when present
- Immutable after creation (consistent with existing FeedbackItem pattern)

**Confidence computation**:

```
confidence = sum(weight for each agreeing reviewer) / sum(weight for all available reviewers)
```

- When all reviewers have weight 1.0, this equals `agreeing_count / total_count` (graceful degradation to ratio)
- When reviewer metadata is absent, confidence is nil and consensus falls back to count-based threshold (backward compatible)

**Consensus threshold**:

- New configurable constant: `CONSENSUS_CONFIDENCE_THRESHOLD` (default: 0.6)
- Consensus is true when `confidence >= CONSENSUS_CONFIDENCE_THRESHOLD`
- Existing `CONSENSUS_THRESHOLD = 3` remains as the fallback when weights are unavailable
- Threshold can be overridden via preset config key `consensus_threshold`

### Defaults

- `CONSENSUS_CONFIDENCE_THRESHOLD = 0.6` (equivalent to "majority of weight agrees")
- When no reviewer weights are available, fall back to existing count-based logic
- FeedbackItems created without confidence are valid (nil confidence, boolean consensus preserved)

## Success Criteria

- [ ] FeedbackItem has `confidence` attribute (Float, 0.0–1.0, nilable)
- [ ] Confidence is computed from weighted reviewer agreement ratio
- [ ] Consensus detection uses confidence threshold when weights are available
- [ ] Fallback to count-based consensus (CONSENSUS_THRESHOLD = 3) when no weights
- [ ] Consensus threshold is configurable via preset config
- [ ] FeedbackItem `to_h` includes confidence when present
- [ ] All existing tests pass (backward compatible)

## Verification Plan

### Confidence Calculation Check
- Test: 2 reviewers (weight 1.0 each) both agree → confidence = 1.0
- Test: 3 reviewers (weights 1.0, 0.5, 0.5), 2 agree (weights 1.0, 0.5) → confidence = 0.75
- Test: Single reviewer → confidence = 1.0 (trivial agreement)
- Test: No reviewer metadata → confidence = nil, consensus uses count fallback

### Threshold Check
- Test: confidence 0.7 with default threshold 0.6 → consensus = true
- Test: confidence 0.4 with default threshold 0.6 → consensus = false
- Test: custom threshold 0.8 → confidence 0.75 yields consensus = false

### Backward Compatibility Check
- Test: FeedbackItem created without confidence field → valid, consensus from boolean
- Test: Legacy preset (no reviewer weights) → count-based consensus unchanged
- Run full test suite — zero regressions
