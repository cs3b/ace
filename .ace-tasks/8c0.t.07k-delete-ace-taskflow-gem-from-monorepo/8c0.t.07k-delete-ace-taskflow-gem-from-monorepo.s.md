# Delete ace-taskflow Gem from Monorepo

## Objective

Remove the ace-taskflow gem entirely from the monorepo after all domain extractions (290, 291, 292) are complete and data migration (293) is verified. ace-taskflow becomes an empty shell with no remaining purpose — everything has moved to standalone gems.

## Behavioral Specification

### Expected Behavior

Complete removal of ace-taskflow:

1. **Verify prerequisites**: Confirm all three domain gems are working independently, data migration is verified, no other gems depend on ace-taskflow
2. **Remove gem directory**: Delete `ace-taskflow/` entirely (lib, test, exe, handbook, config, gemspec)
3. **Remove from monorepo config**: Update `Gemfile`, mise config, any CI references
4. **Remove .ace-taskflow/ data directory**: Delete `.ace-taskflow/` after confirming all data is in `.ace-tasks/`, `.ace-ideas/`, `.ace-retros/`
5. **Update cross-references**: Any CLAUDE.md, README, or config files that reference ace-taskflow

### What gets deleted

**ace-taskflow/ gem directory** (entire tree):
- `lib/ace/taskflow/` — all 11 atoms, 46 molecules, 12 organisms, 3 models, 12 commands, 34 CLI files
- `test/` — all 85 test files
- `handbook/` — remaining release workflows (2), release guides (5+), release templates (~15), handbook README
- `exe/ace-taskflow`, `exe/ace-release` — obsolete executables
- `.ace-defaults/taskflow/` — old config and presets
- `ace-taskflow.gemspec`, `Gemfile`, `Rakefile`, etc.

**Note**: By this point, all valuable code has already been git-moved or adapted into ace-task, ace-idea, ace-retro. What remains is:
- Release-specific code (dropped — release concept gone): ~25 files
- Old multi-domain CLI shell: ~10 files
- Shared utilities already duplicated into domain gems: ~30 files
- Release-scoped handbook content: ~20 files

**.ace-taskflow/ data directory**:
- `.ace-taskflow/v.0.9.0/tasks/` — migrated to `.ace-tasks/` in task 293
- `.ace-taskflow/v.0.9.0/ideas/` — migrated to `.ace-ideas/` in task 293
- `.ace-taskflow/v.0.9.0/retros/` — migrated to `.ace-retros/` in task 293
