# Resolve intent into catalog and skill phases

## Behavioral Specification

### User Experience
- **Input**: Users provide explicit requested steps, known recipes, or high-level intents such as `run tests, reorganize commits, push to remote`.
- **Process**: Smart create resolves phrases to phase definitions and skill-backed workflows, uses catalog rules as guidance, and preserves explicit user intent unless a hard ordering or prerequisite rule must override the requested order.
- **Output**: Users receive an assignment whose phases reflect what they asked for, without needing to understand internal catalog or metadata structures.

### Expected Behavior

This slice makes smart create genuinely useful beyond one tracer path:

1. Catalog phases can expose phrase-matching hints so explicit step requests resolve deterministically.
2. Skill-backed phases can still expand from `assign.source` metadata when the user requests a higher-level capability such as task work.
3. Catalog rules remain advisory: they add prerequisites, ordering fixes, and defaults, but they do not override clear user intent without an explicit rule reason.
4. Common explicit requests used in this redesign are resolvable, including tests, reorganize commits, push, changelog squashing, and rebasing with main.
5. The created phase order is explainable from the user's request plus a small set of hard ordering rules.

### Interface Contract

```bash
/as-assign-create "run tests, reorganize commits, push to remote"
/as-assign-create "squash changelog, rebase with origin main, update pr description"
/as-assign-create "work on task 123 and create a PR"
```

**Error Handling:**
- Unknown explicit phrase: return the phrase that failed and the closest catalog/skill candidates.
- Conflicting explicit order: preserve intent where possible, but state the hard rule that changed ordering.

**Edge Cases:**
- Explicit duplicate steps are normalized into one phase unless the request clearly asks for repetition.
- A skill-backed intent may expand into multiple phases while still appearing as one user request.
- Explicit phrases take precedence over recipe defaults when both are present.

### Success Criteria

- [ ] **Explicit step matching works**: Common requested phrases resolve into expected phases.
- [ ] **Skill-backed expansion survives**: High-level capabilities still expand through `assign.source` metadata where defined.
- [ ] **Guidance stays advisory**: Catalog rules help shape assignments without turning create into a rigid preset-only flow.
- [ ] **Ordering is explainable**: Reordered outputs can be justified by named hard rules rather than opaque heuristics.

### Validation Questions

- [x] **Should catalog control composition strictly?** -> No; it guides and validates, but explicit input is primary.
- [x] **Should skill metadata remain part of the resolution story?** -> Yes; retain `assign.source` expansion for richer phases.
