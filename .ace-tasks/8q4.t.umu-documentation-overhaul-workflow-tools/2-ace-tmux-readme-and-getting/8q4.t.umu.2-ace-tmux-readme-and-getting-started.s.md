### Expected Behavior

New README should convey: "Composable tmux session management via YAML presets -- add windows on the fly without restarting anything."

Key messaging:
- Dynamic windows: add a window to a running session without restarting (no other tmux manager does this)
- Composable at every level: pane presets compose into window presets, which compose into session presets
- Nested layouts: go beyond tmux's five built-in layouts with arbitrary YAML-defined split trees
- Config cascade: project, user, and gem-default presets with deep-merge override

Current README problems to fix:
- Full YAML preset example with nested layout is guide-level content (good for getting-started, too detailed for landing page)
- ASCII art layout diagram is tutorial material, not landing page material
- Links to docs/usage.md which should be deleted (reference info belongs in --help)
- Missing demo GIF and benefit-oriented feature bullets
