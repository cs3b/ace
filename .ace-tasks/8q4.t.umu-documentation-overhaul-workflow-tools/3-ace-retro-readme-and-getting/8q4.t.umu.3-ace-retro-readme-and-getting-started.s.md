# ace-retro README and Getting Started

## Objective

Rewrite ace-retro's README from 99-line reference manual into a landing page. The current README lists implementation details (raw b36ts IDs, folder-based storage patterns) before user value, then provides exhaustive CLI usage examples for all 5 commands and a full Ruby API section with code samples. The frontmatter schema section duplicates --help content. Create a tutorial-style getting-started.md.

## Behavioral Specification

### User Experience
- **Input**: Current ace-retro/README.md (99 lines)
- **Process**: Rewrite README as landing page, create getting-started tutorial, write VHS tape, create usage.md + handbook.md, update gemspec, audit handbook
- **Output**: Rewritten README.md (~60 lines), new docs/getting-started.md, new VHS tape + GIF committed to docs/demo/, new docs/usage.md, docs/handbook.md, updated gemspec

### Expected Behavior

New README should convey: "Lightweight retrospective management -- create, tag, and archive retros from the command line."

Key messaging:
- Simple 5-command pattern (create, show, list, move, update) for managing retrospectives
- Multiple retro types (standard, conversation-analysis, self-review) for different reflection needs
- Task linking connects retros to the work they reflect on
- Archive support with date partitioning keeps your workspace clean

Current README problems to fix:
- Features section leads with implementation details (b36ts IDs, folder storage patterns) instead of user benefits
- CLI Usage section is a complete command reference that duplicates --help
- Ruby API section with full code samples is developer reference, not landing page content
- Frontmatter schema section shows raw YAML that belongs in --help or getting-started
