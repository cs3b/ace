# Add smart auto-sort with B36TS position pinning to ace-task

## Behavioral Context

**Issue**: After extracting ace-task from ace-taskflow, the remaining feature gap was item ordering. ace-taskflow had float-based sort values requiring rebalancing. A cleaner design was needed using smart auto-sort as default with B36TS-based position overrides.

**Key Behavioral Requirements**:
- Tasks should sort by computed score (priority weight x age) by default
- In-progress tasks should float to the top, blocked tasks to the bottom
- Users can pin tasks to specific positions using B36TS values
- Backward compatibility via `--sort id` for chronological ordering

## Objective

Implemented smart auto-sort with B36TS position pinning across ace-support-items and ace-task.

## Scope of Work

- Added `SortScoreCalculator` atom for computing sort scores
- Added `PositionGenerator` atom for B36TS position value generation
- Added `SmartSorter` molecule for pinned-first + score-descending sort
- Integrated smart sort into `TaskManager#list`, list command, update command, and status command

## Implementation Summary

### What Was Done

- **Sort Score Formula**: `priority_weight x 100 + age_days` (age capped at 90). In-progress boost +1000, blocked penalty x0.1. Priority weights: critical=4, high=3, medium=2, low=1.
- **Position Pinning**: B36TS-based position field in frontmatter. Convenience keywords: `first` (early B36TS), `last` (current time), `after:<ref>`, `before:<ref>`. Pinned tasks always sort before auto-sorted tasks.
- **SmartSorter**: Partitions items into pinned (sorted by position asc) and unpinned (sorted by score desc), concatenates.
- **CLI Integration**: `ace-task list --sort smart|id|priority|created`, `ace-task update <ref> --position first|last|after:ref|before:ref`, `ace-task update <ref> --remove position`.
