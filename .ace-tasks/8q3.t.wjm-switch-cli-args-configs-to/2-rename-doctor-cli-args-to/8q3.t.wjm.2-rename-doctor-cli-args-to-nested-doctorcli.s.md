# Rename doctor_cli_args to nested doctor.cli_args with arrays

## Behavioral Specification

### User Experience
- **Input**: YAML config with `doctor.cli_args` (nested) instead of flat `doctor_cli_args`
- **Process**: `config.dig` paths updated to match new nesting; values are arrays
- **Output**: Same doctor agent behavior — correct CLI flags passed to agent sessions

### Expected Behavior

The flat `doctor_cli_args` key is renamed to nested `doctor.cli_args` to match the existing `plan.cli_args` naming convention. All values switch to YAML array format. The `plan.cli_args` values also switch to array format (already nested, just format change).

Config before:
```yaml
task:
  doctor_cli_args:
    gemini: "yolo"
  plan:
    cli_args:
      claude: "--permission-mode plan"
```

Config after:
```yaml
task:
  doctor:
    cli_args:
      gemini:
        - "yolo"
  plan:
    cli_args:
      claude:
        - "--permission-mode plan"
```

### Interface Contract

```ruby
# Before:
config.dig("task", "doctor_cli_args")
# => {"gemini" => "yolo"}

# After:
config.dig("task", "doctor", "cli_args")
# => {"gemini" => ["yolo"]}

# Same pattern for idea and retro:
config.dig("idea", "doctor", "cli_args")
config.dig("retro", "doctor", "cli_args")
```

**Error Handling:**
- Missing `doctor.cli_args` key: fallback to `{}` (existing `|| {}` pattern)
- Empty arrays: no cli_args passed (same as nil)

**Edge Cases:**
- `provider_cli_args` returns array from config — passed directly to `query_options[:cli_args]`
- ArgsNormalizer handles arrays (via wjm.0 foundation)
