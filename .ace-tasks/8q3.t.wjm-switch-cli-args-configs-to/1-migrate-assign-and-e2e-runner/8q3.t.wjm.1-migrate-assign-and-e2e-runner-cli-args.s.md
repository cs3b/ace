# Migrate assign and e2e-runner cli_args to array format

## Behavioral Specification

### User Experience
- **Input**: YAML config with `providers.cli_args` entries as arrays instead of strings
- **Process**: `merge_cli_args` flattens array and string inputs; CliProviderAdapter returns arrays from config
- **Output**: Same CLI commands as before — arrays are flattened and normalized downstream

### Expected Behavior

Convert `providers.cli_args` values from strings to YAML arrays in ace-assign and ace-test-runner-e2e. Update `merge_cli_args` to handle array inputs via `flat_map { |v| Array(v) }`. Update CliProviderAdapter hardcoded fallback defaults to arrays. Update Codex defaults to use proper multi-flag values.

Config before:
```yaml
cli_args:
  claude: "dangerously-skip-permissions"
  codex: "full-auto"
```

Config after:
```yaml
cli_args:
  claude:
    - "dangerously-skip-permissions"
  codex:
    - "--sandbox danger-full-access"
    - "--ask-for-approval never"
```

### Interface Contract

```ruby
# merge_cli_args handles both arrays and strings
merge_cli_args(["dangerously-skip-permissions"], "--model-settings x")
# => "dangerously-skip-permissions --model-settings x"

merge_cli_args(["--sandbox danger-full-access", "--ask-for-approval never"], nil)
# => "--sandbox danger-full-access --ask-for-approval never"
```

**Error Handling:**
- nil/empty inputs: filtered out (existing behavior preserved)
- Mixed string+array merge: `Array(v)` wraps strings in arrays before flattening

**Edge Cases:**
- Single-element array: `["full-auto"]` → flattened to `"full-auto"` (same as old string)
