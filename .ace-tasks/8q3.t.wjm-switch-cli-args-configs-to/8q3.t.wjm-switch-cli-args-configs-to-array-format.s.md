# Switch cli_args configs to array format with unified naming

## Behavioral Specification

### User Experience
- **Input**: YAML config files with per-provider CLI agent flags (e.g., sandbox mode, approval settings)
- **Process**: Config values are read, merged with user-provided args, normalized, and appended to CLI commands
- **Output**: Correctly constructed shell commands with all specified flags

### Expected Behavior

All `cli_args` entries in YAML configs use array format instead of single strings. Each array element represents a logical flag group (e.g., `"--sandbox danger-full-access"`). The flat `doctor_cli_args` key is renamed to nested `doctor.cli_args` for consistency with the existing `plan.cli_args` pattern.

Users can now specify multiple independent flags per provider:
```yaml
providers:
  cli_args:
    codex:
      - "--sandbox danger-full-access"
      - "--ask-for-approval never"
```

### Interface Contract

```yaml
# YAML config — array format (all packages)
providers:
  cli_args:
    provider_name:
      - "flag-one"
      - "--flag-two value"

# Nested context — unified naming
task:
  doctor:
    cli_args:        # was: doctor_cli_args
      gemini:
        - "yolo"
  plan:
    cli_args:
      claude:
        - "--permission-mode plan"
```

**Error Handling:**
- Empty arrays: treated as no args (same as nil)
- Mixed string/array in merge: `merge_cli_args` flattens both formats

**Edge Cases:**
- Multi-word array elements: `"--sandbox danger-full-access"` tokenized via Shellwords
- Single-element arrays: `["full-auto"]` behaves identically to old `"full-auto"`

## Objective

Enable specifying multiple CLI flags per agent in configs (e.g., `--sandbox danger-full-access --ask-for-approval never` for Codex) and unify the inconsistent `doctor_cli_args` / `cli_args` naming convention.

## Scope of Work

- **Config scope**: 7 YAML files across 6 gems + 2 project overrides
- **Code scope**: ArgsNormalizer, merge_cli_args helpers, config.dig paths, CliProviderAdapter defaults
- **Test scope**: Tests in 6 gems updated for array format

## Out of Scope

- Adding new provider flags beyond what's specified
- Changing the ArgsNormalizer auto-prefix (`--`) behavior
- Modifying QueryInterface or provider client command builders
