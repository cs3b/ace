# Add skill protocol discovery and gem skill registrations

## Behavioral Specification

### User Experience
- **Input**: Developers and agents ask nav to list or resolve skills using `skill://` the same way they already use `wfi://`, `guide://`, and `agent://`.
- **Process**: Registered gems expose canonical `handbook/skills` sources through the nav protocol system, and the registry applies the same precedence behavior used elsewhere in nav.
- **Output**: Users can discover all package-owned skills and resolve one canonical skill path without knowing which gem or provider tree currently hosts it.

### Expected Behavior

This slice adds the shared discovery layer used by both nav and `ace-assign`:
1. A new `skill` protocol is registered in nav.
2. Gems register canonical skill sources under `skill-sources`.
3. The protocol resolves to `handbook/skills` within registered gems.
4. `skill://*` lists available canonical skills across registered packages.
5. `skill://<name>` resolves a specific canonical skill path.
6. Duplicate names follow deterministic precedence, matching nav’s registered-source order.

### Interface Contract

```bash
# List canonical package skills
ace-nav skill://*

# Resolve one canonical skill
ace-nav skill://as-task-work
ace-nav skill://as-e2e-run
```

```yaml
# Example gem source registration
name: ace-task
type: gem
description: Canonical skills from ace-task
priority: 10
config:
  relative_path: handbook/skills
  pattern: "*/SKILL.md"
```

**Error Handling:**
- Unknown `skill://` target returns the normal nav not-found behavior with the searched source scope.
- Duplicate skill names resolve deterministically and provide enough source information to explain which package won.
- A registered gem without `handbook/skills` contributes no results rather than failing the whole protocol.

**Edge Cases:**
- `skill://*` lists all package skills, not only the subset already consumed by `ace-assign`.
- Packages may be partially migrated; `skill://` still lists what exists without assuming every gem is complete.
- A project-level override may shadow a gem-provided canonical skill through existing nav precedence.
