# ace-compressor: agent mode prompt-guided minification

## Behavioral Specification

### User Experience
- **Input**: A single Markdown or text source passed to `ace-compressor ... --mode agent`.
- **Process**: The CLI builds the same normalized semantic baseline learned from exact mode, runs an `ace-llm` prompt flow over that structured content to minify and deduplicate it, validates fidelity, and either emits agent output or degrades explicitly to exact.
- **Output**: A high-density pack that keeps almost all instructions, rules, commands, numeric facts, and tables while removing redundancy and unnecessary narrative weight.

### Expected Behavior
This mode is not a summary mode. It should simplify and minify content rather than replace it with broad prose summaries. The final result should preserve instruction-bearing and reasoning-critical units with near-lossless fidelity while compressing repeated or low-value narrative expression more aggressively than exact mode. Compared with compact mode, it should handle dense technical and rule-heavy inputs with better quality and fewer outright refusals.

### Interface Contract
```bash
ace-compressor docs/vision.md --mode agent
ace-compressor docs/architecture.md --mode agent --format stats
ace-compressor docs/decisions.md --mode agent --verbose
```

**Operating Modes / Defaults:**
- `--mode agent` is a new top-level compressor mode alongside `exact` and `compact`.
- Agent mode is single-source first in this task. Corpus-level multi-file agent compression is deferred.
- Provider and model selection use existing `ace-llm` configuration and alias resolution; no new compressor-specific provider flags are required in this phase.
- Prompt composition must be overrideable via `ace-bundle` resources and config, not hardcoded prompt strings.

**Error Handling:**
- If `ace-llm` is unavailable or the agent result fails fidelity validation, the CLI emits exact-mode output, adds explicit fallback metadata, prints degraded-mode messaging, and exits `0`.
- Unsupported binary or empty input errors stay explicit and mode-aware, matching existing compressor behavior.
- Validation failure must never silently downgrade fidelity without either `FALLBACK|...` or explicit loss markers.

**Edge Cases:**
- Rule-heavy docs should no longer rely on compact-style refusal semantics; they either validate under agent mode or degrade explicitly to exact.
- Repeated examples or commands may compress to references when mimicry is not required, but unique executable instructions must survive.
- Tables and numeric facts must remain present or be reduced with explicit fidelity/loss signaling.

### Success Criteria
- [ ] `ace-compressor <file> --mode agent` exists and produces a runnable single-source path.
- [ ] Successful agent-mode runs emit `H|ContextPack/3|agent`.
- [ ] Agent mode preserves rule, instruction, numeric, command, and table content while removing redundancy rather than generating broad summaries.
- [ ] Dense technical and rule-heavy docs achieve better quality than current compact behavior.
- [ ] Fallback to exact is explicit, exit-`0`, and detectable from runtime output.
- [ ] Prompt assembly is resource-driven and overrideable through `ace-bundle`/`ace-llm` patterns.
