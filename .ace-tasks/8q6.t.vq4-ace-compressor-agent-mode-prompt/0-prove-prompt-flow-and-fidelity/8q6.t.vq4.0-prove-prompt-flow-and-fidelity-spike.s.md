# prove prompt flow and fidelity spike

## Behavioral Specification

### User Experience
- **Input**: One representative source such as `docs/architecture.md` or `docs/decisions.md` run through `--mode agent`.
- **Process**: The compressor assembles the agent prompt through `ace-bundle` resources, hands structured content to `ace-llm`, and validates the returned minified pack against the exact baseline.
- **Output**: A proof that the prompt-flow architecture, fidelity gates, and runtime markers can work end to end for one source.

### Expected Behavior
This spike validates the target architecture without trying to solve all source classes at once. It must prove that agent mode can operate on structured compressor input rather than raw markdown, and that the validator can distinguish an acceptable minified result from one that needs exact fallback.

### Interface Contract
```bash
ace-compressor docs/architecture.md --mode agent --verbose
```

**Error Handling:**
- Missing prompt resources or provider access must fail the spike scenario clearly.
- Invalid agent output must be detectable through a validator outcome, not manual eyeballing.

**Edge Cases:**
- The spike may choose one representative source class, but it must document which behaviors are validated and which remain deferred.
- Prompt composition must be resource-driven from day one.

### Success Criteria
- [ ] One end-to-end single-source spike path is fully specified.
- [ ] The spike proves agent mode works on structured compressor input rather than raw markdown summarization.
- [ ] The surviving concepts for final agent mode are made explicit.
- [ ] The spike leaves a concrete concept inventory for the next subtasks.

- [x] The first slice is a spike because this is a core execution-path redesign.

- [x] Validator-visible success/failure is mandatory in the spike.
- [x] Corpus-level behavior is deferred.

Prove that agent mode can be built as a prompt-composed, validator-checked compression path instead of a raw summarization shortcut.

## Scope of Work

- **System Behavior Scope**: Prompt assembly, agent output validation, and architecture proof.
- **Interface Scope**: Single-source spike contract only.
