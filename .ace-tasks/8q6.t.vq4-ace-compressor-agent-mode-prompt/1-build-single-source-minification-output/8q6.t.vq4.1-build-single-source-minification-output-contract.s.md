# build single-source minification output contract

## Behavioral Specification

### User Experience
- **Input**: A single Markdown or text file passed to `ace-compressor ... --mode agent`.
- **Process**: The CLI emits a prompt-guided minified pack that removes redundancy, canonicalizes repeated patterns, and keeps the structured content agents actually need.
- **Output**: A successful run emits `ContextPack/3|agent` and a pack that is smaller than exact mode without collapsing into summary text.

### Expected Behavior
This slice delivers the first runnable single-source agent-mode output. It should preserve rules, commands, numeric facts, tables, and reasoning-critical examples while compressing verbose narrative, duplicated explanations, and repeated examples more aggressively than exact mode. It must be visibly different from compact mode by targeting better retention on dense technical and rule-heavy material.

### Interface Contract
```bash
ace-compressor docs/vision.md --mode agent --format stdio
ace-compressor docs/architecture.md --mode agent --format stats
ace-compressor docs/decisions.md --mode agent --verbose
```

**Error Handling:**
- Successful agent output must identify itself clearly as `agent` mode.
- Any content reduction that removes executable examples or table rows must be explicit through existing loss/reference markers.

**Edge Cases:**
- Rule-heavy content may still succeed under agent mode if fidelity gates pass.
- Example and code compression must preserve unique instructions and mimicry-required cases.

### Success Criteria
- [ ] Single-source `--mode agent` emits `H|ContextPack/3|agent`.
- [ ] Narrative docs compress more than exact mode without replacing the result with broad section summaries.
- [ ] Dense technical docs retain facts and rules while shedding boilerplate narrative.
- [ ] Rule-heavy docs can produce validated agent output when fidelity is preserved.

### Validation Questions

- [x] Rule-heavy content may succeed under agent mode.
- [x] Summary text is not the primary compression mechanism.
- [x] Loss/reference markers stay explicit where reductions occur.
