# Surface forked agent last message on stall (provider-agnostic)

## Behavioral Specification

### User Experience

- **Input**: `ace-assign fork-run --assignment <id>@<root>` (existing command, no change)
- **Process**: When a forked agent stalls and the subtree does not complete, the system captures the agent's last message and surfaces it
- **Output**: Error message includes the agent's last message; phase metadata stores `stall_reason`; `ace-assign status` shows the stall reason

### Expected Behavior

When `fork-run` detects an incomplete subtree after the forked session ends:

1. Read the agent's last message from a cache file
2. Display it in the error output so the orchestrator/user can see WHY the fork stalled
3. Write a `stall_reason` field to the active phase's frontmatter for persistent visibility
4. `ace-assign status` shows `Stall Reason:` line when the current phase has one

**Before** (current behavior):
```
Error: Fork subtree 010.03 did not complete within spawned session. Current phase: 010.03.04 (release).
```

**After** (new behavior):
```
Error: Fork subtree 010.03 did not complete within spawned session. Current phase: 010.03.04 (release).

Agent's last message:
I hit an unexpected state change and need your direction before I continue.
After `bundle install`, `git status` now shows many unrelated file changes...
```

### Two-Layer Capture Strategy

**Layer 1 — All providers (normal exit):**
- `ForkSessionLauncher.launch` captures the return value from `query_interface.query()` (which includes `{ text: ... }`)
- Writes `result[:text]` to `.cache/ace-assign/<id>/sessions/<fork_root>-last-message.md`
- Works for all providers when the agent process exits (even if subtree is incomplete)

**Layer 2 — Codex timeout resilience:**
- Pass `last_message_file:` through `QueryInterface` → `generation_opts` → `CodexClient.generate`
- CodexClient maps it to `--output-last-message <file>` flag
- Codex writes to this file progressively, so it's available even when SafeCapture kills the process on timeout
- Non-codex providers on timeout: file doesn't exist → error says "last message unavailable"

### Interface Contract

```bash
# ace-assign status output (new field)
Current Phase: 010.03.04 - release
Current Status: in_progress
Stall Reason: I hit an unexpected state change and need your direction...

# Phase frontmatter (new optional field)
---
name: release
status: in_progress
stall_reason: "I hit an unexpected state change..."
---

# Last message file location
.cache/ace-assign/<id>/sessions/<fork_root>-last-message.md
```

**Error Handling:**
- Non-codex providers on timeout: file doesn't exist → error omits "Agent's last message" section
- Last message file doesn't exist or is empty: error message omits "Agent's last message" section
- Very long messages (>2000 chars): truncated with "... (truncated)" suffix in error output; full text remains on disk

### Success Criteria

- [ ] Fork stall error includes agent's last message when available
- [ ] `stall_reason` persisted in active phase frontmatter on stall
- [ ] `ace-assign status` displays stall reason
- [ ] Works with all CLI providers (claude, codex, pi, opencode, gemini)
- [ ] Codex gets timeout-resilient capture via `--output-last-message`
- [ ] Non-codex providers degrade gracefully on timeout
- [ ] Phase model backward-compatible (`stall_reason` defaults to nil)
- [ ] Unit tests for modified components

### Key Design Details

- **Path convention**: `<assignment.cache_dir>/sessions/<fork_root>-last-message.md` — parallels existing `pids/` subdirectory pattern
- **Path reconstruction**: ForkRun reconstructs path from `assignment.cache_dir` + `root_phase.number` (deterministic, no need to thread it through errors)
- **Truncation**: >2000 chars truncated with `"... (truncated)"` suffix in error output; full text remains on disk
- **Layer 1 write guard**: Skip writing if file already exists and is non-empty (Codex may have written it natively)
- **PhaseWriter#update_frontmatter** already accepts arbitrary hash — just pass `{ "stall_reason" => text }`
- **Non-codex timeout**: Graceful degradation — error omits "Agent's last message" section
