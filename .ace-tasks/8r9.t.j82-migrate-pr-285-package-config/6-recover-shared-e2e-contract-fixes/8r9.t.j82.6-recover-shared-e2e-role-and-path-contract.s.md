# Recover shared E2E role and path contract fixes from fix/e2e

## Objective
- Recover the shared E2E role, path, and instruction contract fixes from `fix/e2e` without pulling in package-local migration work or tmux-specific churn.

## User Experience
- The restarted E2E contract uses one consistent set of role names and path conventions across shared config and assignment guidance.
- Developers and agents see `test-e2e/scenarios` and `test-e2e/integration` in shared config and task-driving instructions instead of stale `test/e2e` references.
- The E2E runner and verifier roles are explicitly separated so read-only verification is not conflated with goal-seeking execution.

- `.ace/llm/config.yml` renames the restarted E2E roles from the old executor-only shape to explicit `e2e-runner` and `e2e-verifier` roles.

- `.ace/docs/config.yml` ignores scenario markdown under `test-e2e/scenarios` rather than the obsolete `test/e2e` path.
