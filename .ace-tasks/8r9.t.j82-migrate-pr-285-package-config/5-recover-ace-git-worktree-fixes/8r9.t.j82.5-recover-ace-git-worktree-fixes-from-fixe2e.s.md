# Recover ace-git-worktree fixes from fix/e2e

## Objective
- Recover the real worktree cleanup hardening from `fix/e2e`.

## User Experience
- Pruning stale worktrees aggressively expires immediately instead of waiting for default prune timing.
- Removing a broken worktree surfaces a hard failure when the directory still exists after cleanup, instead of only warning and continuing.

## Expected Behavior
- `git worktree prune` runs with `--expire now`.
- Removing a worktree fails if the directory still exists after cleanup.
- Directory-removal failures return an error result instead of only warning and continuing.
- Regression tests cover prune behavior and stuck-directory failure handling.

## Interface Contract
- Inputs:
  - `git diff $(git merge-base main fix/e2e)..fix/e2e -- ace-git-worktree/lib/ace/git/worktree/molecules/worktree_remover.rb ace-git-worktree/test/commands/prune_command_test.rb ace-git-worktree/test/molecules/worktree_remover_test.rb`
- Outputs:
  - prune hardening plus failure-result semantics for stuck directory cleanup
- Validation conditions:
  - prune passes `--expire now`
  - directory cleanup never degrades to warning-only continuation

## Success Criteria
- [ ] prune command hardening is specified
- [ ] directory cleanup failure handling is specified
- [ ] prune and remover regression tests are included

## Verification Plan
- [ ] `ace-test ace-git-worktree`

- [ ] Confirm remover coverage fails when the directory still exists after cleanup.

## Out Of Scope
