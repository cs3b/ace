# Recover non-migration code and config fixes from fix/e2e

## Objective
- Recover substantive package and monorepo config fixes that were mixed into `fix/e2e`, without redoing the restarted E2E migration already owned by `8r9.t.hzr` and `8r9.t.i05`.

## User Experience
- The user gets one orchestrator task that names exactly which fixes should be brought back from `fix/e2e`.
- Each child task owns one package or one monorepo config slice.
- Migration churn, retros, and patch-bucket artifacts are explicitly excluded.

## Expected Behavior
1. The task uses `git diff main..fix/e2e` as the source of truth.
2. Only non-migration fixes are captured here.
3. The task decomposes into one draft subtask for each package/config area that needs recovery.
4. Work already covered by `8r9.t.hzr` and `8r9.t.i05` is deferred to those tasks instead of being duplicated here.

## Interface Contract
- Inputs:
  - branch diff `main..fix/e2e`
  - task ownership boundaries from `8r9.t.hzr` and `8r9.t.i05`
- Outputs:
  - one orchestrator task
  - one subtask per package/config recovery slice
- Validation condition:
  - every included subtask describes a real code or config fix that is not part of the restarted E2E migration
