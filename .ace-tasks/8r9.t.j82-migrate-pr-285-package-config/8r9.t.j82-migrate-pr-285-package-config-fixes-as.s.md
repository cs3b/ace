# Recover non-migration code and config fixes from fix/e2e

## Objective
- Recover substantive package fixes plus shared restarted-E2E contract fixes that were mixed into `fix/e2e`, without redoing the package migration and scenario rewrites already owned by `8r9.t.hzr` and `8r9.t.i05`.

## User Experience
- The user gets one orchestrator task that names exactly which fixes should be brought back from `fix/e2e`.
- Each child task owns one package or one monorepo config slice.
- Shared config and instruction changes that affect multiple packages are grouped into one explicit child instead of being left implicit.
- Migration churn, retros, and patch-bucket artifacts are explicitly excluded.

## Expected Behavior
1. The task uses `git diff $(git merge-base main fix/e2e)..fix/e2e` as the source of truth.
2. Only non-migration fixes are captured here.
3. The task decomposes into one draft subtask for each package or shared-contract area that needs recovery.
4. Shared E2E role and path contract updates are captured in a dedicated child instead of being hidden inside package slices.
5. Work already covered by `8r9.t.hzr` and `8r9.t.i05` is deferred to those tasks instead of being duplicated here.

## Interface Contract
- Inputs:
  - branch diff `$(git merge-base main fix/e2e)..fix/e2e`
  - task ownership boundaries from `8r9.t.hzr` and `8r9.t.i05`
- Outputs:
  - one orchestrator task
  - one subtask per package or shared-contract recovery slice
- Validation condition:
  - every included subtask describes a real code or config fix that exists on `fix/e2e` and is not already owned by the restarted E2E migration tasks
