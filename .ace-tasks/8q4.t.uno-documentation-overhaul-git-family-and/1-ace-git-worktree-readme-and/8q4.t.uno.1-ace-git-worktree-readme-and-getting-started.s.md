# ace-git-worktree README and Getting Started

## Objective

Rewrite ace-git-worktree's README from a 737-line exhaustive reference into a landing page. The current README contains full command reference with every flag documented, PR/branch workflow details, configuration YAML with all options, troubleshooting FAQ with 8+ scenarios, architecture overview, dependency compatibility tables, and duplicated troubleshooting sections. Create a tutorial-style getting-started.md.

## Behavioral Specification

### User Experience
- **Input**: Current ace-git-worktree/README.md (737 lines)
- **Process**: Rewrite README as landing page, create getting-started tutorial, write VHS tape, create usage.md + handbook.md, update gemspec, audit handbook
- **Output**: Rewritten README.md (~60 lines), new docs/getting-started.md, new VHS tape + GIF committed to docs/demo/, new docs/usage.md, docs/handbook.md, updated gemspec

### Expected Behavior

New README should convey: "Task-aware git worktree management -- start working on any task in an isolated environment with one command."

Key messaging:
- One command creates an isolated worktree linked to a task, PR, or branch
- Automatic task status tracking and metadata management via ace-taskflow
- PR worktrees streamline code review workflows
- Configurable naming, hooks, and cleanup for team consistency

Current README problems to fix:
- 737 lines is massively over-documented for a landing page
- Full command reference with every flag for create, list, switch, remove, prune, config (belongs in --help)
- Duplicated troubleshooting section (appears twice in the file)
- Configuration YAML with all options and template variables (belongs in getting-started)
- Architecture section explaining ATOM pattern (belongs in developer docs)
- Dependency compatibility table (belongs in CHANGELOG or docs)
- Mono-repo development setup instructions (belongs in CONTRIBUTING)

### Interface Contract

README.md (~60 lines):
- Tagline: "Task-aware git worktree management -- isolated environments in one command"
- Demo GIF
- Why section (value-first, no commands)
- Works With (integration packages)
- Agent Skills (package-owned only)
- Features: task-linked worktrees, PR worktrees, auto-status tracking, configurable hooks
- Documentation links to getting-started.md and --help
- Part of ACE footer

docs/getting-started.md:
- Prerequisites (Ruby, Git 2.0+, optional: ace-taskflow, mise)
- First worktree: `ace-git-worktree create --task 081`
- Switching and working: `cd $(ace-git-worktree switch 081)`
- PR worktrees: `ace-git-worktree create --pr 26`
- Cleanup: `ace-git-worktree remove --task 081`
- Configuration basics
- Common Commands table
- What to try next (hooks, branch worktrees, bulk operations)

VHS tape: Show `ace-git-worktree create --task 081` creating an isolated worktree, then `ace-git-worktree list --show-tasks`
