# ace-git README and Getting Started

## Objective

Rewrite ace-git's README from a 306-line reference manual into a landing page. The current README contains full workflow documentation with multi-step bash examples, configuration YAML schemas, template override instructions, integration notes, and a development section. Create a tutorial-style getting-started.md. Delete docs/usage.md (218 lines of CLI reference that duplicates --help).

## Behavioral Specification

### User Experience
- **Input**: Current ace-git/README.md (306 lines), docs/usage.md (218 lines)
- **Process**: Rewrite README as landing page, create getting-started tutorial, write VHS tape, delete usage.md
- **Output**: Rewritten README.md (~60 lines), new docs/getting-started.md, new VHS tape, deleted docs/usage.md

### Expected Behavior

New README should convey: "Git workflows and context commands for developers and AI agents -- rebase, PR creation, and commit reorganization with guardrails."

Key messaging:
- Provides guided workflows for rebase, PR creation, and commit reorganization
- CLI commands (status, diff, branch, pr) give instant repo context
- Changelog-preserving rebase prevents version file conflicts
- Works seamlessly with ace-git-worktree and ace-git-commit

Current README problems to fix:
- Full workflow step-by-step bash examples (306 lines -- belongs in getting-started or workflow files themselves)
- Configuration YAML schemas with all options documented inline (belongs in --help or getting-started)
- Template override instructions with mkdir/cat examples (belongs in getting-started)
- "Usage Examples" section with 3 multi-step scenarios is reference, not landing page material

### Interface Contract

README.md (~60 lines):
- Tagline: "Git workflows and context commands for developers and AI agents"
- Demo GIF
- Why section (2-3 sentences)
- Quick Start: `gem install ace-git && ace-git status`
- Features: rebase with changelog preservation, PR templates, commit reorganization, smart diff
- Documentation links to getting-started.md and --help
- Part of ACE footer

docs/getting-started.md:
- Prerequisites (Ruby, Git 2.23+, optional: GitHub CLI)
- First command: `ace-git status`
- Exploring diff: `ace-git diff`
- Using a workflow: `ace-nav wfi://git/rebase`
- Configuration basics
- What to try next (PR workflow, reorganize commits)

VHS tape: Show `ace-git status` displaying repo context, then `ace-git diff --format summary`
