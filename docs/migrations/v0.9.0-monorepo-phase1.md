# Monorepo Migration - Phase 1 (v0.9.0)

## Overview

Version 0.9.0 marks a major architectural shift from a multi-repository submodule structure to a unified monorepo containing modular Ruby gems. This migration simplifies development, testing, and dependency management while maintaining clear separation of concerns through focused gems.

## What Changed

### Architecture Transformation

**Before (≤ v0.8.x):**
```
ace/
├── dev-handbook/          # Git submodule
├── dev-tools/             # Git submodule
└── dev-taskflow/          # Git submodule
```

**After (≥ v0.9.0):**
```
ace/
├── ace-core/              # Foundation gem
├── ace-bundle/           # Context loading
├── ace-git-commit/        # Git commit tool
├── ace-llm/               # LLM integration
├── ace-llm-providers-cli/ # CLI provider support
├── ace-nav/               # Resource navigation
├── ace-taskflow/          # Task management
├── ace-test-runner/       # Test execution
├── ace-test-support/      # Testing infrastructure
├── dev-handbook/          # Legacy (being migrated)
├── dev-tools/             # Legacy (being migrated)
└── .ace/                  # Configuration root
```

### New Gems and Commands

All functionality is now packaged as focused Ruby gems:

| Gem | Executable(s) | Purpose |
|-----|---------------|---------|
| **ace-core** | `ace-framework` | Configuration cascade, shared utilities |
| **ace-bundle** | `ace-bundle` | Project context loading with smart caching |
| **ace-git-commit** | `ace-git-commit` | LLM-powered git commits |
| **ace-llm** | `ace-llm-query` | Multi-provider LLM integration |
| **ace-llm-providers-cli** | `ace-llm-providers-cli-check` | CLI-based LLM provider support |
| **ace-nav** | `ace-nav` | Resource discovery and wfi:// protocol |
| **ace-taskflow** | `ace-taskflow` | Task, release, idea management |
| **ace-test-runner** | `ace-test`, `ace-test-suite` | Test execution and reporting |
| **ace-test-support** | _(library only)_ | Shared testing infrastructure |

### Configuration System Overhaul

**Old Configuration:**
- Per-submodule configuration files
- Scattered ENV loading
- No standardized cascade

**New Configuration (.ace/ cascade):**
```
./.ace/          # Project-specific (highest priority)
~/.ace/          # User defaults
gem defaults/    # Gem-bundled defaults (lowest priority)
```

Each gem has its own namespace:
- `.ace/core/config/core.yml`
- `.ace/context/config/context.yml`
- `.ace/git/config/git.yml`
- etc.

**Resolution:** Nearest/deepest wins (project overrides user overrides gem defaults)

### Command Name Changes

| Old Command | New Command | Notes |
|-------------|-------------|-------|
| `context` | `ace-bundle` | Gem-prefixed |
| `ace-gc` | `ace-git-commit` | Full name, also called ace-git |
| `capture` | `ace-idea create` | Integrated into taskflow |
| Tests varied | `ace-test`, `ace-test-suite` | Standardized |

### ATOM Architecture

All gems now follow the ATOM pattern:

```
lib/ace/your_gem/
├── atoms/         # Pure functions, no side effects
├── molecules/     # Composed operations
├── organisms/     # Business logic orchestration
└── models/        # Data structures
```

Benefits:
- Consistent code organization
- Clear separation of concerns
- Highly testable components
- Easy to understand and navigate

## Breaking Changes

### 1. No More Git Submodules

**Impact:** Submodule commands and workflows are removed.

**Migration:**
```bash
# Old
git submodule update --init --recursive
git submodule foreach git pull

# New (not needed - single repo)
git pull
```

### 2. Config Path Changes

**Impact:** Configuration must move to `.ace/` directories.

**Migration:**
```bash
# If you had custom configs in old locations
mkdir -p .ace/{core,context,git,taskflow}
# Move your configs to appropriate .ace/ subdirectories
```

### 3. Command Names

**Impact:** Some commands renamed with gem prefixes.

**Migration:** Update scripts and aliases:
```bash
# Update your aliases
alias ctx='ace-bundle'
alias gc='ace-git-commit'
```

### 4. Bundle Exec Required (Development)

**Impact:** During development from monorepo root, use `bundle exec`.

**Workaround:**
```bash
# Development
bundle exec ace-bundle

# After gem install (production)
ace-bundle  # Works without bundle exec
```

## Migration Steps

### For End Users

1. **Update Repository:**
   ```bash
   cd ace
   git pull origin main
   ```

2. **Install Dependencies:**
   ```bash
   bundle install
   ```

3. **Migrate Configuration (if you had custom configs):**
   ```bash
   # Example: Move context config
   mkdir -p .ace/context/config
   # Copy your old context configs to .ace/context/config/context.yml
   ```

4. **Test Commands:**
   ```bash
   bundle exec ace-bundle --list
   bundle exec ace-task list --current
   ```

5. **Install Gems (Optional):**
   ```bash
   # For production use without bundle exec
   gem install ace-bundle ace-taskflow ace-git-commit
   ```

### For Developers

1. **Clone Repository:**
   ```bash
   git clone https://github.com/cs3b/ace.git
   cd ace
   ```

2. **Install Dependencies:**
   ```bash
   bundle install
   ```

3. **Run Tests:**
   ```bash
   # All tests
   bundle exec ace-test-suite

   # Specific gem
   cd ace-core
   bundle exec rake test
   ```

4. **Understand Structure:**
   - Read `docs/architecture.md` for ATOM pattern
   - Read `docs/ace-gems.g.md` for gem development
   - Each gem has its own README

5. **Development Workflow:**
   ```bash
   # Make changes in ace-*/lib/
   # Write tests in ace-*/test/
   # Run tests: bundle exec rake test
   # Run gem command: bundle exec ace-your-gem
   ```

### For CI/CD Pipelines

1. **Update Checkout:**
   ```yaml
   # Old (with submodules)
   - uses: actions/checkout@v3
     with:
       submodules: recursive

   # New (no submodules)
   - uses: actions/checkout@v3
   ```

2. **Update Bundle Install:**
   ```yaml
   - name: Install dependencies
     run: bundle install
   ```

3. **Update Test Commands:**
   ```yaml
   # Old (varied per submodule)
   - run: cd dev-tools && rake test

   # New (unified)
   - run: bundle exec ace-test-suite
   # Or test specific gems:
   - run: cd ace-core && bundle exec rake test
   ```

4. **Matrix Testing Example:**
   ```yaml
   strategy:
     matrix:
       ruby: ['3.0', '3.1', '3.2', '3.3']
       gem: ['ace-core', 'ace-bundle', 'ace-test-runner']
   steps:
     - run: cd ${{ matrix.gem }} && bundle exec rake test
   ```

- ✅ Consistent command naming (`ace-*` prefix)

- ✅ Consistent error handling

## Troubleshooting

### "Command not found: ace-bundle"

**Cause:** Not using `bundle exec` in development or gem not installed.

**Solution:**
```bash
# Development (from monorepo)
bundle exec ace-bundle

# Production
gem install ace-bundle
```

### "Config file not found"

**Cause:** Looking in old config locations.

**Solution:** Move configs to `.ace/` directories:
```bash
mkdir -p .ace/context/config
# Create .ace/context/config/context.yml
```

### "Submodule missing"

**Cause:** Old checkout with submodules.

**Solution:**
```bash
# Remove old submodule references
git submodule deinit -f --all
rm -rf .git/modules/

# Pull latest
git pull origin main
bundle install
```

### Tests Failing

**Cause:** Old dependencies or cache.

**Solution:**
```bash
# Clean and reinstall
rm -rf .bundle vendor/bundle
bundle install

# Clear test cache
rm -rf ace-*/coverage ace-*/.cache
bundle exec ace-test-suite
```
