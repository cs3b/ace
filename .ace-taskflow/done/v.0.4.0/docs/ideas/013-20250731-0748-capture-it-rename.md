# Rename `ideas-manager capture` to `capture-it`

## Intention

To rename the `ideas-manager capture` CLI command to `capture-it` and update all relevant references across project documentation to ensure consistency and clarity.

## Problem It Solves

**Observed Issues:**
- The command `ideas-manager capture` is perceived as too verbose and profound, while a simpler name like `capture-it` would be more user-friendly.
- All references to `ideas-manager capture` in project documents are outdated and need to be updated to reflect the new command name.
- Inconsistent naming can lead to confusion for users and AI agents interacting with the toolkit.

**Impact:**
- Users may struggle to find or correctly use the command due to the verbose name.
- Documentation becomes outdated and misleading, hindering onboarding and understanding of the toolkit's capabilities.
- Automation scripts or workflows that rely on the old command name will fail.

## Key Patterns from Reflections

- **Naming Conventions**: The need for simple, descriptive, and consistent naming for tools and commands is crucial for usability and maintainability (implied by the user's request for simplicity).
- **Documentation Synchronization**: Project documentation must be kept in sync with code changes, especially for executable tools and their interfaces (as evidenced by the need to update references).
- **CLI Tooling**: The project has a significant number of CLI tools, and maintaining their clarity and consistency is a recurring theme.
- **ADR-004: Consistent Path Standards**: While focused on file paths, this ADR highlights the project's commitment to consistent naming and standardization across all elements.
