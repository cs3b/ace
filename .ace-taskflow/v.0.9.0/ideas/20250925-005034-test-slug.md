## Problem
`ace-taskflow` generates unique, URL-friendly slugs for idea files (e.g., `20250925-004814-add-support-for-task-dependencies-in-ace-taskflow.md`). The current slug generation logic, likely within `ace-taskflow/lib/ace/taskflow/organisms/idea_writer.rb`, lacks dedicated test coverage. This exposes the system to potential issues such as non-unique slugs, slugs with invalid characters for file paths, or inconsistent formatting when titles contain special characters or are excessively long. Without robust tests, any future modifications to the slug generation could inadvertently introduce critical bugs, impacting the discoverability and integrity of idea documents.

## Solution
Implement a new, comprehensive test suite within the `ace-taskflow` gem specifically for the idea slug generation mechanism. This suite will validate the conversion of raw idea titles into their corresponding slug format, ensuring they adhere to defined standards for uniqueness, character validity, and length.

## Implementation Approach
1.  **Identify Slug Generation Logic**: Pinpoint the exact `Atom` or `Molecule` responsible for transforming an idea title into its slug. This is likely part of the `Ace::Taskflow::Organisms::IdeaWriter` or a helper `Molecule` it utilizes.
2.  **Create Test File**: Add a new test file, e.g., `ace-taskflow/test/molecules/slug_generator_test.rb` (if a dedicated molecule is extracted) or extend `ace-taskflow/test/organisms/idea_writer_test.rb`.
3.  **Develop Test Cases**: Cover various scenarios, including:
    *   Basic alphanumeric titles (e.g., 'My New Idea' -> 'my-new-idea').
    *   Titles with special characters (e.g., 'Fix: UI/UX Issues!' -> 'fix-ui-ux-issues').
    *   Titles with multiple spaces or leading/trailing spaces.
    *   Titles that are empty or consist only of special characters.
    *   Titles exceeding a maximum length, ensuring proper truncation and hyphenation.
    *   Titles that could lead to duplicate slugs (if uniqueness logic is part of this component).
4.  **Leverage `ace-test-support`**: Utilize shared testing infrastructure from `ace-test-support` for assertions and test setup.

## Considerations
-   **ATOM Pattern Adherence**: If the slug generation logic is complex, consider extracting it into a dedicated `Ace::Taskflow::Molecules::SlugGenerator` (or even an `Ace::Core::Atoms::Slugifier` if general slugging is needed elsewhere) to maintain a clean separation of concerns and enable focused testing.
-   **Determinism**: Ensure the slug generation is deterministic, producing the same slug for the same input title every time.
-   **Uniqueness**: While `IdeaWriter` ensures overall file uniqueness, the slug generation molecule/atom should focus on producing a valid *base* slug. Uniqueness handling (e.g., appending a timestamp or counter) would be an `Organism`'s responsibility.
-   **Configuration**: Consider if any aspects of slug generation (e.g., max length, allowed characters) should be configurable via the `.ace/` configuration cascade.

## Benefits
-   **Increased Robustness**: Ensures idea file paths are always valid and consistent, preventing file system errors or navigation issues.
-   **Improved Maintainability**: Allows for safe refactoring or enhancement of slug generation logic in the future.
-   **Enhanced Reliability**: Guarantees that `ace-taskflow` ideas are reliably stored and discoverable.
-   **Better User Experience**: Consistent and predictable idea slugs improve the overall experience for both human developers and AI agents interacting with `ace-taskflow`.
