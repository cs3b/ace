# Migrate ace-* Gems to New API

## Objective

Update all ace-* gems to use the new ace-config API with gem_path parameter.

## Relationship to Task 143

**Task 143** moves hardcoded defaults from Ruby code into `.ace.example/` files using existing `Ace::Core` API.

**This task (157.09)** updates gems to use the new `Ace::Config.create()` API after extraction:
- Replace `Ace::Core::Organisms::ConfigResolver` with `Ace::Config.create`
- Add `gem_path:` parameter for gem defaults
- Remove any remaining `load_gem_defaults` methods

The work is NOT duplicated - 143 standardizes defaults location, 157.09 switches to new API.

## Standard Pattern

```ruby
# lib/ace/my_gem.rb
module Ace
  module MyGem
    def self.config
      @config ||= begin
        gem_root = Gem.loaded_specs["ace-my-gem"]&.gem_dir ||
                   File.expand_path("../..", __dir__)
        Ace::Config.create(
          config_dir: ".ace",
          defaults_dir: ".ace-defaults",
          gem_path: gem_root
        ).resolve_for(["my-gem/config.yml"]).data
      end
    end

    def self.reset_config!
      @config = nil
    end
  end
end
```
