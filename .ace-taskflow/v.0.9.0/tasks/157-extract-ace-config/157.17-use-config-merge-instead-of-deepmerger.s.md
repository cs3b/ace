# Update ace-* gems to use config.merge() instead of DeepMerger

## Objective

Update all ace-* gems to use the new `config.merge()` API instead of calling `Ace::Config::Atoms::DeepMerger` directly. This centralizes merge logic in the Config object and provides a cleaner API.

## Behavioral Specification

### Current Pattern (to replace)

```ruby
# Gems currently do this:
config = resolver.resolve_for(["docs/config.yml", "docs/config.yaml"])
data = config.data  # Extract raw hash
merged = Ace::Config::Atoms::DeepMerger.merge(data, cli_options)
```

### New Pattern

```ruby
# Should become:
config = resolver.resolve_for(["docs/config.yml", "docs/config.yaml"])
merged_config = config.merge(cli_options)  # Returns Config object
```

## Implementation Notes

- Each gem needs to keep the Config object instead of extracting `.data`
- Replace `DeepMerger.merge(data, options)` with `config.merge(options)`
- Some gems may need to adjust how they return/use the merged result
- Tests should verify merged config works correctly
