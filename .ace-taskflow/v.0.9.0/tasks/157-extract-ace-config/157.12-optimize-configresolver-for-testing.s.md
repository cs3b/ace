# Improve ConfigResolver performance for test environments

## Objective

ace-git unit tests are slow (5.7s for 29 organism tests) because ConfigResolver performs filesystem searches on every call. Add test mode to ace-config that skips filesystem searches and returns empty/mock config immediately.

**Dependency**: This task runs AFTER Task 157 (ace-config extraction) completes.

## Behavioral Specification

### User Experience
- **Input**: Test runner executes ace-git test suite
- **Process**: Tests complete quickly without filesystem searches for config files
- **Output**: Tests pass in < 1s (currently 5.7s)

### Expected Behavior

ConfigResolver should NOT perform filesystem searches in test environments. In test mode:
1. Skip filesystem searches entirely
2. Return empty config immediately
3. Allow tests to provide mock config if needed

### Interface Contract

```ruby
# Test Mode Detection (Priority Order):
# 1. Constructor parameter (explicit)
Ace::Config.create(test_mode: true).resolve  # Returns empty immediately

# 2. Class-level flag
Ace::Config.test_mode = true
Ace::Config.create.resolve  # Returns empty immediately

# 3. Auto-detection (ENV or Minitest defined)
# ACE_CONFIG_TEST_MODE=1 or defined?(Minitest)

# Mock config injection
Ace::Config.create(test_mode: true, mock_config: { "key" => "value" }).resolve

# Opt-out for integration tests
with_real_config do
  # Uses real filesystem-based config loading
end
```

### Success Criteria
- [ ] **Performance**: ace-git organism tests run in < 1s (currently 5.7s)
- [ ] **Test Independence**: Tests don't depend on filesystem state
- [ ] **Backward Compatibility**: Non-test code unaffected
- [ ] **Opt-out Available**: `with_real_config` helper for integration tests
