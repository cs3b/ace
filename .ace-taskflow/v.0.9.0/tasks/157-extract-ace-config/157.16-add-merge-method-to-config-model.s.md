# Add merge() method to Config model in ace-config

## Objective

Add a `merge()` method to the Config model as a more intuitive alias for the existing `with()` method. This provides a clearer API for gems to use when merging CLI options or runtime overrides.

## Background

The Config model already has a `with()` method that wraps DeepMerger:

```ruby
# Current implementation in ace-config/lib/ace/config/models/config.rb
def with(other_data)
  merged_data = Atoms::DeepMerger.merge(data, other_data, array_strategy: merge_strategy)
  self.class.new(merged_data, source: "#{source}+merged", merge_strategy: merge_strategy)
end
```

However, `merge()` is a more intuitive name that gems should use.

## Behavioral Specification

### Interface Contract

```ruby
# Add merge() as primary method, keep with() as alias for backward compatibility
config = resolver.resolve_for(["docs/config.yml", "docs/config.yaml"])

# New intuitive API
merged_config = config.merge(cli_options)

# Still works (backward compat)
merged_config = config.with(cli_options)
```

### Expected Behavior

1. `merge()` behaves identically to existing `with()` method
2. Returns a new Config object with merged data
3. Preserves merge_strategy setting
4. Updates source to indicate merge occurred

## Implementation

```ruby
# In ace-config/lib/ace/config/models/config.rb

# Rename with() to merge() and add alias
def merge(other_data)
  merged_data = Atoms::DeepMerger.merge(data, other_data, array_strategy: merge_strategy)
  self.class.new(merged_data, source: "#{source}+merged", merge_strategy: merge_strategy)
end
alias_method :with, :merge  # Backward compatibility
```
