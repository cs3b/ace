# Spike: ace-support-items Foundation + Minimal ace-task End-to-End

## Behavioral Specification

### User Experience
- **Input**: Task title string
- **Process**: Generate b36ts → split into `xxx.t.yyy` → create folder + spec file → load it back
- **Output**: Created task with correct ID, folder, and file content; can be shown via shortcut

### Expected Behavior

Validate the entire new architecture end-to-end with a minimal working path:

1. Generate a 6-char b36ts ID (e.g., `8ppq7w`)
2. Split into task format: `8pp.t.q7w`
3. Create folder `.ace-tasks/8pp.t.q7w-fix-login/` with spec file `8pp.t.q7w-fix-login.s.md`
4. Load the task back by scanning the directory
5. Resolve a shortcut (`q7w`) to the correct task (using `mtime` for ambiguity resolution)

This spike creates two gem scaffolds and validates the concept before investing in full implementation.

### Interface Contract

```bash
# Minimal: create and show a task
ace-task create "Fix login bug"
# Output: Created task 8pp.t.q7w
# Creates: .ace-tasks/8pp.t.q7w-fix-login/8pp.t.q7w-fix-login.s.md

ace-task show q7w
# Output: shows the task (basic formatted output)
```

- [ ] `ItemIdParser` parses all ref forms: full `8pp.t.q7w`, short `t.q7w`, suffix `q7w`, subtask `8pp.t.q7w.a`, raw idea `8ppq7w`
