# ace-task CLI Commands and Polish

## Behavioral Specification

### User Experience
- **Input**: CLI commands via terminal: `ace-task create|show|list|move|update`
- **Process**: dry-cli registry dispatches to command classes, which delegate to TaskManager
- **Output**: Formatted terminal output, task files created/modified on disk

### Expected Behavior

Wire the TaskManager organism to 5 CLI commands following existing ACE patterns (dry-cli, `Ace::Core::CLI::DryCli::Base`):

1. **create**: `ace-task create [TITLE] [--priority P] [--tags T] [--child-of REF] [--in FOLDER] [--dry-run]`
2. **show**: `ace-task show REF [--path | --content | --tree]`
3. **list**: `ace-task list [--status S] [--priority P] [--tags T] [--in FOLDER] [--root PATH]`
4. **move**: `ace-task move REF --to FOLDER`
5. **update**: `ace-task update REF [--set K=V]... [--add K=V]... [--remove K=V]...`

Plus standard `version` and `help` commands via `VersionCommand.build` / `HelpCommand.build`.

### Interface Contract

```bash
# Full command signatures
ace-task create "Fix auth bug" --priority high --tags auth,security --in next
ace-task create "Setup DB" --child-of t.q7w                # Subtask creation
ace-task create "Quick task"                                # Minimal (defaults: pending, medium)
ace-task create "Preview only" --dry-run                    # Show what would be created

ace-task show q7w                   # Formatted task display
ace-task show q7w --path            # Output only file path
ace-task show q7w --content         # Raw markdown content
ace-task show q7w --tree            # Parent + subtask tree view

ace-task list                       # All tasks, flat
ace-task list --status pending      # Filter by status
ace-task list --in maybe            # Tasks in _maybe/ folder
ace-task list --root sprint-3       # Tasks under custom subpath
ace-task list --tags urgent         # Filter by tag

ace-task move q7w --to archive      # Move to _archive/
ace-task move q7w --to maybe        # Move to _maybe/
ace-task move q7w --to sprint-3     # Move to custom path

ace-task update q7w --set status=done --set priority=high
ace-task update q7w --add tags=shipped
ace-task update q7w --remove deps=8pp.t.xyz
ace-task update q7w --set worktree.branch=my-branch

ace-task version                    # Show gem version
ace-task help                       # Show all commands
```

**Exit Codes:**
- 0: Success
- 1: General error (via `Ace::Core::CLI::Error`)
- 130: SIGINT

### Success Criteria

- [ ] All 5 commands work end-to-end from terminal
- [ ] `--set` accepts multiple values: `--set status=done --set priority=high`
- [ ] `--add` accepts multiple values: `--add tags=a --add tags=b`
- [ ] `--remove` accepts multiple values
- [ ] `--in FOLDER` auto-maps to `_FOLDER` prefix
- [ ] `--child-of` creates subtask in parent folder
- [ ] `--dry-run` on create shows what would be created without writing
- [ ] `--path`, `--content`, `--tree` output modes work on show
- [ ] Error messages are actionable (tell user what to do)
- [ ] SIGINT handled with exit code 130
- [ ] `.ace-defaults/task/config.yml` provides sensible defaults
- [ ] CHANGELOG.md and README.md written for both ace-support-items and ace-task
- [ ] Integration tests cover CLI roundtrips for all commands
- [ ] `ace-test ace-task` passes all tests
