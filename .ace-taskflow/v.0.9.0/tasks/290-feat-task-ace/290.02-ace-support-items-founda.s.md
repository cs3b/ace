# ace-task Core Operations and Model

## Behavioral Specification

### User Experience
- **Input**: Task references (any shortcut form), field update triples (--set/--add/--remove), folder targets
- **Process**: Full CRUD operations on tasks without CLI — TaskManager organism orchestrates all operations
- **Output**: Task objects loaded, created, updated, moved with full metadata

### Expected Behavior

Complete the core task operations layer that the CLI commands will delegate to:

1. **TaskScanner**: Wraps DirectoryScanner for `.s.md` files (excluding `.idea.s.md`). Returns typed scan results. Must exclude subtask files from primary scan. Within a folder containing both `8pp.t.q7w-fix-login.s.md` and `8pp.t.q7w.a-setup-db.s.md`, the primary file is identified by matching the folder's ID prefix (without `.{char}` suffix). TaskLoader then loads subtask files separately when building the parent Task.
2. **TaskResolver**: Wraps ShortcutResolver for task ID format (`xxx.t.yyy`). Handles subtask refs (`xxx.t.yyy.a`).
3. **TaskLoader**: Loads a single task from disk — parses frontmatter, detects subtask relationships from folder co-location.
4. **TaskCreator**: Full creation with LLM slug generation (fallback to deterministic), b36ts ID generation, folder+file creation.
5. **SubtaskCreator**: Scans parent folder for existing subtask files, allocates next base36 char (a-z then 0-9, max 36), creates subtask file. Raises `RangeError` if limit exceeded.
6. **FieldUpdater** (in ace-support-items): `--set` delegates to FrontmatterEditor (handles nested dot keys). `--add` appends to array (creates array if scalar). `--remove` removes from array.
7. **FolderMover** (in ace-support-items): Moves item folder between locations. Maps short names (`archive` → `_archive`). Creates target dir if it does not exist. Must call `DatePartitionPath.compute(time)` when target is `_archive` — this atom already exists in ace-support-items (added during 291). Do not reimplement. Ensure ace-b36ts >= 0.7.5 for correct ISO week attribution. Must handle `Errno::EXDEV` for cross-filesystem moves — follow `IdeaMover.atomic_move` pattern: try `File.rename`, rescue with `FileUtils.cp_r` + `FileUtils.rm_rf`.
8. **TaskManager**: Single organism orchestrating create, show, list, update, move, create_subtask.

### Interface Contract (Ruby API)

```ruby
manager = Ace::Task::Organisms::TaskManager.new

# Create
task = manager.create("Fix login bug", priority: "high", tags: ["auth"])
# => Task(id: "8pp.t.q7w", status: "pending", ...)

# Show
task = manager.show("q7w")
# => Task with full metadata, subtasks loaded

# List with filters
tasks = manager.list(status: "pending", in_folder: "next")
# => Array of Task objects

# Update fields
manager.update("q7w", set: { "status" => "done" }, add: { "tags" => "shipped" })

# Move
manager.move("q7w", to: "archive")

# Create subtask
subtask = manager.create_subtask("q7w", "Setup database")
# => Task(id: "8pp.t.q7w.a", parent: "8pp.t.q7w")
```

### Frontmatter schema for tasks
```yaml
---
id: 8pp.t.q7w
status: pending          # pending, in-progress, done, blocked, draft, skipped, cancelled
priority: medium         # critical, high, medium, low
created_at: 2026-02-26 19:15:00
estimate:
dependencies: []         # Array of task IDs
tags: []
---
```
Subtask adds `parent: 8pp.t.q7w`. Orchestrator adds `subtasks: [8pp.t.q7w.a, 8pp.t.q7w.b]`.
