# Unified subject resolution in ace-review

## Objective

Enhance SubjectExtractor to parse `type:value` syntax and delegate all content resolution to ace-context.

## Behavioral Specification

### User Experience

```bash
# Clean type:value syntax
ace-review --preset security --subject diff:origin/main..HEAD
ace-review --preset code --subject pr:123
ace-review --preset docs --subject files:docs/**/*.md
ace-review --preset spec --subject task:145

# Keywords (existing, still work)
ace-review --preset code --subject staged
ace-review --preset code --subject working

# Backward compatible (existing, still work)
ace-review --subject 'diffs: ["origin/main..HEAD"]'
ace-review --subject "HEAD~5..HEAD"
```

### Subject Type Resolution

| Input | Resolves To (ace-context config) |
|-------|----------------------------------|
| `diff:range` | `{ "diffs" => ["range"] }` |
| `pr:123` | `{ "pr" => 123 }` |
| `files:pattern` | `{ "files" => ["pattern"] }` |
| `task:ref` | Task lookup → relevant files |
| `staged` | `{ "diffs" => ["--staged"] }` |
| `working` | `{ "diffs" => [""] }` (unstaged) |

### Parsing Precedence

Subject input is parsed in this order (first match wins):

1. **Typed subjects** (`type:value`) - explicit, highest priority
2. **Keywords** (`staged`, `working`) - convenience shortcuts
3. **YAML detection** - starts with `{` or contains valid YAML keys
4. **Auto-detect** - git range patterns, file globs

### Keyword Behavior Note

`staged` and `working` keywords map to ace-context diffs configuration for consistency with typed subjects. This differs from the current direct `@git.staged_diff` call in SubjectExtractor, but ensures all subject resolution flows through ace-context.

### Subtask Resolution

For subtask references like `task:145.01`:
- Resolution finds the subtask file (e.g., `145.01-context-pr-support.s.md`)
- Returns files pattern for the subtask directory only
- Does NOT include parent task files
- Glob pattern: `{subtask_dir}/**/*`

Example: `task:145.01` → `{ "files" => [".ace-taskflow/v.0.9.0/tasks/145-feat-review-ace/145.01*/**/*"] }`

### Error Messages

**Unknown type:**
```
Error: Unknown subject type 'commit'.

Supported types: diff, pr, files, task
Did you mean: diff:abc123 (for commit range)?
```

**Invalid format:**
```
Error: Invalid subject format. Expected 'type:value' or keyword.

Valid formats:
  --subject diff:<range>     Git diff range
  --subject pr:<number>      GitHub PR
  --subject files:<pattern>  File pattern
  --subject task:<ref>       Task reference
  --subject staged           Staged changes
  --subject working          Unstaged changes
```

## Implementation Notes

### Affected Components

1. **SubjectExtractor** (`lib/ace/review/molecules/subject_extractor.rb`)
   - Add `parse_typed_subject(input)` method
   - Called first in `extract_from_string`
   - Returns ace-context config hash or nil (fall through)

### Simple Implementation

```ruby
def extract_from_string(input)
  # Try typed subject first (new)
  if (typed_config = parse_typed_subject(input))
    return use_ace_context(typed_config)
  end

  # Existing logic unchanged...
  # YAML parsing, keywords, auto-detection
end

def parse_typed_subject(input)
  case input
  when /^diff:(.+)$/
    { "diffs" => [$1] }
  when /^pr:(.+)$/
    { "pr" => $1 }  # ace-context handles the rest (145.01)
  when /^files:(.+)$/
    { "files" => $1.split(",").map(&:strip) }
  when /^task:(.+)$/
    resolve_task_subject($1)
  else
    nil  # Fall through to existing parsing
  end
end

def resolve_task_subject(ref)
  # Validate ref format to prevent injection (alphanumeric, dots, dashes only)
  unless ref.match?(/\A[\w.\-]+\z/)
    raise ArgumentError, "Invalid task reference format: #{ref}"
  end

  # Use Open3 with array args for safe subprocess execution
  require 'open3'
  stdout, stderr, status = Open3.capture3("ace-taskflow", "task", ref, "--path")

  if !status.success?
    raise TaskNotFoundError, "Task '#{ref}' not found. Run `ace-taskflow task #{ref}` for details."
  end

  task_path = stdout.strip
  if task_path.empty?
    raise TaskPathNotFoundError, "Task '#{ref}' exists but has no path. Check task status."
  end

  { "files" => ["#{File.dirname(task_path)}/**/*"] }
end
```

### Key Simplification

**No custom PR logic needed** - `pr:123` simply becomes `{ "pr" => 123 }` and ace-context (145.01) handles all the `gh` CLI interaction.

## Out of Scope

- Subject presets (`--subject-preset`) - Phase 2
- New subject types beyond diff, pr, files, task
- Changes to `--pr` flag behavior (stays as full PR mode)

### Note: `--pr` vs `--subject pr:`

| Flag | Behavior |
|------|----------|
| `--pr 123` | Full PR mode using GhPrFetcher directly (existing behavior) |
| `--subject pr:123` | Subject-only mode via ace-context (new, 145.01) |

The `--pr` flag continues to use ace-review's GhPrFetcher for full PR mode (includes metadata, comments). The `--subject pr:` syntax only fetches diff content through ace-context.
