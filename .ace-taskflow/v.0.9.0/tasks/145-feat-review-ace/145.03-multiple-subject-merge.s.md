# Multiple --subject flags with merge

## Objective

Allow multiple `--subject` flags on a single command, merging them into a combined ace-context configuration.

## Behavioral Specification

### User Experience

```bash
# Combine diff + files
ace-review --preset code --subject diff:HEAD~3..HEAD --subject files:docs/**/*.md

# Combine multiple file patterns
ace-review --preset code --subject files:lib/**/*.rb --subject files:test/**/*_test.rb

# Combine PR + additional files
ace-review --preset code --subject pr:123 --subject files:docs/CHANGELOG.md

# Multiple PRs
ace-review --preset code --subject pr:123 --subject pr:456
```

### Merge Behavior

| Scenario | Input | Result |
|----------|-------|--------|
| Same type (arrays) | `files:a` + `files:b` | `files: [a, b]` |
| Same type (scalar) | `pr:123` + `pr:456` | `pr: [123, 456]` |
| Different types | `diff:range` + `files:*.rb` | `diffs: [range], files: [*.rb]` |
| Mixed | `pr:123` + `diff:HEAD~1` | `pr: [123], diffs: [HEAD~1]` |

**Rule:** Arrays concatenate, scalars become arrays when multiple values provided.

### Interface Contract

```bash
# Thor CLI - repeatable option
ace-review --subject diff:HEAD~3 --subject files:lib/**/*.rb --auto-execute

# Equivalent to YAML
ace-review --subject 'diffs: ["HEAD~3"], files: ["lib/**/*.rb"]' --auto-execute
```

### Error Handling

No conflicts - all subject types merge additively:
- Multiple `diff:` → merged into `diffs: [...]`
- Multiple `pr:` → merged into `pr: [...]`
- Multiple `files:` → merged into `files: [...]`
- Multiple `task:` → merged into `files: [...]` (resolved paths)

## Success Criteria

- [ ] CLI accepts multiple `--subject` flags
- [ ] Same-type subjects merge (arrays concatenate)
- [ ] Different-type subjects combine into single config
- [ ] Keywords (`staged`, `working`) can combine with typed subjects
- [ ] Backward compatible - single `--subject` unchanged
- [ ] Documentation updated with examples

### SubjectExtractor Changes

```ruby
# lib/ace/review/molecules/subject_extractor.rb

def extract(subject_config)
  return "" unless subject_config

  case subject_config
  when Array
    extract_and_merge_multiple(subject_config)  # NEW
  when String
    extract_from_string(subject_config)
  when Hash
    extract_from_hash(subject_config)
  else
    ""
  end
end

private

def extract_and_merge_multiple(subjects)
  merged_config = subjects.reduce({}) do |merged, subject|
    config = resolve_single_subject(subject)
    deep_merge_arrays(merged, config)
  end

  use_ace_context(merged_config)
end

def deep_merge_arrays(base, overlay)
  overlay.each do |key, value|
    if base[key].is_a?(Array) && value.is_a?(Array)
      base[key] = base[key] + value
    elsif base[key].is_a?(Array)
      base[key] = base[key] + [value]
    elsif base.key?(key)
      base[key] = [base[key], value].flatten
    else
      base[key] = value
    end
  end
  base
end

def resolve_single_subject(subject)
  case subject
  when String
    parse_typed_subject(subject) || parse_legacy_to_config(subject)
  when Hash
    subject
  else
    {}
  end
end
```

### Order of Operations

1. CLI collects all `--subject` values into array
2. SubjectExtractor detects array input
3. Each subject parsed individually (type:value or legacy)
4. Results deep-merged with array concatenation
5. Final merged config sent to ace-context

## Out of Scope

- Subject presets (Phase 2)
- Conflict detection/warnings (merge is always additive)
- Deduplication of identical subjects
