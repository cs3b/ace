# Fix pr: array handling in ace-context - multiple PRs only shows one

## Problem

When a preset has `pr: [75, 77]` (YAML array), only one PR diff appears in output.

## Root Cause

`has_processed_section_content?` in context_loader.rb:822-829 does NOT check for `_processed_diffs`, causing `merge_contexts` to discard PR diffs.

```ruby
# Current - missing _processed_diffs check:
def has_processed_section_content?(context)
  context.sections.any? do |_name, data|
    processed_files = data[:_processed_files] || data['_processed_files'] || []
    processed_commands = data[:_processed_commands] || data['_processed_commands'] || []
    processed_files.any? || processed_commands.any?  # <-- Missing _processed_diffs!
  end
end
```

## Scope

Fix the `_processed_diffs` handling across ace-context to ensure multiple PRs in array format are all included in output.

## Deliverables

- [ ] Fix `has_processed_section_content?` to check `_processed_diffs`
- [ ] Fix `has_diffs_content?` in context_loader.rb
- [ ] Fix `has_diffs_content?` in section_processor.rb
- [ ] Fix `merge_section_data` to merge `_processed_diffs` arrays
- [ ] Add tests for multiple PR handling
- [ ] Add comma syntax support in ace-review (`--subject pr:123,456`)

1. **context_loader.rb:822-829** - `has_processed_section_content?`


3. **section_processor.rb:566-568** - `has_diffs_content?`
   ```ruby
   section_data[:_processed_diffs] || section_data['_processed_diffs']
   ```

4. **section_processor.rb:489-495** - `merge_section_data`
   ```ruby
   # Also merge processed diffs (from PR fetches)
   merged_processed = merged[:_processed_diffs] || merged['_processed_diffs'] || []
   new_processed = new_section[:_processed_diffs] || new_section['_processed_diffs'] || []
   if merged_processed.any? || new_processed.any?
     merged[:_processed_diffs] = merged_processed + new_processed
   end
   ```
