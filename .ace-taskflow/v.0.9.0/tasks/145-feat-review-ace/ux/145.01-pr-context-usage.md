# PR Support in ace-context - Usage Guide

## Overview

The `pr:` configuration key enables loading PR diffs directly into ace-context, making it easy to include pull request changes in your context for analysis, review, or documentation.

## Command Types

### YAML Configuration (Primary Method)

Use `pr:` key in YAML configuration files or presets:

```yaml
# config.yml
files: ["lib/**/*.rb"]
diffs: ["origin/main..HEAD"]
pr: 123  # NEW - PR number
```

### CLI Usage

```bash
# Load context with PR support via config file
ace-context -f config.yml
```

## Usage Scenarios

### Scenario 1: Review Current Repository PR

**Goal:** Load diff for PR #123 from the current repository

**Configuration:**
```yaml
# review-pr-123.yml
pr: 123
```

**Command:**
```bash
ace-context -f review-pr-123.yml
```

**Expected Output:**
```xml
<diff source="pr:123">
diff --git a/lib/foo.rb b/lib/foo.rb
index abc123..def456 100644
--- a/lib/foo.rb
+++ b/lib/foo.rb
@@ -1,3 +1,4 @@
 class Foo
+  # new method
 end
</diff>
```

### Scenario 2: Cross-Repository PR Reference

**Goal:** Load diff from a PR in another repository

**Configuration:**
```yaml
# external-pr.yml
pr: "owner/repo#456"
```

**Expected Output:**
```xml
<diff source="pr:owner/repo#456">
... diff content from owner/repo PR #456 ...
</diff>
```

### Scenario 3: Multiple PRs

**Goal:** Load diffs from multiple PRs for comparison

**Configuration:**
```yaml
# compare-prs.yml
pr:
  - 123
  - "owner/repo#456"
  - "https://github.com/org/project/pull/789"
```

**Expected Output:**
```xml
<diff source="pr:123">
... diff from PR #123 ...
</diff>
<diff source="pr:owner/repo#456">
... diff from owner/repo #456 ...
</diff>
<diff source="pr:org/project#789">
... diff from org/project #789 ...
</diff>
```

### Scenario 4: Error Handling

**gh not installed:**
```
Error: GitHub CLI (gh) not installed. Install with: brew install gh
```

**Not authenticated:**
```
Error: Not authenticated with GitHub. Run: gh auth login
```

**PR not found:**
```
Error: PR #99999 not found in repository
```

## Tips and Best Practices

1. **Requires gh CLI:** Install with `brew install gh` and authenticate with `gh auth login`
2. **PR diffs are cached:** Repeated requests for the same PR are fast
3. **Combine with other sources:** Use `pr:` alongside `files:` and `diffs:` for comprehensive context
4. **Network required:** PR fetching requires internet access
