# Add pr: support to ace-context

## Objective

Add `pr:` as a new source type in ace-context, enabling PR diff content to be loaded alongside files, diffs, and commands.

## Behavioral Specification

### User Experience

```yaml
# ace-context configuration with pr: support
files: ["lib/**/*.rb"]
diffs: ["origin/main..HEAD"]
pr: 123                                    # NEW - PR number
pr: "owner/repo#456"                       # Cross-repo PR
pr: "https://github.com/owner/repo/pull/789"  # Full URL
```

### Interface Contract

```bash
# Load PR diff via ace-context
ace-context -f config.yml  # where config.yml has pr: 123

# Inline via ace-review (after 145.02)
ace-review --subject pr:123
```

### Expected Output Format

PR diff content is wrapped in `<diff>` tags with `source` attribute indicating the PR:

```xml
<diff source="pr:123">
diff --git a/lib/foo.rb b/lib/foo.rb
index abc123..def456 100644
--- a/lib/foo.rb
+++ b/lib/foo.rb
@@ -1,3 +1,4 @@
 class Foo
+  # new method
 end
</diff>
```

For multiple PRs, each gets its own `<diff>` block:

```xml
<diff source="pr:123">
  ... diff content ...
</diff>
<diff source="pr:456">
  ... diff content ...
</diff>
```

### Multiple PR Support

```yaml
# Single PR
pr: 123

# Multiple PRs (array syntax)
pr:
  - 123
  - "owner/repo#456"
  - "https://github.com/org/project/pull/789"
```

## Implementation Notes

### Pipeline Integration

PR processing occurs in ContextLoader after other content sources are resolved:

1. `process_template_config` processes `files:`, `diffs:`, `commands:`
2. **NEW**: After line ~854, add `pr:` processing
3. PR diffs are treated as additional diff content (not a new section type)
4. Output uses existing `<diff>` wrapper with `source="pr:{id}"` attribute

### Affected Components

1. **Config Schema** - Add `pr:` as valid key (single value or array)
2. **New Atom: PrIdentifierParser** (`lib/ace/context/atoms/pr_identifier_parser.rb`)
   - Pure function: parse identifier → structured hash
   - No side effects, easily testable
3. **New Molecule: GhPrExecutor** (`lib/ace/context/molecules/gh_pr_executor.rb`)
   - Side effects: execute `gh pr diff` safely
   - Uses Open3.capture3 with array args
   - Handles errors gracefully
4. **ContextLoader** - Integrate PR extraction alongside files/diffs/commands
5. **Tests** - Unit tests for parsing, integration tests with mocked gh CLI

### Testing Strategy

Follow the GhCliExecutor pattern from `ace-review/lib/ace/review/molecules/gh_pr_fetcher.rb`:

1. **Unit tests** for `PrIdentifierParser` (pure atom, no mocking needed)
2. **Integration tests** use protected method mocking pattern:
   ```ruby
   class TestGhPrExecutor < GhPrExecutor
     def execute_gh_command(args)
       # Return mock data based on args
     end
   end
   ```
3. **Negative path tests**:
   - [ ] `gh` not installed → clear error message
   - [ ] Auth failure → suggests `gh auth login`
   - [ ] PR not found → returns nil or raises
   - [ ] Network timeout → graceful degradation

Reference: `docs/testing-patterns.md` for CLI mocking patterns.

### PR Identifier Parsing (Atom - Pure Function)

```ruby
# lib/ace/context/atoms/pr_identifier_parser.rb
module Ace::Context::Atoms
  module PrIdentifierParser
    ParseResult = Data.define(:number, :repo)

    def self.call(input)
      case input.to_s
      when /^(\d+)$/                           # Simple number
        ParseResult.new(number: $1, repo: nil)
      when /^([^\/]+\/[^#]+)#(\d+)$/           # owner/repo#123
        ParseResult.new(number: $2, repo: $1)
      when /github\.com\/([^\/]+\/[^\/]+)\/pull\/(\d+)/  # Full URL
        ParseResult.new(number: $2, repo: $1)
      else
        raise ArgumentError, "Invalid PR identifier: #{input}"
      end
    end
  end
end
```

### GH PR Executor (Molecule - Side Effects)

```ruby
# lib/ace/context/molecules/gh_pr_executor.rb
module Ace::Context::Molecules
  class GhPrExecutor
    def initialize(identifier)
      @parsed = Atoms::PrIdentifierParser.call(identifier)
    end

    def fetch_diff
      stdout, stderr, status = execute_gh_command(build_args)
      raise GhCommandError, format_error(stderr) unless status.success?
      stdout
    end

    protected

    # Override in tests for mocking
    def execute_gh_command(args)
      Open3.capture3(*args)
    end

    private

    def build_args
      args = ["gh", "pr", "diff", @parsed.number]
      args += ["-R", @parsed.repo] if @parsed.repo
      args
    end

    def format_error(stderr)
      case stderr
      when /not found/i
        "PR ##{@parsed.number} not found in repository"
      when /auth/i
        "Not authenticated with GitHub. Run: gh auth login"
      else
        "gh pr diff failed: #{stderr}"
      end
    end
  end

  class GhCommandError < StandardError; end
end
```

### Error Messages

```
Error: GitHub CLI (gh) not installed. Install with: brew install gh
Error: PR #123 not found in repository
Error: Not authenticated with GitHub. Run: gh auth login
```
