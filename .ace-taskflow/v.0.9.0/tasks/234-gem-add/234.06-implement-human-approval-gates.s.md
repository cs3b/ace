# Human gates

## Objective

Pause workflow for human approval with explicit resume commands.

## Scope

Add human gate support to workflow execution:
- Gate step type pauses workflow
- State persisted for resume
- Resume with approve/reject commands

## Deliverables

- [ ] Gate step type in workflow parser
- [ ] `ace-coworker resume --approve` command
- [ ] `ace-coworker resume --reject --reason "..."` command
- [ ] Gate state in job.json

**Workflow:**
```yaml
steps:
  - name: review-gate
    gate: human
    prompt: "Review implementation before creating PR"
```

**CLI:**
```bash
# Approve and continue
ace-coworker resume --approve

# Reject with reason (pauses workflow for manual intervention)
ace-coworker resume --reject --reason "Need to fix X first"
```

**Job state:**
```json
{
  "status": "paused",
  "gate": {
    "step": "review-gate",
    "prompt": "Review implementation before creating PR",
    "waiting": true
  }
}
```

## Rejection Behavior

When a gate is rejected:
1. **State**: Status set to `rejected` (not `paused` or `running`)
2. **No auto-rewind**: Does NOT automatically re-execute previous step
3. **Manual resolution**: User must manually address the issue, then use `ace-coworker retry` to re-attempt current step
4. **Side effects**: Automated side effects from previous steps are NOT rolled back (commits, files written, etc.)

This conservative approach avoids unintended state corruption. Users can manually rewind if needed via `ace-coworker goto --step <n>`.

## Acceptance Criteria

- [ ] Gate step pauses workflow
- [ ] Gate state persisted in job.json
- [ ] `resume --approve` continues workflow
- [ ] `resume --reject` sets status to `rejected`, logs reason, does not auto-rewind
- [ ] Gate events logged in log.jsonl
