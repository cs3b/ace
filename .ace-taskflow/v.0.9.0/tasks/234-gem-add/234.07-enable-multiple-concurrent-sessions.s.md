# Multi-session support

## Objective

Enable multiple concurrent sessions in same directory with auto-resume.

## Scope

Add multi-session management:
- Session IDs using ace-timestamp format
- List all sessions
- Auto-resume existing session for same task
- Session-specific status commands

## Deliverables

- [ ] Session ID generation (ace-timestamp)
- [ ] `ace-coworker list` command
- [ ] Auto-resume logic in `start` command
- [ ] `--session` flag for status/resume commands

**Session structure:**
```
.cache/ace-coworker/
├── 8or5kx/              # Session for task 228
│   ├── job.json
│   ├── log.jsonl
│   └── reports/
├── 8or7ab/              # Session for task 229
│   ├── job.json
│   ├── log.jsonl
│   └── reports/
└── 8or9cd/              # Another session for task 228 (different workflow)
    ├── job.json
    ├── log.jsonl
    └── reports/
```

**CLI:**
```bash
# List all sessions
ace-coworker list
# SESSION   TASK   WORKFLOW          STATUS    STEP      PROGRESS
# 8or5kx    228    task-completion   running   test      3/5
# 8or7ab    229    bug-fix           paused    review    2/4

# Status of specific session
ace-coworker status --session 8or5kx

# Start auto-resumes if session exists for task
ace-coworker start --task 228
# → Resumes existing session 8or5kx (most recently updated active session)

# Force new session even if one exists
ace-coworker start --task 228 --new-session
# → Creates new session 8orabc

# Resume specific session explicitly
ace-coworker start --session 8or9cd
# → Resumes session 8or9cd regardless of task
```

## Multi-Session Resume Rules

When `start --task X` is called with multiple existing sessions for task X:
1. **Filter**: Only consider sessions with status `running` or `paused` (not `completed`, `rejected`, `failed`)
2. **Select**: Resume the most recently updated session (by `updated_at` in job.json)
3. **Override**: Use `--session <id>` to explicitly select a different session
4. **New**: Use `--new-session` to create a fresh session instead of resuming

## Acceptance Criteria

- [ ] Session IDs use ace-timestamp format
- [ ] `ace-coworker list` shows all sessions with status
- [ ] `start --task X` auto-resumes most recently updated active session
- [ ] `--session` flag works on status/resume
- [ ] `--new-session` flag creates fresh session for task
- [ ] Multiple sessions can coexist
