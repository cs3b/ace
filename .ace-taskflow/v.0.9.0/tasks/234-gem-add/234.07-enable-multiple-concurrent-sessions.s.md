# Multi-session support

## Objective

Enable multiple concurrent sessions in same directory with auto-resume.

## Scope

Add multi-session management:
- Session IDs using ace-timestamp format
- List all sessions
- Auto-resume existing session for same task
- Session-specific status commands

## Deliverables

- [ ] Session ID generation (ace-timestamp)
- [ ] `ace-coworker list` command
- [ ] Auto-resume logic in `start` command
- [ ] `--session` flag for status/resume commands

**Session structure:**
```
.cache/ace-coworker/
├── 8or5kx/              # Session for task 228
│   ├── job.json
│   ├── log.jsonl
│   └── reports/
├── 8or7ab/              # Session for task 229
│   ├── job.json
│   ├── log.jsonl
│   └── reports/
└── 8or9cd/              # Another session for task 228 (different workflow)
    ├── job.json
    ├── log.jsonl
    └── reports/
```

**CLI:**
```bash
# List all sessions
ace-coworker list
# SESSION   TASK   WORKFLOW          STATUS    STEP      PROGRESS
# 8or5kx    228    task-completion   running   test      3/5
# 8or7ab    229    bug-fix           paused    review    2/4

# Status of specific session
ace-coworker status --session 8or5kx

# Start auto-resumes if session exists for task
ace-coworker start --task 228
# → Resumes existing session 8or5kx
```

## Acceptance Criteria

- [ ] Session IDs use ace-timestamp format
- [ ] `ace-coworker list` shows all sessions with status
- [ ] `start --task X` auto-resumes existing session
- [ ] `--session` flag works on status/resume
- [ ] Multiple sessions can coexist
