# Full task-completion workflow

## Objective

Create production 8-step workflow for complete task lifecycle.

## Scope

Define and test complete task-completion workflow:
- All 8 steps with proper instructions
- Context bundles per step (wfi:// protocols)
- Integration test end-to-end

## Deliverables

- [ ] `work-on-task.wf.yml` with 8 steps
- [ ] Workflow instructions for each step
- [ ] Integration test

## Workflow Parameter Interpolation

Workflow files use shell-style variable interpolation:
- `$task` → replaced with `--task` argument value
- `$session` → replaced with current session ID
- Variables are expanded before instruction execution

Example: `ace-bundle wfi://work-on-task $task` with `--task 228` becomes `ace-bundle wfi://work-on-task 228`

**Not supported in MVP:**
- Environment variable fallback (`${VAR:-default}`)
- Nested expansion
- Command substitution

**Full workflow:**
```yaml
name: task-completion
steps:
  - name: implement
    instructions: ace-bundle wfi://work-on-task $task
    verifications:
      - ace-test passes
    retries: 5

  - name: commit
    instructions: ace-bundle wfi://commit

  - name: test
    instructions: ace-test
    retries: 5

  - name: review-gate
    gate: human
    prompt: "Review before PR"

  - name: create-pr
    instructions: ace-bundle wfi://create-pr

  - name: self-review
    instructions: ace-bundle wfi://review-pr

  - name: apply-feedback
    instructions: ace-bundle wfi://apply-feedback
    verifications:
      - ace-test passes

  - name: update-pr
    instructions: ace-bundle wfi://update-pr
```

## Integration Test Scope

**Test coverage:**
1. Create session with `start --task` using test workflow
2. Verify job.json created with correct schema
3. Execute each step via `report` command
4. Verify step transitions in job.json
5. Test retry behavior with mock failures
6. Test human gate pause/resume
7. Verify final `completed` status

**Test fixtures:**
- `test/fixtures/workflows/test-3-step.wf.yml` - minimal 3-step workflow
- `test/fixtures/reports/sample-report.md` - valid report file

**Pass criteria:**
- All commands return exit code 0 for success cases
- job.json state matches expected after each operation
- log.jsonl contains expected events

## Acceptance Criteria

- [ ] Each step has proper wfi:// instructions
- [ ] Verifications on implementation steps
- [ ] Human gate before PR
