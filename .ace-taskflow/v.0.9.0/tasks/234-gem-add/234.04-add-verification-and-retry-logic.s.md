# Verifications and retries

## Objective

Automatically verify step outcomes and retry on failure with configurable limits.

## Scope

Add verification and retry logic to workflow execution:
- Verification commands run after step completion
- Retry on verification failure with configurable limits
- Stop on repeated same errors

## Deliverables

- [ ] Verification execution after step completion
- [ ] Retry logic with configurable limits
- [ ] Repeated failure detection

**Workflow additions:**
```yaml
steps:
  - name: implement
    instructions: ace-bundle wfi://work-on-task $task
    verifications:
      - ace-test passes
      - no lint errors
    retries: 5
    on_repeated_failure: 3
```

**Job state additions:**
```json
{
  "steps": [{
    "name": "implement",
    "attempts": 2,
    "last_error": "3 tests failed",
    "error_history": ["3 tests failed", "3 tests failed"]
  }]
}
```

## Error Comparison Rules

"Same error" is determined by **exact string match** of the error message:
- Verification failures: the failing verification command output (first 500 chars, trimmed)
- Exit code failures: `"exit code {n}"`
- Timeout failures: `"timeout after {n}s"`

Repeated failure detection:
1. After each failure, compare `last_error` with previous entries in `error_history`
2. Count consecutive identical errors (from most recent backwards)
3. If count >= `on_repeated_failure`, transition to `failed` status

Example: `["timeout", "3 tests failed", "3 tests failed", "3 tests failed"]` → 3 consecutive same errors → workflow stops.

## Acceptance Criteria

- [ ] Verifications run after step completion
- [ ] Failed verification triggers retry (up to limit)
- [ ] Same error repeated 3 times stops workflow
- [ ] Attempts and errors tracked in job.json
- [ ] Failures logged in log.jsonl
