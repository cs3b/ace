# MVP - Single session workflow

## Objective

Create the smallest working ace-coworker that can execute a two-step workflow with state persistence.

## Behavioral Specification

### User Experience

**Input:** User invokes `/ace:coworker-work-on 228`

**Process:**
1. Agent calls `ace-coworker start --task 228` → gets step 1 instructions
2. Agent follows instructions (wfi://work-on-task)
3. Agent calls `ace-coworker report <file>` → stores report, advances to step 2
4. Agent follows step 2 instructions (wfi://draft-release)
5. Agent calls `ace-coworker report <file>` → workflow complete

**Output:** Task completed through two-step workflow with state tracked

### Interface Contract

```bash
# Start workflow - creates session, returns first step instructions
ace-coworker start --task 228
# Output: Session created, step instructions printed

# Check current step and instructions
ace-coworker status
# Output: Current step name, instructions, progress

# Store agent report and advance to next step
ace-coworker report <file>
# Output: Report stored, advanced to next step (or workflow complete)
```

### Success Criteria

- [ ] `ace-coworker start --task X` creates session in `.cache/ace-coworker/{session-id}/`
- [ ] `ace-coworker status` shows current step and instructions
- [ ] `ace-coworker report` stores report and advances step
- [ ] Two-step workflow (implement → release) runs end-to-end
- [ ] State persists in job.json (can resume after interruption)

## Scope of Work

### Deliverables

**Create:**
- `ace-coworker/` gem directory with standard structure
- `lib/ace/coworker/job.rb` - job.json read/write
- `exe/ace-coworker` - CLI with start, status, report commands
- `.ace-defaults/coworker/workflows/work-on-task.wf.yml` - two-step workflow

**Session Files:**
```
.cache/ace-coworker/{session-id}/
├── job.json      # task, workflow, current step, status
└── reports/      # (placeholder for task 234.02)
```

**job.json Schema (v1):**
```json
{
  "version": 1,
  "task_id": "228",
  "workflow": "work-on-task",
  "status": "running",
  "current_step": 0,
  "steps": [
    {"name": "implement", "status": "completed"},
    {"name": "release", "status": "pending"}
  ],
  "created_at": "2026-01-28T01:30:00Z",
  "updated_at": "2026-01-28T01:35:00Z"
}
```

**Required keys:**
- `version`: Schema version (integer, currently 1)
- `task_id`: Task identifier (string)
- `workflow`: Workflow name (string)
- `status`: Session status: `running`, `paused`, `completed`, `rejected`, `failed`
- `current_step`: Zero-based index into steps array
- `steps`: Array of step objects with at minimum `name` and `status`
- `created_at`, `updated_at`: ISO 8601 timestamps

Subsequent subtasks extend this schema (234.02 adds log fields, 234.04 adds attempts/errors, 234.06 adds gate state).
