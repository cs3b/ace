# Logging and reports

## Objective

Track what goes to agent (delegation) and what comes back (report) with structured logging.

## Scope

Add logging and reporting capabilities to ace-coworker:
- JSONL event stream for all workflow events
- Markdown delegation files (what instructions we gave)
- Markdown report files (what agent returned)

## Deliverables

- [ ] `lib/ace/coworker/log.rb` - JSONL log writer
- [ ] Delegation capture on step start
- [ ] Report storage on `ace-coworker report`

**Session Files:**
```
.cache/ace-coworker/{session-id}/
├── job.json
├── log.jsonl                         # NEW: event stream
└── reports/
    ├── 001-implement-delegation.md   # NEW: what we asked
    ├── 001-implement-report.md       # NEW: what came back
    ├── 002-release-delegation.md
    └── 002-release-report.md
```

**Log events:**
```jsonl
{"ts":"...","event":"step_started","step":"implement"}
{"ts":"...","event":"delegation_sent","step":"implement","file":"reports/001-implement-delegation.md"}
{"ts":"...","event":"report_received","step":"implement","file":"reports/001-implement-report.md"}
{"ts":"...","event":"step_completed","step":"implement"}
```

## Acceptance Criteria

- [ ] log.jsonl created on workflow start
- [ ] All events logged with timestamps
- [ ] Delegation markdown written on step start
- [ ] Report markdown written on `ace-coworker report`
- [ ] Reports numbered sequentially (001-, 002-, etc.)
