# Logging and reports

## Objective

Track what goes to agent (delegation) and what comes back (report) with structured logging.

## Scope

Add logging and reporting capabilities to ace-coworker:
- JSONL event stream for all workflow events
- Markdown delegation files (what instructions we gave)
- Markdown report files (what agent returned)

## Deliverables

- [ ] `lib/ace/coworker/log.rb` - JSONL log writer
- [ ] Delegation capture on step start
- [ ] Report storage on `ace-coworker report`

**Session Files:**
```
.cache/ace-coworker/{session-id}/
├── job.json
├── log.jsonl                             # NEW: event stream
└── reports/
    ├── 001-01-implement-delegation.md    # NEW: step 1, attempt 1
    ├── 001-01-implement-report.md        # NEW: what came back
    ├── 001-02-implement-delegation.md    # Retry: step 1, attempt 2
    ├── 001-02-implement-report.md
    ├── 002-01-release-delegation.md      # Step 2, attempt 1
    └── 002-01-release-report.md
```

**File naming convention:** `{step_number}-{attempt_number}-{step_name}-{type}.md`
- Step number: 3-digit zero-padded (001, 002, ...)
- Attempt number: 2-digit zero-padded (01, 02, ...)
- This prevents overwrites during retries

**Log events:**
```jsonl
{"ts":"8or5kx","event":"step_started","step":"implement","step_index":0,"attempt":1,"session_id":"8or5kx"}
{"ts":"8or5ky","event":"delegation_sent","step":"implement","attempt":1,"file":"reports/001-01-implement-delegation.md"}
{"ts":"8or5lz","event":"report_received","step":"implement","attempt":1,"file":"reports/001-01-implement-report.md"}
{"ts":"8or5m0","event":"step_completed","step":"implement","attempt":1}
```

**Log event schema:**
- `ts`: 6-character ace-timestamp (UTC, via ace-support-timestamp)
- `event`: Event type (step_started, delegation_sent, report_received, step_completed, step_failed, workflow_paused, workflow_completed, etc.)
- `step`: Step name
- `step_index`: Zero-based step index (optional, for correlation)
- `attempt`: Attempt number for this step (1-based)
- `session_id`: Session identifier (for multi-session correlation)

## Acceptance Criteria

- [ ] log.jsonl created on workflow start
- [ ] All events logged with timestamps
- [ ] Delegation markdown written on step start
- [ ] Report markdown written on `ace-coworker report`
- [ ] Reports numbered sequentially (001-, 002-, etc.)
