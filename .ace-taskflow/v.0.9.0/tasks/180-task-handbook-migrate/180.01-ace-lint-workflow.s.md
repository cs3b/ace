# Design ace-lint Workflow

## Objective

Create a comprehensive lint workflow for ace-lint package that includes:
1. A `run-lint.wf.md` workflow with autofix option
2. A lint report processing workflow that creates tasks for manual fixes

## Background

From task 180 discussion: The existing `fix-linting-issue-from.wf.md` in dev-handbook is insufficient. We need a proper lint workflow that:
- Runs linting with optional autofix
- Generates a structured report
- Processes the report to create tasks for issues that need manual intervention
- Includes proposal for how to fix non-autofix issues

## Scope of Work

### Phase 1: run-lint.wf.md

Create workflow in `ace-lint/handbook/workflow-instructions/run-lint.wf.md`:

- **Input**: File paths or patterns to lint
- **Options**:
  - `--autofix` / `--no-autofix`
  - `--format` (json, text, markdown)
- **Output**: Lint report (stored in `.ace-lint/reports/`)

### Phase 2: process-lint-report.wf.md

Create workflow to process lint reports:

- **Input**: Lint report path
- **Process**:
  1. Parse lint issues
  2. Group by severity/type
  3. For each non-autofix issue:
     - Create task draft with:
       - Issue description
       - File location
       - Proposal for how to fix (if determinable)
- **Output**: Task drafts in `.ace-taskflow/` structure

### Phase 3: Integration

- Update ace-lint CLI to support workflow invocation
- Add .claude/commands for `/ace:lint` and `/ace:lint-process`

## Success Criteria

- [ ] run-lint.wf.md created in ace-lint
- [ ] process-lint-report.wf.md created in ace-lint
- [ ] Reports generated in structured format
- [ ] Tasks created for manual-fix issues include fix proposals
- [ ] .claude/commands integration working

## Out of Scope

- Actual linting logic (handled by ace-lint atoms)
- Migrating fix-linting-issue-from.wf.md content
