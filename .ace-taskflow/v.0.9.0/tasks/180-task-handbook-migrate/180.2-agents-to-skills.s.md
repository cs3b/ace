# Convert Agents to Skills

## Objective

Convert all existing agents (both in dev-handbook and already migrated to ace-* packages) from the `.ag.md` agent format to the skill format. Skills are the preferred pattern going forward.

## Background

From task 180 discussion: Agents should be converted to skills rather than migrated as-is. This applies to:
- 6 remaining dev-handbook agents
- 5 already migrated agents in ace-* packages
- review-agents templates

## Scope of Work

### Phase 1: Define Skill Format

Establish the skill format pattern:
- File naming: `*.skill.md` or skill definition in `.claude/commands/`
- Frontmatter requirements
- Integration with Claude Code skill system

### Phase 2: Convert dev-handbook Agents

For each of the 6 agents:
1. Read agent definition
2. Extract capabilities and prompts
3. Create equivalent skill definition
4. Add to appropriate ace-* package

### Phase 3: Convert Migrated Agents

For each of the 5 already-migrated agents:
1. Create skill version alongside existing agent
2. Update .claude/commands to use skill
3. Deprecate agent file (or keep for backward compatibility)

### Phase 4: Review-Agents Templates

Convert review-agents templates to skill format in ace-handbook.

## Success Criteria

- [ ] All 11 agents converted to skills
- [ ] Skills registered in .claude/commands
- [ ] Old agent files deprecated or removed
- [ ] Documentation updated

## Out of Scope

- Enhancing agent/skill capabilities
- Creating new agents/skills
