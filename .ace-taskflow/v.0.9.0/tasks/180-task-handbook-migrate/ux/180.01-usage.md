# ace-lint Workflow Usage Documentation

## Overview

Two complementary workflows for AI-assisted lint management:

1. **run-lint.wf.md** - Run linting with optional autofix and report generation
2. **process-lint-report.wf.md** - Create tasks from lint issues needing manual fixes

## Command Types

### Claude Code Commands (Slash Commands)

Use in Claude Code conversation:

```
/ace:lint [files...] [--autofix]
/ace:lint-process [report-path]
```

### CLI Commands (Terminal)

Run in terminal:

```bash
ace-lint docs/*.md --fix
ace-lint --type yaml config.yml
```

## Usage Scenarios

### Scenario 1: Quick Lint Check

**Goal**: Validate markdown files before commit

**Claude Code**:
```
/ace:lint docs/*.md
```

**Terminal**:
```bash
ace-lint docs/*.md
```

**Expected Output**:
```
docs/README.md: OK (formatted)
docs/architecture.md: OK (formatted)
============================================================
Validated: 2 files
All files passed
```

### Scenario 2: Lint with Autofix

**Goal**: Fix formatting issues automatically

**Claude Code**:
```
/ace:lint docs/*.md --autofix
```

**Terminal**:
```bash
ace-lint docs/*.md --fix
```

**Expected Output**:
```
docs/README.md: OK (formatted)
docs/architecture.md: fixed
============================================================
Validated: 2 files
All files passed
1 file auto-fixed
```

### Scenario 3: Generate Lint Report

**Goal**: Generate structured report for later processing

**Claude Code**:
```
/ace:lint .ace-taskflow/**/*.md --report
```

**Expected Output**:
```
Linting 45 files...
docs/task-001.md: 2 issues
docs/task-002.md: 1 issue
============================================================
Report saved: .ace-lint/reports/lint-2026-01-07-123456.json
3 issues found (1 auto-fixable, 2 manual)
```

### Scenario 4: Process Lint Report into Tasks

**Goal**: Create tasks for manual-fix issues

**Claude Code**:
```
/ace:lint-process .ace-lint/reports/lint-2026-01-07-123456.json
```

**Expected Output**:
```
Processing lint report...
Found 2 issues requiring manual fixes

Creating task drafts:
- .ace-taskflow/v.0.9.0/tasks/lint/fix-yaml-frontmatter-001.md
- .ace-taskflow/v.0.9.0/tasks/lint/fix-broken-link-001.md

Each task includes:
- Issue location and description
- Proposed fix approach
- Related context
```

### Scenario 5: Full Lint Pipeline

**Goal**: Complete lint workflow from check to task creation

**Claude Code**:
```
# Step 1: Run with autofix first
/ace:lint **/*.md --autofix --report

# Step 2: Review remaining issues
# (Report shows what couldn't be auto-fixed)

# Step 3: Create tasks for manual fixes
/ace:lint-process .ace-lint/reports/latest.json
```

## Command Reference

### /ace:lint

**Purpose**: Run linting with optional autofix and report generation

**Syntax**:
```
/ace:lint [files...] [options]
```

**Options**:
- `--autofix`: Apply automatic fixes where possible
- `--report`: Generate structured report (JSON)
- `--type TYPE`: Force file type (markdown, yaml, frontmatter)
- `--quiet`: Suppress detailed output

**Internal Implementation**:
- Uses `ace-lint` CLI with `--fix` for autofix
- Generates JSON report to `.ace-lint/reports/`
- Report includes file, line, message, severity, auto-fixable flag

### /ace:lint-process

**Purpose**: Process lint report and create tasks for manual fixes

**Syntax**:
```
/ace:lint-process [report-path]
```

**Arguments**:
- `report-path`: Path to lint report JSON (default: `.ace-lint/reports/latest.json`)

**Internal Implementation**:
- Parses lint report JSON
- Filters for non-auto-fixable issues
- Groups by severity and type
- Creates task drafts using ace-taskflow
- Includes fix proposals based on lint rule

## Report Format

Lint reports are JSON files with structure:

```json
{
  "generated_at": "2026-01-07T12:34:56Z",
  "files_checked": 45,
  "total_issues": 3,
  "auto_fixable": 1,
  "manual_required": 2,
  "issues": [
    {
      "file": "docs/task-001.md",
      "line": 42,
      "column": 15,
      "message": "Invalid YAML frontmatter: missing required field 'status'",
      "severity": "error",
      "rule": "frontmatter-required-fields",
      "auto_fixable": false,
      "fix_proposal": "Add 'status: draft' to frontmatter"
    }
  ]
}
```

## Tips and Best Practices

1. **Run autofix first** - Always run with `--autofix` before processing to minimize manual work

2. **Review reports before processing** - Check the report to understand issue scope

3. **Batch similar issues** - The process workflow groups similar issues into single tasks when possible

4. **Fix proposals are suggestions** - Review proposed fixes, they may need context-aware adjustments

5. **Use quiet mode in CI** - Use `--quiet` for cleaner CI output

## Migration from fix-linting-issue-from

The old `fix-linting-issue-from.wf.md` workflow is replaced by this two-step approach:

| Old Workflow | New Workflow |
|--------------|--------------|
| Manual error file assignment | Automated report generation |
| Parse .lint-errors-N.md files | JSON report with full metadata |
| No autofix | Autofix integrated |
| No fix proposals | AI-generated fix proposals |

Key improvements:
- Structured JSON reports instead of markdown error lists
- Autofix reduces manual work
- Fix proposals guide resolution
- Task integration via ace-taskflow
