# 122.01 - Hierarchical Task ID Parser

## Scope

Extend `TaskReferenceParser` to support hierarchical task IDs including parent tasks, orchestrators, and subtasks.

## Interface

```ruby
# Current behavior (maintain backward compatibility)
TaskReferenceParser.parse("121")
# => { number: "121", subtask: nil, qualified: false }

# New hierarchical support
TaskReferenceParser.parse("121.00")
# => { number: "121", subtask: "00", qualified: false }

TaskReferenceParser.parse("121.01")
# => { number: "121", subtask: "01", qualified: false }

# Qualified references with subtasks
TaskReferenceParser.parse("v.0.9.0+task.121.01")
# => { release: "v.0.9.0", number: "121", subtask: "01", qualified: true }
```

## Behavior

### Reference Formats Supported

| Format | Example | Result |
|--------|---------|--------|
| Simple number | `121` | Parent/single task |
| Orchestrator | `121.00` | Explicit orchestrator |
| Subtask | `121.01` | Subtask of 121 |
| Qualified | `v.0.9.0+task.121` | Release-specific parent |
| Qualified subtask | `v.0.9.0+task.121.01` | Release-specific subtask |

### Resolution Rules

1. `121` -> Find orchestrator (121.00) OR single task file (121-*.s.md)
2. `121.00` -> Find orchestrator file explicitly
3. `121.01` -> Find subtask file (121.01-*.s.md)
