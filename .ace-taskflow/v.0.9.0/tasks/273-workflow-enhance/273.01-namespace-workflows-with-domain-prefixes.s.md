# Namespace Workflows with Domain Prefixes

## Behavioral Specification

### User Experience
- **Input**: User invokes workflows via `wfi://` protocol or `/ace:` slash commands
- **Process**: Workflows are organized into logical domain namespaces using subdirectories, making discovery and browsing intuitive
- **Output**: All ~88 workflows are addressable via `wfi://namespace/action` URIs and `ace:namespace-action` slash commands

### Expected Behavior

Currently all workflows live flat in `handbook/workflow-instructions/` with no domain grouping. `create-task`, `create-pr`, `create-adr`, `create-e2e-test` all start with `create-`, making browsing by domain impossible. Additionally, `research.wf.md` exists in both ace-handbook and ace-search, with ace-search's version completely shadowed.

After this change:
- Workflows are organized into **14 namespace subdirectories** within each package's `handbook/workflow-instructions/`
- Users can browse by namespace: `wfi://task/*` lists all task workflows
- Name collisions are resolved: `wfi://handbook/research` vs `wfi://search/research`
- Slash commands use hyphen: `/ace:task-work 148` instead of `/ace:work-on-task 148`
- Names are shorter by dropping redundant domain words: `create-e2e-test` → `e2e/create`
- Multi-item variants use `-batch` suffix: `task/work` vs `task/work-batch`

### Namespaces (14 prefixes)

| Prefix | Domain | Count |
|--------|--------|-------|
| `task` | Task lifecycle | 16 |

| `retro` | Retrospectives | 2 |
| `release` | Release and changelog | 4 |
| `git` | Commit, PR, rebase, worktree, secrets | 10 |
| `review` | Code review and feedback | 4 |
| `test` | Unit/integration testing | 5 |
| `e2e` | End-to-end testing | 10 |
| `docs` | Documentation and ADRs | 10 |
| `search` | Search and research | 3 |
| `handbook` | Meta management, research orchestration | 11 |
| `assign` | Assignment workflows | 5 |
| `lint` | Linting | 2 |

Plus 3 flat (no namespace): bundle, b36ts, integration/update-claude.

### Interface Contract

```bash
# Before
ace-bundle wfi://work-on-task        # flat name
/ace:work-on-task 148                # slash command

# After
ace-bundle wfi://task/work           # namespaced URI
/ace:task-work 148                   # hyphenated slash command

# Browsing by namespace
ace-nav wfi://task/*                 # list all task workflows
ace-nav wfi://e2e/*                  # list all e2e workflows
```
