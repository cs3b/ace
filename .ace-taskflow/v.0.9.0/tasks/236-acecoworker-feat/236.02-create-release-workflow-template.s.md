# Create release workflow template

## Objective

Create a multi-step workflow template for ace-coworker that guides the release process from version bump through publication.

## Scope

Create `.ace-defaults/coworker/workflows/release.wf.yml` with sequential steps:

1. **Bump** - Update version numbers (semver)
2. **Changelog** - Update CHANGELOG.md with release notes
3. **Test** - Run full test suite to verify release readiness
4. **Tag** - Create git tag for the release
5. **Publish** - Publish to package registry (gem push, npm publish, etc.)

## Deliverables

- [ ] `release.wf.yml` workflow definition
- [ ] Step instructions for each phase
- [ ] Integration with existing `/ace:draft-release` and `/ace:publish-release` workflows

## Acceptance Criteria

- [ ] Workflow loads via `ace-coworker start --workflow release --version 1.2.0`
- [ ] Each step provides clear instructions to the agent
- [ ] Version parameter drives the bump step
- [ ] Human gate before publish step (configurable)
