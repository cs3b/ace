# Create documentation workflow template

## Objective

Create a multi-step workflow template for ace-coworker that guides documentation updates from gap identification through commit.

## Scope

Create `.ace-defaults/coworker/workflows/documentation.wf.yml` with sequential steps:

1. **Identify** - Find documentation gaps (outdated, missing, incorrect)
2. **Write** - Create or update documentation content
3. **Review** - Self-review for accuracy and completeness
4. **Commit** - Create atomic commit with docs changes

## Deliverables

- [ ] `documentation.wf.yml` workflow definition
- [ ] Step instructions for each phase
- [ ] Integration with existing `/ace:update-docs` workflow

## Acceptance Criteria

- [ ] Workflow loads via `ace-coworker start --workflow documentation --scope ace-review`
- [ ] Each step provides clear instructions to the agent
- [ ] Scope parameter limits documentation focus (gem, directory, or topic)
- [ ] Works with ace-docs for status checking
