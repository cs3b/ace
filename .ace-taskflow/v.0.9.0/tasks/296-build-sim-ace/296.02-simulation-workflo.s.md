# Phase 2: implement ace-sim package from proven phase-1 contracts

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-sim run` with scenario, source, step list, providers, and repeat count.
- **Process**: Package executes step-chain simulation using phase-1-proven contracts and writes deterministic artifacts.
- **Output**: Comparison-ready run outputs and optional writeback preview.

### Expected Behavior
- Create standalone `ace-sim` package with namespace `Ace::Sim`.
- Implement generic simulation CLI `ace-sim run` as the primary public interface.
- Preserve next-phase simulation as a built-in scenario on the generic runner.
- Enforce step chaining: step N consumes output from step N-1.
- Maintain support for multi-provider and same-provider repeat runs.
- Keep behavior aligned with phase-1 usage contract; no contract drift.

### Interface Contract

```bash
ace-sim run \
  --scenario next-phase \
  --source <idea-ref|task-ref> \
  --steps draft,plan \
  --provider <providerA:model> \
  --provider <providerB:model> \
  --repeat 2 \
  --dry-run

ace-sim run \
  --scenario next-phase \
  --source <idea-ref|task-ref> \
  --steps draft,plan \
  --provider <provider:model> \
  --repeat 2 \
  --no-writeback
```
