# Prove Simulation Pipeline with Workflows Only (No Code)

## Objective

Validate the multi-step simulation flow using ONLY existing tools (ace-bundle + ace-llm CLI + shell scripting). Zero new Ruby code. This proves the approach works before any gem implementation begins.

## Behavioral Specification

### User Experience
- **Input**: Maintainer runs shell commands against a real idea file using ace-bundle presets and ace-llm
- **Process**: Multi-step pipeline executes: draft step -> plan step -> synthesis. Each step's output feeds into the next.
- **Output**: Cached artifacts demonstrating step chaining, multi-provider comparison, and synthesis

### Expected Behavior
- Define simulation as YAML config specifying steps, workflow URIs, providers, output format
- All LLM calls use ace-llm with config-driven settings (no hardcoded values)
- All context uses ace-bundle presets (no inline YAML)
- Step outputs chain: step N output becomes input for step N+1
- Multi-provider comparison: same step run on 2+ providers produces separate artifacts
- No new Ruby code, no gem directories, no package scaffolds

### Interface Contract

**Happy-path usage example** (the core deliverable of this subtask):

```bash
RUN_ID=$(ace-b36ts)
mkdir -p .cache/ace-sim/demo/$RUN_ID/{step-01-draft,step-02-plan,synthesis}
SOURCE="path/to/my-idea.s.md"

# Step 1: Draft simulation
ace-bundle sim/step-draft-context > .cache/ace-sim/demo/$RUN_ID/step-01-draft/system.md
ace-llm "$(cat $SOURCE)" \
  --system .cache/ace-sim/demo/$RUN_ID/step-01-draft/system.md \
  -m google:flash \
  > .cache/ace-sim/demo/$RUN_ID/step-01-draft/output.md

# Step 2: Plan simulation (chained from draft output)
ace-bundle sim/step-plan-context > .cache/ace-sim/demo/$RUN_ID/step-02-plan/system.md
ace-llm "$(cat $SOURCE)\n---\nDraft output:\n$(cat .cache/ace-sim/demo/$RUN_ID/step-01-draft/output.md)" \
  --system .cache/ace-sim/demo/$RUN_ID/step-02-plan/system.md \
  -m google:flash \
  > .cache/ace-sim/demo/$RUN_ID/step-02-plan/output.md

# Synthesis
ace-llm "Synthesize findings:\n$(cat .cache/ace-sim/demo/$RUN_ID/step-01-draft/output.md)\n$(cat .cache/ace-sim/demo/$RUN_ID/step-02-plan/output.md)" \
  --system handbook/prompts/sim/synthesis.system.md \
  -m google:flash \
  > .cache/ace-sim/demo/$RUN_ID/synthesis/report.md
```

**Simulation config YAML format**:

```yaml
# .ace-defaults/sim/simulations/review-next-phase.yml
name: review-next-phase
description: Simulate idea->draft->plan lifecycle
steps:
  - name: draft
    workflow: wfi://task/draft
    context_preset: sim/step-draft-context
    providers: [google:flash]
    output_format: yaml
  - name: plan
    workflow: wfi://task/plan
    context_preset: sim/step-plan-context
    providers: [google:flash]
    input: previous
    output_format: yaml
synthesis:
  prompt: handbook/prompts/sim/synthesis.system.md
  provider: google:flash
```

### Verification Evidence

- **Command(s)**: The demo shell script (`docs/examples/sim-demo.sh`) run against a real idea file
