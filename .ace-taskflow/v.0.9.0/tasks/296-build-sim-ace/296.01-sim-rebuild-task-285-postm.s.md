# Phase 1: prove simulation workflows/prompts/examples without package code

## Behavioral Specification

### User Experience
- **Input**: Maintainer runs simulation proof commands against idea/task source references.
- **Process**: Workflow docs, prompt contracts, and examples are authored in task-local assets and executed in read-only mode.
- **Output**: Runnable artifacts demonstrate stage chaining, comparison behavior, and evidence classification without adding package code.

### Expected Behavior
- Define simulation as linear steps where each step consumes previous-step output.
- Provide scenario contract for next-phase simulation (`draft`, `plan`) as v1 proof target.
- Support multi-provider comparison (at least 2 providers) and repeat runs on one provider.
- All proof artifacts live under task-local assets and cache outputs.
- Proof execution must be independent from `ace-taskflow review-next-phase` (assume command does not exist).
- No new package directories (`ace-sim/`) and no Ruby implementation files are created in this phase.

### Interface Contract

```bash
# Generate run-id once (copy output value into <run-id>)
mise exec -- ace-b36ts encode now

# Render draft prompt from source input using task-local bundle config
mise exec -- ace-bundle -f .ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/proof/proof-bundle.yml --output cache

# Draft stage run with provider A
mise exec -- ace-llm google:gflash --system .ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/proof/draft.system.md --prompt .cache/ace-sim/proof/<run-id>/draft.prompt.md --output .cache/ace-sim/proof/<run-id>/draft-output.yml

# Re-render bundle after draft output exists; plan prompt must include draft-output
mise exec -- ace-bundle -f .ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/proof/proof-bundle.yml --output cache

# Plan stage run with provider B (consumes rendered plan.prompt built from draft-output)
mise exec -- ace-llm codex:mini --system .ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/proof/plan.system.md --prompt .cache/ace-sim/proof/<run-id>/plan.prompt.md --output .cache/ace-sim/proof/<run-id>/plan-output.yml

# Repeat run for variance check (same provider)
mise exec -- ace-llm google:gflash --system .ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/proof/plan.system.md --prompt .cache/ace-sim/proof/<run-id>/plan.prompt.md --output .cache/ace-sim/proof/<run-id>/plan-repeat-output.yml

# Synthesis producer (explicit command)
mise exec -- ace-llm codex:mini --system .ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/proof/synthesis.system.md --prompt .cache/ace-sim/proof/<run-id>/synthesis.prompt.md --output .cache/ace-sim/proof/<run-id>/synthesis.yml

# Comparison producer (explicit command)
mise exec -- ace-llm codex:mini --system .ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/proof/comparison.system.md --prompt .cache/ace-sim/proof/<run-id>/comparison.prompt.md --output .cache/ace-sim/proof/<run-id>/comparison.yml
```

Phase-1 proof config (`ux/proof/proof-config.yml`) must follow:

```yaml
scenario: next-phase
steps:
  - draft
  - plan
input:
  type: taskflow_ref
  value: "<ref>"
providers:
  - google:gflash
  - codex:mini
repeat: 2
writeback: false
cache_root: .cache/ace-sim/proof
```

`proof-config.yml` is a normative contract document in phase 1; it defines the required scenario schema and defaults, and is not directly consumed by a phase-1 runner.

Prompt contracts for phase-1 deliverables:
- `draft.system.md`: must instruct model to produce structured `draft-output.yml` for the `draft` step only.
- `plan.system.md`: must instruct model to produce structured `plan-output.yml` based on prior draft content.
- `synthesis.system.md`: must instruct model to summarize run outputs across providers/repeats into `synthesis.yml`.
- `comparison.system.md`: must instruct model to emit provider/repeat delta analysis into `comparison.yml`.
- `proof-bundle.yml`: must define render rules proving `plan.prompt.md` contains content derived from `.cache/ace-sim/proof/<run-id>/draft-output.yml`.

## Objective

Prove simulation behavior and UX contract in runnable form before any ace-sim package implementation begins.

## Scope of Work

- **In scope**:
  - task-local assets in `.ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/`
  - workflow/prompt and output schema contracts
  - proof runs and evidence capture
- **Out of scope**:
  - `ace-sim` gem scaffold
  - Ruby source code extraction/migration
  - package CLI implementation

## Deliverables

- `ux/phase-1-happy-path.md` usage document with command flow and expected artifacts
- `ux/proof/proof-bundle.yml` task-local bundle config for stage prompt assembly
- `ux/proof/proof-config.yml` task-local scenario+steps+providers config contract
- `ux/proof/draft.system.md` and `ux/proof/plan.system.md` prompt contracts
- `ux/proof/synthesis.system.md` and `ux/proof/comparison.system.md` producer prompt contracts
- Simulation contract notes for stage inputs/outputs and comparison semantics
- Evidence matrix table in `ux/phase-1-happy-path.md` covering provider A/B and repeated same-provider run

## Locked Decisions

### [HIGH] Provider baseline (resolved)
- Mandatory proof providers are `google:gflash` and `codex:mini`.

### [HIGH] Repeat-run threshold (resolved)
- Minimum repeat-run threshold is `2` on `google:gflash`.

### [HIGH] Read/write safety (resolved)
- Phase-1 proof is read-only and must not write back to source ideas/tasks.

### [HIGH] Run-id format (resolved)
- Run-id format uses `ace-b36ts` output and is reused across all artifact paths in one proof run.
