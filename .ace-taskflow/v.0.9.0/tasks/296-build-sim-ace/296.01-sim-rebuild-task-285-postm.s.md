# Phase 1: prove simulation workflows/prompts/examples without package code

## Behavioral Specification

### User Experience
- **Input**: Maintainer runs simulation proof commands against idea/task source references.
- **Process**: Workflow docs, prompt contracts, and examples are authored in task-local assets and executed in read-only mode.
- **Output**: Runnable artifacts demonstrate stage chaining, comparison behavior, and evidence classification without adding package code.

### Expected Behavior
- Define simulation as linear steps where each step consumes previous-step output.
- Provide scenario contract for next-phase simulation (`draft`, `plan`) as v1 proof target.
- Support multi-provider comparison (at least 2 providers) and repeat runs on one provider.
- All proof artifacts live under task-local assets and cache outputs.
- No new package directories (`ace-sim/`) and no Ruby implementation files are created in this phase.

### Interface Contract

```bash
# Canonical proof run (read-only)
mise exec -- ace-taskflow review-next-phase --source <ref> --modes draft,plan --no-writeback --verbose

# Provider variance run (provider A)
mise exec -- ace-taskflow review-next-phase --source <ref> --modes draft,plan --no-writeback --verbose --model <providerA:model>

# Provider variance run (provider B)
mise exec -- ace-taskflow review-next-phase --source <ref> --modes draft,plan --no-writeback --verbose --model <providerB:model>
```

### Verification Evidence

- **Command(s)**:
  - `mise exec -- ace-taskflow review-next-phase --source <ref> --modes draft,plan --no-writeback --verbose`
  - one run for provider A, one run for provider B, and one repeated run on provider A
- **Artifacts**:
  - `.cache/ace-taskflow/simulations/<run-id>/synthesis.yml`
  - `.cache/ace-taskflow/simulations/<run-id>/writeback-preview.md`
  - `.ace-taskflow/v.0.9.0/tasks/296-build-sim-ace/ux/phase-1-happy-path.md`
- **Expected vs Observed**:
  - pass when all required artifacts exist and outputs show step-to-step propagation
- **Failure Classification**:
  - `implementation failure | provider unavailable | environment issue`
