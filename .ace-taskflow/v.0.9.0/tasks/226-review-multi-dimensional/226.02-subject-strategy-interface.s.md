# 226.02: Subject Strategy Interface + Full Strategy

## Goal

Define the strategy pattern interface and implement the `full` (no-split) strategy.

## Deliverables

1. `ace-review/lib/ace/review/molecules/subject_strategy.rb` - Interface + factory
2. `ace-review/lib/ace/review/molecules/strategies/full_strategy.rb` - Pass-through
3. Update `ReviewManager` to use strategy interface
4. Unit tests

## Interface

```ruby
# Factory method
strategy = Ace::Review::Molecules::SubjectStrategy.for(:full, config)

# Check if strategy can handle the subject
strategy.can_handle?(subject_text, model_context_limit)
# → true/false

# Prepare subject for review (returns array of review units)
strategy.prepare(subject_text, context)
# → [{ content: "full subject...", metadata: {} }]
```

## Full Strategy Behavior

The `full` strategy is the simplest - it passes the complete subject through unchanged:

```ruby
class FullStrategy
  def can_handle?(subject, model_context)
    TokenEstimator.estimate(subject) < model_context * 0.85
  end

  def prepare(subject, context)
    [{
      content: subject,
      metadata: { strategy: :full, chunk_index: 0, total_chunks: 1 }
    }]
  end
end
```
