# 226.04: Adaptive Strategy Selection

## Goal

Auto-select strategy based on model capabilities + subject size.

## Deliverables

1. Update `SubjectStrategy.for(:adaptive, ...)` to auto-select
2. Configuration schema in preset/config
3. Integration tests

## Logic

```ruby
class AdaptiveStrategy
  def initialize(config)
    @config = config
  end

  def select_strategy(subject, model)
    estimated_tokens = TokenEstimator.estimate(subject)
    model_limit = ContextLimitResolver.resolve(model)

    # Leave 15% headroom for system prompt and context
    available = model_limit * 0.85

    if estimated_tokens < available
      FullStrategy.new(@config)
    else
      ChunkedStrategy.new(@config)
    end
  end

  def prepare(subject, context, model:)
    strategy = select_strategy(subject, model)
    strategy.prepare(subject, context)
  end
end
```

## Configuration

```yaml
# In preset YAML
subject_strategy:
  type: adaptive  # or: full, chunked, progressive
  headroom: 0.15  # Reserve 15% for system prompt (default)
  chunking:
    max_tokens_per_chunk: 100000
    include_change_summary: true
```

## Acceptance Criteria

- [ ] Gemini 2M uses `full` for most PRs (< 1.7M tokens)
- [ ] Claude 200k uses `chunked` for large PRs (> 170k tokens)
- [ ] Strategy configurable at preset level

- [ ] Explicit strategy type overrides adaptive selection
