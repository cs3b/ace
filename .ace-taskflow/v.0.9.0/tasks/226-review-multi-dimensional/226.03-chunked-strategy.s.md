# 226.03: Chunked Strategy

## Goal

Split large diffs into chunks at file boundaries, review in parallel, synthesize.

## Deliverables

1. `ace-review/lib/ace/review/atoms/diff_boundary_finder.rb` - Parse diff into file blocks
2. `ace-review/lib/ace/review/molecules/strategies/chunked_strategy.rb` - Chunk strategy
3. Unit tests

## Interface

```ruby
# Parse diff into file blocks
blocks = Ace::Review::Atoms::DiffBoundaryFinder.parse(diff_text)
# → [
#     { path: "lib/foo.rb", content: "diff --git...", lines: 45 },
#     { path: "test/foo_test.rb", content: "diff --git...", lines: 30 }
#   ]

# Chunked strategy
strategy = Ace::Review::Molecules::SubjectStrategy.for(:chunked, {
  max_tokens_per_chunk: 100_000,
  include_change_summary: true
})

strategy.prepare(subject, context)
# → [
#     { content: "Summary: Changes in lib/foo.rb, test/foo_test.rb\n\ndiff --git...",
#       metadata: { strategy: :chunked, chunk_index: 0, total_chunks: 2, files: ["lib/foo.rb"] } },
#     { content: "Summary: Changes in lib/foo.rb, test/foo_test.rb\n\ndiff --git...",
#       metadata: { strategy: :chunked, chunk_index: 1, total_chunks: 2, files: ["test/foo_test.rb"] } }
#   ]
```

## Chunking Rules

1. Never split within a file (atomic file boundaries)
2. Each chunk includes summary of ALL files being changed (for context)
3. Chunks are sized to fit within `max_tokens_per_chunk`
4. If single file exceeds limit, include it alone (may overflow, but don't split mid-file)

## Acceptance Criteria

- [ ] DiffBoundaryFinder parses `diff --git` boundaries correctly
- [ ] Chunks never split mid-file
- [ ] Each chunk includes summary of all changes for context
- [ ] ChunkedStrategy returns multiple review units for large diffs
- [ ] Single-file diffs return single chunk
- [ ] All tests pass
