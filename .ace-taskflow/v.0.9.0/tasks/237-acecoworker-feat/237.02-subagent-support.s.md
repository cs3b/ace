## Technical Approach

### Guide Structure

The subagent-delegation.g.md guide should cover:

1. **When to Use Subagents**
   - Long-running tasks (>15 min agent work)
   - Parallel workloads (multiple independent tasks)
   - Context isolation needs (fresh context for focused work)
   - Resource-heavy operations (large codebase exploration)

2. **When NOT to Use Subagents**
   - Quick tasks (<5 min)
   - Tasks requiring session state access
   - Sequential dependencies within same context
   - Simple file operations

3. **Instruction Patterns**
   - Task tool prompt structure
   - What information to pass
   - Expected response format
   - Result capture in reports

4. **Context Isolation Model**
   - What subagents can access (filesystem, git, env)
   - What subagents cannot access (parent session, conversation)
   - Implications for workflow design

5. **Anti-patterns**
   - Over-delegation (too many subagents)
   - Under-specified prompts
   - Missing result capture
   - Session state assumptions

### Subagent Instruction Pattern

Template for steps that spawn subagents:

```yaml
steps:
  - name: implement-via-subagent
    instructions: |
      Use the Task tool to delegate implementation:

      **Task prompt:**
      ```
      Implement task {{taskref}} following the task specification.

      Working directory: current
      Required: summary of changes, files modified, test results
      ```

      **After subagent returns:**
      1. Review the implementation summary
      2. Verify key files were modified as expected
      3. Run `ace-test` to confirm tests pass

      Report with: ace-coworker report <file>
```

### Context Isolation Model

**Isolated (Subagent has its own):**
1. Instruction context - receives only the Task tool prompt
2. Session scope - no access to parent's ace-coworker session
3. Conversation history - fresh context

**Shared (Both can access):**
1. Working directory - same file system
2. Git state - same branch and repository
3. Environment variables - inherited

### Example Preset Structure

```yaml
# with-subagent.yml
name: with-subagent
description: Workflow demonstrating subagent delegation

steps:
  - name: prepare
    instructions: |
      Review the task requirements and prepare for implementation.
      Report with: ace-coworker report <file>

  - name: implement-via-subagent
    instructions: |
      Delegate implementation to a subagent using Task tool:
      [... Task tool pattern ...]
      Report with: ace-coworker report <file>

  - name: verify
    instructions: |
      Verify the subagent's work:
      - Run tests: ace-test
      - Check lint: ace-lint
      Report with: ace-coworker report <file>
```
