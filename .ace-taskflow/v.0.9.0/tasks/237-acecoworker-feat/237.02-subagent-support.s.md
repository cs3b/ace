## Technical Approach

### Subagent Instruction Pattern

Instructions tell the agent when to use Task tool:

```yaml
steps:
  - name: implement
    instructions: |
      Use the Task tool to delegate implementation to a subagent:

      Task prompt:
      """
      Implement task {{taskref}} following wfi://work-on-task

      Working directory: current
      Report back: summary of changes, files modified, test results
      """

      After subagent returns, capture results and continue.
      Report with: ace-coworker report <file>
```

### Context Isolation Model

**Isolated (Subagent has its own):**
1. Instruction context - receives only the Task tool prompt
2. Session scope - no access to parent's ace-coworker session
3. Conversation history - fresh context

**Shared (Both can access):**
1. Working directory - same file system
2. Git state - same branch and repository
3. Environment variables - inherited

### Example Preset

```yaml
# with-subagent.yml
name: with-subagent
description: Workflow demonstrating subagent delegation

steps:
  - name: prepare
    instructions: |
      Review the task requirements and prepare for implementation.
      Report with: ace-coworker report <file>

  - name: implement-via-subagent
    instructions: |
      Delegate implementation to a subagent using Task tool:

      Task prompt:
      """
      Execute task {{taskref}} implementation.
      Working directory: current
      Report: summary, files changed, test results
      """

      Capture subagent output in your report.
      Report with: ace-coworker report <file>

  - name: verify
    instructions: |
      Verify the subagent's work:
      - Run tests: ace-test
      - Check lint: ace-lint
      Report with: ace-coworker report <file>
```
