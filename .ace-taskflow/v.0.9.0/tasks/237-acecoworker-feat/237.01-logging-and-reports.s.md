# Logging and Reports

## Overview

Track delegations and agent responses for debugging and audit purposes. Simplified from 234.02.

## Objective

Add lightweight logging and timestamping to ace-coworker sessions:
- Event log file per session (append-only)
- Log step transitions: started, completed, failed, retried
- Add timestamps to report frontmatter
- No JSONL complexity, no separate delegation files

## Scope of Work

### Simplified from 234.02

**Included:**
- Single `log.txt` file per session (human-readable, append-only)
- Log key events: session_started, step_started, step_completed, step_failed, session_completed
- Add `completed_at` timestamp to step frontmatter when reporting
- Timestamps use ISO 8601 format

**Excluded (from original 234.02):**
- JSONL event stream
- Separate delegation/report files in reports/ directory
- Complex event serialization with ace-timestamp encoding
- Attempt tracking in filenames

### Deliverables

#### Create

- `ace-coworker/lib/ace/coworker/molecules/event_logger.rb` - Append-only log writer
- `ace-coworker/test/molecules/event_logger_test.rb` - Logger tests

#### Modify

- `ace-coworker/lib/ace/coworker/organisms/workflow_executor.rb` - Log events at transitions
- `ace-coworker/lib/ace/coworker/molecules/step_writer.rb` - Add completed_at to frontmatter
- `ace-coworker/lib/ace/coworker.rb` - Add require for event_logger

## Technical Approach

### Log Format

Simple human-readable format:
```
2026-01-28T12:00:00Z session_started session=abc123 preset=work-on-task
2026-01-28T12:00:01Z step_started step=010-implement
2026-01-28T12:15:00Z step_completed step=010-implement
2026-01-28T12:15:01Z step_started step=020-test
2026-01-28T12:20:00Z step_failed step=020-test error="3 tests failed"
2026-01-28T12:25:00Z step_started step=021-fix
2026-01-28T12:30:00Z step_completed step=021-fix
2026-01-28T12:35:00Z session_completed
```

### Step Frontmatter Extension

```yaml
---
status: done
completed_at: '2026-01-28T12:15:00Z'
---
```
