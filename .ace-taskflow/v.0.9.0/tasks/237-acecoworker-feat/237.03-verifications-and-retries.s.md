# Verifications and Retries

## Overview

Run verification commands after steps to catch failures. Simplified from 234.04.

## Objective

Add lightweight verification support:
- `verification:` field in step frontmatter
- `ace-coworker report --verify` runs verification command
- If verification fails, auto-mark step as failed with output
- Manual retry via existing `ace-coworker retry`

## Scope of Work

### Simplified from 234.04

**Included:**
- Single verification command per step (in frontmatter)
- `--verify` flag on report command
- Verification output stored in step error field
- Uses existing retry command for recovery

**Excluded (from original 234.04):**
- Multiple verification commands per step
- Error normalization heuristics
- Repeated failure detection
- Auto-retry loops
- verification_retries/on_repeated_failure config

## Technical Approach

### Architecture Pattern

VerificationRunner follows ATOM Molecule pattern:
- Single responsibility: execute shell command and capture output
- Controlled side effect: subprocess execution
- Returns structured result (no exceptions for non-zero exit)

### Step Frontmatter Extension

```yaml
---
status: in_progress
verification: ace-test
---
```

### Verification Flow

```
ace-coworker report report.md --verify
  │
  ├── 1. Store report content (existing advance logic)
  ├── 2. Read current step's verification field
  ├── 3. If verification present:
  │     ├── Execute command via VerificationRunner
  │     ├── Exit 0 → mark step done, advance (normal flow)
  │     └── Exit !0 → mark step failed, store output, halt
  ├── 4. If no verification: normal flow (mark done, advance)
  └── 5. Display result with verification status
```

### CLI Interface

```bash
# Report with verification
ace-coworker report changes.md --verify
# Output (success):
# Step 020 (implement) completed
# Verification: ace-test passed
# Advancing to step 030

# Output (failure):
# Step 020 (implement) failed verification
# Verification: ace-test failed (exit 1)
# Error output stored in step file
# Use 'ace-coworker retry 020' to retry

# Report without verification (existing behavior)
ace-coworker report changes.md
# Completes step without running verification
```

### VerificationRunner Molecule API

```ruby
module Ace::Coworker::Molecules
  class VerificationRunner
    DEFAULT_TIMEOUT = 300  # 5 minutes

    # Execute verification command
    # @param command [String] Shell command to run
    # @param timeout [Integer] Timeout in seconds (default: 300)
    # @return [Hash] { success:, exit_code:, output:, duration: }
    def self.run(command, timeout: DEFAULT_TIMEOUT)
      start_time = Process.clock_gettime(Process::CLOCK_MONOTONIC)

      # Use Open3.capture2e for combined stdout/stderr
      output, status = Open3.capture2e(command, timeout: timeout)

      duration = Process.clock_gettime(Process::CLOCK_MONOTONIC) - start_time

      {
        success: status.success?,
        exit_code: status.exitstatus,
        output: output,
        duration: duration
      }
    rescue Timeout::Error
      {
        success: false,
        exit_code: -1,
        output: "Command timed out after #{timeout} seconds",
        duration: timeout
      }
    end
  end
end
```

### Report Command Changes

```ruby
class Report < Dry::CLI::Command
  # ... existing ...
  option :verify, type: :boolean, default: false,
         desc: "Run verification command after report"

  def call(file:, verify: false, **options)
    # 1. Get current step info before advancing
    executor = Organisms::WorkflowExecutor.new
    status = executor.status
    current = status[:current]

    if verify && current
      verification = current.verification
      if verification
        # Run verification
        result = Molecules::VerificationRunner.run(verification)
        unless result[:success]
          # Mark failed instead of completing
          executor.fail("Verification failed (exit #{result[:exit_code]}): #{result[:output]}")
          # Output failure message
          puts "Step #{current.number} (#{current.name}) failed verification"
          puts "Verification: #{verification} failed (exit #{result[:exit_code]})"
          puts "Use 'ace-coworker retry #{current.number}' to retry"
          return
        end
      end
    end

    # 2. Normal advance flow
    result = executor.advance(file)
    # ... existing output logic ...
  end
end
```
