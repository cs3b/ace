# 233.07: Multi-Dimensional Synthesis

## Goal

Parallel reviewer execution with weighted synthesis and critical finding highlighting.

## Deliverables

1. Update `MultiModelExecutor` for multi-reviewer execution
2. Update `ReportSynthesizer` for weighted merge + critical findings
3. Integration tests

## Execution Flow

```
Subject → Routed to multiple reviewers (parallel)
    ├── code-quality reviewer → findings
    ├── security reviewer → findings (critical)
    └── test-coverage reviewer → findings
            ↓
    Weighted synthesis → Final report
```

## Multi-Reviewer Execution

```ruby
class MultiModelExecutor
  # Thread safety: Each thread gets its own LLM client instance
  # to avoid race conditions with shared connection state
  def execute_reviewers(reviewers, subject, context)
    results = reviewers.map do |reviewer|
      Thread.new do
        # Create per-thread client instance for thread safety
        client = create_client_for_model(reviewer.model)

        filtered_subject = reviewer.filter_subject(subject)
        enhanced_prompt = reviewer.enhance_system_prompt(context[:system_prompt])

        begin
          execute_single(
            client: client,
            system_prompt: enhanced_prompt,
            user_prompt: context[:user_prompt],
            subject: filtered_subject
          )
        rescue StandardError => e
          # Return error result instead of raising
          { error: e.message, reviewer: reviewer.name }
        end
      end
    end.map(&:value)

    results.zip(reviewers).map do |result, reviewer|
      {
        reviewer: reviewer,
        response: result,
        weight: reviewer.weight,
        critical: reviewer.critical,
        failed: result.is_a?(Hash) && result[:error]
      }
    end
  end
end
```

## Weighted Synthesis

```ruby
class ReportSynthesizer
  MAX_SYNTHESIS_INPUT_TOKENS = 150_000  # Conservative limit for synthesis model

  def synthesize_multi_dimensional(results)
    # Filter out failed reviewers
    successful = results.reject { |r| r[:failed] }

    # Group by focus area
    by_focus = successful.group_by { |r| r[:reviewer].focus }

    # Condense if combined output exceeds synthesis limit
    condensed_results = condense_if_needed(successful)

    # Build synthesis prompt
    prompt = build_synthesis_prompt(condensed_results)

    # Critical findings section (extracted before condensation)
    critical_findings = successful
      .select { |r| r[:critical] }
      .flat_map { |r| extract_findings(r[:response]) }

    # Synthesize
    synthesis = execute_synthesis(prompt)

    # Inject critical findings at top
    if critical_findings.any?
      synthesis = "## Critical Findings\n\n#{critical_findings.join("\n")}\n\n#{synthesis}"
    end

    # Add failed reviewer notice if any
    failed = results.select { |r| r[:failed] }
    if failed.any?
      notice = failed.map { |r| "- #{r[:reviewer].name}: #{r[:response][:error]}" }.join("\n")
      synthesis = "## Reviewer Failures\n\n#{notice}\n\n#{synthesis}"
    end

    synthesis
  end

  private

  # Map-reduce condensation when inputs exceed synthesis limit
  def condense_if_needed(results)
    total_tokens = results.sum { |r| TokenEstimator.estimate(r[:response].to_s) }

    return results if total_tokens <= MAX_SYNTHESIS_INPUT_TOKENS

    # Condense each result to its key findings only
    results.map do |r|
      condensed_response = extract_key_findings(r[:response], max_tokens: MAX_SYNTHESIS_INPUT_TOKENS / results.size)
      r.merge(response: condensed_response, condensed: true)
    end
  end
end
```
