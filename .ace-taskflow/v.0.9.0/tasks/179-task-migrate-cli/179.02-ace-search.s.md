## Implementation Notes

### Default Command Routing Pattern

dry-cli doesn't support Thor's `method_missing` for unknown commands. Use this pattern:

```ruby
# Option 1: Wrapper in exe/ace-search that pre-processes ARGV
#!/usr/bin/env ruby
require_relative "../lib/ace/search"

# If first arg isn't a known command, prepend "search"
known_commands = %w[search version --help -h --version]
unless known_commands.include?(ARGV.first) || ARGV.empty?
  ARGV.unshift("search")
end

result = Dry::CLI.new(Ace::Search::CLI).call
exit(result.is_a?(Integer) ? result : 0)
```

```ruby
# Option 2: Custom Registry with before_call hook (advanced)
module Ace::Search
  class CLI < Dry::CLI::Registry
    # Register default command at root
    register nil, Commands::Search  # nil = root command

    # Alternative: Handle in before hook
    def self.call(arguments: ARGV)
      # Transform unknown first args to "search <arg>"
      # ...
    end
  end
end
```

The exe wrapper approach (Option 1) is simpler and recommended for the spike.
