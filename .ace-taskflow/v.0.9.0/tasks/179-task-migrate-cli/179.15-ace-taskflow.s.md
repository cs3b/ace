# Migrate ace-taskflow to dry-cli (Complex)

## Objective

Migrate ace-taskflow CLI from Thor to dry-cli. This is the most complex migration due to:
- Many subcommands (task, tasks, idea, ideas, release, releases, retro, retros, status, etc.)
- Default command routing (unknown → task)
- Class options shared across commands
- Cache clearing on start

## CLI Analysis

**Complexity**: Very Complex
**Commands**: task, tasks, idea, ideas, release, releases, retro, retros, status, doctor, migrate, config
**Aliases**: context → status
**Default routing**: Unknown commands → task (e.g., `ace-taskflow 150` → `ace-taskflow task 150`)
**Class options**: json, markdown, status, stats, tree, format, limit, etc.

## Scope of Work

#### Modify

- `ace-taskflow/lib/ace/taskflow/cli.rb` - Full rewrite
- `ace-taskflow/ace-taskflow.gemspec`
- `ace-taskflow/exe/ace-taskflow`
