# Migrate ace-git-worktree to dry-cli (Complex)

## Objective

Migrate ace-git-worktree CLI from Thor to dry-cli. This is a complex migration due to:
- Multiple subcommands with aliases
- Nested command structure
- Heavy use of `stop_on_unknown_option!` workarounds

## CLI Analysis

**Complexity**: Complex
**Commands**: `create`, `list`, `switch`, `remove`, `prune`, `config`
**Aliases**: `ls` → list, `rm` → remove, `cd` → switch
**Options per command**: task, pr, dry-run, force, show-tasks, etc.
**Workarounds to remove**: `stop_on_unknown_option!`, help flag checks

## Implementation Notes

### Custom Help Text Preservation

Current Thor implementation has custom help:
```ruby
def self.help(shell, subcommand = false)
  super
  shell.say "Task-Aware Worktrees:"
  # ...
end
```

dry-cli approach options:
1. **Banner method**: Override `banner` in root command for custom header
2. **After registration**: Add custom output in exe entry point
3. **Example method**: Use `example` DSL for usage examples

Recommended: Use `example` DSL in each command for contextual help:
```ruby
class Create < Dry::CLI::Command
  desc "Create a new worktree"
  example [
    "--task 081          # Create worktree for task",
    "--pr 123            # Create worktree for PR",
    "feature/new-auth    # Create with branch name"
  ]
end
```

Accept that root-level custom help may be simplified.
