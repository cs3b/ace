# Streamline tools.md after merge

## Behavioral Specification

### User Experience
- **Input**: User opens combined tools.md (after task 218.03 merge)
- **Process**: Finds well-organized, scannable command reference
- **Output**: Quick answers without scrolling through verbose examples

### Expected Behavior

After the merge (task 218.03), tools.md needs streamlining:
1. Group commands by category (not type)
2. Reduce example verbosity - show pattern, not variations
3. Remove redundant explanations
4. Target ~150 lines combined (from post-merge ~250)

### Interface Contract

**Streamlined structure:**
```markdown
# ACE Commands Reference

## Quick Reference
[Top 6-8 commands, both types]

## Task Management
- Claude: `/ace:work-on-task`, `/ace:draft-task`
- CLI: `ace-taskflow task`, `ace-taskflow tasks`

## Git Operations
- Claude: `/ace:commit`, `/ace:create-pr`
- CLI: `ace-git-commit`, `ace-git status`

## Code Review
[...]

## Documentation & Testing
[...]
```

### Success Criteria

- [ ] **Concise**: ~150 lines (reduced from post-merge ~250)
- [ ] **Scannable**: Commands grouped by purpose, not type
- [ ] **Examples reduced**: One pattern per command, not variations
- [ ] **Quick Reference**: Top commands prominently displayed

## Objective

Make tools.md a quick-reference document rather than exhaustive tutorial. Detailed usage lives in each gem's usage.md.
