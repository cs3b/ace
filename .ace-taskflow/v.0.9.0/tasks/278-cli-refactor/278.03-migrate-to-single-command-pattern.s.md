# ace-search: migrate to single-command pattern

## Scope

Migrate ace-search from registry pattern to single-command pattern using `Dry::CLI.new(command).call`.

**Binary:** `ace-search`
**CLI module:** `ace-search/lib/ace/search/cli.rb`
**Pattern:** single-command
**Current commands:** search (only real command)
**Current routing:** Custom `start()` with path-aware `known_command?` — skips args starting with `/` or `.`

## Code Changes

- [ ] Remove registry (`extend Dry::CLI::Registry`)
- [ ] Remove custom `start()` method and `known_command?`
- [ ] Remove `KNOWN_COMMANDS`, `DEFAULT_COMMAND` constants
- [ ] Switch to `Dry::CLI.new(Commands::Search).call` in exe/ace-search
- [ ] Handle `--version` as a command option (not registered subcommand)
- [ ] Path-like first args (e.g., `ace-search ./version`) no longer need special handling — single-command pattern passes all args to the command directly

## Update References

- [ ] Update any workflow files (.wf.md), guides (.g.md), docs/usage.md, README.md referencing ace-search CLI patterns

## Acceptance Criteria

- [ ] `ace-search --help` shows dry-cli built-in help, exits 0
- [ ] `ace-search "pattern" /path` works (no subcommand needed)
- [ ] `ace-search --version` prints version
- [ ] `ace-test` passes in ace-search
