# Base36 Compact ID Format Implementation

## Overview

Orchestrator for replacing 14-character timestamps with 6-character Base36 compact IDs across the ACE mono-repo. This involves creating a new `ace-support-timestamp` package with configurable encoding, then migrating all affected packages (ace-taskflow, ace-prompt, ace-test-runner) to use the new format.

## Subtasks

- **01**: Create ace-support-timestamp Package with Base36 Compact IDs
- **02**: Migrate ace-taskflow to Base36 Compact IDs
- **03**: Migrate ace-prompt to Base36 Compact IDs
- **04**: Migrate ace-test-runner to Base36 Compact IDs

## Dependencies

- Subtask 02 (ace-taskflow migration) depends on Subtask 01 (package creation)
- Subtask 03 (ace-prompt migration) depends on Subtask 01 (package creation)
- Subtask 04 (ace-test-runner migration) depends on Subtask 01 (package creation)

## Success Criteria

- [ ] All packages use Base36 compact IDs by default
- [ ] Existing timestamp-based directories still work (dual-format detection)
- [ ] CLI tool available for encode/decode/migrate operations
- [ ] All tests pass with no regressions
- [ ] Documentation updated
