# Migrate ace-git-secrets to Base36 Compact IDs and Group Files

## Objective

Update ace-git-secrets to use the new ace-support-timestamp package for generating Base36 compact IDs (6 characters) for scan report filenames. Additionally, restructure cache directory to use the standard sessions/ subdirectory grouping pattern. Ensure dual-format detection for existing timestamp-based files.

## Behavioral Specification

### User Experience

**Before**: Scattered files in cache root
```
.cache/ace-git-secrets/20251222-144833-providers.md
.cache/ace-git-secrets/20251222-144833-report.json
```

**After**: Grouped in sessions/ subdirectory with Base36 IDs
```
.cache/ace-git-secrets/sessions/000000-providers.md
.cache/ace-git-secrets/sessions/000000-report.json
```

**Existing Behavior**: Existing timestamp files continue to work

### Expected Behavior

**New Scans**:
- Default to Base36 compact ID format
- Use ace-support-timestamp CompactIdEncoder
- Generate 6-character lowercase IDs
- Save to sessions/ subdirectory (NEW - folder grouping improvement)

**Existing Scans**:
- Dual-format detection recognizes both formats
- Both old (root) and new (sessions/) paths work
- No migration required
- Read/load operations work transparently

### Interface Contract

```ruby
# ScanReport uses new encoder
filename = "#{Ace::Timestamp::Atoms::CompactIdEncoder.encode(Time.now)}-report.json"
# => "000000-report.json"

# Path construction
cache_path = File.join(cache_root, "sessions", filename)
# => ".cache/ace-git-secrets/sessions/000000-report.json"
```

### Success Criteria

- [ ] New scan reports use 6-character Base36 IDs
- [ ] New reports saved to sessions/ subdirectory (grouped)
- [ ] Existing timestamp reports still load correctly
- [ ] Config defaults updated
- [ ] Tests pass for both formats
