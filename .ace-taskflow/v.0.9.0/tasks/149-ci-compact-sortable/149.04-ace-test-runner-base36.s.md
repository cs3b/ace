# Migrate ace-test-runner to Base36 Compact IDs

## Objective

Update ace-test-runner to use the new ace-support-timestamp package for generating Base36 compact IDs (6 characters) for test report filenames. Ensure dual-format detection for existing timestamp-based reports.

## Behavioral Specification

### User Experience

**Before**: Report files named `report-20251117-231038.json`
**After**: Report files named `report-000000.json` (with year_zero=2000)

**Existing Behavior**: Existing timestamp reports continue to work

### Expected Behavior

**New Reports**:
- Default to Base36 compact ID format
- Use ace-support-timestamp CompactIdEncoder
- Generate 6-character lowercase IDs

**Existing Reports**:
- Dual-format detection recognizes both formats
- No migration required
- Read/load operations work transparently

### Interface Contract

```ruby
# TimestampGenerator uses new encoder
generator = Ace::TestRunner::Atoms::TimestampGenerator.new
generator.filename_timestamp
# => "000000"  # Base36 format
```

### Success Criteria

- [ ] New reports use 6-character Base36 IDs
- [ ] Existing timestamp reports still load correctly
- [ ] Config defaults updated
- [ ] Tests pass for both formats

## Scope of Work

### Deliverables

#### Modify

- `lib/ace/test_runner/atoms/timestamp_generator.rb`
  - Add ace-support-timestamp dependency
  - Use CompactIdEncoder.encode()

- `.ace-defaults/test-runner/config.yml`
  - Add format configuration

- `lib/ace/test_runner.gemspec`
  - Add dependency: `ace-support-timestamp ~> 0.1`

## Out of Scope

- ❌ Migrating existing report files
