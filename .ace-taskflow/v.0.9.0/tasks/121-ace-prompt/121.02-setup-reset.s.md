# 121.02 - Setup & Reset Commands

## Scope

Template initialization and reset workflow using `tmpl://` protocol.

## Interface

```bash
ace-prompt setup               # Initialize with default template
ace-prompt setup --template tmpl://custom/template
ace-prompt reset               # Archive current, restore template
ace-prompt reset --force       # Overwrite without archiving
```

## Behavior

### Setup
1. Create directory structure if not exists
2. Resolve template via `tmpl://ace-prompt/base-prompt` protocol
3. Copy template content to `the-prompt.md`
4. Skip if file already exists (unless --force)

### Reset
1. Archive current `the-prompt.md` (unless --force)
2. Restore template content to `the-prompt.md`

## Template Format

```markdown
---
context:
  presets: []
  files: []
  commands: []
---

# Prompt

[Your prompt here]

## Context Requirements

[Describe what context is needed]

## Expected Output

[Describe expected results]
```

### Edge Cases for Testing

#### TemplateResolver
- Template protocol not found (`tmpl://invalid/template`)
- Missing ace-nav dependency (graceful fallback)
- Malformed protocol URL
- Template file doesn't exist at resolved path
- Empty protocol response

#### TemplateManager
- Target directory doesn't exist (should create)
- Target file already exists (setup should warn/skip)
- Template with frontmatter preservation
- Template with placeholder variables
- Template file is empty

#### Reset Command
- No existing prompt to archive (skip archive)
- `--force` flag behavior (no archive)
- Template unavailable during reset
- Prompt file is symlink (handle correctly)
- Archive directory doesn't exist (create it)

#### Setup Command
- Directory already exists with content
- `--force` overwrites existing
- Permission denied on directory creation
- Template resolution fails (helpful error)

echo "Modified content" >> .cache/ace-prompt/prompts/the-prompt.md

# Expected: Archive created, template restored
