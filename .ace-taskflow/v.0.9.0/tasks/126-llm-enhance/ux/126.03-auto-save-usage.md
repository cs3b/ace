# Auto-Save Detection - Usage Guide

## Overview

Auto-save detection automatically saves ace-review reports to the appropriate task or release directory based on your git branch context. This feature is **opt-in** (disabled by default) and can be configured via `.ace/review/config.yml`.

**Key Benefits:**
- Automatically organizes review reports alongside task files
- Reduces manual file management overhead
- Maintains consistent directory structure across projects
- Supports both task-specific and release-level reviews

## Command Types

### Bash CLI Commands

All examples use standard bash shell syntax with ace-review as a CLI tool.

**Syntax:**
```bash
ace-review [options]
```

## Configuration

### Enable Auto-Save

Edit `.ace/review/config.yml`:

```yaml
defaults:
  auto_save: true  # Enable automatic saving
  auto_save_branch_patterns:
    - '^(\d+(?:\.\d+)?)-'     # Standard: 121-feature, 121.01-subtask
    - '^feature/(\d+)-'         # Feature branches: feature/123-name
  auto_save_release_fallback: true  # Save to release dir when no task detected
```

**Default Configuration (Disabled):**
```yaml
defaults:
  auto_save: false  # Disabled by default - must opt-in
```

## Branch Pattern Matching

The auto-save feature detects task IDs from your git branch name using configurable regex patterns.

**Default Pattern:** `^(\d+(?:\.\d+)?)-`

**Examples:**
- `121-feature` → Detects task **121**
- `121.01-subtask` → Detects task **121.01**
- `042-answer-everything` → Detects task **42**
- `main` → No task detected (uses release directory)
- `feature-123` → No match (number not at start)

**Custom Patterns:**

Configure additional patterns for your team's git workflow:

```yaml
defaults:
  auto_save_branch_patterns:
    - '^(\d+(?:\.\d+)?)-'       # Standard: 121-feature
    - '^feature/(\d+)-'           # Feature: feature/123-name
    - '^bugfix/(\d+)-'            # Bugfix: bugfix/456-description
```

## Usage Scenarios

### Scenario 1: Auto-Save to Task Directory (Branch Detection)

**Goal:** Review code while working on task 121, automatically save report to task directory

**Branch:** `121-feature-name`

**Command:**
```bash
ace-review --preset pr --auto-execute
```

**Output:**
```
Review completed successfully
Review saved to .cache/ace-review/sessions/review-20251202-143000/review.md
Review saved to .ace-taskflow/v.0.9.0/tasks/121-feature-name/reviews/20251202-143000-gemini-pr-review.md
```

**Directory Structure:**
```
.ace-taskflow/v.0.9.0/
└── tasks/
    └── 121-feature-name/
        └── reviews/
            └── 20251202-143000-gemini-pr-review.md
```

**Notes:**
- Requires `auto_save: true` in config
- Branch pattern must match (default: `^(\d+(?:\.\d+)?)-`)
- Task directory must exist (validated via ace-taskflow)

---

### Scenario 2: Auto-Save to Release Directory (No Task Match)

**Goal:** Review code on main branch, save to release directory

**Branch:** `main`

**Command:**
```bash

```

**Output:**
```
Review completed successfully
Review saved to .cache/ace-review/sessions/review-20251202-144500/review.md

```

**Directory Structure:**
```
.ace-taskflow/v.0.9.0/
└── reviews/
    └── 20251202-144500-gemini-security-review.md
```

**Notes:**
- No task detected from branch name
- Falls back to release directory
- Requires `auto_save_release_fallback: true` in config

### Scenario 3: Multi-Model Review (Synthesis Report Only)

**Goal:** Review with multiple models, save only the synthesis report

**Branch:** `121.01-subtask`

**Command:**

**Output:**

**Directory Structure:**

```
└── tasks/
    └── 121.01-subtask/

            └── 20251202-150000-synthesis-pr-review.md
```

**Notes:**
- Individual model reviews saved only to `.cache/ace-review/sessions/`
- Only synthesis report saved to task directory
- Filename uses "synthesis" instead of model name

### Scenario 4: Explicit Task Override

**Goal:** Save review to specific task, overriding branch detection

**Branch:** `121-feature` (would normally detect task 121)

**Command:**
```bash
ace-review --preset pr --task 114 --auto-execute
```

**Output:**
```
Review completed successfully
Review saved to .ace-taskflow/v.0.9.0/tasks/114-other-task/reviews/20251202-151000-gemini-pr-review.md
```

**Notes:**
- `--task` flag has **highest priority** (overrides auto-detection)
- Use when working on one task but reviewing another
- Task directory must exist

---

### Scenario 5: Disable Auto-Save with Flag

**Goal:** Temporarily disable auto-save even though config enables it

**Branch:** `121-feature`

**Command:**

### Scenario 6: Auto-Save Disabled by Default

**Goal:** Run review without auto-save (default behavior)

**Branch:** `121-feature`

**Config:** `auto_save: false` (default)

**Command:**

**Output:**

**Notes:**
- Default behavior when `auto_save: false` or not configured

## Filename Format

All auto-saved reviews use consistent filename format:

**Pattern:** `{timestamp}-{model-or-synthesis}-{preset}-review.md`

**Components:**
- `timestamp`: `YYYYMMDD-HHMMSS` format
- `model`: Simplified model name (e.g., `gemini-2.5-flash` → `gemini`)
- `synthesis`: Literal "synthesis" for multi-model reviews
- `preset`: The preset name used (e.g., `pr`, `security`)

**Examples:**
- `20251202-143000-gemini-pr-review.md` (single model)
- `20251202-150000-synthesis-pr-review.md` (multi-model)
- `20251202-144500-gpt-security-review.md` (different preset)

## Priority Order

Auto-save follows strict priority order:

1. **Explicit --task flag** (highest priority)
   - Always saves to specified task
   - Overrides branch detection

2. **Auto-detected from branch** (if config enabled)
   - Extracts task ID from branch name
   - Validates task exists via ace-taskflow

3. **Release directory fallback** (if no task detected)
   - Uses `.ace-taskflow/{release}/reviews/`
   - Requires `auto_save_release_fallback: true`

4. **Session directory only** (if auto-save disabled)
   - Default behavior
   - No automatic organization

## Error Handling

### Invalid Task Directory

**Scenario:** Branch matches pattern but task doesn't exist

**Branch:** `999-nonexistent`

**Behavior:**
```
Warning: Task '999' not found. Falling back to release directory.
Review saved to .ace-taskflow/v.0.9.0/reviews/20251202-154000-gemini-pr-review.md
```

### Detached HEAD State

**Scenario:** Git worktree checkout of specific commit

**Branch:** `HEAD` (detached)

**Behavior:**
```
No task detected from branch. Using release directory.
Review saved to .ace-taskflow/v.0.9.0/reviews/20251202-155000-gemini-pr-review.md
```

### Not in Git Repository

**Scenario:** Running ace-review outside git repo

**Behavior:**
- Auto-detection fails silently
- Falls back to release directory (if configured)
- Otherwise, session directory only

## Tips and Best Practices

### 1. Use Standard Branch Naming

Adopt consistent branch naming for reliable auto-detection:

```bash
# Good (matches default pattern)
121-add-feature
121.01-subtask-name
042-bugfix

# Won't match (number not at start)
feature-121
bugfix-121-description
```

### 2. Configure Custom Patterns for Your Workflow

If your team uses different conventions:

```yaml
defaults:
  auto_save_branch_patterns:
    - '^feature/(\d+)-'   # feature/123-name
    - '^(\d+)-'           # 123-name
```

### 3. Review Reports Location

Auto-saved reviews organize naturally:

```
.ace-taskflow/v.0.9.0/
├── tasks/
│   ├── 121-feature/
│   │   └── reviews/           # Task-specific reviews
│   │       ├── 20251202-143000-gemini-pr-review.md
│   │       └── 20251202-150000-synthesis-pr-review.md
│   └── 114-other/
│       └── reviews/
│           └── 20251202-151000-gemini-pr-review.md
└── reviews/                   # Release-level reviews
    └── 20251202-144500-gemini-security-review.md
```

### 4. Multi-Model Workflow

When using multiple models, only synthesis saved automatically:

```bash
# Synthesis report auto-saved, individual reviews in session dir
ace-review --preset pr --model "gemini,gpt-4,claude" --auto-execute

# Find individual model reviews:
ls .cache/ace-review/sessions/review-20251202-*/
```

### 5. Temporary Disable for WIP

Use `--no-auto-save` for work-in-progress reviews:

```bash
# Don't pollute task directory with experimental review
ace-review --preset pr --auto-execute --no-auto-save
```

## Troubleshooting

### Auto-Save Not Working

**Check 1: Configuration**
```bash
cat .ace/review/config.yml | grep auto_save
# Should show: auto_save: true
```

**Check 2: Branch Pattern**
```bash
git rev-parse --abbrev-ref HEAD
# Should match pattern: 121-feature (yes), feature-121 (no)
```

**Check 3: Task Exists**
```bash
ace-taskflow task 121
# Should show task details, not "not found"
```

### Reviews Saving to Wrong Location

**Scenario:** Expected task directory, got release directory

**Cause:** Branch pattern doesn't match or task not found

**Solution:**
1. Verify branch name matches pattern
2. Check task exists: `ace-taskflow task {id}`
3. Use explicit `--task` flag as override

### Multiple Reviews in Session Directory

**Scenario:** Multi-model reviews all in `.cache/`, not task directory

**Expected:** Synthesis report in task directory, individuals in session

**Cause:** This is correct behavior for multi-model reviews

**Note:** Only synthesis report auto-saves to task/release directory

## Migration from Manual Saving

### Before (Manual --task Flag)

```bash
# Required manual --task flag every time
ace-review --preset pr --task 121 --auto-execute
```

### After (Auto-Detection)

```bash
# Just enable config once
echo "defaults:\n  auto_save: true" >> .ace/review/config.yml

# Now works automatically on branch 121-feature
ace-review --preset pr --auto-execute
```

**Benefits:**
- Less typing, fewer errors
- Consistent organization
- Works across all branches automatically

## Internal Implementation

**Note:** The following details are for developers working on ace-review internals.

### Components

1. **Atoms/TaskAutoDetector**
   - Pure function: `extract_from_branch(branch_name, patterns:)`
   - Matches regex patterns against branch name
   - Returns task ID string or nil

2. **Molecules/GitBranchReader**
   - I/O operation: `current_branch`
   - Executes `git rev-parse --abbrev-ref HEAD`
   - Returns branch name, "HEAD" (detached), or nil

3. **Molecules/TaskReportSaver**
   - File operations: `save(task_dir, review_file, review_data)`
   - Creates `reviews/` subdirectory
   - Generates timestamped filename

4. **Organisms/ReviewManager**
   - Orchestrates auto-save workflow
   - Implements priority order
   - Handles fallback logic

### Auto-Save Decision Flow

```
execute_with_llm completes
  ├─→ Check --no-auto-save flag
  │   └─→ TRUE: Skip auto-save
  │
  ├─→ Check config auto_save
  │   └─→ FALSE: Skip auto-save
  │
  ├─→ Check --task flag
  │   └─→ Present: Save to explicit task
  │
  ├─→ Get current branch
  │   └─→ Extract task ID from branch
  │       ├─→ Match found + task exists: Save to detected task
  │       └─→ No match OR task not found: Check release fallback
  │           ├─→ fallback enabled: Save to release directory
  │           └─→ fallback disabled: Session directory only
```
