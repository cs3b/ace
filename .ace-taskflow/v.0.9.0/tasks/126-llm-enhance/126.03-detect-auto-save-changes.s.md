# Auto-Save Detection

## Scope

Automatically detect task context from git branch and save final report to appropriate task or release directory (opt-in via config).

## User Experience

```bash
# On branch "121-feature", auto-detects task 121
ace-review --preset pr --auto-execute
# → Auto-saves to: .ace-taskflow/v.0.9.0/tasks/121-.../reviews/

# On main branch, saves to release directory
ace-review --preset pr --auto-execute
# → Auto-saves to: .ace-taskflow/v.0.9.0/reviews/

# Multi-model saves synthesis report (not individual reports)
ace-review --preset pr --model "gemini,gpt-4" --auto-execute
# → Auto-saves: .ace-taskflow/v.0.9.0/tasks/121-.../reviews/20251201-synthesis-pr-review.md

# Explicit task override (highest priority)
ace-review --preset pr --task 114 --auto-execute
# → Saves to task 114, not branch-detected task

# Disable auto-save
ace-review --preset pr --auto-execute --no-auto-save
# → Only saves to session directory
```

## Config Schema

```yaml
# .ace/review/config.yml
defaults:
  auto_save: false  # Disabled by default (opt-in)
  auto_save_path_pattern: "{task|release}/reviews/{timestamp}-{model}-{preset}-review.md"
```

## Directory Structure

```
.ace-taskflow/v.0.9.0/
├── tasks/
│   └── 121-ace-prompt/
│       └── reviews/                    # Task-specific reviews
│           ├── 20251201-180000-gemini-pr-review.md
│           └── 20251201-180500-synthesis-pr-review.md
└── reviews/                            # Release-level reviews (no task)
    └── 20251201-181000-gpt4-security-review.md
```

## Filename Pattern

`{timestamp}-{model-or-synthesis}-{preset}-review.md`
- timestamp: `YYYYMMDD-HHMMSS`
- model: slugified model name (e.g., `gemini-2.5-flash` → `gemini`)
- synthesis: literal `synthesis` for multi-model
- preset: the preset name used
