# Unified configuration loading and merging defaults across ace-* packages

## Orchestrator Overview

This is an orchestrator task coordinating the migration of configuration patterns across ace-* packages to use the standard `Ace::Core.config.get()` cascade pattern consistently.

## Research Findings

### Current State Assessment

**Packages Using Standard Pattern (Correct):**
- ace-lint, ace-git-diff, ace-docs, ace-search, ace-prompt, ace-review

**Packages Needing Migration:**
- ace-taskflow: Inline YAML defaults in `configuration.rb:198-250`
- ace-git-worktree: Dual systems (`Configuration` module + `ConfigLoader`)
- ace-nav: `DEFAULT_CONFIG` hash, doesn't use ace-core
- ace-test-runner: `DEFAULT_CONFIG` fallback pattern

**Deferred (Separate Task):**
- ace-llm, ace-llm-providers-cli (complex ENV-based systems)

### Anti-Patterns Identified

1. **Hardcoded paths**: `.ace-taskflow/done/` in ace-search, ace-docs
2. **Hardcoded timeouts**: `DEFAULT_TIMEOUT` constants in multiple packages
3. **Inline defaults**: Full config hashes/YAML embedded in Ruby code
4. **Custom YAML loading**: `YAML.load_file` instead of Ace::Core

## Target Pattern

```ruby
# Use Ace::Core cascade with .ace.example/ defaults
def self.config
  @config ||= begin
    defaults = load_from_ace_example('taskflow', 'config')
    user_config = Ace::Core.config.get('ace', 'taskflow') || {}
    Ace::Core::Atoms::DeepMerger.merge(defaults, user_config)
  end
end
```

## Objective

Provide unified, consistent configuration experience across all ace-* packages:
1. All defaults in `.ace.example/` files (not Ruby code)
2. All packages use `Ace::Core.config.get()` pattern
3. Deep merge of user config over gem defaults
4. Hardcoded values become configurable

## Success Criteria

- [ ] All subtasks completed
- [ ] No inline YAML/hash defaults in Ruby code for migrated packages
- [ ] All migrated packages load from `.ace.example/` cascade
