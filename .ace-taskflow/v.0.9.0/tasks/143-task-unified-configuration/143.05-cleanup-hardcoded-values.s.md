# Cleanup Hardcoded Values in Compliant Packages

## Objective

Make hardcoded paths, timeouts, and limits configurable in packages that otherwise follow the standard configuration pattern.

## Current State (Anti-Patterns)

### Hardcoded Paths

**File: `ace-search/lib/ace/search.rb:23`**
```ruby
"exclude" => [
  ".ace-taskflow/done/**/*",   # Hardcoded path
  "dev-taskflow/done/**/*",
]
```

**File: `ace-docs/lib/ace/docs/organisms/document_registry.rb:304`**
```ruby
%r{/\.ace-taskflow/done/}  # Hardcoded regex pattern
```

### Hardcoded Timeouts

**File: `ace-context/lib/ace/context/molecules/gh_pr_executor.rb:14`**
```ruby
DEFAULT_TIMEOUT = 30  # Hardcoded, not configurable
```

**File: `ace-docs/lib/ace/docs.rb:35`**
```ruby
"llm_timeout" => 600  # Hardcoded in Ruby, should be in .ace.example/
```

## Target State

All these values should be:
1. Defined in `.ace.example/<package>/config.yml`
2. Loaded via `Ace::Core.config.get()` pattern
3. Overridable by user config in `.ace/<package>/config.yml`

## Scope of Work

### ace-search

**File: `ace-search/lib/ace/search.rb`**
- Move hardcoded exclude paths to config
- Make `.ace-taskflow/done/` path reference configurable
- Update `.ace.example/search/config.yml`

### ace-docs

**File: `ace-docs/lib/ace/docs.rb`**
- Move `llm_timeout` default to `.ace.example/docs/config.yml`
- Remove hardcoded value from Ruby

**File: `ace-docs/lib/ace/docs/organisms/document_registry.rb`**
- Make the `.ace-taskflow/done/` regex pattern configurable
- Load from config instead of hardcoding

### ace-context

**File: `ace-context/lib/ace/context/molecules/gh_pr_executor.rb`**
- Move `DEFAULT_TIMEOUT = 30` to config
- Load from `Ace::Core.config.get('ace', 'context', 'gh_timeout')` or similar
