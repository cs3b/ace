# Migrate ace-taskflow Configuration

## Objective

Remove inline YAML defaults from `configuration.rb` and ensure all configuration is loaded from `.ace.example/taskflow/config.yml` using the standard `Ace::Core.config.get()` cascade pattern.

## Current State (Anti-Pattern)

**File: `ace-taskflow/lib/ace/taskflow/configuration.rb:198-250`**
```ruby
def default_config_yaml
  <<~YAML
    taskflow:
      root: ".ace-taskflow"
      task_dir: "t"
      directories:
        completed: "_archive"
        backlog: "_backlog"
      # ... many more defaults embedded in Ruby code
  YAML
end
```

**Problem**: Full configuration is embedded as a YAML string in Ruby code instead of being loaded from `.ace.example/`.

## Target State (Standard Pattern)

```ruby
def self.config
  @config ||= begin
    # Load from .ace.example/taskflow/config.yml (gem defaults)
    # Merge with user config from .ace/taskflow/config.yml
    Ace::Core.config.get('ace', 'taskflow') || load_gem_defaults
  end
end
```

## Scope of Work

### Files to Modify

1. **`ace-taskflow/lib/ace/taskflow/configuration.rb`**
   - Remove `default_config_yaml` method (lines 198-250)
   - Update config loading to use `Ace::Core.config.get()` pattern
   - Keep backward compatibility

2. **`ace-taskflow/lib/ace/taskflow/molecules/config_loader.rb`**
   - Update to use ace-core cascade instead of direct `YAML.load_file`
   - Remove custom YAML loading at lines 32, 137

3. **`ace-taskflow/.ace.example/taskflow/config.yml`**
   - Ensure all defaults currently in Ruby are present here
   - This becomes the single source of truth for defaults
