# Migrate Retro Data to .ace-retros/ (One-Time Codemod)

## Objective

Move all existing retro files from `.ace-taskflow/v.0.9.0/retros/` to the new
standalone `.ace-retros/` root directory, making them discoverable by `ace-retro list`,
`ace-retro show`, and other ace-retro commands.

This is a one-time data migration. The migration logic lives in `codemods/` within
this task folder — not as a permanent gem command.

**Key difference from ace-idea migration (291.04)**: Retro source has more structural
variety — 200+ files with mixed naming conventions, an `archived/` subfolder, and
files that are flat `.md` (not in folders). All must be classified, wrapped in folders,
and given proper frontmatter.

## Behavioral Specification

### User Experience
- **Input**: Existing retros at `.ace-taskflow/v.0.9.0/retros/` (~200 files, mixed naming)
- **Process**: Run `ruby codemods/migrate_retros.rb --dry-run` to preview, then without `--dry-run` to execute
- **Output**: All retros accessible via `ace-retro list`, structure normalized, no data loss

### Expected Behavior

#### Source classification (retro lesson: "classify on arrival")

All files in `.ace-taskflow/v.0.9.0/retros/` must be classified at migration time:

| Pattern | Example | Classification | Action |
|---------|---------|---------------|--------|
| B36TS-named | `8prlzl-task-291-ace-idea-gem.md` | `source: "taskflow:v.0.9.0"` | Wrap in folder, migrate |
| Legacy date-prefixed | `2025-01-config-discovery-architecture.md` | `source: "legacy"` | Wrap in folder, generate b36ts ID, migrate |
| Free-form English | `ace-test-runner-bin-integration-learnings.md` | `source: "legacy"` | Wrap in folder, generate b36ts ID, migrate |
| `reflection-*` prefixed | `reflection-minitest-setup-for-ace-core.md` | `source: "legacy"` | Wrap in folder, generate b36ts ID, migrate |
| `archived/` subfolder | `archived/*.md` | `source: "legacy"`, `status: done` | Migrate to `_archive/`, wrap in folders |

**B36TS discriminator** (retro lesson): Use `\A[0-9][0-9a-z]{5}-` to identify B36TS-named files.
Files not matching this pattern are legacy and need a newly generated b36ts ID.

#### Migration steps

1. **Scan** `.ace-taskflow/v.0.9.0/retros/` for all `.md` files (including in `archived/`)
2. **Classify** each file using B36TS discriminator pattern
3. **For B36TS-named files**: Extract existing ID from filename, create folder `{id}-{slug}/`, rename to `{id}-{slug}.retro.md`
4. **For legacy-named files**: Generate new b36ts ID, derive slug from filename, create folder, rename file
5. **Inject frontmatter**: Add/update frontmatter fields — `id:`, `title:` (from filename or first heading), `type:` (infer: "conversation-analysis" if content has conversation analysis structure, "self-review" if reflection/self-review, otherwise "standard"), `status:` (active unless in `archived/`), `created_at:` (from git log `--diff-filter=A` or filename date), `source:`, `migrated_from:`
6. **Handle `archived/` subfolder**: Migrate contents to `.ace-retros/_archive/` with chronological sub-paths
7. **Handle container/group folders** (retro lesson): If any subfolder contains further `.md` files rather than being a flat file, recurse explicitly

#### Target structure

```

├── 8prlzl-task-291-ace-idea-gem/
│   └── 8prlzl-task-291-ace-idea-gem.retro.md
```

## Scope of Work

### Codemod scripts (in `codemods/` folder)

```
.ace-taskflow/v.0.9.0/tasks/292-feat-retro-ace/codemods/
  migrate_retros.rb              # Main migration script with --dry-run support
```

**Approach**: Single codemod with classify-on-arrival (retro lesson from 291.04 polish — avoid
multi-pass classification). Script handles all classification, folder creation, frontmatter
injection, and file moves in one pass.

### Key implementation details

- **B36TS discriminator**: `\A[0-9][0-9a-z]{5}-` (from polish retro)
- **ID generation for legacy files**: Use `Ace::B36ts.encode(Time.now)` with small offsets to avoid collisions
- **Frontmatter injection**: Use `Ace::Support::Markdown` frontmatter parser to read existing frontmatter (if any), merge new fields, write back
- **Git provenance**: `git log --diff-filter=A --format='%aI' -- <path>` for `created_at:` when not derivable from filename
- **Container folder handling**: Explicitly check if `archived/` or any other directory contains sub-files; recurse rather than treating as a single item

## Out of Scope

- Permanent `ace-retro migrate` CLI command (overkill for one-time operation)
- Content transformation (retro body content stays as-is)
- Cross-gem reference linking (retro ↔ task references — future work)
- Frontmatter schema evolution beyond required fields
