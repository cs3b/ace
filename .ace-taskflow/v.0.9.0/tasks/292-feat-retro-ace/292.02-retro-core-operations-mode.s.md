# ace-retro CLI Commands and Polish

## Behavioral Specification

### User Experience
- **Input**: CLI commands via terminal: `ace-retro create|show|list|move|update`
- **Process**: dry-cli registry dispatches to command classes, which delegate to RetroManager
- **Output**: Formatted terminal output, retro files created/modified on disk

### Expected Behavior

Wire RetroManager to 5 CLI commands following the same ACE patterns as ace-task CLI (290.03) and ace-idea CLI (291.03):

1. **create**: `ace-retro create [TITLE] [--type TYPE] [--tags T] [--task-ref REF] [--move-to FOLDER] [--dry-run]`
2. **show**: `ace-retro show REF [--path | --content]`
3. **list**: `ace-retro list [--status S] [--type T] [--tags T] [--in FOLDER] [--root PATH]`
4. **move**: `ace-retro move REF --to FOLDER`
5. **update**: `ace-retro update REF [--set K=V]... [--add K=V]... [--remove K=V]...`

### Interface Contract

```bash
# Create with title
ace-retro create "Sprint Review" --type standard --tags sprint,team --task-ref q7w

# Create with dry-run
ace-retro create "Quick self-review" --type self-review --dry-run

# Show
ace-retro show q7w
ace-retro show 8ppq7w --path           # Just file path
ace-retro show q7w --content            # Raw markdown

# List
ace-retro list                          # All retros, flat
ace-retro list --in archive             # Retros in _archive/
ace-retro list --type standard --status active

# Move
ace-retro move q7w --to archive
# => Moves to _archive/2026-02/8ppq7w-sprint-review/

# Update
ace-retro update q7w --set status=done
ace-retro update q7w --add tags=reviewed --remove tags=in-progress

# Standard
ace-retro version
ace-retro help
```

**Exit Codes:**
- 0: Success
- 1: General error (via `Ace::Core::CLI::Error`)
- 130: SIGINT

### Success Criteria

- [ ] All 5 commands work end-to-end from terminal
- [ ] `--type` sets retro type correctly (standard, conversation-analysis, self-review)
- [ ] `--task-ref` links retro to task and stores in frontmatter
- [ ] `--move-to archive` on create places retro in chronological archive sub-path
- [ ] `--set/--add/--remove` work identically to ace-task and ace-idea update
- [ ] `--in archive` auto-maps to `_archive/` prefix
- [ ] `--type` filter works in list command
- [ ] Error messages are actionable
- [ ] SIGINT handled with exit code 130
- [ ] `.ace-defaults/retro/config.yml` provides sensible defaults
- [ ] CHANGELOG.md and README.md written
- [ ] Integration tests cover CLI roundtrips for all commands
- [ ] `ace-test ace-retro` passes all tests
