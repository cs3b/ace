# Prerequisite: Add --format json to ace-assign status

## Objective

Add structured JSON output to `ace-assign status` CLI command so ace-overseer can query assignment state via the public interface without reading internal files.

## Behavioral Specification

### User Experience

```bash
# Existing behavior (unchanged)
ace-assign status
# Output: formatted table with hierarchy

# New: structured JSON output
ace-assign status --format json
# Output: JSON with assignment metadata, phase states, progress
```

### Interface Contract

```bash
ace-assign status --format json
# {
#   "assignment": { "id": "8or5kx", "name": "work-on-task-230", "state": "running" },
#   "phases": [
#     { "number": "010", "name": "onboard", "status": "done" },
#     { "number": "020", "name": "work-on-task", "status": "in_progress" }
#   ],
#   "current_phase": { "number": "020", "name": "work-on-task", "skill": "ace:work-on-task" },
#   "progress": "1/5 done"
# }

ace-assign status --format json --assignment abc123
# Same structure, targeting specific assignment
```

### Success Criteria

- [ ] `ace-assign status --format json` outputs valid JSON to stdout
- [ ] JSON contains assignment id, name, state
- [ ] JSON contains all phases with number, name, status
- [ ] JSON contains current phase info (or null if completed)
- [ ] JSON contains progress string
- [ ] Default format ("table") behavior is unchanged
- [ ] Existing tests still pass
