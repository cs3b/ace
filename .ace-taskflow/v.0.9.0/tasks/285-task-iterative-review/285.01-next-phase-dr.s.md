# Simulation Session Framework + Cache Contract (b36ts)

## Behavioral Specification

### User Experience
- **Input**: Reviewer invokes next-phase simulation from idea/task context with artifact reference and optional mode overrides.
- **Process**: System creates a simulation session ID using base36 timestamp semantics, executes stage simulations in memory, and persists intermediate artifacts to a deterministic cache directory.
- **Output**: A complete run bundle exists at `.cache/ace-taskflow/simulations/<b36ts-id>/` with stage outputs, synthesis output, and write-back preview.

### Expected Behavior

Simulation sessions must be structured, deterministic, and auditable.

A run must:
- Create unique `run_id` compatible with `ace-b36ts` style IDs
- Create a cache directory at `.cache/ace-taskflow/simulations/<run_id>/`
- Persist all intermediate artifacts (request context, stage outputs, synthesis data)
- Produce a run summary and write-back preview without mutating downstream artifacts

### Interface Contract

```bash
# Logical invocation shape (interface contract)
ace-taskflow review next-phase --source <idea|task-ref> [--modes draft,plan] [--no-writeback]

# Session artifact root
.cache/ace-taskflow/simulations/<b36ts-id>/
```

Expected cache artifacts:
- `request.yml`
- `stage-idea-draft.yml` (when idea path includes draft stage)
- `stage-idea-plan.yml` (when idea path includes plan stage)
- `stage-task-plan.yml` (when task path includes plan stage)
- `synthesis.yml`
- `writeback-preview.md`
- `run-summary.md`

**Error Handling:**
- Missing source artifact: fail fast with actionable error, no partial write-back.
- Cache write failure: fail run and report path/permission issue.
- Stage simulation failure: persist partial run metadata plus explicit failed stage.

**Edge Cases:**
- Re-run with same source: each run gets a new `<b36ts-id>` directory.
- Empty simulation output from a stage: synthesis still runs and records "no findings" for that stage.

### Success Criteria

- [ ] Every simulation run generates a unique `<b36ts-id>` cache directory
- [ ] Intermediate artifacts are fully persisted under the session directory
- [ ] No downstream draft/task/plan artifacts are created by the simulation run
- [ ] Run summary and write-back preview are always generated (or explicit failure file)

### Validation Questions

- [x] Run identifier format source? -> `ace-b36ts` compatible base36 timestamp IDs.
- [x] Where should intermediate artifacts live? -> `.cache/ace-taskflow/simulations/<b36ts-id>/`.
- [x] Should simulation mutate downstream artifacts? -> No, intermediate outputs are cache-only.

Define the foundational simulation session contract so all next-phase review flows share the same ID, storage, and artifact semantics.
