# Implement reverse synthesis ordering and simulation demo coverage

## Behavioral Specification

### User Experience
- **Input**: User runs idea simulation with `--modes draft,plan`.
- **Process**: Synthesis merges questions/refinements from stage outputs with reverse ordering priority.
- **Output**: Final synthesis and writeback surface `plan` questions before `draft` questions.

### Expected Behavior
- Enhance `SimulationSynthesisBuilder` to support reverse stage ordering for question/refinement aggregation.
- Default behavior for next-phase synthesis uses reverse order for multi-stage idea runs.
- Keep uniqueness filtering while preserving first-seen order under reversed aggregation.
- Add or update tests proving `plan -> draft` ordering.
- Add demo-oriented verification notes in task docs:
  - exact command to run on idea `8pp5z7-llm-enhance`
  - expected artifacts and where reverse order is observable

### Interface Contract

```bash
ace-taskflow review-next-phase --source 8pp5z7-llm-enhance --modes draft,plan --no-writeback --verbose
```

Expected synthesized ordering:
- `questions` list begins with plan-stage questions
- draft-stage questions follow

### Verification Evidence

- **Command**: `mise exec -- ace-taskflow review-next-phase --source 8pp5z7-llm-enhance --modes draft,plan --no-writeback --verbose`
- **Artifacts**: `.cache/ace-taskflow/simulations/<run-id>/synthesis.yml`, `.cache/ace-taskflow/simulations/<run-id>/writeback-preview.md`
- **Expected vs Observed**: `pass` when `plan` questions precede `draft` questions in both synthesis and writeback preview.
- **Failure Classification**: `implementation failure` when ordering is incorrect with successful stage outputs; `provider unavailable` when model call cannot execute.

- [ ] Reverse stage ordering is implemented and covered by tests
