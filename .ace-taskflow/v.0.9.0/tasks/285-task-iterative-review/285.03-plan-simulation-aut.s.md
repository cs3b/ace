# Task-State Plan Simulation + Auto/Manual Trigger Controls

## Behavioral Specification

### User Experience
- **Input**: User/reviewer triggers simulation from an existing task artifact, either manually or via auto-review behavior.
- **Process**: System simulates planning for the selected task, synthesizes blockers/questions, and applies final questions back to the task artifact.
- **Output**: Updated task with final simulation-derived questions/refinements, plus full cached run artifacts.

### Expected Behavior

Task-origin simulation behavior:
- Run plan-stage simulation for the selected task artifact.
- Produce structured synthesis from plan output.
- Write final synthesized questions/refinements back to the task artifact.

Trigger behavior:
- Manual mode must always be available for explicit invocation.
- Auto-review mode is enabled by default for idea and task workflows.
- Users can disable/enable via configuration cascade and per-run CLI overrides.

Configuration precedence:
- CLI flags > project `.ace/` > user `~/.ace/` > gem defaults

### Interface Contract

```bash
# Manual trigger (logical contract)
review-next-phase --source <task-ref> --modes plan

# Auto-review overrides (logical contract)
--next-phase-review
--no-next-phase-review
--next-phase-modes <csv>
```

Proposed config keys (taskflow namespace):
- `review.next_phase.enabled`
- `review.next_phase.auto.idea`
- `review.next_phase.auto.task`
- `review.next_phase.cache_dir` (default: `.cache/ace-taskflow/simulations`)

**Error Handling:**
- Invalid task reference: fail with actionable resolution guidance.
- Auto-trigger enabled but simulation fails: preserve primary workflow behavior and report non-blocking simulation failure.
- Disabled by config with explicit CLI enable: CLI override must force run.

**Edge Cases:**
- Both auto-trigger and manual trigger requested: execute once, avoid duplicate runs.
- Task already has open review questions: merge/append deterministically instead of overwriting unrelated content.

### Success Criteria

- [ ] Task-state plan simulation produces final questions/refinements written to source task artifact
- [ ] Manual trigger path works independently of auto-trigger state
- [ ] Auto-trigger defaults are ON for idea and task flows
- [ ] Config cascade and CLI override precedence are explicit and deterministic
- [ ] Intermediate artifacts remain cache-only under simulation session directory

### Validation Questions

- [x] Default auto-review mode? -> Enabled by default for both idea and task flows.
- [x] Where do final questions belong? -> The reviewed source artifact (`task` file).
- [x] Where do intermediate artifacts belong? -> `.cache/ace-taskflow/simulations/<b36ts-id>/`.
