# Idea-State Next-Phase Simulation (Draft + Plan) and Write-Back

## Behavioral Specification

### User Experience
- **Input**: User/reviewer selects an idea artifact and triggers next-phase review simulation.
- **Process**: System simulates draft-task behavior first, then simulates plan-task behavior using the simulated draft context.
- **Output**: Final synthesized questions/refinements are written to the source idea artifact; full intermediate results remain in the cache session directory.

### Expected Behavior

For idea-origin reviews, simulation is a two-stage chain:
1. Simulate drafting from the source idea.
2. Simulate planning based on the simulated draft output.
3. Synthesize both stages into final guidance.
4. Write final guidance back to the idea artifact.

Write-back rules:
- Only final synthesized questions/refinements are persisted to the idea file.
- Intermediate draft/plan simulation artifacts are never written as downstream task/plan files.

### Interface Contract

```bash
# Logical workflow contract
review-next-phase --source <idea-ref> --modes draft,plan
```

Cache outputs (inside `.cache/ace-taskflow/simulations/<b36ts-id>/`):
- `stage-idea-draft.yml`
- `stage-idea-plan.yml`
- `synthesis.yml`
- `writeback-preview.md`

Idea write-back contract:
- Add/update a dedicated section for simulation-derived questions/refinements
- Preserve existing idea content and frontmatter structure

**Error Handling:**
- Draft simulation fails: planning stage is skipped, run marked failed with clear reason.
- Plan simulation fails: draft stage artifacts remain, synthesis captures partial result and unresolved gaps.
- Write-back failure: cache run remains intact, user receives explicit manual-apply instructions.

**Edge Cases:**
- Idea already contains prior simulation questions: update in-place with latest run, avoid duplicate blocks.
- Sparse ideas: synthesize structural questions instead of failing when details are incomplete.

### Success Criteria

- [ ] Idea simulations execute `draft` then `plan` stages in deterministic order
- [ ] Final synthesized questions/refinements are written back to the source idea artifact
- [ ] No downstream task artifacts are created by simulation
- [ ] Stage artifacts and synthesis outputs are persisted to cache session directory

### Validation Questions

- [x] Should final questions be stored only in cache? -> No, final questions must be written back to the reviewed artifact.
- [x] Should simulated draft output create real task files? -> No, simulation outputs remain cache-only.
- [x] Should plan stage depend on draft stage output? -> Yes, plan simulation consumes simulated draft context.

## Objective

Surface missing context and decision gaps at the idea stage by simulating both draft and planning phases before committing to workflow state transitions.
