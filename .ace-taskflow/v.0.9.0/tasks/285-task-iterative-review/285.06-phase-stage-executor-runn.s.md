# Implement next-phase LLM stage executor and runner integration

## Behavioral Specification

### User Experience
- **Input**: User runs `ace-taskflow review-next-phase` for idea/task source with selected modes.
- **Process**: Runner invokes a default stage executor that loads simulation workflow content, calls `Ace::LLM::QueryInterface`, and normalizes model output.
- **Output**: Real LLM-derived stage artifacts are persisted instead of fallback hardcoded placeholders.

### Expected Behavior
- Add a molecule (or equivalent integration unit) to execute simulation stages:
  - mode-to-workflow mapping (`draft`, `plan`, `work` where applicable)
  - source artifact loading
  - previous stage payload propagation
  - LLM call with read-only sandbox intent
  - robust response parsing and normalization to runner schema
- Wire the executor as default in `ReviewNextPhase`/`NextPhaseSimulationRunner` path while preserving injectable override for tests.
- Preserve existing failure behavior:
  - stage partials and run artifacts continue to persist
  - no silent fallback to hardcoded placeholder when LLM call fails
- Keep cache/writeback responsibilities in existing molecules.

### Interface Contract
- Executor callable signature:
  - `resolved_source:`
  - `mode:`
  - `run_id:`
  - `previous_stage_output:`
- Executor return payload:
  - `status`
  - `findings[]`
  - `questions[]`
  - `refinements[]`
  - optional `unresolved_gaps[]`
- LLM call contract uses `Ace::LLM::QueryInterface.query(...)` with read-only sandbox metadata.

- **Command**: `mise exec -- ace-test ace-taskflow/test/molecules/next_phase_stage_executor_test.rb ace-taskflow/test/organisms/next_phase_simulation_runner_test.rb`

- **Failure Classification**: `implementation failure` if executor wiring/parsing assertions fail; `provider unavailable` when runtime query fails due external provider reachability/auth.

### Success Criteria
- [ ] Runner default stage execution path uses new LLM stage executor
- [ ] Stage payloads are parsed/normalized to deterministic schema
- [ ] Previous stage output is passed into downstream stage execution
- [ ] Existing tests for partial failure artifact persistence still pass with new executor wiring

### Validation Questions
