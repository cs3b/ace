# Work-On Simulation Extension Subtask

## Behavioral Specification

### User Experience
- **Input**: User/reviewer opts into extension mode for simulating work-on phase from a prepared upstream context.
- **Process**: System runs a separate work-on simulation path that reuses the same session/cache/write-back contracts but does not alter default idea/task simulation behavior.
- **Output**: Work-on extension simulation artifacts are produced in cache and optional final guidance can be written back to the reviewed source artifact when mode is enabled.

### Expected Behavior

This subtask defines an extension, not baseline behavior.

Rules:
- Work-on simulation must be disabled by default.
- Enabling work-on simulation must require explicit activation (config or CLI mode selection).
- Existing idea/task simulation paths must remain unchanged when extension is disabled.
- Extension runs must reuse `.cache/ace-taskflow/simulations/<b36ts-id>/` artifact structure.

### Interface Contract

```bash
# Extension mode (logical contract)
review-next-phase --source <idea|task-ref> --modes plan,work

# Optional config key
review.next_phase.include_work_simulation: false  # default
```

**Error Handling:**
- Work simulation requested without prerequisite context: fail extension stage with explicit prerequisite guidance.
- Mixed-mode run where work stage fails: preserve earlier stage artifacts and synthesis, annotate failure in summary.

**Edge Cases:**
- Extension enabled globally but disabled by CLI for one run: CLI must take precedence.
- Unknown mode token in CLI (`--next-phase-modes`): reject with clear valid-mode list.

### Success Criteria

- [ ] Work-on simulation is defined as an optional extension, not default behavior
- [ ] Extension activation requires explicit mode/config enablement
- [ ] Baseline idea/task simulation behavior remains unaffected when extension is off
- [ ] Extension reuses existing session and cache contracts

### Validation Questions

- [x] Should work-on simulation be in baseline scope? -> No, it is a dedicated extension subtask.
- [x] Default extension mode? -> Disabled (`false`).
- [x] Should extension share cache contract? -> Yes, same session directory and artifact model.

## Objective

Prepare a safe extension path for future work-on simulation without introducing scope or behavior churn into the initial idea/task simulation rollout.

## Scope of Work

- **User Experience Scope**: Explicit opt-in behavior for work simulation.
- **System Behavior Scope**: Extension boundaries, mode gating, and compatibility with baseline simulation flows.
- **Interface Scope**: Extension mode contract and optional config key.

### Deliverables

#### Behavioral Specifications
- Work simulation opt-in semantics
- Mode gating and compatibility guarantees
- Extension stage failure behavior

#### Validation Artifacts
- Mode compatibility scenarios (`draft,plan` vs `plan,work`)
- Extension-off regression scenarios

## Out of Scope

- ❌ Implementing baseline idea/task simulation contracts
- ❌ Auto-trigger defaults for baseline flows
- ❌ Real execution of work-on-task phase or state transitions
