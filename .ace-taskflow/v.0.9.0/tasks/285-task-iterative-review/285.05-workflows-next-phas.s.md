# Add LLM simulation workflows for next-phase draft and plan (read-only)

## Behavioral Specification

### User Experience
- **Input**: Reviewer runs `ace-taskflow review-next-phase --source <idea-ref> --modes draft,plan [--no-writeback]`.
- **Process**: Runtime loads dedicated simulation workflows for draft and plan analysis and passes source content plus upstream stage context to the LLM.
- **Output**: Stage artifacts include deterministic, structured analysis payloads that can be consumed by synthesis and writeback molecules.

### Expected Behavior
- Add two workflow files:
  - `ace-taskflow/handbook/workflow-instructions/task/simulate-next-phase-draft.wf.md`
  - `ace-taskflow/handbook/workflow-instructions/task/simulate-next-phase-plan.wf.md`
- Workflows are simulation-only and must state no downstream draft/plan/work artifact writes.
- Draft workflow analyzes an idea as input and returns:
  - `status`
  - `findings[]`
  - `questions[]`
  - `refinements[]`
  - optional `unresolved_gaps[]`
- Plan workflow accepts prior draft stage context and returns the same schema.
- Workflow instructions include failure guidance for malformed source content and insufficient context.

### Interface Contract

```bash
# Workflow resolution targets
wfi://task/simulate-next-phase-draft
wfi://task/simulate-next-phase-plan
```

Stage output schema (both workflows):
- `status: ok|partial|failed`
- `findings: []`
- `questions: []`
- `refinements: []`
- `unresolved_gaps: []` (optional)

### Success Criteria
- [ ] Both simulation workflow files exist and resolve through `ace-bundle wfi://...`
- [ ] Workflows explicitly enforce read-only simulation semantics
- [ ] Output schema is consistent across draft and plan simulation stages
- [ ] Plan workflow contract consumes previous stage context

### Validation Questions
- [ ] Should workflows include model-specific prompts or remain provider-agnostic?
- [ ] Should unresolved gaps be mandatory when status is `partial`?

## Objective

Provide explicit, reusable workflow contracts so next-phase simulation can invoke LLM analysis deterministically instead of placeholder stage output.

## Scope of Work

- **In scope**: Simulation workflow instruction files, schema definition, read-only guardrails, stage-context contract.
- **Out of scope**: Runner execution code, synthesis ordering logic, CLI contract changes.

## Deliverables

- `simulate-next-phase-draft.wf.md` workflow
- `simulate-next-phase-plan.wf.md` workflow
- Structured output contract examples embedded in both files
