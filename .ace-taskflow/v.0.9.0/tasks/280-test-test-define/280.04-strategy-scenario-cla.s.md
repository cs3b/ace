# Test Grouping Strategy and Scenario Classification

## Scope

Define standard tag groups for E2E scenarios, then classify all ~33 existing scenarios with appropriate tags and cost-tiers. This subtask uses the tag infrastructure from 280.03 to create a meaningful, queryable test taxonomy.

> **Pilot note**: The ace-b36ts pilot (280.01) consolidated 4 procedural scenarios into 1 goal-mode scenario, reducing the total from 36 to ~33. The cost-tier "smoke" was validated as appropriate for the goal-based scenario.

> **Dependency note**: 280.02 must define what smoke / happy-path / deep mean for goal-mode scenarios (runner config, complexity, verification depth) before this task can identify which existing scenarios are candidates for goal-mode conversion at each tier. The current classification is tag-only; once 280.02 delivers tier semantics, this task should also map conversion candidates per tier.

## Behavioral Spec

### Standard Tag Groups

| Group | Tag | Purpose | Exec Time Target | Selection Criteria |
|-------|-----|---------|------------------|--------------------|
| Smoke | `smoke` | Fast confidence check | <20% of total suite time | Core happy-path per package, minimal setup |
| Happy Path | `happy-path` | Standard functional coverage | ~50% of suite time | Primary use-cases, expected inputs |
| Use-Case | `use-case:<verb-noun>` | Workflow-specific selection | Varies | Agent-driven selection by task context |
| Deep | `deep` | Edge cases, error paths, complex flows | Remaining ~30% | Multi-step workflows, error recovery, boundary conditions |

### Use-Case Tag Vocabulary

Namespaced tags that enable agent-driven test selection:
- `use-case:lint` — Linting and code quality
- `use-case:commit` — Git commit generation
- `use-case:review` — Code review analysis
- `use-case:assign` — Assignment workflow
- `use-case:bundle` — Context loading and bundling
- `use-case:nav` — Resource discovery and navigation
- `use-case:secrets` — Security scanning
- `use-case:worktree` — Git worktree management
- `use-case:overseer` — Task orchestration
- `use-case:prep` — Prompt preparation

### Cost-Tier Classification

| Tier | Tag | LLM Calls | Typical Duration | Example |
|------|-----|-----------|------------------|---------|
| smoke | `cost-tier:smoke` | 0-1 | <30s | CLI flag validation, basic output check |
| standard | `cost-tier:standard` | 1-3 | 1-5min | Single workflow execution |
| deep | `cost-tier:deep` | 3+ | 5-15min | Multi-step workflow, error recovery |

### Classification Requirements

- Every scenario gets: at least one group tag + cost-tier + priority
- Every package must have at least one `smoke` scenario
- Use-case tags assigned based on the workflow domain
- Multiple group tags allowed (a scenario can be both `smoke` and `happy-path`)
