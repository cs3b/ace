# Move ace-lint CLI Integration Tests to E2E

## Objective

Reduce ace-lint test execution time from 10.87s to ~2-3s by moving CLI integration tests (which spawn real ace-lint binary via Open3.capture3) to E2E test suite and adding mocks to unit tests.

## Scope of Work

- **Test Optimization**: Move CLI integration tests to E2E suite
- **Mock Enhancement**: Add Open3.capture3 mocks to unit tests
- **Performance Target**: 75% reduction in test execution time (~8s saved)

### Deliverables

#### Create
- `ace-lint/test/e2e/MT-LINT-004-cli-exit-codes.mt.md`
- `ace-lint/test/e2e/MT-LINT-005-doctor-command.mt.md`

#### Delete
- `ace-lint/test/integration/cli_integration_test.rb` (10 tests)
- `ace-lint/test/integration/doctor_integration_test.rb` (5 tests)

#### Modify
- `ace-lint/test/molecules/validator_executor_test.rb` (add Open3 mocks)
- `ace-lint/CHANGELOG.md`
