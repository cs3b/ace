# Multi-Session Support

## Overview

Enable multiple concurrent sessions with listing, selection, and filtering. Extends existing single-session model with explicit session selection while maintaining backwards compatibility.

## Objective

Add multi-session management capabilities:
- Explicit session selection via `.current` symlink
- `ace-coworker list` command to show all sessions
- `ace-coworker select` command for session switching
- Enhanced `status` showing current job + other sessions
- `--session` flag on all commands for explicit targeting
- `ACE_COWORKER_SESSION` env var for workflow context propagation

## Technical Research Findings

### Existing Infrastructure

The codebase already has partial multi-session support:

1. **SessionManager.list()** (lines 121-132 of `session_manager.rb`):
   - Implemented but unused
   - Returns all sessions sorted by `updated_at` descending

2. **`.latest` symlink** (lines 174-190):
   - Updated on every session create/update
   - Provides O(1) active session lookup
   - Used by `find_active()` as fast path

3. **QueueState.summary** (lines 98-106):
   - Returns `{total, done, in_progress, pending, failed}` stats
   - Ready for session list display

4. **Registered Commands** (`cli.rb` line 42):
   - `create, status, report, fail, add, retry, prepare`
   - Note: No `start` or `resume` - original spec referenced wrong commands

### Session State Derivation

Sessions can be in one of these states (to be computed from QueueState):
- **running**: Has `in_progress` step
- **paused**: Has `pending` but no `in_progress` (interrupted)
- **completed**: All steps `done`
- **failed**: Has `failed` step(s)

## Technical Approach

### Symlink Strategy: `.latest` + `.current`

```
.cache/ace-coworker/
├── .latest → abc123/          # Auto-updated on any activity
├── .current → def456/         # Explicit user selection (optional)
├── abc123/
│   └── session.yaml
└── def456/
    └── session.yaml
```

**Resolution order for `find_current()`:**
1. `--session <id>` flag (highest priority)
2. `ACE_COWORKER_SESSION` env var
3. `.current` symlink (explicit selection)
4. `.latest` symlink (auto, most recent)
5. Scan all sessions (fallback)

### SessionDiscoverer Molecule

```ruby
module Ace::Coworker::Molecules
  class SessionDiscoverer
    def initialize(cache_base:, queue_scanner:)
      @cache_base = cache_base
      @queue_scanner = queue_scanner
    end

    # Find all sessions with computed state
    # @return [Array<SessionInfo>]
    def find_all(include_completed: false)
      sessions = session_manager.list
      sessions.map { |s| enrich_session(s) }
              .select { |si| include_completed || !si.completed? }
    end

    # Find sessions by task reference
    def find_by_task(task_ref:, active_only: true)
      find_all(include_completed: !active_only)
        .select { |si| si.session.task_ref == task_ref }
    end

    private

    def enrich_session(session)
      state = @queue_scanner.scan(session.jobs_dir, session: session)
      SessionInfo.new(session: session, queue_state: state)
    end
  end
end
```

### SessionInfo Model

```ruby
module Ace::Coworker::Models
  class SessionInfo
    attr_reader :session, :queue_state

    def initialize(session:, queue_state:)
      @session = session
      @queue_state = queue_state
    end

    # Computed state: :running, :paused, :completed, :failed
    def state
      return :empty if queue_state.empty?
      return :failed if queue_state.failed.any?
      return :completed if queue_state.complete?
      return :running if queue_state.current
      :paused
    end

    def progress
      s = queue_state.summary
      "#{s[:done]}/#{s[:total]}"
    end

    def current_step
      queue_state.current&.name || "-"
    end

    # Delegate to session
    def id = session.id
    def name = session.name
    def updated_at = session.updated_at
  end
end
```

### CLI Interface

```bash
# List sessions
ace-coworker list                    # Active sessions
ace-coworker list --all              # Include completed
ace-coworker list --task 148         # Filter by task
ace-coworker list --format json      # JSON output

# Select session
ace-coworker select abc123           # Set .current
ace-coworker select --clear          # Remove .current

# Status with session
ace-coworker status                  # Current + other sessions
ace-coworker status --session abc123 # Specific session
ace-coworker status --all            # Show completed too
ace-coworker status --brief          # Summary only

# Other commands with --session
ace-coworker report result.md --session abc123
ace-coworker fail --session abc123 --message "blocked"
```
