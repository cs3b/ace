# Multi-Session Support

## Overview

Enable multiple concurrent sessions with listing and filtering. Based on 234.07.

## Objective

Add multi-session management:
- `ace-coworker list` command to show all sessions
- Session status filtering: active, completed, failed
- `--task` filter to find sessions for a task
- Auto-resume when starting same task again

## Scope of Work

### From 234.07

**Included:**
- `ace-coworker list` command with table output
- `--format json` for programmatic use
- `--task` filter by task reference
- `--all` to include completed sessions
- Auto-resume: `start --task X` resumes existing active session
- `--new-session` to force new session creation
- `--session` flag on status/resume commands

**Excluded:**
- Session TTL/expiration
- Session cleanup/pruning
- Cross-directory discovery
- Session locking
- Remote synchronization

```

│   └── jobs/

│   ├── session.yaml
│   └── jobs/
└── ghi789/              # Another session for task 148

    └── jobs/
```

### CLI Interface

```bash
# List all active sessions
ace-coworker list
# Output:
# SESSION  TASK  STATUS   STEP     PROGRESS
# abc123   148   running  020-impl  2/5
# def456   149   paused   030-gate  3/4

# List with filters
ace-coworker list --task 148
ace-coworker list --all           # Include completed
ace-coworker list --format json   # JSON output

# Auto-resume existing session
ace-coworker start --task 148
# Output:
# Resuming existing session abc123 for task 148
# ...

# Force new session
ace-coworker start --task 148 --new-session
# Output:
# Created new session xyz999 for task 148

# Operate on specific session
ace-coworker status --session abc123
ace-coworker resume --session abc123 --approve
```

### Auto-Resume Logic

```
ace-coworker start --task X
  │
  ├── --session <id> → Resume that session
  ├── --new-session → Create new session
  └── (default) →
        ├── Find active sessions for task X
        │     ├── Found → Resume most recent
        │     └── None → Create new session
```

### Session Info Schema

```yaml
# session.yaml additions
task_ref: "148"            # Task reference for filtering
created_at: '2026-01-28T10:00:00Z'
updated_at: '2026-01-28T12:30:00Z'
```
