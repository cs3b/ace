# Human Approval Gates

## Overview

Pause workflow for human approval at critical points. Based on 234.06.

## Objective

Add gate support to workflow execution:
- Gate step type pauses workflow
- State persisted for resume
- Resume with approve/reject commands
- Simple rejection handling (mark session, don't rewind)

## Scope of Work

### From 234.06

**Included:**
- `gate: true` field in step frontmatter
- `status: paused` when gate step reached
- `ace-coworker resume --approve` command
- `ace-coworker resume --reject --reason "..."` command
- Gate state in session.yaml

**Excluded:**
- Multiple gate types (only "human" gate)
- Timeout gates
- Conditional gates
- Auto-rewind on rejection
- External API/webhook approval

## Technical Approach

### Step Frontmatter Extension

```yaml
---
status: paused
gate: true
gate_prompt: "Review implementation before creating PR"
---
```

### Session State for Gates

```yaml
# session.yaml
id: abc123
status: paused
gate:
  step: 030-review-gate
  paused_at: '2026-01-28T12:00:00Z'
  prompt: "Review implementation before creating PR"
```

### CLI Interface

```bash
# Check status shows gate
ace-coworker status
# Output:
# Session: abc123
# Status: paused at gate
# Step: 030-review-gate
#
# Gate prompt:
# Review implementation before creating PR
#
# To continue: ace-coworker resume --approve
# To reject: ace-coworker resume --reject --reason "..."

# Approve and continue
ace-coworker resume --approve
# Output:
# Gate approved
# Advancing to step 040: create-pr

# Reject with reason
ace-coworker resume --reject --reason "Need to fix X first"
# Output:
# Gate rejected: Need to fix X first
# Session marked as rejected
# Manual recovery required
```

### Workflow Flow

```
Step completes → Check next step
  │
  ├── Regular step → Continue normally
  └── Gate step →
        ├── Set status: paused
        ├── Store gate state in session
        └── Wait for ace-coworker resume
              │
              ├── --approve → Clear gate, advance
              └── --reject → Mark session rejected
```

### Rejection Behavior

When rejected:
1. Status set to `rejected`
2. No auto-rewind (previous steps stay done)
3. Manual recovery: edit session.yaml or start fresh
4. Commits/files from previous steps NOT rolled back
