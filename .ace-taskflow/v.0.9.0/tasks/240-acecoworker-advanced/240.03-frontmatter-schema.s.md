# Frontmatter Schema

## Overview

Declarative session configuration in task/workflow files. Based on 234.08.

## Objective

Add `coworker:` frontmatter schema for declarative configuration:
- Parse `coworker:` section from .wf.md files
- Define steps, presets, variables declaratively
- Session hierarchy (parent/child sessions)
- Simplify preset management

## Scope of Work

### From 234.08

**Included:**
- `coworker:` frontmatter block parsing
- Session configuration: required, inherit, parent
- Workflow steps defined in frontmatter
- Parent-child session linking
- `--tree` option on list command

**Simplified:**
- Fewer configuration options than 234.08
- No context/delegation/gates blocks (use simpler patterns)
- Focus on core schema, not full feature set

## Technical Approach

### Core Schema

```yaml
---
coworker:
  # Session configuration
  session:
    required: true        # Require tracked session
    parent: "abc123"      # Parent session ID (optional)

  # Workflow definition
  workflow:
    preset: work-on-task  # Use existing preset
    # OR inline steps:
    steps:
      - name: implement
        instructions: |
          Execute task implementation.
          Report with: ace-coworker report <file>
      - name: test
        instructions: |
          Run tests.
          Report with: ace-coworker report <file>
        verification: ace-test

  # Variables for interpolation
  variables:
    taskref: "148"
    branch: "148-feature"
---

# Task 148: Implement Feature

...task content...
```

### Session Hierarchy

Child sessions link to parents:

```yaml
# Child session.yaml
id: abc123.1
parent: abc123
task_ref: "148.01"
```

```
abc123           # Parent (task 148)
├── abc123.1     # Child (subtask 148.01)
└── abc123.2     # Child (subtask 148.02)
```

### CoworkerConfigParser Atom

```ruby
module Ace::Coworker::Atoms
  class CoworkerConfigParser
    # Parse coworker configuration from frontmatter
    # @param frontmatter [Hash] Parsed YAML frontmatter
    # @return [Hash] { config:, valid:, errors: }
    def self.parse(frontmatter)
      coworker = frontmatter["coworker"] || frontmatter[:coworker]
      return empty_result unless coworker

      config = {
        session_required: dig_bool(coworker, %w[session required], false),
        session_parent: dig_string(coworker, %w[session parent]),
        workflow_preset: dig_string(coworker, %w[workflow preset]),
        workflow_steps: dig_array(coworker, %w[workflow steps]),
        variables: dig_hash(coworker, %w[variables])
      }

      { config: config, valid: true, errors: [] }
    end
  end
end
```

### CLI Interface

```bash
# Prepare from file with coworker: frontmatter
ace-coworker prepare --file task-148.s.md
# → Reads coworker: section
# → Uses workflow.preset or workflow.steps
# → Applies variables

# Create child session
ace-coworker prepare --file task-148.01.s.md --parent abc123
# → Creates abc123.1
# → Links to parent

# List with hierarchy
ace-coworker list --tree
# Output:
# SESSION    TASK     STATUS    STEP
# abc123     148      running   orchestrate
# ├─ abc123.1  148.01   completed   2/2
# ├─ abc123.2  148.02   running     1/2
# └─ abc123.3  148.03   pending     0/2
```

### Usage Example

Task file with inline workflow:

```yaml
---
id: v.0.9.0+task.148
status: pending

coworker:
  session:
    required: true
  workflow:
    steps:
      - name: implement
        instructions: |
          Implement task 148 following the spec below.
          Report with: ace-coworker report <file>
      - name: test

          Report with: ace-coworker report <file>

        instructions: |

          Report with: ace-coworker report <file>
---

...
```
