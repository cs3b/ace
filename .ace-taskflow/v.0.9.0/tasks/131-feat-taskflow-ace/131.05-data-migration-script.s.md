# Data Migration: Physical Folder Renames

## Description

Create a migration script/command to physically rename folders from old naming convention to new `_` prefix convention. This should be run after code changes are deployed.

## Acceptance Criteria

- [ ] Create `ace-taskflow migrate` command (or similar)
- [ ] Support `--dry-run` mode to preview changes
- [ ] Rename `.ace-taskflow/done/` → `_archive/`
- [ ] Rename `.ace-taskflow/backlog/` → `_backlog/`
- [ ] Rename `v.*/tasks/done/` → `_archive/` (all releases)
- [ ] Rename `v.*/ideas/done/` → `_archive/` (all releases)
- [ ] Handle errors gracefully (folder already exists, permissions, etc.)
- [ ] Preserve git history with proper rename detection

## Implementation Notes

### Migration Scope

```
.ace-taskflow/
├── done/           → _archive/      (published releases)
├── backlog/        → _backlog/      (future releases)
└── v.0.9.0/
    ├── tasks/done/ → tasks/_archive/
    └── ideas/done/ → ideas/_archive/
```

### Command Interface

```bash
# Preview changes
ace-taskflow migrate --dry-run

# Execute migration
ace-taskflow migrate

# Migrate specific scope
ace-taskflow migrate --scope releases    # Only done → _archive at root
ace-taskflow migrate --scope tasks       # Only tasks/done → _archive
ace-taskflow migrate --scope ideas       # Only ideas/done → _archive
```

### Implementation Options

1. **Ruby script in ace-taskflow** (recommended)
   - Add `commands/migrate_command.rb`
   - Use FileUtils for atomic renames
   - Git-aware: use `git mv` when in git repo

2. **Shell script** (simpler but less portable)
   - Could be in `scripts/migrate-folders.sh`

### Error Handling

- Skip if target folder already exists
- Warn if source folder is empty
- Handle nested done/ folders in archived releases
