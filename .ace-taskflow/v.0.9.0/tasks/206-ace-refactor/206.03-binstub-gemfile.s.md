# Add ace-bundle Binstub and Gemfile Entry

## Behavioral Specification

### User Experience

- **Input**: Developer has ace-bundle with updated config
- **Process**: Create binstub and add to mono-repo Gemfile
- **Output**: Both `ace-context` and `ace-bundle` commands work

### Expected Behavior

1. **Binstub Creation**: `bin/ace-bundle` created
   - Points to `ace-bundle/exe/ace-bundle`
   - Executable permissions set

2. **Gemfile Update**: Add ace-bundle (keep ace-context)
   - Both gems in Gemfile
   - `bundle install` updates lockfile

3. **Coexistence**: Both commands work
   - `bin/ace-context --version` shows 0.28.2
   - `bin/ace-bundle --version` shows 0.29.0

### Success Criteria

- [ ] **Binstub**: `bin/ace-bundle` exists and is executable
- [ ] **Gemfile**: Both ace-context and ace-bundle listed
- [ ] **Both Work**: Both CLI commands function correctly
- [ ] **Lockfile**: Gemfile.lock includes both gems

### Validation Questions

- [ ] **Order**: Should ace-bundle come before or after ace-context in Gemfile?

## Objective

Add ace-bundle to mono-repo tooling so both packages can coexist during migration.

## Scope of Work

- Create `bin/ace-bundle` binstub
- Update root `Gemfile` (add ace-bundle, keep ace-context)
- Run `bundle install`
