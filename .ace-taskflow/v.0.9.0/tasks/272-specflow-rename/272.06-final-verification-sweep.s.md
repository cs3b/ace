# Final verification sweep

## Behavioral Specification

### User Experience

After all rename subtasks (272.01–272.05) are complete, an agent runs this final sweep to catch any stale references that slipped through the scoped subtasks. The result is a clean repo with zero unintentional references to the old naming.

### Expected Behavior

This task performs exhaustive searches across the entire repository for any remaining references to the old `ace-taskflow`, `Ace::Taskflow`, or `.00-orchestrator` naming patterns. Any findings are either fixed or documented as intentional exceptions.

### Success Criteria

- [ ] `grep -r "ace-taskflow"` across repo (excluding `.git/`) returns zero hits outside intentional exceptions
- [ ] `grep -r "\.00-orchestrator\|\.00-"` returns zero hits outside intentional exceptions
- [ ] `grep -r "Ace::Taskflow\|ace_taskflow"` returns zero hits outside intentional exceptions
- [ ] `ace-test-suite` passes
- [ ] All nav protocols resolve correctly
- [ ] Intentional exceptions documented (retro historical references, CHANGELOG history, etc.)

## Scope of Work

### Exhaustive Search

Run the following searches across the entire repo (excluding `.git/`):

1. **Gem name references**: `grep -r "ace-taskflow"` — catches any remaining gem name, require paths, CLI references
2. **`.00` orchestrator references**: `grep -r "\.00-orchestrator\|\.00-"` — catches any lingering `.00` patterns not cleaned by task 271
3. **Ruby namespace references**: `grep -r "Ace::Taskflow\|ace_taskflow"` — catches module names, underscored gem names, test helpers

### Validation

- Run `ace-test-suite` to verify all tests pass with the fully renamed codebase
- Run `ace-nav --sources` and spot-check protocol resolution (e.g., `ace-nav spec://121`)

### Exception Documentation

Document any intentional remaining references:
- **Retros**: Historical records that reflect what was true at time of writing — leave as-is
- **CHANGELOG**: Historical entries describing the rename itself — leave as-is
- **Git history**: Commit messages referencing old names — immutable, no action needed

### Deliverables

#### Modify
- Any files with stale references found during the sweep

#### Create
- None expected (this is a verification task)

## Implementation Notes

### Dependencies

This task must run **after** all other 272 subtasks complete:
- 272.03 (cross-gem Ruby dependencies)
- 272.04 (nav protocols and Claude skills)
- 272.05 (docs, CI, and miscellaneous)

### Approach

This is primarily a verification task, not a scoped rename task. The expectation is that subtasks 01–05 handled the vast majority of changes. This sweep catches edge cases and provides confidence that the rename is complete.
