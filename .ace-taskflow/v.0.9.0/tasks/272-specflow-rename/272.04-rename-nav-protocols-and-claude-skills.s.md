# Rename nav protocols and Claude skills

## Behavioral Specification

### User Experience

After this change, nav protocols reference ace-specflow, Claude skills invoke `ace-specflow` commands, and workflow instructions use the new naming. Protocol URLs like `spec://121` work (replacing `task://121`).

### Expected Behavior

The nav protocol layer and Claude skill definitions are updated to reflect the specflow rename. This includes protocol source files, skill SKILL.md definitions, and workflow instruction files.

### Success Criteria

- [ ] 4 nav protocol source files renamed: `ace-taskflow.yml` → `ace-specflow.yml` in `.ace/nav/protocols/*/`
- [ ] Same 4 files renamed in `.ace-defaults/nav/protocols/*/`
- [ ] `task.yml` protocol → `spec.yml` with updated command template
- [ ] 34 Claude skill SKILL.md files updated: `source: ace-taskflow` → `source: ace-specflow`
- [ ] Skill permission patterns updated: `Bash(ace-taskflow:*)` → `Bash(ace-specflow:*)`
- [ ] Skill directories with "task" in name renamed to spec equivalents
- [ ] ~15 workflow instruction `.wf.md` files updated
- [ ] All protocol resolution works with new names

## Objective

Update all nav protocol definitions, Claude skill configurations, and workflow instruction files from taskflow/task naming to specflow/spec naming.

## Scope of Work

### Nav Protocol Files (rename + content)
- `.ace/nav/protocols/*/ace-taskflow.yml` → `ace-specflow.yml` (4 source files)
- `.ace-defaults/nav/protocols/*/ace-taskflow.yml` → `ace-specflow.yml` (4 files in gem defaults)
- `task.yml` protocol → `spec.yml` with `command_template: "ace-specflow spec %{ref}"`
- Update all internal references within protocol YAML files

### Claude Skill Definitions (34 SKILL.md files)
- `source: ace-taskflow` → `source: ace-specflow`
- `Bash(ace-taskflow:*)` → `Bash(ace-specflow:*)` permission patterns
- Any skill content referencing `ace-taskflow` commands or paths

### Skill Directory Renames
- Directories with "task" in the name that specifically reference taskflow concepts:
  - e.g., `ace_task-finder/` → `ace_spec-finder/`
  - e.g., `ace_manage-task-status/` → `ace_manage-spec-status/`
- Evaluate each on case-by-case basis — some "task" references may be generic

### Workflow Instruction Files (~15 `.wf.md` files)
- Update references to `ace-taskflow` commands across gems
- Update references to `.ace-taskflow/` paths
- Update references to `task` subcommand → `spec`

### Deliverables

#### Rename
- 8 nav protocol YAML files (4 project + 4 defaults)
- Skill directories with taskflow-specific "task" names

#### Modify
- 34 Claude skill SKILL.md files
- ~15 workflow instruction files
- Protocol YAML content (command templates, paths)

## Implementation Notes

### Skill Directory Rename Judgment
Not all skill directories with "task" should be renamed. Only rename those that specifically refer to ace-taskflow's concept of a task/spec:
- `ace_task-finder` → `ace_spec-finder` (yes — finds taskflow tasks)
- `ace_manage-task-status` → `ace_manage-spec-status` (yes — manages taskflow status)
- `ace_work-on-task` → evaluate whether "task" here means a taskflow task or a generic work item

### Protocol Resolution
After renaming `task.yml` → `spec.yml`, verify that `ace-nav spec://121` resolves correctly. The old `task://` protocol should either be removed or kept as a deprecated alias.
