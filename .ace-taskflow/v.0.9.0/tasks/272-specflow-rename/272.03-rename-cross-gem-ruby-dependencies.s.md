# Rename cross-gem Ruby dependencies

## Behavioral Specification

### User Experience

After this change, all gems that depend on ace-taskflow now depend on ace-specflow. All Ruby `require` statements, class references, method names, and subprocess calls use the new specflow/spec naming.

### Expected Behavior

Three gems have direct dependencies on ace-taskflow: ace-git-worktree, ace-review, and ace-prompt-prep. Additionally, several gems invoke `ace-taskflow` as a subprocess via Open3/system/backtick calls. All of these must be updated.

### Success Criteria

- [ ] 3 gemspec dependency updates (ace-git-worktree, ace-review, ace-prompt-prep)
- [ ] All `require "ace/taskflow"` → `require "ace/specflow"` (7 files)
- [ ] All Ruby class references updated (e.g., `Ace::Taskflow::Organisms::TaskManager` → `Ace::Specflow::Organisms::SpecManager`)
- [ ] CLI subprocess calls updated: `"ace-taskflow"` → `"ace-specflow"` (~20 source files)
- [ ] Method names updated: `ace_taskflow_available?` → `ace_specflow_available?`, `execute_ace_taskflow_command` → `execute_ace_specflow_command`
- [ ] ace-assign preset references updated: `ace-taskflow task done` → `ace-specflow spec done`
- [ ] All affected test files updated
- [ ] Full test suite passes across all affected gems

## Objective

Update all cross-gem Ruby dependencies, require paths, class references, subprocess invocations, and method names from taskflow/task to specflow/spec.

## Scope of Work

### Gemspec Dependencies
- `ace-git-worktree/ace-git-worktree.gemspec`: `add_dependency "ace-taskflow"` → `"ace-specflow"`
- `ace-review/ace-review.gemspec`: same
- `ace-prompt-prep/ace-prompt-prep.gemspec`: same

### Ruby Require Paths (7 files)
- All `require "ace/taskflow"` → `require "ace/specflow"`
- All `require "ace/taskflow/..."` → `require "ace/specflow/..."`

### Class References
- `Ace::Taskflow::Organisms::TaskManager` → `Ace::Specflow::Organisms::SpecManager`
- `Ace::Taskflow::Molecules::TaskLoader` → `Ace::Specflow::Molecules::SpecLoader`
- `Ace::Taskflow::Atoms::TaskReferenceParser` → `Ace::Specflow::Atoms::SpecReferenceParser`
- All other `Ace::Taskflow::*` references

### Subprocess Calls (~20 source files)
- `Open3.capture2("ace-taskflow", ...)` → `Open3.capture2("ace-specflow", ...)`
- `system("ace-taskflow", ...)` → `system("ace-specflow", ...)`
- Backtick invocations: `` `ace-taskflow ...` `` → `` `ace-specflow ...` ``
- String interpolation: `"ace-taskflow task #{ref}"` → `"ace-specflow spec #{ref}"`

### Method Names
- `ace_taskflow_available?` → `ace_specflow_available?`
- `execute_ace_taskflow_command` → `execute_ace_specflow_command`
- Any other methods with `taskflow` or `task` in the name that reference this gem

### ace-assign Preset References
- Phase definitions referencing `ace-taskflow task done` → `ace-specflow spec done`
- Catalog entries with taskflow subprocess calls

### Test Files
- All test files in affected gems that reference taskflow classes, commands, or methods

### Deliverables

#### Modify
- 3 gemspec files
- ~7 Ruby require files
- ~20 source files with subprocess calls
- Method definitions and call sites
- ace-assign catalog/preset files
- All corresponding test files

## Implementation Notes

### Search Strategy
Use `grep -r "ace-taskflow\|ace_taskflow\|Ace::Taskflow\|taskflow" --include="*.rb" --include="*.gemspec"` across the monorepo to find all references. Cross-reference with `grep -r "ace-taskflow" --include="*.yml" --include="*.yaml"` for config files.

### Ordering
This subtask depends on 272.01 (gem core) and 272.02 (data directory) because the gem name and paths must exist before other gems can reference them.
