# Rename gem core (ace-taskflow → ace-specflow)

## Behavioral Specification

### User Experience

After this change, the gem is named `ace-specflow`. The executable is `ace-specflow`. CLI subcommands change from `task`/`tasks` to `spec`/`specs`, with deprecated aliases for the old names that print a warning.

### Expected Behavior

The core gem directory, module namespace, class names, and CLI entry points all change from taskflow/task to specflow/spec. This is the foundation that all other subtasks build on.

### Success Criteria

- [ ] Directory renamed: `ace-taskflow/` → `ace-specflow/`
- [ ] Gemspec renamed: `ace-taskflow.gemspec` → `ace-specflow.gemspec`
- [ ] Module renamed: `Ace::Taskflow` → `Ace::Specflow` throughout
- [ ] All files under `lib/ace/taskflow/` → `lib/ace/specflow/`
- [ ] Executable renamed: `exe/ace-taskflow` → `exe/ace-specflow`
- [ ] CLI subcommands: `spec`/`specs` primary, `task`/`tasks` deprecated aliases
- [ ] All internal class renames completed (see scope)
- [ ] Config key: `config["specflow"]`
- [ ] Test base class: `AceSpecflowTestCase`
- [ ] All tests pass within the gem

## Objective

Rename the ace-taskflow gem's core identity — directory, module, classes, executable, and CLI subcommands.

## Scope of Work

### Directory and File Renames
- `ace-taskflow/` → `ace-specflow/`
- `ace-taskflow/ace-taskflow.gemspec` → `ace-specflow/ace-specflow.gemspec`
- `ace-taskflow/lib/ace/taskflow.rb` → `ace-specflow/lib/ace/specflow.rb`
- `ace-taskflow/lib/ace/taskflow/` → `ace-specflow/lib/ace/specflow/` (all files)
- `ace-taskflow/exe/ace-taskflow` → `ace-specflow/exe/ace-specflow`

### Module and Class Renames
- `Ace::Taskflow` → `Ace::Specflow`
- `TaskManager` → `SpecManager`
- `TaskLoader` → `SpecLoader`
- `TaskReferenceParser` → `SpecReferenceParser`
- `TaskFilter` → `SpecFilter`
- `TaskflowDoctor` → `SpecflowDoctor`
- `TaskflowContextLoader` → `SpecflowContextLoader`
- `AceTaskflowTestCase` → `AceSpecflowTestCase`

### CLI Changes
- Primary subcommands: `spec` (show), `specs` (list)
- Deprecated aliases: `task` → `spec`, `tasks` → `specs` (with deprecation warning)
- Config key in YAML: `taskflow:` → `specflow:`

### Test Files
- All test files under `ace-specflow/test/` updated for new class names and paths
- Test base class renamed

### Deliverables

#### Rename
- Entire `ace-taskflow/` directory → `ace-specflow/`
- All internal file paths accordingly

#### Modify
- Every Ruby source file in the gem (module/class references)
- Every test file in the gem
- Gemspec metadata (name, require paths, executables)

## Implementation Notes

### Approach
Use a combination of `git mv` for directory/file renames and find-and-replace for content changes. Process in order: directories first, then file renames, then content updates.

### CLI Deprecation Pattern
```ruby
desc "task REF", "Show spec details (DEPRECATED: use 'spec')"
def task(ref)
  warn "[DEPRECATED] 'ace-specflow task' is deprecated, use 'ace-specflow spec'"
  spec(ref)
end
```
