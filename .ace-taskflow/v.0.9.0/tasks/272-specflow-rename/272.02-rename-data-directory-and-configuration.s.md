# Rename data directory and configuration

## Behavioral Specification

### User Experience

After this change, the project data directory is `.ace-specflow/` instead of `.ace-taskflow/`. Configuration lives at `.ace/specflow/config.yml`. Inside release directories, the `tasks/` subdirectory becomes `specs/`.

### Expected Behavior

All configuration and data paths transition from taskflow naming to specflow naming. A migration script handles the rename for existing projects.

### Success Criteria

- [ ] `.ace-taskflow/` → `.ace-specflow/` (project data directory)
- [ ] `.ace/taskflow/config.yml` → `.ace/specflow/config.yml`
- [ ] YAML key `taskflow:` → `specflow:` in config files
- [ ] `*/tasks/` → `*/specs/` inside release directories
- [ ] Default config: `root: ".ace-specflow"`
- [ ] `ACE_TASKFLOW_PATH` → `ACE_SPECFLOW_PATH` in mise.toml files
- [ ] `.ace-defaults/taskflow/config.yml` → `.ace-defaults/specflow/config.yml`
- [ ] Migration script handles rename idempotently

## Objective

Rename all data directories and configuration paths from taskflow to specflow, including the project-level `.ace-taskflow/` directory and all configuration files.

## Scope of Work

### Data Directory Rename
- `.ace-taskflow/` → `.ace-specflow/` (entire project data tree)
- Inside each release: `tasks/` → `specs/` (e.g., `v.0.9.0/tasks/` → `v.0.9.0/specs/`)
- `retros/`, `ideas/`, `docs/` subdirectories: unchanged

### Configuration File Renames
- `.ace/taskflow/config.yml` → `.ace/specflow/config.yml`
- `.ace-defaults/taskflow/config.yml` → `.ace-defaults/specflow/config.yml` (inside gem)
- YAML key: `taskflow:` → `specflow:` in all config files

### Environment Variables
- `ACE_TASKFLOW_PATH` → `ACE_SPECFLOW_PATH` in 3 `mise.toml` files

### Default Configuration Updates
- `root: ".ace-taskflow"` → `root: ".ace-specflow"` in config loader defaults
- `config["taskflow"]` → `config["specflow"]` in configuration.rb

### Migration Script
- Rename `.ace-taskflow/` → `.ace-specflow/` in project root
- Rename `tasks/` → `specs/` within each release directory
- Rename `.ace/taskflow/` → `.ace/specflow/`
- Must be idempotent (safe to run multiple times)
- Should handle both fresh installs and existing projects

### Deliverables

#### Rename
- `.ace-taskflow/` → `.ace-specflow/` (entire tree)
- `.ace/taskflow/` → `.ace/specflow/`
- `.ace-defaults/taskflow/` → `.ace-defaults/specflow/` (inside gem)

#### Modify
- Configuration loader defaults
- mise.toml files (3)
- All YAML config files with `taskflow:` key

#### Create
- Migration script (or extend the one from subtask 01)

## Implementation Notes

### Ordering
This subtask can run in parallel with 272.01 (gem core rename). The data directory rename is independent of the code rename, though both must complete before cross-gem updates (272.03).

### Risk
The `.ace-taskflow/` directory contains all task specs, retros, and ideas. The rename must preserve git history. Use `git mv` to maintain traceability.
