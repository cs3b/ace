# Rename docs, CI, and miscellaneous

## Behavioral Specification

### User Experience

After this change, all documentation, CI configuration, and miscellaneous project files reference ace-specflow instead of ace-taskflow. The rename is complete and consistent across the entire repository.

### Expected Behavior

This subtask covers the "long tail" of references that don't fit neatly into the gem core, data directory, cross-gem, or skill categories. It ensures no stale ace-taskflow references remain anywhere in the repository.

### Success Criteria

- [ ] `Gemfile` + `Gemfile.lock` updated (gem name and path)
- [ ] `Rakefile` package list updated
- [ ] `.github/workflows/test.yml` matrix entry updated
- [ ] `.simplecov` group name updated
- [ ] `.gitignore` path patterns updated
- [ ] `bin/ace-taskflow` shim → `bin/ace-specflow`
- [ ] `.ace/git/commit.yml` glob patterns updated
- [ ] `.ace/test/suite.yml` package name and path updated
- [ ] `.ace/review/config.yml` path references updated
- [ ] ~10 docs/*.md files updated (architecture, tools, blueprint)
- [ ] `README.md`, `CLAUDE.md`, `DEVELOPMENT.md` updated
- [ ] ADR files referencing ace-taskflow updated
- [ ] Other gem READMEs updated
- [ ] E2E test fixtures (~5 files) updated
- [ ] CHANGELOG entries note the rename
- [ ] `grep -r "ace-taskflow"` returns zero hits across repo

## Objective

Update all remaining references to ace-taskflow across documentation, CI configuration, project tooling config, and miscellaneous files to complete the specflow rename.

## Scope of Work

### Build and Dependency Files
- `Gemfile`: gem name `"ace-taskflow"` → `"ace-specflow"`, path reference
- `Gemfile.lock`: regenerate after Gemfile update
- `Rakefile`: package list entry

### CI Configuration
- `.github/workflows/test.yml`: matrix entry for the gem package
- Any other CI files referencing the gem name or path

### Project Tooling Config
- `.simplecov`: group name for coverage reporting
- `.gitignore`: any path patterns referencing `.ace-taskflow`
- `bin/ace-taskflow` → `bin/ace-specflow` (shim/binstub)
- `.ace/git/commit.yml`: glob patterns for commit message context
- `.ace/test/suite.yml`: package name and path
- `.ace/review/config.yml`: path references

### Documentation (~10+ files)
- `docs/architecture.md` (or similar)
- `docs/tools.md`
- `docs/blueprint.md` (or similar)
- Other docs referencing ace-taskflow
- `README.md` (project root)
- `CLAUDE.md` (project root)
- `DEVELOPMENT.md`
- ADR files (`docs/adrs/` or similar) referencing ace-taskflow
- Other gem `README.md` files that mention ace-taskflow

### E2E Test Fixtures
- ~5 fixture files that reference ace-taskflow paths or commands
- Test helper files with hardcoded paths

### CHANGELOG
- Add entry noting the rename from ace-taskflow to ace-specflow
- Do not rewrite history of previous entries

#### Modify
- `Gemfile`, `Rakefile`
