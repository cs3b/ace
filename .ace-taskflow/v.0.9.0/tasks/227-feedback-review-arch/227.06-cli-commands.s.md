# CLI Commands for Feedback Management

## Behavioral Specification

### User Experience
- **Input**: CLI commands like `ace-review feedback list --status draft`
- **Process**: System queries/updates feedback items via FeedbackManager
- **Output**: Formatted terminal output or status messages

### Expected Behavior

New `feedback` subcommand group for ace-review CLI:

```bash
# List feedback items
ace-review feedback list [--status STATUS] [--priority PRIORITY] [--archived]

# Show single item
ace-review feedback show ID

# Verify item (mark as valid/invalid)
ace-review feedback verify ID --valid|--invalid [--research "..."]

# Skip item (not applicable now)
ace-review feedback skip ID [--reason "..."]

# Resolve item (mark as done)
ace-review feedback resolve ID --resolution "..."
```

Output formats should be:
- `list`: Table with id, title (truncated), status, priority, files count
- `show`: Full item details including all sections
- `verify/skip/resolve`: Success message with updated status

### Interface Contract

```ruby
# FeedbackListCommand
class FeedbackListCommand
  def execute(options)
    # options: status, priority, archived
    # Output: formatted table
  end
end

# FeedbackShowCommand
class FeedbackShowCommand
  def execute(id)
    # Output: full item details
  end
end

# FeedbackVerifyCommand
class FeedbackVerifyCommand
  def execute(id, valid:, research: nil)
    # Output: success/failure message
  end
end

# FeedbackSkipCommand
class FeedbackSkipCommand
  def execute(id, reason: nil)
    # Output: success/failure message
  end
end

# FeedbackResolveCommand
class FeedbackResolveCommand
  def execute(id, resolution:)
    # Output: success/failure message
  end
end
```

**CLI Registration:**
```ruby
# In cli.rb
register :feedback do |cmd|
  cmd.subcommand :list, FeedbackListCommand
  cmd.subcommand :show, FeedbackShowCommand
  cmd.subcommand :verify, FeedbackVerifyCommand
  cmd.subcommand :skip, FeedbackSkipCommand
  cmd.subcommand :resolve, FeedbackResolveCommand
end
```

**Error Handling:**
- Item not found: Print error, exit 1
- Invalid options: Print usage, exit 1
- Manager error: Print error message, exit 1

**Edge Cases:**
- Empty list: Print "No feedback items found"
- ID prefix match: Support partial ID matching (first 3+ chars)
