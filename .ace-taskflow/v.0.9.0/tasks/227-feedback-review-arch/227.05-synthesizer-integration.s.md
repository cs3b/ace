# Synthesizer Integration

## Behavioral Specification

### User Experience
- **Input**: Review completes with `--feedback` (default) or `--no-feedback` flag
- **Process**: After synthesis, system extracts feedback items if enabled
- **Output**: Feedback files created alongside synthesis report

### Expected Behavior

Integrate FeedbackManager with the existing review workflow:

1. **Default behavior**: Reviews extract feedback after synthesis
2. **Opt-out**: `--no-feedback` flag disables extraction
3. **Backward compatible**: Synthesis report still generated as before
4. **Additive**: Feedback files complement, don't replace synthesis

The integration points are:
- ReviewManager (Organism) orchestrates both ReportSynthesizer and FeedbackManager
- ReportSynthesizer generates synthesis report only (no FeedbackManager dependency)
- ReviewManager calls FeedbackManager.extract_and_save after synthesis completes
- ReviewOptions model gains `feedback` boolean option

**Note**: This preserves ATOM hierarchy - Organisms coordinate Molecules, not vice versa.

### Interface Contract

```ruby
# Updated ReviewOptions
class ReviewOptions
  attr_accessor :feedback  # default: true
end

# ReportSynthesizer (Molecule) - NO FeedbackManager dependency
class ReportSynthesizer
  # No changes to initialize - no feedback_manager injection

  def synthesize(reports, options: {})
    # Existing synthesis logic only
    generate_synthesis(reports)
  end
end

# Updated ReviewManager (Organism) - Orchestrates both
class ReviewManager
  def initialize(..., feedback_manager: nil)
    # ... existing initialization ...
    @feedback_manager = feedback_manager
  end

  def execute_multi_model(options)
    # ... existing code ...

    # Step 1: Generate synthesis (Molecule call)
    synthesis_result = synthesizer.synthesize(model_results, options: options)

    # Step 2: Extract feedback if enabled (Organism call)
    if options.feedback && @feedback_manager
      feedback_result = @feedback_manager.extract_and_save(model_results)
      synthesis_result[:feedback] = feedback_result
    end

    synthesis_result
  end
end
```

**Architecture Note**: FeedbackManager (Organism) is coordinated by ReviewManager (Organism), not injected into ReportSynthesizer (Molecule). This preserves the ATOM hierarchy where Molecules have no Organism dependencies.

**CLI Changes:**
```bash
# Default (feedback enabled)
ace-review --preset code-pr

# Feedback disabled
ace-review --preset code-pr --no-feedback
```

**Error Handling:**
- Extraction failure: Log warning, don't fail overall review
- No FeedbackManager: Skip extraction silently (backward compat)

**Edge Cases:**
- Single-model review: Still extracts feedback from single report
- Empty synthesis: No findings to extract, return empty
