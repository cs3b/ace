# Task Integration

## Behavioral Specification

### User Experience
- **Input**: Review run with `--task` flag or auto-save from feature branch
- **Process**: Feedback saved to task directory alongside reviews
- **Output**: `{task_dir}/feedback/` contains feedback files

### Expected Behavior

When reviews are associated with a task, feedback files should be saved to the task's feedback directory:

```
{task_dir}/
├── feedback/
│   ├── 8o7abc-security-sql-injection.s.md
│   ├── 8o7abd-perf-n-plus-one-query.s.md
│   └── _archived/
│       └── 8o7abf-resolved-item.s.md
└── reviews/
    ├── 8o7ab0-gemini-code-pr-review.md
    └── 8o7ab2-synthesis.md
```

The integration modifies TaskReportSaver to:
1. Create feedback directory structure
2. Save feedback files from extraction
3. Support archiving within task directory

### Interface Contract

```ruby
# Updated TaskReportSaver

  # Existing methods unchanged...

  # New: Save feedback to task directory
  def save_feedback(feedback_items)
    # => { success: true, paths: [...] }

  def archive_feedback(feedback_id)
    # => { success: true, path: "feedback/_archived/..." }
end

# Updated ReviewManager integration
class ReviewManager
  def execute_with_task(options)
    # ... existing flow ...

    # After synthesis with feedback extraction
    if options.task && synthesis_result[:feedback]
      task_saver.save_feedback(synthesis_result[:feedback][:items])
    end
  end
end
```

**Directory Structure:**
```
.ace-taskflow/v.X.X/tasks/{task-id}/
├── feedback/
│   ├── {id}-{slug}.s.md
│   └── _archived/
└── reviews/
    └── ...
```

**Error Handling:**
- Task directory not found: Log warning, skip save
- Feedback write failure: Log error, continue with others
- Archive failure: Log error, keep file in place

**Edge Cases:**
- No feedback items: Don't create empty directory
- Mixed save (some succeed, some fail): Return partial result
- Task without reviews dir: Create both feedback/ and reviews/

### Success Criteria

- [ ] Feedback files saved to `{task_dir}/feedback/`
- [ ] Archived feedback moved to `{task_dir}/feedback/_archived/`
- [ ] Directory created only when feedback exists
- [ ] Works with both --task flag and auto-save
- [ ] Partial failures handled gracefully
