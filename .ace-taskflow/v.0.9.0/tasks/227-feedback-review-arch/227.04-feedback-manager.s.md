# Feedback Manager (Organism)

## Behavioral Specification

### User Experience
- **Input**: Commands to extract, list, verify, resolve feedback items
- **Process**: Coordinates extractor, file I/O, and state transitions
- **Output**: Feedback items with updated status, files archived when appropriate

### Expected Behavior

The FeedbackManager organism orchestrates the entire feedback lifecycle:

1. **Extract and Save**: Run extraction on reports, save as files
2. **List**: Return filtered list of feedback items
3. **Transition**: Move items between states with validation
4. **Verify**: Mark as pending/invalid/skip with research notes
5. **Resolve**: Mark as done with resolution notes, archive

Status transitions follow the workflow:
```
draft -> pending (verified as valid)
draft -> invalid (false positive)
draft -> skip (not applicable now)
pending -> done (addressed/fixed)
```

Items in terminal states (invalid, skip, done) are automatically archived.

### Interface Contract

```ruby
# FeedbackManager organism
class FeedbackManager
  def initialize(base_dir:, llm_client: nil)

  # Extraction
  def extract_and_save(reports, reviewer: nil)
    # => { success: true, items: [...], count: 5 }

  # Listing
  def list(status: nil, priority: nil, include_archived: false)
    # => Array<FeedbackItem>

  def find(id)
    # => FeedbackItem or nil

  # State transitions
  def verify(id, valid:, research: nil)
    # valid: true -> pending, false -> invalid
    # => { success: true, item: FeedbackItem }

  def skip(id, reason: nil)
    # => { success: true, item: FeedbackItem }

  def resolve(id, resolution:)
    # => { success: true, item: FeedbackItem }

  # Statistics
  def stats
    # => { total: 10, draft: 3, pending: 5, done: 2 }
end
```

**Error Handling:**
- Item not found: Return `{ success: false, error: "Item not found" }`
- Invalid transition: Return `{ success: false, error: "Cannot transition from X to Y" }`
- File I/O error: Return `{ success: false, error: "..." }`

**Edge Cases:**
- Verify already-verified item: No-op if same state, error if different
- Resolve non-pending item: Error with message
- Archive failure: Keep item in place, log error
