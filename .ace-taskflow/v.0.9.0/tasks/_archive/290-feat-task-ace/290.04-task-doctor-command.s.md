# ace-task Doctor Command

## Behavioral Specification

### User Experience
- **Input**: CLI command `ace-task doctor` with optional flags for scoping, auto-fix, and output format
- **Process**: Scans all tasks, validates structure/frontmatter/scope consistency, optionally auto-fixes safe issues
- **Output**: Health report with score, categorized issues, and fix results

### Expected Behavior

Provide a self-sufficient health check system for the ace-task gem, since ace-taskflow will be deleted. This mirrors `ace-idea doctor` exactly in architecture:

1. **Structure validation**: Folder naming matches `{id}-{slug}` convention, spec file naming correct, subtask files placed in parent folder (not standalone folders), no stale backups, no empty directories.
2. **Frontmatter validation**: Required fields present (`id`, `status`, `title`), ID format valid (`xxx.t.yyy`), status enum valid, parent/subtask consistency (subtask file has `parent:` field matching parent ID, parent file lists subtask in `subtasks:` array).
3. **Scope/status consistency**: Terminal-status tasks belong in `_archive/`, non-terminal tasks not in `_archive/`, `_maybe/` tasks not terminal.
4. **Auto-fix**: Safe issues fixed automatically (missing `created_at`, wrong archive status, stale backups, empty dirs). Complex issues reported for manual or agent-assisted fix.
5. **Agent-assisted fix**: Remaining issues passed to an LLM agent session for interactive repair.

### Interface Contract

```bash
# Run all health checks
ace-task doctor

# Auto-fix safe issues
ace-task doctor --auto-fix
ace-task doctor --auto-fix --dry-run        # Preview fixes without applying

# Agent-assisted fix for complex issues
ace-task doctor --auto-fix-with-agent
ace-task doctor --auto-fix-with-agent --model gemini:flash-latest

# Scoped checks
ace-task doctor --check structure
ace-task doctor --check frontmatter
ace-task doctor --check scope

# Output formats
ace-task doctor --json                      # JSON for tool integration
ace-task doctor --verbose                   # Show all warnings
ace-task doctor --errors-only               # Suppress warnings
ace-task doctor --quiet                     # Exit code only (0=healthy, 1=issues)
```

**Exit Codes:**
- 0: All checks pass
- 1: Issues found (via `Ace::Core::CLI::Error`)

### Success Criteria

- [ ] `TaskValidationRules` atom defines valid statuses (`pending`, `in-progress`, `done`, `blocked`, `draft`, `skipped`, `cancelled`), terminal statuses, required fields, scope consistency checks, and parent/subtask consistency checks
- [ ] `TaskStructureValidator` molecule checks folder naming for split ID format (`xxx.t.yyy-slug/`), spec file naming, subtask file placement in parent folder, stale backups, empty directories

- [ ] `TaskDoctorFixer` molecule auto-fixes safe issues: missing `created_at`, wrong archive status, stale backups, empty dirs, missing `status` field

- [ ] All numeric CLI options add `.to_i`/`.to_f` in consuming layer (dry-cli does NOT coerce — values arrive as strings)
- [ ] Full test coverage at atoms/molecules/organisms layers
- [ ] `ace-test ace-task` passes all tests

### Task-specific validation rules (vs idea)

| Rule | Ideas | Tasks |
|------|-------|-------|
| Valid statuses | pending, in-progress, done, obsolete | pending, in-progress, done, blocked, draft, skipped, cancelled |
| Terminal statuses | done, obsolete | done, skipped, cancelled |
| Required fields | id, status, title | id, status, title |
| ID format | 6-char raw (`8ppq7w`) | 10-char split (`8pp.t.q7w`) |
| Subtask validation | N/A | parent field, subtask char allocation, folder co-location |
| Folder naming | `{6char}-{slug}/` | `{xxx.t.yyy}-{slug}/` |
