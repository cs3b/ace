# Phase 3: markdown-first artifact chain finalization (unplanned)

## Behavioral Context

**Issue**: Phase 2 (`296.02`) completed with a YAML-oriented chain contract (`input.yml/output.yml`) that did not match the final intended runtime model.

**Key Behavioral Requirements**:
- `ace-sim run --source` must be a readable file path.
- Runtime chain must be markdown-first: `input.md -> user.bundle.md -> user.prompt.md -> output.md`.
- Next-step handoff must use previous `output.md` as the next `input.md`.
- `session.yml` and `synthesis.yml` remain top-level metadata outputs.

## Objective

Finalize `ace-sim` runtime and package defaults to a markdown-first artifact chain and document the completed behavior as task-296 lineage.

## Scope of Work

- Reworked runtime execution from YAML step artifacts to markdown-first chain artifacts.
- Enforced strict file-path source handling in CLI/runtime for step-1 input copy.
- Replaced default step templates with sectioned, prompt-ready bundle configs.
- Updated tests, docs, and E2E scenario assets to the final artifact contract.
- Released `ace-sim` through `v0.2.0` and `v0.3.0` to capture final-state behavior.
