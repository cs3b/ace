# Inventory current naming surfaces and blast radius

## Behavioral Specification

### User Experience
- **Input**: Current repository naming terms (`ACE`, `Agentic Coding Environment`, `ADE`) and candidate slugs (`ace`, `ace-meta`, `agentic-coding-environment`)
- **Process**: Author receives a concrete inventory of where names appear and what each change would affect
- **Output**: A complete map of update surfaces and risk notes for each surface

### Expected Behavior
The subtask produces a traceable inventory covering repository metadata, root docs, per-gem READMEs, gemspec summaries/descriptions,
workflow/skill references, badges, and links. Each surface is labeled as required update, optional update, or do-not-change.

### Interface Contract
```bash
# Discovery commands expected during implementation phase
rg -n "Agentic Coding Environment|Agentic Development Environment|ACE|ace-meta"
rg --files -g "*.gemspec"
```

### Success Criteria
- [ ] Inventory includes GitHub metadata surfaces and local repository files
- [ ] Every update candidate is tagged with ownership and risk level
- [ ] No implementation changes are proposed in this phase

## Scope

- Root-level docs and metadata files
- Per-package `README.md` and `*.gemspec` files
- Taskflow/workflow references that include repository naming

## Deliverables

- [ ] Naming surface inventory table
- [ ] Blast-radius summary with priority labels (critical/high/medium/low)
- [ ] Recommended sequencing constraints for later subtasks
