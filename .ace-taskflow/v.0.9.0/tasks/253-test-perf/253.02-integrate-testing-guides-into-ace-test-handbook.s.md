# Integrate Testing Guides into ace-test Handbook

## Behavioral Specification

### User Experience
- **Input**: Agent or developer needs testing guidance via `ace-bundle guide://test-*` protocols
- **Process**: Guides are discoverable through ace-nav and loadable via ace-bundle
- **Output**: Complete testing guidance available inline without leaving workflow

### Expected Behavior

When a developer or AI agent needs testing guidance, they can:
1. Discover available guides via `ace-nav guide://*test*`
2. Load any guide via `ace-bundle guide://test-layer-decision`
3. Reference guides from workflows using standard `guide://` protocol

The 6 new guides integrate seamlessly with existing ace-test documentation.

### Interface Contract

```bash
# Discovery
ace-nav guide://*test*
# Returns: test-layer-decision, test-mocking-patterns, test-suite-health,
#          test-responsibility-map, test-review-checklist, testing-strategy

# Loading
ace-bundle guide://test-layer-decision
# Returns: Full guide content for inline use

# Cross-reference in workflows
ace-bundle wfi://plan-tests
# Guide references within workflow resolve correctly
```

### Success Criteria

- [ ] 6 guides moved to `ace-test/handbook/guides/`
- [ ] All guides follow ACE guide conventions (frontmatter, structure)
- [ ] Guides discoverable via `ace-nav guide://*test*`
- [ ] Protocol references work: `guide://test-layer-decision`
- [ ] Cross-references between guides resolve correctly
- [ ] SUMMARY.md navigation index updated
