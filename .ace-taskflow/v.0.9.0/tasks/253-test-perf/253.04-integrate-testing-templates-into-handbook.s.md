# Integrate Testing Templates into ace-test Handbook

## Behavioral Specification

### User Experience
- **Input**: Agent needs structured template via `ace-bundle tmpl://test-responsibility-map`
- **Process**: Template loads with placeholder sections ready for population
- **Output**: Filled template as structured artifact (audit report, checklist, etc.)

### Expected Behavior

When an agent needs a testing template:
1. Discover templates via `ace-nav tmpl://*test*`
2. Load template via `ace-bundle tmpl://test-performance-audit`
3. Populate placeholders with context-specific content
4. Save as structured artifact

Templates enable consistent documentation across test audits and reviews.

### Interface Contract

```bash
# Discovery
ace-nav tmpl://*test*
# Returns: test-responsibility-map, test-performance-audit,
#          test-review-checklist, e2e-sandbox-checklist

# Loading
ace-bundle tmpl://test-performance-audit
# Returns: Template with placeholders for population

# Output examples
# - .ace-taskflow/retro/test-audit-2026-01.md
# - Test PR review checklist in PR description
```

### Success Criteria

- [ ] 4 templates moved to `ace-test/handbook/templates/`
- [ ] All templates follow `.template.md` naming convention
- [ ] Templates discoverable via `ace-nav tmpl://*test*`
- [ ] Placeholder format consistent with ACE conventions
- [ ] Templates embedded in corresponding workflows (ADR-002)

## Objective

Provide reusable templates for test documentation, ensuring consistent structure across performance audits, responsibility maps, and review checklists.
