# ace-e2e-test Command

## Objective

Add `ace-e2e-test` CLI command to ace-test-e2e-runner for executing single E2E tests or all tests in a package via LLM.

## Behavioral Specification

### User Experience

- **Input**: Package name and optional test ID
- **Process**:
  1. Discover *.mt.md files in `{package}/test/e2e/`
  2. Build prompt with system instructions + test scenario
  3. Execute via `Ace::LLM::QueryInterface.query()` with cli_args
  4. Parse JSON results from LLM response
  5. Write reports to `.cache/ace-test-e2e/{timestamp}/`
- **Output**: Structured summary with pass/fail status

### Interface Contract

```bash
# Run specific test
ace-e2e-test ace-lint MT-LINT-001
# Output:
# Running E2E test: MT-LINT-001 (ace-lint)
# Executing via google:gemini-2.5-flash...
# Result: PASS
# Report: .cache/ace-test-e2e/8xyz12/MT-LINT-001/summary.r.md

# Run all tests in package
ace-e2e-test ace-lint
# Output:
# Discovered 5 E2E tests in ace-lint
# [1/5] MT-LINT-001: PASS (45s)
# [2/5] MT-LINT-002: PASS (32s)
# ...
# Summary: 5/5 passed

# With CLI args for provider
ace-e2e-test ace-coworker MT-COWORKER-001 --cli-args dangerously-skip-permissions
```

### Success Criteria

- [ ] Test discovery finds all *.mt.md files
- [ ] Prompt builder creates valid LLM prompts
- [ ] LLM execution returns structured results
- [ ] `--cli-args` passed through to LLM provider
- [ ] Reports written in standard format
- [ ] CLI shows progress and summary

## Report Path Contract

```

    experience.r.md
```

Use `ace-timestamp encode` for timestamp generation (6-char Base36).
