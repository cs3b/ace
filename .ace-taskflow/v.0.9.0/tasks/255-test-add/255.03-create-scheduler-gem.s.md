# Create ace-scheduler Gem Structure

## Objective

Create the gem skeleton for `ace-scheduler` following ACE gem patterns with ATOM architecture.

## Behavioral Specification

### User Experience
- **Input**: N/A (gem structure only)
- **Process**: Create gem with proper directory structure, gemspec, and base classes
- **Output**: Working gem that can be required and has version command

### Interface Contract

```bash
# After this task, these should work:
ace-scheduler --version
ace-scheduler --help
```

### Success Criteria

- [ ] Gem directory created at `ace-scheduler/`
- [ ] Gemspec with correct dependencies (ace-support-core, dry-cli, fugit)
- [ ] ATOM directory structure
- [ ] CLI registry with dry-cli
- [ ] Version command works
- [ ] Default config structure defined

### Deliverables


## Implementation Plan

### Execution Steps

- [ ] Create gem directory structure
  ```bash
  mkdir -p ace-scheduler/{exe,lib/ace/scheduler/cli/commands,test,.ace-defaults/scheduler}
  ```

- [ ] Create gemspec with dependencies
  ```ruby
  # ace-scheduler.gemspec
  spec.add_dependency "ace-support-core", ">= 0.1"
  spec.add_dependency "ace-support-config", ">= 0.1"
  spec.add_dependency "dry-cli", "~> 1.0"
  spec.add_dependency "fugit", "~> 1.8"  # Cron parsing
  ```
  > TEST: Gemspec Valid
  > Assert: Gem can be built
  > Command: cd ace-scheduler && gem build ace-scheduler.gemspec

- [ ] Create version.rb with initial version 0.1.0

- [ ] Create CLI registry with dry-cli

- [ ] Create executable file (ace-scheduler)

- [ ] Create bin/ wrapper script for monorepo

- [ ] Create default config files
  ```yaml
  # .ace-defaults/scheduler/config.yml
  version: 1
  tasks: {}
  events: {}
  state_dir: ".ace/scheduler/state"
  ```

  ```yaml
  # .ace/scheduler/config.yml (project default)
  tasks:
    daily-tests:
      description: "Run test suite with profiling"
      cron: "0 9 * * *"
      command: ace-test-suite --profile 10
      enabled: true

    weekly-e2e:
      description: "Full E2E test run"
      cron: "0 6 * * 1"
      command: ace-e2e-test-suite --all
      enabled: true

  events:
    pr-merged:
      description: "Post-merge verification"
      triggers:
        - command: ace-test-suite --changed
        - command: ace-e2e-test --affected

    release-started:
      description: "Pre-release gates"
      triggers:
        - command: ace-test-suite
        - command: ace-e2e-test-suite --all
  ```

- [ ] Create state directory structure
  ```
  .ace/scheduler/state/
  ├── .gitkeep
  └── .gitignore  # Ignore history/, *.lock
  ```

- [ ] Add to root Gemfile for development

- [ ] Verify gem loads and version command works
  > TEST: Gem Loads
  > Assert: Version command returns version
  > Command: ace-scheduler --version
