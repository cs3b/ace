# Implement E2E Test Execution and CLI

## Objective

Implement the core E2E test execution logic: test discovery, prompt building, LLM execution, result parsing, and CLI commands.

## Behavioral Specification

### User Experience
- **Input**: Package name and optional test ID
- **Process**:
  1. Discover *.mt.md files in `{package}/test/e2e/`
  2. Build prompt with system instructions + test scenario
  3. Execute via `Ace::LLM::QueryInterface.query()`
  4. Parse JSON results from LLM response
  5. Write reports to `.cache/ace-test-e2e/{timestamp}/`
- **Output**: Structured summary with pass/fail status

### Interface Contract

```bash
# Run specific test
ace-e2e-test ace-lint MT-LINT-001
# Output:
# Running E2E test: MT-LINT-001 (ace-lint)
# Executing via google:gemini-2.5-flash...
# Result: PASS
# Report: .cache/ace-test-e2e/8xyz12/MT-LINT-001/summary.r.md

# Run all tests in package
ace-e2e-test ace-lint
# Output:
# Discovered 5 E2E tests in ace-lint
# [1/5] MT-LINT-001: PASS (45s)
# [2/5] MT-LINT-002: PASS (32s)
# ...
# Summary: 5/5 passed

# Run full suite (sequential)
ace-e2e-test-suite
# Output:
# Discovered 31 E2E tests across 10 packages
# Running sequentially...
# ...

# Run full suite (parallel)
ace-e2e-test-suite --parallel 4
# Output:
# Running 4 tests in parallel...
```
