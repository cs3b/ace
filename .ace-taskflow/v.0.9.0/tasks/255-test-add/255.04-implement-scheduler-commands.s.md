# Implement Scheduler Commands

## Objective

Implement all ace-scheduler CLI commands: list, run, emit, status, and cron management.

## Behavioral Specification

### User Experience

**List Tasks:**
- **Input**: `ace-scheduler list`
- **Output**: LLM-friendly table of tasks and events

**Run Task:**
- **Input**: `ace-scheduler run daily-tests`
- **Output**: Executes command, shows result

**Emit Event:**
- **Input**: `ace-scheduler emit pr-merged`
- **Output**: Triggers all associated actions

**Status:**
- **Input**: `ace-scheduler status`
- **Output**: Next scheduled runs, last results

**Cron Management:**
- **Input**: `ace-scheduler cron install`
- **Output**: Installs crontab entries

- [ ] Implement CronInstaller molecule
  ```ruby
  # CRITICAL: Crontab operations must be atomic to prevent partial writes
  # on failure. Build complete crontab in memory before writing.
  def install
    entries = build_entries
    existing = `crontab -l 2>/dev/null`.lines

    # Remove old ace-scheduler entries
    filtered = existing.reject { |l| l.include?('ace-scheduler') }

    # Build complete new crontab BEFORE writing (atomic preparation)
    new_crontab = filtered + entries.map { |e| "#{e} # ace-scheduler\n" }
    new_content = new_crontab.join

    # Validate content before writing
    raise "Empty crontab content" if new_content.strip.empty? && !filtered.empty?

    # Write atomically via pipe to crontab
    result = IO.popen('crontab -', 'w', err: [:child, :out]) do |io|
      io.write(new_content)
    end

    unless $?.success?
      # Rollback: restore original crontab
      IO.popen('crontab -', 'w') { |io| io.write(existing.join) }
      raise "Failed to install crontab, rolled back to previous state"
    end
  end

  def uninstall
    existing = `crontab -l 2>/dev/null`.lines
    filtered = existing.reject { |l| l.include?('ace-scheduler') }
    IO.popen('crontab -', 'w') { |io| io.write(filtered.join) }
  end
  ```

## Concurrency Strategy

Task executions are serialized via exclusive file locking:

- StateManager acquires exclusive lock on `.ace/scheduler/state/.lock` before any state operation
- If a cron job triggers while another is running, it waits for the lock
- This prevents state corruption and ensures predictable execution order
- Lock timeout: 10 minutes (configurable via `config.yml`)

This is simpler than parallel execution with job queuing and sufficient for local scheduling needs.
