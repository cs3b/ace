# E2E Test Runner CLI and Local Scheduler

## Objective

Replace CI-focused task 253.06 (GitHub Actions performance gate) with local tooling that better fits our workflow:

1. **ace-e2e-test / ace-e2e-test-suite** - CLI tools for E2E test execution via LLM
2. **ace-scheduler** - Local cron and event-driven task automation with LLM-friendly output

This addresses the gap that E2E tests currently have no CLI runner (only agent-driven skills) and provides local scheduling for maintenance tasks instead of CI-based gates.

## Behavioral Specification

### User Experience

**E2E Test CLI:**
- **Input**: Package name and optional test ID
- **Process**: Discovers tests, builds prompts, executes via LLM, collects results
- **Output**: Structured report with pass/fail status, summary, and detailed results

**Scheduler:**
- **Input**: Task name or event to emit
- **Process**: Executes configured commands on schedule or event trigger
- **Output**: LLM-friendly structured output showing task status and results

### Interface Contract

```bash
# E2E Test Commands
ace-e2e-test ace-lint                    # Run all E2E tests for ace-lint
ace-e2e-test ace-lint MT-LINT-001        # Run specific test
ace-e2e-test-suite                       # Run all E2E tests (sequential)
ace-e2e-test-suite --parallel 4          # Parallel execution

# Scheduler Commands
ace-scheduler list                 # List all tasks and events
ace-scheduler run daily-tests      # Run a task immediately
ace-scheduler emit pr-merged       # Emit event, trigger actions
ace-scheduler status               # Show next scheduled runs
ace-scheduler cron install         # Install to system cron
ace-scheduler cron uninstall       # Remove from system cron
```

### Success Criteria

- [ ] `ace-e2e-test` can run single E2E tests via LLM and produce reports
- [ ] `ace-e2e-test-suite` can run all E2E tests with parallel execution
- [ ] `ace-scheduler` can list, run, and manage scheduled tasks
- [ ] Event-driven triggers work (e.g., `ace-scheduler emit pr-merged`)
- [ ] Release workflows integrate with scheduler events

## Architecture Decisions

| Question | Decision |
|----------|----------|
| Where should CLIs live? | New gems: `ace-test-e2e-runner-cli`, `ace-scheduler` |
| How to call LLM? | Library via `Ace::LLM::QueryInterface.query()` |
| Parallel execution? | Process-based (subprocess per test, like ace-test-suite) |
| Scheduler state? | File-based in `.ace/scheduler/state/` |
