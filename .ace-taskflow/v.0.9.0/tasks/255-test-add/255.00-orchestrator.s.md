# E2E Test Runner CLI

## Objective

Add CLI commands for running E2E tests via LLM providers. This extends existing packages (ace-llm, ace-test-e2e-runner) rather than creating new gems.

Key capability: Pass CLI args to LLM providers for flags like `--dangerously-skip-permissions`:

```bash
ace-e2e-test ace-coworker MT-COWORKER-001 --cli-args dangerously-skip-permissions
```

## Behavioral Specification

### User Experience

**Single Test Execution:**
- **Input**: Package name and test ID
- **Process**: Build prompt, execute via LLM with CLI args, collect results
- **Output**: Structured report with pass/fail status

**Suite Execution:**
- **Input**: Optional package filter
- **Process**: Discover all tests, execute with parallelism
- **Output**: Aggregated report across all packages

### Interface Contract

```bash
# Run specific test
ace-e2e-test ace-lint MT-LINT-001
ace-e2e-test ace-coworker MT-COWORKER-001 --cli-args dangerously-skip-permissions

# Run all tests in package
ace-e2e-test ace-lint

# Run full suite
ace-e2e-test-suite
ace-e2e-test-suite --parallel 4
```

### Success Criteria

- [ ] `--cli-args` passes flags to CLI-based LLM providers
- [ ] `ace-e2e-test` can run single tests via LLM
- [ ] `ace-e2e-test-suite` can run all tests with parallel execution
- [ ] Reports written to `.cache/ace-test-e2e/`
