# Release Pipeline Integration

## Objective

Integrate E2E tests and scheduler into the release workflow, replacing the CI-based performance gate (253.06) with local automation.

## Behavioral Specification

### User Experience

**During Release Draft:**
- E2E tests run automatically via `ace-scheduler emit release-started`
- Results verified before release tasks created

**During Release Publish:**
- Final verification runs
- E2E results checked
- Performance profiles captured

### Interface Contract

```bash
# When running /ace:draft-release
# Step added: ace-scheduler emit release-started

# When running /ace:publish-release
# Step added: Verify E2E results
```

### Success Criteria

- [ ] Release workflows emit scheduler events
- [ ] E2E tests run before release finalization
- [ ] Performance profiles auto-captured
- [ ] Documentation updated

  - **Schedule**: 9:00 AM daily

  ### release-started

  ```bash
  ace-scheduler emit release-started
  ```

- [ ] Configure default project scheduler config

  # .ace/scheduler/config.yml
  version: 1

      cron: "0 9 * * *"
      command: ace-test-suite --profile 10
      enabled: true
      timeout: 3600

    weekly-e2e:
      description: "Full E2E test run"

      timeout: 1800

      description: "Post-merge verification"

      description: "Pre-release gates"
      triggers:
        - command: ace-test-suite
          timeout: 1800

          timeout: 7200
  ```
