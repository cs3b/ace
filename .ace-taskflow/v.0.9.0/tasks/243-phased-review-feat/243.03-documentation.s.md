# Phased Review Presets

## Overview

Phased review applies Kent Beck's iterative principle to code review:
1. **Make it work** (correctness) - Fix bugs first
2. **Make it fast** (quality) - Then optimize
3. **Make it beautiful** (polish) - Refine last

## Available Presets

### correctness
Focus: Bugs, logic errors, missing functionality, security
Blocking: Medium+ severity findings
Usage: `ace-review --preset correctness`

### quality
Focus: Performance, architecture, standards compliance
Blocking: High+ severity findings
Usage: `ace-review --preset quality`

### polish
Focus: Simplification, naming, readability
Blocking: None (suggestions only)
Usage: `ace-review --preset polish`

## Standalone Usage

Run individual phases independently:
- ace-review --preset correctness
- ace-review --preset quality
- ace-review --preset polish

## Phase-Specific Output

Each preset guides the LLM to focus on phase-appropriate concerns:
- Correctness phase ignores style issues
- Quality phase assumes correctness, focuses on structure
- Polish phase provides non-blocking suggestions only
```

### ace-coworker Guide (`ace-coworker/handbook/guides/phased-review.g.md`)

```markdown
# Phased Review Workflow

## Overview

The phased-review coworker preset orchestrates a complete review cycle:
1. Run correctness review (up to 3 cycles)
2. Auto-apply verified fixes
3. Run quality review (up to 2 cycles)
4. Auto-apply verified fixes
5. Run polish review (1 cycle, non-blocking)

## Configuration

```yaml
# In job.yaml
preset: phased-review
```

## Cycle Control

Each phase has configurable cycle limits:
- max_cycles: Maximum review/fix iterations
- min_improvement: Stop if < X% fewer findings (diminishing returns)

## Customization

Override default phase configuration:
```yaml
preset: phased-review
phase_overrides:
  correctness:
    max_cycles: 5
  polish:
    enabled: false
```

## Integration with Verification

Phased review uses Task 242's verification flow:
1. Review generates findings
2. Findings are verified via Chain of Verification
3. Only verified findings are auto-applied
4. Uncertain findings flagged for human review
```
