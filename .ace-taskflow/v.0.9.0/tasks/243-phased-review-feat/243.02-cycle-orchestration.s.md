# 243.02: Cycle Orchestration

## Objective

Create PhaseCycleManager for ace-coworker that orchestrates multi-phase review workflows with cycle control logic (max_cycles, min_improvement) and integration with Task 242's verification flow.

## Behavioral Specification

### User Experience

- **Input**: Coworker workflow with phased-review preset
- **Process**: Execute phases sequentially, run cycles within each phase until no blocking issues
- **Output**: Complete review with findings organized by phase, applied verified fixes

### Phase Cycle Flow

```
for each phase:
  cycle_count = 0
  previous_findings = infinity

  while cycle_count < max_cycles:
    run ace-review with phase preset
    current_findings = count blocking findings

    if current_findings == 0:
      break  # Phase complete, no blocking issues

    if cycle_count > 0:
      improvement = (previous_findings - current_findings) / previous_findings
      if improvement < min_improvement:
        break  # Diminishing returns, move on

    apply verified fixes (Task 242)
    previous_findings = current_findings
    cycle_count++

  proceed to next phase
```

### Interface Contract

```ruby
manager = Ace::Coworker::Molecules::PhaseCycleManager.new(session:)

# Configure phases from preset
manager.load_phases(preset_config)

# Execute full phased review
result = manager.execute
# => { phases: [...], total_cycles: 5, findings: [...] }

# Phase result structure
result.phases[0]
# => { name: :correctness, cycles: 2, findings_resolved: 5, status: :complete }
```

## Implementation Notes

### Cycle Decision Logic

```ruby
def should_continue_cycle?(phase_config, cycle_count, current_findings, previous_findings)
  return false if cycle_count >= phase_config.max_cycles
  return false if current_findings == 0

  if cycle_count > 0 && previous_findings > 0
    improvement = (previous_findings - current_findings).to_f / previous_findings
    return false if improvement < phase_config.min_improvement
  end

  true
end
```

### Integration with Task 242

After each review cycle:
1. Extract feedback items (Task 227)
2. Verify findings (Task 242)
3. Apply `verified` items automatically
4. Re-run review to check remaining issues

### State Tracking

Track per-phase metrics for reporting:
- `cycles_run`: How many review cycles executed
- `findings_initial`: Findings from first cycle
- `findings_resolved`: How many findings were fixed
- `findings_remaining`: Blocking findings still open
- `status`: `:complete` | `:max_cycles_reached` | `:diminishing_returns`
