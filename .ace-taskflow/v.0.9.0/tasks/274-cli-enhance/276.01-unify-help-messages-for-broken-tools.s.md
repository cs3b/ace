# Fix ace-test-suite, ace-test-e2e-sh, ace-framework, and ace-models Help Output

## Behavioral Specification

### User Experience

- **Input**: User runs `bin/ace-test-suite --help`, `bin/ace-test-e2e-sh --help`, `bin/ace-framework`, `bin/ace-models cache --help`
- **Process**: No errors, no FATAL messages, exit 0
- **Output**: Each tool produces help output visually matching the dry-cli tools from tasks 274/275

### Expected Behavior

**ace-test-suite `--help`** (currently OptionParser style):
```
ace-test-suite - Run all tests across the monorepo

Usage: ace-test-suite [OPTIONS]

Options:
  -c, --config FILE       Configuration file
  -p, --parallel N        Maximum parallel processes
  -q, --quiet             Suppress non-essential output
  -h, --help              Show this help
      --version           Show version

Examples:
  $ ace-test-suite                    Run all package tests
  $ ace-test-suite --parallel 4       Run with 4 parallel workers

Run 'ace-test-suite --help' for full details.
```

**ace-test-e2e-sh `--help`** (currently FATAL):
```
ace-test-e2e-sh - Run a single E2E test scenario in a sandbox

Usage: ace-test-e2e-sh <test-dir> [command...]

Arguments:
  test-dir    Path to E2E test scenario directory
  command     Command to run inside the sandbox

Examples:
  $ ace-test-e2e-sh ace-lint/test/e2e/TS-LINT-001
```

**ace-framework** — loads without LoadError. Either:
- Remove `bin/ace-framework` symlink if the gem no longer exists, or
- Fix the exe path to point to the correct location

**ace-models groups** (`ace-models cache --help`, etc.):
- Exits 0 (not 1)
- Output goes to stdout (not stderr)
- Output appears once (not twice)

### Interface Contract

| Tool | File | Fix |
|------|------|-----|
| ace-test-suite | `ace-test-runner/exe/ace-test-suite` | Replace `puts opts` with ALL-CAPS formatted help block |
| ace-test-e2e-sh | shell script (find path) | Add `if [ "$1" = "--help" ] || [ "$1" = "-h" ]` guard at top |
| ace-framework | `bin/ace-framework` | Remove symlink or fix target path |
| ace-models groups | `ace-support-models/lib/ace/support/models/cli.rb` | Fix subcommand group help to exit 0 and write to stdout |

**Error handling:**
- `ace-test-e2e-sh --help` must exit 0
- `ace-test-e2e-sh` with no args should still print usage and exit 1
