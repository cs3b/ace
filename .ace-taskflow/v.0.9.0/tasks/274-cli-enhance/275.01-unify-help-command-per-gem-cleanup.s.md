# Apply Standardized Help Format Across All 20 dry-cli Gems

## Behavioral Specification

### User Experience

- **Input**: Developer runs `bin/ace-docs analyze --help` or `bin/ace-taskflow --help`
- **Process**: No errors, no doubled names, grouped commands where useful
- **Output**: Clean, consistent help across every tool

### Expected Behavior

**1. Duplicate examples fixed** (4 gems)

Before (`ace-docs analyze --help`):
```
EXAMPLES
  $ ace-docs analyze ace-docs analyze README.md
```
After:
```
EXAMPLES
  $ ace-docs analyze README.md    Analyze a documentation file
```

**2. Canonical option descriptions** (all 20 gems)

All `--quiet`, `--verbose`, `--debug` option strings replaced with:
- `"Suppress non-essential output"` (quiet)
- `"Show verbose output"` (verbose)
- `"Show debug output"` (debug)

**3. Grouped commands** (ace-taskflow `--help`)

```
COMMANDS
  Task Management
    task REF                        Show or manage a task (default)
    task create                     Create a new task
    task start REF                  Start working on a task
    task done REF                   Mark task as done
    tasks                           List tasks in current release

  Idea Management
    idea REF                        Show or manage an idea
    ideas                           List ideas

  Release & Retro
    release / releases              Show or list releases
    retro / retros                  Show or list retros

  Utilities
    status                          Show project status
    doctor                          Check configuration health
    config                          Show configuration
    migrate                         Run migrations
```

**4. Fixed examples in ace-taskflow task add-dependency**

Before: `ace-taskflow task add-dependency 034 --depends-on 031`
After: `ace-taskflow task add-dependency 034 031`

**5. Deprecated command hidden**

`ace-review feedback skip` no longer appears in `ace-review feedback --help`.
