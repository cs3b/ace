# Add context subcommand to ace-taskflow

## Objective

Add `ace-taskflow context` subcommand that combines repo context with task-aware information.

## Behavioral Specification

### Interface Contract

```bash
# Basic usage
ace-taskflow context              # Full task-aware context as markdown

# Use in ace-context preset
# commands:
#   - ace-taskflow context
```

### Output Format (Markdown)

```markdown
# Taskflow Context

## Repository
**Branch:** 140-add-git-context
**PR:** #75 - Add Git Context to ace-context (open → main)

## Current Task
**ID:** v.0.9.0+task.140
**Title:** Enhance ace-context with Dynamic Git Branch and PR Information
**Status:** in-progress
**Path:** .ace-taskflow/v.0.9.0/tasks/140-feat-context-ace/140-context-dynamic-git-branch.s.md

## Release: v.0.9.0
**Progress:** 7% (1/15)
**Status:** active

## Recent Tasks
- v.0.9.0+task.154 🟢 Implement task reorganization workflow
- v.0.9.0+task.155 🟢 Add fix-bug and analyze-bug workflows

## Next Tasks
- v.0.9.0+task.139 🟡 Secure Git History
```

### Error Handling

- Not in git repo: Show only taskflow info (release, tasks)
- No task for branch pattern: Omit "Current Task" section
- ace-context-repo not available: Show warning, continue with taskflow-only info

### Implementation

```ruby
class ContextCommand < Thor
  desc "context", "Show task-aware context (branch, PR, task, release)"

  def context
    # 1. Get repo context from ace-context-repo (optional)
    repo_ctx = fetch_repo_context

    # 2. Resolve task from branch pattern
    task = resolve_task(repo_ctx&.dig(:task_pattern))

    # 3. Get release and task summaries (reuse existing code)
    release = current_release
    recent = recent_tasks(limit: 3)
    next_tasks = next_tasks(limit: 3)

    # 4. Format and output
    puts format_context(repo_ctx, task, release, recent, next_tasks)
  end
end
```

## Success Criteria

- [ ] `ace-taskflow context` outputs combined repo + task context
- [ ] Resolves task from branch pattern using existing task finder
- [ ] Includes release progress and recent/next tasks
- [ ] Works in ace-context preset `commands:` section
- [ ] Graceful degradation without ace-context-repo

- ❌ PR fetching (delegated to ace-context-repo)
