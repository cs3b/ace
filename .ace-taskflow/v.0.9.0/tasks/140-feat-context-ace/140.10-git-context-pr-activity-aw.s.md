# Enhance ace-git context with PR activity awareness

## Problem

The merge bug in task 140 revealed a gap: `ace-git context` doesn't show enough about what's happening in the repository. Developers/agents can't see recent PR activity, leading to incorrect assumptions (e.g., thinking PRs aren't merged when they are).

## Objective

Enhance `ace-git context` to show PR activity around the current branch, providing awareness of:
- What was recently merged (context of completed work)
- What else is open (context of parallel work)

## Behavioral Specification

### User Experience

- **Input**: `ace-git context` (existing command)
- **Process**: Fetches PR data via `gh` CLI, formats concisely
- **Output**: Enhanced context with PR activity section

### Expected Behavior

When running `ace-git context`, in addition to current output, show:
1. **Recent Merged PRs** (last 3): One line per PR with number, title, merge date
2. **Open PRs** (excluding current branch): One line per PR with number, title, author

### Interface Contract

```bash
# Current output (unchanged)
ace-git context
# Branch: 140-enhance-ace-context...
# Task Pattern: 140
# PR: #85 (if current branch has PR)

# NEW: PR Activity section added
## PR Activity
### Recently Merged
- #84: Update ace-taskflow to use ace-git (merged 2h ago)
- #82: Create ace-git package (merged 1d ago)
- #80: Fix lint issues (merged 2d ago)

### Open PRs
- #83: Add documentation updates (@username)
- #81: Refactor search module (@other)
```

**Flags:**
- `--no-pr`: Skips ALL PR-related output (current + activity)
- `--json`: Includes PR activity in JSON output

**Error Handling:**
- gh CLI not installed: Skip PR activity section gracefully
- No recent merged PRs: Show "No recently merged PRs"
- No open PRs: Show "No other open PRs"

## Acceptance Criteria

- [ ] AC1: `ace-git context` shows last 3 merged PRs (one line each)
- [ ] AC2: `ace-git context` shows open PRs excluding current branch's PR
- [ ] AC3: Output is concise (one line per PR with key info)
- [ ] AC4: `--no-pr` flag skips all PR output including activity
- [ ] AC5: `--json` includes PR activity in structured output
- [ ] AC6: Graceful degradation when gh CLI unavailable
- [ ] AC7: Tests cover new functionality

## Out of Scope

- Changing existing context output format
- PR review status or checks
- Detailed PR metadata beyond one-line summary
