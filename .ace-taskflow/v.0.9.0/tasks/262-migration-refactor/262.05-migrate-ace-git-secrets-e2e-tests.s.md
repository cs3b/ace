# Migrate ace-git-secrets E2E Tests

## Scope

Convert 3 `.mt.md` files (18 TCs) in `ace-git-secrets/test/e2e/` to the new directory-based per-TC format. All test cases are independent.

## Migration Inventory

| Source File | Target Directory | TCs | Key Fixtures |
|-------------|-----------------|-----|-------------|
| `MT-SECRETS-001-full-scan-workflow.mt.md` | `TS-SECRETS-001-full-scan-workflow/` | 8 | README.md, config.env (GITHUB_TOKEN), test/mock_tokens.json, .ace/git-secrets/config.yml, real_config.env |
| `MT-SECRETS-002-rewrite-workflow.mt.md` | `TS-SECRETS-002-rewrite-workflow/` | 5 | README.md, config.txt (token), .cache/ace-git-secrets/broken-report.json |
| `MT-SECRETS-003-config-cascade.mt.md` | `TS-SECRETS-003-config-cascade/` | 5 | README.md only (config files created dynamically in TCs) |

**Total: 3 scenarios, 18 TCs, 9 unique fixtures**

## Technical Approach

### Setup Pattern

All scenarios:
```yaml
setup:
  - git-init
  - copy-fixtures
  - env:
      PROJECT_ROOT_PATH: .
```

### Special Considerations

- **Fixture tokens:** config.env and real_config.env contain test secret tokens — extract as real files
- **Nested directories:** .ace/git-secrets/ and .cache/ace-git-secrets/ structures preserved in fixtures
- **Config cascade testing (003):** Config files created dynamically within TCs, NOT in fixtures
