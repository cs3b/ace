# Migrate ace-support-timestamp E2E Tests

## Scope

Convert 4 `.mt.md` files (46 TCs) in `ace-support-timestamp/test/e2e/` to the new directory-based per-TC format. All test cases are independent. Highest TC count of all packages (46 TCs) but NO fixture files — pure CLI testing.

## Migration Inventory

| Source File | Target Directory | TCs | Fixtures |

| `MT-TIMESTAMP-001-core-encode-decode-roundtrip.mt.md` | `TS-TIMESTAMP-001-core-encode-decode-roundtrip/` | 11 | NONE |

**Total: 4 scenarios, 46 TCs, 0 fixtures**

## Technical Approach

### Minimal Setup (all 4 scenarios)

```yaml
setup: []
```

No `git-init`, no `copy-fixtures`, no `env` needed. Pure CLI tests that invoke `ace-timestamp encode` and `ace-timestamp decode` with various flags and inputs.

### Key Characteristics

- **No fixtures directory** needed for any scenario
- All test data is generated via bash commands (date, echo) or hardcoded strings in TC steps
- Config files (if any) are created dynamically within TCs
- Highest volume of TC extraction (46 individual files) but simplest per-file work
