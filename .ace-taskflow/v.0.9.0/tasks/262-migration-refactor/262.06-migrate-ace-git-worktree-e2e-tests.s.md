# Migrate ace-git-worktree E2E Tests

## Scope

Convert 2 `.mt.md` files (21 TCs) in `ace-git-worktree/test/e2e/` to the new directory-based per-TC format. All test cases are independent.

## Migration Inventory

| Source File | Target Directory | TCs | Key Fixtures | Special |
|-------------|-----------------|-----|-------------|---------|
| `MT-WORKTREE-001-basic-worktree-lifecycle.mt.md` | `TS-WORKTREE-001-basic-worktree-lifecycle/` | 10 | NONE (git-only setup) | Feature branches created via `run:` |
| `MT-WORKTREE-002-task-aware-worktree.mt.md` | `TS-WORKTREE-002-task-aware-worktree/` | 11 | .ace-taskflow/ task specs (999, 888, 777), .ace/taskflow/config.yml | Taskflow integration |

**Total: 2 scenarios, 21 TCs, 5 fixtures (all in TS-WORKTREE-002)**

## Technical Approach

### TS-WORKTREE-001 (git-only, no fixtures)

```yaml
setup:
  - git-init
  - run: |
      git checkout -b feature/test-worktree --quiet
      echo "Feature content" > feature.txt
      git add feature.txt && git commit -m "Add feature" --quiet
      git checkout main --quiet
      git checkout -b bugfix/test-fix --quiet
      echo "Bugfix content" > bugfix.txt
      git add bugfix.txt && git commit -m "Add bugfix" --quiet
      git checkout main --quiet
  - env:
      PROJECT_ROOT_PATH: .
```

No `fixtures/` directory needed — all test data is git structure.

### TS-WORKTREE-002 (taskflow fixtures)

```yaml
setup:
  - git-init
  - copy-fixtures
  - run: |
      mkdir -p .ace-wt
      git add .ace-taskflow/ .ace/ && git commit -m "Add taskflow" --quiet
  - env:
      PROJECT_ROOT_PATH: .
```

Fixtures include nested `.ace-taskflow/v.test/tasks/` structure with task spec files.
