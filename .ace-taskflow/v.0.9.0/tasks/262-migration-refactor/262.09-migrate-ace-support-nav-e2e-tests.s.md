# Migrate ace-support-nav E2E Tests

## Scope

Convert 1 `.mt.md` file (8 TCs) in `ace-support-nav/test/e2e/` to the new directory-based per-TC format. All test cases are independent.

## Migration Inventory

| Source File | Target Directory | TCs | Key Fixtures |
|-------------|-----------------|-----|-------------|
| `MT-NAV-001-extension-inference.mt.md` | `TS-NAV-001-extension-inference/` | 8 | .ace/nav/protocols/{guide,wfi}.yml, guide-sources/local.yml, wfi-sources/local.yml, handbook/guides/{3 files}, handbook/workflows/{2 files} |

**Total: 1 scenario, 8 TCs, 9 fixtures (deeply nested directory structure)**

## Technical Approach

### Setup Pattern

```yaml
setup:
  - git-init
  - copy-fixtures
  - env:
      PROJECT_ROOT_PATH: .
```

### Fixture Structure

```
fixtures/
├── .ace/nav/protocols/
│   ├── guide.yml
│   ├── wfi.yml
│   ├── guide-sources/local.yml
│   └── wfi-sources/local.yml
└── handbook/
    ├── guides/
    │   ├── markdown-style.g.md
    │   ├── coding-standards.guide.md
    │   └── general-info.md
    └── workflows/
        ├── deploy-process.wf.md
        └── review-process.workflow.md
```

Key: Multiple extension patterns (.g.md, .guide.md, .md, .wf.md, .workflow.md) are the core of what's being tested.
