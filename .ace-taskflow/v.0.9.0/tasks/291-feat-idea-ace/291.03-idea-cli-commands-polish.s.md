# ace-idea CLI Commands and Polish

## Behavioral Specification

### User Experience
- **Input**: CLI commands via terminal: `ace-idea create|show|list|move|update`
- **Process**: dry-cli registry dispatches to command classes, which delegate to IdeaManager
- **Output**: Formatted terminal output, idea files created/modified on disk

### Expected Behavior

Wire IdeaManager to 5 CLI commands following the same ACE patterns as ace-task CLI (290.03):

1. **create**: `ace-idea create [CONTENT] [--title T] [--tags T] [--move-to FOLDER] [--clipboard] [--llm-enhance] [--dry-run]`
2. **show**: `ace-idea show REF [--path | --content]`
3. **list**: `ace-idea list [--status S] [--tags T] [--in FOLDER] [--root PATH]`
4. **move**: `ace-idea move REF --to FOLDER`
5. **update**: `ace-idea update REF [--set K=V]... [--add K=V]... [--remove K=V]...`

### Interface Contract

```bash
# Create with content
ace-idea create "We should add dark mode" --title "Dark mode" --tags ux,design --move-to next

# Create from clipboard
ace-idea create --clipboard --llm-enhance --move-to next

# Create with dry-run
ace-idea create "raw thought" --dry-run

# Show
ace-idea show q7w
ace-idea show 8ppq7w --path           # Just file path
ace-idea show q7w --content            # Raw markdown

# List
ace-idea list                          # All ideas, flat
ace-idea list --in maybe               # Ideas in _maybe/
ace-idea list --tags ux --status pending

# Move
ace-idea move q7w --to archive
ace-idea move q7w --to maybe

# Update
ace-idea update q7w --set status=done
ace-idea update q7w --add tags=implemented --remove tags=pending-review

# Standard
ace-idea version
ace-idea help
```

**Exit Codes:**
- 0: Success
- 1: General error (via `Ace::Core::CLI::Error`)
- 130: SIGINT

### Success Criteria

- [ ] All 5 commands work end-to-end from terminal
- [ ] `--clipboard` captures and creates idea from system clipboard
- [ ] `--llm-enhance` sends content through LLM, falls back gracefully on failure
- [ ] `--move-to FOLDER` on create places idea immediately in target folder
- [ ] `--set/--add/--remove` work identically to ace-task update
- [ ] `--in FOLDER` auto-maps to `_FOLDER` prefix
- [ ] Error messages are actionable
- [ ] SIGINT handled with exit code 130
- [ ] `.ace-defaults/idea/config.yml` provides sensible defaults
- [ ] CHANGELOG.md and README.md written
- [ ] Integration tests cover CLI roundtrips for all commands
- [ ] `ace-test ace-idea` passes all tests
